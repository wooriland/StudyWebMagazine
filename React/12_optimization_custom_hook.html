최적화와 Custom Hook

React에서 state나 props가 변경되면 컴포넌트는 다시 렌더링됩니다.
이때 무거운 계산, 함수 재생성, 자식 컴포넌트 렌더링이 반복되면 성능 부담이 생길 수 있습니다.
이번 단원에서는 계산 결과를 기억하는 useMemo, 함수 자체를 기억하는 useCallback,
자식 컴포넌트 렌더링 결과를 기억하는 React.memo, 그리고 반복되는 Hook 로직을 재사용 함수로 분리하는 Custom Hook을 학습합니다.
핵심은 “무조건 최적화”가 아니라 불필요한 계산과 렌더링을 필요한 지점에서 줄이는 것입니다.

반복 로직을 Hook으로 재사용한다.

3. 최적화와 Custom Hook 한눈에 보기

최적화 도구들은 모두 “기억해 두고 재사용한다”는 공통점이 있습니다. 하지만 기억하는 대상은 서로 다릅니다.

도구 기억하는 것 대표 목적 주의점
useMemo 계산 결과값 무거운 계산, 필터링, 정렬 결과 재사용 가벼운 계산에 무조건 쓰면 코드만 복잡해질 수 있습니다.
useCallback 함수 자체 자식에게 전달하는 함수 참조 유지 함수 안에서 사용하는 state나 props를 의존성 배열에 넣어야 합니다.
React.memo 컴포넌트 렌더링 결과 props가 같을 때 자식 컴포넌트 렌더링 생략 props 비교 비용이 있으므로 무조건 사용하지 않습니다.
Custom Hook 반복되는 Hook 로직 입력값, 토글, fetch, form 로직 재사용 이름은 use로 시작하고 Hook 규칙을 지켜야 합니다.

4. 핵심 개념 설명

4-1. 최적화는 불필요한 일을 줄이는 것이다

최적화는 React를 빠르게 만드는 마법 문법이 아닙니다. state나 props 변경으로 컴포넌트가 다시 호출될 때, 정말 다시 할 필요가 없는 계산이나 렌더링을 줄이는 과정입니다.

리렌더링 자체는 정상

state 변경
↓
컴포넌트 함수 다시 호출
↓
JSX 다시 계산
↓
Virtual DOM 비교
↓
변경된 부분 실제 DOM 반영

React는 state가 바뀌면 컴포넌트를 다시 계산합니다. 이것은 정상적인 동작입니다.

최적화가 필요한 상황

무거운 계산 반복
큰 배열 필터링 반복
함수 props 매번 새로 생성
memo 자식 컴포넌트의 불필요한 렌더링

계산 비용이나 렌더링 비용이 실제로 부담될 때 최적화를 고려합니다.

4-2. Memoization은 계산 결과 캐시다

Memoization은 한 번 계산한 결과나 함수를 저장해두고, 조건이 바뀌지 않았다면 다시 계산하지 않고 이전 값을 재사용하는 기법입니다.

Memoization 흐름

첫 번째 계산
→ 직접 계산
→ 결과를 캐시에 저장

다음 렌더링
→ 의존성 배열 확인
→ 조건이 같으면 캐시 결과 재사용
→ 조건이 바뀌면 다시 계산

4-3. useMemo는 계산 결과값을 기억한다

useMemo는 콜백 함수의 반환값을 메모이제이션합니다. 의존성 배열 값이 바뀔 때만 다시 계산하고, 바뀌지 않으면 이전 계산 결과를 재사용합니다. 

const filteredProducts = useMemo(() => {
    return products.filter((product) =>
        product.name.includes(keyword)
    );
}, [products, keyword]);

products나 keyword가 바뀌면 다시 계산하고, 두 값이 그대로라면 이전 filteredProducts를 재사용합니다.

4-4. useCallback은 함수 자체를 기억한다

useCallback은 함수 자체를 메모이제이션합니다. 부모 컴포넌트가 리렌더링될 때 자식 컴포넌트에 전달하는 함수가 매번 새로 생성되는 것을 줄일 수 있습니다. 

const handleSelect = useCallback((id) => {
    setSelectedId(id);
}, []);

의존성 배열이 바뀌지 않으면 이전 함수 참조를 재사용합니다. 특히 React.memo로 감싼 자식 컴포넌트에 함수 props를 전달할 때 자주 함께 사용됩니다.

4-5. useMemo와 useCallback의 차이

구분 useMemo useCallback
기억하는 것 계산 결과값 함수 자체
반환값 값 함수
대표 목적 무거운 계산 반복 방지 함수 참조 재생성 방지
대표 예 필터링 결과, 정렬 결과, 합계 계산 자식에게 전달하는 이벤트 핸들러
기본 형식 useMemo(() => value, [deps]) useCallback(() => {}, [deps])

4-6. React.memo는 컴포넌트 렌더링 결과를 기억한다

React.memo()는 Hook은 아니지만 React 최적화에서 자주 함께 사용됩니다. 부모가 리렌더링되어도 자식에게 전달된 props가 이전과 같다면 자식 컴포넌트의 렌더링을 생략할 수 있습니다. 

function ProductItem({ product, onSelect }) {
    return (
        <li>
            <span>{product.name}</span>
            <button onClick={() => onSelect(product.id)}>
                선택
            </button>
        </li>
    );
}

export default React.memo(ProductItem);

React.memo 동작 흐름

부모 컴포넌트 리렌더링
↓
React.memo가 자식 props 비교
↓
props가 이전과 같음
↓
자식 컴포넌트 렌더링 생략
↓
이전 렌더링 결과 재사용

4-7. Custom Hook은 반복되는 Hook 로직을 재사용한다

Custom Hook은 이름이 use로 시작하는 재사용 함수입니다. 내부에서 useState, useEffect 같은 React Hook을 호출할 수 있고, 여러 컴포넌트에서 반복되는 로직을 분리해 사용할 수 있습니다. 

import { useState } from "react";

export default function useInput(initialValue = "") {
    const [value, setValue] = useState(initialValue);

    function handleChange(event) {
        setValue(event.target.value);
    }

    function reset() {
        setValue(initialValue);
    }

    return {
        value,
        onChange: handleChange,
        reset
    };
}

Custom Hook은 JSX를 반환하는 컴포넌트가 아니라, 상태값과 이벤트 함수, 로직 결과를 반환하는 재사용 함수입니다.

4-8. 최적화 도구는 무조건 쓰지 않는다

useMemo, useCallback, React.memo 자체도 비용이 있습니다. 의존성 배열 비교, props 비교, 메모리 사용, 코드 복잡도가 생깁니다.
따라서 최적화는 필요한 지점에 선택적으로 적용해야 합니다.

useMemo 고려 무거운 계산

큰 배열 필터링, 정렬, 그룹화, 합계 계산이 반복될 때 고려합니다.

useCallback 고려 함수 참조

memo 자식 컴포넌트에 함수 props를 전달할 때 고려합니다.

React.memo 고려 자식 렌더링

자식 컴포넌트가 무겁고 props가 자주 바뀌지 않을 때 고려합니다.

Custom Hook 고려 로직 재사용

입력값, 토글, fetch, form 로직이 여러 컴포넌트에서 반복될 때 고려합니다.

useCallback 없음

→ handleSelect 함수 A

두 번째 렌더링
→ handleSelect 함수 B

A !== B

함수 내용은 같아도 렌더링마다 새 함수 참조가 만들어질 수 있습니다.

→
useCallback 사용 
const handleSelect =
    useCallback((id) => {
        setSelectedId(id);
    }, []);

의존성이 바뀌지 않으면 이전 함수 참조를 재사용합니다.

5-3. React.memo는 자식 앞의 출입 검사원이다

1 부모 리렌더링

부모 state가 바뀌어 다시 렌더링됩니다.

React.memo가 자식 props를
이전 props와 비교합니다.

3 props 같음

자식 렌더링을 생략하고
이전 결과를 사용합니다.

4 props 다름

자식 컴포넌트를 다시 렌더링합니다.

5-4. Custom Hook은 반복 로직을 담은 공구함이다

반복되는 입력 로직

function calculateTotalPrice(items) {
    console.log("무거운 합계 계산 실행");

    return items.reduce((sum, item) => sum + item.price, 0);

export default function App() {
    const [selectedId, setSelectedId] = useState(null);
    const [count, setCount] = useState(0);

Mistakes

7. 자주 하는 실수

  • 실수 1. useMemo를 모든 계산에 사용함 price * count처럼 가벼운 계산은 그냥 계산하는 편이 더 읽기 쉽습니다. useMemo는 무거운 계산에 선택적으로 사용합니다.
  • 실수 2. useCallback을 모든 함수에 사용함 자식 컴포넌트에 props로 전달하지 않거나 최적화와 관련 없는 함수라면 굳이 useCallback이 필요 없을 수 있습니다.
  • 실수 3. React.memo를 모든 컴포넌트에 사용함 컴포넌트가 가볍거나 props가 매번 바뀐다면 props 비교 비용 때문에 효과가 작을 수 있습니다.
  • 실수 4. useMemo의 의존성 배열을 잘못 작성함 products와 keyword를 사용한다면 의존성 배열에 [products, keyword]를 넣어야 최신 결과를 보장할 수 있습니다.
  • 실수 5. useCallback의 의존성 배열을 비워서 오래된 값을 사용함 콜백 함수 안에서 사용하는 state나 props가 있다면 의존성 배열에 포함해야 합니다. 또는 함수형 업데이트를 사용해 의존성을 줄일 수 있습니다.
  • 실수 6. useMemo 안에서 side effect를 실행함 useMemo는 계산 결과를 반환하기 위한 Hook입니다. API 요청, localStorage 저장, DOM 조작은 useEffect에서 처리하는 것이 좋습니다.
  • 실수 7. React.memo를 썼는데 props로 매번 새 객체를 전달함 option={{ showPrice: true }}처럼 작성하면 렌더링마다 새 객체가 됩니다. 필요한 경우 객체를 useMemo로 고정하거나 primitive props로 전달합니다.
  • 실수 8. React.memo를 썼는데 함수 props가 매번 새로 생성됨 onSelect={(id) => setSelectedId(id)}는 렌더링마다 새 함수입니다. memo 자식에게 전달한다면 useCallback을 고려할 수 있습니다.
  • 실수 9. Custom Hook 이름을 use로 시작하지 않음 Custom Hook은 useInput, useToggle, useFetch처럼 use로 시작해야 Hook 규칙을 따르는 함수임을 명확히 알 수 있습니다.
  • 실수 10. Custom Hook을 조건문 안에서 호출함 Custom Hook도 내부에서 Hook을 호출하므로 Hook 규칙을 따릅니다. 조건문, 반복문, 중첩 함수 안에서 호출하면 안 됩니다.
  • 실수 11. Custom Hook이 JSX를 반환해야 한다고 생각함 Custom Hook은 컴포넌트가 아닙니다. JSX보다 상태값, 이벤트 함수, 로직 결과 등을 반환하는 것이 일반적입니다.
  • 실수 12. 성능 측정 없이 감으로 최적화함 최적화 도구도 비용이 있으므로 렌더링 로그나 Profiler로 문제가 되는 지점을 확인하고 적용하는 것이 좋습니다.
Interview

8. 면접 포인트

Q1. Memoization이란 무엇인가요?

Memoization은 동일한 입력에 대해 이전에 계산한 결과를 저장해 두었다가 재사용하는 기법입니다. React에서는 useMemo, useCallback, React.memo가 관련된 최적화 도구입니다.

Q2. useMemo는 무엇인가요?

useMemo는 계산 결과값을 캐싱하는 Hook입니다. 무거운 계산이나 큰 배열 필터링, 정렬 결과를 의존성 배열이 바뀔 때만 다시 계산하도록 할 때 사용합니다.

Q3. useCallback은 무엇인가요?

useCallback은 함수 자체를 캐싱하는 Hook입니다. 주로 React.memo로 감싼 자식 컴포넌트에 함수 props를 전달할 때 함수 참조가 매번 바뀌는 것을 줄이기 위해 사용합니다.

Q4. useMemo와 useCallback의 차이는 무엇인가요?

useMemo는 함수 실행 결과를 메모이제이션하고, useCallback은 함수 자체를 메모이제이션합니다. 쉽게 말해 useMemo는 값 캐시, useCallback은 함수 캐시입니다.

Q5. React.memo는 무엇인가요?

React.memo는 컴포넌트를 메모이제이션하는 고차 컴포넌트입니다. props가 변경되지 않았다면 자식 컴포넌트의 불필요한 리렌더링을 줄일 수 있습니다.

Q6. React.memo와 useMemo의 차이는 무엇인가요?

React.memo는 컴포넌트 자체를 감싸서 props가 같으면 렌더링을 생략하게 하는 기능이고, useMemo는 컴포넌트 내부의 계산 결과를 캐싱하는 Hook입니다.

Q7. 최적화 Hook을 무조건 쓰면 안 되는 이유는 무엇인가요?

최적화 도구도 비용이 있습니다. 의존성 비교, props 비교, 메모리 사용, 코드 복잡도가 추가됩니다. 따라서 계산이 무겁거나 렌더링 비용이 큰 경우에 선택적으로 사용하는 것이 좋습니다.

Q8. useCallback을 사용했는데도 자식 컴포넌트가 계속 렌더링될 수 있는 이유는 무엇인가요?

useCallback은 함수 참조만 안정화합니다. 자식에게 전달되는 다른 props가 바뀌거나 객체 리터럴이 매번 새로 생성되면 React.memo가 있어도 자식은 다시 렌더링될 수 있습니다.

Q9. Custom Hook이란 무엇인가요?

Custom Hook은 컴포넌트 간에 반복되는 Hook 로직을 재사용하기 위해 만든 함수입니다.
이름은 use로 시작하고, 내부에서 useState나 useEffect 같은 Hook을 사용할 수 있습니다.

Q10. Custom Hook은 컴포넌트인가요?

Custom Hook은 컴포넌트가 아닙니다. 화면을 렌더링하기보다는 상태 관리나 effect 같은 로직을 재사용하기 위한 함수입니다.

Q11. Custom Hook의 이름이 use로 시작해야 하는 이유는 무엇인가요?

Custom Hook은 이름이 use로 시작해야 합니다. 그래야 React Hook 규칙을 따르는 함수임을 알 수 있고, lint 도구도 Hook 규칙을 검사할 수 있습니다.

Q12. useMemo 안에서 API 요청을 하면 안 되는 이유는 무엇인가요?

useMemo는 순수 계산 결과를 캐싱하기 위한 용도입니다. API 요청 같은 side effect는 렌더링 이후 작업이므로 useEffect에서 처리하는 것이 맞습니다.

Q13. React.memo와 useCallback은 왜 함께 사용되는 경우가 많나요?

React.memo는 props 비교를 통해 리렌더링을 줄입니다.
함수 props는 부모가 리렌더링될 때 새로 생성되기 쉬우므로 useCallback으로 함수 참조를 고정하면 React.memo와 함께 효과를 낼 수 있습니다.

Summary

9. 이번 단원 정리

반드시 기억할 구분

Memoization
→ 이전 계산 결과나 함수를 기억해 재사용하는 최적화 기법

useMemo
→ 계산 결과값 캐시
→ const value = useMemo(() => 계산결과, [deps])

useCallback
→ 함수 자체 캐시
→ const callback = useCallback(() => {}, [deps])

React.memo
→ 컴포넌트 렌더링 결과 캐시
→ props가 같으면 이전 렌더링 결과 재사용

Custom Hook
→ use로 시작하는 재사용 함수
→ 내부에서 React Hook 사용 가능
→ 반복되는 Hook 로직 분리

무거운 계산
→ useMemo 고려

memo 컴포넌트에 함수 전달
→ useCallback 고려

자식 props가 자주 안 바뀜
→ React.memo 고려

반복되는 입력/토글/fetch 로직
→ Custom Hook 고려

주의
→ 무조건 사용하지 않는다.
→ 의존성 배열을 정확히 작성한다.
→ props가 매번 바뀌면 React.memo 효과가 줄어든다.
→ Custom Hook도 Hook 규칙을 지킨다.

핵심 체크리스트

  • Memoization은 이전 결과를 기억하고 같은 조건에서 재사용하는 최적화 기법입니다.
  • useMemo는 계산 결과값을 캐시합니다.
  • useCallback은 함수 자체를 캐시합니다.
  • React.memo는 컴포넌트 렌더링 결과를 캐시합니다.
  • useMemo와 useCallback은 의존성 배열을 정확히 작성해야 합니다.
  • 최적화 Hook은 무조건 쓰는 것이 아니라 필요한 경우에만 사용합니다.
  • 가벼운 계산에는 useMemo가 필요 없을 수 있습니다.
  • 자식에게 전달하지 않는 함수에는 useCallback이 필요 없을 수 있습니다.
  • props가 매번 바뀌는 컴포넌트에는 React.memo 효과가 작을 수 있습니다.
  • Custom Hook은 use로 시작하는 재사용 함수입니다.
  • Custom Hook은 JSX를 반환하는 컴포넌트가 아니라 반복되는 Hook 로직을 반환하는 함수입니다.
  • Custom Hook도 Hook 규칙을 지켜야 합니다.

학습 메모

이번 단원은 React 앱이 커졌을 때 성능과 유지보수성을 함께 생각하는 단계입니다.
useMemo, useCallback, React.memo는 불필요한 일을 줄이는 도구이고, Custom Hook은 반복되는 Hook 로직을 재사용 가능한 공구함으로 분리하는 방법입니다.

    const [value, setValue] = useState(initialValue);
    return { value, setValue };
