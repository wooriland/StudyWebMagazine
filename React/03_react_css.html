Overview

1. 이번 단원은 무엇을 배우는가?

React 프로젝트도 최종적으로는 브라우저에서 HTML, CSS, JavaScript로 실행됩니다. 따라서 CSS 문법 자체가 사라지는 것은 아닙니다.
다만 React에서는 화면을 JSX와 컴포넌트 단위로 작성하기 때문에 CSS를 적용하는 방식과 범위를 정확히 구분해야 합니다.
일반 HTML에서는 class 속성을 사용하지만, React JSX에서는 className을 사용합니다.
또한 inline style은 문자열이 아니라 JavaScript 객체로 작성해야 하며, 컴포넌트 단위 스타일이 필요할 때는 CSS Module을 사용할 수 있습니다.

전역 CSS index.css, App.css, src/assets CSS처럼
여러 컴포넌트에 영향을 줄 수 있는 스타일입니다.
inline style JSX 태그에 style={{ ... }} 형태로 직접 넣는
JavaScript 객체 기반 스타일입니다.
CSS Module Component.module.css 파일을 사용해
클래스 이름 충돌을 줄이는
컴포넌트 중심 스타일 방식입니다.

이번 단원의 한 줄 정리

4. 핵심 개념 설명

4-1. React에서도 CSS는 필요하다

React는 화면을 컴포넌트로 만드는 JavaScript 라이브러리입니다. 하지만 React를 사용한다고 해서 CSS가 사라지는 것은 아닙니다.
브라우저에 최종적으로 보이는 화면은 여전히 HTML, CSS, JavaScript로 구성됩니다.

HTML 방식

<div class="card">카드 내용</div>

일반 HTML에서는 class 속성으로 CSS 클래스를 지정합니다.

React JSX 방식

<div className="card">카드 내용</div>

React JSX에서는 class가 아니라 className을 사용합니다.

4-2. public 폴더 CSS

public 폴더에 CSS 파일을 넣으면 index.html에서 <link> 태그로 직접 연결합니다. 이 방식은 Vite가 CSS를 번들링하지 않고 그대로 제공합니다. 

public/
└─ styles.css
<!-- index.html -->
<link rel="stylesheet" href="/styles.css" />

경로 주의

  • public/styles.css라고 연결하지 않습니다.
  • /styles.css처럼 루트 경로로 접근합니다.
  • 이 CSS는 전역으로 적용되며 Vite 번들링 대상이 아닙니다.

4-3. src/assets CSS

src/assets 폴더 안의 CSS는 JavaScript 또는 JSX 파일에서 import해서 사용합니다. 이 방식은 Vite가 번들링 과정에서 CSS를 함께 처리하고 최적화합니다. 

src/
└─ assets/
   └─ styles.css
import "./assets/styles.css";

export default function App() {
    return (
        <section className="asset-card">
            <h2>src/assets CSS 사용</h2>
            <p>이 CSS는 App.jsx에서 import했습니다.</p>
        </section>
    );
}

중요한 오해 방지

src/assets/styles.css를 App.jsx에서 import했다고 해서 그 CSS가 자동으로 App 컴포넌트에만 갇히는 것은 아닙니다.
일반 CSS 파일은 import해도 전역 스타일처럼 동작할 수 있습니다.

4-4. index.css와 App.css

Vite React 기본 프로젝트에는 보통 src/index.css와 src/App.css가 있습니다. index.css는 앱 전체의 기본 스타일에 적합하고, App.css는 App 화면 중심의 스타일에 적합합니다.

index.css 앱 전체 기본 스타일

main.jsx에서 import하며, body, 기본 폰트, 배경색, reset 스타일에 적합합니다.

App.css App 화면 중심 스타일

App.jsx에서 import하며, App 화면의 큰 레이아웃이나 hero 영역 스타일에 적합합니다.

일반 CSS 전역 충돌 가능

파일명이 App.css라도 자동으로 App 컴포넌트에만 제한되지는 않습니다.

CSS Module 컴포넌트 중심

특정 컴포넌트 전용 스타일이 필요하다면 Component.module.css를 사용합니다.

4-5. CSS 적용 순서와 우선순위

React라고 해서 CSS 우선순위 규칙이 사라지는 것은 아닙니다. 기본적으로 CSS 선택자 우선순위, 로드 순서, inline style 여부가 영향을 줍니다.
교안에서는 public CSS가 가장 낮고, index.css, App.css처럼 import 계층에 따라 적용될 수 있다고 정리합니다.

1 public CSS

index.html에서
먼저 연결되는 전역 스타일

2 index.css

main.jsx에서 import되는 앱 기본 스타일

3 App.css

App.jsx에서 import되는 App 중심 스타일

4 inline style

태그에 직접 지정하는 JavaScript 객체 스타일

4-6. inline style은 style={{ ... }}

React JSX에서 inline style은 문자열이 아니라 JavaScript 객체로 작성합니다. 바깥 중괄호는 JSX 안에서 JavaScript 표현식을 쓰기 위한 것이고, 안쪽 중괄호는 실제 스타일 객체입니다. 

const titleStyle = {
    color: "#ffffff",
    backgroundColor: "#164e63",
    fontWeight: "bold",
    borderRadius: "12px",
    padding: "16px"
};

export default function App() {
    return (
        <h1 style={titleStyle}>
            React에서 CSS 다루기
        </h1>
    );
}
CSS 문법 JSX inline style
background-color backgroundColor
font-weight fontWeight
border-radius borderRadius
margin-bottom marginBottom
text-align textAlign

4-7. CSS Module

CSS Module은 클래스 이름 충돌을 줄이기 위한 방식입니다. 파일명은 Component.module.css 형태로 만들고, 컴포넌트에서는 styles 객체로 import해서 사용합니다. 

/* StudyButton.module.css */
.button {
    padding: 12px 18px;
    background-color: #164e63;
    color: #ecfeff;
    border-radius: 999px;
}
import styles from "./StudyButton.module.css";

export default function StudyButton() {
    return (
        <button className={styles.button}>
            CSS Module 버튼
        </button>
    );
}

6. CSS 적용 방식 선택 기준

React에서는 하나의 CSS 방식만 사용하는 것이 아니라, 목적과 범위에 따라 여러 방식을 함께 사용합니다.

상황 추천 방식 이유
전체 배경, 기본 폰트, reset index.css 앱 전체 기본값에 적합합니다.
App 화면의 큰 레이아웃 App.css 초기 학습에서 단순하고 직관적입니다.
여러 컴포넌트에서 공유하는 CSS src/assets/*.css import해서 공용 리소스로 사용할 수 있습니다.
특정 컴포넌트 전용 스타일 CSS Module 클래스 이름 충돌을 줄일 수 있습니다.
조건에 따라 바뀌는 간단한 스타일 inline style JavaScript 변수로 제어하기 쉽습니다.
HTML에서 직접 연결해야 하는 정적 CSS public CSS 파일을 그대로 제공할 수 있습니다.

선택 기준 요약

전역 스타일이 필요하다
→ index.css, App.css, src/assets CSS

컴포넌트 전용 스타일이 필요하다
→ Component.module.css

태그 하나의 동적 스타일이 필요하다
→ style={{ ... }}

HTML에서 직접 연결해야 한다
→ public/styles.css

            : styles.badge;

    return (
        <span className={badgeClassName}>
            {children}
        </span>
    );
}

Mistakes

8. 자주 하는 실수

  • 실수 1. JSX에서 class를 사용함 React JSX에서는 class="study-card"가 아니라 className="study-card"를 사용합니다.
  • 실수 2. inline style을 문자열로 작성함 style="color: red;"가 아니라 style={{ color: "red" }}처럼 JavaScript 객체로 작성합니다.
  • 실수 3. inline style 속성명을 kebab-case로 작성함 background-color가 아니라 backgroundColor처럼 camelCase로 작성해야 합니다.
  • 실수 4. style={{}}의 중괄호 의미를 모름 바깥 중괄호는 JSX 표현식이고, 안쪽 중괄호는 JavaScript 객체입니다.
  • 실수 5. CSS Module을 일반 CSS처럼 사용함 CSS Module은 import "./Button.module.css"가 아니라 import styles from "./Button.module.css"로 가져오고, className={styles.button}으로 사용합니다.
  • 실수 6. CSS Module 파일명을 잘못 작성함 StudyButton.css는 일반 CSS입니다. CSS Module로 사용하려면 StudyButton.module.css처럼 작성해야 합니다.
  • 실수 7. 일반 CSS import를 컴포넌트 전용이라고 오해함 일반 CSS는 컴포넌트에서 import해도 전역 스타일처럼 적용될 수 있습니다. 컴포넌트 전용 스타일이 필요하면 CSS Module을 사용합니다.
  • 실수 8. public CSS를 JSX에서 import하려고 함 public/styles.css는 보통 index.html에서 <link rel="stylesheet" href="/styles.css" />로 연결합니다.
  • 실수 9. public CSS 경로를 /public/styles.css로 씀 public/styles.css는 /styles.css로 접근합니다. /public/styles.css가 아닙니다.
  • 실수 10. 전역 클래스 이름을 너무 흔하게 지음 .box, .title, .button 같은 이름은 충돌하기 쉽습니다. 전역 CSS에서는 구체적인 클래스명을 쓰거나 CSS Module을 사용합니다.
  • 실수 11. 모든 스타일을 inline style로 처리하려고 함 inline style은 간단한 동적 스타일에는 좋지만, hover, media query, 복잡한 레이아웃에는 CSS 파일이나 CSS Module이 더 적합합니다.
  • 실수 12. CSS import 순서를 생각하지 않음 같은 선택자가 여러 CSS 파일에 있다면 선택자 우선순위와 로드 순서에 따라 결과가 달라질 수 있습니다.
Interview

9. 면접 포인트

Q1. React에서 CSS를 적용하는 방식에는 어떤 것들이 있나요?

일반 CSS 파일을 import하는 방식, public 폴더의 CSS를 index.html에서 <link>로 연결하는 방식, JSX 태그에 style={{ ... }}로 inline style을 적용하는 방식, 그리고 *.module.css를 사용하는 CSS Module 방식이 있습니다.

Q2. JSX에서 class 대신 className을 쓰는 이유는 무엇인가요?

JSX는 HTML 문자열이 아니라 JavaScript 안에서 사용되는 확장 문법입니다. React DOM은 HTML 속성을 JavaScript 스타일에 맞게 다루기 때문에 class 대신 className을 사용합니다.

Q3. public CSS와 src/assets CSS의 차이는 무엇인가요?

public CSS는 index.html에서 직접 연결하고, Vite가 번들링하지 않습니다. src/assets CSS는 JS/JSX 파일에서 import해서 사용하며, Vite가 번들링 과정에서 최적화합니다.

Q4. 일반 CSS 파일을 import하면 해당 컴포넌트에만 적용되나요?

아닙니다. 일반 CSS 파일은 컴포넌트에서 import하더라도 전역 스타일처럼 적용될 수 있습니다. 컴포넌트 범위로 스타일을 제한하고 싶다면 CSS Module을 사용하는 것이 좋습니다.

Q5. React inline style은 어떻게 작성하나요?

React inline style은 문자열이 아니라 JavaScript 객체로 작성합니다.
예를 들어 style={{ color: "red", backgroundColor: "black" }}처럼 작성하며, CSS 속성명은 camelCase를 사용합니다.

Q6. style={{}}에서 중괄호가 두 번 들어가는 이유는 무엇인가요?

바깥 중괄호는 JSX 안에서 JavaScript 표현식을 사용하기 위한 것이고, 안쪽 중괄호는 실제 스타일 객체입니다. 즉 style={ { color: "red" } } 구조입니다.

Q7. CSS Module은 무엇인가요?

CSS Module은 CSS 클래스 이름을 모듈화해서 클래스명 충돌을 줄이는 방식입니다.
Button.module.css 파일을 만들고, 컴포넌트에서 import styles from "./Button.module.css"로 가져온 뒤 className={styles.button}처럼 사용합니다.

Q8. CSS Module의 장점은 무엇인가요?

CSS Module의 가장 큰 장점은 클래스 이름 충돌을 줄일 수 있다는 점입니다.
여러 컴포넌트에서 같은 .button, .title 같은 이름을 사용해도 빌드 과정에서 고유한 클래스명으로 변환되어 충돌 가능성이 낮아집니다.

Q9. CSS Module과 일반 CSS import의 차이는 무엇인가요?

일반 CSS는 import "./Card.css"처럼 가져오고 className="card"처럼 문자열 클래스명을 사용합니다.
CSS Module은 import styles from "./Card.module.css"로 가져오고 className={styles.card}처럼 객체 속성으로 클래스명을 참조합니다.

Q10. 모든 스타일을 inline style로 작성해도 되나요?

가능은 하지만 권장되는 방식은 아닙니다.
inline style은 조건에 따라 동적으로 바뀌는 간단한 스타일에는 유용하지만, hover, media query, 복잡한 레이아웃, 재사용 스타일에는 CSS 파일이나 CSS Module이 더 적합합니다.

Q11. React에서 CSS 우선순위는 어떻게 생각해야 하나요?

React에서도 CSS의 기본 우선순위 규칙은 동일합니다. 선택자 우선순위, 로드 순서, inline style 여부가 영향을 줍니다.
일반 CSS는 전역 충돌 가능성이 있으므로 컴포넌트 단위 스타일에는 CSS Module을 사용할 수 있습니다.

Summary

10. 이번 단원 정리

반드시 기억할 문법

HTML class
→ React JSX className

HTML style="color:red"
→ React JSX style={{ color: "red" }}

CSS background-color
→ JSX inline style backgroundColor

일반 CSS
→ import "./App.css"

CSS Module
→ import styles from "./Button.module.css"
→ className={styles.button}

핵심 체크리스트

  • React에서도 CSS는 사용하지만 JSX 문법 때문에 className을 사용합니다.
  • public CSS는 index.html에서 <link>로 연결합니다.
  • src/assets CSS는 JS/JSX 파일에서 import해서 사용합니다.
  • 일반 CSS 파일은 import해도 전역적으로 적용될 수 있습니다.
  • inline style은 style={{ JavaScript 객체 }} 형태로 작성합니다.
  • inline style 속성명은 backgroundColor처럼 camelCase를 사용합니다.
  • CSS Module은 Component.module.css 파일명으로 작성합니다.
  • CSS Module은 styles 객체로 import하고 styles.클래스명으로 사용합니다.
  • 컴포넌트 전용 스타일이 필요하면 일반 CSS보다 CSS Module이 안전합니다.
  • 전역 CSS를 많이 사용할수록 클래스 이름 충돌에 주의해야 합니다.

학습 메모

이번 단원은 CSS 문법 자체를 새로 배우는 단원이 아니라, React 프로젝트 안에서 CSS의 위치, 적용 방식, 적용 범위를 구분하는 단원입니다.
특히 className, style={{}}, module.css는 이후 JSX와 컴포넌트 학습에서도 계속 등장합니다.

오늘의 핵심 구분

전역 CSS
→ index.css, App.css, src/assets CSS, public CSS

컴포넌트 전용 CSS
→ Component.module.css

HTML의 class
→ React JSX에서는 className

HTML의 style 문자열
→ React JSX에서는 style={{ JavaScript 객체 }}

가장 많이 틀리는 부분

React에서 class를 그대로 쓰거나, inline style을 style="color:red"처럼 문자열로 작성하는 실수가 많습니다.
JSX는 HTML처럼 보이지만 JavaScript 문법에 가까우므로 className과 style={{}}를 습관화해야 합니다.
