컴포넌트 설계

React 화면은 하나의 큰 HTML 덩어리처럼 작성하는 것이 아니라,
Header, Footer, Card, Button 같은 작은 UI 조각을 만들고 다시 조립하는 방식으로 구성합니다.
이번 단원에서는 컴포넌트의 의미, 함수형 컴포넌트와 클래스형 컴포넌트의 차이,
컴포넌트 이름을 대문자로 시작해야 하는 이유, props를 통한 재사용, 그리고 App.jsx에서 컴포넌트를 조립하는 흐름을 학습합니다.

1. 이번 단원은 무엇을 배우는가?

앞 단원까지는 JSX가 무엇인지, 브라우저가 화면을 어떻게 렌더링하는지, React가 Virtual DOM으로 변경점을 어떻게 계산하는지 공부했습니다.
이제는 그 화면을 실제로 어떤 단위로 나누어 작성할지 배워야 합니다.
React에서 화면은 보통 App.jsx 하나에 모든 내용을 쌓아 두지 않습니다.
화면의 상단은 Header, 반복되는 카드는 CourseCard, 하단 안내는 Footer처럼 역할이 명확한 컴포넌트로 나누고, App.jsx에서는 이 컴포넌트들을 조립합니다.

컴포넌트 UI의 한 부분을 담당하는
재사용 가능한 코드 조각입니다.
함수형 컴포넌트 JavaScript 함수로 작성하고
props를 입력받아 JSX를 반환합니다.
컴포넌트 설계 화면을 역할별 블록으로 나누고
다시 조립하는 사고방식입니다.

이번 단원의 한 줄 정리

컴포넌트는 React 화면을 구성하는 재사용 가능한 레고 블록이고, 함수형 컴포넌트는 props를 입력받아 JSX를 반환하는 JavaScript 함수입니다.

4. 핵심 개념 설명

4-1. 컴포넌트는 재사용 가능한 UI 조각이다

컴포넌트는 React에서 화면을 구성하는 기본 단위입니다. 교안에서는 컴포넌트를 “재사용이 가능한 HTML 코드 조각”이라고 설명합니다.
React 관점에서 조금 더 정확히 말하면, 컴포넌트는 UI의 한 부분을 담당하고 JSX를 반환하는 JavaScript 함수 또는 클래스입니다.

컴포넌트로 나누기 전

export default function App() {
    return (
        <main>
            <header>
                <h1>React 학습 페이지</h1>
            </header>

            <section>
                <h2>JSX 완전 이해</h2>
                <p>JSX는 JavaScript 확장 문법입니다.</p>
            </section>

            <footer>
                <p>React.js With Vite</p>
            </footer>
        </main>
    );
}

화면이 커질수록 App.jsx가 길어지고 수정 위치를 찾기 어려워집니다.

컴포넌트로 나눈 후

export default function App() {
    return (
        <main>
            <Header />
            <CourseCard />
            <CourseCard />
            <Footer />
        </main>
    );
}

App.jsx는 화면의 세부 내용을 직접 쓰기보다 컴포넌트를 조립합니다.

4-2. 함수형 컴포넌트와 클래스형 컴포넌트

교안에서는 컴포넌트를 함수형 Component와 클래스형 Component로 나눕니다.
초기 React에서는 함수형 컴포넌트를 Stateless, 클래스형 컴포넌트를 Stateful로 설명했지만,
React 16.8 이후 Hooks가 도입되면서 함수형 컴포넌트에서도 state와 생명주기 관리가 가능해졌습니다.

구분 함수형 컴포넌트 클래스형 컴포넌트
작성 방식 JavaScript 함수 JavaScript class
호출 관점 함수 호출 생성자 함수 호출
반환 위치 함수의 return render() 메서드
과거 역할 UI 표현 중심, Stateless state와 생명주기 관리, Stateful
현재 흐름 Hooks로 state와 생명주기 관리 가능 기존 코드 유지보수에서 주로 만남
학습 우선순위 현재 React 학습의 중심 개념 비교와 기존 코드 읽기용

4-3. 함수형 컴포넌트는 JSX를 반환하는 JavaScript 함수다

함수형 컴포넌트는 JavaScript 함수로 작성하고 JSX를 반환합니다. props를 입력으로 받을 수 있으며, 같은 컴포넌트라도 props 값에 따라 다른 화면을 보여줄 수 있습니다.

입력 props

부모 컴포넌트가 자식 컴포넌트에 속성처럼 전달하는 값입니다.

<CourseCard title="JSX" />
처리 함수 본문

필요한 JavaScript 로직을 작성하고 화면에 사용할 값을 준비합니다.

function CourseCard(props) {
    return ...;
}
출력 JSX

컴포넌트가 최종적으로 반환하는 UI 구조입니다.

return (
    <article>
        <h2>{props.title}</h2>
    </article>
);
재사용 다른 props

같은 컴포넌트에 다른 props를 주어 다른 내용을 표시합니다.

<CourseCard title="JSX" />
<CourseCard title="Virtual DOM" />

4-4. 컴포넌트 이름은 대문자로 시작한다

JSX는 태그 이름을 보고 HTML 태그인지 React 컴포넌트인지 판단합니다. 소문자로 시작하면 HTML 기본 태그로 해석하고, 대문자로 시작하면 React 컴포넌트로 해석합니다.

HTML 태그

<header>
    <h1>문서 헤더</h1>
</header>

<header>는 소문자로 시작하므로 HTML 기본 태그입니다.

React 컴포넌트

function Header() {
    return <h1>Header</h1>;
}

<Header />

<Header />는 대문자로 시작하므로 사용자 정의 컴포넌트입니다.

4-5. 함수형 컴포넌트를 작성하는 세 가지 방식

함수 선언문

export default function Header() {
    return (
        <header>
            <h1>React 컴포넌트</h1>
        </header>
    );
}

초보자가 읽기 가장 쉽고 함수 이름이 명확합니다.

함수 표현식

const Header = function () {
    return (
        <header>
            <h1>React 컴포넌트</h1>
        </header>
    );
};

export default Header;

함수를 변수에 담아 사용하는 방식입니다.

화살표 함수

const Header = () => {
    return (
        <header>
            <h1>React 컴포넌트</h1>
        </header>
    );
};

export default Header;

React 코드에서 자주 보이는 방식입니다.

화살표 함수 축약

const Header = () => (
    <header>
        <h1>React 컴포넌트</h1>
    </header>
);

export default Header;

반환문 하나만 있다면 return을 생략할 수 있습니다.

화살표 함수 주의

  • {}를 쓰면 return이 필요합니다.
  • ()를 쓰면 JSX를 바로 반환할 수 있습니다.
  • 컴포넌트가 아무것도 반환하지 않으면 화면에 표시되지 않습니다.

4-6. props는 컴포넌트에 전달되는 입력값이다

교안에서는 <Header name="가길동" />처럼 속성이 있다면 { name: "가길동" } 형태로 함수의 파라미터에 전달된다고 설명합니다.
즉 props는 부모가 자식 컴포넌트에 속성 형태로 전달하는 입력값입니다.

컴포넌트 사용 
<Header title="React 학습 페이지" />
→
함수 호출처럼 이해 
Header({
    title: "React 학습 페이지"
});
function Header(props) {
    return (
        <header>
            <h1>{props.title}</h1>
        </header>
    );
}

props는 다음 단원에서 더 깊게 배우지만, 이번 단원에서는 “같은 컴포넌트를 다른 내용으로 재사용하기 위한 입력값”으로 이해하면 됩니다.

6. 컴포넌트를 나누는 기준

컴포넌트를 잘 나누려면 기준이 필요합니다. 무조건 작게 쪼개는 것이 아니라 역할이 명확하고 재사용 가치가 있는 단위로 나누는 것이 좋습니다.

기준 설명 예시
화면 영역이 명확한가? 상단, 본문, 하단처럼 역할이 나뉘는가? Header, Main, Footer
반복되는 UI인가? 같은 모양이 여러 번 등장하는가? CourseCard, ProductCard
독립적으로 이름 붙일 수 있는가? “이건 무엇이다”라고 설명할 수 있는가? ProfileCard, SearchBox
재사용 가능성이 있는가? 다른 페이지나 다른 위치에서도 쓸 수 있는가? Button, Modal, Badge
코드가 너무 길어졌는가? App.jsx가 읽기 어려울 정도로 길어졌는가? 섹션별 컴포넌트 분리

나누기 좋은 단위

Header
Footer
반복되는 Card
큰 Section
재사용되는 Button
Layout

역할이 명확하고 재사용 가능성이 있는 단위입니다.

처음부터 나누지 않아도 되는 단위

한 번만 쓰는 작은 p 태그
의미 없는 wrapper div
너무 단순한 span
단순 장식용 텍스트 조각

파일만 많아지고 오히려 구조가 복잡해질 수 있습니다.

설계 기준 요약

컴포넌트 분리의 핵심은 “작게”가 아니라 “역할이 명확한가”입니다. 컴포넌트 이름을 봤을 때 이 컴포넌트가 무엇을 담당하는지 바로 떠올라야 합니다.

Mistakes

8. 자주 하는 실수

  • 실수 1. 컴포넌트명을 소문자로 시작함 React 컴포넌트 이름은 대문자로 시작해야 합니다. 소문자로 시작하면 JSX가 HTML 태그로 해석할 수 있습니다.
  • 실수 2. JSX를 반환하지 않음 함수형 컴포넌트는 JSX를 return해야 합니다. JSX를 작성만 하고 반환하지 않으면 화면에 표시되지 않습니다.
  • 실수 3. 화살표 함수에서 return을 빼먹음 화살표 함수에서 {}를 사용하면 return이 필요합니다. JSX만 바로 반환하려면 ()를 사용합니다.
  • 실수 4. export와 import를 빠뜨림 컴포넌트 파일에서 export default를 해야 다른 파일에서 import할 수 있고, App.jsx에서도 정확한 상대 경로로 import해야 합니다.
  • 실수 5. 컴포넌트를 너무 잘게 나눔 처음부터 모든 div, p, span을 컴포넌트로 만들 필요는 없습니다. 역할이 명확하거나 반복되는 UI를 기준으로 나누는 것이 좋습니다.
  • 실수 6. 컴포넌트가 너무 많은 일을 함 CourseCard가 카드뿐 아니라 Header, Footer, 전체 목록까지 담당하면 역할이 모호해집니다. 컴포넌트는 이름과 책임이 일치해야 합니다.
  • 실수 7. props를 자식에서 직접 바꾸려고 함 props는 부모에서 자식에게 전달되는 읽기 전용 입력값으로 생각해야 합니다. 변경이 필요하면 이벤트로 부모에게 요청하는 구조가 필요합니다.
  • 실수 8. App.jsx에 모든 코드를 계속 쌓음 컴포넌트를 배운 뒤에는 App.jsx가 모든 JSX를 직접 쓰는 파일이 아니라, 컴포넌트를 조립하는 최상위 컴포넌트가 되어야 합니다.
  • 실수 9. 컴포넌트 파일명과 함수명이 달라서 헷갈림 초보 단계에서는 Header.jsx → Header, Footer.jsx → Footer, CourseCard.jsx → CourseCard처럼 파일명과 컴포넌트명을 맞추는 것이 좋습니다.
  • 실수 10. 클래스형 컴포넌트를 지금 주력으로 공부하려고 함 클래스형 컴포넌트는 기존 코드 이해용으로 알아두면 좋지만, 현재 학습은 함수형 컴포넌트와 Hooks 중심으로 진행하는 것이 자연스럽습니다.
Interview

9. 면접 포인트

Q1. React에서 컴포넌트란 무엇인가요?

컴포넌트는 React 화면을 구성하는 재사용 가능한 UI 단위입니다. 각 컴포넌트는 JSX를 반환하고, 다른 컴포넌트와 조합되어 전체 화면을 만듭니다.

Q2. 왜 화면을 컴포넌트로 나누나요?

화면을 역할별로 나누면 코드가 읽기 쉬워지고, 반복되는 UI를 재사용할 수 있으며, 수정 범위가 작아집니다. 또한 props, state, hook을 연결하기 쉬운 구조가 됩니다.

Q3. 함수형 컴포넌트란 무엇인가요?

함수형 컴포넌트는 props를 입력받아 JSX를 반환하는 JavaScript 함수입니다. 현재 React에서는 Hooks와 함께 함수형 컴포넌트를 중심으로 개발합니다.

Q4. 클래스형 컴포넌트란 무엇인가요?

클래스형 컴포넌트는 React.Component를 상속해 만들고, render() 메서드에서 JSX를 반환하는 컴포넌트입니다. 과거에는 state와 생명주기 관리를 위해 많이 사용했습니다.

Q5. 함수형 컴포넌트와 클래스형 컴포넌트의 차이는 무엇인가요?

함수형 컴포넌트는 JavaScript 함수로 작성하고 JSX를 반환합니다. 클래스형 컴포넌트는 class 문법으로 작성하고 render()에서 JSX를 반환합니다.
Hooks 도입 이후 함수형 컴포넌트에서도 state와 생명주기 관리가 가능해져 현재는 함수형 컴포넌트가 주로 사용됩니다.

Q6. 현재 React에서 함수형 컴포넌트를 많이 사용하는 이유는 무엇인가요?

Hooks 이후 함수형 컴포넌트에서도 상태 관리와 생명주기 처리가 가능해졌습니다. 그래서 더 간결하고 재사용하기 쉬운 함수형 컴포넌트를 중심으로 개발합니다.

Q7. 컴포넌트 이름은 왜 대문자로 시작해야 하나요?

JSX에서 소문자로 시작하는 태그는 HTML 태그로 처리되고, 대문자로 시작하는 태그는 React 컴포넌트로 처리됩니다. 그래서 사용자 정의 컴포넌트는 반드시 대문자로 시작해야 합니다.

Q8. props는 무엇인가요?

props는 부모가 자식 컴포넌트에 전달하는 읽기 전용 데이터입니다. JSX 태그의 속성처럼 전달하고, 자식 컴포넌트는 이를 받아 화면에 출력합니다.

Q9. props를 자식 컴포넌트에서 직접 수정하면 안 되는 이유는 무엇인가요?

props는 부모가 내려준 읽기 전용 값입니다. 자식이 직접 수정하면 데이터 흐름이 깨지므로, 변경이 필요하면 이벤트를 통해 부모에게 요청하고 부모가 state를 변경해야 합니다.

Q10. App.jsx의 역할은 무엇인가요?

App.jsx는 React 앱의 최상위 컴포넌트입니다. 컴포넌트를 분리하면 App.jsx는 Header, Footer, Card 같은 하위 컴포넌트를 조립해 전체 화면 구조를 구성하는 역할을 합니다.

Q11. 컴포넌트를 나누는 기준은 무엇인가요?

Header, Footer, Card처럼 역할이 분명하거나 반복되는 UI는 컴포넌트로 분리하기 좋습니다. 컴포넌트 분리의 핵심은 파일을 많이 만드는 것이 아니라 책임을 명확히 나누는 것입니다.

Summary

→ 재사용 가능한 UI 조각

→ JavaScript 함수
→ Props 입력 가능
→ JSX 반환
→ 현재 React 학습의 중심

클래스형 컴포넌트
→ class 문법
→ render()에서 JSX 반환
→ 기존 코드 이해용으로 알아두기

컴포넌트 이름
→ 반드시 대문자로 시작

props
→ 부모가 자식에게 전달하는 읽기 전용 입력값

App.jsx
→ 모든 화면을 직접 쓰는 파일에서

핵심 체크리스트

  • 컴포넌트는 재사용 가능한 UI 조각입니다.
  • React 화면은 작은 컴포넌트를 조립해서 만듭니다.
  • 함수형 컴포넌트는 JavaScript 함수이며 JSX를 반환합니다.
  • 클래스형 컴포넌트는 render()에서 JSX를 반환합니다.
  • 현재 React 학습은 함수형 컴포넌트 중심으로 진행하는 것이 좋습니다.
  • 사용자 정의 컴포넌트 이름은 대문자로 시작해야 합니다.
  • props는 부모가 자식에게 전달하는 입력값입니다.
  • props는 읽기 전용으로 생각해야 합니다.
  • App.jsx는 하위 컴포넌트를 조립하는 최상위 컴포넌트입니다.
  • 컴포넌트 분리는 “작게”가 아니라 “역할이 명확하게” 하는 것이 중요합니다.

학습 메모

이번 단원은 React 화면을 어떤 단위로 나눌지 배우는 단원입니다.
컴포넌트는 단순히 파일을 나누는 기술이 아니라, 화면의 책임을 나누고 재사용 가능한 구조를 만드는 React의 핵심 사고방식입니다.

App
├─ Header
├─ CourseCard
├─ CourseCard
├─ CourseCard
└─ Footer

가장 많이 틀리는 부분

컴포넌트명을 소문자로 시작하거나, 화살표 함수에서 return을 빼먹거나, 컴포넌트 파일을 만들고 export / import를 빠뜨리는 실수가 많습니다.
사용자 정의 컴포넌트는 대문자로 시작하고, JSX를 반드시 반환해야 합니다.

다음 단원 연결

다음 단원에서는 props, children, state를 본격적으로 공부합니다. 이번 단원에서 만든 CourseCard처럼 같은 컴포넌트에 다른 값을 전달하는 방식이 다음 단원의 핵심 출발점입니다.

08. props, children, state
