ES6 모듈 import/export

React 프로젝트는 하나의 파일에 모든 코드를 작성하지 않습니다.
컴포넌트, 함수, 상수, API 요청 함수, Custom Hook, 데이터 파일을 역할별로 분리하고, 필요한 곳에서 다시 가져와 조립합니다.
이번 단원에서는 ES6 전체 문법이 아니라, React 파일 분리에서 가장 자주 사용하는 import와 export만 집중해서 공부합니다.
특히 named export/import와 default export/import의 차이, { } 사용 여부, 파일 경로 작성법, React 실무 파일 분리 흐름을 정확히 정리합니다.

Overview

1. 이번 단원은 무엇을 배우는가?

처음 React를 배울 때는 모든 코드를 App.jsx 하나에 작성할 수 있습니다.
하지만 프로젝트가 커지면 컴포넌트, 함수, 상수, API 요청 코드가 많아지면서 한 파일만으로는 관리하기 어려워집니다. 

src/
├─ App.jsx
├─ components/
│  ├─ Header.jsx
│  ├─ Button.jsx
│  └─ ProductCard.jsx
├─ api/
│  └─ productApi.js
├─ constants/
│  └─ routes.js
└─ utils/
   └─ price.js

이렇게 파일을 나누면 자연스럽게 “다른 파일에 있는 컴포넌트나 함수를 어떻게 가져오지?”라는 질문이 생깁니다. 이때 사용하는 문법이 바로 export와 import입니다.

파일 분리 역할 나누기

컴포넌트, 함수, 상수, 데이터를 각각 다른 파일에 둡니다.

export 밖으로 내보내기

다른 파일에서 사용할 수 있도록 값을 공개합니다.

import 가져와 사용하기

다른 파일에서 내보낸 값을 현재 파일로 가져옵니다.

이번 단원의 한 줄 정리

export는 파일 밖으로 내보내는 문법이고, import는 다른 파일의 코드를 가져오는 문법이며,
named 방식은 이름표가 붙은 여러 물건을 { }로 꺼내는 방식이고, default 방식은 파일의 대표 물건 하나를 { } 없이 꺼내는 방식입니다.

4. 핵심 개념 설명

4-1. ES6 모듈은 파일을 나누고 연결하는 문법이다

ES6 모듈은 JavaScript 코드를 파일 단위로 나누고, 필요한 값만 주고받을 수 있게 하는 문법입니다.
React에서는 이 문법을 통해 컴포넌트, 유틸 함수, API 함수, 상수, Hook을 여러 파일로 분리합니다. 

export
→ 현재 파일의 값을 밖으로 내보내기

import
→ 다른 파일에서 내보낸 값을 가져오기

4-2. 모듈은 하나의 파일 단위 코드 조각이다

하나의 .js 또는 .jsx 파일을 모듈로 볼 수 있습니다. 파일 안에는 컴포넌트, 함수, 상수, 객체, 배열 등이 들어갈 수 있습니다.

컴포넌트 모듈

Header.jsx
Button.jsx
ProductCard.jsx

화면을 구성하는 UI 조각을 담습니다.

유틸 모듈

price.js
date.js
validation.js

여러 곳에서 재사용할 함수를 담습니다.

상수 모듈

routes.js
roles.js
config.js

변하지 않는 고정값을 모아 둡니다.

API 모듈

productApi.js
authApi.js
boardApi.js

서버 요청 함수를 분리해 관리합니다.

4-3. named export와 named import

named export는 이름을 붙여 값을 내보내는 방식입니다. 한 파일에서 여러 개를 내보낼 수 있으며, import할 때는 { } 안에 export된 이름을 정확히 작성해야 합니다.

내보내는 파일

// utils/math.js
export function add(a, b) {
    return a + b;
}

export function multiply(a, b) {
    return a * b;
}

가져오는 파일

// App.jsx
import { add, multiply } from "./utils/math.js";

console.log(add(10, 20));
console.log(multiply(3, 4));

4-4. default export와 default import

default export는 한 파일의 대표값 하나를 내보내는 방식입니다. React 컴포넌트 파일에서 자주 사용하며, import할 때는 { }를 쓰지 않습니다.

내보내는 파일

// components/Header.jsx
export default function Header() {
    return (
        <header>
            <h1>Header</h1>
        </header>
    );
}

가져오는 파일

// App.jsx
import Header from "./components/Header.jsx";

export default function App() {
    return <Header />;
}

4-5. named와 default의 차이

구분 named default
내보내기 export const name = ... export default Component
가져오기 import { name } from "./file.js" import Component from "./file.js"
{ } 사용 사용함 사용하지 않음
이름 규칙 export한 이름과 맞춰야 함 가져오는 쪽에서 이름 지정 가능
한 파일에서 개수 여러 개 가능 한 개만 가능
주 사용 예 함수, 상수, API 함수 대표 컴포넌트 하나

4-6. named와 default를 함께 사용하기

한 파일에서 default export와 named export를 함께 사용할 수 있습니다. 가져올 때는 일반적으로 default import를 먼저 작성하고, named import를 뒤에 { }로 작성합니다. 

// ProductCard.jsx
export const CARD_TYPE = "PRODUCT";

export function formatProductName(name) {
    return `[상품] ${name}`;
}

export default function ProductCard({ name, price }) {
    return (
        <article>
            <h2>{formatProductName(name)}</h2>
            <p>{price.toLocaleString()}원</p>
        </article>
    );
}
// App.jsx
import ProductCard, { CARD_TYPE, formatProductName } from "./ProductCard.jsx";

4-7. import 경로는 현재 파일 기준이다

import 경로는 항상 현재 작성 중인 파일의 위치를 기준으로 계산합니다. 

src/
├─ components/
│  └─ ProductCard.jsx
└─ utils/
   └─ price.js
// ProductCard.jsx에서 price.js 가져오기

5. 그림처럼 이해하기

5-1. 파일을 택배 상자처럼 이해하기

파일 하나를 택배 상자로 생각하면 export와 import의 역할을 쉽게 이해할 수 있습니다.

파일 택배 상자

컴포넌트, 함수, 상수 같은 물건을 담고 있습니다.

export 송장 붙이기

상자 밖으로 보낼 수 있게 이름표를 붙입니다.

import 택배 받기

다른 파일에서 보낸 물건을 현재 파일로 가져옵니다.

5-2. named export는 여러 물건에 각각 이름표 붙이기

math.js 상자 
export function add() {}
export function multiply() {}

add, multiply 각각 이름표가 붙은 물건입니다.

→
App.jsx에서 꺼내기 
import { add, multiply } from "./math.js";

이름표를 정확히 보고 { } 안에서 골라 가져옵니다.

5-3. default export는 대표 상품 하나 보내기

Header.jsx 상자 
export default function Header() {
    return <header>Header</header>;
}

파일의 대표 상품 하나를 기본으로 내보냅니다.

→
App.jsx에서 받기 
import Header from "./Header.jsx";

대표 상품 하나를 받는 것이므로 { }가 없습니다.

5-4. { }는 이름표 있는 물건을 골라 꺼내겠다는 뜻이다

import { add } from "./math.js" named import

math.js에서 add라는 이름표가 붙은 값을 정확히 가져옵니다.

import Header from "./Header.jsx" default import

Header.jsx의 대표값 하나를 가져오므로 { }를 쓰지 않습니다.

6-6. src/App.jsx

import StudyCard from "./components/StudyCard.jsx";
import lessons from "./data/lessons.js";
import { SITE_NAME, CHAPTER_TITLE } from "./constants/site.js";
import { add, multiply, getAverage } from "./utils/math.js";
import "./App.css";

export default function App() {
    const total = add(10, 20);
    const result = multiply(3, 4);
    const average = getAverage(90, 3);

    return (
        <main className="app-page">
            <header className="page-hero">
                <p>{SITE_NAME}</p>
                <h1>{CHAPTER_TITLE}</h1>
                <p>파일을 나누고 import/export로 연결하는 예제입니다.</p>
            </header>

            <section className="card">
                <h2>named import 결과</h2>
                <p>10 + 20 = {total}</p>
                <p>3 × 4 = {result}</p>
                <p>90 / 3 = {average}</p>
            </section>

            <section className="lesson-list">
                {lessons.map((lesson) => (
                    <StudyCard
                        key={lesson.id}
                        title={lesson.title}
                        description={lesson.description}
                    />
                ))}
            </section>
        </main>
    );
}

7. React 파일 분리 실무 연결

React 프로젝트에서 import/export는 단순한 문법이 아니라, 프로젝트 구조를 유지보수 가능하게 만드는 핵심 도구입니다.

컴포넌트 분리

// components/Header.jsx
export default function Header() {
    return <header>Header</header>;
}

// App.jsx
import Header from "./components/Header.jsx";

대표 컴포넌트 하나는 default export를 자주 사용합니다.

유틸 함수 분리

// utils/price.js
export function formatPrice(price) {
    return price.toLocaleString() + "원";
}

// ProductCard.jsx
import { formatPrice } from "../utils/price.js";

여러 재사용 함수는 named export가 적합합니다.

상수 분리

// constants/routes.js
export const ROUTES = {
    HOME: "/",
    LOGIN: "/login"
};

// Header.jsx
import { ROUTES } from "../constants/routes.js";

라우트 경로, API 주소, 권한 코드는 상수 파일로 분리합니다.

API 함수 분리

// api/productApi.js
export async function getProducts() {
    const response = await fetch("/products");
    return response.json();
}

// ProductList.jsx
import { getProducts } from "../api/productApi.js";

서버 통신 코드는 API 파일로 분리하면 컴포넌트가 깔끔해집니다.

Custom Hook 분리

// hooks/useInput.js
export default function useInput(initialValue = "") {
    // useState 등 Hook 로직
}

// SearchForm.jsx
import useInput from "../hooks/useInput.js";

반복되는 Hook 로직은 Custom Hook으로 분리합니다.

CSS import

// App.jsx
import "./App.css";

CSS import는 변수나 함수를 가져오는 것이 아니라
스타일 파일을 적용하는 의미입니다.

Mistakes

8. 자주 하는 실수

  • 실수 1. named import에 {}를 빼먹음 export function add()로 내보냈다면 import { add } from "./math.js"처럼 {}가 필요합니다.
  • 실수 2. default import에 {}를 붙임 export default function StudyCard()로 내보냈다면 import StudyCard from "./StudyCard.jsx"처럼 {} 없이 가져옵니다.
  • 실수 3. named import 이름을 마음대로 바꿈 export function add()를 import { plus }로 가져오면 오류가 납니다. 별칭이 필요하면 import { add as plus }를 사용합니다.
  • 실수 4. default export를 한 파일에 여러 번 사용함 export default는 한 파일에서 한 번만 가능합니다. 여러 개를 내보내려면 named export를 사용합니다.
  • 실수 5. import 경로를 현재 파일 기준으로 생각하지 않음 ./는 현재 폴더, ../는 한 단계 위 폴더입니다. import 경로는 항상 현재 파일 위치 기준으로 계산합니다.
  • 실수 6. default import 이름과 파일명을 너무 다르게 씀 문법적으로 가능하더라도 import Banana from "./Header.jsx"처럼 쓰면 읽는 사람이 혼란스럽습니다.
  • 실수 7. CSS import와 JavaScript 값 import를 헷갈림 import "./App.css"는 스타일 적용이고, import { add } from "./math.js"는 값을 가져오는 것입니다.
  • 실수 8. export하지 않은 값을 import하려고 함 파일 안에 함수가 있어도 export하지 않았다면 다른 파일에서 가져올 수 없습니다.
  • 실수 9. named와 default를 섞어 쓸 때 순서를 헷갈림 혼합 import는 import DefaultName, { name1, name2 } from "./module.js" 형태가 일반적입니다.
  • 실수 10. React 전용 문법이라고 착각함 import/export는 React 전용 문법이 아니라 ES6 JavaScript 모듈 문법입니다. React에서 파일 분리를 많이 하므로 자주 볼 뿐입니다.
Interview

9. 면접 포인트

Q1. ES6 모듈이란 무엇인가요?

ES6 모듈은 파일 단위로 코드를 분리하고 재사용하기 위한 문법입니다. 한 파일에서 변수, 함수, 컴포넌트 등을 export하고, 다른 파일에서 import해서 사용할 수 있습니다.

Q2. export와 import의 역할은 무엇인가요?

export는 현재 모듈의 값이나 함수를 외부로 공개하는 문법이고, import는 다른 모듈에서 공개한 값을 현재 파일로 가져오는 문법입니다.

Q3. named export란 무엇인가요?

named export는 여러 값을 각각 이름을 붙여 내보내는 방식입니다. 가져올 때는 export된 이름과 동일한 이름을 { } 안에 작성해야 합니다.

Q4. default export란 무엇인가요?

default export는 모듈의 기본 내보내기입니다. 한 파일에서 하나만 사용할 수 있고, React 컴포넌트 파일에서 대표 컴포넌트를 내보낼 때 자주 사용합니다.

Q5. named export와 default export의 차이는 무엇인가요?

named export는 여러 값을 내보낼 수 있고 import할 때 이름을 맞춰 { }로 가져옵니다.
default export는 파일의 대표값 하나를 내보내며 import할 때 { } 없이 원하는 이름으로 가져올 수 있습니다.

Q6. 한 파일에서 named export와 default export를 함께 사용할 수 있나요?

가능합니다. 가져올 때는 일반적으로 import ProductCard, { CARD_TYPE } from "./ProductCard.jsx"처럼 default import를 먼저 쓰고, named import를 뒤에 작성합니다.

Q7. React에서 import/export를 왜 많이 사용하나요?

React 프로젝트는 컴포넌트와 로직을 여러 파일로 나누어 관리합니다. import/export를 사용하면 컴포넌트, API 함수, 유틸 함수, 상수 등을 재사용할 수 있고 유지보수성이 좋아집니다.

Q8. import 경로에서 ./와 ../의 차이는 무엇인가요?

./는 현재 디렉터리를 의미하고, ../는 상위 디렉터리를 의미합니다. import 경로는 현재 작성 중인 파일의 위치를 기준으로 계산해야 합니다.

Q9. default import에서 이름을 마음대로 바꿔도 되나요?

문법적으로는 가능합니다. 다만 유지보수를 위해 보통 파일명이나 컴포넌트명과 같은 이름으로 가져오는 것이 좋습니다.

Q10. named import의 이름을 바꾸고 싶으면 어떻게 하나요?

as 키워드를 사용해 별칭을 줄 수 있습니다. 예를 들어 import { add as plus } from "./math.js"처럼 작성합니다.

핵심 체크리스트

  • ES6 모듈은 파일 단위로 코드를 나누고 연결하는 문법입니다.
  • export는 현재 파일의 값을 밖으로 내보냅니다.
  • import는 다른 파일에서 내보낸 값을 가져옵니다.
  • named export는 한 파일에서 여러 개 사용할 수 있습니다.
  • named import는 { }를 사용하고 이름을 정확히 맞춰야 합니다.
  • default export는 한 파일에서 한 번만 사용할 수 있습니다.
  • default import는 { }를 사용하지 않고 이름을 가져오는 쪽에서 정할 수 있습니다.
  • 컴포넌트 파일은 default export를 많이 사용합니다.
  • 유틸 함수, 상수, API 함수는 named export를 많이 사용합니다.
  • import 경로는 현재 파일 위치를 기준으로 작성합니다.

학습 메모

이번 단원은 React 파일 구조를 읽는 데 매우 중요합니다. App.jsx에서 다른 컴포넌트를 가져오고, API 함수나 유틸 함수를 분리해 재사용하려면 import/export를 정확히 이해해야 합니다.

오늘의 핵심 문법

// named export
export const SITE_NAME = "React";
export function add(a, b) {
    return a + b;
}

// named import
import { SITE_NAME, add } from "./module.js";

// default export
export default function Header() {
    return <header>Header</header>;
}

// default import
import Header from "./Header.jsx";

// mixed import
import ProductCard, { CARD_TYPE } from "./ProductCard.jsx";

가장 많이 틀리는 부분

가장 많이 틀리는 부분은 { }입니다. named import는 { }가 필요하고, default import는 { }를 쓰지 않습니다. 또한 default export는 한 파일에서 한 번만 가능하다는 점을 꼭 기억해야 합니다.
