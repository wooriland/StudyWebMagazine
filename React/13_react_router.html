React Router

React는 하나의 index.html을 기반으로 동작하는 SPA 구조입니다.
그래서 여러 HTML 파일을 이동하는 방식이 아니라, 현재 URL 주소에 따라 어떤 컴포넌트를 보여줄지 React가 결정해야 합니다.
이번 단원에서는 BrowserRouter, Routes, Route, Link, NavLink, Outlet, URL parameter, query string, useNavigate를 사용해
주소별 화면 전환 구조를 만드는 방법을 학습합니다.
핵심은 React Router를 SPA에서 URL에 따라 화면 컴포넌트를 바꾸는 주소별 화면 스위치로 이해하는 것입니다.

보여주는 SPA 라우팅을 이해한다.

4. 핵심 개념 설명

4-1. React Router는 주소별 화면 스위치다

React Router는 현재 브라우저 주소를 확인하고, 등록된 Route 목록 중에서 일치하는 컴포넌트를 렌더링합니다.

/ Home

메인 화면

/about About

소개 화면

/products ProductList

상품 목록

/products/1 ProductDetail

상품 상세

4-2. SPA 라우팅 원리

전통적인 웹은 주소를 이동할 때마다 서버가 새 HTML을 보내고 브라우저가 전체 페이지를 다시 그릴 수 있습니다.
반면 React SPA는 하나의 index.html을 기반으로 현재 URL에 맞는 컴포넌트만 바꿔 보여줍니다.

전통적인 페이지 이동

사용자가 /about 클릭
↓
서버에 /about 요청
↓
새 HTML 응답
↓
브라우저 전체 새로고침

페이지 전체가 다시 로드될 수 있고, 기존 클라이언트 상태가 초기화될 수 있습니다.

React Router 이동

사용자가 /about 클릭
↓
React Router가 주소 변경
↓
서버에서 새 HTML을 다시 받지 않음
↓
About 컴포넌트 렌더링

앱은 유지되고, 현재 URL에 맞는 화면 컴포넌트만 바뀝니다.

4-3. BrowserRouter, Routes, Route

라우터를 사용하려면 먼저 BrowserRouter로 앱을 감싸고, Routes 안에 Route를 작성합니다. 

import { BrowserRouter } from "react-router-dom";

ReactDOM.createRoot(document.getElementById("root")).render(
    <BrowserRouter>
        <App />
    </BrowserRouter>
);
import { Routes, Route } from "react-router-dom";

export default function App() {
    return (
        <Routes>
            <Route path="/" element={<Home />} />
            <Route path="/about" element={<About />} />
        </Routes>
    );
}

4-4. Link와 NavLink

앱 내부 이동에는 <a> 태그보다 Link 또는 NavLink를 사용합니다. Link는 새로고침 없는 이동을 담당하고, NavLink는 현재 경로와 일치할 때 active 스타일을 줄 수 있습니다.

내부 이동에 a 태그 사용

<a href="/about">About</a>

브라우저가 서버에 새 요청을 보내 전체 새로고침이 발생할 수 있습니다.

Link 사용

<Link to="/about">About</Link>

React Router가 주소만 변경하고, 앱 내부에서 화면을 전환합니다.

4-5. Outlet과 중첩 라우팅

중첩 라우팅은 공통 레이아웃을 유지하고 본문만 교체하는 구조입니다. 부모 컴포넌트 안에 Outlet을 두면, 현재 URL과 일치하는 자식 라우트가 그 위치에 렌더링됩니다. 

import { Outlet } from "react-router-dom";

export default function Layout() {
    return (
        <>
            <header>공통 헤더</header>
            <main>
                <Outlet />
            </main>
        </>
    );
}
<Routes>
    <Route path="/" element={<Layout />}>
        <Route index element={<Home />} />
        <Route path="about" element={<About />} />
        <Route path="products" element={<ProductList />} />
    </Route>
</Routes>

4-6. URL parameter와 query string

URL parameter는 특정 자원의 ID처럼 주소 경로 안에 들어가는 값이고, query string은 검색어, 필터, 정렬, 페이지 번호처럼 선택 조건을 표현하는 값입니다.

구분 예시 읽는 Hook 주 사용 목적
URL parameter /products/3 useParams() 상품 ID, 게시글 ID처럼 특정 자원 식별
query string /search?keyword=react&page=2 useSearchParams() 검색어, 필터, 정렬, 페이지 번호

4-7. useNavigate

useNavigate는 사용자가 링크를 클릭하는 것이 아니라 코드 로직에 따라 페이지를 이동해야 할 때 사용합니다.
예를 들어 로그인 성공 후 메인 페이지로 이동하거나, 저장 완료 후 상세 페이지로 이동할 때 사용할 수 있습니다. 

import { useNavigate } from "react-router-dom";

export default function Login() {
    const navigate = useNavigate();

    function handleLogin() {
        // 로그인 성공 처리
        navigate("/mypage");
    }

    return (
        <button onClick={handleLogin}>
            로그인
        </button>
    );
}

6-4. src/App.jsx

import { Routes, Route } from "react-router-dom";
import "./App.css";

import Layout from "./components/Layout.jsx";
import Home from "./pages/Home.jsx";
import About from "./pages/About.jsx";
import ProductList from "./pages/ProductList.jsx";
import ProductDetail from "./pages/ProductDetail.jsx";
import Search from "./pages/Search.jsx";
import NotFound from "./pages/NotFound.jsx";

export default function App() {
    return (
        <Routes>
            <Route path="/" element={<Layout />}>
                <Route index element={<Home />} />
                <Route path="about" element={<About />} />
                <Route path="products" element={<ProductList />} />
                <Route path="products/:productId" element={<ProductDetail />} />
                <Route path="search" element={<Search />} />
                <Route path="*" element={<NotFound />} />
            </Route>
        </Routes>
    );
}

Layout을 부모 라우트로 두고, 자식 라우트들이 Outlet 자리에 렌더링됩니다.

6-6. src/pages/Home.jsx

import { Link, useNavigate } from "react-router-dom";

export default function Home() {
    const navigate = useNavigate();

    function goFirstProduct() {
        navigate("/products/1");
    }

    return (
        <section className="page-card">
            <h2>Home</h2>
            <p>React Router 실습 메인 페이지입니다.</p>

            <div className="link-group">
                <Link to="/about">소개 페이지로 이동</Link>
                <Link to="/products">상품 목록으로 이동</Link>
                <Link to="/search?keyword=router">router 검색</Link>
            </div>

            <button onClick={goFirstProduct}>
                코드로 1번 상품 상세 이동
            </button>
        </section>
    );
}

Link는 클릭 이동, useNavigate는 코드 로직 이동에 사용합니다.

                    <li key={product.id}>
                        <span>{product.name}</span>
                        <Link to={`/products/${product.id}`}>
                            상세 보기
                        </Link>
                    </li>
                ))}
            </ul>
        </section>
    );
}

Mistakes

7. 자주 하는 실수

  • 실수 1. 내부 이동에 <a> 태그를 사용함 앱 내부 이동에 <a href="/about">를 사용하면 전체 새로고침이 발생할 수 있습니다. 내부 이동은 Link 또는 NavLink를 사용합니다.
  • 실수 2. BrowserRouter로 감싸지 않음 BrowserRouter가 없으면 Routes, Link, useNavigate 같은 라우터 기능이 제대로 동작하지 않습니다.
  • 실수 3. Route를 Routes 밖에 작성함 Route는 반드시 Routes 내부에서 사용해야 합니다.
  • 실수 4. element에 컴포넌트 함수를 그대로 넣음 element={About}이 아니라 element={<About />}처럼 JSX를 전달해야 합니다.
  • 실수 5. 중첩 라우팅에서 Outlet을 빼먹음 부모 Layout 안에 Outlet이 없으면 자식 라우트 화면이 들어갈 자리가 없습니다.
  • 실수 6. URL parameter와 query string을 혼동함 /products/1은 특정 상품 ID를 가리키는 parameter이고, /search?keyword=react는 검색 조건을 나타내는 query string입니다.
  • 실수 7. useParams 값이 문자열이라는 점을 잊음 useParams()로 읽은 값은 기본적으로 문자열입니다. 숫자 비교가 필요하면 Number(productId)처럼 변환합니다.
  • 실수 8. NavLink의 isActive 사용법을 헷갈림 className="active"를 고정하면 항상 active가 됩니다. className={({ isActive }) => ...} 형태로 작성해야 현재 경로에 맞게 active 처리가 됩니다.
  • 실수 9. 중첩 라우트의 path에 /를 습관적으로 붙임 중첩 라우팅의 자식 path는 보통 부모 경로 기준의 상대 경로로 작성합니다. 예를 들어 path="about"은 부모 / 기준으로 /about이 됩니다.
  • 실수 10. 없는 주소 처리를 하지 않음 path="*" 라우트를 두어 잘못된 주소에 대한 NotFound 화면을 제공하는 것이 좋습니다.
  • 실수 11. useNavigate를 컴포넌트 밖에서 호출함 useNavigate는 Hook이므로 함수형 컴포넌트나 Custom Hook 내부에서 호출해야 합니다.
  • 실수 12. 배포 후 새로고침 404 문제를 이해하지 못함 SPA는 서버가 어떤 경로 요청이 와도 index.html로 돌려주는 fallback 설정이 필요할 수 있습니다. 배포 후 /about 새로고침에서 404가 난다면 서버 설정을 확인해야 합니다.
Interview

8. 면접 포인트

Q1. React Router란 무엇인가요?

React Router는 React 애플리케이션에서 브라우저 URL에 따라 다른 화면 컴포넌트를 렌더링하게 해 주는 라이브러리입니다.
SPA 구조에서 페이지 새로고침 없이 화면 전환을 구현할 때 사용합니다.

Q2. SPA 라우팅이란 무엇인가요?

SPA 라우팅은 하나의 HTML 문서를 기반으로 브라우저 주소만 변경하고, 현재 URL에 맞는 컴포넌트를 클라이언트에서 렌더링하는 방식입니다.
그래서 전체 페이지 새로고침 없이 화면 전환이 가능합니다.

Q3. BrowserRouter는 왜 필요한가요?

BrowserRouter는 React Router 기능의 기반이 되는 컴포넌트입니다. 앱을 BrowserRouter로 감싸야 Link, Routes, Route, useNavigate 같은 라우터 기능을 사용할 수 있습니다.

Q4. Routes와 Route의 차이는 무엇인가요?

Routes는 Route들을 묶는 컨테이너이고, Route는 path와 element를 연결해 해당 URL에 어떤 컴포넌트를 보여줄지 정의합니다.

Q5. Route의 element에는 무엇을 넣어야 하나요?

Route의 element에는 컴포넌트 함수 자체가 아니라 JSX 형태인 <About />을 전달해야 합니다.

Q6. Link를 사용하는 이유는 무엇인가요?

일반 <a> 태그는 브라우저의 기본 페이지 이동으로 전체 새로고침을 만들 수 있습니다.
반면 Link는 React Router를 통해 주소만 변경하고 해당 컴포넌트를 렌더링하므로 SPA 상태를 유지할 수 있습니다.

Q7. NavLink는 Link와 무엇이 다른가요?

NavLink는 Link의 확장 형태로, 현재 경로와 일치하는지 isActive 값을 통해 확인할 수 있습니다. 메뉴의 활성화 스타일을 줄 때 사용합니다.

Q8. Outlet은 무엇인가요?

Outlet은 부모 라우트 컴포넌트 안에서 자식 라우트가 들어갈 자리입니다. 공통 Layout을 유지하고 본문만 바꾸는 구조를 만들 때 사용합니다.

Q9. URL parameter는 언제 사용하나요?

URL parameter는 /products/1처럼 특정 자원을 식별하는 값을 경로에 포함할 때 사용합니다. 컴포넌트에서는 useParams로 값을 읽을 수 있습니다.

Q10. query string은 언제 사용하나요?

query string은 검색 조건이나 필터 조건처럼 선택적인 값을 URL에 담을 때 사용합니다. React Router에서는 useLocation이나 useSearchParams로 값을 읽을 수 있습니다.

Q11. useParams와 useSearchParams의 차이는 무엇인가요?

useParams는 /products/:id 같은 경로 파라미터를 읽을 때 사용하고, useSearchParams는 /search?keyword=react 같은 쿼리 스트링 값을 읽거나 변경할 때 사용합니다.

Q12. useNavigate는 언제 사용하나요?

useNavigate는 로그인 성공 후 메인 페이지로 이동하거나, 저장 완료 후 상세 페이지로 이동하는 것처럼 코드에서 직접 라우팅을 제어해야 할 때 사용합니다.

Q13. React Router에서 NotFound 페이지는 어떻게 만들 수 있나요?

path="*"를 사용해 앞의 Route들과 매칭되지 않는 모든 URL을 NotFound 컴포넌트로 처리할 수 있습니다.

Q14. React Router를 사용한 SPA에서 새로고침 시 404가 나는 이유는 무엇인가요?

React Router는 클라이언트에서 라우팅을 처리합니다.
하지만 새로고침하면 서버에 해당 경로로 직접 요청이 가기 때문에,
서버가 모든 SPA 경로를 index.html로 fallback하도록 설정하지 않으면 404가 발생할 수 있습니다.

핵심 체크리스트

  • React Router는 SPA에서 URL에 따라 화면 컴포넌트를 바꾸는 라이브러리입니다.
  • BrowserRouter는 앱 전체를 라우터 기능으로 감쌉니다.
  • Routes는 여러 Route를 묶는 영역입니다.
  • Route는 path와 element를 연결합니다.
  • element에는 <Component /> 형태의 JSX를 전달합니다.
  • 앱 내부 이동은 <a>보다 Link를 사용합니다.
  • NavLink는 현재 경로와 일치할 때 active 스타일을 줄 수 있습니다.
  • Outlet은 중첩 라우팅에서 자식 화면이 들어가는 자리입니다.
  • URL parameter는 useParams로 읽습니다.
  • query string은 useSearchParams로 읽거나 수정할 수 있습니다.
  • useNavigate는 코드 로직으로 페이지를 이동할 때 사용합니다.
  • path="*"로 없는 주소를 처리하는 NotFound 화면을 만들 수 있습니다.

학습 메모

이번 단원은 React가 여러 페이지처럼 보이는 화면을 만드는 핵심 단계입니다. React Router는 하나의 React 앱 안에서 주소에 따라 컴포넌트를 바꾸는 구조입니다.
BrowserRouter로 앱을 감싸고, Routes와 Route로 주소와 화면을 연결하며, Link, NavLink, Outlet, useParams, useSearchParams, useNavigate를 상황에 맞게 사용합니다.

오늘의 핵심 문법

<BrowserRouter>
    <App />
</BrowserRouter>

<Routes>
    <Route path="/" element={<Layout />}>
        <Route index element={<Home />} />
        <Route path="about" element={<About />} />
        <Route path="products/:productId" element={<ProductDetail />} />
        <Route path="*" element={<NotFound />} />
    </Route>
</Routes>

<Link to="/about">About</Link>

const { productId } = useParams();

const [searchParams, setSearchParams] = useSearchParams();

const navigate = useNavigate();

가장 많이 틀리는 부분

내부 이동에 <a> 태그를 사용하거나, Route를 Routes 밖에 작성하거나, element={About}처럼 컴포넌트 함수를 그대로 넣는 실수가 많습니다.
내부 이동은 Link, 라우트 연결은 element={<About />}, 중첩 라우팅에는 Outlet이 필요하다는 점을 기억해야 합니다.
