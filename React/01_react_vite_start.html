3. 핵심 개념 설명

3-1. React란?

React는 사용자 인터페이스를 만들기 위한 JavaScript 라이브러리입니다.
웹 화면을 하나의 큰 HTML 덩어리로 작성하는 대신, 버튼, 메뉴, 카드, 입력창, 목록 같은 작은 UI 조각으로 나누어 만드는 방식을 사용합니다.

전통적인 방식

const title = document.querySelector("#title");
title.textContent = "Hello React";

JavaScript로 실제 DOM 요소를 직접 찾고 내용을 수정합니다.

React 방식

export default function App() {
    const title = "Hello React";

    return <h1>{title}</h1>;
}

화면을 반환하는 컴포넌트를 만들고,
데이터가 바뀌면 React가 화면 갱신을 처리합니다.

3-2. Vite란?

Vite는 React 자체가 아닙니다. Vite는 React 프로젝트를 빠르게 생성하고, 개발 서버를 실행하고, 배포용 파일로 빌드해 주는 프론트엔드 개발 도구입니다.

빠른 개발 서버 Dev Server

처음 실행할 때 전체 프로젝트를 한 번에 모두 번들링하지 않고,
필요한 모듈을 중심으로 빠르게 로드합니다.

빠른 HMR Hot Module Replacement

코드를 수정하면 전체 페이지를 새로고침하지 않고
변경된 부분을 빠르게 화면에 반영합니다.

배포용 빌드 npm run build

개발용 파일을 실제 서버에 올릴 수 있는 최적화된 정적 파일로 변환합니다.

간단한 설정 vite.config.js

개발 서버 포트, 브라우저 자동 열기, 빌드 결과 폴더 등을 설정할 수 있습니다.

3-3. Node.js, NPM, Yarn, VS Code의 역할

도구 역할 처음 기억할 문장
Node.js 브라우저 밖에서 JavaScript 기반 개발 도구를 실행하는 환경 Vite와 NPM을 실행하려면 Node.js가 필요하다.
NPM React, Vite 같은 패키지를 설치하고 관리하는 패키지 매니저 npm install은 필요한 부품을 실제로 설치한다.
Yarn NPM과 비슷한 역할을 하는 대체 패키지 매니저 초보 단계에서는 NPM 하나로 통일하는 것이 좋다.
VS Code 코드를 작성하고 터미널 명령어를 실행하는 코드 에디터 React 코드는 VS Code에서 작성하고 터미널로 실행한다.
React Developer Tools 브라우저 개발자 도구에서 React 컴포넌트 구조를 확인하는 확장 도구 나중에 props, state, 컴포넌트 구조를 확인할 때 유용하다.

5. Vite 프로젝트 생성 실습

아래 명령어는 Vite를 사용해 React 프로젝트를 생성하고 실행하는 기본 흐름입니다. 터미널은 VS Code의 내장 터미널을 사용해도 됩니다.

5-1. 설치 확인

node --version
npm --version

버전 번호가 출력되면 Node.js와 NPM이 정상적으로 설치된 것입니다. Node.js를 방금 설치했다면 VS Code 터미널을 닫고 다시 열어야 인식되는 경우가 있습니다.

5-2. Vite React 프로젝트 생성

npm create vite@latest react-vite-start -- --template react

react-vite-start라는 폴더를 만들고, React 템플릿을 기반으로 프로젝트를 생성합니다.

5-3. 프로젝트 폴더로 이동

cd react-vite-start

npm install과 npm run dev는 프로젝트 폴더 안에서 실행해야 합니다.

5-4. 패키지 설치

npm install

package.json에 적힌 React, React DOM, Vite 관련 패키지를 실제로 설치합니다.

5-5. 개발 서버 실행

npm run dev

터미널에 출력되는 Local 주소를 브라우저에서 열면 React 화면을 확인할 수 있습니다.

5-6. 배포용 빌드

npm run build

실제 배포에 사용할 수 있는 파일이 dist 폴더에 생성됩니다.

5-7. src/App.jsx 수정 예제

프로젝트가 실행되면 src/App.jsx 파일을 아래처럼 수정해 봅니다. 저장 후 브라우저 화면이 바로 갱신되면 Vite 개발 서버와 HMR이 정상적으로 동작하는 것입니다. 

import "./App.css";

function ToolCard({ name, role, command }) {
    return (
        <article className="tool-card">
            <h2>{name}</h2>
            <p>{role}</p>
            <code>{command}</code>
        </article>
    );
}

export default function App() {
    return (
        <main className="app">
            <section className="hero">
                <p className="eyebrow">React.js With Vite</p>
                <h1>React와 Vite 시작하기</h1>
                <p>
                    React는 화면을 컴포넌트로 만들고,
                    Vite는 React 프로젝트를 빠르게 실행하고 빌드할 수 있게 도와줍니다.
                </p>
            </section>

            <section className="tool-grid" aria-label="React 개발 도구 목록">
                <ToolCard
                    name="React"
                    role="사용자 인터페이스를 컴포넌트로 만드는 JavaScript 라이브러리"
                    command="화면 = 컴포넌트 조합"
                />

                <ToolCard
                    name="Vite"
                    role="개발 서버 실행, HMR, 빌드를 도와주는 프론트엔드 개발 도구"
                    command="npm run dev"
                />

                <ToolCard
                    name="NPM"
                    role="React와 Vite 같은 패키지를 설치하고 관리하는 도구"
                    command="npm install"
                />
            </section>
        </main>
    );
}

예제에서 지금 이해할 것

  • App은 화면을 반환하는 React 컴포넌트입니다.
  • ToolCard는 반복해서 사용할 수 있는 작은 컴포넌트입니다.
  • name, role, command는 부모 컴포넌트가 자식 컴포넌트에 전달한 값입니다.
  • className은 JSX에서 CSS 클래스를 지정할 때 사용하는 속성입니다.
  • 파일을 저장하면 Vite가 변경 내용을 감지해 브라우저 화면을 빠르게 갱신합니다.

Mistakes

7. 자주 하는 실수

  • 실수 1. Node.js 설치 후 터미널을 다시 열지 않음 Node.js를 설치했는데 node --version이 동작하지 않는다면 VS Code 터미널을 닫고 다시 열거나 VS Code를 재시작합니다.
  • 실수 2. 프로젝트 폴더로 이동하지 않고 npm install 실행 npm install은 현재 위치의 package.json을 기준으로 실행됩니다. 반드시 cd react-vite-start 후 실행해야 합니다.
  • 실수 3. npm install을 빼먹고 npm run dev 실행 프로젝트 틀만 생성된 상태에서는 패키지가 실제로 설치되지 않았을 수 있습니다. npm install로 node_modules를 먼저 만들어야 합니다.
  • 실수 4. npm dev라고 입력함 Vite 개발 서버는 npm run dev로 실행합니다. dev는 package.json의 scripts에 등록된 이름입니다.
  • 실수 5. NPM과 Yarn을 섞어서 사용 처음에는 NPM 하나로 통일하는 것이 좋습니다. NPM과 Yarn을 섞으면 잠금 파일이 여러 개 생겨 의존성 관리가 혼란스러울 수 있습니다.
  • 실수 6. 터미널에 출력된 주소가 아닌 주소로 접속 Vite는 보통 http://localhost:5173/에서 실행되지만, 포트가 사용 중이면 다른 번호로 실행될 수 있습니다. 터미널에 출력된 Local 주소를 그대로 사용합니다.
  • 실수 7. CRA 자료와 Vite 자료를 섞어서 따라 함 npx create-react-app은 CRA 방식입니다. 이번 교안은 Vite 기준이므로 npm create vite@latest 명령어를 사용합니다.

8. 면접 포인트

Q1. React는 무엇인가요?

React는 사용자 인터페이스를 컴포넌트 단위로 만들기 위한 JavaScript 라이브러리입니다.
화면을 작은 UI 조각으로 나누고, 데이터를 기준으로 화면을 표현하며, 데이터가 바뀌면 React가 화면 갱신을 관리합니다.

Q2. React는 프레임워크인가요, 라이브러리인가요?

React는 일반적으로 라이브러리라고 부릅니다. React 자체는 UI 구성에 집중하고, 라우팅, API 통신, 빌드, 전역 상태 관리 등은 다른 도구와 함께 구성합니다.

Q3. Vite는 무엇인가요?

Vite는 빠른 개발 서버와 빌드 기능을 제공하는 프론트엔드 개발 도구입니다. 개발 중에는 빠른 HMR을 제공하고, 배포 시에는 최적화된 정적 파일을 생성합니다.

Q4. React와 Vite의 차이는 무엇인가요?

React는 UI를 만드는 라이브러리이고, Vite는 React 프로젝트를 실행하고 빌드하는 개발 도구입니다. 즉 React는 화면을 만드는 역할, Vite는 개발 환경과 빌드 흐름을 담당합니다.

Q5. Node.js는 왜 필요한가요?

React 앱은 최종적으로 브라우저에서 실행되지만, 개발 중에는 패키지 설치, Vite 실행, JSX 변환, 빌드 같은 작업이 필요합니다.
이런 개발 도구들이 Node.js 환경에서 실행되므로 Node.js가 필요합니다.

Q6. npm install은 어떤 역할을 하나요?

npm install은 package.json에 적힌 의존성 목록을 읽고 필요한 패키지를 실제로 설치합니다. 설치된 패키지는 node_modules 폴더에 저장됩니다.

Q7. npm run dev와 npm run build의 차이는 무엇인가요?

npm run dev는 개발 서버를 실행하는 명령어이고, npm run build는 배포용 파일을 만드는 명령어입니다. 개발 중에는 dev, 배포 전에는 build를 사용합니다.

Q8. Vite와 CRA의 차이는 무엇인가요?

CRA는 예전 React 프로젝트 생성 도구이고, Vite는 빠른 개발 서버, 빠른 HMR, 최적화된 빌드 환경을 제공하는 모던 프론트엔드 도구입니다.
현재 새 React 프로젝트 학습에는 Vite 기준으로 시작하는 것이 자연스럽습니다.

9. 이번 단원 정리

반드시 기억할 실행 순서

npm create vite@latest react-vite-start -- --template react
cd react-vite-start
npm install
npm run dev

체크포인트

  • React는 화면을 컴포넌트로 만드는 JavaScript 라이브러리입니다.
  • Vite는 React 프로젝트를 빠르게 실행하고 빌드하는 개발 도구입니다.
  • Node.js는 Vite와 NPM 같은 개발 도구를 실행하기 위해 필요합니다.
  • NPM은 패키지를 설치하고 프로젝트 실행 스크립트를 관리합니다.
  • npm install은 프로젝트 실행에 필요한 패키지를 실제로 설치합니다.
  • npm run dev는 개발 서버를 실행합니다.
  • npm run build는 배포용 파일을 생성합니다.
  • 처음에는 NPM과 Yarn을 섞어 쓰지 않고 NPM 기준으로 진행하는 것이 좋습니다.

학습 메모

이번 단원은 React 문법을 깊게 배우는 단계가 아니라, React 프로젝트를 만들고 실행하는 전체 개발 흐름을 이해하는 단계입니다.

node --version
npm --version
npm create vite@latest react-vite-start -- --template react
cd react-vite-start
npm install
npm run dev

가장 많이 틀리는 부분

프로젝트를 생성한 뒤 바로 npm run dev를 실행하는 실수가 많습니다. 반드시 프로젝트 폴더로 이동한 뒤 npm install을 먼저 실행하세요.
