Overview

1. 이번 단원은 무엇을 배우는가?

1단원에서는 Vite로 React 프로젝트를 생성하고 실행했습니다. 이제는 생성된 프로젝트 안에 어떤 파일이 있고, 각 파일이 어떤 역할을 하는지 읽어야 합니다.
React 프로젝트는 일반 HTML처럼 index.html 하나에 모든 화면을 작성하지 않습니다.
핵심은 세 파일의 연결입니다. index.html은 React 앱이 들어갈 자리를 만들고, main.jsx는 그 자리에 React를 연결하고, App.jsx는 실제 화면 내용을 작성합니다.

App.jsx 실제 화면 내용을 작성하는
최상위 React 컴포넌트입니다.

4. 핵심 개념 설명

4-1. index.html은 React 앱의 빈 자리다

Vite React 프로젝트의 index.html은 브라우저가 처음 읽는 HTML 파일입니다. 하지만 일반 HTML 페이지처럼 모든 화면을 여기에 직접 작성하지 않습니다.
가장 중요한 코드는 아래 두 줄입니다. 

<div id="root"></div>
<script type="module" src="/src/main.jsx"></script>

<div id="root"></div>는 React 화면이 들어갈 자리입니다. 처음에는 비어 있지만, main.jsx가 이 요소를 찾아서 React 컴포넌트를 넣습니다.
<script type="module">은 src/main.jsx 파일을 ES Module 방식으로 불러옵니다.

4-2. main.jsx는 React 앱의 시작점이다

main.jsx는 실제 화면 내용을 많이 작성하는 파일이 아니라, React 앱을 브라우저 DOM에 연결하는 시작 파일입니다. 

import React from "react";
import ReactDOM from "react-dom/client";
import "./index.css";
import App from "./App.jsx";

ReactDOM.createRoot(document.getElementById("root")).render(
    <React.StrictMode>
        <App />
    </React.StrictMode>
);
App 가져오기 import

import App from "./App.jsx";로 최상위 컴포넌트를 가져옵니다.

root 찾기 DOM 선택

document.getElementById("root")로 index.html의 빈 자리를 찾습니다.

React root 생성 createRoot

React가 관리할 렌더링 기준점을 만듭니다.

App 렌더링 render

<App /> 컴포넌트를 #root 안에 표시합니다.

4-3. App.jsx는 실제 화면을 작성하는 파일이다

App.jsx는 React 앱의 최상위 화면 컴포넌트입니다. 초기 학습에서는 가장 많이 수정하는 파일입니다. 

export default function App() {
    return (
        <>
            <h1>Vite 프로젝트 구조 읽기</h1>
            <p>이 화면은 App.jsx에서 작성했습니다.</p>
        </>
    );
}

세 파일의 역할을 한 문장으로 정리

index.html은 자리를 만들고, main.jsx는 React를 연결하고, App.jsx는 실제 화면을 만듭니다.

4-4. package.json은 실행 명령어와 의존성 설명서다

package.json은 프로젝트 이름, 필요한 패키지, 개발용 패키지, 실행 명령어를 관리합니다. npm run dev와 npm run build도 이 파일의 scripts에 정의된 명령어입니다. 

{
    "scripts": {
        "dev": "vite",
        "build": "vite build",
        "preview": "vite preview"
    },
    "dependencies": {
        "react": "^19.0.0",
        "react-dom": "^19.0.0"
    },
    "devDependencies": {
        "@vitejs/plugin-react": "latest",
        "vite": "latest"
    }
}

4-5. vite.config.js는 Vite의 설정판이다

vite.config.js는 개발 서버 포트, 브라우저 자동 열기, React 플러그인, 빌드 결과 폴더 등을 설정하는 파일입니다. 

import { defineConfig } from "vite";
import react from "@vitejs/plugin-react";

export default defineConfig({
    plugins: [react()],
    base: "/",
    server: {
        port: 9090,
        open: true
    },
    build: {
        outDir: "dist"
    }
});

Assets & Env

6. public, src/assets, .env 구분하기

6-1. public과 src/assets 차이

두 폴더 모두 이미지, CSS, JSON 같은 정적 리소스를 둘 수 있습니다. 하지만 사용하는 방식과 Vite가 처리하는 방식이 다릅니다.

구분 public src/assets
접근 방식 /vite.svg처럼 URL 경로로 접근 import logo from "./assets/react.svg"처럼 import
Vite 처리 번들링하지 않고 그대로 제공 번들링하고 최적화
파일명 원래 이름 유지 빌드 시 해시가 붙을 수 있음
주 사용처 favicon, robots.txt, 고정 파일명 이미지 컴포넌트에서 사용하는 이미지, CSS, JSON
기억법 그대로 공개하는 창고 앱 내부에서 import하는 자재 창고

public 파일 사용

// public/vite.svg
<img src="/vite.svg" alt="Vite 로고" />

public/vite.svg라고 쓰지 않고, 루트 경로인 /vite.svg로 사용합니다.

src/assets 파일 사용

// src/assets/react.svg
import reactLogo from "./assets/react.svg";

<img src={reactLogo} alt="React 로고" />

src/assets 안의 파일은 컴포넌트에서 import해서 사용합니다.

6-2. .env 환경 변수

.env 파일은 프로젝트 루트에 만들고, React 클라이언트 코드에서 사용할 변수는 반드시 VITE_로 시작해야 합니다.

.env 
VITE_APP_NAME=Vite 구조 학습 앱
VITE_LESSON_TITLE=02. Vite 프로젝트 구조 읽기
→
App.jsx 
{import.meta.env.VITE_APP_NAME}
{import.meta.env.VITE_LESSON_TITLE}
→
브라우저 화면

환경 변수 값이 React 화면에 출력됩니다.

.env 사용 시 주의

  • .env는 package.json과 같은 프로젝트 루트에 둡니다.
  • React 코드에서 사용할 변수는 VITE_ 접두사를 붙입니다.
  • .env를 새로 만들거나 수정한 뒤 값이 반영되지 않으면 개발 서버를 재시작합니다.
  • VITE_가 붙은 값은 브라우저 코드에 노출될 수 있으므로 비밀번호나 Secret Key를 넣지 않습니다.

6-3. .env 파일 종류와 우선순위

파일 역할
.env 모든 모드에서 공통으로 적용되는 기본 환경 변수
.env.local Git에 올리지 않는 로컬 개발자별 설정
.env.development 개발 모드 전용 설정
.env.production 배포 모드 전용 설정
.env.development.local 개발 모드에서 개인 로컬 설정
.env.production.local 배포 모드에서 개인 로컬 설정

우선순위

.env
<
.env.[mode]
<
.env.local
<
.env.[mode].local

처음 학습할 때는 모든 파일을 만들 필요는 없습니다. 이번 단원에서는 .env 파일 하나만 만들어도 충분합니다.

                    작성 기준: <strong>{author}</strong>

            <section className="structure-grid">

                    <h2>index.html</h2>
                    <p>
                        브라우저가 처음 읽는 HTML 파일입니다.
                        React 앱이 들어갈 #root를 준비합니다.
                    </p>
                </article>

                <article className="structure-card">
                    <h2>main.jsx</h2>
                    <p>
                        document.getElementById("root")로 root를 찾고,
                        <App />을 렌더링합니다.
                    </p>
                </article>

                <article className="structure-card">
                    <h2>App.jsx</h2>
                    <p>
                        실제 화면 내용을 작성하는 최상위 컴포넌트입니다.
                    </p>
                </article>

                <article className="structure-card">
                    <h2>.env</h2>
                    <p>
                        VITE_로 시작하는 환경 변수를

            <section className="logo-row" aria-label="정적 리소스 사용 예">

    );
}

예제에서 확인할 것

  • index.html의 #root는 React 화면이 들어갈 자리입니다.
  • main.jsx는 App.jsx를 가져와 #root에 렌더링합니다.
  • App.jsx는 실제 화면 내용을 작성하는 파일입니다.
  • public/vite.svg는 /vite.svg 경로로 사용합니다.
  • src/assets/react.svg는 import해서 사용합니다.
  • .env의 VITE_ 변수는 import.meta.env로 접근합니다.
Mistakes

8. 자주 하는 실수

  • 실수 1. index.html에 모든 화면을 직접 작성하려고 함 React 프로젝트에서 index.html은 전체 화면을 작성하는 곳이 아니라, React 앱이 들어갈 #root와 main.jsx 연결을 담당하는 HTML 템플릿입니다.
  • 실수 2. id="root"를 지우거나 이름을 바꿈 main.jsx가 document.getElementById("root")로 찾고 있다면 index.html에도 반드시 id="root"가 있어야 합니다.
  • 실수 3. main.jsx와 App.jsx의 역할을 헷갈림 main.jsx는 React를 DOM에 연결하는 시작 파일이고, App.jsx는 실제 화면 내용을 작성하는 최상위 컴포넌트입니다.
  • 실수 4. App import 경로를 잘못 작성함 main.jsx와 App.jsx가 같은 src 폴더에 있다면 import App from "./App.jsx";처럼 상대 경로를 사용합니다.
  • 실수 5. public 파일을 /public/파일명으로 접근함 public/vite.svg는 JSX에서 /vite.svg로 접근합니다. /public/vite.svg가 아닙니다.
  • 실수 6. src/assets 파일을 import하지 않고 URL처럼 사용함 src/assets/react.svg는 import reactLogo from "./assets/react.svg";처럼 import해서 사용하는 것이 기본입니다.
  • 실수 7. .env 파일을 src 폴더 안에 만듦 .env 파일은 package.json과 같은 프로젝트 루트에 만들어야 합니다.
  • 실수 8. 환경 변수 이름에 VITE_를 붙이지 않음 React 클라이언트 코드에서 사용할 Vite 환경 변수는 VITE_APP_NAME처럼 VITE_로 시작해야 합니다.
  • 실수 9. .env 수정 후 서버를 재시작하지 않음 .env를 새로 만들거나 수정했는데 값이 반영되지 않으면 Ctrl + C로 서버를 종료한 뒤 npm run dev를 다시 실행합니다.
  • 실수 10. VITE_ 환경 변수에 Secret Key를 넣음 VITE_가 붙은 값은 브라우저 코드에 노출될 수 있습니다. DB 비밀번호, 서버 Secret Key, 결제 Secret Key 같은 민감한 값은 넣지 않습니다.
Interview

9. 면접 포인트

Q1. Vite React 프로젝트에서 index.html의 역할은 무엇인가요?

index.html은 브라우저가 처음 읽는 HTML 템플릿입니다.
React 앱이 렌더링될 <div id="root"></div>를 준비하고, <script type="module" src="/src/main.jsx">로 React 앱의 시작 파일을 불러옵니다.

Q2. root는 왜 중요한가요?

root는 React 컴포넌트 트리가 들어갈 기준 DOM 요소입니다. main.jsx에서 document.getElementById("root")로 이 요소를 찾고, 그 안에 <App />을 렌더링합니다.

Q3. main.jsx는 어떤 역할을 하나요?

main.jsx는 React 앱의 시작점입니다. App.jsx를 import하고, index.html의 #root 요소를 찾아 <App />을 렌더링합니다.

Q4. App.jsx는 어떤 역할을 하나요?

App.jsx는 React 앱의 최상위 UI 컴포넌트입니다. 실제 화면에 표시될 JSX를 반환하고, 이후 다른 컴포넌트들을 조합해 전체 화면을 구성합니다.

Q5. package.json은 왜 중요한가요?

package.json은 프로젝트의 의존성과 실행 스크립트를 관리하는 파일입니다.
npm install은 이 파일의 의존성을 기준으로 패키지를 설치하고, npm run dev는 scripts.dev에 정의된 명령어를 실행합니다.

Q6. vite.config.js는 언제 수정하나요?

개발 서버 포트, 브라우저 자동 열기, React 플러그인, 빌드 결과 폴더, 배포 기준 경로 등을 변경해야 할 때 vite.config.js를 수정합니다.

Q7. public과 src/assets의 차이는 무엇인가요?

public은 빌드 시 그대로 복사되는 정적 리소스 폴더이고, /logo.png처럼 루트 경로로 접근합니다.
반면 src/assets는 import logo from "./assets/logo.png"처럼 import해서 사용하며, Vite가 번들링과 최적화를 수행합니다.

Q8. Vite에서 환경 변수 이름에 왜 VITE_를 붙이나요?

Vite는 클라이언트 코드에 노출할 환경 변수를 제한하기 위해 기본적으로 VITE_ 접두사가 붙은 변수만 import.meta.env로 노출합니다.
따라서 React 컴포넌트에서 사용할 환경 변수는 VITE_API_URL처럼 작성해야 합니다.

Q9. .env에 Secret Key를 넣어도 되나요?

넣으면 안 됩니다. React 앱은 브라우저에서 실행되므로 VITE_ 환경 변수는 사용자에게 노출될 수 있습니다.
DB 비밀번호, 서버 Secret Key, 결제 Secret Key 같은 민감한 값은 프론트엔드 .env에 넣지 않습니다.

Q10. “Target container is not a DOM element” 오류는 왜 발생하나요?

React가 렌더링하려는 DOM 요소를 찾지 못했을 때 발생할 수 있습니다.
보통 main.jsx에서 찾는 document.getElementById("root")와 index.html의 실제 id 값이 일치하지 않을 때 발생합니다.

10. 이번 단원 정리

반드시 기억할 실행 흐름

index.html
→ src/main.jsx
→ src/App.jsx
→ 브라우저 화면

핵심 체크리스트

  • index.html은 React 앱이 들어갈 #root를 준비합니다.
  • main.jsx는 #root를 찾아 <App />을 렌더링합니다.
  • App.jsx는 실제 화면 내용을 작성하는 최상위 컴포넌트입니다.
  • package.json은 의존성과 실행 스크립트를 관리합니다.
  • vite.config.js는 Vite 개발 서버와 빌드 설정을 관리합니다.
  • public 파일은 /파일명 경로로 사용합니다.
  • src/assets 파일은 import해서 사용합니다.
  • .env 변수는 VITE_ 접두사를 붙이고 import.meta.env로 접근합니다.
  • VITE_ 환경 변수에는 비밀번호나 Secret Key를 넣지 않습니다.
