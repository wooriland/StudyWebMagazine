3. 핵심 개념 설명

3-1. useState는 값을 기억하게 해 주는 Hook이다

함수형 컴포넌트는 JavaScript 함수입니다. 함수 안의 일반 변수는 React가 상태 변화로 추적하지 않습니다.
그래서 화면에 보여지는 값이 변해야 한다면 일반 변수 대신 state로 관리해야 합니다.

일반 변수 사용

export default function Counter() {
    let count = 0;

    function increase() {
        count = count + 1;
        console.log(count);
    }

    return (
        <button onClick={increase}>
            {count}
        </button>
    );
}

count는 일반 변수이므로 React가 화면 상태로 추적하지 않습니다.

useState 사용

import { useState } from "react";

export default function Counter() {
    const [count, setCount] = useState(0);

    return (
        <button onClick={() => setCount(count + 1)}>
            {count}
        </button>
    );
}

setCount를 호출하면 React가 state 변경을 알고 화면을 다시 렌더링합니다.

3-2. Hook은 아무 곳에서나 호출하면 안 된다

Hook은 함수처럼 보이지만 React가 렌더링 순서를 추적하기 위해 규칙을 가집니다. useState는 함수형 컴포넌트의 최상위에서 호출해야 합니다.

가능 컴포넌트 최상위 
export default function App() {
    const [name, setName] = useState("");

    return <input />;
}

함수형 컴포넌트의 가장 바깥 레벨에서 호출합니다.

불가능 조건문 안 
if (isReady) {
    const [name, setName] = useState("");
}

조건에 따라 Hook 호출 순서가 달라질 수 있습니다.

불가능 이벤트 함수 안 
function handleClick() {
    const [count, setCount] = useState(0);
}

Hook은 이벤트 핸들러 내부에서 호출하지 않습니다.

가능 Custom Hook 
function useFormInput() {
    const [value, setValue] = useState("");
    return [value, setValue];
}

직접 만든 Custom Hook 안에서는 Hook을 호출할 수 있습니다.

3-3. useState의 기본 형식

const [state, setState] = useState(초기값);
코드 의미 예시
state 현재 상태값 count, name, form
setState 상태를 변경하는 함수 setCount, setName, setForm
useState(초기값) 초기 state 생성 useState(0), useState("")
useState(() => 초기값) 초기값 계산을 처음 렌더링 때만 실행 useState(() => createInitialItems())

3-4. state 변경 시 리렌더링이 일어난다

state 변경 함수가 호출되면 React는 컴포넌트를 다시 실행하고, 새 JSX를 만든 뒤 이전 Virtual DOM과 비교해 바뀐 부분만 실제 DOM에 반영합니다.

리렌더링 흐름

사용자 이벤트 발생
↓
setState 호출
↓
state 변경
↓
함수형 컴포넌트 다시 호출
↓
새 JSX 반환
↓
새 Virtual DOM 생성
↓
이전 Virtual DOM과 비교
↓
변경된 부분만 실제 DOM 반영

3-5. state는 직접 대입하지 않는다

잘못된 예

count = count + 1;

JavaScript 변수만 바꾸는 것이므로 React가 변경을 모를 수 있습니다.

올바른 예

setCount(count + 1);

React에게 상태가 바뀌었음을 알리고 리렌더링을 요청합니다.

3-6. 객체와 배열 state는 새 객체, 새 배열로 업데이트한다

객체와 배열은 Reference Type입니다. 내부 값만 직접 수정하고 같은 객체나 배열을 다시 전달하면 React가 변경을 제대로 감지하지 못할 수 있습니다.

객체 직접 수정

form.name = "가길동";
setForm(form);

기존 객체를 직접 수정하고 같은 객체를 다시 전달합니다.

새 객체로 업데이트

setForm({
    ...form,
    name: "가길동"
});

기존 객체를 복사하고 변경할 값만 덮어쓴 새 객체를 전달합니다.

배열 직접 수정

items.push("새 항목");
setItems(items);

기존 배열의 내부를 직접 바꿉니다.

새 배열로 업데이트

setItems([
    ...items,
    "새 항목"
]);

기존 배열을 복사하고 새 항목을 추가한 새 배열을 전달합니다.

3-7. 폼 입력은 value와 onChange를 state와 연결한다

React에서 입력 폼을 state로 관리하려면 입력 요소의 현재 값은 value로 연결하고, 사용자의 입력은 onChange에서 읽어 state 변경 함수로 반영합니다. 

const [name, setName] = useState("");

<input
    value={name}
    onChange={(event) => setName(event.target.value)}
/>

입력값 동기화 흐름

사용자가 입력창에 글자 입력
↓
onChange 이벤트 발생
↓
event.target.value로 현재 입력값 읽기
↓
setName으로 state 변경
↓
컴포넌트 리렌더링
↓
value={name}에 새 state 값 반영
↓
입력창 화면 갱신

3-8. 입력 요소별 state 연결 방식

입력 요소 state 연결 이벤트에서 읽는 값
text input value={name} event.target.value
textarea value={intro} event.target.value
select value={role} event.target.value
checkbox checked={agree} event.target.checked

3-9. form 제출은 onSubmit과 preventDefault로 처리한다

HTML form은 기본적으로 제출 시 페이지를 새로고침하려고 합니다. React에서는 보통 event.preventDefault()로 기본 제출 동작을 막고, 현재 state 값으로 제출 로직을 처리합니다. 

function handleSubmit(event) {
    event.preventDefault();
    console.log(form);
}

<form onSubmit={handleSubmit}>
    <button type="submit">제출</button>
</form>

4-1. useState는 기억하는 변수다

일반 변수는 칠판에 잠깐 적어 둔 메모처럼 React가 변경을 추적하지 않습니다. state는 React가 관리하는 기록장에 적어 둔 값처럼, 변경되면 React가 다시 화면을 계산합니다.

일반 변수 칠판 메모

const [count, setCount] = useState(0);

5-2. src/App.jsx

import { useState } from "react";
import "./App.css";

export default function App() {
    const [count, setCount] = useState(0);

    const [form, setForm] = useState({
        name: "",
        email: "",
        intro: "",
        role: "student",
        agree: false
    });

    function handleIncrease() {
        setCount((prevCount) => prevCount + 1);
    }

    function handleDecrease() {
        setCount((prevCount) => prevCount - 1);
    }

    function handleReset() {
        setCount(0);
    }

    function handleChange(event) {
        const { name, value, type, checked } = event.target;

        setForm({
            ...form,
            [name]: type === "checkbox" ? checked : value
        });
    }

    function handleSubmit(event) {
        event.preventDefault();

        alert(
            `이름: ${form.name}
이메일: ${form.email}
자기소개: ${form.intro}
역할: ${form.role}
동의 여부: ${form.agree ? "동의" : "미동의"}`
        );
    }

    return (
        <main className="app-page">
            <header className="page-hero">
                <p>React.js With Vite</p>
                <h1>useState와 폼 입력</h1>
                <p>
                    클릭, 입력, 선택처럼 변하는 값을 state로 관리합니다.
                </p>
            </header>

            <section className="card">
                <h2>1. 카운터</h2>
                <p className="count-text">현재 숫자: {count}</p>

                <div className="button-group">
                    <button onClick={handleIncrease}>증가</button>
                    <button onClick={handleDecrease}>감소</button>
                    <button onClick={handleReset}>초기화</button>
                </div>
            </section>

            <section className="card">
                <h2>2. 프로필 폼</h2>

                <form onSubmit={handleSubmit} className="profile-form">
                    <label>
                        이름
                        <input
                            type="text"
                            name="name"
                            value={form.name}
                            onChange={handleChange}
                            placeholder="이름을 입력하세요"
                        />
                    </label>

                    <label>
                        이메일
                        <input
                            type="email"
                            name="email"
                            value={form.email}
                            onChange={handleChange}
                            placeholder="email@example.com"
                        />
                    </label>

                    <label>
                        자기소개
                        <textarea
                            name="intro"
                            value={form.intro}
                            onChange={handleChange}
                            placeholder="간단한 자기소개를 입력하세요"
                        />
                    </label>

                    <label>
                        역할
                        <select
                            name="role"
                            value={form.role}
                            onChange={handleChange}
                        >
                            <option value="student">학습자</option>
                            <option value="mentor">멘토</option>
                            <option value="admin">관리자</option>
                        </select>
                    </label>

                    <label className="checkbox-label">
                        <input
                            type="checkbox"
                            name="agree"
                            checked={form.agree}
                            onChange={handleChange}
                        />
                        개인정보 수집에 동의합니다.
                    </label>

                    <button type="submit">
                        제출
                    </button>
                </form>
            </section>

            <section className="card preview-card">
                <h2>3. 입력값 미리보기</h2>
                <p>이름: {form.name || "아직 입력하지 않음"}</p>
                <p>이메일: {form.email || "아직 입력하지 않음"}</p>
                <p>자기소개: {form.intro || "아직 입력하지 않음"}</p>
                <p>역할: {form.role}</p>
                <p>동의 여부: {form.agree ? "동의" : "미동의"}</p>
            </section>
        </main>
    );
}

Mistakes

6. 자주 하는 실수

  • 실수 1. state를 일반 변수처럼 직접 바꿈 count = count + 1처럼 직접 대입하면 React가 변경을 모를 수 있습니다. 반드시 setCount(count + 1)처럼 변경 함수를 사용해야 합니다.
  • 실수 2. Hook을 조건문이나 반복문 안에서 호출함 useState는 함수형 컴포넌트의 최상위에서 호출해야 합니다. 조건문, 반복문, 이벤트 함수 안에서 호출하면 안 됩니다.
  • 실수 3. value만 쓰고 onChange를 빼먹음 <input value={name} />처럼 작성하면 입력값이 state에 반영되지 않아 입력창이 읽기 전용처럼 동작할 수 있습니다.
  • 실수 4. onChange에서 event.target.value를 읽지 않음 사용자가 입력한 값을 state에 저장하려면 event.target.value를 읽어 setState에 전달해야 합니다.
  • 실수 5. checkbox에 value를 연결함 체크박스는 value가 아니라 checked를 state와 연결하고, event.target.checked로 선택 여부를 읽습니다.
  • 실수 6. 객체 state를 직접 수정함 form.name = "가길동"처럼 직접 수정하지 말고, setForm({ ...form, name: "가길동" })처럼 새 객체를 만들어야 합니다.
  • 실수 7. 배열 state를 직접 수정함 items.push("새 항목")처럼 직접 수정하지 말고, setItems([...items, "새 항목"])처럼 새 배열을 만들어야 합니다.
  • 실수 8. 이벤트 함수 실행 결과를 onClick에 전달함 onClick={setCount(count + 1)}가 아니라 onClick={() => setCount(count + 1)}처럼 나중에 실행할 함수를 전달해야 합니다.
  • 실수 9. form 제출 시 새로고침을 막지 않음 onSubmit에서는 event.preventDefault()를 호출해 브라우저의 기본 새로고침 동작을 막아야 합니다.
  • 실수 10. 입력값이 많아졌는데 state 구조를 정리하지 않음 입력값이 많아지면 각각의 state로 관리하기보다 form 객체 state로 묶어 관리하는 것이 편할 수 있습니다.
Interview

7. 면접 포인트

Q1. Hook이란 무엇인가요?

Hook은 함수형 컴포넌트에서 state나 생명주기 관련 기능을 사용할 수 있게 해 주는 함수입니다. 대표적으로 useState, useEffect가 있습니다.

Q2. Hook의 기본 규칙은 무엇인가요?

Hook은 함수형 컴포넌트 또는 Custom Hook의 최상위에서 호출해야 합니다. 조건문, 반복문, 중첩 함수, 이벤트 핸들러 안에서 호출하면 안 됩니다.

Q3. useState는 무엇인가요?

useState는 함수형 컴포넌트에서 상태를 만들기 위한 Hook입니다.
[state, setState] 형태의 배열을 반환하고, setState를 호출하면 상태가 변경되며 컴포넌트가 리렌더링됩니다.

Q4. const [count, setCount] = useState(0)의 의미는 무엇인가요?

count는 현재 상태값이고, setCount는 그 상태를 변경하는 함수입니다. useState(0)은 초기값을 0으로 설정한다는 의미입니다.

Q5. state를 직접 대입하면 안 되는 이유는 무엇인가요?

state는 React가 관리하는 값이기 때문에 직접 대입하지 않고 setState 함수로 변경해야 합니다. 그래야 React가 상태 변경을 감지하고 컴포넌트를 다시 렌더링합니다.

Q6. state가 변경되면 React에서는 어떤 일이 일어나나요?

state가 변경되면 React는 컴포넌트를 다시 렌더링합니다. 이 과정에서 새 JSX가 만들어지고, React는 이전 Virtual DOM과 비교해 바뀐 부분만 실제 DOM에 반영합니다.

Q7. 동일한 컴포넌트를 여러 번 사용하면 state는 공유되나요?

공유되지 않습니다. 동일한 컴포넌트를 여러 번 렌더링해도 각 컴포넌트는 독립적인 state를 가집니다. 그래서 한 Counter의 count를 변경해도 다른 Counter의 count는 바뀌지 않습니다.

Q8. React에서 입력창을 state로 관리하려면 어떻게 해야 하나요?

입력 요소의 value를 state와 연결하고, onChange에서 event.target.value를 읽어 state를 변경합니다.

Q9. controlled component란 무엇인가요?

Controlled component는 입력 요소의 값이 브라우저 DOM이 아니라 React state에 의해 제어되는 컴포넌트입니다.
value={state}와 onChange={...setState...}를 사용해 입력값과 state를 동기화합니다.

Q10. checkbox는 text input과 무엇이 다른가요?

text input, textarea, select는 보통 value와 event.target.value를 사용합니다. checkbox는 체크 여부가 중요하므로 checked와 event.target.checked를 사용합니다.

Q11. 객체 state는 어떻게 업데이트해야 하나요?

객체 state는 직접 수정하지 않고 새 객체를 만들어 업데이트해야 합니다. 예를 들어 setForm({ ...form, name: "가길동" })처럼 작성합니다.

Q12. form 제출 시 event.preventDefault()를 사용하는 이유는 무엇인가요?

form 제출 시 브라우저의 기본 새로고침 동작을 막기 위해 사용합니다. 그래야 React state를 유지한 상태로 제출 로직을 처리할 수 있습니다.

  • Hook은 함수형 컴포넌트 또는 Custom Hook의 최상위에서 호출해야 합니다.
  • state는 직접 대입하지 않고 setState 함수로 변경해야 합니다.
  • state 변경 시 컴포넌트가 다시 호출되고 화면이 갱신됩니다.
  • 동일한 컴포넌트를 여러 번 사용해도 각 컴포넌트의 state는 독립적입니다.
  • 입력창은 value와 onChange를 state와 함께 연결합니다.
  • event.target.value는 사용자가 입력한 현재 값을 의미합니다.
  • checkbox는 checked와 event.target.checked를 사용합니다.
  • 객체 state는 직접 수정하지 않고 새 객체를 만들어 업데이트합니다.
  • form 제출 시 event.preventDefault()로 새로고침을 막습니다.

학습 메모

이번 단원은 React에서 “값이 변하면 화면도 변한다”는 흐름을 직접 구현하는 단원입니다.
state는 React가 기억하는 값이고, setState는 React에게 상태 변경을 알리는 함수입니다.
특히 폼 입력에서는 value와 onChange를 함께 연결하는 구조를 반드시 익혀야 합니다.

오늘의 핵심 문법

const [name, setName] = useState("");

<input
    value={name}
    onChange={(event) => setName(event.target.value)}
/>

가장 많이 틀리는 부분

state를 count = count + 1처럼 직접 대입하거나, 입력창에 value만 연결하고 onChange를 빠뜨리는 실수가 많습니다.
state는 반드시 변경 함수로 바꾸고, 입력 요소는 state와 이벤트를 함께 연결해야 합니다.
