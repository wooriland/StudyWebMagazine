4. 핵심 개념 설명

4-1. Bootstrap은 완성된 UI 프레임워크다

Bootstrap은 버튼, 카드, 폼, 네비게이션, 모달 같은 UI를 빠르게 만들 수 있도록 미리 준비된 CSS 클래스와 JavaScript 기능을 제공하는 프론트엔드 UI 프레임워크입니다.
직접 CSS를 하나하나 작성하지 않아도, 이미 준비된 클래스명을 붙여 빠르게 화면을 구성할 수 있습니다.

직접 CSS 작성

.my-button {
    padding: 8px 16px;
    background-color: #dc3545;
    color: white;
    border: none;
    border-radius: 6px;
    font-weight: 600;
}

버튼 모양, 색상, 여백, hover 효과를 직접 설계해야 합니다.

Bootstrap 클래스 사용

<button className="btn btn-danger">
    Danger
</button>

미리 준비된 btn, btn-danger 클래스를 붙여 스타일을 적용합니다.

4-2. CDN 방식

CDN 방식은 Bootstrap 파일을 프로젝트 안에 직접 설치하지 않고, 외부 주소에서 CSS와 JavaScript를 불러오는 방식입니다.
교안에서는 index.html에 Bootstrap CSS와 JS CDN을 추가하는 방식을 제시합니다. 

<link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.2/dist/css/bootstrap.min.css" rel="stylesheet">
<script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.2/dist/js/bootstrap.bundle.min.js"></script>

React에서 Bootstrap 예제를 복사할 때 주의

  • Bootstrap 문서는 일반 HTML 예제가 많아 class를 사용합니다.
  • React JSX에서는 반드시 className으로 바꿔야 합니다.
  • 예: class="btn btn-primary" → className="btn btn-primary"

4-3. Bootstrap CSS와 Bootstrap JS

Bootstrap CDN에는 CSS와 JavaScript가 따로 있습니다.
CSS는 버튼 색상, 카드 모양, 폼 스타일, 그리드, 여백 같은 디자인을 담당하고,
JS는 모달, 드롭다운, Collapse, Navbar toggle 같은 동작을 담당합니다.

bootstrap.min.css 디자인 담당

버튼 색상, 카드 모양, 폼 스타일, 그리드, 여백, 배경색 같은 화면 스타일을 제공합니다.

bootstrap.bundle.min.js 동작 담당

모달, 드롭다운, Collapse, Navbar toggle, Toast 같은 동작형 UI에 필요할 수 있습니다.

CSS만 연결한 경우 스타일만 적용

버튼이나 카드 모양은 보이지만, 일부 동작형 UI는 작동하지 않을 수 있습니다.

React 컴포넌트 방식 Reactstrap

동작형 UI를 React 컴포넌트와 props 중심으로 다룰 수 있습니다.

4-4. UI 라이브러리와 Reactstrap

교안에서는 Bootstrap 같은 화면 디자인 프레임워크를 React 컴포넌트에서 직접 import해서 사용할 수 있도록 UI 엘리먼트를 컴포넌트로 제공하는 라이브러리들이 있다고 설명합니다.
예시로 Material-UI, React Toolbox, React Bootstrap, reactstrap 등이 언급됩니다.

도구 역할
Bootstrap CSS 클래스와 JavaScript 동작을 제공하는 UI 프레임워크
Reactstrap Bootstrap UI를 React 컴포넌트처럼 사용할 수 있게 해 주는 3rd party 라이브러리
React Bootstrap Bootstrap을 React 컴포넌트 형태로 사용할 수 있게 해 주는 또 다른 라이브러리
Material UI Material Design 기반 React UI 컴포넌트 라이브러리

4-5. bootstrap과 reactstrap의 차이

교안에서는 bootstrap을 Bootstrap CSS와 관련된 라이브러리로, reactstrap을 Bootstrap의 JavaScript 부분을 React로 변환한 3rd party 라이브러리로 설명합니다.

6. Bootstrap CDN 방식

CDN 방식은 가장 빠르게 Bootstrap을 붙여 볼 수 있는 방법입니다.
Vite React 프로젝트의 루트에 있는 index.html에 Bootstrap CSS와 JS 주소를 넣고, JSX에서는 Bootstrap 클래스를 className으로 사용합니다.

6-1. index.html

<!doctype html>
<html lang="ko">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Bootstrap CDN 실습</title>

    <link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.2/dist/css/bootstrap.min.css" rel="stylesheet">
  </head>
  <body>
    <div id="root"></div>

    <script type="module" src="/src/main.jsx"></script>
    <script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.2/dist/js/bootstrap.bundle.min.js"></script>
  </body>
</html>

6-2. src/App.jsx

export default function App() {
    return (
        <main className="container py-5">
            <section className="p-4 mb-4 bg-light border rounded">
                <h1 className="mb-3">Bootstrap CDN 방식</h1>
                <p className="text-muted">
                    index.html에 Bootstrap CDN을 연결하고,
                    JSX에서는 className으로 Bootstrap 클래스를 사용합니다.
                </p>

                <button className="btn btn-primary me-2">
                    Primary
                </button>

                <button className="btn btn-danger">
                    Danger
                </button>
            </section>

            <section className="card">
                <div className="card-body">
                    <h2 className="card-title">Bootstrap Card</h2>
                    <p className="card-text">
                        이 카드는 React 컴포넌트 라이브러리를 사용한 것이 아니라,
                        Bootstrap 클래스만 붙인 HTML 구조입니다.
                    </p>
                    <button className="btn btn-success">
                        확인
                    </button>
                </div>
            </section>
        </main>
    );
}

7. 패키지 방식과 Reactstrap

패키지 방식은 Bootstrap 관련 라이브러리를 프로젝트에 설치하고 JS/JSX 파일에서 import해서 사용하는 방식입니다. 교안에서는 bootstrap과 reactstrap을 함께 설치합니다.

7-1. 패키지 설치

npm install --save bootstrap reactstrap

최신 NPM 환경에서는 --save를 생략해도 됩니다. 

npm install bootstrap reactstrap

7-2. package.json 확인

"dependencies": {
    "bootstrap": "...",
    "reactstrap": "..."
}

설치 후 package.json에 두 패키지가 추가되었는지 확인합니다.

7-3. src/main.jsx

import React from "react";
import ReactDOM from "react-dom/client";
import "bootstrap/dist/css/bootstrap.min.css";
import "./index.css";
import App from "./App.jsx";

ReactDOM.createRoot(document.getElementById("root")).render(
    <React.StrictMode>
        <App />
    </React.StrictMode>
);

중요한 줄은 import "bootstrap/dist/css/bootstrap.min.css";입니다.
이 import가 없으면 Reactstrap 컴포넌트가 렌더링되어도 Bootstrap 스타일이 제대로 보이지 않을 수 있습니다.

8. 재사용 가능한 Button 컴포넌트 만들기

Reactstrap의 Button을 한 번 더 감싸서 우리 프로젝트에서 사용할 AppButton 컴포넌트를 만들 수 있습니다.
아직 props를 깊게 배우지 않았더라도, “값을 받아 버튼 모양을 바꾸는 컴포넌트”라고 이해하면 됩니다.

8-1. src/components/AppButton.jsx

import { Button } from "reactstrap";

export default function AppButton({
    color = "primary",
    outline = false,
    children
}) {
    return (
        <Button color={color} outline={outline} className="me-2 mb-2">
            {children}
        </Button>
    );
}

8-2. src/App.jsx

import AppButton from "./components/AppButton.jsx";

export default function App() {
    return (
        <main className="container py-5">
            <h1 className="mb-4">재사용 가능한 Button 컴포넌트</h1>

            <AppButton color="primary">
                저장
            </AppButton>

            <AppButton color="danger">
                삭제
            </AppButton>

            <AppButton color="success" outline>
                완료
            </AppButton>
        </main>
    );
}

Mistakes

9. 자주 하는 실수

  • 실수 1. Bootstrap 문서의 class를 그대로 복사함 Bootstrap 문서는 일반 HTML 예제가 많습니다. React JSX에서는 class="btn btn-primary"가 아니라 className="btn btn-primary"를 사용해야 합니다.
  • 실수 2. CDN CSS만 넣고 JS 동작까지 기대함 CSS만 연결하면 버튼이나 카드 스타일은 보이지만, 모달, 드롭다운, Collapse, Navbar toggle 같은 동작형 UI는 bootstrap.bundle.min.js가 필요할 수 있습니다.
  • 실수 3. Reactstrap만 설치하고 Bootstrap CSS를 import하지 않음 reactstrap 컴포넌트는 렌더링되더라도 bootstrap/dist/css/bootstrap.min.css를 import하지 않으면 Bootstrap 스타일이 제대로 보이지 않을 수 있습니다.
  • 실수 4. Button을 import하지 않고 사용함 <Button>은 HTML 기본 태그가 아니라 reactstrap에서 가져와야 하는 React 컴포넌트입니다.
  • 실수 5. <button>과 <Button>을 헷갈림 소문자 <button>은 HTML 태그이고, 대문자 <Button>은 React 컴포넌트입니다.
  • 실수 6. Reactstrap Button에 className="btn btn-danger"만 사용함 Reactstrap의 Button은 color="danger" 같은 prop을 제공합니다. className은 추가 여백이나 커스텀 스타일을 붙일 때 함께 사용할 수 있습니다.
  • 실수 7. main.jsx에 화면 요소를 계속 추가함 교안 예제처럼 설치 확인용으로 main.jsx에 <Button>을 넣을 수는 있지만, 실제 화면 구성은 보통 App.jsx나 별도 컴포넌트에서 작성하는 것이 좋습니다.
  • 실수 8. CDN 방식과 패키지 방식을 동시에 중복 적용함 index.html에 CDN을 넣고 main.jsx에서도 Bootstrap CSS를 import하면 같은 스타일이 중복 로드될 수 있습니다.
  • 실수 9. 패키지 설치 후 개발 서버 오류를 헷갈림 Cannot find module 'reactstrap' 같은 오류가 계속 보이면 설치 위치와 package.json을 확인하고, 필요하면 개발 서버를 재시작합니다.
  • 실수 10. color="red"처럼 임의의 CSS 색상명을 넣음 Reactstrap Button의 color는 primary, danger, success 같은 Bootstrap 의미 색상 이름을 사용합니다.
Interview

10. 면접 포인트

Q1. Bootstrap은 무엇인가요?

Bootstrap은 버튼, 카드, 폼, 네비게이션, 모달 같은 UI를 빠르게 만들 수 있도록 미리 준비된 CSS 클래스와 JavaScript 동작을 제공하는 프론트엔드 UI 프레임워크입니다.

Q2. React에서 Bootstrap을 적용하는 방법은 무엇이 있나요?

크게 두 가지가 있습니다. 하나는 index.html에 Bootstrap CDN을 연결하는 방식이고,
다른 하나는 bootstrap, reactstrap 같은 패키지를 설치해 JS/JSX 파일에서 import해서 사용하는 방식입니다.

Q3. CDN 방식과 패키지 방식의 차이는 무엇인가요?

CDN 방식은 index.html에서 외부 Bootstrap CSS와 JS 주소를 직접 연결합니다.
패키지 방식은 NPM으로 Bootstrap 관련 패키지를 설치하고 import를 통해 프로젝트 구조 안에서 관리합니다.

Q4. Bootstrap CSS와 Bootstrap JS의 차이는 무엇인가요?

Bootstrap CSS는 버튼 색상, 카드 모양, 그리드, 폼 스타일 같은 디자인을 담당합니다.
Bootstrap JS는 모달, 드롭다운, Collapse, Navbar toggle 같은 동작형 UI를 담당합니다.

Q5. React에서 Bootstrap 예제를 복사할 때 주의할 점은 무엇인가요?

Bootstrap 문서의 예제는 일반 HTML 기준인 경우가 많아 class를 사용합니다. React JSX에서는 class 대신 className으로 바꿔야 합니다.

Q6. bootstrap 패키지와 reactstrap 패키지의 차이는 무엇인가요?

bootstrap은 Bootstrap CSS 스타일을 사용하기 위한 패키지이고,
reactstrap은 Bootstrap 기반 UI를 React 컴포넌트로 사용할 수 있게 해 주는 라이브러리입니다.

Q7. Reactstrap의 Button을 사용하려면 무엇을 해야 하나요?

먼저 npm install --save bootstrap reactstrap으로 패키지를 설치합니다.
그다음 bootstrap/dist/css/bootstrap.min.css를 import하고,
import { Button } from "reactstrap";로 Button 컴포넌트를 가져옵니다.

Q8. <button>과 <Button>은 무엇이 다른가요?

소문자 <button>은 브라우저가 기본으로 제공하는 HTML 태그입니다.
대문자 <Button>은 reactstrap에서 import한 React 컴포넌트입니다.

Q9. Reactstrap에서 color="danger"는 무엇인가요?

color="danger"는 Reactstrap Button 컴포넌트에 전달하는 prop입니다. 이 prop에 따라 Bootstrap의 danger 스타일이 적용된 버튼이 만들어집니다.

Q10. 실제 React 프로젝트에서는 CDN 방식과 패키지 방식 중 무엇이 더 적합한가요?

빠른 테스트나 간단한 데모에는 CDN 방식이 편합니다.
하지만 실제 React 프로젝트에서는 package.json으로 의존성과 버전을 관리할 수 있고,
Vite 번들링 흐름에 포함할 수 있는 패키지 방식이 더 적합한 경우가 많습니다.

반드시 기억할 구분

CDN 방식

핵심 체크리스트

  • Bootstrap은 완성된 UI 부품 상자처럼 사용할 수 있는 프론트엔드 UI 프레임워크입니다.
  • CDN 방식은 index.html에 Bootstrap CSS와 JS 주소를 연결합니다.
  • React JSX에서는 Bootstrap 예제의 class를 className으로 바꿔야 합니다.
  • Bootstrap CSS는 디자인을 담당하고, Bootstrap JS는 동작형 UI를 담당합니다.
  • 패키지 방식에서는 bootstrap과 reactstrap을 설치합니다.
  • Bootstrap CSS는 import "bootstrap/dist/css/bootstrap.min.css";로 가져옵니다.
  • Reactstrap의 Button은 import { Button } from "reactstrap";로 가져옵니다.
  • <button>은 HTML 태그이고, <Button>은 React 컴포넌트입니다.
  • Reactstrap에서는 color="danger"처럼 prop으로 버튼 스타일을 지정할 수 있습니다.
  • 실제 React 프로젝트에서는 CDN보다 패키지 방식이 의존성 관리에 더 적합한 경우가 많습니다.

학습 메모

이번 단원은 Bootstrap 클래스 자체를 모두 외우는 단원이 아니라, React 프로젝트에서 Bootstrap을 어떤 방식으로 연결하고 사용할지를 구분하는 단원입니다.
CDN은 빠른 확인에 좋고, 패키지 방식은 React 프로젝트 구조와 의존성 관리에 더 잘 맞습니다.

오늘의 핵심 명령어와 import

npm install --save bootstrap reactstrap

import "bootstrap/dist/css/bootstrap.min.css";
import { Button } from "reactstrap";

가장 많이 틀리는 부분

Bootstrap 문서의 class를 그대로 복사하거나, Reactstrap의 Button을 import하지 않고 사용하는 실수가 많습니다.
React JSX에서는 className을 사용하고, 대문자 Button은 반드시 import해야 하는 컴포넌트임을 기억하세요.
