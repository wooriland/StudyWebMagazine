3. JSX 핵심 문법 한눈에 보기

JSX는 HTML처럼 보이지만 JavaScript 규칙을 따릅니다. 그래서 HTML에서 익숙한 문법이 JSX에서는 조금 다르게 쓰입니다.

구분 HTML / 일반 JS React JSX 이유
CSS 클래스 class="box" className="box" JSX는 JavaScript에 가까운 문법이므로 React DOM 속성명을 사용합니다.
클릭 이벤트 onclick="..." onClick={handleClick} 이벤트 속성은 camelCase로 쓰고 함수 참조를 전달합니다.
inline style style="color:red" style={{ color: "red" }} JSX style은 문자열이 아니라 JavaScript 객체입니다.
여러 태그 반환 HTML에서는 여러 태그 가능 <>...</> 컴포넌트는 하나의 최상위 요소를 반환해야 합니다.
자식 없는 태그 <img> <img /> JSX에서는 태그를 명확하게 닫아야 합니다.
리스트 출력 반복으로 HTML 작성 map() 또는 JSX 배열 배열을 JSX 목록으로 만들고 각 항목에 key를 둡니다.

JSX를 읽는 핵심 기준

JSX
→ JavaScript 안에 들어온 UI 설계도

{ }
→ JSX 안에서 JavaScript 표현식을 넣는 창문

className / onClick
→ JavaScript 스타일의 React DOM 속성명

style={{ }}
→ JSX 표현식 안에 JavaScript 객체를 넣는 구조

Fragment
→ DOM에 불필요한 요소를 추가하지 않고 여러 태그를 묶는 투명 포장지

4. 핵심 개념 설명

4-1. JSX는 HTML이 아니다

React 컴포넌트에서 반환하는 <h1>Hello React</h1> 같은 코드는 HTML처럼 보입니다. 하지만 정확히는 HTML 문서가 아니라 JavaScript 코드 안에서 UI를 표현하는 JSX입니다.

HTML 문서

<h1>Hello React</h1>

브라우저가 직접 읽는 마크업 문서 안에 작성합니다.

JSX

export default function App() {
    return (
        <h1>Hello React</h1>
    );
}

JavaScript 함수 안에서 UI 구조를 반환합니다.

4-2. JSX는 React.createElement()로 변환된다

교안에서는 JSX가 Babel에 의해 React.createElement() 호출 형태로 변환되며, 아래 두 예제는 같은 의미라고 설명합니다.

JSX 코드 
const element = (
    <h1 className="greeting">
        Hello, React!
    </h1>
);

사람이 읽기 쉬운 태그 형태의 UI 설계도입니다.

→
변환된 JavaScript 
const element = React.createElement(
    "h1",
    {
        className: "greeting"
    },
    "Hello, React!"
);

React가 이해할 수 있는 JavaScript 함수 호출 형태입니다.

핵심 이해

JSX는 브라우저에 그대로 전달되는 HTML 문자열이 아닙니다. React가 화면을 만들 수 있도록 JavaScript 코드로 변환되는 문법입니다.

4-3. JSX 태그에는 HTML 태그와 컴포넌트 태그가 있다

JSX 태그는 HTML 기본 태그일 수도 있고, 사용자가 만든 React 컴포넌트일 수도 있습니다. 소문자로 시작하면 HTML 태그로 해석되고, 대문자로 시작하면 React 컴포넌트로 해석됩니다.

소문자 태그 HTML 기본 태그 
<button>버튼</button>
<section>영역</section>
<p>문단</p>

브라우저가 아는 기본 HTML 요소로 처리됩니다.

대문자 태그 React 컴포넌트 
<App />
<Header />
<StudyCard />
<Button />

사용자가 만들거나 라이브러리에서 import한 컴포넌트로 처리됩니다.

앞 단원 연결 Reactstrap Button 
import { Button } from "reactstrap";

<Button color="danger">
    Danger
</Button>

<Button>은 HTML 태그가 아니라 import한 React 컴포넌트입니다.

이름 규칙 대문자로 시작 
function StudyBadge() {
    return <span>Badge</span>;
}

<StudyBadge />

사용자 정의 컴포넌트 이름은 대문자로 시작해야 합니다.

4-4. JSX 안에서 JavaScript를 쓰려면 {}를 사용한다

JSX 안에 JavaScript 변수, 계산식, 함수 호출 결과를 넣고 싶을 때는 중괄호 {}를 사용합니다. 이것을 JSX 표현식이라고 생각하면 됩니다.

변수 출력

const title = "JSX 완전 이해";

return <h1>{title}</h1>;

계산식 출력

const price = 10000;
const count = 3;

return <p>총 가격: {price * count}원</p>;

함수 결과 출력

function getGreeting(name) {
    return `${name}님, 환영합니다.`;
}

return <h1>{getGreeting("React")}</h1>;

배열 결과 출력

const skills = ["JSX", "Component", "Props"];

return <p>{skills.join(", ")}</p>;

4-5. JSX 속성 작성 규칙

JSX 속성값이 문자열이면 따옴표를 사용하고, JavaScript 값이면 중괄호를 사용합니다.

상황 작성 방식 예시
문자열 속성명="문자열" className="card"
변수 속성명={변수} src={imageUrl}
숫자 속성명={숫자} count={3}
boolean 속성명={true} disabled={true}
함수 속성명={함수명} onClick={handleClick}
객체 속성명={{ key: value }} style={{ color: "red" }}

4-6. JSX 속성명은 camelCase를 사용한다

JSX는 HTML보다 JavaScript에 가깝기 때문에 속성 이름도 JavaScript 스타일을 따릅니다. 그래서 class 대신 className, onclick 대신 onClick을 사용합니다.

HTML JSX
class className
onclick onClick
tabindex tabIndex
readonly readOnly
for htmlFor

4-7. JSX에서 inline style은 style={{ ... }}로 작성한다

JSX에서 inline style은 문자열이 아니라 JavaScript 객체로 작성합니다. 바깥 중괄호는 JSX 표현식이고, 안쪽 중괄호는 실제 스타일 객체입니다. 

const titleStyle = {
    color: "#ffffff",
    backgroundColor: "#164e63",
    borderRadius: "12px",
    padding: "16px",
    fontWeight: "bold"
};

export default function App() {
    return (
        <h1 style={titleStyle}>
            JSX 완전 이해
        </h1>
    );
}
CSS JSX inline style
background-color backgroundColor
font-weight fontWeight
border-radius borderRadius
margin-bottom marginBottom
text-align textAlign

4-8. JSX 주석은 {/* 주석 */}으로 작성한다

JSX 안에서는 HTML 주석 <!-- 주석 -->을 사용할 수 없습니다. JSX 안의 주석은 {/* 주석 */} 형태로 작성합니다. 

export default function App() {
    return (
        <div>
            {/* 제목 영역 */}
            <h1>JSX 완전 이해</h1>
        </div>
    );
}

5. 최상위 태그와 Fragment

React 컴포넌트가 JSX를 반환할 때는 하나의 최상위 요소가 필요합니다. 여러 태그를 나란히 반환하려면 <div>나 Fragment로 감싸야 합니다.

잘못된 예

export default function App() {
    return (
        <h1>JSX 완전 이해</h1>
        <p>JSX는 JavaScript 확장 문법입니다.</p>
    );
}

<h1>과 <p>가 하나의 최상위 요소로 묶이지 않았습니다.

<div>로 감싸기

export default function App() {
    return (
        <div>
            <h1>JSX 완전 이해</h1>
            <p>JSX는 JavaScript 확장 문법입니다.</p>
        </div>
    );
}

오류는 해결되지만 실제 DOM에 <div>가 추가됩니다.

Fragment 사용

export default function App() {
    return (
        <>
            <h1>JSX 완전 이해</h1>
            <p>JSX는 JavaScript 확장 문법입니다.</p>
        </>
    );
}

여러 요소를 묶지만 실제 DOM에는 불필요한 태그가 추가되지 않습니다.

명시적 Fragment

import { Fragment } from "react";

export default function App() {
    return (
        <Fragment>
            <h1>JSX 완전 이해</h1>
            <p>Fragment를 명시적으로 사용합니다.</p>
        </Fragment>
    );
}

짧은 문법 <></> 대신 Fragment를 import해서 사용할 수 있습니다.

방식 실제 DOM 추가 특징
<div>...</div> 추가됨 레이아웃이나 스타일에 영향을 줄 수 있습니다.
<React.Fragment>...</React.Fragment> 추가 안 됨 React 객체를 통해 명시적으로 사용합니다.
<Fragment>...</Fragment> 추가 안 됨 Fragment import가 필요합니다.
<>...</> 추가 안 됨 가장 간단한 Fragment 문법입니다.

5-1. 자식이 없는 태그는 self closing

JSX에서 자식이 없는 태그는 반드시 닫아야 합니다.

잘못된 예

<img src="/logo.png">

JSX에서는 자식이 없는 태그도 명확히 닫아야 합니다.

올바른 예

<img src="/logo.png" alt="로고" />

<Header />
<StudyBadge />

HTML 태그와 컴포넌트 모두 자식이 없다면 />로 닫습니다.

7. 변수 출력, 조건 렌더링, 리스트 렌더링

7-1. 변수와 함수 결과 출력

JSX 안에서 JavaScript 변수나 함수 결과를 출력할 때는 {}를 사용합니다.

변수 출력

const lessonTitle = "JSX 완전 이해";

return <h1>{lessonTitle}</h1>;

함수 결과 출력

function getLevel(score) {
    if (score >= 90) return "상";
    if (score >= 70) return "중";
    return "하";
}

return <p>레벨: {getLevel(85)}</p>;

7-2. 조건 렌더링

JSX 안에서는 if 문을 직접 넣기보다 삼항 연산자나 && 연산자를 사용합니다.

삼항 연산자

{isLogin ? (
    <p>로그인한 사용자입니다.</p>
) : (
    <p>로그인이 필요합니다.</p>
)}

조건이 참일 때와 거짓일 때의 화면을 모두 지정합니다.

&& 연산자

{score >= 80 && (
    <p className="success-message">
        다음 단원으로 넘어갈 준비가 되었습니다.
    </p>
)}

조건이 참일 때만 특정 JSX를 보여줍니다.

7-3. 리스트 렌더링

배열 데이터를 JSX 목록으로 출력할 때는 map()을 자주 사용합니다. 교안에는 for 루프 안에서 JSX를 배열에 넣는 예제가 나오지만, 실제 React 코드에서는 map()도 많이 사용합니다.

교안식 for 루프 예

const numbers = [1, 2, 3, 4, 5];
const listItems = [];

for (let i = 0; i < numbers.length; i++) {
    listItems.push(
        <li key={numbers[i]}>
            {numbers[i]}
        </li>
    );
}

const List = () => {
    return <ul>{listItems}</ul>;
};

map 사용 예

const skills = ["JSX", "React.createElement", "Fragment"];

return (
    <ul>
        {skills.map((skill) => (
            <li key={skill}>{skill}</li>
        ))}
    </ul>
);

리스트 렌더링 핵심

배열을 JSX 목록으로 출력할 때는 각 항목에 key가 필요합니다. key는 React가 각 항목을 구분하기 위한 이름표입니다.

Mistakes

9. 자주 하는 실수

  • 실수 1. JSX를 HTML이라고 생각함 React 함수 안에 HTML을 쓰는 것이 아니라 JSX를 씁니다. JSX는 HTML처럼 보이지만 JavaScript 확장 문법입니다.
  • 실수 2. class를 그대로 사용함 JSX에서는 class="card"가 아니라 className="card"를 사용합니다.
  • 실수 3. onclick을 그대로 사용함 JSX 이벤트 속성은 camelCase입니다. onclick이 아니라 onClick을 사용합니다.
  • 실수 4. style을 문자열로 작성함 style="color:red"가 아니라 style={{ color: "red" }}처럼 JavaScript 객체를 전달합니다.
  • 실수 5. inline style 속성을 kebab-case로 씀 JSX inline style에서는 background-color가 아니라 backgroundColor처럼 camelCase를 사용합니다.
  • 실수 6. return 안에 형제 태그를 그대로 나열함 여러 태그를 반환할 때는 하나의 최상위 태그나 Fragment로 감싸야 합니다.
  • 실수 7. return 안에서 if 문을 직접 사용함 JSX 반환부 안에는 표현식을 넣어야 합니다. 조건 렌더링에는 삼항 연산자나 &&를 주로 사용합니다.
  • 실수 8. 리스트 렌더링에서 key를 빼먹음 배열을 JSX 목록으로 출력할 때는 각 항목에 key를 지정해야 합니다.
  • 실수 9. 컴포넌트 이름을 소문자로 시작함 React 컴포넌트 이름은 대문자로 시작해야 합니다. 소문자로 시작하면 HTML 태그로 해석될 수 있습니다.
  • 실수 10. 자식이 없는 태그를 닫지 않음 JSX에서는 <img />, <Header />처럼 self closing을 사용해야 합니다.
  • 실수 11. JSX 주석을 HTML 주석처럼 작성함 JSX 주석은 {/* 주석 */} 형태로 작성합니다.
  • 실수 12. 객체를 JSX에서 그대로 출력하려고 함 객체 전체를 {user}처럼 출력하면 오류가 날 수 있습니다. {user.name}처럼 필요한 속성을 꺼내야 합니다.
Interview

10. 면접 포인트

Q1. JSX는 무엇인가요?

JSX는 JavaScript 안에서 UI 구조를 태그 형태로 작성할 수 있게 해 주는 JavaScript 확장 문법입니다.
React Element를 쉽게 만들기 위해 사용하며, HTML처럼 보이지만 HTML도 문자열도 아닙니다.

Q2. JSX는 HTML인가요?

아닙니다. JSX는 HTML처럼 보이지만 HTML이 아니라 JavaScript 확장 문법입니다. 변환 과정을 거쳐 React가 이해할 수 있는 JavaScript 객체 형태가 됩니다.

Q3. JSX는 무엇으로 변환되나요?

교안 기준으로 JSX는 Babel에 의해 React.createElement() 호출 형태로 변환됩니다.
예를 들어 <h1 className="greeting">Hello</h1>는 React.createElement("h1", { className: "greeting" }, "Hello")와 같은 의미로 이해할 수 있습니다.

Q4. JSX에서 {}는 어떤 의미인가요?

JSX 안에서 JavaScript 표현식을 삽입하는 문법입니다. 변수, 계산식, 함수 호출, 삼항 연산자, 배열의 map() 결과 등을 넣을 수 있습니다.

Q5. JSX에서 class 대신 className을 사용하는 이유는 무엇인가요?

JSX는 HTML 문자열이 아니라 JavaScript 문법에 가까운 확장 문법입니다. React DOM은 JavaScript 스타일의 속성명을 사용하므로 HTML의 class 대신 className을 사용합니다.

Q6. JSX에서 이벤트는 어떻게 작성하나요?

이벤트 속성은 onClick처럼 camelCase로 작성하고, 문자열이 아니라 {handleClick}처럼 함수 참조를 전달합니다.

Q7. JSX에서 inline style은 어떻게 작성하나요?

JSX inline style은 문자열이 아니라 JavaScript 객체로 작성합니다.
예를 들어 style={{ color: "red", backgroundColor: "black" }}처럼 쓰며, CSS 속성명은 camelCase를 사용합니다.

Q8. JSX에서 여러 태그를 반환할 때 왜 Fragment를 사용하나요?

React 컴포넌트는 하나의 최상위 요소를 반환해야 합니다.
여러 태그를 묶기 위해 <div>를 사용하면 실제 DOM에 불필요한 요소가 추가될 수 있으므로, Fragment를 사용하면 DOM 추가 없이 여러 요소를 묶을 수 있습니다.

Q9. JSX의 조건 렌더링은 어떻게 하나요?

JSX 안에서는 표현식을 사용해야 하므로 삼항 연산자나 && 연산자를 많이 사용합니다. 예를 들어 {isLogin ? <p>환영합니다</p> : <p>로그인이 필요합니다</p>}처럼 작성합니다.

Q10. JSX에서 리스트 렌더링을 할 때 key가 왜 필요한가요?

key는 React가 리스트의 각 항목을 식별하기 위한 값입니다. 배열을 렌더링할 때 안정적인 key를 주면 React가 어떤 항목이 추가, 삭제, 변경되었는지 더 정확히 추적할 수 있습니다.

Q11. JSX 안에서 if나 for를 사용할 수 없나요?

JSX를 if 문이나 for 루프 안에서 만들 수는 있습니다.
하지만 return 안의 JSX 중괄호 안에 if 문이나 for 문을 직접 넣는 방식은 사용할 수 없습니다.
JSX 반환부에는 표현식이 들어가야 하므로 삼항 연산자, &&, map()을 주로 사용합니다.

Summary

11. 이번 단원 정리

반드시 기억할 JSX 규칙

JSX
→ HTML이 아니라 JavaScript 확장 문법

JSX 변환
→ React.createElement() 형태로 이해 가능

JSX 표현식
→ {JavaScript 표현식}

HTML class
→ JSX className

HTML onclick
→ JSX onClick

HTML style 문자열
→ JSX style={{ JavaScript 객체 }}

여러 태그 반환
→ Fragment 또는 <></> 사용

자식 없는 태그
→ self closing />

조건 렌더링
→ 삼항 연산자 또는 &&

리스트 렌더링
→ map 또는 반복으로 JSX 배열 생성, key 필요

핵심 체크리스트

  • JSX는 HTML이 아니라 JavaScript 확장 문법입니다.
  • JSX는 React Element를 쉽게 만들기 위한 문법입니다.
  • JSX는 React.createElement() 형태로 변환된다고 이해할 수 있습니다.
  • JSX 안에서 JavaScript 값을 넣을 때는 {}를 사용합니다.
  • JSX에서는 class 대신 className을 사용합니다.
  • 이벤트 속성은 onClick처럼 camelCase로 작성합니다.
  • inline style은 style={{ JavaScript 객체 }} 형태로 작성합니다.
  • 여러 태그를 반환할 때는 Fragment나 최상위 태그로 감싸야 합니다.
  • 자식이 없는 태그는 />로 닫아야 합니다.
  • 조건 렌더링은 삼항 연산자나 &&를 사용합니다.
  • 리스트 렌더링에서는 각 항목에 key를 지정해야 합니다.

학습 메모

이번 단원은 React 문법의 핵심인 JSX를 이해하는 단원입니다.
JSX를 HTML로 착각하면 class, onclick, style="...", 형제 태그 반환 같은 실수를 자주 하게 됩니다. JSX는 JavaScript 안에 들어온 UI 설계도라고 생각하세요.

오늘의 핵심 문법

const title = "JSX 완전 이해";

return (
    <>
        <h1 className="title">{title}</h1>
        <button onClick={handleClick}>클릭</button>
        {isLogin ? <p>환영합니다</p> : <p>로그인이 필요합니다</p>}
    </>
);

가장 많이 틀리는 부분

JSX에서는 class가 아니라 className, onclick이 아니라 onClick, style="color:red"가 아니라 style={{ color: "red" }}를 사용합니다.
또한 여러 태그를 반환할 때는 Fragment로 감싸야 합니다.
