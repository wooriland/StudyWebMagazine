Arrow Function 화살표 함수

화살표 함수는 function 키워드 대신 =>를 사용해 함수를 더 간결하게 작성하는 ES6 문법입니다.
단순히 짧은 함수 문법이 아니라, 자신만의 this와 arguments를 가지지 않는다는 중요한 차이가 있습니다.

02

Arrow Function이란 무엇인가

화살표 함수는 ES6에서 추가된 함수 표현 문법입니다. 기존의 function 키워드 대신 => 기호를 사용합니다.

기존 함수 표현식 
const add = function (a, b) {
    return a + b;
};

console.log(add(10, 20));
// 30
화살표 함수 
const add = (a, b) => {
    return a + b;
};

console.log(add(10, 20));
// 30
한 줄 반환까지 줄인 형태
const add = (a, b) => a + b;

console.log(add(10, 20));
// 30

화살표 함수는 단순 축약형만은 아니다

03

매개변수 개수에 따른 문법 규칙

화살표 함수는 매개변수의 개수에 따라 괄호를 생략할 수 있는 경우와 반드시 써야 하는 경우가 나뉩니다.

매개변수 없음 
const sayHello = () => "Hello";

매개변수가 없으면 반드시 ()를 작성합니다.

매개변수 하나 
const square = x => x * x;

매개변수가 하나이면 괄호를 생략할 수 있습니다.

매개변수 둘 이상 
const add = (a, b) => a + b;

매개변수가 두 개 이상이면 괄호가 필요합니다.

기본값 
const greet = (name = "손님") =>
    `안녕하세요, ${name}님`;

기본값 매개변수도 사용할 수 있습니다.

잘못된 문법과 올바른 문법

잘못된 코드 
const sayHello = => "Hello";
// SyntaxError

매개변수가 없어도 빈 괄호가 필요합니다.

올바른 코드 
const sayHello = () => "Hello";

console.log(sayHello());
// Hello
잘못된 코드 
const add = a, b => a + b;
// SyntaxError

매개변수가 두 개 이상이면 괄호를 생략할 수 없습니다.

올바른 코드 
const add = (a, b) => a + b;

console.log(add(10, 20));
// 30
실무 작성 기준

매개변수가 하나일 때 괄호 생략이 가능하지만, 팀 코드 스타일에서는 일관성을 위해 (x) => x * 2처럼 항상 괄호를 쓰기도 합니다.

04

함수 본문과 반환 방식

화살표 함수는 본문을 작성하는 방식에 따라 return이 필요한 경우와 생략 가능한 경우가 나뉩니다.

Block Body

중괄호 사용

const add = (a, b) => {
    return a + b;
};

{}를 사용하면 여러 줄 로직을 작성할 수 있지만, 값을 반환하려면 return을 직접 써야 합니다.

Expression Body

표현식 자동 반환

const add = (a, b) => a + b;

{} 없이 표현식만 작성하면 그 결과가 자동으로 반환됩니다.

중괄호를 쓰면 자동 반환이 아니다

실수 
const add = (a, b) => {
    a + b;
};

console.log(add(10, 20));
// undefined

{}를 사용했지만 return이 없습니다.

해결 
const add = (a, b) => {
    return a + b;
};

console.log(add(10, 20));
// 30

객체 리터럴 바로 반환하기

화살표 함수에서 객체를 바로 반환하려면 객체 리터럴을 ()로 감싸야 합니다. 그렇지 않으면 {}가 객체가 아니라 함수 본문 블록으로 해석됩니다.

실수 
const createUser = (name, age) => {
    name: name,
    age: age
};

객체를 반환하는 것이 아니라 함수 본문으로 해석됩니다.

해결 
const createUser = (name, age) => ({
    name,
    age
});

console.log(createUser("가길동", 20));
// { name: "가길동", age: 20 }

조건이 복잡하면 여러 줄로 작성한다

읽기 쉬운 여러 줄 화살표 함수
const getStatusMessage = (status) => {
    if (status === "success") {
        return "완료되었습니다.";
    }

    if (status === "error") {
        return "오류가 발생했습니다.";
    }

    return "대기 중입니다.";
};
화살표 함수는 무조건 한 줄로 쓰는 문법이 아닙니다.

로직이 길어지거나 조건이 복잡해지면 {}와 return을 사용하는 편이 더 읽기 좋습니다.

06

화살표 함수의 핵심: Lexical this

가장 중요

화살표 함수는 자신만의 this를 만들지 않습니다. 자신이 정의된 위치의 바깥 스코프 this를 그대로 사용합니다.

일반 함수와 화살표 함수 비교

일반 함수 콜백 
const user = {
    name: "가길동",
    printLater: function () {
        setTimeout(function () {
            console.log(this.name);
        }, 1000);
    }
};

user.printLater();
// undefined 또는 오류
화살표 함수 콜백 
const user = {
    name: "가길동",
    printLater: function () {
        setTimeout(() => {
            console.log(this.name);
        }, 1000);
    }
};

user.printLater();
// 가길동

화살표 함수의 this는 bind()로 바뀌지 않는다

call, apply, bind로 this 변경 불가
const user = {
    name: "가길동"
};

const printName = () => {
    console.log(this.name);
};

printName.call(user);

const boundPrintName =
    printName.bind(user);

boundPrintName();

// 일반 함수처럼 this가 user로 바뀌지 않음
화살표 함수의 this는 호출 방식으로 정해지지 않습니다.

그래서 call(), apply(), bind()로 일반 함수처럼 this를 바꿀 수 없습니다.

객체 메서드에서 this가 필요하면 일반 함수나 메서드 축약 표현을 사용합니다.

생성자 함수

부적합

    this.name = name;
};

    new Person("가길동");

적합

DOM 이벤트에서 요소 this가 필요한 경우

주의

    this.textContent = "클릭됨";
});

화살표 함수의 this는 버튼 요소가 아닙니다.

대안

    event.currentTarget.textContent =
        "클릭됨";

화살표 함수를 사용할 때는 event.currentTarget을 사용합니다.

arguments가 필요한 경우

부적합

    console.log(arguments);
};

대안 
const printArgs = (...args) => {
    console.log(args);
};

printArgs(1, 2, 3);
// [1, 2, 3]

React와 연결되는 지점

const add = (a, b) => {
    a + b;
};

// undefined 반환

{}를 쓰면 return을 직접 작성해야 합니다.

04

객체 반환 시 ()를 생략한다

const createUser = (name) => {
    name: name
};

객체를 바로 반환하려면 ({ name })처럼 작성합니다.

05

객체 메서드를 화살표 함수로 작성한다

const user = {
    name: "가길동",
    printName: () => {
        console.log(this.name);
    }
};

화살표 함수의 this는 객체 자신이 아닙니다.

06

DOM 이벤트에서 this를 요소로 기대한다

button.addEventListener("click", () => {
    this.textContent = "클릭됨";
});

event.currentTarget을 사용하거나 일반 함수를 사용합니다.

07

화살표 함수를 생성자로 사용한다

const Person = (name) => {
    this.name = name;
};

new Person("가길동");

화살표 함수는 생성자로 사용할 수 없습니다.

08

arguments를 사용한다

const sum = () => {
    console.log(arguments);
};

Rest Parameter인 ...args를 사용합니다.

09

React 이벤트 핸들러를 즉시 실행한다

<button onClick={handleDelete(user.id)}>
    삭제
</button>

클릭할 때 실행하려면
() => handleDelete(user.id)를 전달합니다.

10

화살표 함수가 무조건 성능을 좋게 만든다고 생각한다

화살표 함수는 문법을 간결하게 할 뿐,
자동으로 성능을 개선하는 문법은 아닙니다.

11

bind()로 화살표 함수의 this가 바뀐다고 생각한다

화살표 함수의 this는 lexical하게 정해지므로
일반 함수처럼 바뀌지 않습니다.

12

너무 긴 한 줄 함수로 만든다

조건이 복잡하면 여러 줄 본문과
return을 사용하는 편이 더 읽기 좋습니다.

const add = (a, b) => a + b;

    a + b;
};

일반 함수 생성자나 class를 사용합니다.

13

면접 포인트

Q1. 화살표 함수란 무엇인가요?

ES6에서 추가된 함수 표현 문법으로,
function 키워드 대신 =>를 사용해 함수를 간결하게 작성하는 문법입니다.

Q2. 화살표 함수에서 return은 언제 생략할 수 있나요?

중괄호 없이 하나의 표현식으로 작성한 경우 자동으로 반환됩니다.
중괄호를 사용하면 return을 직접 작성해야 합니다.

Q3. 객체를 바로 반환할 때 왜 ()가 필요한가요?

{}만 쓰면 객체 리터럴이 아니라 함수 본문 블록으로 해석됩니다.
따라서 ({ name })처럼 소괄호로 감싸야 합니다.

Q4. 일반 함수와 화살표 함수의 가장 큰 차이는 무엇인가요?

가장 큰 차이는 this입니다.
일반 함수는 호출 방식에 따라 this가 결정되고,
화살표 함수는 바깥 스코프의 this를 사용합니다.

Q5. Lexical this란 무엇인가요?

화살표 함수가 자신만의 this를 만들지 않고
자신이 정의된 위치의 바깥 this를 사용하는 것을 의미합니다.

Q6. 객체 메서드에 화살표 함수를 쓰면 안 되는 이유는?

객체 리터럴은 새로운 this를 만들지 않기 때문에,
화살표 함수의 this가 객체 자신이 되지 않습니다.

Q7. 화살표 함수를 생성자로 사용할 수 있나요?

사용할 수 없습니다.
new와 함께 호출하면 TypeError: ... is not a constructor가 발생합니다.

Q8. 화살표 함수에는 arguments가 있나요?

없습니다. 가변 인자가 필요하면 ...args 형태의 Rest Parameter를 사용합니다.

Q9. DOM 이벤트에서 화살표 함수를 사용할 때 주의할 점은?

화살표 함수의 this는 이벤트 요소가 아닙니다.
요소가 필요하면 일반 함수를 쓰거나 event.currentTarget을 사용합니다.

Q10. React에서 화살표 함수는 어디에 자주 쓰이나요?

함수형 컴포넌트, 이벤트 핸들러, map() 목록 렌더링, 상태 업데이트 함수에서 자주 사용됩니다.

Q11. React에서 onClick={handleClick(id)}가 왜 문제인가요?

클릭할 때 실행되는 것이 아니라 렌더링 중에 바로 실행되기 때문입니다.
클릭 시 실행하려면 onClick={() => handleClick(id)}처럼 작성합니다.

Q12. 화살표 함수가 항상 좋은가요?

아닙니다. 짧은 콜백에는 좋지만 this가 필요한 객체 메서드, 생성자 함수,
DOM 이벤트의 요소 this가 필요한 경우에는 부적합합니다.

30초 면접 답변

화살표 함수는 ES6에서 추가된 함수 표현 문법으로, function 키워드 대신 =>를 사용해 함수를 간결하게 작성할 수 있습니다.
한 줄 표현식은 return을 생략할 수 있고, 배열의 map, filter, reduce 같은 콜백에서 자주 사용됩니다. 하지만 일반 함수와 완전히 같지는 않습니다.
화살표 함수는 자신만의 this와 arguments를 가지지 않고, 바깥 스코프의 this를 사용합니다.
그래서 객체 메서드에서 this가 필요하거나, 생성자 함수로 사용하거나, DOM 이벤트에서 요소를 this로 사용해야 할 때는 피해야 합니다.

이번 단원 최종 정리

Arrow Function

=>를 사용하는 ES6 함수 표현 문법입니다.

매개변수 없음

() => {}처럼 빈 괄호가 필요합니다.

매개변수 하나

x => x * 2처럼 괄호 생략이 가능합니다.

매개변수 둘 이상

(a, b) => a + b처럼 괄호가 필요합니다.

Expression Body

중괄호 없이 표현식을 쓰면 자동 반환됩니다.

Block Body

중괄호를 쓰면 return을 직접 작성해야 합니다.

객체 반환

({ name })처럼 소괄호로 감쌉니다.

Lexical this

자신만의 this가 없고 바깥 this를 사용합니다.

arguments 없음

가변 인자는 ...args를 사용합니다.

생성자 불가

new와 함께 사용할 수 없습니다.

좋은 사용처

짧은 콜백, 배열 메서드, Promise, React 이벤트입니다.

주의 사용처

this가 필요한 객체 메서드와 DOM 이벤트입니다.

한 문장으로 마무리

화살표 함수는 콜백과 간단한 함수 표현을 짧고 읽기 쉽게 만들어 주는 ES6 문법이지만,
자신만의 this와 arguments를 가지지 않기 때문에 객체 메서드, 생성자 함수, DOM 이벤트의 this 의존 코드에서는 일반 함수와 구분해서 사용해야 합니다.

학습 메모

화살표 함수는 “짧게 쓰는 함수”로만 외우면 위험합니다. 가장 중요한 차이는 자기 자신만의 this를 만들지 않는다는 점입니다.

문법 기준

매개변수가 없으면 (), 하나면 괄호 생략 가능, 둘 이상이면 (a, b)가 필요합니다. 중괄호를 쓰면 return도 직접 써야 합니다.

this 기준

일반 함수는 호출 방식으로 this가 결정됩니다. 화살표 함수는 호출 방식이 아니라 바깥 스코프의 this를 사용합니다.

React 기준

React 함수형 컴포넌트에서는 화살표 함수가 이벤트 핸들러, 상태 업데이트 함수, map() 콜백에서 자주 등장합니다. 단, 이벤트 핸들러를 렌더링 중 즉시 실행하지 않도록 주의해야 합니다.
