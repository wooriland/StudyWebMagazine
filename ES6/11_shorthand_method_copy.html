03

Shorthand Property

Shorthand Property는 객체를 만들 때 key 이름과 변수명이 같으면 key: key를 key 하나로 줄여 쓰는 문법입니다.

ES5 방식 
var x = 1;
var y = 2;

var obj = {
    x: x,
    y: y
};

console.log(obj);
// { x: 1, y: 2 }
ES6 Shorthand 
const x = 1;
const y = 2;

const obj = {
    x,
    y
};

console.log(obj);
// { x: 1, y: 2 }

사용자 객체 예제

key와 변수명이 같을 때
const username = "kosmo";
const email = "kosmo@example.com";
const role = "USER";

const user = {
    username,
    email,
    role
};

console.log(user);
// { username: "kosmo", email: "kosmo@example.com", role: "USER" }
핵심 조건

Shorthand Property는 key 이름과 변수명이 같을 때만 사용할 수 있습니다. 이름이 다르면 반드시 key: value 형태로 작성해야 합니다.

Shorthand와 Computed key 비교
const key = "email";
const value = "test@example.com";

    key: value

    [key]: value

Method 축약 표현

Method 축약 표현은 객체 안에 함수를 정의할 때 key: function() {} 대신 key() {} 형태로 작성하는 문법입니다.

ES5 방식 
var obj = {
    name: "가길동",
    walk: function() {
        console.log(this.name + " 걷다");
    }
};

obj.walk();
// 가길동 걷다
ES6 Method 축약 
const obj = {
    name: "가길동",
    walk() {
        console.log(this.name + " 걷다");
    }
};

obj.walk();
// 가길동 걷다

계산 객체 예제

calculator 객체
const calculator = {
    add(a, b) {
        return a + b;
    },

        return a * b;
    }
};

console.log(calculator.add(10, 20));
// 30

console.log(calculator.multiply(3, 4));
// 12

Method 축약 표현과 this

객체 메서드 안에서 this를 사용하면, 메서드를 호출한 객체를 가리킵니다.

this는 호출한 객체를 가리킴
const user = {
    username: "kosmo",

    printName() {
        console.log(this.username);
    }
};

user.printName();
// kosmo

화살표 함수 주의

주의 
const user = {
    username: "kosmo",

    printName: () => {
        console.log(this.username);
    }
};

user.printName();
// undefined

화살표 함수는 자신의 this를 만들지 않습니다.

권장 
const user = {
    username: "kosmo",

    printName() {
        console.log(this.username);
    }
};

user.printName();
// kosmo

객체 자신을 this로 사용해야 하면 Method 축약 표현을 사용합니다.

객체 복사 기초: 값 복사와 주소 복사

기본 자료형은 값이 복사된다

문자열 복사
let str1 = "코스모";
let str2 = str1;

str2 = "인재개발원";

console.log(str1);
// 코스모

console.log(str2);
// 인재개발원

문자열, 숫자, boolean 같은 기본 자료형은 값을 복사합니다. 한쪽 값을 바꿔도 다른 변수에는 영향을 주지 않습니다.

객체는 주소가 복사된다

객체 단순 대입
const obj1 = {
    x: 1,
    y: 2
};

const obj2 = obj1;

obj2.x = 10;

console.log(obj1.x);
// 10

console.log(obj2.x);
// 10
주의

const obj2 = obj1은 객체 복사가 아니라 주소 공유입니다. obj2를 수정하면 obj1도 바뀐 것처럼 보입니다.

얕은 복사와 깊은 복사

얕은 복사

얕은 복사는 겉 객체만 새로 만들고, 중첩 객체는 원본과 공유될 수 있는 복사입니다.

Spread 얕은 복사 예제
const obj1 = {
    x: 1,
    y: 2,
    z: {
        z1: 10,
        z2: 20
    }
};

const obj2 = {
    ...obj1
};

obj2.x = 100;
obj2.z.z1 = 999;

console.log(obj1.x);
// 1

console.log(obj1.z.z1);
// 999

깊은 복사

깊은 복사는 겉 객체뿐 아니라 중첩 객체까지 모두 새로 복사하는 방식입니다.

구분 의미 중첩 객체 대표 예
단순 대입 주소 공유 공유 obj2 = obj1
얕은 복사 겉 객체만 새로 복사 공유될 수 있음 { ...obj }
깊은 복사 중첩 객체까지 새로 복사 분리 deepCopy(obj)

Object.assign()으로 객체 복사

Object.assign()은 원본 객체의 속성을 대상 객체로 복사합니다. 첫 번째 인자로 빈 객체 {}를 넣으면 새 객체를 만들 수 있습니다.

Object.assign 기본 복사
const obj1 = {
    x: 1,
    y: 2
};

const obj2 = Object.assign({}, obj1);

obj2.x = 10;

console.log(obj1.x);
// 1

console.log(obj2.x);
// 10

중첩 객체에서는 얕은 복사

중첩 객체 공유
const obj1 = {
    x: 1,
    z: {
        z1: 10
    }
};

const obj2 = Object.assign({}, obj1);

obj2.z.z1 = 100;

console.log(obj1.z.z1);
// 100

console.log(obj2.z.z1);
// 100

덮어쓰기 순서

update가 뒤 
const base = {
    name: "가길동",
    age: 20
};

const update = {
    age: 21
};

const result = Object.assign({}, base, update);

console.log(result);
// { name: "가길동", age: 21 }
base가 뒤 
const result = Object.assign({}, update, base);

console.log(result);
// { age: 20, name: "가길동" }

깊은 복사 구현 방식

재귀 함수로 깊은 복사

학습용 deepCopy 함수
function deepCopy(obj) {
    if (obj === null || typeof obj !== "object") {
        return obj;
    }

    const copy = {};

    for (let key in obj) {
        copy[key] = deepCopy(obj[key]);
    }

    return copy;
}

const obj1 = {
    x: 1,
    y: 2,
    z: {
        z1: 10,
        z2: 20
    }
};

const obj2 = deepCopy(obj1);

obj2.z.z1 = 100;

console.log(obj1.z.z1);
// 10

console.log(obj2.z.z1);
// 100

    ...user
};

updatedUser.company.contacts.tel = "02-999-9999";

중첩 객체가 원본과 공유되어 원본도 바뀔 수 있습니다.

06

Object.assign()이 항상 깊은 복사라고 생각한다

const obj2 = Object.assign({}, obj1);

Object.assign()도 기본적으로 얕은 복사입니다.

07

React 상태를 직접 수정한다

user.email = "new@example.com";

setUser(user);

상태는 직접 수정하지 않고 새 객체로 업데이트해야 합니다.

08

객체 Spread 순서를 잘못 쓴다

const updatedUser = {
    email: "new@example.com",
    ...user
};

뒤의 ...user가 새 email 값을 다시 덮어쓸 수 있습니다.

16

면접 포인트

Q1. Shorthand Property란?

객체를 만들 때 key 이름과 변수명이 같으면 key: key를 key 하나로 줄여 쓰는 ES6 문법입니다.

Q2. Shorthand는 언제 사용할 수 있나요?

key 이름과 변수명이 같을 때 사용할 수 있습니다. 다르면 key: value 형태로 명시해야 합니다.

Q3. Method 축약 표현이란?

객체 안의 함수를 key: function() {} 대신 key() {} 형태로 작성하는 문법입니다.

Q4. 객체 메서드에서 화살표 함수를 주의해야 하는 이유는?

화살표 함수는 자신의 this를 가지지 않으므로,
객체 자신을 this로 참조해야 할 때는 일반 함수나 Method 축약을 사용해야 합니다.

Q5. 객체를 =로 대입하면 복사인가요?

객체 자체가 복사되는 것이 아니라 참조 주소가 복사됩니다.
그래서 두 변수가 같은 객체를 바라봅니다.

Q6. 얕은 복사란?

겉 객체만 새로 만들고, 중첩 객체는 원본과 공유될 수 있는 복사입니다.

Q7. 깊은 복사란?

겉 객체뿐 아니라 중첩 객체까지 모두 새 객체로 복사하는 것입니다.

Q8. Object.assign과 Spread의 공통점은?

둘 다 객체의 1단계 속성을 새 객체로 복사할 수 있지만, 기본적으로 얕은 복사입니다.

Q9. React에서 객체 복사가 중요한 이유는?

React 상태를 직접 수정하지 않고 새 객체로 만들어 업데이트해야
상태 변경을 예측 가능하게 관리할 수 있기 때문입니다.

Q10. 중첩 객체 상태는 어떻게 수정하나요?

수정하려는 깊이까지 Spread로 새 객체를 만들어야 합니다.

Method 축약 표현은 객체 안의 함수를 walk: function() {} 대신 walk() {}처럼 작성하는 문법입니다.
객체 복사에서는 기본 자료형과 달리 객체는 대입 시 값이 아니라 참조 주소가 복사되므로 주의해야 합니다.
Object.assign({}, obj)나 { ...obj }를 사용하면 새 객체를 만들 수 있지만 기본적으로 얕은 복사입니다.
React에서는 상태를 직접 수정하지 않고 { ...state, key: value }처럼 새 객체를 만들어 업데이트합니다.

학습 메모

이번 단원은 ES6 객체 문법과 React 상태 관리가 직접 연결되는 중요한 단원입니다.
Shorthand와 Method 축약은 객체를 간결하게 만들기 위한 문법이고, 객체 복사는 원본 데이터를 지키기 위한 개념입니다.

Shorthand 기준

{ username }은 { username: username }과 같습니다. 단, username 변수가 실제로 있어야 합니다.

Method 기준

객체 메서드는 printName() {}처럼 줄여 쓸 수 있습니다. 메서드 안에서 객체 자신을 this로 사용해야 한다면 화살표 함수보다 Method 축약 표현이 안전합니다.

복사 기준

obj2 = obj1은 복사가 아니라 주소 공유입니다. { ...obj1 }은 새 객체를 만들지만, 중첩 객체는 여전히 공유될 수 있습니다.

React 기준

상태 업데이트는 직접 수정하지 말고 setState(prev => ({ ...prev, key: value }))처럼 새 객체를 만들어 처리합니다. 중첩 상태는 수정하려는 깊이까지 Spread해야 합니다.
