02

동기와 비동기

Promise를 이해하려면 먼저 동기와 비동기를 구분해야 합니다. 동기는 앞 작업이 끝날 때까지 기다리고, 비동기는 작업이 끝나기를 기다리지 않고 다음 코드를 먼저 실행합니다.

동기 처리

순서대로 실행되는 동기 코드
function add1(param) {
    return param;
}

function add2(param) {
    return param + 2;
}

function add3(param) {
    return param + 3;
}

const result1 = add1(1);
console.log("첫번째:", result1);

const result2 = add2(result1);
console.log("두번째:", result2);

const result3 = add3(result2);
console.log("세번째:", result3);

// 첫번째: 1
// 두번째: 3
// 세번째: 6

비동기 처리

setTimeout은 나중에 실행됨
console.log("1. 시작");

setTimeout(() => {
    console.log("2. 2초 뒤 실행");
}, 2000);

console.log("3. 끝");

// 1. 시작
// 3. 끝
// 2. 2초 뒤 실행
핵심 이해

비동기 함수는 작업 완료를 기다리지 않고 다음 코드를 먼저 실행합니다. 그래서 비동기 작업의 결과는 콜백, Promise, async/await 흐름 안에서 처리해야 합니다.

비동기 결과를 바로 쓰면 생기는 문제

결과가 아직 없어서 NaN
let result2;

setTimeout(() => {
    result2 = 3;
}, 2000);

const result3 = result2 + 3;

console.log(result3);
// NaN

setTimeout() 내부는 나중에 실행됩니다. 따라서 result2 + 3이 실행되는 시점에는 result2가 아직 undefined입니다.

콜백 함수와 콜백 지옥

콜백 함수란?

콜백 함수는 다른 함수에 인자로 전달되어 나중에 실행되는 함수입니다. 비동기 작업에서는 “작업이 끝나면 이 함수를 실행해 주세요”라는 의미로 사용됩니다.

콜백 함수 기본
function greet(callback) {
    callback();
}

function sayHello() {
    console.log("Hello!");
}

greet(sayHello);
// Hello!

콜백으로 비동기 순서 제어하기

순서는 맞지만 중첩이 깊어짐
function add1(param, callback) {
    callback(param);
}

function add2(param, callback) {
    setTimeout(() => {
        callback(param + 2);
    }, 2000);
}

function add3(param, callback) {
    callback(param + 3);
}

add1(1, function(result1) {
    console.log("첫번째:", result1);

    add2(result1, function(result2) {
        console.log("두번째:", result2);

        add3(result2, function(result3) {
            console.log("세번째:", result3);
        });
    });
});

Promise란 무엇인가

Promise는 비동기 작업의 성공 또는 실패 결과를 나중에 처리하기 위한 객체입니다. 성공하면 resolve(), 실패하면 reject()를 호출합니다.

new Promise((resolve, reject) => { ... })
Promise 기본 구조
const promise = new Promise((resolve, reject) => {
    const success = true;

    if (success) {
        resolve("성공 결과");
    } else {
        reject("실패 이유");
    }
});

promise
    .then((result) => {
        console.log(result);
    })
    .catch((error) => {
        console.log(error);
    });
주문 번호표 비유

카페에서 주문하면 당장 커피를 받는 것이 아니라 주문 번호표를 받습니다.
커피가 준비되면 성공, 재료가 없으면 실패입니다. Promise도 지금 당장은 결과가 없지만 나중에 성공 또는 실패 결과를 받는 구조입니다.

05

Promise의 세 가지 상태

Promise는 pending에서 시작하고, 성공하면 fulfilled, 실패하면 rejected 상태가 됩니다.

pending

대기

아직 성공도 실패도 아닌 상태입니다.
resolve와 reject가 아직 호출되지 않았습니다.

fulfilled

성공

resolve(value)가 호출된 상태입니다.
결과는 then()에서 받습니다.

rejected

실패

reject(error)가 호출된 상태입니다.
실패 이유는 catch()에서 받습니다.

pending 예제

끝나지 않는 Promise
const promise = new Promise(() => {});

console.log(promise);
// Promise { <pending> }

fulfilled 예제

resolve 호출
const promise = new Promise((resolve) => {
    resolve("성공");
});

promise.then((result) => {
    console.log(result);
});
// 성공

rejected 예제

reject 호출
const promise = new Promise((resolve, reject) => {
    reject("에러 발생");
});

promise.catch((error) => {
    console.log(error);
});
// 에러 발생
상태는 한 번만 바뀝니다.

reject("실패")가 먼저 호출되면 뒤에 resolve("성공")을 호출해도 무시됩니다.

07

then(), catch(), finally()

then

성공 처리

Promise가 fulfilled 상태가 되었을 때 실행됩니다.

catch

실패 처리

Promise가 rejected 상태가 되었을 때 실행됩니다.

finally

마무리

성공과 실패에 관계없이 마지막에 항상 실행됩니다.

성공 흐름

then으로 성공 결과 받기
const promise = new Promise((resolve) => {
    setTimeout(() => {
        resolve("데이터 불러오기 성공");
    }, 1000);
});

promise.then((result) => {
    console.log(result);
});
// 데이터 불러오기 성공

실패 흐름

catch로 실패 이유 받기
const promise = new Promise((resolve, reject) => {
    setTimeout(() => {
        reject("데이터 불러오기 실패");
    }, 1000);
});

promise.catch((error) => {
    console.log(error);
});
// 데이터 불러오기 실패

마무리 흐름

finally는 성공/실패와 관계없이 실행
function asyncTask(success) {
    return new Promise((resolve, reject) => {
        setTimeout(() => {
            success ? resolve("완료") : reject("실패");
        }, 1000);
    });
}

asyncTask(true)
    .then((result) => {
        console.log(result);
    })
    .catch((error) => {
        console.log(error);
    })
    .finally(() => {
        console.log("무조건 실행");
    });
React 연결

React에서는 finally()를 setLoading(false)처럼 로딩 종료 처리에 자주 사용합니다.

Promise 체이닝

Promise의 강력한 점은 then()을 이어 붙여 순차적인 비동기 흐름을 만들 수 있다는 것입니다.

then에서 반환한 값은 다음 then으로 전달
Promise.resolve(10)
    .then((value) => {
        console.log("첫 번째 then:", value);
        return value + 10;
    })
    .then((value) => {
        console.log("두 번째 then:", value);
        return value * 2;
    })
    .then((value) => {
        console.log("세 번째 then:", value);
    });

// 첫 번째 then: 10
// 두 번째 then: 20
// 세 번째 then: 40

then 안에서 Promise를 return하기

정상 
delayPrint(1, 1000)
    .then(() => {
        return delayPrint(2, 500);
    })
    .then(() => {
        return delayPrint(3, 200);
    });

Promise를 return하면 다음 then이 앞 작업 완료를 기다립니다.

주의 
delayPrint(1, 1000)
    .then(() => {
        delayPrint(2, 500);
    })
    .then(() => {
        delayPrint(3, 200);
    });

return하지 않으면 다음 then이 기다릴 Promise가 없습니다.

체이닝 핵심

then() 안에서 다음 비동기 작업을 실행할 때는 그 Promise를 return해야 흐름이 정확히 이어집니다.

function getUser(username) {
    return new Promise((resolve, reject) => {
        setTimeout(() => {

                    password: "1234",
                    name: "김길동"
                };

                resolve(user);
            } catch (error) {
                reject(error);
            }
        }, 1000);
    });
}

function getPosts(username) {

                    { no: 2, title: "제목2" }
                ];

                resolve(posts);
            } catch (error) {
                reject(error);
            }
        }, 1000);
    });
}

function getComments(postNo) {

                { cno: 1, comment: "댓글1" },

            ];

            resolve(comments);
        }, 1000);
    });
}

Promise.all()과 Promise.allSettled()

Promise.all()

Promise.all()은 여러 Promise를 동시에 실행하고, 모든 Promise가 성공해야 성공합니다.

모두 성공하면 결과 배열 반환
const promise1 = Promise.resolve("A");

const promise2 = new Promise((resolve) => {
    setTimeout(() => resolve("B"), 1000);
});

const promise3 = new Promise((resolve) => {
    setTimeout(() => resolve("C"), 500);
});

Promise.all([promise1, promise2, promise3])
    .then((results) => {
        console.log(results);
    })
    .catch((error) => {
        console.log(error);
    });

// ["A", "B", "C"]
결과 순서

결과 배열은 실제 완료 순서가 아니라 Promise.all()에 넣은 배열 순서대로 나옵니다.

하나라도 실패하면 전체 실패

Promise.all 실패 예
Promise.all([
    Promise.resolve("성공1"),
    Promise.reject("실패2"),
    Promise.resolve("성공3")
])
    .then((results) => {
        console.log(results);
    })
    .catch((error) => {
        console.log(error);
    });

// 실패2

Promise.allSettled()

Promise.allSettled()는 성공과 실패에 관계없이 모든 Promise가 끝날 때까지 기다린 뒤, 각 결과를 배열로 반환합니다.

성공/실패 결과를 모두 받기
Promise.allSettled([
    Promise.resolve("성공1"),
    Promise.reject("실패2"),
    Promise.resolve("성공3")
])
    .then((results) => {
        console.log(results);
    });

/*
[
    { status: "fulfilled", value: "성공1" },
    { status: "rejected", reason: "실패2" },
    { status: "fulfilled", value: "성공3" }
]
*/
메서드 하나 실패하면? 결과 사용 상황
Promise.all() 바로 실패 처리 catch() 모든 데이터가 반드시 필요할 때
Promise.allSettled() 그래도 모두 기다림 성공·실패 결과 배열 일부 실패도 결과로 확인해야 할 때

Promise.any()와 Promise.race()

Promise.any()

Promise.any()는 여러 Promise 중 가장 먼저 성공한 결과를 사용합니다. 실패한 Promise는 무시하고, 모두 실패한 경우에만 catch()로 이동합니다.

가장 먼저 성공한 결과
const promise1 = new Promise((resolve) => {
    setTimeout(() => resolve("첫번째 성공"), 2000);
});

const promise2 = new Promise((resolve) => {
    setTimeout(() => resolve("두번째 성공"), 3000);
});

const promise3 = new Promise((resolve) => {
    setTimeout(() => resolve("세번째 성공"), 1000);
});

Promise.any([promise1, promise2, promise3])
    .then((result) => {
        console.log(result);
    })
    .catch((error) => {
        console.log(error);
    });

// 세번째 성공

Promise.race()

Promise.race()는 성공이든 실패든 가장 먼저 끝난 Promise의 결과를 사용합니다.

가장 먼저 끝난 Promise가 실패한 경우
const promise1 = new Promise((resolve) => {
    setTimeout(() => resolve("첫번째 성공"), 2000);
});

const promise2 = new Promise((resolve, reject) => {
    setTimeout(() => reject("두번째 실패"), 1000);
});

Promise.race([promise1, promise2])
    .then((result) => {
        console.log("성공:", result);
    })
    .catch((error) => {
        console.log("실패:", error);
    });

// 실패: 두번째 실패

timeout 처리 예제

fetch와 timeout 경쟁
function timeout(ms) {
    return new Promise((resolve, reject) => {
        setTimeout(() => {
            reject("요청 시간이 초과되었습니다.");
        }, ms);
    });
}

Promise.race([
    fetch("/api/users"),
    timeout(3000)
])
    .then((response) => {
        return response.json();
    })
    .then((data) => {
        console.log(data);
    })
    .catch((error) => {
        console.log(error);
    });
메서드 기준 실패 처리
Promise.any() 가장 먼저 성공한 Promise 모두 실패해야 catch
Promise.race() 가장 먼저 끝난 Promise 가장 먼저 끝난 것이 실패면 catch

React와 연결되는 지점

React에서 Promise는 서버 통신과 거의 항상 함께 등장합니다. Promise 상태는 React의 data, loading, error 상태로 연결됩니다.

pending loading

요청 중

fulfilled data

성공 데이터 저장

rejected error

실패 메시지 저장

finally loading false

로딩 종료

서버 데이터 불러오기

UserList.jsx
import { useEffect, useState } from "react";

function UserList() {
    const [users, setUsers] = useState([]);
    const [loading, setLoading] = useState(false);
    const [error, setError] = useState("");

    useEffect(() => {
        setLoading(true);

        fetch("/api/users")
            .then((response) => {
                if (!response.ok) {
                    return Promise.reject("사용자 목록 조회 실패");
                }

                return response.json();
            })
            .then((data) => {
                setUsers(data);
            })
            .catch((error) => {
                setError(String(error));
            })
            .finally(() => {
                setLoading(false);
            });
    }, []);

    if (loading) {
        return <p>로딩 중...</p>;
    }

    if (error) {
        return <p>{error}</p>;
    }

    return (
        <ul>
            {users.map((user) => (
                <li key={user.id}>
                    {user.username}
                </li>
            ))}
        </ul>
    );
}

export default UserList;

자주 하는 실수

01

비동기 결과를 일반 변수처럼 바로 사용한다

let result;

    result = 10;
}, 1000);

비동기 결과는 나중에 나오므로 Promise 흐름 안에서 처리해야 합니다.

02

resolve나 reject를 호출하지 않는다

const promise = new Promise((resolve, reject) => {
    const data = "success";
});

이 Promise는 계속 pending 상태로 남습니다.

03

then 안에서 Promise를 return하지 않는다

getUser("kim")
    .then((user) => {
        getPosts(user.username);
    })
    .then((posts) => {
        console.log(posts);
    });

다음 then이 getPosts의 완료를 기다리지 못합니다.

04

catch를 작성하지 않는다

fetch("/api/users")
    .then((response) => response.json())
    .then((data) => {
        console.log(data);
    });

실패했을 때 에러를 처리할 수 없습니다.

05

fetch가 404, 500을 자동으로 catch로 보낸다고 생각한다

fetch("/api/wrong-url")
    .then((response) => response.json())
    .catch((error) => {
        console.log(error);
    });

HTTP 상태 실패는 response.ok를 확인하는 것이 안전합니다.

06

Promise.all에서 일부 실패 결과도 받을 수 있다고 생각한다

Promise.all([
    Promise.resolve("A"),
    Promise.reject("B 실패"),
    Promise.resolve("C")
]);

하나라도 실패하면 전체가 실패합니다. 일부 실패도 보고 싶으면 allSettled를 사용합니다.

07

Promise.any와 Promise.race를 헷갈린다

Promise.any(promises);
Promise.race(promises);

any는 가장 먼저 성공한 것, race는 가장 먼저 끝난 것입니다.

08

finally에서 성공 결과를 처리하려고 한다

fetchData()
    .finally((data) => {
        console.log(data);
    });

성공 결과는 then에서 처리하고, finally는 마무리 작업에 사용합니다.

면접 포인트

Q1. Promise란?

비동기 작업의 미래 결과를 표현하는 ES6 표준 객체입니다.
성공하면 resolve(), 실패하면 reject()를 호출합니다.

Q2. Promise의 상태는?

pending, fulfilled, rejected 세 가지입니다.

Q3. resolve와 reject의 차이는?

resolve는 성공 결과를 then()으로, reject는 실패 이유를 catch()로 전달합니다.

Q4. then, catch, finally의 역할은?

then()은 성공 처리, catch()는 실패 처리, finally()는 마무리 처리입니다.

Q5. Promise가 콜백 지옥을 어떻게 개선하나요?

중첩 콜백 대신 then() 체이닝을 사용해 비동기 흐름을 위에서 아래로 읽게 해 줍니다.

Q6. then 안에서 Promise를 return해야 하는 이유는?

다음 then()이 앞의 비동기 작업 완료를 기다리게 하기 위해서입니다.

Q7. Promise.all과 allSettled의 차이는?

all()은 모두 성공해야 성공하고, 하나라도 실패하면 catch로 갑니다.
allSettled()는 성공·실패와 관계없이 모두 완료 결과를 반환합니다.

Q8. Promise.any와 race의 차이는?

any()는 가장 먼저 성공한 결과를 사용하고,
race()는 성공이든 실패든 가장 먼저 끝난 결과를 사용합니다.

Q9. fetch와 Promise의 관계는?

fetch()는 Promise를 반환하는 브라우저 API입니다. 응답을 받은 뒤 then()에서 처리합니다.

Q10. React에서 Promise는 어디에 쓰이나요?

서버 요청, 로그인, 목록 조회, 로딩 상태, 에러 상태, 성공 데이터 관리에 사용됩니다.

30초 면접 답변

Promise는 비동기 작업의 미래 결과를 표현하는 ES6 표준 객체입니다.
Promise는 처음에는 pending 상태이고, 작업이 성공하면 resolve()가 호출되어 fulfilled 상태가 되며 결과는 then()에서 처리합니다.
실패하면 reject()가 호출되어 rejected 상태가 되고 에러는 catch()에서 처리합니다.
finally()는 성공과 실패에 관계없이 마지막에 실행됩니다. React에서는 서버 요청, 로딩 상태, 에러 상태, 성공 데이터를 관리할 때 Promise가 자주 사용됩니다.

이번 단원 최종 정리

Promise

비동기 작업의 미래 결과를
표현하는 객체입니다.

pending

아직 성공도 실패도 아닌 대기 상태입니다.

fulfilled

resolve()가 호출된 성공 상태입니다.

rejected

reject()가 호출된 실패 상태입니다.

then

성공 결과를 처리합니다.

catch

실패 이유를 처리합니다.

finally

성공·실패와 관계없이 마무리합니다.

executor

Promise 생성 시 즉시 실행되는 함수입니다.

체이닝

then()을 이어 붙여 순차 흐름을 만듭니다.

fetch

Promise를 반환하는 브라우저 API입니다.

Promise.all

모두 성공해야 성공합니다.

React

data, loading, error 상태와 연결됩니다.

한 문장으로 마무리

Promise는 비동기 작업의 성공 또는 실패 결과를 나중에 처리하기 위한 ES6 표준 객체이며, pending, fulfilled, rejected 상태를 가집니다.
성공은 then(), 실패는 catch(), 마무리는 finally()로 처리하고, React에서는 서버 요청, 로딩 상태, 에러 상태, 성공 데이터를 체계적으로 관리하는 데 사용됩니다.

학습 메모

Promise는 다음 단원인 async / await를 이해하기 위한 핵심 기초입니다.
async/await는 Promise를 더 동기 코드처럼 읽기 쉽게 만든 문법이므로, 먼저 Promise의 상태와 then(), catch(), finally() 흐름을 정확히 이해해야 합니다.

상태 기준

Promise는 pending에서 시작하고, 성공하면 fulfilled, 실패하면 rejected가 됩니다. 상태는 한 번만 바뀝니다.

처리 기준

성공 결과는 then(), 실패 이유는 catch(), 성공·실패와 관계없는 정리 작업은 finally()에서 처리합니다.

체이닝 기준

then() 안에서 다음 Promise를 실행할 때는 반드시 return해야 다음 then()이 기다릴 수 있습니다.

React 기준

Promise의 pending은 loading, fulfilled는 data, rejected는 error 상태와 연결해서 생각하면 이해하기 쉽습니다.
