왜 구조분해 할당을 사용하는가

구조분해 할당은 코드를 짧게 만드는 문법이기도 하지만, 더 중요한 목적은 필요한 데이터가 무엇인지 명확하게 보여 주는 것입니다.

기존 방식 
const user = {
    id: 1,
    username: "kosmo",
    email: "kosmo@example.com",
    role: "USER"
};

const username = user.username;
const email = user.email;

console.log(username, email);
구조분해 방식 
const user = {
    id: 1,
    username: "kosmo",
    email: "kosmo@example.com",
    role: "USER"
};

const { username, email } = user;

console.log(username, email);

03

배열 구조분해 기본

배열 구조분해는 []를 사용하고, 배열의 값을 순서대로 변수에 할당합니다.

배열 기본 구조분해
const arr = [1, 2, 3];

const [a, b, c] = arr;

console.log(a);
// 1

console.log(b);
// 2

console.log(c);
// 3

배열은 순서가 중요하다

순서 기준 할당
const userInfo = ["kosmo", "kosmo@example.com", "USER"];

const [username, email, role] = userInfo;

console.log(username);
// kosmo

console.log(email);
// kosmo@example.com

console.log(role);
// USER

값이 부족하면 undefined

오른쪽 배열에 값이 부족한 경우
const [a, b] = [1];

console.log(a);
// 1

console.log(b);
// undefined

값이 많으면 남는 값은 무시

받을 변수가 없는 값은 무시
const [a, b] = [1, 2, 3, 4];

console.log(a);
// 1

console.log(b);
// 2
04

배열 구조분해 활용 패턴

일부 값 건너뛰기

쉼표를 사용하면 특정 위치의 값을 건너뛸 수 있습니다.

중간 값 무시
const numbers = [1, 2, 3];

const [a, , c] = numbers;

console.log(a);
// 1

console.log(c);
// 3

배열 기본값

값이 없을 때 
const [a, b, c = 3] = [1, 2];

console.log(a);
// 1

console.log(b);
// 2

console.log(c);
// 3
null은 기본값 적용 안 됨 
const [value = "기본값"] = [null];

console.log(value);
// null
기본값 적용 기준

구조분해 기본값은 값이 undefined일 때만 적용됩니다. null은 실제 값으로 취급됩니다.

변수 값 교환

임시 변수 없이 값 교환
let a = 1;
let b = 3;

[a, b] = [b, a];

console.log(a);
// 3

console.log(b);
// 1

함수가 반환한 배열 분해

반환 배열을 바로 구조분해
function getPoint() {
    return [10, 20];
}

const [x, y] = getPoint();

console.log(x);
// 10

console.log(y);
// 20

배열 Rest 구조분해

정상 
const numbers = [1, 2, 3, 4, 5];

const [first, second, ...rest] = numbers;

console.log(first);
// 1

console.log(rest);
// [3, 4, 5]
오류 
const [first, ...rest, last] = [1, 2, 3, 4];

// SyntaxError:
// Rest element must be last element
05

객체 구조분해 기본

객체 구조분해는 {}를 사용하고, 배열과 다르게 key 이름을 기준으로 값을 꺼냅니다.

객체 기본 구조분해
const user = {
    username: "kosmo",
    email: "kosmo@example.com",
    role: "USER"
};

const { username, email } = user;

console.log(username);
// kosmo

console.log(email);
// kosmo@example.com

객체 구조분해는 순서가 중요하지 않다

순서가 달라도 key 이름으로 찾음
const user = {
    username: "kosmo",
    email: "kosmo@example.com",
    role: "USER"
};

const { role, username } = user;

console.log(role);
// USER

console.log(username);
// kosmo

key 이름이 다르면 undefined

존재하지 않는 key 구조분해
const user = {
    username: "kosmo",
    email: "kosmo@example.com"
};

const { name } = user;

console.log(name);
// undefined

기존 방식과 비교

기존 방식 
const obj = {
    name: "kosmo",
    location: "가산동",
    contacts: "010"
};

const name = obj.name;
const location = obj["location"];
const contacts = obj.contacts;
구조분해 방식 
const obj = {
    name: "kosmo",
    location: "가산동",
    contacts: "010"
};

const { name, location, contacts } = obj;
06

객체 구조분해: 변수명 변경과 기본값

변수명 변경

객체의 key 이름과 다른 변수명으로 값을 받고 싶을 때는 key: 새변수명 형태를 사용합니다.

username 값을 userId 변수로 받기
const user = {
    username: "kosmo",
    email: "kosmo@example.com"
};

const { username: userId, email: userEmail } = user;

console.log(userId);
// kosmo

console.log(userEmail);
// kosmo@example.com
주의

{ username: userId }는 username 변수를 만드는 문법이 아닙니다. username key의 값을 userId 변수에 저장하는 문법입니다.

객체 기본값

key가 없을 때 기본값 사용
const user = {
    username: "kosmo"
};

const { username, role = "USER" } = user;

console.log(username);
// kosmo

console.log(role);
// USER

변수명 변경 + 기본값

새 변수명과 기본값 함께 사용
const obj = {
    a: 3
};

const { a: aa = 10, b: bb = 5 } = obj;

console.log(aa);
// 3

console.log(bb);
// 5
key 그대로 받기 const { username } = user
변수명 변경 const { username: userId } =
user
기본값 const { role = "USER" } = user
변경 + 기본값 const { role: userRole =
"USER" } = user
07

변수 선언 후 객체 구조분해 재할당

변수를 먼저 선언한 뒤 객체 구조분해로 값을 할당할 때는 전체 할당문을 괄호로 감싸야 합니다.

오류 
let a;
let b;

{ a, b } = { a: 1, b: 2 };

JavaScript가 { a, b }를 객체 구조분해가 아니라 블록으로 해석할 수 있습니다.

정상 
let a;
let b;

({ a, b } = { a: 1, b: 2 });

console.log(a);
// 1

console.log(b);
// 2

괄호로 감싸면 객체 구조분해 할당식으로 해석됩니다.

기억법

const { a, b } = obj처럼 선언과 동시에 할당할 때는 괄호가 필요 없습니다.
하지만 let a, b를 먼저 선언하고 나중에 객체 구조분해로 재할당할 때는 ({ a, b } = obj)처럼 괄호가 필요합니다.

08

중첩 구조분해

객체 안 객체, 객체 안 배열, 배열 안 객체처럼 데이터가 중첩되어 있을 때도 구조분해를 사용할 수 있습니다.

중첩 객체 구조분해

profile 안의 name과 email 꺼내기
const user = {
    id: 1,
    profile: {
        name: "가길동",
        email: "gil@example.com"
    }
};

const {
    profile: {
        name,
        email
    }
} = user;

console.log(name);
// 가길동

console.log(email);
// gil@example.com
중간 객체 변수는 자동 생성되지 않습니다.

위 코드에서는 name과 email만 생성됩니다. profile 변수도 필요하면 따로 꺼내야 합니다.

profile 자체도 함께 꺼내기
const {
    profile,
    profile: {
        name,
        email
    }
} = user;

console.log(profile);
// { name: "가길동", email: "gil@example.com" }

객체 안 배열 구조분해

metadata.translations[0].title 꺼내기
const metadata = {
    title: "Scratchpad",
    translations: [
        {
            locale: "de",
            title: "JavaScript-Umgebung"
        }
    ],
    url: "/en-US/docs/Tools/Scratchpad"
};

const {
    title: englishTitle,
    translations: [
        {
            title: localeTitle
        }
    ]
} = metadata;

console.log(englishTitle);
// Scratchpad

console.log(localeTitle);
// JavaScript-Umgebung

배열 안 객체 구조분해

첫 번째 객체의 username 꺼내기
const users = [
    { id: 1, username: "kosmo" },
    { id: 2, username: "gildong" }
];

const [{ username }] = users;

console.log(username);
// kosmo

for...of와 구조분해

배열 안의 객체를 순회할 때 구조분해를 함께 사용하면 반복문 안에서 필요한 값만 바로 꺼낼 수 있습니다.

for...of에서 중첩 객체 구조분해
const people = [
    {
        name: "Mike Smith",
        family: {
            father: "Harry Smith"
        },
        age: 35
    },
    {
        name: "Tom Jones",
        family: {
            father: "Richard Jones"
        },
        age: 25
    }
];

for (const { name, family: { father } } of people) {
    console.log(`${name}의 아버지는 ${father}입니다`);
}

// Mike Smith의 아버지는 Harry Smith입니다
// Tom Jones의 아버지는 Richard Jones입니다
기존 방식 
for (const person of people) {
    const name = person.name;
    const father = person.family.father;

    console.log(`${name}의 아버지는 ${father}입니다`);
}
구조분해 방식 
for (const { name, family: { father } } of people) {
    console.log(`${name}의 아버지는 ${father}입니다`);
}

함수 파라미터 구조분해

함수에 객체를 전달할 때 함수 내부에서 다시 값을 꺼내지 않고, 파라미터 자리에서 필요한 값만 바로 꺼낼 수 있습니다.

중첩 객체 파라미터 구조분해

fullName 안쪽 값 꺼내기
const getFullName = ({
    displayName,
    fullName: {
        firstName,
        lastName
    }
}) => {
    console.log(`${displayName}닉은 ${firstName} ${lastName}입니다`);
};

const user = {
    id: 42,
    displayName: "jdoe",
    fullName: {
        firstName: "John",
        lastName: "Doe"
    }
};

함수 파라미터 기본값

옵션 객체 기본값 처리
function drawChart({
    size = "big",
    coords = { x: 0, y: 0 },
    radius = 25
} = {}) {
    console.log(size, coords, radius);
}

drawChart({
    coords: { x: 18, y: 30 },
    radius: 30
});

// big { x: 18, y: 30 } 30

drawChart();
// big { x: 0, y: 0 } 25
= {}가 중요한 이유

함수 호출 시 인자를 전달하지 않으면 파라미터 값은 undefined입니다. 이 상태에서 객체 구조분해를 시도하면 오류가 발생할 수 있으므로 = {}로 기본 객체를 지정합니다.

Computed Property Name과 Rest 구조분해

Computed Property Name과 구조분해

객체 구조분해에서도 변수에 저장된 key로 값을 꺼낼 수 있습니다.

오해 
let key = "z";

let { key: foo } = {
    z: "bar"
};

console.log(foo);
// undefined

key라는 고정 속성을 찾기 때문에 undefined입니다.

정상 
let key = "z";

let { [key]: foo } = {
    z: "bar"
};

console.log(foo);
// bar

변수 key의 값인 z 속성을 찾습니다.

배열 Rest 구조분해

남은 배열 요소 모으기
const numbers = [1, 2, 3, 4, 5];

const [first, ...rest] = numbers;

console.log(first);
// 1

console.log(rest);
// [2, 3, 4, 5]

객체 Rest 구조분해

특정 key를 제외한 나머지 모으기
const user = {
    id: 1,
    username: "kosmo",
    email: "kosmo@example.com",
    role: "USER"
};

const { id, ...userInfo } = user;

console.log(id);
// 1

console.log(userInfo);
// { username: "kosmo", email: "kosmo@example.com", role: "USER" }

    );
}
useState const [state, setState]
props function Card({ title })
event const { name, value }
API const { data } = response

자주 하는 실수

01

배열 구조분해와 객체 구조분해를 헷갈린다

const { a, b } = [1, 2];

배열은 [], 객체는 {}를 사용해야 합니다.

02

객체 key 이름을 틀린다

const user = {
    username: "kosmo"
};

const { name } = user;

console.log(name);
// undefined

객체 구조분해는 key 이름이 정확히 일치해야 합니다.

03

변수명 변경 문법을 오해한다

const { username: userId } = user;

console.log(username);
// ReferenceError

생성되는 변수는 userId입니다.

04

객체 재할당에서 괄호를 빼먹는다

let a;
let b;

{ a, b } = { a: 1, b: 2 };

({ a, b } = obj)처럼 괄호가 필요합니다.

05

함수 파라미터 구조분해에서 기본 객체를 빼먹는다

function printUser({ username }) {
    console.log(username);
}

printUser();

function printUser({ username } = {})처럼 기본 객체를 지정해야 안전합니다.

06

중첩 구조분해에서 중간 객체 변수도 생긴다고 착각한다

const {
    profile: {
        name
    }
} = user;

console.log(profile);
// ReferenceError

profile도 필요하면 따로 꺼내야 합니다.

07

Rest 요소를 마지막에 두지 않는다

const [first, ...rest, last] = arr;

Rest 요소는 항상 마지막에 와야 합니다.

08

기본값이 null에도 적용된다고 생각한다

const { role = "USER" } = {
    role: null
};

console.log(role);
// null

기본값은 undefined일 때만 적용됩니다.

09

너무 깊은 구조분해를 한 줄로 작성한다

const { a: { b: { c: { d } } } } = obj;

가능은 하지만 가독성이 떨어지면 단계별로 나누는 것이 좋습니다.

10

React props가 undefined일 수 있는데 바로 중첩 구조분해한다

function UserCard({ user: { username } }) {
    return <p>{username}</p>;
}

user가 없을 수 있다면 기본값이나 조건 처리가 필요합니다.

면접 포인트

Q1. 구조분해 할당이란?

배열이나 객체 안의 값을 꺼내 변수에 바로 할당하는 ES6 문법입니다.

Q2. 배열 구조분해와 객체 구조분해의 차이는?

배열 구조분해는 순서를 기준으로 값을 꺼내고,
객체 구조분해는 key 이름을 기준으로 값을 꺼냅니다.

Q3. 배열에서 값을 건너뛰려면?

쉼표를 사용합니다. const [a, , c] = [1, 2, 3]처럼 작성합니다.

Q4. 기본값은 언제 사용되나요?

구조분해 대상 값이 undefined일 때만 기본값이 사용됩니다. null에는 적용되지 않습니다.

Q5. 객체 구조분해에서 변수명을 바꾸려면?

const { username: userId } = user처럼 key: 새변수명 형식을 사용합니다.

Q6. 객체 구조분해 재할당에서 괄호가 필요한 이유는?

괄호가 없으면 JavaScript가 {}를 객체 구조분해가 아니라 블록으로 해석할 수 있기 때문입니다.

Q7. 함수 파라미터 구조분해란?

함수 인자로 받은 객체에서 필요한 값을 파라미터 자리에서 바로 꺼내는 방식입니다.

Q8. 파라미터 구조분해에서 = {}는 왜 필요한가요?

인자를 전달하지 않으면 undefined에서 구조분해를 시도하게 되어
오류가 발생할 수 있기 때문입니다.

Q9. 중첩 구조분해의 단점은?

너무 깊게 작성하면 가독성이 떨어지고 중간 객체가 없을 때 오류가 발생할 수 있습니다.

Q10. React에서 어디에 많이 사용되나요?

useState, props, event 객체, API 응답, 배열 map(), rest props 처리에서 자주 사용됩니다.

30초 면접 답변

구조분해 할당은 배열이나 객체 안의 값을 꺼내 변수에 바로 할당하는 ES6 문법입니다.
배열 구조분해는 순서를 기준으로 값을 꺼내고, 객체 구조분해는 key 이름을 기준으로 값을 꺼냅니다.
예를 들어 const [a, b] = [1, 2]는 배열의 첫 번째, 두 번째 값을 각각 변수에 담고, const { username, email } = user는 객체에서 해당 key 값을 꺼냅니다.
기본값을 지정할 수도 있고, 객체 구조분해에서는 username: userId처럼 변수명을 바꿀 수도 있습니다. React에서는 useState, props, event, API 응답 처리에서 자주 사용됩니다.

한 문장으로 마무리

Destructuring Assignment는 배열과 객체 안의 값을 필요한 변수에 바로 꺼내 담는 ES6 문법이며, 배열은 순서 기준으로, 객체는 key 기준으로 값을 할당합니다.
기본값, 변수명 변경, Rest, 중첩 구조분해, 함수 파라미터 구조분해까지 활용하면
JavaScript와 React 코드에서 반복되는 접근 코드를 줄이고 필요한 데이터의 의도를 더 명확하게 표현할 수 있습니다.

학습 메모

구조분해 할당은 React를 공부하면서 가장 자주 마주치는 ES6 문법 중 하나입니다.
처음에는 문법 모양이 복잡해 보이지만, “배열은 순서, 객체는 key”라는 기준만 잡으면 대부분의 코드를 읽을 수 있습니다.

배열 기준

배열 구조분해는 순서가 중요합니다. const [a, b] = [10, 20]에서 a는 첫 번째 값, b는 두 번째 값을 받습니다.

객체 기준

객체 구조분해는 key 이름이 중요합니다. const { username } = user는 user.username 값을 꺼내는 문법입니다.

변수명 변경 기준

const { username: userId } = user에서 실제로 만들어지는 변수는 userId입니다. username 변수는 만들어지지 않습니다.

React 기준

useState는 배열 구조분해, props와 event는 객체 구조분해, rest props는 객체 Rest 구조분해와 연결됩니다.
