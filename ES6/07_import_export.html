01

이번 단원의 핵심

핵심 문장

export는 현재 파일의 코드를 다른 파일에서 사용할 수 있도록 내보내는 문법이고, import는 다른 파일이 내보낸 코드를 현재 파일로 가져오는 문법입니다.

필요한 파일에서 필요한 코드만 가져옵니다. 이때 사용하는 핵심 문법이 import와 export입니다.

03

모듈 기본: export와 import

모듈은 기능 단위로 분리된 JavaScript 파일입니다. 현재 파일의 코드를 다른 파일에서 쓰게 하려면 export하고, 다른 파일의 코드를 현재 파일에서 쓰려면 import합니다.

utils/format.js export function formatPrice
→
ProductCard.jsx import { formatPrice }
내보내는 파일 
// utils/format.js

export function formatPrice(price) {
    return `${price.toLocaleString("ko-KR")}원`;
}
가져오는 파일 
// components/ProductCard.jsx

import { formatPrice } from "../utils/format.js";

console.log(formatPrice(30000));
// 30,000원

모듈 스코프

모듈 파일 안에서 선언한 변수는 자동으로 전역 변수가 되지 않습니다. 다른 파일에서 사용하려면 반드시 export와 import가 필요합니다.

공유되지 않음 
// user.js
const username = "가길동";

// app.js
console.log(username);
// ReferenceError
명시적 공유 
// user.js
export const username = "가길동";

// app.js
import { username } from "./user.js";

console.log(username);
// 가길동
04

CommonJS와 ES Module

교안에서는 CommonJS 방식과 ES6 모듈 방식을 비교합니다.
CommonJS에서는 모듈을 가져올 때 require()를 사용하고, 내보낼 때 module.exports나 exports.함수명 형태를 사용합니다. ES6에서는 import와 export를 사용합니다.

CommonJS 
// math.js
function add(a, b) {
    return a + b;
}

module.exports = {
    add
};

// app.js
const { add } = require("./math");

console.log(add(10, 20));
ES Module 
// math.js
export function add(a, b) {
    return a + b;
}

// app.js
import { add } from "./math.js";

console.log(add(10, 20));
구분 CommonJS ES Module
가져오기 require() import
내보내기 module.exports export
주 사용 환경 Node.js 구형 방식 브라우저, React, Vite, Webpack
문법 위치 동적으로 호출 가능 일반적으로 파일 상단에 정적으로 작성
05

Named Export와 Named Import

named export는 이름이 있는 값을 내보내는 방식입니다. 한 파일에서 여러 개를 내보낼 수 있고, 가져올 때는 {} 안에 정확한 이름을 적어야 합니다.

선언과 동시에 named export
// math.js

export function add(a, b) {
    return a + b;
}

export function multiply(a, b) {
    return a * b;
}
named import
// app.js

import { add, multiply } from "./math.js";

console.log(add(10, 20));
// 30

console.log(multiply(3, 4));
// 12

선언 후 모아서 export

파일 마지막에서 export 목록 정리
const username = "가길동";

function add(a, b) {
    return a + b;
}

function multiply(a, b) {
    return a * b;
}

export { username, add, multiply };

잘못된 named export

오류 
const username = "가길동";

export username;
// SyntaxError
정상 
const username = "가길동";

export { username };

// 또는
export const username = "가길동";
Named Export
add multiply username

여러 값을 이름표와 함께 내보내는 방식입니다.

06

별칭 Import: as

named import는 export한 이름과 import 이름이 일치해야 합니다. 하지만 as를 사용하면 현재 파일에서 사용할 이름을 바꿀 수 있습니다.

add를 sum이라는 이름으로 사용
// math.js
export function add(a, b) {
    return a + b;
}

// app.js
import { add as sum } from "./math.js";

console.log(sum(10, 20));
// 30

이름 충돌을 피할 때

서로 다른 파일의 같은 이름
import { format as formatDate } from "./dateUtil.js";
import { format as formatPrice } from "./priceUtil.js";

console.log(formatDate(new Date()));
console.log(formatPrice(30000));

더 명확한 이름으로 바꾸고 싶을 때

get을 의미 있는 이름으로 변경
import { get as getUser } from "./userApi.js";
import { get as getProduct } from "./productApi.js";

getUser(1);
getProduct(100);
별칭 사용 기준

이름이 충돌하거나 현재 파일에서 더 구체적인 이름이 필요할 때 as를 사용합니다.

07

Default Export와 Default Import

default export는 파일의 대표값 하나를 내보내는 방식입니다. 한 파일에서 한 번만 사용할 수 있고, 가져올 때는 {}를 사용하지 않습니다.

default export
// Button.jsx

function Button() {
    return <button>클릭</button>;
}

export default Button;
default import
// App.jsx

import Button from "./Button.jsx";

function App() {
    return <Button />;
}

default export 작성 방식

함수 선언에 바로 붙이기 
export default function add(a, b) {
    return a + b;
}
class 선언에 바로 붙이기 
export default class User {
    constructor(name) {
        this.name = name;
    }
}
선언 후 내보내기 
function add(a, b) {
    return a + b;
}

export default add;

잘못된 default export

오류 
export default const username = "가길동";
// SyntaxError
정상 
const username = "가길동";

export default username;

// 또는
export const username = "가길동";
default export는 한 파일에 하나만 가능합니다.

여러 값을 내보내야 한다면 named export를 사용하거나, 객체 하나로 묶어 default export할 수 있습니다.

08

Named Export와 Default Export 비교

구분 Named Export Default Export
export 개수 여러 개 가능 한 파일에 하나만 가능
export 방식 export const name = ... export default value
import 방식 import { name } from "./file.js" import name from "./file.js"
{} 사용 필요 사용하지 않음
이름 일치 export 이름과 정확히 일치 가져오는 쪽에서 임의 이름 가능
대표 사용처 유틸 함수, 상수, API 함수 대표 컴포넌트, 대표 함수
자주 하는 실수 {} 누락 {}를 잘못 사용
유틸 함수 여러 개 Named Export
상수 여러 개 Named Export
API 함수 여러 개 Named Export
대표 컴포넌트 하나 Default Export
09

Default와 Named를 함께 사용하기

한 파일에서 default export와 named export를 함께 사용할 수 있습니다. 이 경우 가져올 때는 default import가 먼저 오고, named import는 {} 안에 작성합니다.

triangle.js
export const widthHeight = {
    width: 100,
    height: 100
};

export default function area(width, height) {
    return width * height * 0.5;
}
default + named import
import triangleArea, { widthHeight } from "./triangle.js";

const w = widthHeight.width;
const h = widthHeight.height;

console.log(triangleArea(w, h));
// 5000
잘못된 순서 
import { widthHeight }, triangleArea from "./triangle.js";
// SyntaxError
올바른 순서 
import triangleArea, { widthHeight } from "./triangle.js";

Namespace Import: import * as

import * as 이름은 모듈의 named export들을 하나의 객체처럼 묶어서 가져오는 방식입니다.

userApi.js
export function getUsers() {
    console.log("사용자 목록 조회");
}

export function createUser(data) {
    console.log("사용자 생성", data);
}

export function deleteUser(id) {
    console.log("사용자 삭제", id);
}
namespace import
import * as userApi from "./userApi.js";

userApi.getUsers();
userApi.createUser({ username: "kosmo" });
userApi.deleteUser(1);

15

자주 하는 실수

01

Named import에서 {}를 빼먹는다

import add from "./math.js";

named export라면 import { add }처럼 가져와야 합니다.

02

Default import에 {}를 사용한다

import { Button } from "./Button.jsx";

default export라면 import Button처럼 가져옵니다.

03

Named export 이름을 틀린다

import { sum } from "./math.js";

실제 export 이름이 add라면 { add }로 가져와야 합니다.

04

default export를 여러 개 작성한다

export default function add() {}
export default function multiply() {}

한 파일에는 default export를 하나만 작성할 수 있습니다.

05

export default const를 사용한다

export default const API_BASE_URL = "...";

변수 선언 후 export default API_BASE_URL로 내보냅니다.

06

상대 경로를 잘못 작성한다

import Header from "./components/Header.jsx";

현재 파일 위치에 따라 ../components/Header.jsx일 수 있습니다.

07

type="module" 없이 import를 사용한다

<script src="./index.js"></script>

브라우저 직접 실행에서는 type="module"이 필요합니다.

08

default와 named import 순서를 틀린다

import { widthHeight }, triangleArea from "./triangle.js";

import triangleArea, { widthHeight } 순서로 작성합니다.

09

파일명 대소문자를 다르게 import한다

import Header from "../components/header.jsx";

실제 파일명이 Header.jsx라면 대소문자를 정확히 맞춥니다.

10

import한 값을 직접 재할당한다

import { API_BASE_URL } from "./constants.js";

API_BASE_URL = "http://localhost:3000";

import한 바인딩은 읽기 전용이므로 직접 바꾸지 않습니다.

면접 포인트

Q1. 모듈이란 무엇인가요?

기능 단위로 분리된 JavaScript 파일입니다.
변수, 함수, class, 컴포넌트 등을 파일 단위로 나누고
필요한 곳에서 가져와 사용할 수 있게 합니다.

Q2. export와 import의 역할은?

export는 현재 파일의 코드를 밖으로 내보내고,
import는 다른 파일이 내보낸 코드를 현재 파일로 가져옵니다.

Q3. Named Export란?

이름이 있는 값을 내보내는 방식입니다.
여러 개 export할 수 있고 import할 때 { name } 형태를 사용합니다.

Q4. Default Export란?

파일의 대표값 하나를 기본으로 내보내는 방식입니다.
한 파일에 하나만 가능하고 {} 없이 import합니다.

Q5. Named와 Default의 차이는?

Named는 여러 개를 내보내고 이름이 정확히 일치해야 합니다.
Default는 하나만 내보내고 가져오는 쪽에서 이름을 자유롭게 정할 수 있습니다.

Q6. as는 언제 사용하나요?

import 이름이 충돌하거나 현재 파일에서 더 명확한 이름으로 사용하고 싶을 때 사용합니다.

Q7. import * as는 무엇인가요?

모듈의 named export들을 하나의 namespace 객체처럼 묶어서 가져오는 문법입니다.

Q8. Side Effect Import란?

값을 가져오지 않고 모듈 파일을 실행만 하는 import입니다.
CSS import나 초기 설정 파일 실행에 사용됩니다.

Q9. 브라우저에서 import를 쓰려면?

<script type="module" src="./index.js">처럼 type="module"을 지정해야 합니다.

Q10. React에서 컴포넌트는 보통 어떻게 export하나요?

대표 컴포넌트 하나를 내보내는 경우가 많으므로 default export를 자주 사용합니다.

Q11. React Hook은 어떻게 import하나요?

useState, useEffect 같은 Hook은 import { useState }처럼 named import로 가져옵니다.

Q12. API 함수 파일은 어떤 export가 적합한가요?

여러 API 함수를 한 파일에서 내보내는 경우가 많으므로
named export나 namespace import가 적합합니다.

30초 면접 답변

import와 export는 ES6 모듈 문법으로, JavaScript 코드를 파일 단위로 나누고 필요한 기능만 가져오기 위해 사용합니다.
export는 현재 파일의 변수, 함수, class, 컴포넌트를 외부에서 사용할 수 있게 내보내고, import는 다른 파일이 내보낸 값을 가져옵니다.
named export는 여러 개를 내보낼 수 있고 { name } 형태로 정확한 이름을 가져와야 하며, default export는 한 파일의 대표값 하나를 내보내고 {} 없이 가져옵니다.
React에서는 컴포넌트, Hook, API 함수, 유틸 함수, CSS 파일을 모두 import/export로 관리합니다.

한 문장으로 마무리

import와 export는 JavaScript 코드를 파일 단위의 모듈로 나누어 필요한 기능만 명확하게 가져오고 내보내는 ES6 문법이며,
named export, default export, alias import, namespace import를 구분해서 사용하면
React 프로젝트에서 컴포넌트, API 함수, 유틸 함수, 상수 파일을 체계적으로 관리할 수 있습니다.

학습 메모

이번 단원은 React 프로젝트 구조를 이해하는 데 매우 중요합니다.
파일을 나누는 것 자체보다, 어떤 파일이 어떤 기능을 외부에 공개하고 어떤 파일이 그 기능을 가져와 사용하는지 명확히 보는 것이 핵심입니다.

Export 기준

여러 유틸 함수, 상수, API 함수를 내보낼 때는 named export를 자주 사용합니다. 파일의 대표 컴포넌트나 대표 함수 하나를 내보낼 때는 default export를 자주 사용합니다.

Import 기준

named export는 import { name }, default export는 import name입니다. 이 둘을 헷갈리면 대부분의 import 오류가 발생합니다.

React 기준

컴포넌트는 default export, Hook은 named import, API 함수와 상수는 named export, CSS는 side effect import로 연결되는 경우가 많습니다.

오류 기준

does not provide an export named가 나오면 export 이름과 import 이름이 맞는지 확인하고, default 관련 오류가 나오면 named export와 default export를 혼동했는지 확인합니다.
