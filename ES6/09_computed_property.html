Computed Property Name 계산된 프로퍼티 이름

Computed Property Name은 객체의 속성명을 고정된 이름으로 작성하는 대신, 변수나 표현식의 결과로 동적으로 생성하는 ES6 문법입니다.
객체 key 자리에 [표현식]을 사용하면, 표현식의 결과가 실제 속성명이 됩니다.
React에서는 여러 input을 하나의 handler로 처리할 때 { ...form, [name]: value } 형태로 매우 자주 사용됩니다.

변수 key의 값을 속성명으로 사용합니다.

02

왜 Computed Property Name을 사용하는가

일반 객체는 key를 고정된 이름으로 작성합니다.

고정된 key를 가진 객체
const user = {
    username: "kosmo",
    email: "kosmo@example.com",
    role: "USER"
};

console.log(user.username);
// kosmo

하지만 실제 프로젝트에서는 key가 실행 중에 결정되는 경우가 많습니다. 예를 들어 검색 기준이 username일 수도 있고, email일 수도 있습니다.

검색 기준에 따라 key가 달라지는 경우
const searchType = "email";
const keyword = "kosmo@example.com";

const params = {
    [searchType]: keyword
};

console.log(params);
// { email: "kosmo@example.com" }

03

기본 문법: { [표현식]: 값 }

Computed Property Name의 기본 형태는 다음과 같습니다.

const obj = { [expression]: value };

객체 key 자리에 대괄호를 쓰면 대괄호 안의 표현식이 먼저 계산되고, 그 결과가 실제 key가 됩니다.

일반 key 
const key = "email";

const user = {
    key: "test@example.com"
};

console.log(user);
// { key: "test@example.com" }

key라는 글자가 그대로 속성명이 됩니다.

Computed key 
const key = "email";

const user = {
    [key]: "test@example.com"
};

console.log(user);
// { email: "test@example.com" }

변수 key의 값인 email이 속성명이 됩니다.

대괄호 안에 올 수 있는 것

변수 { [field]: value }
문자열 결합 { ["user" + "Name"]: value }
템플릿 리터럴 { [`user_${id}`]: value }
함수 호출 { [makeKey()]: value }
삼항 연산자 { [isMobile ? "mobile" : "desktop"]: value }
숫자 표현식 { [1 + 2]: "three" }
04

ES5 방식과 ES6 방식 비교

ES5에서도 동적 key를 만들 수는 있었습니다. 다만 객체를 먼저 만든 뒤, 객체 바깥에서 대괄호 표기법으로 속성을 추가해야 했습니다.

ES5 방식 
var i = 0;
var obj = {};

obj["foo" + ++i] = i;
obj["foo" + ++i] = i;
obj["foo" + ++i] = i;

console.log(obj.foo1);
// 1

console.log(obj.foo2);
// 2

console.log(obj.foo3);
// 3
ES6 방식 
let i = 0;

const obj = {
    ["foo" + ++i]: i,
    ["foo" + ++i]: i,
    ["foo" + ++i]: i
};

console.log(obj.foo1);
// 1

console.log(obj.foo2);
// 2

console.log(obj.foo3);
// 3
구분 ES5 ES6
동적 key 생성 위치 객체 생성 후 바깥에서 추가 객체 리터럴 내부에서 생성
문법 obj["foo" + i] = value { ["foo" + i]: value }
코드 흐름 객체 생성과 key 추가가 분리됨 객체 구조 안에서 key 생성까지 표현
가독성 동적 속성이 밖에 흩어질 수 있음 객체 생성 의도가 한눈에 보임
05

점 표기법과 대괄호 표기법

Computed Property Name을 이해하려면 객체 속성 접근 방식도 함께 이해해야 합니다.

점 표기법

고정 key 접근
const user = {
    username: "kosmo",
    email: "kosmo@example.com"
};

console.log(user.username);
// kosmo

user.username은 문자 그대로 username이라는 속성에 접근합니다.

대괄호 표기법

문자열 key 접근
const user = {
    username: "kosmo",
    email: "kosmo@example.com"
};

console.log(user["email"]);
// kosmo@example.com

변수로 key 접근

변수의 값을 key로 사용
const user = {
    username: "kosmo",
    email: "kosmo@example.com"
};

const key = "email";

console.log(user[key]);
// kosmo@example.com

console.log(user.key);
// undefined
코드 의미 결과 예시
user.email email이라는 고정 key 접근 kosmo@example.com
user["email"] 문자열 "email" key 접근 kosmo@example.com
user[key] 변수 key의 값을 key로 사용 kosmo@example.com
user.key key라는 고정 key 접근 undefined
06

key와 [key]의 차이

초보자가 가장 많이 헷갈리는 부분은 key와 [key]의 차이입니다.

일반 key 
const key = "email";

const user = {
    key: "test@example.com"
};

console.log(user);
// { key: "test@example.com" }

문자 그대로 key라는 속성명이 만들어집니다.

계산된 key 
const key = "email";

const user = {
    [key]: "test@example.com"
};

console.log(user);
// { email: "test@example.com" }

변수 key의 값인 email이 속성명이 됩니다.

실수 주의

React form에서 name: value라고 쓰면 문자 그대로 name이라는 key가 생깁니다. input의 name 값을 key로 쓰려면 반드시 [name]: value라고 써야 합니다.

07

표현식으로 동적 key 만들기

대괄호 안에는 단순 변수뿐 아니라 문자열 결합, 템플릿 리터럴, 함수 호출 결과, 조건식 등을 사용할 수 있습니다.

변수 값을 key로 사용

const key = "username";

const user = {
    [key]: "kosmo"
};

console.log(user);
// { username: "kosmo" }

문자열 결합

const prefix = "user";
const field = "Name";

const obj = {
    [prefix + field]: "가길동"
};

console.log(obj);
// { userName: "가길동" }

템플릿 리터럴

const id = 100;

const obj = {
    [`user_${id}`]: "가길동"
};

console.log(obj);
// { user_100: "가길동" }

함수 호출 결과

function makeKey(prefix, field) {
    return `${prefix}_${field}`;
}

const obj = {
    [makeKey("user", "name")]: "가길동"
};

console.log(obj);
// { user_name: "가길동" }

삼항 연산자

const isMobile = true;

const config = {
    [isMobile ? "mobileSize" : "desktopSize"]: 4
};

console.log(config);
// { mobileSize: 4 }

반복문으로 key 생성

const result = {};

for (let i = 1; i <= 3; i++) {
    result[`item${i}`] = i;
}

console.log(result);
// { item1: 1, item2: 2, item3: 3 }

const param = "size";

객체 구조분해에서 동적 key 꺼내기

객체를 만들 때뿐 아니라 객체 구조분해 할당에서도 계산된 property name을 사용할 수 있습니다. 변수에 저장된 key로 객체의 값을 꺼낼 때 사용합니다.

변수 key로 값 꺼내기
let key = "z";

let { [key]: foo } = {
    z: "bar"
};

console.log(foo);
// bar

위 코드는 변수 key의 값인 "z"에 해당하는 속성을 꺼내서 foo 변수에 저장합니다.

오해 
let key = "z";

let { key: foo } = {
    z: "bar"
};

console.log(foo);
// undefined

key라는 고정 속성을 찾기 때문에 undefined입니다.

정상 
let key = "z";

let { [key]: foo } = {
    z: "bar"
};

console.log(foo);
// bar

변수 key의 값인 z 속성을 찾습니다.

React와 연결되는 지점

React에서 Computed Property Name이 가장 자주 쓰이는 곳은 여러 input을 하나의 onChange handler로 처리하는 form state입니다.

여러 input을 하나의 handler로 처리

SignupForm.jsx
import { useState } from "react";

function SignupForm() {
    const [form, setForm] = useState({
        username: "",
        email: "",
        password: ""
    });

    const handleChange = (event) => {
        const { name, value } = event.target;

        setForm((prevForm) => ({
            ...prevForm,
            [name]: value
        }));
    };

    return (
        <form>
            <input
                name="username"
                value={form.username}
                onChange={handleChange}
                placeholder="아이디"
            />

            <input
                name="email"
                value={form.email}
                onChange={handleChange}
                placeholder="이메일"
            />

            <input
                name="password"
                value={form.password}
                onChange={handleChange}
                placeholder="비밀번호"
            />
        </form>
    );
}

export default SignupForm;

username과 userName은 다른 key입니다.

05

객체 key 자리에서 표현식을 그냥 쓴다

const config = {
    "mobile" + param: 4
};

표현식 key는 ["mobile" + param]처럼 대괄호가 필요합니다.

06

문자열 "key"를 변수로 착각한다

const key = "email";

const user = {
    ["key"]: "test@example.com"
};

["key"]는 문자열 key이고, 변수 사용은 [key]입니다.

07

Spread 없이 동적 key만 사용한다

setForm({
    [name]: value
});

기존 form의 다른 값이 사라질 수 있습니다. ...prevForm이 필요합니다.

08

name 변수 값을 확인하지 않는다

setForm((prev) => ({
    ...prev,
    [name]: value
}));

console.log(name)으로 실제 key 문자열을 확인해야 합니다.

17

면접 포인트

Q1. Computed Property Name이란?

객체의 속성명을 변수나 표현식의 결과로 동적으로 만드는 ES6 문법입니다.

Q2. 기본 문법은?

객체 리터럴에서 key 자리에 대괄호를 사용해 { [expression]: value } 형태로 작성합니다.

Q3. ES5 방식과 ES6 방식의 차이는?

ES5는 객체를 만든 뒤 obj["동적키"] = value로 추가했고,
ES6는 객체 리터럴 내부에서 { ["동적키"]: value }로 작성할 수 있습니다.

Q4. key와 [key]의 차이는?

key는 문자 그대로 "key"라는 속성명이고, [key]는 변수 key의 값을 속성명으로 사용합니다.

Q5. obj.key와 obj[key]의 차이는?

obj.key는 고정 key에 접근하고, obj[key]는 변수 key의 값을 key로 사용해 접근합니다.

Q6. React에서 어디에 쓰이나요?

여러 input을 하나의 onChange handler로 처리할 때
{ ...form, [name]: value } 형태로 자주 사용합니다.

Q7. React form에서 주의할 점은?

input의 name 속성과 form state의 key가 정확히 일치해야 합니다.

Q8. Spread와 함께 쓰는 이유는?

기존 객체 값을 유지하면서 동적 key 하나만 수정하기 위해
{ ...prevForm, [name]: value } 형태로 함께 사용합니다.

Q9. Shorthand Property와의 차이는?

Shorthand는 { username }처럼 key와 변수명이 같을 때 줄여 쓰는 문법이고,
Computed Property Name은 key 이름 자체를 계산하는 문법입니다.

Q10. API 요청에서 어떻게 활용하나요?

검색 기준이나 조건에 따라 요청 객체의 key가 달라질 때
{ [searchType]: keyword } 형태로 활용할 수 있습니다.

30초 면접 답변

Computed Property Name은 객체의 속성명을 변수나 표현식의 결과로 동적으로 만드는 ES6 문법입니다.
객체 리터럴에서 key 자리에 대괄호를 사용해 { [expression]: value } 형태로 작성합니다.
예를 들어 const key = "email"일 때 { [key]: "test@example.com" }은 { email: "test@example.com" }이 됩니다.
React에서는 여러 input을 하나의 handler로 처리할 때 setForm({ ...form, [name]: value }) 형태로 자주 사용합니다.

18

이번 단원 최종 정리

Computed Property Name

객체 key를 변수나 표현식으로
동적으로 만듭니다.

기본 형태

{ [expression]: value }

핵심 기호

key 자리에 쓰는 대괄호 []입니다.

key

문자 그대로 "key"라는 속성명입니다.

[key]

변수 key의 값을 속성명으로 사용합니다.

obj.key

고정 key에 접근합니다.

obj[key]

변수 값으로 key에 접근합니다.

ES5 방식

obj["foo" + i] = value

ES6 방식

{ ["foo" + i]: value }

React form

{ ...form, [name]: value }

API params

{ [searchType]: keyword }

대표 실수

[name] 대신 name을 쓰는 것입니다.

한 문장으로 마무리

Computed Property Name은 객체의 속성명을 고정된 이름으로 쓰는 대신 변수나 표현식의 결과로 동적으로 생성하는 ES6 문법이며,
특히 React의 form 입력 처리에서 setForm({ ...form, [name]: value })처럼 여러 input을 하나의 handler로 처리할 때 매우 자주 사용됩니다.

학습 메모

이번 단원은 React form 처리를 이해하는 데 매우 중요합니다. key와 [key]의 차이만 정확히 이해해도 Computed Property Name의 핵심은 대부분 잡은 것입니다.

고정 key

{ name: value }는 문자 그대로 name이라는 속성을 만듭니다.

동적 key

{ [name]: value }는 변수 name의 값을 속성명으로 사용합니다.

React 기준

input의 name 속성과 state 객체의 key를 반드시 맞춰야 합니다. 예를 들어 state가 username이면 input도 name="username"이어야 합니다.

Spread 조합

기존 객체 값을 유지하려면 { ...prevForm, [name]: value }처럼 작성합니다. { [name]: value }만 쓰면 기존 값이 사라질 수 있습니다.
