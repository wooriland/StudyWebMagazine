ES6 JavaScript - Chapter 02

Template Literals

Template Literal은 백틱으로 문자열을 작성하고 ${표현식}을 이용해 동적인 값을 필요한 위치에 삽입하는 ES6 문법입니다.
기존의 + 문자열 연결을 더 읽기 쉽게 바꾸고, 여러 줄 메시지, API URL, 화면 문구, React 속성값을 명확하게 작성할 수 있게 합니다.

Expression

실행한 뒤 하나의 값으로 평가되는 JavaScript 코드입니다. Template Literal의 ${} 안에는 변수뿐 아니라 다양한 표현식을 넣을 수 있습니다.

변수 삽입

표현식과 문장의 차이

종류 예시 ${} 안 사용 이유
변수 username 가능 하나의 값으로 평가됨
산술 연산 price * quantity 가능 계산 결과를 반환함
함수 호출 getMessage() 가능 반환값으로 평가됨
삼항 연산 condition ? a : b 가능 선택된 값을 반환함
if문 if (condition) {} 불가능 값 하나를 반환하는 표현식이 아님
for문 for (...) {} 불가능 실행 흐름을 제어하는 문장
변수 선언 const value = 10 불가능 선언문이므로 표현식이 아님
복잡한 로직은 문자열 밖에서 먼저 처리합니다. 
const activeUserSummary = users
    .filter((user) => user.active)
    .map((user) => `${user.name}:${user.age}`)
    .sort()
    .join(", ");

const message =
    `활성 사용자: ${activeUserSummary}`;

Template Literal 안에는 값을 읽거나 간단히 계산하는 표현식만 두는 편이 코드의 의도를 파악하기 쉽습니다.

05

평가 시점과 문자열 변환

Template Literal은 실행 시점의 값으로 완성된다

Template Literal은 원본 변수와 계속 연결되는 반응형 문자열이 아닙니다. 코드가 실행되는 시점의 값을 읽어 완성된 문자열 하나를 만듭니다.

초기 값 count = 1
→
문자열 생성 `현재 값: ${count}`
→
저장된 결과 "현재 값: 1"
→
이후 값 변경 count = 2
이미 생성된 문자열은 자동으로 바뀌지 않는다
let count = 1;

const firstMessage =
    `현재 값: ${count}`;

count = 2;

const secondMessage =
    `현재 값: ${count}`;

console.log(firstMessage);
// 현재 값: 1

console.log(secondMessage);
// 현재 값: 2
최신 값이 필요하면 다시 평가한다
let count = 1;

function getCountMessage() {
    return `현재 값: ${count}`;
}

console.log(getCountMessage());
// 현재 값: 1

count = 2;

console.log(getCountMessage());
// 현재 값: 2

표현식은 왼쪽에서 오른쪽으로 평가된다

함수 호출 순서
function first() {
    console.log("first 실행");
    return "첫 번째";
}

function second() {
    console.log("second 실행");
    return "두 번째";
}

const message =
    `${first()} / ${second()}`;

console.log(message);
출력 순서 
first 실행
second 실행
첫 번째 / 두 번째

표현식 결과의 문자열 변환

값 예시 문자열 결과 주의점
숫자 ${100} "100" 일반적인 숫자 문자열 변환
불리언 ${true} "true" 화면 문구가 필요하면 별도 변환
null ${null} "null" 기본값 처리를 권장
undefined ${undefined} "undefined" 사용자 화면 노출 주의
배열 ${[1, 2, 3]} "1,2,3" join()으로 구분자 지정
일반 객체 ${user} "[object Object]" 프로퍼티 또는 JSON 변환 사용
Symbol ${symbol} TypeError 가능 String(symbol) 사용

중요

Template Literal은 문자열을 조합할 뿐, 삽입값을 검증하거나 안전하게 정화하지 않습니다.

미제공

URL 인코딩

공백과 특수문자를 URL 규칙에 맞게
자동 변환하지 않습니다.

미제공

SQL 인젝션 방지

사용자 입력을 SQL에 안전하게

미제공

    commentInput.value;

function highlight(
    strings,
    ...values
) {
    return strings.reduce(
        (result, text, index) => {
            const value =
                index < values.length
                    ? `[${values[index]}]`
                    : "";

            return result + text + value;
        },
        ""
    );
}

const username = "가길동";
const score = 95;

const message =
    highlight`${username}님의 점수는 ${score}점입니다.`;

console.log(message);
// [가길동]님의 점수는 [95]점입니다.

면접 포인트

Q1. Template Literal이란 무엇인가요?

백틱으로 문자열을 작성하고
${표현식}으로 동적인 값을 삽입할 수 있는 ES6 문자열 문법입니다.

Q2. 일반 문자열과 어떤 차이가 있나요?

+ 연결 없이 표현식을 삽입할 수 있고,
실제 줄바꿈을 포함한 여러 줄 문자열을 작성할 수 있습니다.

Q3. ${} 안에는 변수만 가능한가요?

변수뿐 아니라 산술 연산, 함수 호출, 메서드 호출, 삼항 연산자처럼
하나의 값으로 평가되는 표현식을 사용할 수 있습니다.

Q4. ${} 안에 if문을 넣을 수 있나요?

if는 문장이므로 직접 넣을 수 없습니다. 삼항 연산자나 함수 반환값을 사용합니다.

Q5. 여러 줄 문자열에서 주의할 점은 무엇인가요?

실제 줄바꿈뿐 아니라 코드 들여쓰기와 앞뒤 공백도 문자열에 포함될 수 있습니다.

Q6. 변수가 바뀌면 기존 문자열도 바뀌나요?

아닙니다. Template Literal이 실행된 시점의 값을 사용해 완성된 문자열이 생성됩니다.

Q7. 객체를 그대로 넣으면 어떻게 되나요?

일반 객체는 [object Object]가 나올 수 있으므로
필요한 프로퍼티나 JSON.stringify()를 사용합니다.

Q8. URL 인코딩도 자동으로 처리되나요?

아닙니다. encodeURIComponent()나 URLSearchParams가 별도로 필요할 수 있습니다.

Q9. Template Literal은 XSS를 막아 주나요?

아닙니다. 문자열을 조합할 뿐 삽입값을 정화하지 않으므로
innerHTML 사용 시 보안 검토가 필요합니다.

Q10. Tagged Template이란 무엇인가요?

Template Literal 앞에 함수를 붙여 고정 문자열 조각과 삽입값을 분리해 처리하는 문법입니다.

Q11. String.raw는 무엇인가요?

이스케이프 시퀀스의 역슬래시를 작성된 모양에 가깝게 유지해 주는
내장 Tagged Template 함수입니다.

Q12. React에서는 어디에 사용하나요?

완성 메시지, className, 접근성 속성, API URL, 이미지 URL 등에 자주 사용합니다.

30초 면접 답변

Template Literal은 ES6에서 추가된 문자열 문법으로, 백틱을 사용하고 ${표현식}을 통해 변수, 연산 결과, 함수 반환값을 문자열 안에 삽입할 수 있습니다.
기존의 + 연결보다 최종 문장의 구조를 읽기 쉽고, 실제 줄바꿈을 포함한 여러 줄 문자열도 작성할 수 있습니다.
다만 URL 인코딩, HTML 이스케이프, XSS와 SQL 인젝션 방지는 자동으로 제공하지 않으므로 목적에 맞는 별도 처리가 필요합니다.

이번 단원 최종 정리

백틱

Template Literal의 시작과 끝을 표시합니다.

${표현식}

JavaScript 표현식의 결과를 문자열에 삽입합니다.

문자열 연결

+ 연결보다 실제 출력 구조를 쉽게 읽을 수 있습니다.

여러 줄

코드에 작성한 실제 줄바꿈을 문자열에 포함할 수 있습니다.

평가 시점

문자열을 생성하는 순간의 값이 사용됩니다.

객체 삽입

필요한 프로퍼티나 JSON.stringify()를 사용합니다.

배열 삽입

표시 형식을 제어하려면 join()을 사용합니다.

URL

구조 조합과 값 인코딩은 서로 다른 문제입니다.

HTML

문자열 생성은 가능하지만 XSS 방지는 별도 처리합니다.

String.raw

이스케이프 문자의 역슬래시를 원본 형태에 가깝게 유지합니다.

Tagged Template

문자열 조각과 삽입값을 함수에서 분리해 처리합니다.

React

메시지, 속성, 클래스명, API URL에 활용합니다.

한 문장으로 마무리

Template Literal은 고정된 문장 구조와 ${표현식}의 동적인 결과를 명확하게 구분하여
화면 메시지, 여러 줄 텍스트, API 주소와 React 속성값을 읽기 쉽게 작성하도록 돕는 문법입니다.

학습 메모

Template Literal은 코드 길이를 줄이는 것보다 문자열의 실제 구조가 코드에서 바로 보이게 하는 것이 더 중요한 목적입니다.

가장 먼저 확인할 문법

표현식 삽입은 작은따옴표나 큰따옴표가 아니라 백틱 내부의 ${표현식}에서만 동작합니다.

표현식 작성 기준

변수, 간단한 연산, 프로퍼티 접근과 짧은 함수 호출은 ${} 안에 넣을 수 있습니다. 복잡한 로직은 먼저 변수나 함수로 분리합니다.

실무 보안 기준

Template Literal은 문자열만 조합합니다. URL 인코딩, HTML 정화, SQL 파라미터 바인딩은 각각 목적에 맞는 별도의 도구를 사용합니다.

React에서 기억할 것

완성된 문자열에는 Template Literal이 편리하지만, React UI 구조는 HTML 문자열이 아니라 JSX 요소로 작성합니다.
