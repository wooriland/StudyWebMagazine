왜 class를 사용하는가

객체 하나만 필요하다면 객체 리터럴로 충분합니다. 하지만 같은 구조의 객체를 여러 개 만들어야 한다면 객체 리터럴을 계속 복사하는 방식은 비효율적입니다.

객체 리터럴 방식
const user1 = {
    name: "가길동",
    age: 20,
    introduce() {
        console.log(`${this.name}님의 나이는 ${this.age}세입니다.`);
    }
};

const user2 = {
    name: "나길동",
    age: 25,
    introduce() {
        console.log(`${this.name}님의 나이는 ${this.age}세입니다.`);
    }
};

위 코드는 introduce() 메서드가 반복됩니다. 이런 경우 class를 사용하면 객체의 구조와 동작을 하나의 설계도로 만들 수 있습니다.

class 설계도 방식
class User {
    constructor(name, age) {
        this.name = name;
        this.age = age;
    }

    introduce() {
        console.log(`${this.name}님의 나이는 ${this.age}세입니다.`);
    }
}

const user1 = new User("가길동", 20);
const user2 = new User("나길동", 25);

user1.introduce();
// 가길동님의 나이는 20세입니다.

user2.introduce();
// 나길동님의 나이는 25세입니다.

03

ES5 생성자 함수와 prototype

ES6 class를 이해하려면 먼저 ES5의 생성자 함수와 prototype 방식을 이해해야 합니다.

생성자 함수

new와 생성자 함수
function Person(name) {
    this.name = name;
}

const person = new Person("가길동");

console.log(person.name);
// 가길동

생성자 함수 안에 메서드를 직접 넣는 방식의 문제

인스턴스마다 함수가 새로 만들어짐
function Person(name) {
    this.name = name;

    this.printName = function () {
        console.log(this.name);
    };
}

const person1 = new Person("가길동");
const person2 = new Person("나길동");

console.log(person1.printName === person2.printName);
// false

기능이 같은 printName()이 인스턴스마다 새로 만들어지므로 비효율적입니다. 그래서 공통 메서드는 prototype에 넣습니다.

prototype에 메서드 넣기

공통 메서드 공유
function Person(name) {
    this.name = name;
}

Person.prototype.printName = function () {
    console.log(this.name);
};

const person1 = new Person("가길동");
const person2 = new Person("나길동");

console.log(person1.printName === person2.printName);
// true

person1.printName();
// 가길동

person2.printName();
// 나길동
class와 prototype의 관계

ES6 class로 메서드를 작성해도 내부적으로는 ClassName.prototype.method 구조와 연결됩니다.

constructor - 인스턴스 초기화

constructor는 new로 인스턴스를 만들 때 자동으로 실행되는 특별한 메서드입니다. 주로 인스턴스의 초기 필드를 설정합니다.

constructor 기본 예제
class Person {
    constructor(name, age) {
        this.name = name;
        this.age = age;
    }
}

const person = new Person("가길동", 20);

console.log(person.name);
// 가길동

console.log(person.age);
// 20

constructor는 하나만 가능하다

오류 
class Person {
    constructor(name) {
        this.name = name;
    }

    constructor(name, age) {
        this.name = name;
        this.age = age;
    }
}

// SyntaxError:
// A class may only have one constructor
대안 
class Person {
    constructor(name, age = 0) {
        this.name = name;
        this.age = age;
    }
}

const person1 = new Person("가길동");
const person2 = new Person("나길동", 25);

옵션 객체로 초기화하기

매개변수가 많을 때
class User {

        this.username = username;
        this.email = email;
        this.role = role;

const user = new User({
    username: "kosmo",
    email: "kosmo@example.com"

console.log(user.role);

static - class 자체의 메서드

static 메서드는 인스턴스가 아니라 class 자체에 속한 메서드입니다. 인스턴스를 만들지 않고 ClassName.method() 형태로 호출합니다.

static 메서드
class MathUtil {
    static add(a, b) {
        return a + b;
    }
}

console.log(MathUtil.add(10, 20));
// 30

인스턴스 없이 class 이름으로 호출

instance method instance.method()

인스턴스를 만든 뒤 호출

static 메서드는 인스턴스로 호출할 수 없다

오류 
class MathUtil {

// util.add is not a function
정상 
class MathUtil {

static 사용 예

유틸 기능
class PriceFormatter {
    static format(price) {
        return `${price.toLocaleString("ko-KR")}원`;

console.log(PriceFormatter.format(30000));
// 30,000원

extends - 상속

extends는 부모 class를 상속받아 자식 class를 만들 때 사용합니다. 자식 class는 부모 class의 필드와 메서드를 사용할 수 있습니다.

Animal과 Dog 상속
class Animal {
    constructor(name) {
        this.name = name;
    }

    sleep() {
        console.log(`${this.name} 잠잔다`);
    }
}

class Dog extends Animal {
    bark() {
        console.log(`${this.name} 짖는다`);
    }
}

const dog = new Dog("예삐");

dog.sleep();
// 예삐 잠잔다

dog.bark();
// 예삐 짖는다

상속을 사용하는 이유

공통 기능은 부모, 특화 기능은 자식
class User {
    constructor(name) {
        this.name = name;
    }

    login() {
        console.log(`${this.name} 로그인`);
    }
}

class AdminUser extends User {
    deleteUser(targetName) {
        console.log(`${targetName} 사용자 삭제`);
    }
}

const admin = new AdminUser("관리자");

admin.login();
// 관리자 로그인

admin.deleteUser("가길동");
// 가길동 사용자 삭제

super()와 super.method()

super()는 부모 constructor 호출

부모 초기화 후 자식 필드 추가
class Person {
    constructor(name) {
        this.name = name;
    }
}

class Student extends Person {
    constructor(studentNumber, name) {
        super(name);
        this.studentNumber = studentNumber;
    }
}

const student = new Student("S001", "가길동");

console.log(student.name);
// 가길동

console.log(student.studentNumber);
// S001

자식 class에서는 this보다 super()가 먼저다

오류 
class Student extends Person {
    constructor(studentNumber, name) {
        this.studentNumber = studentNumber;
        super(name);
    }
}

// ReferenceError:
// Must call super constructor before accessing 'this'
정상 
class Student extends Person {
    constructor(studentNumber, name) {
        super(name);
        this.studentNumber = studentNumber;
    }
}

super.method()는 부모 메서드 호출

부모 메서드 호출 후 자식 기능 추가
class Animal {
    constructor(name) {
        this.name = name;
    }

    sleep() {
        console.log(`${this.name} 잠잔다`);
    }
}

class Dog extends Animal {
    bark() {
        super.sleep();
        console.log(`${this.name} 짖는다`);
    }
}

const dog = new Dog("예삐");

dog.bark();

// 예삐 잠잔다
// 예삐 짖는다
부모 메서드를 그냥 이름만으로 호출하면 안 됩니다.

sleep()처럼 호출하면 현재 스코프에서 sleep이라는 함수를 찾습니다. 부모 메서드를 호출하려면 super.sleep() 또는 this.sleep()을 사용합니다.

class 안의 this

class의 constructor와 인스턴스 메서드 안에서 this는 보통 현재 인스턴스를 가리킵니다.

인스턴스 메서드의 this
class Person {
    constructor(name) {
        this.name = name;
    }

    printName() {
        console.log(this.name);
    }
}

const person = new Person("가길동");

person.printName();
// 가길동

class 메서드를 분리하면 this가 사라질 수 있다

문제 
const print = person.printName;

print();

// TypeError:
// Cannot read properties of undefined

print()로 호출하면서 호출 주체가 사라졌습니다.

해결 
const print =
    person.printName.bind(person);

print();
// 가길동

bind(person)으로 this를 인스턴스에 고정합니다.

class field 화살표 함수

분리 호출에도 this 유지
class Person {
    name = "가길동";

    printName = () => {
        console.log(this.name);
    };
}

const person = new Person();

const print = person.printName;

print();
// 가길동
방식 저장 위치 특징
일반 class 메서드 prototype 인스턴스들이 메서드를 공유하지만 분리 호출 시 this 주의
class field 화살표 함수 각 인스턴스 this 바인딩 문제는 줄어들지만 인스턴스마다 함수 생성
class 내부는 strict mode처럼 동작합니다.

호출 주체 없이 class 메서드를 실행하면 this가 window로 자동 연결되지 않고 undefined가 되기 쉽습니다.

보완 개념: public field, private field, getter, setter

교안의 핵심 범위는 constructor, 메서드, static, extends, super()이지만, 현대 JavaScript class를 읽기 위해 아래 개념도 함께 알아두면 좋습니다.

public field

class body에 필드 선언
class Counter {
    count = 0;

    increase() {
        this.count += 1;
    }
}

const counter = new Counter();

counter.increase();

console.log(counter.count);
// 1

private field

#을 붙이면 class 내부에서만 접근 가능
class BankAccount {
    #balance = 0;

    deposit(amount) {
        this.#balance += amount;
    }

    getBalance() {
        return this.#balance;
    }
}

const account = new BankAccount();

account.deposit(10000);

console.log(account.getBalance());
// 10000

getter와 setter

메서드를 프로퍼티처럼 사용
class User {
    constructor(firstName, lastName) {
        this.firstName = firstName;
        this.lastName = lastName;
    }

    get fullName() {
        return `${this.lastName}${this.firstName}`;
    }

    set fullName(value) {
        this.lastName = value.slice(0, 1);
        this.firstName = value.slice(1);
    }
}

const user = new User("길동", "가");

console.log(user.fullName);
// 가길동

user.fullName = "나길동";

console.log(user.lastName);
// 나

면접 포인트

Q1. ES6 class란 무엇인가요?

JavaScript의 prototype 기반 객체 생성 방식을 더 읽기 쉽게 표현하기 위한 문법입니다.

Q2. ES5 생성자 함수와 class의 차이는?

ES5는 생성자 함수와 prototype을 따로 작성하고,
ES6 class는 constructor와 메서드를 하나의 class 블록에 작성합니다.

Q3. constructor의 역할은?

new로 인스턴스를 만들 때 자동 실행되어 초기 필드를 설정합니다.

Q4. class 메서드는 어디에 저장되나요?

일반 class 메서드는 각 인스턴스에 복사되지 않고 class의 prototype에 저장되어 공유됩니다.

Q5. static 메서드는 무엇인가요?

인스턴스가 아니라 class 자체에 속한 메서드이며 ClassName.method()로 호출합니다.

Q6. extends는 무엇인가요?

부모 class를 상속받아 자식 class를 만들 때 사용하는 키워드입니다.

Q7. super()는 무엇인가요?

부모 class의 constructor를 호출하는 문법입니다. 자식 class에서 this를 사용하기 전에 호출해야 합니다.

Q8. super.method()는 무엇인가요?

부모 class의 메서드를 호출하는 문법입니다.
오버라이딩된 메서드에서 부모 기능을 함께 사용할 때 유용합니다.

Q9. class는 호이스팅되나요?

class 이름은 스코프에 등록되지만 초기화 전 접근할 수 없는 TDZ 상태이므로
선언 전에 사용할 수 없습니다.

Q10. class 메서드를 변수에 담아 호출하면 왜 문제가 되나요?

호출 주체가 사라져 this가 undefined가 될 수 있기 때문입니다.

Q11. React class component에서 class 문법은 어떻게 쓰이나요?

extends React.Component, constructor, super(props),
this.state, this.setState()로 연결됩니다.

Q12. 현대 React에서도 class를 알아야 하나요?

현대 React는 함수형 컴포넌트와 Hooks 중심이지만,
기존 class component 코드와 JavaScript 객체지향 문법을 이해하기 위해 필요합니다.

30초 면접 답변

ES6 class는 JavaScript의 prototype 기반 객체 생성 방식을 더 읽기 쉽게 표현하기 위한 문법입니다.
기존 ES5에서는 생성자 함수와 prototype을 따로 작성했지만, ES6 class에서는 constructor와 메서드를 하나의 class 블록 안에 작성할 수 있습니다.
constructor는 new로 인스턴스를 만들 때 실행되어 초기값을 설정하고, 일반 인스턴스 메서드는 prototype에 저장됩니다.
static 메서드는 class 이름으로 호출하고, extends는 상속을 만들며, 자식 class에서 this를 사용하려면 반드시 super()를 먼저 호출해야 합니다.
