02

for...in과 for...of

for...in과 for...of는 이름이 비슷하지만 순회하는 대상이 완전히 다릅니다. for...in은 속성 이름을 순회하고, for...of는 실제 값을 순회합니다.

for...in

key를 본다

0 1 username

객체의 key, 배열의 index, 추가된 속성명을 순회합니다.

for...of

value를 본다

가길동 나길동

iterable의 실제 값을 순회합니다.

객체에서 for...in

객체의 key와 value 순회
const user = {
    name: "가길동",
    age: 20,
    address: "서울"
};

for (const key in user) {
    console.log(key);
    console.log(user[key]);
}

// name
// 가길동
// age
// 20
// address
// 서울

for...in의 변수에는 값이 아니라 속성 이름이 들어옵니다. 따라서 값을 꺼내려면 user[key]처럼 대괄호 접근을 사용합니다.

배열에서 for...in을 사용할 때의 문제

배열에 속성이 추가된 경우
const users = ["가길동", "나길동"];

users.username = "다길동";

for (const index in users) {
    console.log(index, users[index]);
}

// 0 가길동
// 1 나길동
// username 다길동
배열 값 순회에 for...in을 권장하지 않는 이유

for...in은 배열 요소만 순회하는 문법이 아니라 열거 가능한 속성명을 순회합니다. 배열에 추가된 속성이나 프로토타입의 속성까지 포함될 수 있습니다.

프로토타입 속성까지 포함될 수 있다

교안 핵심 예제 흐름
const arr = ["가길동", "나길동"];

Array.prototype.username = "다길동";

for (const index in arr) {
    console.log(arr[index]);
}

// 가길동
// 나길동
// 다길동

delete Array.prototype.username;

실제 배열 요소는 두 개뿐인데, 프로토타입에 추가한 username까지 순회될 수 있습니다. 그래서 배열의 값을 안전하게 순회하려면 for...of를 사용합니다.

배열에서 for...of

배열의 실제 값 순회
const users = ["가길동", "나길동"];

for (const user of users) {
    console.log(user);
}

// 가길동
// 나길동
구분 for...in for...of
순회 대상 key, index, 속성명 value, 실제 값
배열에서 나오는 것 "0", "1" 같은 인덱스 배열 요소 값
객체에서 사용 가능 일반 객체는 바로 불가능
배열 순회 가능하지만 주의 권장
프로토타입 속성 영향 받을 수 있음 배열 값 순회에서는 영향 적음
대표 사용 목적 객체 key 순회 배열, 문자열, Map, Set 값 순회
03

객체 순회와 iterable 순회

for...of는 iterable에 사용할 수 있습니다. 배열, 문자열, Set, Map은 iterable입니다. 하지만 일반 객체는 기본적으로 iterable이 아니므로 for...of로 바로 순회하면 오류가 발생합니다.

Array 
for (const value of [1, 2, 3]) {
    console.log(value);
}
String 
for (const char of "KOSMO") {
    console.log(char);
}
Set 
const roles = new Set(["USER", "ADMIN"]);

for (const role of roles) {
    console.log(role);
}
Map 
const userMap = new Map();

userMap.set("name", "가길동");
userMap.set("age", 20);

for (const [key, value] of userMap) {
    console.log(key, value);
}

일반 객체는 바로 for...of 불가능

오류 
const user = {
    name: "가길동",
    age: 20
};

for (const value of user) {
    console.log(value);
}

// TypeError: user is not iterable
해결 
for (const [key, value] of Object.entries(user)) {
    console.log(key, value);
}

// name 가길동
// age 20

객체 순회 도구

Object.keys()

객체의 key 배열을 반환합니다.

Object.keys(user);
// ["name", "age"]
Object.values()

객체의 value 배열을 반환합니다.

Object.values(user);
// ["가길동", 20]
Object.entries()

key와 value 쌍의 배열을 반환합니다.

Object.entries(user);
// [["name", "가길동"], ["age", 20]]
객체 순회 실무 기준

객체의 key와 value를 모두 사용해야 한다면 Object.entries()와 구조분해 할당을 함께 사용하는 방식이 가장 읽기 쉽습니다.

함수 선언문과 함수 표현식

JavaScript에서 함수는 선언문으로 만들 수도 있고, 값처럼 변수에 저장하는 표현식으로 만들 수도 있습니다. 두 방식은 비슷해 보이지만 호이스팅에서 큰 차이가 있습니다.

함수 선언문 
function add(a, b) {
    return a + b;
}

console.log(add(10, 20));
// 30

함수 이름으로 선언하며, 함수 전체가 호이스팅됩니다.

함수 표현식 
const add = function (a, b) {
    return a + b;
};

console.log(add(10, 20));
// 30

함수를 값처럼 변수에 저장합니다.
변수 초기화 전에는 호출할 수 없습니다.

화살표 함수 
const add = (a, b) => a + b;

console.log(add(10, 20));
// 30

ES6 함수 표현 방식입니다. 콜백 함수에 자주 사용됩니다.

함수 선언문은 선언 전 호출 가능

함수 선언문 호이스팅
console.log(add(10, 20));
// 30

함수 표현식은 초기화 전 호출 불가

콜백 함수

콜백 함수는 다른 함수에 인자로 전달되어, 특정 시점에 실행되는 함수입니다. 배열 메서드의 핵심도 결국 콜백 함수입니다.

콜백 함수 기본 예제
function greet(callback) {
    callback();
}

function sayHello() {
    console.log("Hello");
}

greet(sayHello);
// Hello
sayHello()

괄호를 붙이면 즉시 실행

≠
sayHello

함수 자체를 인자로 전달

배열 메서드의 콜백

함수 표현식 콜백 
const numbers = [1, 2, 3];

const doubled = numbers.map(function (number) {
    return number * 2;
});

console.log(doubled);
// [2, 4, 6]
화살표 함수 콜백 
const numbers = [1, 2, 3];

const doubled = numbers.map((number) => number * 2);

console.log(doubled);
// [2, 4, 6]

배열 메서드는 대부분 (현재값, 인덱스, 원본배열) 형태의 콜백 인자를 받을 수 있습니다.

콜백 함수의 기본 인자
const users = ["가길동", "나길동"];

users.forEach((value, index, array) => {
    console.log(value);
    console.log(index);
    console.log(array);
});

배열 메서드 전체 개요

배열 메서드는 목록 데이터를 다루는 도구입니다. 무엇을 하고 싶은지에 따라 선택하는 메서드가 달라집니다.

찾기 indexOf()

값의 위치를 찾습니다.

포함 여부 includes()

값이 있는지 boolean으로 확인합니다.

필터링 filter()

조건에 맞는 요소만 남깁니다.

반복 작업 forEach()

각 요소로 작업만 수행합니다.

변환 map()

각 요소를 다른 값으로 바꿉니다.

누적 reduce()

배열을 하나의 결과로 합칩니다.

메서드 목적 반환값 원본 변경 React 사용
indexOf() 값의 인덱스 찾기 인덱스 또는 -1 변경 안 함 선택 여부 확인
includes() 포함 여부 확인 true / false 변경 안 함 권한, 선택 여부
filter() 조건에 맞는 요소 선택 새 배열 변경 안 함 삭제, 검색, 필터링
forEach() 각 요소로 작업 수행 undefined 직접 바꾸면 변경 가능 렌더링에는 부적합
map() 요소 변환 새 배열 변경 안 함 목록 렌더링, 수정
reduce() 누적 계산 하나의 결과 변경 안 함 합계, 통계
메서드 선택 기준

새 배열이 필요하면 map() 또는 filter(), 단순 작업만 필요하면 forEach(), 하나의 결과가 필요하면 reduce()를 선택합니다.

indexOf(), lastIndexOf(), includes()

indexOf()

값의 첫 번째 위치 찾기
const numbers = [1, 2, 3, 4, 5];

console.log(numbers.indexOf(3));
// 2

console.log(numbers.indexOf(10));
// -1

indexOf()는 값이 있으면 인덱스를 반환하고, 없으면 -1을 반환합니다.

lastIndexOf()

뒤에서부터 검색한 위치
const numbers = [1, 2, 3, 2, 1];

console.log(numbers.indexOf(2));
// 1

console.log(numbers.lastIndexOf(2));
// 3

includes()

indexOf 방식 
const roles = ["USER", "ADMIN"];

if (roles.indexOf("ADMIN") !== -1) {
    console.log("관리자 권한이 있습니다.");
}
includes 방식 
const roles = ["USER", "ADMIN"];

if (roles.includes("ADMIN")) {
    console.log("관리자 권한이 있습니다.");
}
사용 기준

위치가 필요하면 indexOf(), 포함 여부만 필요하면 includes()가 더 읽기 쉽습니다.

08

filter() - 조건에 맞는 요소만 남기기

filter()는 배열의 각 요소에 대해 조건을 검사하고, 조건이 true인 요소만 모아 새 배열을 반환합니다.

숫자 배열 필터링
const numbers = [1, 2, 3, 4, 5];

const result = numbers.filter((number) => number >= 3);

console.log(result);
// [3, 4, 5]

console.log(numbers);
// [1, 2, 3, 4, 5]

객체 배열 필터링

성인 사용자만 선택
const users = [
    { name: "가길동", age: 20, addr: "가산동" },
    { name: "나길동", age: 25, addr: "나산동" },
    { name: "다길동", age: 40, addr: "다산동" },
    { name: "라길동", age: 10, addr: "라산동" }
];

const adultUsers = users.filter((user) => user.age >= 20);

console.log(adultUsers);

검색 기능

검색어 포함 여부로 필터링
const posts = [
    { id: 1, title: "JavaScript 기초" },
    { id: 2, title: "React 시작하기" },
    { id: 3, title: "JavaScript 배열 메서드" }
];

const keyword = "JavaScript";

const searchedPosts = posts.filter((post) =>
    post.title.includes(keyword)
);

console.log(searchedPosts);

thisArg와 현대적인 대안

thisArg 사용 
function greaterOrEqual(value) {
    return value >= this.min;
}

const condition = {
    min: 3
};

const result = numbers.filter(
    greaterOrEqual,
    condition
);
클로저 사용 
const min = 3;

const result = numbers.filter(
    (value) => value >= min
);
화살표 함수와 thisArg

화살표 함수는 자기 this를 가지지 않으므로 filter()의 두 번째 인자인 thisArg와 함께 사용하는 의미가 거의 없습니다.

09

forEach() - 각 요소로 작업 수행

forEach()는 배열의 각 요소를 순회하면서 콜백 함수를 실행합니다. 주로 콘솔 출력, DOM 이벤트 등록, 부수 작업에 사용합니다.

각 요소 출력
const numbers = [1, 2, 3];

numbers.forEach((number, index) => {
    console.log(`numbers[${index}] = ${number}`);
});

// numbers[0] = 1
// numbers[1] = 2
// numbers[2] = 3

반환값이 없다

실수 
const numbers = [1, 2, 3];

const result = numbers.forEach((number) => {
    return number * 2;
});

console.log(result);
// undefined

forEach()는 콜백의 반환값을 모아 주지 않습니다.

새 배열이 필요할 때 
const doubled = numbers.map(
    (number) => number * 2
);

console.log(doubled);
// [2, 4, 6]

break를 사용할 수 없다

오류 
numbers.forEach((number) => {
    if (number === 3) {
        break;
    }

    console.log(number);
});

// SyntaxError: Illegal break statement
해결 
for (const number of numbers) {
    if (number === 3) {
        break;
    }

    console.log(number);
}
사용 기준

새 배열이 필요하면 map(), 중간에 멈춰야 하면 for...of, 단순 작업만 필요하면 forEach()를 사용합니다.

10

map() - 각 요소를 변환해 새 배열 만들기

map()은 배열의 각 요소를 다른 값으로 변환하고, 그 결과를 모아 새 배열을 반환합니다. React에서 배열을 JSX로 렌더링할 때 가장 많이 사용하는 메서드입니다.

숫자 배열 변환
const numbers = [1, 2, 3, 4, 5];

const doubled = numbers.map((number) => number * 2);

console.log(doubled);
// [2, 4, 6, 8, 10]

console.log(numbers);
// [1, 2, 3, 4, 5]

객체 배열에서 이름만 추출

필요한 프로퍼티만 뽑기
const users = [
    { id: 1, name: "가길동", age: 20 },
    { id: 2, name: "나길동", age: 25 },
    { id: 3, name: "다길동", age: 30 }
];

const names = users.map((user) => user.name);

console.log(names);
// ["가길동", "나길동", "다길동"]

객체를 새 객체로 변환

화면 카드용 데이터 만들기
const userCards = users.map((user) => ({
    id: user.id,

화살표 함수에서 객체를 바로 반환하려면 ({ ... })처럼 소괄호로 감쌉니다. 그렇지 않으면 중괄호가 함수 본문으로 해석됩니다.

11

reduce() - 배열을 하나의 결과로 누적하기

reduce()는 배열의 각 요소를 누적해서 하나의 결과를 만듭니다. 합계, 평균, 통계, 그룹화, 객체 변환 등에 사용할 수 있습니다.

배열.reduce((누적값, 현재값) => 다음누적값, 초기값)
합계 구하기
const numbers = [1, 2, 3, 4];

const sum = numbers.reduce(
    (total, number) => total + number,
    0
);

console.log(sum);
// 10
초기값 0
→
1회차 0 + 1 = 1
→
2회차 1 + 2 = 3
→
3회차 3 + 3 = 6
→
4회차 6 + 4 = 10

초기값이 중요한 이유

위험 
const numbers = [];

const sum = numbers.reduce(
    (total, number) => total + number
);

// TypeError:
// Reduce of empty array with no initial value
안전 
const numbers = [];

const sum = numbers.reduce(
    (total, number) => total + number,
    0
);

console.log(sum);
// 0

장바구니 총액 계산

객체 배열 누적
const cartItems = [
    { name: "키보드", price: 50000, quantity: 1 },
    { name: "마우스", price: 30000, quantity: 2 },
    { name: "모니터", price: 200000, quantity: 1 }
];

const totalPrice = cartItems.reduce(
    (total, item) => total + item.price * item.quantity,
    0
);

console.log(totalPrice);
// 310000

객체로 누적하기

역할별 사용자 수 계산
const users = [
    { name: "가길동", role: "USER" },
    { name: "나길동", role: "ADMIN" },
    { name: "다길동", role: "USER" }
];

const roleCount = users.reduce((result, user) => {
    return {
        ...result,
        [user.role]: (result[user.role] ?? 0) + 1
    };
}, {});

console.log(roleCount);
// { USER: 2, ADMIN: 1 }

배열 메서드 선택 기준

그냥 반복만 한다 forEach()
중간에 멈춰야 한다 for...of
같은 길이의 새 배열을 만든다 map()
일부 요소만 남긴다 filter()
하나의 결과로 합친다 reduce()
포함 여부만 확인한다 includes()

원본 배열 변경 여부

메서드 원본 배열 변경 여부 주의점
indexOf() 변경하지 않음 값이 없으면 -1
includes() 변경하지 않음 포함 여부만 확인
filter() 변경하지 않음 새 배열 반환
forEach() 메서드 자체는 변경하지 않음 콜백 안에서 객체를 직접 바꾸면 원본 객체 변경 가능
map() 변경하지 않음 콜백에서 객체 직접 수정 주의
reduce() 변경하지 않음 초기값 지정 권장

    { id: 3, name: "모니터", price: 200000 }
];

const expensiveProductNames = products
    .filter((product) => product.price >= 50000)
    .map((product) => product.name);

console.log(expensiveProductNames);
// ["키보드", "모니터"]

면접 포인트

Q1. for...in과 for...of의 차이는?

for...in은 객체의 key 또는 배열의 index 같은 속성명을 순회하고,
for...of는 iterable의 실제 value를 순회합니다.

Q2. 배열 순회에 for...in을 권장하지 않는 이유는?

배열 요소뿐 아니라 배열에 추가된 속성이나
프로토타입 체인의 열거 가능한 속성까지 순회될 수 있기 때문입니다.

Q3. 일반 객체를 for...of로 순회할 수 있나요?

기본적으로 불가능합니다. 일반 객체는 iterable이 아니므로
Object.keys(), Object.values(), Object.entries()를 사용합니다.

Q4. 함수 선언문과 함수 표현식의 차이는?

함수 선언문은 함수 전체가 호이스팅되어 선언 전 호출이 가능하지만,
함수 표현식은 변수에 함수가 할당된 후에만 호출할 수 있습니다.

Q5. 콜백 함수란?

다른 함수에 인자로 전달되어 특정 시점에 실행되는 함수입니다.
배열 메서드와 이벤트 처리에서 자주 사용됩니다.

Q6. indexOf()는 무엇을 반환하나요?

특정 값이 처음 등장하는 인덱스를 반환하고, 값이 없으면 -1을 반환합니다.

Q7. filter()는 어떤 메서드인가요?

조건이 true인 요소만 모아 새 배열을 반환하는 메서드입니다. 원본 배열은 변경하지 않습니다.

Q8. forEach()와 map()의 차이는?

forEach()는 작업만 수행하고 반환값은 undefined입니다.
map()은 변환 결과를 모아 새 배열을 반환합니다.

Q9. reduce()는 언제 사용하나요?

배열의 요소들을 누적해서 하나의 결과를 만들 때 사용합니다.
합계, 평균, 통계, 그룹화에 사용할 수 있습니다.

Q10. reduce() 초기값을 넣는 이유는?

빈 배열에서 오류를 방지하고, accumulator의 시작값과 타입을 명확하게 하기 위해서입니다.

Q11. React 목록 렌더링에는 어떤 메서드를 쓰나요?

map()을 사용합니다. 배열 데이터를 JSX 요소 배열로 변환해야 하기 때문입니다.

Q12. React에서 삭제와 수정은 어떤 메서드와 연결되나요?

삭제는 filter(), 수정은 map()과 연결됩니다.
기존 배열을 직접 수정하지 않고 새 배열을 만듭니다.

30초 면접 답변

for...in은 객체의 key나 배열의 index 같은 속성명을 순회하고, for...of는 배열 같은 iterable의 실제 값을 순회합니다. 배열 값 순회에는 for...of가 더 적합합니다.
함수 선언문은 호이스팅되어 선언 전 호출이 가능하지만, 함수 표현식은 변수에 함수가 할당된 뒤에만 호출할 수 있습니다.
배열 메서드 중 filter()는 조건에 맞는 요소만 새 배열로 만들고, map()은 각 요소를 변환해 새 배열을 만들며, forEach()는 반복 작업만 수행하고 반환값이 없습니다.
reduce()는 배열을 하나의 값으로 누적할 때 사용합니다. React에서는 목록 렌더링에 map(), 삭제에 filter(), 합계 계산에 reduce()를 자주 사용합니다.

for...in

객체의 key, 배열의 index 같은 속성명을 순회합니다.

for...of

iterable의 실제 value를 순회합니다.

for...of를 권장합니다.

객체 순회

Object.keys(), Object.values(),

호이스팅되어 선언 전 호출이 가능합니다.

함수 표현식

변수 초기화 이후 호출할 수 있습니다.

콜백 함수

다른 함수에 인자로 전달되어 나중에 실행됩니다.

값의 위치를 반환하고, 없으면 -1을 반환합니다.

포함 여부를 boolean으로 반환합니다.

각 요소로 작업만 수행하며 반환값은 undefined입니다.

map()

각 요소를 변환해 새 배열을 반환합니다.

reduce()

배열을 누적해 하나의 결과를 만듭니다.

React 목록 출력

map()을 사용합니다.

React 삭제·수정

삭제는 filter(), 수정은 map()을 사용합니다.

한 문장으로 마무리

for...of는 배열의 실제 값을 안전하게 순회하기 위한 문법이고,
함수 선언문과 함수 표현식은 호이스팅과 호출 순서에서 차이가 있으며,
filter(), map(), reduce() 같은 배열 메서드는 실무와 React에서 목록 필터링, 화면 렌더링, 합계 계산을 처리하는 핵심 도구입니다.

학습 메모

이번 단원은 JavaScript와 React에서 목록 데이터를 다루는 기본기입니다.
배열을 어떻게 순회하고, 조건으로 걸러내고, 새 형태로 바꾸고, 하나의 값으로 누적하는지를 구분할 수 있어야 합니다.

반복문 기준

배열의 값이 필요하면 for...of, 객체의 key가 필요하면 for...in 또는 Object.keys(), key와 value가 모두 필요하면 Object.entries()를 사용합니다.

배열 메서드 기준

조건으로 거르면 filter(), 새 배열로 변환하면 map(), 작업만 수행하면 forEach(), 하나의 결과로 누적하면 reduce()입니다.

React 기준

목록 렌더링은 map(), 삭제는 filter(), 수정은 map(), 합계 계산은 reduce()로 연결됩니다.
