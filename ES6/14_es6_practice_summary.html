04

변수 선언: let과 const

ES6 이후에는 var보다 let과 const를 사용합니다. 실전에서는 기본적으로 const를 먼저 생각하고, 재할당이 필요한 경우에만 let을 사용합니다.

문법 특징 실전 사용 기준
const 재할당 불가 기본 선택, 함수·객체·배열 선언
let 재할당 가능 검색어, 카운터, 변경되는 임시 값
var 함수 스코프 ES6 이후 코드에서는 되도록 피함
const와 let 사용 기준
const boards = [];
let keyword = "";

keyword = "Promise";
const 객체 주의

const는 변수 재할당을 막는 것이지, 객체 내부 속성 변경까지 막는 것은 아닙니다. 하지만 React 상태에서는 내부 직접 수정도 피하고 새 객체로 업데이트해야 합니다.

const 객체 내부 속성 변경은 가능
const user = {
    username: "kim",
    email: "kim@example.com"
};

user.email = "new@example.com";

console.log(user);
// { username: "kim", email: "new@example.com" }

문자열 처리와 콜백 함수

Template Literals

Template Literals는 백틱과 ${...}를 사용해 문자열 안에 변수를 쉽게 넣는 문법입니다.

기존 방식 
const username = "kim";
const count = 3;

const message =
    username + "님이 작성한 게시글은 " + count + "개입니다.";
Template Literals 
const username = "kim";
const count = 3;

const message = `${username}님이 작성한 게시글은 ${count}개입니다.`;
게시글 표시 문자열 만들기
const board = boards[0];

const message = `[${board.id}] ${board.title} - 작성자: ${board.writer}`;

console.log(message);
// [1001] ES6 let과 const 정리 - 작성자: kim

Arrow Function

화살표 함수는 map, filter, reduce, Promise 콜백, React 이벤트 핸들러에서 자주 사용합니다.

기존 함수 표현식 
const titles = boards.map(function(board) {
    return board.title;
});
화살표 함수 
const titles = boards.map((board) => board.title);
this 주의

객체 메서드에서 this가 필요하다면 화살표 함수보다 Method 축약 표현을 사용하는 것이 안전합니다.

비동기 처리: Promise와 async / await

Promise 방식

then / catch / finally
fetch("/api/boards")
    .then((response) => {
        if (!response.ok) {
            return Promise.reject("게시글 목록 조회 실패");
        }

        return response.json();
    })
    .then((boards) => {
        console.log(boards);
    })
    .catch((error) => {
        console.log(error);
    })
    .finally(() => {
        console.log("요청 종료");
    });

async / await 방식

try ~ catch ~ finally
async function loadBoards() {
    try {
        const response = await fetch("/api/boards");

        if (!response.ok) {
            throw new Error("게시글 목록 조회 실패");
        }

        const boards = await response.json();

        console.log(boards);
    } catch (error) {
        console.log(error.message);
    } finally {
        console.log("요청 종료");
    }
}

loadBoards();

const mockBoards = [
    {
        id: 1001,
        title: "ES6 let과 const 정리",
        writer: "kim",
        category: "JavaScript",
        comments: [
            { id: 1, content: "좋은 정리입니다.", postdate: "2026-06-01" },
            { id: 2, content: "React와 연결해 주세요.", postdate: "2026-06-02" }
        ]
    },
    {
        id: 1002,
        title: "Promise와 async await",
        writer: "lee",
        category: "JavaScript",
        comments: [
            { id: 3, content: "비동기 이해가 됐어요.", postdate: "2026-06-03" }
        ]
    }
];

function fetchBoards() {
    return new Promise((resolve) => {
        setTimeout(() => {
            resolve(mockBoards);
        }, 1000);
    });
}

function createBoardSummaries(boards) {
    return boards.map(({ id, title, writer, comments }) => ({
        id,
        title,
        writer,
        commentCount: comments.length,
        label: `[${id}] ${title} - ${writer} / 댓글 ${comments.length}개`
    }));
}

function filterBoardsByKeyword(boards, keyword) {
    return boards.filter(({ title }) => title.includes(keyword));
}

function updateCommentContent(boards, targetCommentId, content) {
    return boards.map((board) => ({
        ...board,
        comments: board.comments.map((comment) =>
            comment.id === targetCommentId
                ? {
                      ...comment,
                      content,
                      postdate: new Date().toLocaleDateString()
                  }
                : comment
        )
    }));
}

async function runBoardApp() {
    try {
        const boards = await fetchBoards();

        const keyword = "Promise";
        const searchedBoards = filterBoardsByKeyword(boards, keyword);
        const summaries = createBoardSummaries(searchedBoards);

        console.log(`${keyword} 검색 결과: ${summaries.length}건`);
        console.log(summaries);

        const updatedBoards = updateCommentContent(
            boards,
            2,
            "React와 연결한 설명이 좋습니다."
        );

        console.log(updatedBoards);
    } catch (error) {
        console.log(error.message);
    } finally {
        console.log("게시글 앱 실행 종료");
    }
}

runBoardApp();

React에서는 ES6 문법이 거의 모든 코드에 등장합니다. 아래 예제는 게시글 목록 조회, 검색, 댓글 수정, 로딩, 에러, 목록 렌더링을 한 번에 연결합니다.

import { useEffect, useState } from "react";
import { fetchBoards } from "../api/boardApi";
import {
    createBoardSummaries,
    updateCommentContent
} from "../utils/boardUtils";

function BoardPage() {
    const [boards, setBoards] = useState([]);
    const [keyword, setKeyword] = useState("");
    const [loading, setLoading] = useState(false);
    const [error, setError] = useState("");

    useEffect(() => {
        async function loadBoards() {
            try {
                setLoading(true);
                setError("");

                const data = await fetchBoards();

                setBoards(data);
            } catch (error) {
                setError(error.message);
            } finally {
                setLoading(false);
            }
        }

        loadBoards();
    }, []);

    const handleChange = (event) => {
        const { value } = event.target;
        setKeyword(value);
    };

    const handleUpdateComment = (commentId) => {
        setBoards((prevBoards) =>
            updateCommentContent(
                prevBoards,
                commentId,
                "수정된 댓글입니다."
            )
        );
    };

    const filteredBoards = boards.filter(({ title }) =>
        title.includes(keyword)
    );

    const summaries = createBoardSummaries(filteredBoards);

    if (loading) {
        return <p>로딩 중...</p>;
    }

    if (error) {
        return <p>{error}</p>;
    }

    return (
        <main>
            <input
                name="keyword"
                value={keyword}
                onChange={handleChange}
                placeholder="게시글 검색"
            />

            <p>{`검색 결과: ${summaries.length}건`}</p>

            <ul>
                {summaries.map(({ id, label }) => (
                    <li key={id}>{label}</li>
                ))}
            </ul>

            <button onClick={() => handleUpdateComment(2)}>
                2번 댓글 수정
            </button>
        </main>
    );
}

export default BoardPage;

15

자주 하는 실수

01

const 객체는 절대 못 바꾼다고 생각한다

const user = {
    name: "kim"
};

user.name = "lee";
// 가능

const는 재할당을 막는 것이지 객체 내부 변경까지 막는 것은 아닙니다.

02

React 상태 객체를 직접 수정한다

board.title = "수정된 제목";

React 상태는 직접 수정하지 않고 새 객체로 만들어 업데이트해야 합니다.

03

중첩 객체를 얕게만 복사한다

const updatedUser = {
    ...user
};

updatedUser.profile.email = "new@example.com";

중첩 객체가 원본과 공유될 수 있습니다.

04

map과 forEach를 혼동한다

const result = boards.forEach((board) => board.title);

console.log(result);
// undefined

새 배열이 필요하면 map을 사용합니다.

05

filter 결과가 하나라고 착각한다

const board = boards.filter((board) => board.id === 1001);

console.log(board.title);
// undefined

filter는 배열을 반환합니다. 하나만 찾으려면 find를 사용합니다.

06

구조분해 key 이름을 틀린다

const { name } = boards[0];

console.log(name);
// undefined

객체에 실제로 존재하는 key 이름을 기준으로 꺼내야 합니다.

07

named export와 default export를 혼동한다

export function add(a, b) {
    return a + b;
}

import add from "./math.js";
// 오류 가능

named export는 { add } 형태로 가져와야 합니다.

08

await 앞 함수가 Promise를 반환하지 않는다

function task() {
    setTimeout(() => {
        console.log("완료");
    }, 1000);
}

await task();

await가 기다리려면 함수가 Promise를 반환해야 합니다.

17

면접 포인트

Q1. ES6에서 var보다 let, const를 사용하는 이유는?

let과 const는 블록 스코프를 가지므로 변수의 유효 범위가 명확합니다.
기본적으로 const를 사용하고 재할당이 필요할 때 let을 사용합니다.

Q2. Template Literals의 장점은?

문자열 안에 변수를 ${...}로 바로 넣을 수 있어
+ 연산자로 연결하는 방식보다 가독성이 좋습니다.

Q3. 화살표 함수는 언제 많이 사용하나요?

map, filter, reduce, Promise 콜백, React 이벤트 핸들러에서 많이 사용합니다.

Q4. 구조분해 할당을 사용하는 이유는?

객체나 배열에서 필요한 값만 간결하게 꺼낼 수 있고,
함수 파라미터나 React props에서 필요한 값이 명확하게 보입니다.

Q5. React에서 Spread를 많이 사용하는 이유는?

React 상태를 직접 수정하지 않고 새 객체나 새 배열로 업데이트하기 위해 사용합니다.

Q6. Computed Property Name은 언제 사용하나요?

객체 key를 변수 값으로 동적으로 정해야 할 때 사용합니다.
React 입력 처리에서 [name]: value 패턴으로 자주 사용됩니다.

Q7. map, filter, reduce의 차이는?

map은 변환, filter는 조건 필터링, reduce는 하나의 값이나 객체로 누적할 때 사용합니다.

Q8. Promise와 async / await의 관계는?

async / await는 Promise를 기반으로 한 문법이며,
Promise 체이닝을 더 읽기 쉬운 동기 코드 형태로 작성하게 해 줍니다.

Q9. React에서 API 요청 시 어떤 상태를 관리하나요?

보통 data, loading, error 상태를 함께 관리합니다.

Q10. named export와 default export의 차이는?

named export는 여러 개를 내보낼 수 있고 이름을 맞춰 가져와야 합니다.
default export는 한 파일에서 하나만 사용할 수 있고
가져올 때 이름을 자유롭게 정할 수 있습니다.

30초 면접 답변

ES6는 실제 웹 개발에서 코드를 더 안전하고 읽기 쉽게 만들기 위한 문법 집합입니다.
let과 const로 변수 범위를 명확히 하고, Template Literals로 문자열을 쉽게 만들며, 화살표 함수와 map, filter, reduce로 배열 데이터를 가공합니다.
구조분해 할당은 필요한 값을 간결하게 꺼낼 때 사용하고, Spread는 원본 객체나 배열을 직접 수정하지 않고 새 값으로 업데이트할 때 사용합니다.
Promise와 async/await는 API 요청 같은 비동기 작업을 처리할 때 사용하고, React에서는 이 문법들이 상태 관리, 목록 렌더링, 입력 처리, 서버 통신에 거의 매번 함께 사용됩니다.
