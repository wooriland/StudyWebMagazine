ES6 JavaScript - Chapter 00

ES6 학습 로드맵

ES6는 JavaScript를 더 안전하고, 간결하고, 실무 프로젝트에 적합하게 만들어 주는 문법 묶음입니다.
이 단원에서는 ES6 전체 학습 흐름을 먼저 잡고, 이후 단원에서 어떤 개념을 어떤 순서로 공부해야 하는지 정리합니다.

let / const Template Literals this Arrow Function Class Module Promise async / await
01

ES6를 공부한다는 것

ES6를 공부한다는 것은 단순히 새로운 문법 몇 개를 외우는 것이 아닙니다.
JavaScript 코드가 복잡해질 때 생기는 문제를 줄이고, 화면 개발과 서버 통신을 더 예측 가능하게 만드는 방식을 익히는 것입니다.
예전 JavaScript는 var, 문자열 더하기, 콜백 함수, 생성자 함수와 prototype, 전역 스코프에 의존하는 코드가 많았습니다.
ES6는 여기에 let, const, Template Literals, Arrow Function, Class, Module, Spread, Destructuring, Promise 같은 문법을 추가하여 코드를 더 명확하게 표현할 수 있게 만들었습니다.

ES5 방식

var, 문자열 + 연결, 콜백 중첩,
prototype 기반 객체 생성

→
ES6 방식

let/const, Template Literal,
Arrow Function, class, module, Promise

→
현대 웹 개발

React, Vite, REST API, 비동기 통신, 컴포넌트 기반 개발

02

ES6가 필요한 이유

코드의 안전성

var는 함수 스코프를 사용하기 때문에
의도하지 않은 변수 접근이
발생할 수 있습니다.
let과 const는
블록 스코프를 사용하여
변수의 범위를 더 명확하게 제한합니다.

코드의 가독성

문자열 연결, 함수 표현,
객체 생성, 배열 처리 코드가
짧고 명확해집니다.
특히 Template Literals와
Arrow Function은 화면 출력 코드와
콜백 코드를 읽기 쉽게 만듭니다.

프로젝트 구조화

import와 export를 사용하면
파일을 역할별로 나눌 수 있습니다.
컴포넌트, 유틸 함수, API 함수,
데이터 객체를 분리해서
관리할 수 있습니다.

비동기 처리 개선

콜백 지옥을 Promise와
async/await로 개선할 수 있습니다.
서버 API 요청, 파일 처리, 타이머,
외부 데이터 요청 같은 작업을
더 읽기 쉽게 작성할 수 있습니다.

03

전체 학습 흐름

ES6 학습은 아래 순서로 진행하는 것이 좋습니다. 먼저 변수와 스코프를 이해하고, 그 다음 함수와 this, 배열 처리, 객체 문법, 모듈, 비동기 처리 순서로 확장합니다.

01

let, const, Scope

var, let, const의 차이를 배우고 함수 스코프와 블록 스코프를 구분합니다. 앞으로 작성할 모든 JavaScript 코드의 변수 선언 기준을 잡는 단원입니다.

02

Template Literals

백틱과 ${}를 사용해 문자열을 더 간결하게 작성합니다. 메시지 출력, URL 조합, HTML 문자열 작성에서 자주 사용됩니다.

03

this 키워드

this가 어떤 객체를 가리키는지 호출 주체를 기준으로 이해합니다. 일반 함수, 객체 메서드, 생성자 함수, 이벤트 핸들러에서 달라지는 동작을 정리합니다.

04

Arrow Function

화살표 함수의 짧은 문법과 lexical this를 공부합니다. 콜백 함수에서는 유용하지만 객체 메서드나 이벤트의 this가 필요한 경우에는 주의해야 합니다.

05

for...of, 함수, 배열 메서드

for...in과 for...of의 차이, 함수 선언문과 함수 표현식의 차이,
map, filter, reduce 같은 배열 메서드를 배웁니다. React 목록 렌더링과 데이터 가공의 기초가 됩니다.

06

Class

생성자 함수와 prototype 기반 문법을 ES6 class 문법으로 정리합니다. constructor, static, extends, super()를 통해 객체지향 구조를 이해합니다.

07

import와 export

파일을 모듈로 나누고 필요한 값만 가져오는 방법을 배웁니다. named export, default export, alias, 전체 import 방식을 구분합니다.

08

Spread와 Rest

같은 ... 문법이 배열과 객체를 펼칠 때는 Spread, 함수 매개변수에서 남은 인자를 모을 때는 Rest로 쓰인다는 점을 구분합니다.
상태 객체 복사와 함수 인자 처리에 자주 사용됩니다.

09

Computed Property Name

객체의 속성명을 고정하지 않고 변수나 표현식으로 동적으로 생성하는 방법을 배웁니다. 폼 데이터 처리와 동적 옵션 객체를 만들 때 유용합니다.

10

Destructuring Assignment

배열과 객체에서 필요한 값만 꺼내 변수에 할당하는 구조분해 할당을 배웁니다. 함수 파라미터, API 응답 데이터, React props와 Hook 문법을 이해하는 데 필수입니다.

11

Shorthand, Method, 객체 복사

객체 축약 표현, 메서드 축약 표현, 얕은 복사와 깊은 복사를 정리합니다. 원본 데이터를 직접 바꾸지 않고 새 객체를 만드는 방식까지 연결합니다.

12

Promise

비동기 작업의 성공과 실패를 다루는 Promise를 배웁니다. pending, fulfilled, rejected, then, catch, finally, Promise.all의 흐름을 정리합니다.

13

async / await

Promise 기반 비동기 코드를 동기 코드처럼 읽기 쉽게 작성합니다. try~catch를 사용한 에러 처리와 순차 실행 흐름을 정리합니다.

14

ES6 실전 종합 정리

앞에서 배운 문법을 작은 웹 개발 예제로 연결합니다. 변수 선언, 배열 가공, 객체 복사, 모듈 분리, 비동기 API 요청을 한 흐름으로 복습합니다.

04

ES6를 그림처럼 이해하기

ES6 학습은 “문법을 하나씩 암기하는 과정”보다 “데이터가 코드 안에서 어떻게 이동하고 변형되는지 이해하는 과정”으로 보는 것이 좋습니다.

1단계

let, const, Scope

2단계

함수로 동작 만들기

3단계

배열과 객체 다루기

map, filter, reduce,
spread, destructuring

4단계

파일과 비동기 연결하기

import/export,
Promise, async/await

05

ES6 핵심 문법 미리 보기

아래 코드는 앞으로 배울 ES6 문법이 실제로 어떻게 함께 사용되는지 보여주는 작은 예시입니다.
지금은 전부 이해하지 않아도 됩니다. 이번 로드맵에서는 “이런 문법들을 앞으로 배우게 된다” 정도로 보면 됩니다. 

const users = [
    { id: 1, name: "가길동", age: 20, active: true },
    { id: 2, name: "나길동", age: 17, active: false },
    { id: 3, name: "다길동", age: 25, active: true }
];

const getActiveUserNames = (userList = []) => {
    return userList
        .filter(({ active }) => active)
        .map(({ name, age }) => `${name}님 / ${age}세`);
};

console.log(getActiveUserNames(users));
// ["가길동님 / 20세", "다길동님 / 25세"]

06

React와 연결되는 ES6 문법

ES6는 React를 공부하기 전에 반드시 익숙해져야 하는 기반 문법입니다.
React 코드에서 보이는 const, 구조분해 할당, 화살표 함수, map, spread, import/export, async/await는 대부분 ES6 이후 문법입니다.

컴포넌트 작성

const Header = ({ title }) => {
    return <h1>{title}</h1>;
};

화살표 함수, 구조분해 할당, const가 함께 사용됩니다.

목록 렌더링

{users.map((user) => (
    <li key={user.id}>{user.name}</li>
))}

map()은 배열 데이터를 화면 요소로 변환할 때 사용됩니다.

상태 객체 수정

setUser({
    ...user,
    name: "코스모"
});

spread 연산자로 원본 객체를 직접 수정하지 않고 새 객체를 만듭니다.

API 요청

const response = await fetch("/api/users");
const data = await response.json();

Promise와 async/await는 REST API 통신의 기본이 됩니다.

07

학습할 때 자주 헷갈리는 기준

헷갈리는 부분
정리 기준
let과 const 중 무엇을 써야 할까?
기본은 const, 값이 바뀌어야 하는 반복 변수나 임시 변수에는 let을 사용합니다.
const 객체는 수정할 수 없을까?
const는 재할당을 막습니다. 객체나 배열 내부 속성 변경은 가능합니다.
화살표 함수가 항상 좋은가?
짧은 콜백에는 좋지만, 객체 메서드나 이벤트 객체의 this가 필요한 경우에는
일반 함수가 더 적절할 수 있습니다.
Spread와 Rest는 같은 문법인가?
기호는 같지만 역할이 다릅니다. 펼치면 Spread, 남은 값을 모으면 Rest입니다.
Promise와 async/await는 다른 기술인가?
async/await는 Promise를 더 읽기 쉽게 쓰는 문법입니다. 내부 기반은 Promise입니다.
08

면접에서 ES6를 물어보는 이유

ES6 문법은 단순한 문법 암기 문제가 아니라, 지원자가 현대 JavaScript 프로젝트의 코드를 읽고 유지보수할 수 있는지를 확인하는 기준이 됩니다.

Q1. var, let, const의 차이는 무엇인가요?

변수의 스코프와 재선언, 재할당 가능 여부가 다릅니다.
실무에서는 예측 가능한 코드를 위해 기본적으로 const를 사용하고,
값이 바뀌는 경우에만 let을 사용합니다.

Q2. 화살표 함수와 일반 함수의 차이는 무엇인가요?

문법이 짧다는 차이도 있지만 가장 중요한 차이는 this입니다.
일반 함수는 호출 주체에 따라 this가 결정되고,
화살표 함수는 자신을 감싸는 상위 스코프의 this를 사용합니다.

Q3. 얕은 복사와 깊은 복사는 왜 중요한가요?

객체를 복사했다고 생각했는데 내부 객체의 주소가 공유되면
원본 데이터가 함께 바뀔 수 있습니다.
React 상태 관리처럼 원본 불변성이 중요한 환경에서는 반드시 이해해야 합니다.

Q4. Promise와 async/await는 어디에 사용하나요?

서버 API 요청처럼 결과가 나중에 도착하는 비동기 작업에 사용합니다.
Promise는 비동기 결과를 객체로 다루고,
async/await는 그 흐름을 동기 코드처럼 읽기 쉽게 만듭니다.

09

ES6 학습 체크리스트

각 단원을 공부할 때 아래 기준으로 이해했는지 확인하면 좋습니다. 코드가 “왜 이렇게 동작하는지”를 설명할 수 있으면 다음 단원으로 넘어가도 됩니다.

학습 메모

ES6 학습은 React를 잘하기 위한 준비 단계이면서, JavaScript 자체를 실무적으로 다루기 위한 핵심 과정입니다.
이번 로드맵에서는 전체 문법을 넓게 훑고, 다음 단원부터는 let, const, Scope처럼 하나의 주제씩 깊게 공부합니다.

이번 단원의 핵심 문장

ES6는 “짧은 문법”이 아니라, 복잡한 JavaScript 코드를 더 안전하게 나누고, 예측 가능하게 실행하고, 비동기 흐름까지 읽기 쉽게 만들기 위한 현대 JavaScript의 기본 도구입니다.
