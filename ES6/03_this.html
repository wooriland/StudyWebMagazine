ES6 JavaScript - Chapter 03

this 키워드

this는 현재 함수 실행에서 기준이 되는 객체를 가리키는 키워드입니다.
일반 함수에서는 함수가 어디에 작성되었는지가 아니라 어떻게 호출되었는지에 따라 달라지고,
화살표 함수에서는 자신만의 this를 만들지 않고 바깥 스코프의 this를 사용합니다.

핵심 문장

04

전역 this와 strict mode

브라우저의 일반 <script> 환경에서 전역 this는 보통 window입니다. 하지만 ES Module이나 strict mode에서는 다르게 동작할 수 있습니다.

브라우저 일반 script

<script>
    console.log(this === window);
    // true
</script>

ES Module

<script type="module">
    console.log(this);
    // undefined
</script>

일반 함수 호출

non-strict

function printThis() {
    console.log(this === window);
}

printThis();
// true

브라우저 일반 script에서는 호출 주체가 없을 때 window로 연결될 수 있습니다.

strict mode

"use strict";

function printThis() {
    console.log(this);
}

printThis();
// undefined

strict mode에서는 호출 주체 없는 함수의 this가 undefined입니다.

strict mode가 실수를 빨리 보여 주는 이유

실수로 전역 객체를 수정하는 문제
function setName(name) {
    this.name = name;
}

setName("가길동");

// non-strict 브라우저 일반 script에서는
// window.name이 바뀔 수 있음

객체 메서드에서의 this

객체의 프로퍼티에 함수가 들어 있으면 그 함수를 메서드라고 부릅니다. 메서드를 객체.메서드() 형태로 호출하면 this는 그 객체가 됩니다.

객체 메서드 호출
const user = {
    username: "kosmo",
    print: function () {
        console.log(this.username);
        console.log(this === user);
    }
};

user.print();

// kosmo
// true

ES6 메서드 축약 표현

기존 방식

const user = {
    username: "kosmo",
    print: function () {
        console.log(this.username);
    }
};

메서드 축약 표현

const user = {
    username: "kosmo",
    print() {
        console.log(this.username);
    }
};

메서드 축약 표현은 화살표 함수가 아닙니다. 일반 메서드이므로 user.print()로 호출하면 this는 user가 됩니다.

마지막 점 앞의 객체가 this

중첩 객체 메서드
const company = {
    name: "KOSMO",
    user: {
        name: "가길동",
        printName: function () {
            console.log(this.name);
        }
    }
};

company.user.printName();
// 가길동
마지막 점 규칙

a.b.c.method() 형태라면 this는 c입니다.

07

bind(), call(), apply()

JavaScript 함수는 this를 직접 지정해서 실행하거나, 지정된 this를 가진 새 함수로 만들 수 있습니다.

메서드 실행 여부 인자 전달 방식 특징
call() 즉시 실행 쉼표로 나열 fn.call(obj, a, b)
apply() 즉시 실행 배열로 전달 fn.apply(obj, [a, b])
bind() 즉시 실행하지 않음 나중에 전달 this가 고정된 새 함수 반환

call()

function printInfo(age, city) {
    console.log(`${this.name}, ${age}, ${city}`);
}

const user = {
    name: "가길동"
};

printInfo.call(user, 20, "서울");
// 가길동, 20, 서울

apply()

function printInfo(age, city) {
    console.log(`${this.name}, ${age}, ${city}`);
}

const user = {
    name: "가길동"
};

printInfo.apply(user, [20, "서울"]);
// 가길동, 20, 서울

bind()

function printInfo(age, city) {
    console.log(`${this.name}, ${age}, ${city}`);
}

const user = {
    name: "가길동"
};

const boundPrint =
    printInfo.bind(user);

boundPrint(20, "서울");
// 가길동, 20, 서울

bind()는 새 함수를 반환한다

실수 
function printName() {
    console.log(this.name);
}

const user = {
    name: "가길동"
};

printName.bind(user);

printName();
// this가 user로 고정되지 않음

bind()를 호출만 하고 반환된 새 함수를 저장하지 않았습니다.

해결 
const boundPrintName =
    printName.bind(user);

boundPrintName();
// 가길동

this가 고정된 새 함수를 변수에 저장한 뒤 호출합니다.

bind()는 한 번 고정되면 다시 바뀌지 않는다

첫 번째 바인딩이 유지된다
function printName() {
    console.log(this.name);
}

const user1 = {
    name: "가길동"
};

const user2 = {
    name: "나길동"
};

const bound1 =
    printName.bind(user1);

const bound2 =
    bound1.bind(user2);

bound2();
// 가길동
bind()는 원래 함수를 바꾸지 않습니다.

bind()는 this가 고정된 새 함수를 만들어 반환합니다. 따라서 반환값을 저장하거나 바로 사용해야 합니다.

이벤트 핸들러에서의 this

DOM 이벤트에서 일반 함수로 이벤트 핸들러를 등록하면 this는 이벤트 핸들러가 연결된 요소를 가리킵니다.

일반 함수 이벤트 핸들러

    console.log(this === btn);
});

// <button id="btn">클릭</button>
// true

button#btn span 클릭
event.target

실제 이벤트가 시작된 가장 안쪽 요소

event.currentTarget

이벤트 핸들러가 등록된 요소

this

일반 함수 이벤트 핸들러에서는 보통 currentTarget과 같음

target과 currentTarget 비교
<button id="btn">
    <span>클릭</span>
</button>
span을 클릭한 경우
btn.addEventListener("click", function (event) {
    console.log(event.target);
    // span

    console.log(event.currentTarget);
    // button

    console.log(this);
    // button

주의

        event.currentTarget === btn

중첩 함수와 비동기 콜백에서의 this

객체 메서드 안의 중첩 함수

바깥 메서드의 this가 객체라고 해서, 내부의 일반 함수까지 자동으로 같은 this를 받는 것은 아닙니다.

중첩 일반 함수
const user = {
    name: "가길동",
    outer: function () {
        function inner() {
            console.log(this.name);
        }

        inner();
    }
};

user.outer();
// undefined 또는 오류
user.outer()

outer의 this는 user

→
inner()

일반 함수 호출이므로 this는 user가 아님

중첩 함수 해결 방법

방법 1

self 변수

outer: function () {
    const self = this;

    function inner() {
        console.log(self.name);
    }

    inner();
}
방법 2

bind(this)

outer: function () {
    const inner = function () {
        console.log(this.name);
    }.bind(this);

    inner();
}
방법 3

화살표 함수

outer: function () {
    const inner = () => {
        console.log(this.name);
    };

    inner();
}

비동기 콜백에서의 this

setTimeout 안의 일반 함수
const user = {
    name: "가길동",
    printLater: function () {
        setTimeout(function () {
            console.log(this.name);
        }, 1000);
    }
};

user.printLater();
// undefined 또는 오류

printLater()의 this는 user지만, setTimeout()에 전달한 일반 함수는 나중에 별도로 호출됩니다. 따라서 그 내부의 this는 user가 아닙니다.

일반 함수 콜백 
setTimeout(function () {
    console.log(this.name);
}, 1000);

나중에 호출될 때 바깥 객체의 this가 유지되지 않습니다.

화살표 함수 콜백 
setTimeout(() => {
    console.log(this.name);
}, 1000);

바깥 메서드의 this를 그대로 사용합니다.

화살표 함수와 lexical this

핵심

화살표 함수는 자신만의 this를 만들지 않습니다. 대신 자신을 감싸는 바깥 스코프의 this를 사용합니다.

화살표 함수가 유리한 경우

면접 포인트

Q1. this란 무엇인가요?

현재 함수 실행에서 기준이 되는 객체를 가리키는 키워드입니다.
일반 함수에서는 호출 방식에 따라 결정됩니다.

Q2. 일반 함수에서 this는 어떻게 결정되나요?

함수가 어디에 정의되었는지가 아니라 어떻게 호출되었는지로 결정됩니다.
obj.method()이면 this는 obj입니다.

Q3. 메서드를 변수에 담으면 왜 this가 달라지나요?

함수 값만 분리되고 객체와의 호출 관계가 사라지기 때문입니다.
fn()은 호출 주체 없는 일반 함수 호출입니다.

Q4. bind(), call(), apply()의 차이는?

call()과 apply()는 즉시 실행하고, bind()는 this가 고정된 새 함수를 반환합니다.
apply()는 인자를 배열로 전달합니다.

Q5. DOM 이벤트 핸들러에서 일반 함수의 this는?

이벤트 핸들러가 연결된 요소를 가리킵니다.
다만 실무에서는 event.currentTarget을 쓰는 편이 더 명확할 때가 많습니다.

Q6. 화살표 함수의 this는 일반 함수와 어떻게 다른가요?

화살표 함수는 자신만의 this를 만들지 않고 바깥 스코프의 this를 사용합니다.

Q7. 객체 메서드에 화살표 함수를 쓰면 안 되는 이유는?

객체 리터럴은 새로운 this를 만들지 않기 때문에
화살표 함수의 this가 객체 자신이 되지 않습니다.

Q8. 생성자 함수에서 this는?

new로 호출하면 새로 만들어지는 인스턴스 객체를 가리킵니다.

Q9. strict mode에서 호출 주체가 없는 함수의 this는?

undefined입니다. non-strict 브라우저 일반 script에서는 window가 될 수 있습니다.

Q10. class에서 this는?

생성자와 인스턴스 메서드에서 this는 인스턴스를 가리킵니다.
다만 메서드를 분리 호출하면 this가 사라질 수 있습니다.

Q11. React 함수형 컴포넌트에서는 this를 사용하나요?

보통 사용하지 않습니다.
함수형 컴포넌트에서는 props, state, Hook, 이벤트 객체를 사용합니다.

Q12. 비동기 콜백에서 화살표 함수가 유리한 이유는?

setTimeout() 같은 콜백은 나중에 별도 호출되므로 일반 함수의 this가 바뀔 수 있습니다.
화살표 함수는 바깥 this를 유지합니다.

30초 면접 답변

this는 현재 함수 실행의 기준 객체를 가리키는 키워드입니다. 일반 함수의 this는 함수가 어디에 정의되었는지가 아니라 어떻게 호출되었는지에 따라 결정됩니다.
예를 들어 obj.method()로 호출하면 this는 obj지만, const fn = obj.method; fn()처럼 분리해서 호출하면 호출 주체가 사라져 this가 달라집니다.
bind(), call(), apply()로 this를 명시적으로 지정할 수 있고, 화살표 함수는 자신만의 this를 가지지 않아 바깥 스코프의 this를 사용합니다.
React 함수형 컴포넌트에서는 보통 this를 쓰지 않고 props, state, Hook을 사용합니다.

화살표 함수는 user를 this로 가지지 않습니다.

17

이번 단원 최종 정리

this

현재 함수 실행의 기준 객체입니다.

일반 함수

호출 방식에 따라 this가 결정됩니다.

객체 메서드

obj.method()에서 this는 obj입니다.

마지막 점 규칙

a.b.method()에서 this는 b입니다.

메서드 분리

함수만 분리하면 원래 객체와의 연결이 사라집니다.

strict mode

호출 주체가 없는 일반 함수의 this는 undefined입니다.

이벤트 일반 함수

this는 이벤트가 연결된 요소입니다.

생성자 함수

new로 호출하면 this는 새 객체입니다.

bind()

this가 고정된 새 함수를 반환합니다.

call()

this를 지정하고 즉시 실행합니다.

apply()

this를 지정하고 배열 인자로 즉시 실행합니다.

화살표 함수

자기 this가 없고 바깥 this를 사용합니다.

객체 메서드 화살표 함수

this가 객체 자신이 아니므로 주의합니다.

class

this는 인스턴스지만 메서드 분리 호출에 주의합니다.

React 함수형

보통 this 대신 props, state, Hook을 사용합니다.

한 문장으로 마무리

this는 함수가 어디에 작성되었는지를 보는 문법이 아니라 함수가 어떤 방식으로 호출되었는지를 보는 문법이며,
일반 함수는 호출 주체에 따라 this가 달라지고 화살표 함수는 자신만의 this를 만들지 않고 바깥 스코프의 this를 그대로 사용합니다.

학습 메모

이번 단원은 this를 암기하는 단원이 아니라 함수가 어떤 방식으로 호출되는지 읽는 연습을 하는 단원입니다.
함수 내부에서 this를 만나면, 함수 정의 위치보다 호출 위치를 먼저 찾아야 합니다.

가장 중요한 판단 기준

일반 함수라면 호출부에서 점 앞 객체를 확인합니다. obj.method()이면 this는 obj입니다. method()처럼 호출 주체가 없다면 obj가 아닙니다.

화살표 함수 기준

화살표 함수는 자기 this를 만들지 않습니다. 객체 메서드에서 this를 써야 한다면 화살표 함수 대신 일반 함수 또는 메서드 축약 표현을 사용합니다.

React에서 기억할 것

React 함수형 컴포넌트에서는 보통 this를 사용하지 않습니다. 상태와 화면 변경은 useState, props, 이벤트 객체, Hook을 중심으로 처리합니다.
