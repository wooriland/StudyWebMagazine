23. 패키지

패키지는 관련 있는 클래스와 인터페이스를 묶어 관리하는 Java의 주소 체계입니다.
이 페이지에서는 package 선언, import, 디렉토리 구조,
javac -d . 컴파일, 접근 지정자, jar 배포까지 공부합니다.

이번 페이지의 학습 목표

패키지는 Java 프로젝트를 체계적으로 관리하기 위한 기본 구조입니다.
클래스가 많아질수록 패키지를 잘 나누는 것이 중요하며, 패키지는 클래스 이름 충돌을 막고 프로젝트 구조를 읽기 쉽게 만들어줍니다.

패키지 개념

관련 클래스와 인터페이스를 묶어 관리하는 구조를 이해합니다.

package와 import

내가 속한 위치와 외부 클래스를 가져오는 문장을 구분합니다.

컴파일과 실행

javac -d .와 java 패키지명.클래스명 실행 방식을 익힙니다.

실무 구조

controller, service, repository, entity, dto 패키지 구조를 이해합니다.

1. 패키지의 기본 개념

1-1. 패키지란?

패키지는 관련 있는 클래스와 인터페이스를 묶어 관리하는 단위입니다. 파일 시스템의 폴더와 비슷한 개념으로 이해하면 쉽습니다. 

package member;
package member;
→ 이 클래스는 member 패키지에 속한다.

패키지의 역할:
1. 클래스 분류
2. 이름 충돌 방지
3. 프로젝트 구조 정리
4. jar 파일 배포 단위 구성

1-2. 패키지를 사용하는 이유

회원 관련 클래스
→ member 패키지

게시판 관련 클래스
→ board 패키지

상품 관련 클래스
→ product 패키지

관리자 관련 클래스
→ admin 패키지

패키지를 사용하면 클래스가 많아져도 기능별로 코드를 정리할 수 있습니다. 또한 같은 이름의 클래스가 있어도 패키지가 다르면 서로 다른 클래스로 구분됩니다. 

com.ict.project.user.UserService
com.ict.project.admin.UserService
com.ict.project.payment.UserService

1-3. 패키지는 클래스의 전체 주소다

클래스 이름만으로는 같은 이름의 클래스가 충돌할 수 있습니다. 그래서 Java에서는 패키지명과 클래스명을 합친 전체 이름으로 클래스를 구분합니다. 

클래스명:
UserService

전체 클래스 이름:
com.ict.project.user.UserService

즉,
패키지명 + 클래스명 = 클래스의 전체 주소

2. package 선언

2-1. package 선언 위치

package 선언은 Java 파일의 가장 위에 작성해야 합니다. import보다도 먼저 와야 합니다. 

package member;

import java.util.Scanner;

public class MemberMain {

    public static void main(String[] args) {
        System.out.println("회원 관리");
    }

}
올바른 Java 파일 순서:

1. package 선언
2. import 선언
3. class 선언

2-2. 잘못된 package 위치

import java.util.Scanner;

package member; // 오류

public class MemberMain {
}

package 선언은 반드시 import보다 먼저 작성해야 합니다.

2-3. 패키지명 작성 규칙

패키지명은 보통 소문자로 작성합니다. 클래스명은 대문자로 시작하지만, 패키지명은 소문자로 작성하는 것이 Java 관례입니다. 

package member;
package board;
package product;
package com.ict.project.member;
package User;
package AdminPage;
package PropertyService;
구분 작성 관례 예시
패키지명 소문자 member, com.ict.project.user
클래스명 대문자 시작 MemberMain, UserService
메서드명 소문자 시작 login(), findUser()
변수명 소문자 시작 userName, loginId

3. package와 import

3-1. package와 import의 차이

구분 의미 예시
package 현재 클래스가 속한 패키지 선언 package member;
import 다른 패키지의 클래스를 가져와 사용 import java.util.Scanner;

정리: package는 내가 속한 위치이고, import는 내가 사용할 외부 클래스입니다.

3-2. package와 import 함께 사용

package member;

import java.util.Scanner;

public class MemberInput {

    public static void main(String[] args) {

        Scanner scanner = new Scanner(System.in);

        System.out.print("이름 입력: ");
        String name = scanner.nextLine();

        System.out.println("입력한 이름: " + name);

    }

}
package member;
→ 이 클래스는 member 패키지 소속

import java.util.Scanner;
→ java.util 패키지의 Scanner 클래스를 사용

3-3. 특정 클래스 import

import java.util.Scanner;

Scanner scanner = new Scanner(System.in);

3-4. 패키지 안의 모든 클래스 import

import java.util.*;

Scanner scanner = new Scanner(System.in);
ArrayList<String> list = new ArrayList<>();

import java.util.*;은 java.util 패키지 안의 클래스만 가져옵니다. java.util의 하위 패키지까지 자동으로 가져오는 것은 아닙니다.

3-5. java.lang은 자동 import

System, String, Integer, Math 같은 클래스는 java.lang 패키지에 들어 있습니다. java.lang은 Java에서 자동으로 import되므로 따로 import하지 않아도 됩니다. 

System.out.println("Hello");
String name = "홍길동";
int number = Integer.parseInt("100");
double result = Math.sqrt(16);
생략되어 있다고 볼 수 있는 문장:

import java.lang.*;

4. 패키지와 디렉토리 구조

4-1. 패키지는 폴더 구조와 연결된다

Java 파일에 패키지를 선언했다면, 컴파일된 .class 파일도 패키지 구조와 일치하는 디렉토리 안에 있어야 합니다. 

package nooption;

public class Nooption {

    public static void main(String[] args) {
        System.out.println("-d 옵션 사용안하기");
    }

}
프로젝트 루트
└─ nooption
    ├─ Nooption.java
    └─ Nooption.class

전체 클래스 이름:
nooption.Nooption

4-2. 패키지 안의 클래스 접근 방식

패키지 안에 있는 클래스는 점 .을 사용해서 접근합니다. 

java.lang.System
java.util.Scanner
java.util.ArrayList
member.MemberMain
com.ict.project.user.UserService

Java에서는 폴더 구분 기호처럼 /를 쓰는 것이 아니라, 패키지 경로를 .으로 연결합니다.

5. 패키지가 있는 클래스 컴파일과 실행

5-1. -d 옵션 없이 컴파일하기

-d 옵션을 사용하지 않을 경우, 패키지명과 같은 디렉토리를 미리 만들어두는 방식으로 컴파일할 수 있습니다. 

프로젝트 루트
└─ nooption
    └─ Nooption.java

package nooption;

public class Nooption {

    public static void main(String[] args) {
        System.out.println("-d 옵션 사용안하기");
    }

}
javac nooption\Nooption.java
java nooption.Nooption

5-2. -d 옵션으로 컴파일하기

-d 옵션을 사용하면 package 선언에 맞는 디렉토리가 자동 생성되고, 그 안에 .class 파일이 배치됩니다. 

package option;

public class Option {

    public static void main(String[] args) {
        System.out.println("-d 옵션 사용하기");
    }

}
javac -d . Option.java
컴파일 전:

프로젝트 루트
└─ Option.java


컴파일 후:

프로젝트 루트
├─ Option.java
└─ option
    └─ Option.class

java option.Option

5-3. -d 옵션의 의미

javac -d . Option.java

javac
→ Java 컴파일 명령

-d
→ 컴파일된 class 파일의 출력 위치 지정

.
→ 현재 디렉토리

Option.java
→ 컴파일할 Java 파일

즉 javac -d . Option.java는 “현재 디렉토리를 기준으로 패키지 구조에 맞게 .class 파일을 생성하라”는 의미입니다.

5-4. 패키지가 깊은 경우

package com.ict.project.member;

public class MemberMain {
}
javac -d . MemberMain.java
컴파일 결과:

com
└─ ict
└─ project
    └─ member
        └─ MemberMain.class

java com.ict.project.member.MemberMain

6. 접근 지정자와 패키지

6-1. 접근 지정자와 패키지 관계

패키지는 접근 지정자와도 관련이 깊습니다. 특히 접근 지정자를 생략한 default는 같은 패키지 안에서만 접근할 수 있습니다.

접근 지정자 같은 클래스 같은 패키지 자식 클래스 다른 패키지
public 가능 가능 가능 가능
protected 가능 가능 상속 관계면 가능 상속 관계일 때 가능
default 가능 가능 다른 패키지면 불가 불가
private 가능 불가 불가 불가

6-2. default 접근 지정자

class DefaultClass {
}

접근 지정자를 생략하면 default 접근이 됩니다. 이 클래스는 같은 패키지 안에서는 접근 가능하지만, 다른 패키지에서는 접근할 수 없습니다.

6-3. 최상위 클래스에 private/protected 사용 불가

private class Member {
}

protected class Member {
}
public class Member {
}

class Member {
}

일반적인 최상위 클래스에는 public 또는 default만 사용할 수 있습니다.

7. jar 파일 배포

7-1. jar 파일이란?

jar 파일은 여러 .class 파일과 리소스를 하나로 묶은 Java 배포 파일입니다. 개발 중에는 여러 패키지와 여러 클래스 파일로 나뉘어 있지만, 배포할 때는 jar 파일 하나로 묶을 수 있습니다. 

개발 중:
여러 패키지 + 여러 class 파일

배포 시:
jar 파일 하나로 묶음

7-2. jar 생성 명령

jar cvf jar파일명.jar 루트패키지
jar cvf my.jar my
my 패키지 폴더
        ↓
my.jar 파일로 묶음

7-3. Eclipse에서 jar 만들기

루트 패키지 선택
→ 마우스 우클릭
→ Export
→ Java
→ JAR file 선택
→ 필요한 패키지 체크
→ Finish

7-4. Eclipse에서 외부 jar 가져오기

Project 선택
→ 마우스 우클릭
→ Build Path
→ Configure Build Path
→ Add External JARs
→ jar 파일 선택

실무에서는 직접 jar 파일을 추가하기도 하지만, 보통 Maven이나 Gradle 같은 빌드 도구로 외부 라이브러리 의존성을 관리합니다.

import java.util.Scanner;

package member;

그런데 .class 파일이 member 폴더 안에 없으면 실행 문제가 생길 수 있습니다. javac -d . 옵션을 사용하면 패키지 구조에 맞게 폴더가 자동 생성됩니다. 

javac -d . MemberMain.java

private class Member {
}

protected class Member {
}

}

}

실수 6. package와 import를 혼동

package
→ 현재 클래스가 속한 패키지 선언

import

11. 면접 포인트

Q1. 패키지란 무엇인가요?

답변: 패키지는 관련 있는 클래스와 인터페이스를 묶어 관리하는 단위입니다. 파일 시스템의 디렉토리와 비슷한 개념이며, 클래스 이름 충돌을 줄이고 프로젝트 구조를 체계적으로 관리하는 데 사용합니다.

Q2. package 선언은 어디에 작성해야 하나요?

답변: Java 파일의 최상단에 작성해야 합니다. import 문보다도 먼저 와야 합니다.

Q3. 패키지명은 보통 어떻게 작성하나요?

답변: 보통 소문자로 작성합니다. 예를 들어 member, board, product 또는 com.ict.project.member처럼 작성합니다.

Q4. package와 import의 차이는 무엇인가요?

답변: package는 현재 클래스가 속한 패키지를 선언하는 문장입니다. import는 다른 패키지에 있는 클래스를 현재 파일에서 사용할 수 있도록 가져오는 문장입니다.

Q5. 패키지가 선언된 클래스를 실행할 때 주의할 점은 무엇인가요?

답변: 패키지를 포함한 전체 클래스 이름으로 실행해야 합니다. 예를 들어 package member;가 선언된 MemberMain 클래스는 java member.MemberMain 형태로 실행해야 합니다.

Q6. javac -d . 의 의미는 무엇인가요?

답변: -d 옵션은 컴파일된 class 파일을 지정한 위치에 생성하라는 의미입니다.
-d .은 현재 디렉토리를 기준으로 package 선언에 맞는 폴더 구조를 자동 생성하고, 그 안에 class 파일을 배치하라는 뜻입니다.

Q7. default 접근 지정자는 패키지와 어떤 관계가 있나요?

답변: 접근 지정자를 생략하면 default 접근이 됩니다. default 멤버나 클래스는 같은 패키지 안에서는 접근할 수 있지만, 다른 패키지에서는 접근할 수 없습니다.

Q8. java.lang 패키지는 왜 import하지 않아도 되나요?

답변: java.lang 패키지는 Java에서 기본으로 자동 import되기 때문입니다. System, String, Math, Integer 같은 클래스가 java.lang에 포함됩니다.

Q9. import java.util.* 는 하위 패키지까지 모두 가져오나요?

답변: 아닙니다. java.util 패키지 안의 클래스만 가져오며, java.util의 하위 패키지까지 자동으로 가져오지는 않습니다.

Q10. jar 파일은 무엇인가요?

답변: jar 파일은 여러 class 파일과 리소스를 하나로 묶은 Java 배포 파일입니다. 패키지 단위로 작성한 프로그램을 jar로 압축하여 다른 프로젝트나 환경에서 사용할 수 있습니다.

12. 오늘의 핵심 정리

  1. 패키지는 관련 있는 클래스와 인터페이스를 묶어 관리하는 단위다.
  2. 패키지는 파일 관리의 디렉토리와 비슷한 개념이다.
  3. package 선언은 Java 파일의 최상단에 작성한다.
  4. package 선언은 import보다 먼저 와야 한다.
  5. 패키지명은 보통 소문자로 작성한다.
  6. 패키지 안의 클래스는 패키지명.클래스명 형태로 접근한다.
  7. package는 내가 속한 위치, import는 내가 사용할 외부 클래스다.
  8. 패키지가 있는 클래스는 java 패키지명.클래스명 형태로 실행한다.
  9. javac -d .를 사용하면 package 선언에 맞는 폴더 구조가 자동 생성된다.
  10. default 접근 지정자는 같은 패키지 안에서만 접근 가능하다.
  11. 최상위 클래스에는 public 또는 default만 사용할 수 있다.
  12. java.lang은 자동 import된다.
  13. java.util, java.io, java.sql 등은 자주 사용하는 기본 패키지다.
  14. jar 파일은 패키지와 class 파일을 묶어 배포하는 Java 압축 파일이다.

학습 메모

이 페이지는 Java 패키지 단원입니다.
패키지는 관련 있는 클래스와 인터페이스를 묶어 관리하는 구조이며, Java 코드의 주소 체계라고 이해할 수 있습니다.
package는 현재 클래스가 속한 위치를 선언하고, import는 다른 패키지의 클래스를 가져와 사용하는 문장입니다.
패키지가 있는 클래스는 실행할 때 반드시 java 패키지명.클래스명 형태로 실행해야 합니다.
