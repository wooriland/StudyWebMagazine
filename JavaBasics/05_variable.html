05. 변수

변수는 메모리 상의 임시 데이터 저장 공간에 붙이는 이름입니다.
이 페이지에서는 변수 선언, 초기화, 값 변경, 지역변수, 멤버변수, static 변수,
그리고 좋은 변수명 작성법까지 Java 기초 문법의 핵심을 공부합니다.

이번 페이지의 학습 목표

변수는 Java 문법의 가장 기본이 되는 개념입니다.
변수는 단순히 값을 저장하는 상자가 아니라, 값에 의미를 부여하고, 프로그램 안에서 데이터를 재사용하고 변경할 수 있게 해주는 핵심 도구입니다.

변수 개념 이해

변수가 메모리 상의 데이터 저장 공간에 붙인 이름이라는 점을 이해합니다.

선언과 초기화

자료형과 변수명을 사용해 변수를 선언하고 처음 값을 넣는 방법을 익힙니다.

변수 종류 구분

지역변수, 멤버변수, static 변수의 차이를 구분합니다.

좋은 변수명 작성

실무에서 읽기 좋은 변수명을 짓는 습관을 익힙니다.

1. 핵심 개념 설명

1-1. 변수란?

변수는 메모리 상의 임시 데이터 저장 공간에 붙인 이름입니다. 프로그램은 실행 중 여러 값을 다루는데, 그 값을 저장하고 다시 사용하기 위해 변수를 사용합니다. 

int age = 20;
구성 요소 의미
int 저장할 값의 자료형입니다. 여기서는 정수를 의미합니다.
age 변수명입니다. 값을 저장할 공간의 이름입니다.
20 변수에 저장되는 실제 값입니다.

정리: 변수는 값을 저장하는 공간이면서, 그 값이 무엇을 의미하는지 알려주는 이름표입니다.

1-2. 변수는 왜 필요한가?

값을 직접 사용하면 처음에는 간단해 보이지만, 값의 의미가 드러나지 않고 같은 값을 여러 번 사용할 때 관리하기 어려워집니다. 

System.out.println(20);
System.out.println(20 + 1);
System.out.println(20 >= 19);

위 코드에서 20이 나이인지, 점수인지, 개수인지 알기 어렵습니다. 변수를 사용하면 값에 의미를 부여할 수 있습니다. 

int age = 20;

System.out.println(age);
System.out.println(age + 1);
System.out.println(age >= 19);
  • 값을 저장하기 위해 변수를 사용합니다.
  • 값에 의미 있는 이름을 붙이기 위해 변수를 사용합니다.
  • 같은 값을 여러 번 재사용하기 위해 변수를 사용합니다.
  • 나중에 값을 쉽게 변경하기 위해 변수를 사용합니다.

1-3. 변수 선언

변수를 사용하려면 먼저 선언해야 합니다. 변수 선언은 “어떤 종류의 값을 저장할 공간을 만들겠다”는 의미입니다. 

자료형 변수명;
int age;
String name;
double height;
boolean isLogin;

위 코드는 각각 정수, 문자열, 실수, 논리값을 저장할 수 있는 변수를 선언한 예입니다. 아직 값을 넣은 것은 아니며, 변수를 사용할 준비를 한 상태입니다.

1-4. 변수 초기화

변수에 처음 값을 넣는 것을 초기화라고 합니다. 

int age;
age = 20;

선언과 초기화를 한 번에 할 수도 있습니다.

int age = 20;
String name = "홍길동";
double height = 175.5;
boolean isLogin = true;

초급 단계에서는 변수를 선언할 때 바로 초기화하는 습관을 들이면 초기화하지 않은 변수를 사용하는 오류를 줄일 수 있습니다.

1-5. 변수 값 변경

변수는 이름 그대로 값이 변할 수 있습니다. 같은 변수에 새로운 값을 다시 대입하면 기존 값은 새 값으로 바뀝니다. 

int score = 80;

score = 90;

System.out.println(score);

출력 결과는 다음과 같습니다.

90

처음에는 80이 저장되었지만, 이후 90을 다시 대입했기 때문에 최종적으로 출력되는 값은 90입니다.

2. 변수의 종류

2-1. 지역변수

지역변수는 메서드나 블록 안에서 선언된 변수입니다. 선언된 블록 내부에서만 사용할 수 있고, 사용하기 전에 반드시 초기화해야 합니다. 

public class LocalVariableExample {

    public static void main(String[] args) {

        int age = 20;

        System.out.println(age);

    }

}
  • 메서드나 블록 안에서 선언됩니다.
  • 선언된 블록 안에서만 사용할 수 있습니다.
  • 반드시 초기화한 뒤 사용해야 합니다.
  • 메서드 또는 블록 실행이 끝나면 사라집니다.
int age;
System.out.println(age);

위 코드는 지역변수 age를 초기화하지 않고 사용했기 때문에 오류가 발생합니다.

2-2. 멤버변수

멤버변수는 클래스 안, 메서드 밖에서 선언된 변수입니다. 객체가 가지는 속성을 표현할 때 사용합니다. 

public class Student {

    String name;
    int age;

    public void introduce() {
        System.out.println("이름: " + name);
        System.out.println("나이: " + age);
    }

}

위 코드에서 name과 age는 Student 객체가 가지는 속성입니다.

  • 클래스 안, 메서드 밖에서 선언됩니다.
  • 객체의 속성을 표현합니다.
  • 초기화하지 않아도 기본값으로 자동 초기화됩니다.
  • 객체가 존재하는 동안 함께 유지됩니다.

2-3. static 변수

static 변수는 객체가 아니라 클래스에 소속되는 변수입니다. 객체마다 따로 만들어지는 것이 아니라 클래스 단위로 하나만 존재하며, 모든 객체가 공유합니다. 

public class Student {

    static String schoolName = "한국ICT인재개발원";

    String name;

}
Student 클래스
└─ static schoolName = "한국ICT인재개발원"

student1 객체
└─ name = "홍길동"

student2 객체
└─ name = "김철수"

schoolName은 모든 Student 객체가 공유하는 값이고, name은 객체마다 따로 가지는 값입니다.

2-4. 변수 종류 비교

구분 지역변수 멤버변수 static 변수
선언 위치 메서드/블록 안 클래스 안, 메서드 밖 클래스 안, static 키워드 사용
소속 메서드/블록 객체 클래스
초기화 직접 초기화 필요 기본값 자동 초기화 기본값 자동 초기화

3. 변수명 작성 규칙과 관례

3-1. 변수명 규칙

Java 변수명은 아무렇게나 지을 수 없습니다. 문법적으로 허용되는 규칙을 지켜야 합니다.

  • 영문자, 숫자, 언더바 _, 달러 기호 $를 사용할 수 있습니다.
  • 숫자로 시작할 수 없습니다.
  • 예약어를 사용할 수 없습니다.
  • 공백을 사용할 수 없습니다.
  • 대부분의 특수문자는 사용할 수 없습니다.
  • Java는 대소문자를 구분합니다.

3-2. 올바른 변수명과 잘못된 변수명

int age;
int userAge;
String userName;
int score1;
boolean isLogin;
int 4you;
int sharp#4;
int public;
  • 4you는 숫자로 시작하기 때문에 사용할 수 없습니다.
  • sharp#4는 # 특수문자를 포함하기 때문에 사용할 수 없습니다.
  • public은 Java 예약어이기 때문에 사용할 수 없습니다.

3-3. camelCase 관례

Java에서는 변수명을 보통 camelCase 방식으로 작성합니다. 첫 단어는 소문자로 시작하고, 두 번째 단어부터 첫 글자를 대문자로 작성합니다. 

int userAge;
String userName;
double averageScore;
boolean isDeleted;
user age       → userAge
average score  → averageScore
login status   → loginStatus

3-4. boolean 변수명 관례

boolean 변수는 참/거짓을 표현하므로 상태나 가능 여부가 드러나는 이름을 많이 사용합니다. 

boolean isLogin;
boolean isDeleted;
boolean hasPermission;
boolean canAccess;
변수명 의미
isLogin 로그인 상태인가?
isDeleted 삭제되었는가?
hasPermission 권한을 가지고 있는가?
canAccess 접근할 수 있는가?

4. 그림처럼 이해하기

4-1. 변수와 메모리

int age = 20;

메모리 공간
┌──────────────┐
│ 변수명: age   │
│ 값: 20        │
└──────────────┘

값을 다시 대입하면 같은 변수 공간의 값이 변경됩니다.

age = 21;

메모리 공간
┌──────────────┐
│ 변수명: age   │
│ 값: 21        │
└──────────────┘

4-2. 지역변수와 멤버변수

public class Student {

    String name;  ← 멤버변수

    public void study() {
        int hour = 3;  ← 지역변수
    }

}

Student 객체
├─ name

study() 메서드 실행 중
└─ hour

4-3. 변수 사용 범위

class 블록
├─ 멤버변수 사용 가능
│
└─ method 블록
├─ 지역변수 선언 가능
└─ 지역변수는 이 블록 안에서만 사용 가능

5. 코드 예제

5-1. 변수 선언과 초기화

public class VariableBasic {

    public static void main(String[] args) {

        int age = 20;
        String name = "홍길동";
        double height = 175.5;
        boolean isStudent = true;

        System.out.println(name);
        System.out.println(age);
        System.out.println(height);
        System.out.println(isStudent);

    }

}

5-2. 변수 값 변경

public class ChangeValue {

    public static void main(String[] args) {

        int score = 80;

        System.out.println(score);

        score = 90;

        System.out.println(score);

    }

}

출력 결과는 다음과 같습니다.

80
90

5-3. 지역변수, 멤버변수, static 변수 한 번에 보기

public class VariableTypeExample {

    static String academyName = "한국ICT인재개발원";

    String studentName;

    public void study() {

        int studyHour = 3;

        System.out.println(academyName);
        System.out.println(studentName);
        System.out.println(studyHour);

    }

}
  • academyName은 static 변수입니다.
  • studentName은 멤버변수입니다.
  • studyHour는 지역변수입니다.

5-4. static 변수 예제

public class AcademyStudent {

    static String academyName = "한국ICT인재개발원";

    String studentName;

    public static void main(String[] args) {

        AcademyStudent student1 = new AcademyStudent();
        AcademyStudent student2 = new AcademyStudent();

        student1.studentName = "홍길동";
        student2.studentName = "김철수";

        System.out.println(AcademyStudent.academyName);
        System.out.println(student1.studentName);
        System.out.println(student2.studentName);

    }

}

6. 실무 연결

6-1. 변수명은 곧 문서다

실무에서는 변수명이 코드 이해 속도를 크게 좌우합니다. 좋은 변수명은 주석 없이도 값의 의미를 설명합니다. 

int d = 30;
int daysAfterDelete = 30;
int accountRecoveryLimitDays = 30;

6-2. DTO와 Entity도 변수로 구성된다

Spring Boot를 공부하면 DTO나 Entity 클래스에 필드가 많이 등장합니다. 이 필드들은 모두 멤버변수입니다. 

public class UserResponseDto {

    private Long userId;
    private String loginId;
    private String email;
    private boolean active;

}

지금 배우는 변수 개념은 나중에 Request DTO, Response DTO, Entity, Form 객체를 이해하는 기초가 됩니다.

6-3. 상태값은 boolean 변수로 표현한다

boolean isActive;
boolean isDeleted;
boolean hasValidToken;
boolean canRecoverAccount;

boolean 변수명은 조건문에서 자연스럽게 읽히도록 짓는 것이 좋습니다. 

if (canRecoverAccount) {
    restoreAccount();
}

6-4. static 변수는 공통값에 사용된다

public class UserRole {

    public static final String ADMIN = "ADMIN";
    public static final String USER = "USER";

}
if (role.equals(UserRole.ADMIN)) {
    System.out.println("관리자입니다.");
}

실무에서는 모든 객체가 공유해야 하는 공통 상수나 설정값에 static을 활용합니다. 다만 무분별한 static 사용은 코드 결합도를 높일 수 있으므로 주의해야 합니다.

7. 자주 하는 실수

실수 1. 선언하지 않고 사용하기

age = 20;

변수는 사용하기 전에 반드시 선언해야 합니다.

int age = 20;

실수 2. 초기화하지 않은 지역변수 사용하기

int age;
System.out.println(age);

지역변수는 자동 기본값이 없기 때문에 반드시 초기화해야 합니다.

int age = 20;
System.out.println(age);

실수 3. 변수명 대소문자 실수

int userAge = 20;

System.out.println(userage);

Java는 대소문자를 구분합니다. userAge와 userage는 서로 다른 이름입니다.

실수 4. 예약어를 변수명으로 사용하기

int public = 10;
String class = "Java";

public, class는 Java 문법에서 이미 사용 중인 예약어이므로 변수명으로 사용할 수 없습니다.

실수 5. 의미 없는 변수명 사용하기

int a = 20;
int b = 30;

실제 프로젝트에서는 의미 있는 이름을 사용하는 것이 좋습니다.

int userAge = 20;
int maxAge = 30;

8. 면접 포인트

Q1. 변수란 무엇인가요?

답변: 변수는 메모리 상의 데이터를 저장하는 공간에 붙인 이름입니다. Java에서는 자료형과 변수명을 함께 선언하여 값을 저장하고 재사용할 수 있습니다.

Q2. 지역변수와 멤버변수의 차이는 무엇인가요?

답변: 지역변수는 메서드나 블록 안에서 선언되는 변수이며, 선언된 범위 안에서만 사용할 수 있고 반드시 초기화해야 합니다.
멤버변수는 클래스 안, 메서드 밖에서 선언되는 변수이며 객체의 속성을 표현하고 기본값으로 자동 초기화됩니다.

Q3. static 변수는 무엇인가요?

답변: static 변수는 객체가 아니라 클래스에 소속되는 변수입니다.
객체마다 따로 생성되는 것이 아니라 클래스 단위로 하나만 존재하며, 모든 객체가 공유합니다. 보통 클래스명으로 접근합니다.

Q4. 변수명을 지을 때 주의할 점은 무엇인가요?

답변: 변수명은 숫자로 시작할 수 없고, 예약어를 사용할 수 없으며, 공백이나 대부분의 특수문자를 사용할 수 없습니다.
Java에서는 보통 camelCase를 사용하고, 의미 있는 이름을 짓는 것이 중요합니다.

Q5. 좋은 변수명이 중요한 이유는 무엇인가요?

답변: 좋은 변수명은 코드의 의미를 설명하는 역할을 합니다. 의미 있는 변수명을 사용하면 주석이 적어도 코드의 의도를 쉽게 이해할 수 있고, 유지보수성이 좋아집니다.

9. 오늘의 핵심 정리

  1. 변수는 메모리 상의 데이터 저장 공간에 붙인 이름이다.
  2. 변수는 자료형과 함께 선언한다.
  3. 변수에 처음 값을 넣는 것을 초기화라고 한다.
  4. 지역변수는 메서드나 블록 안에서 선언된다.
  5. 지역변수는 반드시 초기화하고 사용해야 한다.
  6. 멤버변수는 클래스 안, 메서드 밖에서 선언된다.
  7. 멤버변수는 객체의 속성을 표현한다.
  8. static 변수는 클래스에 소속되며 모든 객체가 공유한다.
  9. 변수명은 숫자로 시작할 수 없고 예약어를 사용할 수 없다.
  10. 좋은 변수명은 코드의 의미를 설명한다.

학습 메모

이 페이지는 Java 문법의 본격적인 시작점입니다. 변수는 값을 저장하는 기능뿐 아니라 코드의 의미를 드러내는 이름표 역할도 합니다.
지역변수, 멤버변수, static 변수의 차이를 이해하면 이후 자료형, 메서드, 객체지향, DTO, Entity 구조까지 훨씬 쉽게 연결할 수 있습니다.
