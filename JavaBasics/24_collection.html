24. 컬렉션

컬렉션은 여러 객체를 저장, 삭제, 검색하기 위한 Java의 자료구조입니다.
이 페이지에서는 배열과 컬렉션의 차이, List, Set, Map,
ArrayList, HashSet, HashMap, Iterator, 제네릭까지 공부합니다.

이번 페이지의 학습 목표

컬렉션은 Java에서 여러 데이터를 다룰 때 거의 반드시 사용하는 핵심 문법입니다.
배열보다 추가, 삭제, 검색이 편리하고, 데이터의 성격에 따라 List, Set, Map 중 적절한 구조를 선택할 수 있습니다.

배열과 컬렉션 비교

고정 크기 배열과 동적 자료구조 컬렉션의 차이를 이해합니다.

List · Set · Map

반복 도구

Iterator와 Enumeration의 역할을 이해합니다.

제네릭

컬렉션에 저장할 타입을 제한해 안전한 코드를 작성합니다.

1. 컬렉션의 기본 개념

1-1. 컬렉션이란?

컬렉션은 여러 객체를 저장하고 관리하기 위한 Java의 자료구조입니다. 객체를 추가, 삭제, 검색할 수 있으며 배열보다 유연하게 데이터를 다룰 수 있습니다. 

ArrayList<String> list = new ArrayList<>();

list.add("Java");
list.add("Oracle");
list.add("HTML");

컬렉션은 메모리 기반의 작은 데이터 저장소처럼 사용할 수 있습니다. 단, 프로그램이 종료되면 메모리에 저장된 컬렉션 데이터도 사라집니다.

1-2. 배열과 컬렉션의 차이

구분 배열 컬렉션
크기 고정 동적으로 변경 가능
저장 대상 기본형, 참조형 모두 가능 객체만 저장 가능
추가/삭제 불편함 편리함
검색 직접 반복문 사용 다양한 메서드 제공
대표 예 String[] ArrayList, HashSet, HashMap

1-3. 컬렉션에는 객체만 저장할 수 있다

컬렉션에는 기본형을 직접 저장할 수 없습니다. 기본형 값은 Wrapper 클래스 객체로 변환되어 저장됩니다. 

ArrayList<int> numbers = new ArrayList<>(); // 오류
ArrayList<Integer> numbers = new ArrayList<>();

numbers.add(10);
numbers.add(20);
numbers.add(30);
int 10
→ Integer.valueOf(10)
→ 컬렉션에 Integer 객체로 저장

이 과정을 오토박싱이라고 한다.

1-4. 컬렉션 계층 구조

Collection
├─ List
│   ├─ ArrayList
│   └─ Vector
│
└─ Set
    └─ HashSet

Map
├─ HashMap
└─ Hashtable

Map은 넓게는 컬렉션 프레임워크에 포함되지만, 정확히는 Collection 인터페이스를 직접 상속하지 않습니다.
학습 단계에서는 List, Set, Map을 대표 3계열로 이해하면 됩니다.

2. List 계열

2-1. List란?

List는 데이터를 순서 있게 저장하고 중복을 허용하는 구조입니다. 인덱스로 접근할 수 있기 때문에 배열과 비슷하게 사용할 수 있습니다. 

ArrayList

index 0 → Java
index 1 → Oracle
index 2 → Java
index 3 → HTML

특징:
순서 유지
중복 허용
인덱스로 접근

ArrayList<String> list = new ArrayList<>();

list.add("Java");
list.add("Oracle");
list.add("Java");

System.out.println(list);
[Java, Oracle, Java]

2-2. ArrayList

ArrayList는 List 계열에서 가장 많이 사용하는 클래스입니다. 순서가 중요하고 중복을 허용해야 하는 목록 데이터에 적합합니다. 

ArrayList<String> list = new ArrayList<>();

list.add("List0");
list.add("List1");
list.add("List2");

System.out.println(list.size());
System.out.println(list.get(0));
3
List0

2-3. ArrayList 주요 메서드

메서드 의미
add() 객체 추가
get(index) 인덱스로 객체 꺼내기
remove(index) 인덱스로 삭제
remove(object) 객체 값으로 삭제
size() 저장된 객체 수 반환
clear() 전체 삭제
contains() 포함 여부 확인

2-4. Vector

Vector도 List 계열입니다. 예전 Java에서 많이 사용되었고, 동기화가 적용되어 있습니다. 현대 Java에서는 특별한 이유가 없다면 보통 ArrayList를 더 많이 사용합니다. 

Vector<String> list = new Vector<>();

for (int i = 0; i < 10; i++) {
    list.add("List" + i);
}

System.out.println("저장된 객체 수: " + list.size());

ArrayList는 일반적인 목록 처리에 많이 사용하고, Vector는 오래된 동기화 List 계열로 이해하면 됩니다.

3. Set 계열

3-1. Set이란?

Set은 중복을 허용하지 않는 구조입니다. 일반적으로 저장 순서를 보장하지 않습니다. 

HashSet

Java
Oracle
HTML

Java를 여러 번 넣어도 한 번만 저장

특징:
중복 제거
일반적으로 순서 보장 안 함

HashSet<String> set = new HashSet<>();

set.add("Java");
set.add("Oracle");
set.add("Java");

System.out.println(set.size());
System.out.println(set);
2
[Java, Oracle]

3-2. HashSet

HashSet은 Set 계열의 대표 클래스입니다. 중복 제거가 필요할 때 자주 사용합니다. 

HashSet<Long> likedPropertyIds = new HashSet<>();

likedPropertyIds.add(101L);
likedPropertyIds.add(102L);
likedPropertyIds.add(101L);

System.out.println(likedPropertyIds.size());
2

같은 매물 ID가 여러 번 들어와도 HashSet은 중복을 제거해 한 번만 저장합니다.

4. Map 계열

4-1. Map이란?

Map은 데이터를 key와 value의 쌍으로 저장하는 구조입니다. 인덱스가 아니라 key로 value를 찾습니다. 

HashMap

key      value
U     → 일반 회원
A     → 관리자
R     → 공인중개사

특징:
key로 value 조회
key 중복 불가
value 중복 가능

HashMap<String, Integer> scores = new HashMap<>();

scores.put("홍길동", 90);
scores.put("김철수", 85);

System.out.println(scores.get("홍길동"));
90

4-2. HashMap

HashMap은 Map 계열에서 가장 많이 사용하는 클래스입니다. key로 value를 빠르게 찾고 싶을 때 사용합니다. 

HashMap<String, String> userTypes = new HashMap<>();

userTypes.put("U", "일반 회원");
userTypes.put("A", "관리자");
userTypes.put("R", "공인중개사");

System.out.println(userTypes.get("A"));
관리자

4-3. HashMap 주요 메서드

메서드 의미
put(key, value) key와 value 저장
get(key) key로 value 조회
remove(key) key에 해당하는 데이터 삭제
containsKey(key) key 존재 여부 확인
containsValue(value) value 존재 여부 확인
size() 저장된 쌍의 개수
keySet() 모든 key를 Set으로 반환

4-4. Hashtable

Hashtable은 오래된 Map 계열 클래스이고 동기화가 적용되어 있습니다. 현대 Java에서는 일반적으로 HashMap을 더 많이 사용합니다. 

Hashtable<String, Integer> table = new Hashtable<>();

for (int i = 0; i < 10; i++) {
    table.put("List" + i, i);
}

System.out.println("저장된 객체 수: " + table.size());

5-1. Collection 인터페이스 주요 메서드

메서드 의미
int size() 컬렉션에 저장된 객체의 개수 반환
boolean isEmpty() 객체가 비었으면 true, 아니면 false

5-2. Iterator

Iterator는 컬렉션에 저장된 객체를 하나씩 꺼내기 위한 반복 도구입니다.

메서드 의미
hasNext() 다음 객체가 있으면 true
next() 다음 객체를 꺼냄
remove() 현재 반복 중인 객체 삭제 
ArrayList<String> list = new ArrayList<>();

list.add("Java");
list.add("Oracle");
list.add("HTML");

Iterator<String> it = list.iterator();

while (it.hasNext()) {
    System.out.println(it.next());
}

5-4. Enumeration

Enumeration은 Iterator와 비슷하게 객체를 하나씩 꺼내는 오래된 반복 도구입니다. 주로 Vector, Hashtable 같은 오래된 클래스와 함께 사용됩니다. 

Vector<String> list = new Vector<>();

list.add("List0");
list.add("List1");
list.add("List2");

Enumeration<String> e = list.elements();

while (e.hasMoreElements()) {
    System.out.println(e.nextElement());
}

List1
List2

6. 제네릭과 컬렉션

6-1. 제네릭이란?

제네릭은 컬렉션에 어떤 타입의 객체를 저장할지 미리 정하는 문법입니다. <String>, <Integer>, <Long> 같은 형태로 작성합니다. 

ArrayList<String> list = new ArrayList<>();
HashSet<Integer> set = new HashSet<>();
HashMap<String, Integer> map = new HashMap<>();

6-2. 제네릭의 장점

ArrayList<String> list = new ArrayList<>();

list.add("Java");
list.add("Oracle");
// list.add(100); // 오류
  1. 잘못된 타입 저장을 컴파일 단계에서 막을 수 있습니다.
  2. 꺼낼 때 형변환을 줄일 수 있습니다.
  3. 코드가 더 안전하고 읽기 쉬워집니다.

6-3. 제네릭을 쓰지 않았을 때 문제

ArrayList list = new ArrayList();

list.add("Java");
list.add(100);

String value = (String) list.get(1); // 실행 중 오류 가능

타입이 섞여 들어갈 수 있기 때문에 꺼낼 때 형변환 오류가 발생할 수 있습니다.

import java.util.ArrayList;
import java.util.Iterator;

public class IteratorRemoveExample {

    public static void main(String[] args) {

        ArrayList<String> list = new ArrayList<>();

        list.add("Java");
        list.add("Oracle");
        list.add("HTML");

        Iterator<String> it = list.iterator();

        while (it.hasNext()) {
            String value = it.next();

            if (value.equals("Oracle")) {
                it.remove();
            }
        }

        System.out.println(list);

    }

}

8-5. Map 반복 예제

import java.util.HashMap;
import java.util.Iterator;
import java.util.Set;

public class MapIteratorExample {

    public static void main(String[] args) {

        HashMap<String, Integer> map = new HashMap<>();

        map.put("List0", 0);
        map.put("List1", 1);
        map.put("List2", 2);

        Set<String> keys = map.keySet();
        Iterator<String> it = keys.iterator();

        while (it.hasNext()) {
            String key = it.next();
            Integer value = map.get(key);

            System.out.println("key값: " + key + ", value값: " + value);
        }

    }

}

10. 면접 포인트

Q1. 컬렉션이란 무엇인가요?

답변: 컬렉션은 여러 객체를 저장하고 관리하기 위한 Java의 자료구조입니다. 객체를 추가, 삭제, 검색할 수 있으며 배열보다 유연하게 데이터를 다룰 수 있습니다.

Q2. 배열과 컬렉션의 차이는 무엇인가요?

답변: 배열은 크기가 고정되어 있고 기본형과 참조형을 모두 저장할 수 있습니다. 컬렉션은 크기가 동적으로 변할 수 있고 객체만 저장할 수 있습니다.
추가, 삭제, 검색이 많은 경우 컬렉션이 더 편리합니다.

Q3. List, Set, Map의 차이는 무엇인가요?

답변: List는 순서를 유지하고 중복을 허용합니다. Set은 중복을 허용하지 않으며 일반적으로 순서를 보장하지 않습니다.
Map은 key와 value의 쌍으로 데이터를 저장하고 key로 value를 조회합니다.

Q4. ArrayList와 Vector의 차이는 무엇인가요?

답변: 둘 다 List 계열이며 순서가 있고 중복을 허용합니다. Vector는 오래된 클래스이고 동기화가 적용되어 있습니다. 일반적인 상황에서는 ArrayList를 더 많이 사용합니다.

Q5. HashMap과 Hashtable의 차이는 무엇인가요?

답변: 둘 다 key-value 구조로 데이터를 저장합니다. Hashtable은 오래된 클래스이고 동기화가 적용되어 있습니다. 일반적인 상황에서는 HashMap을 더 많이 사용합니다.

Q6. Iterator는 무엇인가요?

답변: Iterator는 컬렉션에 저장된 객체를 하나씩 꺼내기 위한 반복 도구입니다. hasNext()로 다음 요소가 있는지 확인하고, next()로 요소를 꺼냅니다.

Q7. Enumeration은 무엇인가요?

답변: Enumeration은 Iterator와 비슷하게 객체를 하나씩 꺼내는 오래된 반복 도구입니다.
주로 Vector와 Hashtable에서 사용되며, 현대 Java에서는 Iterator나 향상된 for문을 더 많이 사용합니다.

Q8. 컬렉션에 int를 저장하려면 어떻게 해야 하나요?

답변: 컬렉션에는 기본형을 직접 저장할 수 없으므로 Integer를 사용해야 합니다. 예를 들어 ArrayList<Integer> list = new ArrayList<>();처럼 작성합니다.
int 값은 오토박싱되어 Integer 객체로 저장됩니다.

Q9. Map에서 key가 중복되면 어떻게 되나요?

답변: Map에서 key는 중복될 수 없습니다. 같은 key로 다시 put()하면 기존 value가 새로운 value로 덮어써집니다.

Q10. 제네릭을 사용하는 이유는 무엇인가요?

답변: 제네릭은 컬렉션에 저장할 객체 타입을 미리 제한하기 위해 사용합니다.
잘못된 타입이 저장되는 것을 컴파일 단계에서 막을 수 있고, 꺼낼 때 형변환을 줄일 수 있어 코드가 더 안전해집니다.

11. 오늘의 핵심 정리

  1. 컬렉션은 여러 객체를 저장하고 관리하기 위한 Java의 자료구조다.
  2. 컬렉션은 객체만 저장할 수 있다.
  3. 기본형은 Wrapper 클래스로 오토박싱되어 저장된다.
  4. List는 순서가 있고 중복을 허용한다.
  5. Set은 중복을 허용하지 않고 일반적으로 순서를 보장하지 않는다.
  6. Map은 key와 value의 쌍으로 저장한다.
  7. Map은 인덱스가 아니라 key로 value를 찾는다.
  8. ArrayList는 가장 많이 사용하는 List 계열 클래스다.
  9. HashSet은 중복 제거가 필요할 때 사용한다.
  10. HashMap은 key로 value를 빠르게 찾을 때 사용한다.
  11. Vector와 Hashtable은 오래된 동기화 컬렉션이다.
  12. Iterator는 컬렉션의 객체를 하나씩 꺼낼 때 사용한다.
  13. Enumeration은 오래된 반복 도구로 Vector, Hashtable에서 주로 사용된다.
  14. 제네릭을 사용하면 컬렉션에 저장할 타입을 제한할 수 있다.
  15. 실무에서는 DB 조회 결과, API 응답 목록, 중복 제거, key-value 데이터 관리에 컬렉션을 자주 사용한다.

학습 메모

이 페이지는 Java 컬렉션 단원입니다.
컬렉션은 여러 객체를 저장, 삭제, 검색하기 위한 자료구조이며, 데이터의 성격에 따라 List, Set, Map을 선택합니다.
List는 순서와 중복이 중요할 때, Set은 중복 제거가 필요할 때, Map은 key로 value를 빠르게 찾을 때 사용합니다.
컬렉션에는 객체만 저장할 수 있으므로 기본형은 Wrapper 클래스로 다뤄야 합니다.
