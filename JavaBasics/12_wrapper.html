12. Wrapper 클래스

Wrapper 클래스는 기본 자료형 값을 객체처럼 다룰 수 있도록 감싸는 클래스입니다.
이 페이지에서는 기본형과 참조형의 차이, Boxing, Unboxing,
Auto-Boxing, Auto-Unboxing, 문자열 변환, 컬렉션 활용까지 공부합니다.

이번 페이지의 학습 목표

Java에는 int, double, boolean 같은 기본 자료형이 있습니다.
하지만 컬렉션, 제네릭, DTO, Entity, null 표현처럼 객체가 필요한 상황에서는 기본형을 그대로 사용할 수 없습니다.
이때 기본형 값을 객체처럼 감싸는 클래스가 Wrapper 클래스입니다.

Wrapper 개념 이해

기본형 값을 객체처럼 사용하기 위한 Wrapper 클래스의 목적을 이해합니다.

Boxing / Unboxing

기본형과 Wrapper 객체 사이의 변환 과정을 익힙니다.

문자열 변환

parseInt(), valueOf(), String.valueOf()의 차이를 이해합니다.

실무 활용

컬렉션, DTO, Entity ID, null 처리에서 Wrapper 클래스가 필요한 이유를 익힙니다.

1. 핵심 개념 설명

1-1. Wrapper 클래스란?

Wrapper 클래스는 기본 자료형 값을 참조형 객체로 감싸는 클래스입니다. 기본형은 값 자체를 저장하지만, Wrapper 클래스는 그 값을 객체처럼 다룰 수 있게 해줍니다. 

int num = 10;
Integer numObj = 10;
int       → 기본형
Integer   → int 값을 감싸는 Wrapper 클래스

int는 기본형이고, Integer는 int 값을 객체처럼 다루기 위한 클래스입니다.

1-2. 기본형과 Wrapper 클래스 대응

Java의 8가지 기본 자료형은 각각 대응되는 Wrapper 클래스를 가지고 있습니다.

기본 자료형 Wrapper 클래스 주의할 점
byte Byte 첫 글자 대문자
short Short 첫 글자 대문자
int Integer Int가 아님
long Long 첫 글자 대문자
float Float 첫 글자 대문자
double Double 첫 글자 대문자
char Character Char가 아님
boolean Boolean 첫 글자 대문자

암기 포인트: int → Integer, char → Character는 이름이 다르므로 특히 주의합니다.

1-3. 기본형과 참조형 차이

기본형은 값 자체를 저장하고, 참조형은 객체를 가리키는 참조값을 저장합니다. Wrapper 클래스는 참조형이므로 객체처럼 사용할 수 있고, null도 가질 수 있습니다. 

기본형 int

num
┌──────┐
│ 10   │
└──────┘


Wrapper Integer

numObj
┌────────────────┐
│ Integer 객체   │
│ value = 10     │
└────────────────┘

1-4. Boxing

Boxing은 기본형 값을 Wrapper 객체로 감싸는 과정입니다. 예전에는 직접 Wrapper 객체를 만들어야 했습니다. 

int a = 10;

Integer obj = Integer.valueOf(a);
int 10
↓ Boxing
Integer 객체(10)

예전 방식으로는 new Integer(a)처럼 객체를 직접 만들기도 했지만, 현재는 Integer.valueOf() 또는 Auto-Boxing을 사용하는 것이 일반적입니다.

1-5. Unboxing

Unboxing은 Wrapper 객체 안에 들어 있는 기본형 값을 꺼내는 과정입니다. 

Integer obj = Integer.valueOf(10);

int a = obj.intValue();
Integer 객체(10)
↓ Unboxing
int 10

1-6. Auto-Boxing과 Auto-Unboxing

JDK 5 이후부터는 기본형과 Wrapper 클래스 사이의 변환을 Java가 자동으로 처리해줍니다.
기본형이 Wrapper 객체로 자동 변환되는 것을 Auto-Boxing, Wrapper 객체가 기본형으로 자동 변환되는 것을 Auto-Unboxing이라고 합니다. 

int a = 10;

Integer obj = a; // Auto-Boxing

int b = obj;     // Auto-Unboxing
Auto-Boxing
int → Integer

Auto-Unboxing
Integer → int

주의: 자동으로 변환되더라도 내부적으로는 기본형과 객체 사이의 변환이 일어납니다. 특히 Wrapper 객체가 null일 때 Auto-Unboxing이 발생하면 오류가 날 수 있습니다.

2. Wrapper 클래스가 필요한 이유

2-1. 컬렉션은 객체만 저장할 수 있다

Java의 컬렉션은 기본형을 직접 저장할 수 없습니다. 따라서 int를 저장하려면 Integer를 사용해야 합니다. 

ArrayList<int> numbers = new ArrayList<int>();
ArrayList<Integer> numbers = new ArrayList<Integer>();

JDK 5 이후에는 Auto-Boxing 덕분에 아래처럼 자연스럽게 사용할 수 있습니다. 

numbers.add(10);
numbers.add(20);

int first = numbers.get(0);
numbers.add(10)
10 → Integer로 Auto-Boxing

int first = numbers.get(0)
Integer → int로 Auto-Unboxing

2-2. null을 표현할 수 있다

기본형은 null을 저장할 수 없습니다. 하지만 Wrapper 클래스는 참조형이므로 null을 저장할 수 있습니다. 

int age = null;
Integer age = null;

실무에서는 이 차이가 중요합니다. 예를 들어 사용자가 나이를 아직 입력하지 않았다면 null로 표현할 수 있습니다. 

Integer age = null
→ 사용자가 아직 나이를 입력하지 않음

int age = 0
→ 실제 나이가 0인지, 입력하지 않은 것인지 구분 어려움

2-3. Wrapper 클래스는 메서드를 제공한다

Wrapper 클래스는 클래스이기 때문에 유용한 메서드를 제공합니다. 대표적으로 문자열을 숫자로 바꾸는 parseInt()가 있습니다. 

String str = "100";

int num = Integer.parseInt(str);

System.out.println(num + 20);
120

3. 문자열과 숫자 변환

3-1. 문자열을 기본형으로 변환하기

웹 요청, 콘솔 입력, 파일 데이터, JSON 데이터는 숫자처럼 보여도 문자열로 들어오는 경우가 많습니다. 계산하려면 문자열을 숫자형으로 변환해야 합니다. 

int num = Integer.parseInt("100");
long longNum = Long.parseLong("100000");
double doubleNum = Double.parseDouble("3.14");
float floatNum = Float.parseFloat("3.14");
boolean bool = Boolean.parseBoolean("true");
변환 목적 사용 메서드
문자열 → int Integer.parseInt()
문자열 → long Long.parseLong()
문자열 → double Double.parseDouble()
문자열 → float Float.parseFloat()
문자열 → boolean Boolean.parseBoolean()

3-2. 기본형을 문자열로 변환하기

숫자나 boolean 값을 문자열로 바꿀 때는 String.valueOf()를 사용할 수 있습니다. 

int num = 100;

String str1 = String.valueOf(num);
String str2 = Integer.toString(num);
String str3 = num + "";

추천: 초급 단계에서는 의미가 분명한 String.valueOf()를 먼저 익히는 것이 좋습니다.

3-3. parseInt와 valueOf의 차이

Integer.parseInt()와 Integer.valueOf()는 비슷해 보이지만 반환 타입이 다릅니다. 

int num = Integer.parseInt("100");
Integer obj = Integer.valueOf("100");
메서드 반환 결과 의미
Integer.parseInt("100") 기본형 int 계산용 숫자 값으로 변환
Integer.valueOf("100") Wrapper 객체 Integer 객체 형태의 숫자로 변환 
parseInt → int
valueOf  → Integer

5. 코드 예제

5-1. Auto-Boxing / Auto-Unboxing 예제

public class AutoBoxingExample {

    public static void main(String[] args) {

        int a = 10;

        Integer obj = a; // Auto-Boxing

        int b = obj; // Auto-Unboxing

        System.out.println(obj);
        System.out.println(b);

    }

}

5-2. 문자열을 숫자로 변환

public class ParseExample {

        String str = "100";

        int num = Integer.parseInt(str);

        System.out.println(num + 20);

    }

}

5-3. 기본형을 문자열로 변환

public class StringValueOfExample {

    public static void main(String[] args) {

        int num = 100;

    }

}

숫자 100이 문자열 "100"으로 바뀌었기 때문에 + 20은 숫자 덧셈이 아니라 문자열 결합이 됩니다. 

10020

5-4. 컬렉션에서 Wrapper 사용

import java.util.ArrayList;
import java.util.List;

public class WrapperCollectionExample {

    public static void main(String[] args) {

        List<Integer> scores = new ArrayList<>();

        scores.add(90);
        scores.add(80);
        scores.add(70);

        int total = 0;

        for (int i = 0; i < scores.size(); i++) {
            total += scores.get(i);
        }

        System.out.println(total);

    }

}
240

5-5. null 체크 예제

public class WrapperNullExample {

    public static void main(String[] args) {

        Integer age = null;

        if (age != null) {
            int realAge = age;
            System.out.println(realAge);
        } else {
            System.out.println("나이 정보가 없습니다.");
        }

    }

}

6. 실무 연결

6-1. DTO에서 Wrapper 사용

DTO에서는 기본형보다 Wrapper 클래스를 사용하는 경우가 많습니다. 이유는 null을 표현할 수 있기 때문입니다. 

public class UserRequestDto {

    private Long userId;
    private Integer age;
    private Boolean active;

}
age = null → 사용자가 아직 입력하지 않음
age = 0    → 실제 값이 0

6-2. Entity ID에서 Long 사용

Spring/JPA에서는 식별자 타입으로 Long을 자주 사용합니다. 아직 DB에 저장되지 않은 객체는 ID가 없기 때문에 null로 표현할 수 있어야 합니다. 

private Long userId;
private Long propertyId;

6-3. 웹 요청 값 변환

웹 요청이나 콘솔 입력은 문자열로 들어오는 경우가 많습니다. 숫자 계산이나 비교를 하려면 Wrapper 클래스의 변환 메서드를 사용합니다. 

String inputAge = "20";

int age = Integer.parseInt(inputAge);

6-4. null 안전 처리

Wrapper 클래스는 null을 가질 수 있으므로, 기본형으로 꺼내기 전에 null 체크가 필요합니다. 

Integer age = null;

if (age != null) {
    int realAge = age;
    System.out.println(realAge);
} else {
    System.out.println("나이 정보가 없습니다.");
}

7. 자주 하는 실수

실수 1. 컬렉션에 기본형을 직접 사용

ArrayList<int> list = new ArrayList<>();
ArrayList<Integer> list = new ArrayList<>();

실수 2. null인 Wrapper를 기본형으로 언박싱

Integer age = null;

int realAge = age;

위 코드는 실행 중 NullPointerException이 발생할 수 있습니다. 

if (age != null) {
    int realAge = age;
}

실수 3. 숫자가 아닌 문자열을 parseInt에 넣음

int num = Integer.parseInt("abc");

숫자가 아닌 문자열을 숫자로 바꾸려 하면 NumberFormatException이 발생합니다.

실수 4. Wrapper 객체를 무조건 ==로 비교

Integer a = 1000;
Integer b = 1000;

System.out.println(a == b);

Wrapper 클래스는 객체이므로 값 비교에는 equals()를 사용하는 것이 안전합니다. 

System.out.println(a.equals(b));

실수 5. Wrapper 클래스 이름을 잘못 기억함

잘못된 생각:
int  → Int
char → Char

올바른 표현:
int  → Integer
char → Character

8. 면접 포인트

Q1. Wrapper 클래스란 무엇인가요?

답변: Wrapper 클래스는 기본 자료형 값을 객체처럼 사용할 수 있도록 감싸는 클래스입니다.
예를 들어 int의 Wrapper 클래스는 Integer이고, double의 Wrapper 클래스는 Double입니다.
컬렉션처럼 객체만 저장할 수 있는 구조에서 기본형 값을 사용하기 위해 필요합니다.

Q2. Boxing과 Unboxing이란 무엇인가요?

답변: Boxing은 기본형 값을 Wrapper 객체로 감싸는 것이고, Unboxing은 Wrapper 객체에서 기본형 값을 꺼내는 것입니다.
JDK 5 이후에는 Auto-Boxing과 Auto-Unboxing이 지원되어 자동 변환이 가능합니다.

Q3. 컬렉션에서 int 대신 Integer를 쓰는 이유는 무엇인가요?

답변: Java 컬렉션은 객체만 저장할 수 있고 기본형은 직접 저장할 수 없습니다. 그래서 int 값을 저장하려면 Wrapper 클래스인 Integer를 사용해야 합니다.

Q4. Integer.parseInt()와 Integer.valueOf()의 차이는 무엇인가요?

답변: Integer.parseInt()는 문자열을 기본형 int로 변환합니다. Integer.valueOf()는 문자열을 Wrapper 객체 Integer로 변환합니다.

Q5. Wrapper 클래스 사용 시 null과 관련해 주의할 점은 무엇인가요?

답변: Wrapper 클래스는 참조형이므로 null을 가질 수 있습니다. 하지만 null인 Wrapper 객체를 기본형으로 Auto-Unboxing하면 NullPointerException이 발생할 수 있습니다. 따라서 언박싱 전에 null 체크가 필요합니다.

Q6. Wrapper 객체 비교 시 주의할 점은 무엇인가요?

답변: Wrapper 클래스는 객체이므로 값 비교에는 equals()를 사용하는 것이 안전합니다. ==는 객체의 참조 비교가 될 수 있어 의도와 다른 결과가 나올 수 있습니다.

9. 오늘의 핵심 정리

  1. Wrapper 클래스는 기본형 값을 객체처럼 사용하기 위한 클래스다.
  2. int의 Wrapper 클래스는 Integer다.
  3. char의 Wrapper 클래스는 Character다.
  4. Boxing은 기본형을 Wrapper 객체로 감싸는 것이다.
  5. Unboxing은 Wrapper 객체에서 기본형 값을 꺼내는 것이다.
  6. JDK 5 이후 Auto-Boxing과 Auto-Unboxing이 지원된다.
  7. 컬렉션은 객체만 저장하므로 Integer, Long 같은 Wrapper 클래스를 사용한다.
  8. Wrapper 클래스는 null을 표현할 수 있다.
  9. null인 Wrapper를 기본형으로 언박싱하면 NullPointerException이 발생할 수 있다.
  10. 문자열을 숫자로 바꿀 때는 parseInt, parseLong, parseDouble 등을 사용한다.
  11. 기본형을 문자열로 바꿀 때는 String.valueOf()를 사용할 수 있다.
  12. Wrapper 객체의 값 비교는 equals()를 사용하는 것이 안전하다.

학습 메모

이 페이지는 기본 자료형을 객체처럼 다루기 위한 Wrapper 클래스 단원입니다.
Integer, Long, Double, Boolean 같은 Wrapper 클래스는 컬렉션, DTO, Entity, 문자열 변환, null 처리에서 자주 사용됩니다.
특히 int → Integer, char → Character, Auto-Boxing, Auto-Unboxing, null 언박싱 오류는 반드시 기억해야 합니다.
