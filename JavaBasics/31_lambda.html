31. 람다식

람다식은 함수형 인터페이스의 추상 메서드를 짧게 구현하는 문법입니다.
익명 클래스, 함수형 인터페이스, @FunctionalInterface,
람다식 생략 규칙, java.util.function, Stream 활용까지 공부합니다.

이번 페이지의 학습 목표

람다식은 JDK 8부터 지원된 문법으로, 이름 없는 함수를 간결하게 표현합니다.
단독으로 아무 곳에서나 쓰는 문법이 아니라, 반드시 추상 메서드가 하나뿐인 함수형 인터페이스와 함께 사용해야 합니다.

람다식 개념

익명 클래스를 더 짧게 표현하는 구조를 이해합니다.

함수형 인터페이스

추상 메서드가 하나인 인터페이스와 람다식의 관계를 익힙니다.

생략 규칙

매개변수 괄호, 자료형, 중괄호, return 생략 규칙을 정리합니다.

실무 연결

Runnable, Predicate, Function, Stream에서 람다식을 활용합니다.

1. 람다식 기본 개념

1-1. 람다식이란?

람다식은 익명 함수를 간결하게 표현하는 문법입니다. 기존 메서드처럼 이름과 반환 타입을 모두 작성하지 않고, 매개변수와 실행 내용만 짧게 표현합니다. 

반환타입 메서드명(자료형 매개변수들) {
    실행문;
}
(매개변수들) -> {
    실행문;
}

핵심: 람다식은 독립적인 일반 메서드를 새로 만드는 문법이 아니라, 함수형 인터페이스의 추상 메서드를 짧게 구현하는 표현식입니다.

1-2. 메서드와 람다식 비교

public int calc(int a, int b) {
    return a + b;
}
(a, b) -> a + b
(a, b) -> a + b

(a, b)
→ 매개변수

->
→ 람다 연산자

a + b
→ 실행 내용 또는 반환값

1-3. 람다식을 사용하는 이유

  1. 간단한 기능 구현을 짧게 작성할 수 있습니다.
  2. 익명 클래스보다 코드가 간결해집니다.
  3. 일회성 기능 구현에 적합합니다.
  4. 컬렉션, Stream, 스레드 작업에서 자주 사용됩니다.
  5. 코드를 데이터처럼 변수에 저장하거나 메서드에 전달할 수 있습니다.

2. 구현 클래스 → 익명 클래스 → 람다식

2-1. 예제 인터페이스

@FunctionalInterface
interface MyInterface {
    int calc(int a, int b);
}

이 인터페이스는 calc()라는 추상 메서드 하나만 가지고 있습니다. 따라서 람다식으로 구현할 수 있습니다.

2-2. 구현 클래스를 따로 만드는 방식

class MyInterfaceImpl implements MyInterface {

    @Override
    public int calc(int a, int b) {
        return a + b;
    }

}
MyInterface my = new MyInterfaceImpl();

System.out.println(my.calc(1, 2));
3

재사용하기에는 좋지만, 간단한 기능 하나 때문에 별도 클래스를 만들어야 해서 코드가 길어질 수 있습니다.

2-3. 익명 클래스 방식

MyInterface my2 = new MyInterface() {

    @Override
    public int calc(int a, int b) {
        return a + b;
    }

};

System.out.println(my2.calc(1, 2));
3

별도 구현 클래스 파일은 필요 없지만, 일회성 구현치고는 여전히 문법이 무겁습니다.

2-4. 람다식 방식

MyInterface my3 = (a, b) -> a + b;

System.out.println(my3.calc(1, 2));
3
구현 클래스
→ 재사용 가능하지만 코드가 길다.

익명 클래스
→ 별도 파일은 줄지만 여전히 코드가 길다.

람다식
→ 가장 짧고 간결하다.

3. 함수형 인터페이스

3-1. 함수형 인터페이스란?

함수형 인터페이스는 추상 메서드를 하나만 가진 인터페이스입니다. 람다식은 이 하나의 추상 메서드를 구현합니다. 

@FunctionalInterface
interface Calculator {
    int calc(int a, int b);
}
Calculator plus = (a, b) -> a + b;
Calculator plus = (a, b) -> a + b;

의미:
Calculator 인터페이스의 calc(int a, int b)를
람다식으로 구현한 것

3-2. 람다식은 익명 클래스와 같은 역할

Calculator plus = new Calculator() {

    @Override
    public int calc(int a, int b) {
        return a + b;
    }

};
Calculator plus = (a, b) -> a + b;

3-3. 추상 메서드가 2개면 람다식 사용 불가

@FunctionalInterface
interface BadInterface {
    void method1();
    void method2(); // 오류
}

람다식은 함수형 인터페이스의 단 하나의 추상 메서드를 구현하는 문법입니다. 추상 메서드가 여러 개면 어떤 메서드를 구현해야 하는지 알 수 없습니다.

3-4. @FunctionalInterface

@FunctionalInterface는 이 인터페이스가 함수형 인터페이스여야 한다는 표시입니다. 추상 메서드가 2개 이상이면 컴파일 에러를 발생시켜 실수를 막아줍니다. 

@FunctionalInterface
→ 추상 메서드가 하나인지 검사
→ 람다식 사용 가능 여부를 명확하게 표시
→ 협업 시 의도를 분명하게 전달

4. 람다식 문법과 생략 규칙

4-1. 기본 문법

(매개변수들) -> {
    실행문;
}
(a, b) -> {
    return a + b;
}
(a, b) -> a + b

4-2. 생략 규칙 표

상황 생략 가능 여부 예시
매개변수 자료형 생략 가능 (int a, int b) -> a + b
(a, b) -> a + b
매개변수 1개 괄호 생략 가능 value -> value.length()
매개변수 없음 빈 괄호 필수 () -> System.out.println("실행")
매개변수 2개 이상 괄호 필수 (a, b) -> a + b
실행문 1개 중괄호 생략 가능 name -> System.out.println(name)
한 줄 return return과 중괄호 함께 생략 가능 (a, b) -> a + b

4-3. 매개변수 자료형 생략

(int a, int b) -> a + b
(a, b) -> a + b

함수형 인터페이스의 추상 메서드에 매개변수 타입이 이미 정해져 있으므로, 컴파일러가 타입을 추론할 수 있습니다.

4-4. 매개변수가 없는 경우

() -> System.out.println("실행")

매개변수가 없으면 빈 괄호 ()를 반드시 써야 합니다.

4-5. 매개변수가 하나인 경우

(value) -> System.out.println(value)
value -> System.out.println(value)

매개변수가 하나면 괄호를 생략할 수 있습니다.

4-6. return 생략 규칙 주의

(a, b) -> return a + b; // 오류
(a, b) -> a + b
(a, b) -> {
    return a + b;
}

return을 쓰려면 중괄호가 필요합니다. 중괄호를 생략하면 return도 함께 생략해야 합니다.

5. java.util.function 패키지

5-1. 대표 함수형 인터페이스

인터페이스 추상 메서드 의미
Runnable void run() 매개변수도 없고 반환값도 없음
Supplier<T> T get() 매개변수는 없고 반환값만 있음
Consumer<T> void accept(T t) 매개변수만 있고 반환값은 없음
Function<T, R> R apply(T t) 매개변수 하나를 받아 결과 반환
Predicate<T> boolean test(T t) 조건식 표현, boolean 반환
BiConsumer<T, U> void accept(T t, U u) 매개변수 2개, 반환값 없음
BiPredicate<T, U> boolean test(T t, U u) 매개변수 2개, boolean 반환
BiFunction<T, U, R> R apply(T t, U u) 매개변수 2개를 받아 결과 반환

5-2. Runnable

Runnable task = () -> System.out.println("작업 실행");

task.run();
Runnable
→ 입력 없음
→ 반환 없음
→ run()

5-3. Supplier

Supplier<String> supplier = () -> "Hello Lambda";

System.out.println(supplier.get());
Supplier<T>
→ 입력 없음
→ T 타입 결과 반환

5-4. Consumer

Consumer<String> printer = value -> System.out.println(value);

printer.accept("Java");
Consumer<T>
→ T 타입 입력 있음
→ 반환 없음

5-5. Function

Function<String, Integer> lengthFunction = value -> value.length();

System.out.println(lengthFunction.apply("Lambda"));
Function<T, R>
→ T 타입 입력
→ R 타입 결과 반환

5-6. Predicate

Predicate<String> predicate = value -> value.indexOf("@") != -1;

String email = "kildong@naver.com";

System.out.println(predicate.test(email) ? "이메일 형식이다." : "이메일 형식이 아니다.");
Predicate<T>
→ T 타입 입력
→ boolean 반환
→ 조건 검사에 적합

5-7. BiFunction

BiFunction<Integer, Integer, Integer> plus = (a, b) -> a + b;

System.out.println(plus.apply(10, 20));
BiFunction<T, U, R>
→ T 타입 입력 1개
→ U 타입 입력 1개
→ R 타입 결과 반환

6. 실무 연결

6-1. Thread와 람다식

Thread thread = new Thread(() -> {
    for (int i = 1; i <= 5; i++) {
        System.out.println("스레드 작업: " + i);
    }
});

thread.start();

Runnable은 추상 메서드가 run() 하나뿐이므로 람다식으로 표현할 수 있습니다.

6-2. 컬렉션 반복 처리

List<String> names = Arrays.asList("Java", "Spring", "Oracle");

names.forEach(name -> System.out.println(name));

컬렉션의 각 요소를 처리할 때 람다식을 사용하면 반복문보다 간결하게 표현할 수 있습니다.

6-3. Stream filter와 forEach

List<String> names = Arrays.asList("Java", "Spring", "Oracle");

names.stream()
    .filter(name -> name.length() >= 5)
    .forEach(name -> System.out.println(name));
Spring
Oracle
filter(name -> name.length() >= 5)
→ 조건을 전달
→ Predicate 형태

forEach(name -> System.out.println(name))
→ 처리 방법을 전달
→ Consumer 형태

6-4. 이메일 형식 검증

Predicate<String> emailCheck = value -> value.indexOf("@") != -1;

String email = "test@example.com";

if (emailCheck.test(email)) {
    System.out.println("이메일 형식입니다.");
} else {
    System.out.println("이메일 형식이 아닙니다.");
}

6-5. 계산 기능 교체

@FunctionalInterface
interface Calculator {
    int calc(int a, int b);
}

Calculator plus = (a, b) -> a + b;
Calculator minus = (a, b) -> a - b;
Calculator multiply = (a, b) -> a * b;

System.out.println(plus.calc(10, 5));
System.out.println(minus.calc(10, 5));
System.out.println(multiply.calc(10, 5));

하나의 함수형 인터페이스를 기준으로 여러 계산 방식을 간단히 교체할 수 있습니다.

7. 코드 예제

7-1. 구현 클래스 → 익명 클래스 → 람다식

@FunctionalInterface
interface MyInterface {
    int calc(int a, int b);
}

class MyInterfaceImpl implements MyInterface {

    @Override
    public int calc(int a, int b) {
        return a + b;
    }

}

public class LambdaBasicExample {

    public static void main(String[] args) {

        MyInterface my1 = new MyInterfaceImpl();

        MyInterface my2 = new MyInterface() {
            @Override
            public int calc(int a, int b) {
                return a + b;
            }
        };

        MyInterface my3 = (a, b) -> a + b;

        System.out.println(my1.calc(1, 2));
        System.out.println(my2.calc(1, 2));
        System.out.println(my3.calc(1, 2));
    }
}

7-2. Runnable 람다식

public class RunnableLambdaExample {

    public static void main(String[] args) {

        Runnable task = () -> System.out.println("작업 실행");

        task.run();

    }
}

7-3. Predicate 예제

import java.util.function.Predicate;

public class PredicateExample {

    public static void main(String[] args) {

        Predicate<String> hasAt = value -> value.indexOf("@") != -1;

        String email = "kildong@naver.com";

        System.out.println(hasAt.test(email) ? "이메일 형식이다." : "이메일 형식이 아니다.");

    }
}

7-4. Function 예제

import java.util.function.Function;

public class FunctionExample {

    public static void main(String[] args) {

        Function<String, Integer> lengthFunction = value -> value.length();

        System.out.println(lengthFunction.apply("Lambda"));

    }
}

7-5. Stream과 람다식

import java.util.Arrays;
import java.util.List;

public class StreamLambdaExample {

    public static void main(String[] args) {

        List<String> names = Arrays.asList("Java", "Spring", "Oracle");

        names.stream()
            .filter(name -> name.length() >= 5)
            .forEach(name -> System.out.println(name));

    }
}

8. 자주 하는 실수

실수 1. 추상 메서드가 2개인 인터페이스에 람다식을 사용함

interface BadInterface {
    void method1();
    void method2();
}

BadInterface bad = () -> System.out.println("실행"); // 오류

실수 2. 매개변수가 2개인데 괄호 생략

a, b -> a + b // 오류
(a, b) -> a + b

실수 3. 매개변수가 없는데 괄호 생략

-> System.out.println("실행") // 오류
() -> System.out.println("실행")

실수 4. 중괄호 없이 return 사용

(a, b) -> return a + b; // 오류
(a, b) -> a + b
(a, b) -> {
    return a + b;
}

실수 5. 반환 타입을 직접 씀

int (a, b) -> a + b // 오류

람다식에는 반환 타입을 직접 쓰지 않습니다. 대상 함수형 인터페이스를 통해 컴파일러가 추론합니다.

실수 6. 람다식을 아무 변수에나 저장하려 함

var calc = (a, b) -> a + b; // 상황에 따라 오류
Calculator calc = (a, b) -> a + b;

람다식은 반드시 함수형 인터페이스 타입에 대입되거나, 함수형 인터페이스를 요구하는 메서드 인자로 전달되어야 합니다.

9. 면접 포인트

Q1. 람다식이란 무엇인가요?

답변: 람다식은 JDK 8부터 지원되는 익명 함수 표현식입니다. 함수형 인터페이스의 추상 메서드를 간결하게 구현할 때 사용합니다.

Q2. 함수형 인터페이스란 무엇인가요?

답변: 함수형 인터페이스는 추상 메서드를 하나만 가진 인터페이스입니다. 람다식은 이 하나의 추상 메서드를 구현하는 방식으로 사용됩니다.

Q3. @FunctionalInterface는 무엇인가요?

답변: @FunctionalInterface는 해당 인터페이스가 함수형 인터페이스인지 컴파일러가 검사하도록 하는 애노테이션입니다. 추상 메서드가 2개 이상이면 컴파일 에러가 발생합니다.

Q4. 익명 클래스와 람다식의 차이는 무엇인가요?

답변: 익명 클래스는 이름 없는 클래스를 즉석에서 만들어 메서드를 오버라이딩하는 방식입니다. 람다식은 함수형 인터페이스의 단 하나의 추상 메서드를 더 간결하게 구현하는 표현식입니다. 람다식은 코드가 짧지만 함수형 인터페이스에만 사용할 수 있습니다.

Q5. 람다식에서 매개변수 타입을 생략할 수 있는 이유는 무엇인가요?

답변: 람다식이 대입되는 함수형 인터페이스의 추상 메서드에 이미 매개변수 타입이 정의되어 있기 때문입니다. 컴파일러가 대상 타입을 보고 매개변수 타입을 추론할 수 있습니다.

Q6. 람다식의 생략 규칙은 무엇인가요?

답변: 매개변수 타입은 생략할 수 있습니다. 매개변수가 하나면 괄호를 생략할 수 있습니다. 실행문이 한 줄이면 중괄호를 생략할 수 있습니다. 한 줄짜리 return문이면 return과 중괄호를 함께 생략할 수 있습니다. 단, 매개변수가 없거나 두 개 이상이면 괄호가 필요합니다.

Q7. Runnable은 람다식으로 표현할 수 있나요?

답변: 네. Runnable은 run()이라는 추상 메서드 하나만 가진 함수형 인터페이스이므로 람다식으로 표현할 수 있습니다. 예를 들어 Runnable r = () -> System.out.println("실행");처럼 사용할 수 있습니다.

Q8. Predicate는 언제 사용하나요?

답변: Predicate는 조건 검사를 표현할 때 사용합니다. 하나의 값을 받아 boolean을 반환하는 test() 메서드를 가지고 있습니다. Stream의 filter() 메서드에서 자주 사용됩니다.

Q9. Function과 Consumer의 차이는 무엇인가요?

답변: Function은 입력값을 받아 결과값을 반환합니다. Consumer는 입력값을 받아 처리만 하고 반환값은 없습니다.

Q10. 람다식은 실무에서 어디에 자주 사용되나요?

답변: 컬렉션 반복 처리, Stream의 filter, map, forEach, 이벤트 처리, 스레드 작업 정의, 조건 검증, 간단한 계산 로직 전달 등에 자주 사용됩니다.

10. 오늘의 핵심 정리

  1. 람다식은 JDK 8부터 지원된다.
  2. 람다식은 익명 함수를 생성하기 위한 표현식이다.
  3. 람다식은 함수형 인터페이스의 추상 메서드를 간결하게 구현한다.
  4. 함수형 인터페이스는 추상 메서드를 하나만 가진 인터페이스다.
  5. @FunctionalInterface는 함수형 인터페이스 조건을 컴파일러가 검사하게 한다.
  6. 람다식 기본 구조는 (매개변수들) -> { 실행문; }이다.
  7. 매개변수의 자료형은 생략할 수 있다.
  8. 매개변수가 한 개면 괄호를 생략할 수 있다.
  9. 매개변수가 없거나 두 개 이상이면 괄호가 필요하다.
  10. 실행문이 한 줄이면 중괄호를 생략할 수 있다.
  11. 실행문이 한 줄인 return문이면 return과 중괄호를 함께 생략할 수 있다.
  12. Runnable은 매개변수도 반환값도 없는 대표적인 함수형 인터페이스다.
  13. Supplier는 입력 없이 값을 반환한다.
  14. Consumer는 입력은 있고 반환은 없다.
  15. Function은 입력 하나를 받아 결과 하나를 반환한다.
  16. Predicate는 입력 하나를 받아 boolean을 반환한다.
  17. BiFunction은 입력 두 개를 받아 결과 하나를 반환한다.
  18. 람다식은 Stream의 filter, map, forEach 같은 메서드에서 자주 사용된다.

학습 메모

이 페이지는 Java 람다식 단원입니다. 람다식은 함수형 인터페이스의 추상 메서드를 짧게 구현하는 문법입니다.
추상 메서드가 하나뿐인 인터페이스에만 사용할 수 있으며, @FunctionalInterface를 붙이면 컴파일러가 함수형 인터페이스 조건을 검사합니다.
람다식은 Runnable, Predicate, Function, Consumer, Stream의 filter, forEach 같은 곳에서 자주 사용됩니다.
