32. Stream API

Stream API는 배열과 컬렉션 데이터를 원본은 유지한 채 흐름처럼 처리하는 도구입니다.
stream(), 중간 연산, 최종 연산, filter(), map(), flatMap(),
reduce(), Optional, 메서드 참조까지 공부합니다.

이번 페이지의 학습 목표

Stream API는 반복문을 직접 작성하지 않고 배열이나 컬렉션 데이터를 처리하는 기능입니다.
데이터 원본을 직접 바꾸지 않고, 생성 → 중간 연산 → 최종 연산의 흐름으로 데이터를 걸러내고, 변환하고, 집계하고, 출력할 수 있습니다.

Stream 개념

배열과 컬렉션 데이터를 흐름처럼 처리하는 구조를 이해합니다.

중간 연산

filter, map, flatMap, sorted를 익힙니다.

최종 연산

forEach, count, reduce, max, findFirst를 익힙니다.

실무 감각

Stream 재사용 불가, Optional 처리, 메서드 참조를 이해합니다.

1. Stream API 기본 개념

1-1. Stream API란?

Stream API는 배열이나 컬렉션에 저장된 데이터를 반복문 없이 처리할 수 있게 해주는 Java의 데이터 처리 도구입니다. JDK 8부터 지원되며, 람다식과 함께 자주 사용됩니다. 

List<String> names = Arrays.asList("Java", "Spring", "Oracle");

names.stream()
    .filter(name -> name.length() >= 5)
    .forEach(System.out::println);
Spring
Oracle
names를 Stream으로 만들고
→ 길이가 5 이상인 값만 남기고
→ 하나씩 출력한다.

1-2. Stream의 핵심 특징

  1. 배열이나 컬렉션 데이터를 처리합니다.
  2. 원본 데이터를 직접 변경하지 않습니다.
  3. 반복문을 직접 작성하지 않고 내부 반복으로 처리합니다.
  4. 람다식과 함께 자주 사용합니다.
  5. 중간 연산과 최종 연산으로 구성됩니다.
  6. 최종 연산이 실행되면 Stream은 닫힙니다.
  7. 한 번 닫힌 Stream은 다시 사용할 수 없습니다.

1-3. Stream 처리 흐름

데이터 원본
[Java, Spring, Oracle, HTML]

        ↓ stream()

Stream 생성
Java → Spring → Oracle → HTML

        ↓ filter(name -> name.length() >= 5)

중간 연산
Spring → Oracle

        ↓ forEach(System.out::println)

최종 연산
Spring 출력
Oracle 출력

        ↓

Stream 닫힘

2. Stream과 반복문 비교

2-1. 반복문 방식

List<String> names = Arrays.asList("Java", "Spring", "Oracle");

for (String name : names) {
    if (name.length() >= 5) {
        System.out.println(name);
    }
}

2-2. Stream 방식

names.stream()
    .filter(name -> name.length() >= 5)
    .forEach(System.out::println);

Stream은 “어떻게 반복할지”보다 “무엇을 처리할지”를 중심으로 코드를 작성합니다.

2-3. 비교 표

구분 반복문 Stream
처리 방식 외부 반복 내부 반복
작성 방식 개발자가 반복 흐름 직접 작성 Stream이 내부적으로 반복 처리
주요 문법 for, while filter, map, forEach
람다식 활용 필수 아님 자주 사용
가독성 조건이 많아지면 길어짐 처리 흐름을 체인으로 표현

3. Stream 생성 방법

3-1. 컬렉션에서 Stream 생성

List<String> names = Arrays.asList("Java", "Spring", "Oracle");

Stream<String> stream = names.stream();

3-2. 배열에서 Stream 생성

String[] names = {"Java", "Spring", "Oracle"};

Stream<String> stream1 = Stream.of(names);
Stream<String> stream2 = Arrays.stream(names);

3-3. 배열 일부만 Stream으로 만들기

Arrays.stream(names, 1, 3)
    .forEach(System.out::println);

Arrays.stream(array, start, end)에서 시작 인덱스는 포함하고, 끝 인덱스는 포함하지 않습니다.

4. 중간 연산과 최종 연산

4-1. 중간 연산

중간 연산은 Stream을 다시 반환하는 연산입니다. 여러 개를 연결할 수 있고, 최종 연산이 실행되기 전까지 실제 처리가 지연될 수 있습니다. 

중간 연산
→ 데이터를 걸러내거나 변환
→ Stream 반환
→ 계속 연결 가능

대표:
filter()
map()
flatMap()
distinct()
sorted()
limit()
skip()

4-2. 최종 연산

최종 연산은 Stream을 실제로 소비하거나 결과를 만드는 연산입니다. 최종 연산 후 Stream은 닫히므로 다시 사용할 수 없습니다. 

최종 연산
→ 출력, 집계, 검색, 결과 생성
→ Stream 닫힘
→ 재사용 불가

대표:
forEach()
count()
reduce()
max()
min()
findFirst()
findAny()
collect()

5. 주요 중간 연산

5-1. filter()

filter()는 조건에 맞는 요소만 통과시키는 연산입니다. 

List<String> names = Arrays.asList("Java", "Spring", "Oracle");

names.stream()
    .filter(name -> name.length() >= 5)
    .forEach(System.out::println);
Spring
Oracle

5-2. map()

map()은 요소를 1:1로 다른 값으로 변환합니다. 

names.stream()
    .map(name -> name.length())
    .forEach(System.out::println);
4
6
6
Java   → 4
Spring → 6
Oracle → 6

5-3. flatMap()

flatMap()은 중첩된 데이터를 풀어서 하나의 Stream으로 만듭니다. 

List<List<String>> listOfLists = Arrays.asList(
    Arrays.asList("북한산", "관악산", "도봉산"),
    Arrays.asList("일본", "베트남", "태국"),
    Arrays.asList("자바", "파이썬")
);

listOfLists.stream()
        .flatMap(List::stream)
        .forEach(System.out::println);
map()
→ 요소 하나를 다른 값 하나로 변환

flatMap()
→ 요소 안의 여러 값을 풀어서 하나의 Stream으로 합침

List<String> names = Arrays.asList("Java", "Spring", "Java", "Oracle");

names.stream()
    .distinct()
    .forEach(System.out::println);

6. 주요 최종 연산

6-1. forEach()

names.stream()
    .forEach(name -> System.out.println(name));
names.stream()
    .forEach(System.out::println);

6-2. count()

long count = names.stream()
                .filter(name -> name.length() >= 5)
                .count();

System.out.println(count);
2

6-3. reduce()

List<Integer> numbers = Arrays.asList(1, 2, 3, 4, 5);

int sum = numbers.stream()
                .reduce(0, (a, b) -> a + b);

System.out.println(sum);
15
초기값 0
0 + 1 = 1
1 + 2 = 3
3 + 3 = 6
6 + 4 = 10
10 + 5 = 15

6-4. max(), min()

List<Integer> numbers = Arrays.asList(10, 30, 20);

Integer max = numbers.stream()
                    .max((a, b) -> a - b)
                    .orElse(0);

Integer min = numbers.stream()
                    .min((a, b) -> a - b)
                    .orElse(0);

max()와 min()은 결과가 없을 수도 있으므로 Optional을 반환합니다. 그래서 orElse() 같은 처리가 필요합니다.

6-5. findFirst(), findAny()

String first = names.stream()
                    .filter(name -> name.length() >= 5)
                    .findFirst()
                    .orElse("없음");
String any = names.stream()
                .filter(name -> name.length() >= 5)
                .findAny()
                .orElse("없음");

findFirst()는 첫 번째 요소를 찾고, findAny()는 조건에 맞는 아무 요소 하나를 찾습니다. 병렬 Stream에서는 findAny()의 순서가 보장되지 않을 수 있습니다.

7. Stream 재사용 불가

7-1. 한 번 사용한 Stream은 다시 사용할 수 없다

Stream<String> stream = names.stream();

stream.forEach(System.out::println);

stream.count(); // 오류
forEach()
→ 최종 연산
→ Stream 닫힘

닫힌 Stream에 count() 다시 호출
→ 오류

names.stream().forEach(System.out::println);

long count = names.stream().count();

8. 메서드 참조 ::

8-1. 메서드 참조란?

::는 메서드 참조 연산자입니다. 람다식을 더 짧고 읽기 쉽게 표현할 때 사용합니다. 

value -> System.out.println(value)
System.out::println

8-2. 정적 메서드 참조

DoubleUnaryOperator sqrtFunction = Math::sqrt;

double result = sqrtFunction.applyAsDouble(16);

System.out.println(result);
4.0

8-3. 생성자 참조

String[] names = {"가길동", "나길동"};

Stream<String> namesStream = Arrays.stream(names);

String[] result = namesStream.toArray(String[]::new);
String[]::new

의미:
length -> new String[length]

12. 면접 포인트

Q1. Stream API란 무엇인가요?

답변: Stream API는 배열이나 컬렉션의 데이터를 반복문 없이 처리할 수 있게 해주는 Java의 데이터 처리 라이브러리입니다.
원본 데이터를 직접 변경하지 않고, 중간 연산과 최종 연산을 연결해 데이터를 처리합니다.

Q2. 중간 연산과 최종 연산의 차이는 무엇인가요?

답변: 중간 연산은 Stream을 반환하며 여러 개를 연결할 수 있습니다. 최종 연산은 Stream을 소비해 결과를 만들거나 출력하며, 최종 연산 후 Stream은 닫힙니다.

Q3. Stream은 재사용할 수 있나요?

답변: 아니요. Stream은 최종 연산을 수행하면 닫히기 때문에 다시 사용할 수 없습니다. 다시 처리하려면 stream()을 호출해서 새로운 Stream을 생성해야 합니다.

Q4. filter()와 map()의 차이는 무엇인가요?

답변: filter()는 조건에 맞는 요소만 통과시키는 연산입니다. map()은 각 요소를 다른 값으로 1:1 변환하는 연산입니다.

Q5. map()과 flatMap()의 차이는 무엇인가요?

답변: map()은 요소 하나를 다른 값 하나로 변환합니다. flatMap()은 요소 안에 들어 있는 여러 값을 풀어서 하나의 Stream으로 합칩니다.

학습 메모

이 페이지는 Java Stream API 단원입니다. Stream은 배열과 컬렉션 데이터를 원본은 유지한 채 처리하는 흐름입니다.
Stream은 생성 → 중간 연산 → 최종 연산 순서로 사용하며, 최종 연산이 실행되면 닫혀서 다시 사용할 수 없습니다.
filter()는 조건 필터링, map()은 1:1 변환, flatMap()은 중첩 데이터 평탄화에 사용합니다.
