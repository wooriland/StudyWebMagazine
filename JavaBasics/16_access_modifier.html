16. 접근 지정자

접근 지정자는 클래스, 필드, 메서드를 어디까지 공개할지 정하는 문법입니다.
이 페이지에서는 private, default, protected, public의 차이와
캡슐화, 필드 보호, public 메서드 제공 방식을 공부합니다.

이번 페이지의 학습 목표

접근 지정자는 객체 내부 데이터를 보호하고, 필요한 기능만 외부에 공개하기 위한 문법입니다.
모든 필드와 메서드를 무조건 공개하면 객체 상태가 쉽게 망가질 수 있고, 반대로 모두 숨기면 외부에서 사용할 수 없습니다.
그래서 접근 지정자는 “누가 어디까지 사용할 수 있는가”를 정하는 기준입니다.

접근 범위 이해

private, default, protected, public의 범위를 익힙니다.

캡슐화 연결

필드는 숨기고 메서드로 접근하게 하는 구조를 이해합니다.

클래스 제한 이해

최상위 클래스에는 public과 생략형만 가능하다는 점을 익힙니다.

실무 패턴 이해

필드는 private, 외부 기능은 public으로 제공하는 패턴을 익힙니다.

1. 접근 지정자의 기본 개념

1-1. 접근 지정자란?

접근 지정자는 클래스, 멤버 변수, 메서드에 대해 다른 클래스에서 접근할 수 있는 범위를 정하는 문법입니다. 

public class User {

    private String password;

    public String name;

}
public
→ 외부에서 접근 가능

private
→ 자기 클래스 안에서만 접근 가능

즉 접근 지정자는 “이 데이터나 기능을 어디까지 공개할 것인가?”를 결정하는 장치입니다.

1-2. 접근 지정자가 필요한 이유

객체지향에서는 객체의 내부 데이터를 외부에서 마음대로 바꾸지 못하게 보호하는 것이 중요합니다.
예를 들어 계좌의 잔고를 외부에서 직접 수정할 수 있다면 잘못된 값이 들어갈 수 있습니다. 

class Account {
    int balance;
}

Account account = new Account();

account.balance = -100000;

위 코드는 은행 계좌 입장에서 말이 되지 않는 상태를 만들 수 있습니다. 그래서 중요한 데이터는 private으로 숨기고, 정해진 메서드를 통해서만 변경하도록 만드는 것이 좋습니다. 

class Account {

    private int balance;

    public void deposit(int amount) {
        if (amount > 0) {
            balance += amount;
        }
    }

    public int getBalance() {
        return balance;
    }

}

1-3. 접근 지정자의 종류와 범위

Java의 대표적인 접근 지정자는 네 가지입니다. 접근 범위는 private이 가장 좁고, public이 가장 넓습니다. 

좁음                                  넓음
private → default(package) → protected → public
접근 지정자 접근 범위 주 사용 위치
private 자기 클래스 안에서만 접근 가능 중요한 필드, 내부 보조 메서드
default 같은 패키지 안에서만 접근 가능 패키지 내부에서만 사용할 클래스나 멤버
protected 같은 패키지 또는 다른 패키지의 상속 관계에서 접근 가능 상속받은 자식 클래스에 열어줄 멤버
public 어디서나 접근 가능 외부에서 사용해야 하는 클래스와 메서드

2. 접근 지정자별 이해

2-1. private

private은 가장 좁은 접근 범위입니다. 자기 클래스 안에서만 접근할 수 있습니다. 

class Account {

    private int balance;

    public void deposit(int amount) {
        balance += amount; // 같은 클래스 안이므로 접근 가능
    }

}

하지만 외부 클래스에서는 직접 접근할 수 없습니다. 

Account account = new Account();

account.balance = 10000; // 오류

사용 기준: 비밀번호, 잔고, 내부 상태값처럼 외부에서 직접 변경하면 위험한 데이터는 보통 private으로 숨깁니다.

2-2. default

default는 접근 지정자를 아무것도 쓰지 않은 상태입니다. 같은 패키지 안에서는 접근할 수 있지만, 다른 패키지에서는 접근할 수 없습니다. 

class User {

    String name;

}
같은 패키지
→ 접근 가능

다른 패키지
→ 접근 불가

default는 package-private이라고도 부릅니다. Java에서 멤버 접근 제어용으로 default라는 키워드를 직접 쓰지는 않습니다.

2-3. protected

protected는 같은 패키지에서는 접근 가능하고, 다른 패키지에서는 상속 관계일 때 접근할 수 있습니다. 

public class Parent {

    protected String name = "부모";

}
public class Child extends Parent {

    public void printName() {
        System.out.println(name);
    }

}
protected 접근 가능 조건

1. 같은 패키지
2. 다른 패키지라도 상속 관계

protected는 상속 단원과 깊게 연결된다.

2-4. public

public은 가장 넓은 접근 범위입니다. 같은 클래스, 같은 패키지, 다른 패키지 어디서나 접근할 수 있습니다. 

public class User {

    public String name;

    public void printName() {
        System.out.println(name);
    }

}

하지만 모든 것을 public으로 만들면 객체 내부 데이터가 너무 많이 노출됩니다. 따라서 실무에서는 필요한 클래스와 메서드만 공개하는 것이 좋습니다.

실무 흐름: 클래스는 public, 필드는 private, 외부에서 필요한 기능은 public 메서드로 제공하는 경우가 많습니다.

3. 클래스 앞 접근 지정자 제한

3-1. 최상위 클래스에 가능한 접근 지정자

일반적인 최상위 클래스 앞에는 public과 생략형만 붙일 수 있습니다. 

public class User {
}
class User {
}

반면 최상위 클래스 앞에 private이나 protected를 붙일 수 없습니다. 

private class User {
}
protected class User {
}

단, 이것은 일반적인 최상위 클래스 기준입니다. 나중에 배우는 내부 클래스에서는 상황이 달라질 수 있습니다.

3-2. 접근 지정자를 붙일 수 있는 곳

접근 지정자는 클래스, 상수, 멤버 변수, 메서드에 붙일 수 있습니다. 

public class User {
}

private String password;

public static final int MAX_COUNT = 100;

public void login() {
}

접근 지정자와 static, final 같은 지정자를 함께 사용할 때는 보통 아래 순서로 작성합니다. 

public static final int MAX_LOGIN_FAIL_COUNT = 5;

private static final String DEFAULT_ROLE = "USER";

4. 접근 지정자와 캡슐화

4-1. 캡슐화와 접근 지정자

접근 지정자는 캡슐화를 구현하는 핵심 문법입니다. 캡슐화는 객체의 내부 데이터를 외부에서 함부로 바꾸지 못하게 보호하고, 정해진 메서드를 통해 접근하게 만드는 방식입니다. 

class Account {

    private int balance;

    public void setBalance(int balance) {
        if (balance >= 0) {
            this.balance = balance;
        }
    }

    public int getBalance() {
        return balance;
    }

}
private int balance
→ 외부에서 직접 수정하지 못하게 보호

public setBalance()
→ 값을 넣는 통로

public getBalance()
→ 값을 꺼내는 통로

4-2. public 메서드는 공식 통로다

public 메서드는 외부 클래스가 객체를 사용할 수 있는 공식 통로입니다. 따라서 public으로 열기 전에 검증 로직과 객체 상태 안전성을 고려해야 합니다. 

public class User {

    private String password;

    public void changePassword(String newPassword) {
        if (newPassword != null && newPassword.length() >= 8) {
            this.password = newPassword;
        }
    }

}

위 코드에서는 비밀번호가 null이 아니고 8자 이상일 때만 변경됩니다. 필드를 직접 열어두는 것보다 안전합니다.

4-3. private 보조 메서드

외부에서 직접 호출할 필요가 없는 내부 검증 기능은 private 메서드로 숨깁니다. 

public class Account {

    private int balance;

    public void deposit(int amount) {
        if (isValidAmount(amount)) {
            balance += amount;
        }
    }

    private boolean isValidAmount(int amount) {
        return amount > 0;
    }

    public int getBalance() {
        return balance;
    }

}
deposit()
→ 외부에서 호출하는 공개 메서드

isValidAmount()
→ Account 내부에서만 쓰는 검증 메서드

getBalance()
→ 외부에서 잔고를 읽는 공개 메서드

5. 그림처럼 이해하기

5-1. 접근 범위 그림

private
└─ 자기 클래스 안에서만 접근 가능

default
└─ 같은 패키지 안에서 접근 가능

protected
└─ 같은 패키지 접근 가능
└─ 다른 패키지라도 상속 관계면 접근 가능

public
└─ 어디서나 접근 가능

5-2. 집으로 비유하기

private
→ 내 방 안에서만 볼 수 있는 개인 일기

default
→ 같은 집 안 가족끼리만 볼 수 있는 메모

protected
→ 가족 + 상속받은 자식에게 열어주는 자료

public
→ 누구나 볼 수 있는 공지사항

5-3. 객체 보호 흐름

외부 클래스
↓ 직접 접근 차단
private 필드
↑
public 메서드로만 접근
↑
검증 로직 수행

6. 코드 예제

6-1. private 필드 보호

public class Account {

    private int balance;

    public void deposit(int amount) {
        if (amount > 0) {
            balance += amount;
        }
    }

    public int getBalance() {
        return balance;
    }

}
public class AccountMain {

    public static void main(String[] args) {

        Account account = new Account();

        account.deposit(10000);

        System.out.println(account.getBalance());

    }

}

6-2. 직접 접근 불가 예제

public class AccountMain {

    public static void main(String[] args) {

        Account account = new Account();

        account.balance = 10000; // 오류

    }

}

balance가 private이면 외부 클래스에서 직접 접근할 수 없습니다.

6-3. 접근 지정자 종합 예제

public class AccessExample {

    public String publicValue = "public";
    protected String protectedValue = "protected";
    String defaultValue = "default";
    private String privateValue = "private";

    public void publicMethod() {
        System.out.println("public method");
    }

    protected void protectedMethod() {
        System.out.println("protected method");
    }

    void defaultMethod() {
        System.out.println("default method");
    }

    private void privateMethod() {
        System.out.println("private method");
    }

    public void printAll() {
        System.out.println(publicValue);
        System.out.println(protectedValue);
        System.out.println(defaultValue);
        System.out.println(privateValue);

        publicMethod();
        protectedMethod();
        defaultMethod();
        privateMethod();
    }

}

같은 클래스 내부에서는 모든 접근 지정자의 멤버에 접근할 수 있습니다.

6-4. 실무형 User 예제

public class User {

    private Long userId;
    private String loginId;
    private String password;
    private String email;

    public Long getUserId() {
        return userId;
    }

    public String getLoginId() {
        return loginId;
    }

    public void changePassword(String newPassword) {
        if (newPassword != null && newPassword.length() >= 8) {
            this.password = newPassword;
        }
    }

}

필드는 모두 private으로 숨기고, 외부에 필요한 조회나 변경 기능만 public 메서드로 제공합니다.

7. 실무 연결

7-1. 실무에서 자주 쓰는 접근 지정자 패턴

클래스
→ public

필드
→ private

외부에서 필요한 조회 메서드
→ public getter

외부에서 필요한 변경 메서드
→ public method

내부 검증용 메서드
→ private method

7-2. Entity / DTO에서 사용

public class LoginRequestDto {

    private String loginId;
    private String password;

    public String getLoginId() {
        return loginId;
    }

    public String getPassword() {
        return password;
    }

}

DTO는 요청 데이터를 담지만, 필드를 직접 열기보다 private으로 숨기고 메서드를 통해 접근하게 만들 수 있습니다.

7-3. protected는 상속 구조에서 의미가 커진다

public class BaseService {

    protected void log(String message) {
        System.out.println(message);
    }

}

public class UserService extends BaseService {

    public void createUser() {
        log("회원 생성");
    }

}

log()는 외부에 완전히 공개할 필요는 없지만, 상속받은 서비스 클래스에서는 사용할 수 있게 열어둘 수 있습니다.

7-4. 접근 지정자 선택 기준

  1. 외부에 공개해야 하는 클래스라면 public class를 사용합니다.
  2. 객체 내부 데이터라면 private field로 숨깁니다.
  3. 외부에서 사용해야 하는 기능이라면 public method로 제공합니다.
  4. 클래스 내부에서만 쓰는 보조 기능이라면 private method로 숨깁니다.
  5. 같은 패키지 안에서만 쓸 클래스나 멤버라면 default를 사용할 수 있습니다.
  6. 상속받은 자식에게 열어줄 기능이라면 protected를 사용할 수 있습니다.

8. 자주 하는 실수

실수 1. 모든 필드를 public으로 둠

public class User {

    public String password;

}

이렇게 하면 외부에서 비밀번호를 아무 값으로나 바꿀 수 있습니다. 

private String password;

public void changePassword(String newPassword) {
    if (newPassword != null && newPassword.length() >= 8) {
        this.password = newPassword;
    }
}

실수 2. private 필드에 외부에서 직접 접근하려 함

Account account = new Account();

account.balance = 10000;
account.deposit(10000);

System.out.println(account.getBalance());

실수 3. 최상위 클래스 앞에 private을 붙임

private class User {
}
public class User {
}
class User {
}

실수 4. protected를 public처럼 생각함

protected는 어디서나 접근 가능한 것이 아니다.

가능:
- 같은 패키지
- 다른 패키지의 상속 관계

불가능:
- 다른 패키지이면서 상속 관계가 아닌 경우

실수 5. default를 명시적으로 쓰려고 함

default String name;
String name;

접근 지정자를 생략하면 default 접근입니다.

실수 6. public 메서드에 검증 로직을 넣지 않음

public void setBalance(int balance) {
    this.balance = balance;
}
public void setBalance(int balance) {
    if (balance >= 0) {
        this.balance = balance;
    }
}

9. 면접 포인트

Q1. 접근 지정자란 무엇인가요?

답변: 접근 지정자는 클래스, 필드, 메서드에 대해 다른 클래스에서 접근할 수 있는 범위를 지정하는 문법입니다.
객체 내부 데이터를 보호하고, 필요한 기능만 외부에 공개하기 위해 사용합니다.

Q2. Java의 접근 지정자 종류와 범위는 무엇인가요?

답변: private은 자기 클래스 안에서만 접근 가능합니다. default는 같은 패키지 안에서만 접근 가능합니다.
protected는 같은 패키지에서 접근 가능하고, 다른 패키지에서는 상속 관계일 때 접근 가능합니다. public은 어디서나 접근 가능합니다.

Q3. 접근 범위가 좁은 순서대로 말해보세요.

답변: private < default(package) < protected < public 순서입니다.

Q4. private은 언제 사용하나요?

답변: 외부에서 직접 접근하면 안 되는 필드나 내부 보조 메서드에 사용합니다.
예를 들어 비밀번호, 잔고 같은 중요한 데이터는 private으로 숨기고, public 메서드를 통해 검증 후 접근하게 하는 것이 좋습니다.

Q5. default 접근 지정자는 무엇인가요?

답변: 접근 지정자를 생략한 상태를 default 또는 package-private이라고 합니다. 같은 패키지 안에서는 접근 가능하지만, 다른 패키지에서는 접근할 수 없습니다.

Q6. protected는 언제 접근 가능한가요?

답변: protected 멤버는 같은 패키지 안에서는 접근 가능하고, 다른 패키지에서는 상속 관계가 있을 때 접근 가능합니다.

Q7. class 앞에는 어떤 접근 지정자를 붙일 수 있나요?

답변: 일반적인 최상위 class 앞에는 public과 생략형만 붙일 수 있습니다. private과 protected는 최상위 class 앞에 붙일 수 없습니다.

Q8. 접근 지정자와 캡슐화의 관계는 무엇인가요?

답변: 접근 지정자는 캡슐화를 구현하는 핵심 문법입니다.
필드를 private으로 숨기고, public getter/setter 또는 의미 있는 메서드를 통해 접근하게 하면 객체 내부 데이터를 보호하고 잘못된 값이 들어오는 것을 막을 수 있습니다.

10. 오늘의 핵심 정리

  1. 접근 지정자는 클래스와 클래스 간의 접근 범위를 지정한다.
  2. 접근 범위는 private < default < protected < public 순서다.
  3. private은 자기 클래스 안에서만 접근 가능하다.
  4. default는 같은 패키지 안에서만 접근 가능하다.
  5. protected는 같은 패키지와 상속 관계에서 접근 가능하다.
  6. public은 어디서나 접근 가능하다.
  7. 최상위 class 앞에는 public과 생략형만 붙일 수 있다.
  8. private과 protected는 일반 최상위 class 앞에는 붙일 수 없다.
  9. 필드는 보통 private으로 숨긴다.
  10. 외부에 필요한 기능은 public 메서드로 제공한다.
  11. 내부 검증용 메서드는 private으로 숨긴다.
  12. 접근 지정자는 캡슐화를 구현하는 핵심 문법이다.

학습 메모

이 페이지는 Java 접근 지정자 단원입니다.
접근 지정자는 클래스와 멤버의 공개 범위를 정하는 문법이며, private, default, protected, public 순서로 접근 범위가 넓어집니다.
특히 필드는 private으로 숨기고, 외부에 필요한 기능은 public 메서드로 제공하는 패턴은 다음 단원인 캡슐화와 직접 연결됩니다.
