25. 추상 클래스

추상 클래스는 공통 기능은 부모가 제공하고, 반드시 달라져야 하는 기능은 자식에게 구현을 강제하는 클래스입니다.
이 페이지에서는 abstract class, 추상 메서드, 직접 객체 생성 불가,
자식 클래스의 오버라이딩 의무, 추상 클래스와 다형성까지 공부합니다.

이번 페이지의 학습 목표

추상 클래스는 직접 객체를 만들기 위한 클래스라기보다, 자식 클래스들이 공통으로 따라야 할 구조를 정해주는 부모 클래스입니다.
공통 필드와 공통 메서드는 부모가 제공하고, 자식마다 다르게 동작해야 하는 메서드는 추상 메서드로 선언해 구현을 강제합니다.

추상 클래스 이해

abstract class가 왜 직접 객체 생성이 불가능한지 이해합니다.

추상 메서드 이해

몸체 없는 메서드가 자식 클래스의 구현 의무가 되는 구조를 익힙니다.

상속과 오버라이딩

추상 클래스를 상속받은 자식 클래스가 반드시 구현해야 하는 내용을 이해합니다.

다형성 연결

추상 클래스 타입으로 여러 자식 객체를 다루는 구조를 이해합니다.

1. 추상 클래스의 기본 개념

1-1. 추상 클래스란?

추상 클래스는 abstract class로 선언하는 클래스입니다. 직접 객체를 만들기보다는 자식 클래스들이 공통으로 가져야 할 구조를 정해주는 부모 클래스 역할을 합니다. 

abstract class Animal {
}
class Animal
→ 일반 클래스

abstract class Animal
→ 추상 클래스

추상 클래스
→ 미완성 설계도
→ 직접 객체 생성 불가
→ 자식 클래스의 공통 부모 역할

1-2. 추상 메서드란?

추상 메서드는 메서드 몸체가 없는 메서드입니다. 선언만 있고 실행 내용은 없습니다. 

public abstract void sound();
public void sound() {
    System.out.println("소리를 냅니다.");
}
구분 형태 의미
일반 메서드 public void sound() { } 선언부와 구현부가 모두 있음
추상 메서드 public abstract void sound(); 선언부만 있고 구현부가 없음

1-3. 추상 메서드가 있으면 클래스도 abstract여야 한다

class Animal {

    public abstract void sound();

}

추상 메서드를 하나라도 가진 클래스는 반드시 클래스 앞에도 abstract를 붙여야 합니다. 

abstract class Animal {

    public abstract void sound();

}

정리: 추상 메서드가 1개 이상 있다면 그 클래스는 반드시 추상 클래스여야 합니다.

1-4. 추상 클래스는 직접 객체 생성이 불가능하다

abstract class Animal {

    public abstract void sound();

}
Animal animal = new Animal(); // 오류

추상 클래스는 아직 구현되지 않은 추상 메서드를 가질 수 있기 때문에 직접 객체를 만들 수 없습니다. 만약 animal.sound()를 호출했는데 실행할 코드가 없다면 문제가 되기 때문입니다. 

Animal animal = new Animal();

animal.sound();

문제:
sound()가 미완성 메서드라서 실행할 코드가 없음

그래서 Java는 추상 클래스의 직접 객체 생성을 막는다.

2. 추상 클래스를 상속받은 자식 클래스

2-1. 자식 클래스는 추상 메서드를 반드시 구현해야 한다

추상 클래스를 상속받은 자식 클래스는 부모의 추상 메서드를 반드시 오버라이딩해야 합니다. 

abstract class Animal {

    public abstract void sound();

}

class Dog extends Animal {

    @Override
    public void sound() {
        System.out.println("멍멍");
    }

}
Dog dog = new Dog();
dog.sound();
멍멍

2-2. 추상 메서드를 구현하지 않으면 오류

abstract class Animal {

    public abstract void sound();

}

class Dog extends Animal {
}

Dog가 부모의 추상 메서드 sound()를 구현하지 않았기 때문에 오류가 발생합니다.

해결 방법 1: 자식 클래스에서 추상 메서드를 구현합니다.

class Dog extends Animal {

    @Override
    public void sound() {
        System.out.println("멍멍");
    }

}

해결 방법 2: 자식 클래스도 추상 클래스로 선언합니다.

abstract class Dog extends Animal {
}

2-3. 추상 메서드가 없어도 추상 클래스가 될 수 있다

추상 메서드가 없어도 객체 생성을 막고 상속 전용 부모 클래스로만 사용하고 싶다면 추상 클래스로 선언할 수 있습니다. 

abstract class BaseEntity {

    private Long id;

    public Long getId() {
        return id;
    }

}
BaseEntity entity = new BaseEntity(); // 오류

BaseEntity 자체로 객체를 만들 필요는 없고, UserEntity, ProductEntity 같은 자식 클래스의 공통 부모로만 사용할 수 있습니다.

2-4. 추상 클래스 타입 참조변수는 가능하다

추상 클래스는 직접 객체 생성은 불가능하지만 타입으로 사용할 수 있습니다. 

Animal animal = new Dog();
Animal animal
→ 추상 클래스 타입 참조변수

new Dog()
→ 추상 클래스를 상속받아 완성한 자식 객체

즉,
추상 클래스 타입으로 자식 객체를 참조할 수 있다.

3. 일반 메서드, 생성자, 필드를 가진 추상 클래스

3-1. 추상 클래스는 일반 메서드도 가질 수 있다

추상 클래스는 추상 메서드만 가지는 것이 아닙니다. 일반 필드, 생성자, 일반 메서드도 가질 수 있습니다. 

abstract class Animal {

    String name;

    public Animal(String name) {
        this.name = name;
    }

    public void breathe() {
        System.out.println(name + "이 숨을 쉽니다.");
    }

    public abstract void sound();

}
공통으로 이미 구현된 기능
→ 일반 메서드로 제공

자식마다 반드시 다르게 구현해야 하는 기능
→ 추상 메서드로 강제

3-2. 추상 클래스에도 생성자가 있다

추상 클래스는 직접 객체 생성은 불가능하지만 생성자를 가질 수 있습니다. 자식 객체가 생성될 때 부모 영역도 함께 만들어지므로, 부모 영역을 초기화하기 위해 추상 클래스의 생성자가 실행됩니다. 

abstract class Animal {

    String name;

    public Animal(String name) {
        this.name = name;
    }

}

class Dog extends Animal {

    public Dog(String name) {
        super(name);
    }

}
new Dog("바둑이")
→ Animal 영역 생성
→ Animal 생성자 실행
→ Dog 영역 생성
→ Dog 생성자 실행

3-3. 추상 클래스 구조 그림

Animal 추상 클래스

┌──────────────────────────┐
│ name                     │
│ breathe() 구현 완료      │
│ sound() 미완성           │
└──────────────────────────┘
            ↑ extends

Dog 일반 클래스

┌──────────────────────────┐
│ name                     │
│ breathe() 상속받음       │
│ sound() → 멍멍 구현      │
└──────────────────────────┘

Cat 일반 클래스

┌──────────────────────────┐
│ name                     │
│ breathe() 상속받음       │
│ sound() → 야옹 구현      │
└──────────────────────────┘

4. 추상 클래스와 다형성

4-1. 추상 클래스 타입으로 여러 자식 객체 다루기

abstract class Animal {

    public abstract void sound();

}

class Dog extends Animal {

    @Override
    public void sound() {
        System.out.println("멍멍");
    }

}

    @Override

Animal animal1 = new Dog();

animal1.sound();
animal2.sound();
멍멍
야옹

참조변수 타입은 모두 Animal이지만, 실제 객체가 Dog인지 Cat인지에 따라 실행되는 sound()가 달라집니다.

4-2. 배열과 다형성

Animal[] animals = {
    new Dog(),

for (Animal animal : animals) {
    animal.sound();
}
멍멍
야옹

Dog
Cat

반복문:
animal.sound()

6. 실무 연결

6-1. 공통 Entity 부모

abstract class BaseEntity {

    private LocalDateTime createdAt;
    private LocalDateTime updatedAt;

    public LocalDateTime getCreatedAt() {
        return createdAt;
    }

    public LocalDateTime getUpdatedAt() {
        return updatedAt;
    }

}

class UserEntity extends BaseEntity {

    private Long userId;
    private String loginId;

}

class PropertyEntity extends BaseEntity {

    private Long propertyId;
    private String address;

}

BaseEntity 자체는 객체로 만들 필요가 없고, 여러 Entity가 공통 필드를 물려받기 위한 부모로만 사용됩니다.

6-2. 결제 방식 공통 부모

abstract class Payment {

    protected int amount;

    public Payment(int amount) {
        this.amount = amount;
    }

    public void printAmount() {
        System.out.println("결제 금액: " + amount);
    }

    public abstract void pay();

}

class CardPayment extends Payment {

    public CardPayment(int amount) {
        super(amount);
    }

    @Override
    public void pay() {
        System.out.println("카드로 결제합니다.");
    }

}

class KakaoPayment extends Payment {

    public KakaoPayment(int amount) {
        super(amount);
    }

    @Override
    public void pay() {
        System.out.println("카카오페이로 결제합니다.");
    }

}
Payment[] payments = {
    new CardPayment(10000),
    new KakaoPayment(20000)
};

for (Payment payment : payments) {
    payment.printAmount();
    payment.pay();
}

6-3. 보고서 생성 흐름

전체 흐름은 부모가 정하고, 세부 분석 방식은 자식이 구현하게 만들 수 있습니다. 

abstract class ReportGenerator {

    public void generate() {
        loadData();
        analyze();
        printReport();
    }

    protected void loadData() {
        System.out.println("데이터를 불러옵니다.");
    }

    protected abstract void analyze();

    protected void printReport() {
        System.out.println("보고서를 출력합니다.");
    }

}

class UserReportGenerator extends ReportGenerator {

    @Override
    protected void analyze() {
        System.out.println("회원 데이터를 분석합니다.");
    }

}

class PropertyReportGenerator extends ReportGenerator {

    @Override
    protected void analyze() {
        System.out.println("매물 데이터를 분석합니다.");
    }

}
공통 흐름:
generate(), loadData(), printReport()

자식마다 다른 부분:
analyze()

8. 자주 하는 실수

실수 1. 추상 메서드가 있는데 클래스에 abstract를 안 붙임

class Animal {

    public abstract void sound();

}
abstract class Animal {

    public abstract void sound();

}

실수 2. 추상 메서드에 몸체를 작성함

public abstract void sound() {
    System.out.println("소리");
}
public abstract void sound();

실수 3. 추상 클래스를 직접 new 하려고 함

Animal animal = new Animal();
Animal animal = new Dog();

실수 4. 자식 클래스에서 추상 메서드를 구현하지 않음

class Dog extends Animal {
}
class Dog extends Animal {

    @Override
    public void sound() {
        System.out.println("멍멍");
    }

}

실수 5. 추상 클래스의 생성자를 불필요하다고 생각함

abstract class Animal {

    String name;

    public Animal(String name) {
        this.name = name;
    }

}

추상 클래스는 직접 객체 생성은 못 하지만 생성자를 가질 수 있습니다. 자식 객체가 만들어질 때 부모 영역도 초기화되어야 하기 때문입니다.

실수 6. 추상 클래스와 인터페이스를 같은 것으로 생각함

추상 클래스
→ 공통 필드, 공통 메서드, 생성자, 추상 메서드를 함께 가질 수 있음

인터페이스
→ 구현해야 할 규칙이나 기능 명세에 가까움

9. 면접 포인트

Q1. 추상 클래스란 무엇인가요?

답변: 추상 클래스는 abstract class로 선언하는 클래스이며, 직접 객체를 생성할 수 없고 자식 클래스에게 공통 기능과 구현해야 할 추상 메서드를 제공하는 클래스입니다.

Q2. 추상 메서드란 무엇인가요?

답변: 추상 메서드는 메서드 몸체 없이 선언만 되어 있는 메서드입니다. abstract 키워드를 붙이며, 자식 클래스에서 반드시 오버라이딩해야 합니다.

Q3. 추상 메서드를 가진 클래스는 반드시 추상 클래스여야 하나요?

답변: 네. 추상 메서드를 하나라도 가진 클래스는 반드시 class 앞에 abstract를 붙여 추상 클래스로 선언해야 합니다.

Q4. 추상 클래스는 객체 생성이 가능한가요?

답변: 직접 객체 생성은 불가능합니다. 하지만 추상 클래스 타입으로 참조변수를 선언하고, 그 추상 클래스를 상속받아 구현한 자식 객체를 참조할 수는 있습니다.
예를 들어 Animal animal = new Dog();는 가능합니다.

Q5. 추상 클래스를 상속받은 자식 클래스는 무엇을 해야 하나요?

답변: 부모 추상 클래스의 추상 메서드를 반드시 오버라이딩해야 합니다. 만약 구현하지 않는다면 자식 클래스도 추상 클래스로 선언해야 합니다.

Q6. 추상 메서드가 없어도 추상 클래스가 될 수 있나요?

답변: 가능합니다. 추상 메서드가 없어도 인스턴스 생성을 막고 상속 전용 부모 클래스로 사용하고 싶다면 추상 클래스로 선언할 수 있습니다.

Q7. 추상 클래스에 생성자가 있을 수 있나요?

답변: 있을 수 있습니다. 추상 클래스는 직접 객체 생성은 못 하지만, 자식 객체가 생성될 때 부모 영역을 초기화하기 위해 추상 클래스의 생성자가 실행됩니다.

Q8. 추상 클래스와 일반 클래스의 차이는 무엇인가요?

답변: 일반 클래스는 직접 객체 생성이 가능하지만, 추상 클래스는 직접 객체 생성이 불가능합니다. 또한 추상 클래스는 추상 메서드를 가질 수 있고, 자식 클래스에게 특정 메서드 구현을 강제할 수 있습니다.

Q9. 추상 클래스와 인터페이스의 차이는 무엇인가요?

답변: 추상 클래스는 공통 필드, 생성자, 일반 메서드, 추상 메서드를 함께 가질 수 있고, 상속 관계에서 공통 기능과 강제 구현을 함께 제공할 때 사용합니다. 인터페이스는 주로 클래스가 반드시 구현해야 할 규칙이나 기능 명세를 정의할 때 사용합니다.

Q10. 추상 클래스를 사용하는 이유는 무엇인가요?

답변: 여러 자식 클래스가 공통으로 사용할 필드와 메서드를 제공하면서, 자식마다 반드시 다르게 구현해야 하는 메서드를 강제하기 위해 사용합니다. 이를 통해 코드 중복을 줄이고 일관된 구조를 유지할 수 있습니다.

10. 오늘의 핵심 정리

  1. 추상 클래스는 abstract class로 선언한다.
  2. 추상 메서드는 abstract 키워드를 붙이고 몸체 없이 선언한다.
  3. 추상 메서드를 하나라도 가진 클래스는 반드시 추상 클래스여야 한다.
  4. 추상 메서드가 없어도 객체 생성을 막기 위해 추상 클래스로 만들 수 있다.
  5. 추상 클래스는 직접 new 해서 객체 생성할 수 없다.
  6. 추상 클래스는 타입 선언은 가능하다.
  7. 추상 클래스 타입 참조변수는 자식 객체를 참조할 수 있다.
  8. 추상 클래스를 상속받은 자식 클래스는 추상 메서드를 반드시 오버라이딩해야 한다.
  9. 추상 메서드를 구현하지 않은 자식 클래스는 그 자체도 추상 클래스가 되어야 한다.
  10. 추상 클래스는 일반 필드, 일반 메서드, 생성자, 상수를 가질 수 있다.
  11. 추상 클래스의 생성자는 자식 객체 생성 시 부모 영역 초기화를 위해 실행된다.
  12. 추상 클래스는 공통 기능 제공과 강제 구현을 동시에 할 때 유용하다.
  13. 추상 클래스는 상속과 다형성 구조에서 부모 타입으로 자주 사용된다.

학습 메모

이 페이지는 Java 추상 클래스 단원입니다. 추상 클래스는 직접 객체를 만들기보다 자식 클래스의 공통 부모 역할을 합니다.
일반 필드, 일반 메서드, 생성자를 가질 수 있고, 자식 클래스가 반드시 구현해야 하는 메서드는 추상 메서드로 선언합니다.
추상 클래스는 직접 new 할 수 없지만, 추상 클래스 타입 참조변수로 자식 객체를 다룰 수 있다는 점이 핵심입니다.
