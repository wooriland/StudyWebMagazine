08. 형 변환

형 변환은 값의 자료형을 다른 자료형으로 바꾸는 과정입니다.
이 페이지에서는 자동 형 변환, 강제 형 변환, 문자열과 숫자 변환,
그리고 형 변환 과정에서 발생할 수 있는 데이터 손실과 연산 결과를 공부합니다.

이번 페이지의 학습 목표

형 변환은 단순히 자료형 이름을 바꾸는 문법이 아니라,
값이 안전하게 유지되는지, 소수점이나 범위 손실이 발생하지 않는지, 연산 결과가 의도한 대로 나오는지를 판단하는 중요한 개념입니다.

형 변환 개념 이해

값의 자료형을 다른 자료형으로 바꾸는 과정을 이해합니다.

자동/강제 변환 구분

Java가 자동으로 처리하는 변환과 개발자가 직접 지정하는 변환을 구분합니다.

데이터 손실 이해

큰 타입에서 작은 타입으로 변환할 때 발생할 수 있는 손실을 이해합니다.

실무 변환 활용

문자열 입력값, 평균 계산, DTO 변환에서 형 변환이 쓰이는 흐름을 익힙니다.

1. 핵심 개념 설명

1-1. 형 변환이란?

형 변환은 값의 자료형을 다른 자료형으로 바꾸는 것입니다. 서로 다른 자료형끼리 값을 대입하거나 연산할 때 Java는 필요한 경우 자료형을 변환합니다. 

int num = 10;
double result = num;
int 10
↓
double 10.0

위 코드에서는 int 값 10이 double 값 10.0으로 변환됩니다.

1-2. 형 변환이 필요한 이유

Java는 자료형을 엄격하게 구분합니다. 그래서 서로 다른 자료형끼리 대입, 연산, 비교, 메서드 전달을 할 때 자료형을 맞추는 과정이 필요합니다.

  • 서로 다른 숫자 타입끼리 연산할 때 형 변환이 발생합니다.
  • 사용자 입력처럼 문자열로 들어온 값을 숫자로 바꿀 때 형 변환이 필요합니다.
  • 정수 나눗셈에서 소수점 결과를 얻기 위해 형 변환이 필요합니다.
  • DTO나 API 응답에서 숫자를 문자열로 보여줄 때 형 변환이 필요합니다.

1-3. 형 변환의 종류

Java의 형 변환은 크게 자동 형 변환과 강제 형 변환으로 나뉩니다.

구분 자동 형 변환 강제 형 변환
처리 주체 Java가 자동 처리 개발자가 직접 지정
방향 작은 타입 → 큰 타입 큰 타입 → 작은 타입
데이터 손실 거의 없음 발생 가능
문법 별도 표시 없음 (자료형) 사용
예시 double d = 10; int i = (int) 10.7;

1-4. 자동 형 변환

자동 형 변환은 Java가 알아서 자료형을 바꿔주는 경우입니다. 보통 작은 타입의 값을 더 큰 타입에 넣을 때 자동으로 발생합니다. 

int intValue = 10;
double doubleValue = intValue;

System.out.println(intValue);
System.out.println(doubleValue);

출력 결과는 다음과 같습니다.

10
10.0
byte → short → int → long → float → double

char → int

다만 long → float처럼 자동 변환이 가능하더라도 정밀도 손실이 발생할 수 있으므로 항상 값이 안전하게 유지되는지 생각해야 합니다.

1-5. 강제 형 변환

강제 형 변환은 개발자가 직접 (자료형)을 붙여 변환하는 방식입니다. 보통 큰 타입의 값을 작은 타입에 넣을 때 필요합니다. 

double doubleValue = 10.7;
int intValue = (int) doubleValue;

System.out.println(intValue);

출력 결과는 다음과 같습니다.

10

주의: 실수를 정수로 강제 형 변환하면 반올림되지 않고 소수점 아래가 버려집니다.

1-6. 정수형끼리의 형 변환

작은 정수형에서 큰 정수형으로 이동할 때는 자동 형 변환이 가능합니다. 

int num = 100;
long longNum = num;

하지만 큰 정수형에서 작은 정수형으로 이동할 때는 강제 형 변환이 필요합니다. 

long longNum = 100L;
int num = longNum;
long longNum = 100L;
int num = (int) longNum;

1-7. 실수형과 정수형의 형 변환

정수형에서 실수형으로 변환하는 것은 보통 자동으로 처리됩니다. 

int num = 10;
double result = num;

반대로 실수형에서 정수형으로 변환할 때는 강제 형 변환이 필요하며, 소수점 아래가 버려집니다. 

double pi = 3.14;
int result = (int) pi;

System.out.println(result);
3
3.14  → 3
9.99  → 9
-3.14 → -3

1-8. char와 int 형 변환

char는 문자 하나를 저장하지만 내부적으로는 숫자 코드값을 가집니다. 그래서 char는 int로 변환될 수 있습니다. 

char ch = 'A';
            int code = ch;

            System.out.println(code);
65

반대로 숫자 코드값을 문자로 바꿀 수도 있습니다.

int code = 66;
            char ch = (char) code;

            System.out.println(ch);
B

1-9. 연산 중 자동 형 변환

서로 다른 자료형끼리 연산하면 더 큰 타입으로 자동 변환된 뒤 계산되는 경우가 많습니다. 

int num = 10;
double result = num + 3.5;

System.out.println(result);
13.5
10 + 3.5
↓
10.0 + 3.5
↓
13.5

1-10. byte, short, char의 연산 결과

byte, short, char는 연산할 때 보통 int로 변환됩니다. 

byte a = 10;
byte b = 20;

byte result = a + b;

올바른 방식은 결과를 int로 받는 것입니다.

byte a = 10;
byte b = 20;

int result = a + b;

반드시 byte로 받고 싶다면 강제 형 변환이 필요합니다.

byte result = (byte)(a + b);

2. 문자열과 숫자 변환

2-1. 문자열 숫자와 실제 숫자

문자열 "100"과 숫자 100은 겉보기에는 비슷하지만 전혀 다른 자료형입니다. 

"100" → 문자열 String
100   → 숫자 int

String str = "100";
int num = 100;

2-2. 문자열을 숫자로 변환

문자열을 숫자로 바꾸려면 Wrapper 클래스의 변환 메서드를 사용합니다. 

int num = Integer.parseInt("100");
long longNum = Long.parseLong("1000");
double doubleNum = Double.parseDouble("3.14");
boolean bool = Boolean.parseBoolean("true");

문자열 숫자를 실제 숫자로 바꿔야 산술 연산이 가능합니다. 

String str = "100";

int num = Integer.parseInt(str);

System.out.println(num + 10);
110

2-3. 숫자를 문자열로 변환

숫자를 문자열로 바꿀 때는 String.valueOf()를 사용할 수 있습니다. 

int num = 100;

String str = String.valueOf(num);

System.out.println(str);

num + "" 방식도 가능하지만, 의미가 분명한 String.valueOf()를 사용하는 편이 좋습니다.

2-4. 문자열 결합과 형 변환

Java에서 문자열과 숫자를 +로 연결하면 숫자가 문자열로 변환되어 결합됩니다. 

System.out.println("나이: " + 20);
나이: 20

하지만 계산 순서에 주의해야 합니다. 

System.out.println("결과: " + 10 + 20);
결과: 1020

계산을 먼저 하고 싶다면 괄호를 사용합니다.

System.out.println("결과: " + (10 + 20));
결과: 30

4. 코드 예제

4-1. 자동 형 변환 예제

public class AutoCastingExample {

    public static void main(String[] args) {

        int intValue = 10;
        double doubleValue = intValue;

        System.out.println(intValue);
        System.out.println(doubleValue);

    }

}

4-2. 강제 형 변환 예제

public class ManualCastingExample {

    public static void main(String[] args) {

        double doubleValue = 10.7;
        int intValue = (int) doubleValue;

        System.out.println(doubleValue);
        System.out.println(intValue);

    }

}

출력 결과는 다음과 같습니다.

10.7
10

4-3. char와 int 변환 예제

public class CharCastingExample {

    public static void main(String[] args) {

        char ch = 'A';

        System.out.println(code);

        int nextCode = 66;
        char nextChar = (char) nextCode;

        System.out.println(nextChar);

}

A
65
B

5. 실무 연결

5-1. 사용자 입력은 보통 문자열로 들어온다

콘솔 입력, 웹 요청 파라미터, JSON 데이터는 숫자처럼 보여도 문자열로 들어오는 경우가 많습니다. 계산이나 비교를 하려면 숫자형으로 변환해야 합니다. 

String inputAge = "20";
int age = Integer.parseInt(inputAge);
"page=1"
"size=10"
"price=350000000"

문자열 입력값 → 숫자 변환 필요

5-2. 평균 계산에서는 형 변환이 중요하다

정수끼리 나누면 결과도 정수로 계산됩니다. 소수점까지 얻고 싶다면 한쪽을 double로 변환해야 합니다. 

int totalScore = 250;
int subjectCount = 3;

double average = totalScore / subjectCount;
double average = (double) totalScore / subjectCount;

5-3. DB와 API 값 변환

DB나 API에서 가져온 값이 Java에서 원하는 타입과 다를 수 있습니다. 이런 경우 적절한 변환 메서드를 사용해야 합니다. 

Number count = 10;
int result = count.intValue();
long price = Long.parseLong("350000000");

5-4. DTO 변환에서도 형 변환이 등장한다

Entity에는 숫자로 저장된 값을 화면 DTO에서는 문자열로 보여줄 수 있고, 반대로 입력 폼에서 문자열로 받은 값을 숫자로 바꿔 저장할 수도 있습니다. 

Long dealAmount = 350000000L;
String displayAmount = String.valueOf(dealAmount) + "원";
String inputAmount = "350000000";
Long dealAmount = Long.parseLong(inputAmount);

5-5. 실수 계산과 금액 처리

실수형은 소수 계산에서 오차가 발생할 수 있습니다. 금액처럼 정확도가 중요한 값은 double 변환을 조심해야 하며, 실무에서는 BigDecimal을 고려합니다. 

BigDecimal price = new BigDecimal("350000000");

6. 자주 하는 실수

실수 1. 강제 형 변환의 소수점 손실을 모름

double value = 9.99;
int result = (int) value;

결과는 9입니다. 반올림이 아니라 소수점 아래가 버려집니다.

실수 2. 정수 나눗셈 결과를 double에 담으면 소수점이 살아난다고 생각함

int total = 10;
int count = 3;

double average = total / count;

올바른 예시는 다음과 같습니다.

double average = (double) total / count;

실수 3. byte 연산 결과를 byte로 바로 받음

byte a = 10;
byte b = 20;

byte result = a + b;

올바른 예시는 다음과 같습니다.

int result = a + b;

실수 4. 문자열 숫자를 숫자처럼 더한다고 생각함

String num = "100";

System.out.println(num + 10);

출력 결과는 10010입니다. 숫자 계산을 원하면 변환해야 합니다.

int number = Integer.parseInt(num);
System.out.println(number + 10);

실수 5. parseInt에 숫자가 아닌 문자열을 넣음

int number = Integer.parseInt("abc");

이 코드는 실행 중 NumberFormatException이 발생합니다. 문자열을 숫자로 바꿀 때는 정말 숫자 형태인지 확인해야 합니다.

실수 6. 큰 값을 작은 타입으로 변환하면서 범위 초과를 무시함

int num = 130;
byte b = (byte) num;

byte 범위를 넘는 값이므로 결과가 예상과 달라질 수 있습니다.

실수 7. 문자열 결합 순서를 착각함

System.out.println("결과: " + 10 + 20);

출력 결과는 결과: 1020입니다.

System.out.println("결과: " + (10 + 20));

출력 결과는 결과: 30입니다.

7. 면접 포인트

Q1. 형 변환이란 무엇인가요?

답변: 형 변환은 값의 자료형을 다른 자료형으로 바꾸는 것입니다. Java에서는 작은 타입에서 큰 타입으로 변환되는 자동 형 변환과, 개발자가 직접 명시하는 강제 형 변환이 있습니다.

Q2. 자동 형 변환과 강제 형 변환의 차이는 무엇인가요?

답변: 자동 형 변환은 데이터 손실 가능성이 낮은 방향으로 Java가 자동 처리하는 변환입니다. 보통 작은 타입에서 큰 타입으로 변환될 때 발생합니다.
강제 형 변환은 큰 타입에서 작은 타입으로 변환할 때 개발자가 직접 (자료형)을 붙여 수행하며, 데이터 손실이 발생할 수 있습니다.

Q3. 실수를 정수로 형 변환하면 어떻게 되나요?

답변: 실수를 정수로 강제 형 변환하면 소수점 아래가 버려집니다. 반올림되지 않고 단순히 정수 부분만 남습니다. 예를 들어 (int) 3.9는 3이 됩니다.

Q4. byte끼리 연산하면 결과 타입은 무엇인가요?

답변: byte, short, char는 연산 시 보통 int로 변환됩니다. 따라서 byte끼리 더해도 결과 타입은 byte가 아니라 int로 처리됩니다.

Q5. 문자열 "100"을 숫자 100으로 바꾸려면 어떻게 하나요?

답변: Integer.parseInt("100")을 사용하면 문자열을 int로 변환할 수 있습니다. long은 Long.parseLong(), double은 Double.parseDouble()을 사용할 수 있습니다.

Q6. 정수 나눗셈에서 소수점까지 결과를 얻으려면 어떻게 해야 하나요?

답변: 정수끼리 나누면 정수 나눗셈이 먼저 수행되므로 소수점이 사라집니다. 소수점 결과를 얻으려면 피연산자 중 하나를 double로 형 변환해야 합니다.
예를 들어 (double) total / count처럼 작성합니다.

Q7. 강제 형 변환 시 주의할 점은 무엇인가요?

답변: 강제 형 변환은 데이터 손실이 발생할 수 있습니다.
실수를 정수로 바꾸면 소수점이 사라지고, 큰 정수를 작은 타입으로 바꾸면 범위 초과로 예상과 다른 값이 나올 수 있습니다.
따라서 변환 전 값의 범위와 손실 가능성을 확인해야 합니다.

8. 오늘의 핵심 정리

  1. 형 변환은 값의 자료형을 다른 자료형으로 바꾸는 것이다.
  2. 자동 형 변환은 작은 타입에서 큰 타입으로 갈 때 주로 발생한다.
  3. 강제 형 변환은 큰 타입에서 작은 타입으로 갈 때 개발자가 직접 지정한다.
  4. 강제 형 변환은 데이터 손실이 발생할 수 있다.
  5. 실수를 정수로 바꾸면 소수점 아래가 버려진다.
  6. byte, short, char 연산 결과는 보통 int가 된다.
  7. 문자열 숫자는 parseInt, parseLong, parseDouble 등으로 변환한다.
  8. 숫자를 문자열로 바꿀 때는 String.valueOf()를 사용할 수 있다.
  9. 문자열 결합에서는 계산 순서에 주의해야 한다.
  10. 정수 나눗셈에서 소수점 결과를 얻으려면 한쪽을 double로 형 변환해야 한다.

학습 메모

이 페이지는 Java에서 자료형이 서로 달라질 때 값이 어떻게 이동하는지 공부하는 형 변환 단원입니다.
자동 형 변환과 강제 형 변환을 구분하고, 소수점 손실, 범위 초과, 문자열 숫자 변환, 정수 나눗셈 문제를 이해하면
이후 연산자, 조건문, 반복문, 사용자 입력 처리, DTO 변환과 API 데이터 처리까지 훨씬 안정적으로 다룰 수 있습니다.
