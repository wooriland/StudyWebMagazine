Java Basic · Chapter 01

01. Java 개발환경

Java를 공부하기 전에 가장 먼저 이해해야 하는 것은 개발환경입니다.
JDK를 설치하고, JAVA_HOME과 Path를 설정하고, Eclipse에서 UTF-8 인코딩을 맞추는 과정은
단순한 설치 작업이 아니라 Java 프로그램이 어떻게 만들어지고 실행되는지 이해하는 출발점입니다.

이번 페이지의 학습 목표

이 페이지에서는 Java 개발환경을 구성하는 핵심 요소인
JDK, JRE, JVM, javac, java, JAVA_HOME, Path, Eclipse 설정을 공부합니다.
단순히 설치 방법만 외우는 것이 아니라,
Java 소스코드가 어떤 과정을 거쳐 실행되는지까지 연결해서 이해하는 것이 목표입니다.

개념 이해

JDK, JRE, JVM의 차이를 구분합니다.

실행 구조 이해

.java 파일이 .class 파일로 바뀌고
JVM에서 실행되는 흐름을 이해합니다.

환경 설정 이해

JAVA_HOME과 Path를 왜 설정하는지 이해합니다.

실무 연결

Spring Boot, Gradle, Maven, Tomcat에서
JDK 설정이 왜 중요한지 연결합니다.

1. 핵심 개념 설명

1-1. JDK란?

JDK는 Java Development Kit의 약자입니다.
이름 그대로 Java 프로그램을 개발하기 위한 도구 모음입니다.
Java 개발자는 단순히 Java 프로그램을 실행하는 것만 하는 것이 아니라,
직접 소스코드를 작성하고, 컴파일하고, 실행하고, 필요하면 문서화까지 해야 합니다.
이 모든 작업을 가능하게 해주는 것이 JDK입니다. 

JDK
├─ JRE
│   ├─ JVM
│   └─ 기본 클래스 라이브러리
│
├─ javac.exe    → Java 소스코드 컴파일 도구
├─ java.exe     → 컴파일된 class 파일 실행 도구
├─ javadoc.exe  → API 문서 생성 도구
└─ jar.exe      → Java 파일 압축 및 배포 도구

즉, JDK는 Java 개발을 위한 가장 큰 묶음입니다.
개발자는 JDK를 설치해야 javac 명령어로 Java 파일을 컴파일할 수 있고,
java 명령어로 컴파일된 프로그램을 실행할 수 있습니다.

1-2. JDK, JRE, JVM의 차이

Java를 처음 공부할 때 가장 헷갈리는 개념이 바로 JDK, JRE, JVM입니다.
세 단어가 비슷해 보이지만 역할은 분명히 다릅니다.

구분 이름 역할 비유
JDK Java Development Kit Java 개발 도구 전체 요리 도구 세트
JRE Java Runtime Environment Java 실행 환경 요리가 이루어지는 주방 환경
JVM Java Virtual Machine 바이트코드를 실제로 실행하는 가상 머신 실제로 조리하는 장치

정리:
JDK는 개발 도구이고, JRE는 실행 환경이며, JVM은 Java 프로그램을 실제로 실행하는 엔진입니다.
개발자는 보통 JDK를 설치해야 합니다.

1-3. Java 실행 과정

Java는 작성한 소스코드를 바로 실행하지 않습니다.
먼저 사람이 읽을 수 있는 .java 파일을 JVM이 읽을 수 있는 .class 파일로 변환해야 합니다.
이때 생성되는 .class 파일을 바이트코드라고 합니다. 

Hello.java
   ↓ javac 명령어로 컴파일
Hello.class
   ↓ java 명령어로 실행
JVM이 class 파일 로드
   ↓
코드 검증
   ↓
실행
   ↓
결과 출력

이 구조 덕분에 Java는 운영체제에 직접 의존하지 않습니다.
Windows, macOS, Linux마다 JVM만 설치되어 있다면 같은 .class 파일을 실행할 수 있습니다.
이것이 Java의 대표적인 특징인 플랫폼 독립성입니다.

1-4. 바이트코드란?

바이트코드는 Java 소스코드를 컴파일한 결과물입니다.
사람이 직접 읽기 쉬운 코드는 아니지만, JVM이 해석하고 실행할 수 있는 중간 코드입니다.
Java가 운영체제에 바로 맞춰 실행파일을 만드는 방식이 아니라,
JVM이 이해할 수 있는 바이트코드를 만드는 방식을 선택했기 때문에
“한 번 작성하면 여러 환경에서 실행할 수 있다”는 장점이 생깁니다. 

개발자가 작성한 코드
public class Hello {
    public static void main(String[] args) {
        System.out.println("Hello Java");
    }
}

        ↓ 컴파일

JVM이 실행할 수 있는 바이트코드
Hello.class

2. 코드 예제

2-1. 가장 기본적인 Java 프로그램

아래 코드는 Java에서 가장 기본이 되는 프로그램 구조입니다.
public class Hello는 클래스 선언이고,
public static void main(String[] args)는 프로그램의 시작 지점입니다. 

public class Hello {

    public static void main(String[] args) {

        System.out.println("Hello Java");

    }

}

Java 프로그램은 실행될 때 main 메서드를 가장 먼저 찾습니다.
따라서 Java 기초 단계에서는 main 메서드 구조를 반드시 익숙하게 만들어야 합니다.

2-2. 컴파일 명령어

Java 소스파일을 작성한 뒤에는 javac 명령어로 컴파일합니다. 

javac Hello.java

컴파일에 성공하면 같은 폴더에 다음 파일이 생성됩니다. 

Hello.class

2-3. 실행 명령어

컴파일된 class 파일은 java 명령어로 실행합니다. 이때 확장자 .class는 붙이지 않습니다. 

java Hello

실행 결과는 다음과 같습니다.

Hello Java

2-4. Java 버전 확인

JDK가 제대로 설치되었는지 확인하려면 명령 프롬프트에서 다음 명령어를 입력합니다. 

java -version
javac -version

java -version은 Java 실행 환경을 확인하는 명령어이고,
javac -version은 Java 컴파일러가 제대로 동작하는지 확인하는 명령어입니다.
만약 java는 되는데 javac가 되지 않는다면, JRE만 설치했거나 Path 설정이 잘못되었을 가능성이 있습니다.

3. JAVA_HOME과 Path

3-1. JAVA_HOME이란?

JAVA_HOME은 JDK가 설치된 위치를 저장하는 환경 변수입니다.
여러 개발 도구는 Java가 어디에 설치되어 있는지 찾기 위해 JAVA_HOME 값을 확인합니다.

변수명

JAVA_HOME

값 예시

C:\Program Files\Java\jdk-21

중요한 점은 JAVA_HOME에는 보통 bin 폴더까지 넣지 않는다는 것입니다. JDK의 루트 경로까지만 등록합니다.

3-2. Path란?

Path는 터미널이나 명령 프롬프트에서 명령어를 입력했을 때,
운영체제가 실행 파일을 찾기 위해 확인하는 경로 목록입니다.
우리가 아무 폴더에서나 java, javac 명령어를 사용할 수 있는 이유는 Path에 JDK의 bin 경로가 등록되어 있기 때문입니다. 

%JAVA_HOME%\bin

위 경로를 Path에 추가하면 운영체제는 다음 실행 파일들을 찾을 수 있습니다.

  • java.exe
  • javac.exe
  • javadoc.exe
  • jar.exe

3-3. JAVA_HOME 설정에서 자주 하는 실수

잘못된 예시는 다음과 같습니다.

C:\Program Files\Java\jdk-21\bin

위처럼 JAVA_HOME에 bin까지 넣으면 다른 도구에서 JDK 루트 경로를 제대로 찾지 못할 수 있습니다.

올바른 예시는 다음과 같습니다.

C:\Program Files\Java\jdk-21

그리고 Path에는 다음 값을 추가합니다.

%JAVA_HOME%\bin

4. Eclipse 설정

4-1. Installed JREs 설정

Eclipse를 설치한 뒤에는 Eclipse가 어떤 JDK를 사용할지 확인해야 합니다.
특히 내장 JDK가 아니라 직접 설치한 JDK를 사용하고 싶다면 Installed JREs 설정을 확인해야 합니다. 

Window
 └─ Preferences
     └─ Java
         └─ Installed JREs
             └─ C:\Program Files\Java\jdk-버전

이 설정을 통해 Eclipse 프로젝트가 어떤 Java 버전으로 컴파일되고 실행될지 결정됩니다.

4-2. Workspace UTF-8 설정

Java 소스코드에 한글 주석이나 한글 문자열이 들어갈 경우,
인코딩 설정이 맞지 않으면 글자가 깨질 수 있습니다. 그래서 Eclipse의 Workspace 인코딩을 UTF-8로 맞춰두는 것이 좋습니다. 

Window
 └─ Preferences
     └─ General
         └─ Workspace
             └─ Text file encoding
                 └─ Other: UTF-8

HTML 문서에서 <meta charset="UTF-8">을 설정하는 것처럼,
Java 개발환경에서도 UTF-8을 맞춰두면 한글 깨짐 문제를 예방할 수 있습니다.

4-3. module-info.java 체크 해제

Java Project를 만들 때 Create module-info.java 옵션이 보일 수 있습니다.
초급 단계에서는 Java 모듈 시스템까지 함께 다루면 학습 난이도가 올라가기 때문에,
처음에는 체크를 해제하고 기본 Java 프로젝트로 공부하는 것이 좋습니다.

처음 Java 문법을 공부할 때는
클래스, main 메서드, 변수, 자료형, 조건문, 반복문, 객체지향 개념을 먼저 익히는 것이 우선입니다.

5. 실습 체크리스트

아래 항목을 직접 확인하면서 Java 개발환경이 제대로 준비되었는지 점검합니다.

  • JDK 설치 위치 확인하기
  • JAVA_HOME 환경 변수 등록하기
  • Path에 %JAVA_HOME%\bin 등록하기
  • 명령 프롬프트에서 java -version 확인하기
  • 명령 프롬프트에서 javac -version 확인하기
  • Eclipse의 Installed JREs 설정 확인하기
  • Workspace 인코딩을 UTF-8로 설정하기
  • Java Project 생성 시 module-info.java 체크 해제하기
  • Hello.java 파일 만들기
  • javac Hello.java로 컴파일하기
  • java Hello로 실행하기

6. 자주 하는 실수

실수 1. javac 명령어가 안 되는 경우

java -version은 되는데 javac -version이 안 되는 경우가 있습니다.
이 경우는 JDK가 제대로 설치되지 않았거나, Path에 JDK의 bin 경로가 등록되지 않은 경우가 많습니다. 

'javac'은 내부 또는 외부 명령, 실행할 수 있는 프로그램, 또는
배치 파일이 아닙니다.

실수 2. 파일명과 public class 이름이 다른 경우

Java에서는 public class 이름과 파일명이 같아야 합니다. 

// 파일명: Test.java

public class Hello {
    public static void main(String[] args) {
        System.out.println("Hello Java");
    }
}

위 코드는 파일명과 public class 이름이 다르기 때문에 문제가 발생합니다.

// 파일명: Hello.java

public class Hello {
    public static void main(String[] args) {
        System.out.println("Hello Java");
    }
}

실수 3. java 실행 시 확장자를 붙이는 경우

컴파일할 때는 .java 확장자를 붙이지만, 실행할 때는 class 이름만 사용합니다. 

java Hello.class
java Hello

실수 4. UTF-8 설정을 하지 않아 한글이 깨지는 경우

소스 파일 인코딩과 컴파일러가 해석하는 인코딩이 다르면 한글이 깨질 수 있습니다.
특히 한글 주석이나 한글 문자열을 많이 사용하는 학습 단계에서는 UTF-8 설정을 통일하는 것이 중요합니다.

7. 실무 연결

7-1. Spring Boot도 결국 JDK 위에서 실행된다

나중에 Spring Boot 백엔드 프로젝트를 만들 때도 JDK 설정은 매우 중요합니다.
Spring Boot 애플리케이션은 Java로 작성되고 JVM 위에서 실행됩니다.
Gradle이나 Maven 같은 빌드 도구도 JDK를 기준으로 프로젝트를 컴파일합니다. 

Spring Boot 프로젝트
   ↓
Gradle 또는 Maven 빌드
   ↓
Java 코드 컴파일
   ↓
JVM 위에서 실행
   ↓
웹 서버 동작

7-2. JDK 버전 충돌은 실무에서 자주 발생한다

실무에서는 프로젝트마다 요구하는 JDK 버전이 다를 수 있습니다.
어떤 프로젝트는 JDK 8을 사용하고, 어떤 프로젝트는 JDK 17이나 JDK 21을 사용할 수 있습니다.
이때 JAVA_HOME이 잘못 잡혀 있으면 프로젝트가 실행되지 않거나 빌드가 실패할 수 있습니다.

  • Spring Boot 버전과 JDK 버전이 맞지 않는 경우
  • Gradle이 사용하는 Java 버전과 Eclipse가 사용하는 Java 버전이 다른 경우
  • 환경 변수 JAVA_HOME이 예전 JDK를 가리키는 경우
  • Path에 여러 JDK 경로가 섞여 있는 경우

7-3. 인코딩 문제는 웹 개발에서도 이어진다

Java에서 UTF-8 설정을 맞추는 습관은 웹 개발에서도 그대로 이어집니다.
HTML, CSS, JavaScript, Java, DB, JSON, API 응답까지 인코딩이 일치해야 한글 데이터가 깨지지 않습니다. 

HTML 문서      → UTF-8
Java 소스코드  → UTF-8
DB 문자셋      → UTF-8
API 응답       → UTF-8

모두 맞아야 한글 깨짐을 예방할 수 있다.

8. 면접 포인트

Q1. JDK와 JRE의 차이는 무엇인가요?

답변:
JDK는 Java Development Kit으로 Java 프로그램을 개발하기 위한 도구입니다.
JRE는 Java Runtime Environment로 Java 프로그램을 실행하기 위한 환경입니다.
JDK 안에는 JRE가 포함되어 있으며, 개발자는 컴파일 도구가 필요하기 때문에 JDK를 설치해야 합니다.

Q2. JVM은 어떤 역할을 하나요?

답변:
JVM은 Java Virtual Machine으로 Java 바이트코드를 실행하는 가상 머신입니다.
class 파일을 로드하고, 코드를 검증하고, 메모리를 관리하며, 프로그램을 실행합니다.
또한 Garbage Collection을 통해 사용하지 않는 메모리를 정리합니다.

Q3. Java가 플랫폼 독립적이라고 하는 이유는 무엇인가요?

답변:
Java는 운영체제에 직접 맞는 실행 파일을 만드는 것이 아니라,
JVM이 실행할 수 있는 바이트코드를 만듭니다.
운영체제마다 JVM만 설치되어 있다면 같은 바이트코드를 실행할 수 있기 때문에 플랫폼 독립적이라고 합니다.

Q4. javac와 java 명령어의 차이는 무엇인가요?

답변:
javac는 Java 소스 파일인 .java 파일을 컴파일하여
.class 바이트코드 파일을 생성하는 명령어입니다.
java는 생성된 class 파일을 JVM 위에서 실행하는 명령어입니다.

Q5. JAVA_HOME은 왜 설정하나요?

답변:
JAVA_HOME은 JDK가 설치된 위치를 알려주는 환경 변수입니다.
Eclipse, Gradle, Maven, Tomcat 같은 개발 도구가 Java 위치를 찾을 때 사용합니다.
Path에는 JAVA_HOME의 bin 폴더를 등록해서 터미널 어디서든 java, javac 명령어를 사용할 수 있게 합니다.

9. 오늘의 핵심 정리

  1. JDK는 Java 개발 도구 전체를 의미한다.
  2. JRE는 Java 프로그램을 실행하기 위한 환경이다.
  3. JVM은 Java 바이트코드를 실제로 실행하는 가상 머신이다.
  4. Java 소스파일은 .java이고, 컴파일 결과는 .class 파일이다.
  5. javac는 컴파일 명령어이고, java는 실행 명령어이다.
  6. JAVA_HOME에는 JDK의 루트 경로를 등록한다.
  7. Path에는 %JAVA_HOME%\bin을 등록한다.
  8. Eclipse에서는 Installed JREs와 UTF-8 설정을 확인해야 한다.
  9. 초급 단계에서는 module-info.java 생성을 체크 해제하고 시작하는 것이 좋다.
  10. Java 개발환경 이해는 Spring Boot, Android, 서버 개발의 기초가 된다.
