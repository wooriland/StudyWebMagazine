18. this와 super

this는 현재 객체 자기 자신을 의미하고,
super는 현재 객체의 부모 클래스 영역을 의미합니다.
이 페이지에서는 필드와 매개변수 구분, 부모 멤버 접근, static 메서드 제한, this()와 super()의 차이를 공부합니다.

이번 페이지의 학습 목표

<> this와 super는 객체지향 문법에서 자주 등장하는 키워드입니다.
this는 현재 객체의 멤버를 명확하게 가리킬 때 사용하고, super는 상속 관계에서 부모 클래스의 멤버를 명확하게 가리킬 때 사용합니다.

this 이해

현재 객체 자신의 인스턴스 멤버에 접근하는 방법을 익힙니다.

필드와 매개변수 구분

this.name = name; 구조를 정확히 이해합니다.

super 이해

상속 관계에서 부모 클래스의 필드와 메서드에 접근하는 방법을 익힙니다.

제한 사항 이해

static 메서드, 부모 private 멤버 접근 제한을 이해합니다.

1. this의 기본 개념

1-1. this란?

this는 현재 객체 자기 자신을 가리키는 키워드입니다. 현재 객체가 가지고 있는 인스턴스 필드나 인스턴스 메서드에 접근할 때 사용할 수 있습니다. 

this.name
this.balance
this.printInfo()
this.name
→ 현재 객체의 name 멤버 변수

this.balance
→ 현재 객체의 balance 멤버 변수

this.printInfo()
→ 현재 객체의 printInfo() 메서드

1-2. this가 필요한 이유

this가 가장 많이 필요한 상황은 멤버 변수와 매개변수 이름이 같을 때입니다. 

private int balance;

public void setBalance(int balance) {
    balance = balance;
}

위 코드는 의도한 대로 동작하지 않습니다. 왼쪽과 오른쪽의 balance가 모두 매개변수로 해석될 수 있기 때문입니다. 

balance = balance;
→ 매개변수 balance = 매개변수 balance

결과:
멤버 변수 balance에 값이 들어가지 않음

private int balance;

public void setBalance(int balance) {
    this.balance = balance;
}
this.balance
→ 현재 객체의 멤버 변수 balance

balance
→ 메서드로 전달받은 매개변수 balance

1-3. 캡슐화와 this

캡슐화 단원에서 setter를 만들 때 this가 자주 등장합니다. 필드명과 매개변수명을 같게 쓰면 코드 의미가 자연스럽고, this로 멤버 변수를 명확히 구분할 수 있습니다. 

class Account {

    private String name;
    private String accountNo;
    private int balance;

    public void setName(String name) {
        this.name = name;
    }

    public void setAccountNo(String accountNo) {
        this.accountNo = accountNo;
    }

    public void setBalance(int balance) {
        if (balance >= 0) {
            this.balance = balance;
        }
    }

}

1-4. this는 객체마다 달라진다

this는 “현재 이 메서드를 호출한 객체”를 의미합니다. 같은 메서드라도 어떤 객체가 호출했는지에 따라 this가 가리키는 대상이 달라집니다. 

User user1 = new User();
User user2 = new User();

user1.setLoginId("test01");
user2.setLoginId("test02");
user1.setLoginId("test01")
→ this = user1 객체

user2.setLoginId("test02")
→ this = user2 객체

1-5. this는 static 메서드에서 사용할 수 없다

this는 현재 객체를 의미합니다. 그런데 static 메서드는 객체 생성 없이 클래스명으로 호출할 수 있으므로 현재 객체라는 개념이 없습니다. 

class User {

    private String name;

    public static void print() {
        System.out.println(this.name); // 오류
    }

}
인스턴스 메서드
→ 객체가 있어야 호출 가능
→ this 사용 가능

static 메서드
→ 객체 없이 클래스명으로 호출 가능
→ this 사용 불가

2. super의 기본 개념

2-1. super란?

super는 부모 클래스의 멤버에 접근할 때 사용하는 키워드입니다. 상속 관계에서 자식 클래스가 부모 클래스의 필드나 메서드를 명확하게 가리킬 때 사용합니다. 

class Parent {
    String name = "부모";
}

class Child extends Parent {

    String name = "자식";

    public void printName() {
        System.out.println(this.name);
        System.out.println(super.name);
    }

}
this.name
→ 현재 Child 객체의 name

super.name
→ Parent 영역의 name

2-2. super가 필요한 이유

자식 클래스가 부모 클래스와 같은 이름의 필드나 메서드를 가질 수 있습니다. 이때 그냥 이름만 쓰면 자식 멤버가 우선됩니다. 부모 멤버를 명확히 가리키려면 super를 사용합니다. 

class Parent {
    String message = "부모 메시지";
}

class Child extends Parent {

    String message = "자식 메시지";

    public void printMessage() {
        System.out.println(message);
        System.out.println(super.message);
    }

}
message
→ 자식 메시지

super.message
→ 부모 메시지

2-3. super로 부모 메서드 호출

super는 부모 클래스의 메서드를 호출할 때도 사용할 수 있습니다. 특히 자식 클래스가 부모 메서드를 오버라이딩했을 때 부모의 원래 기능을 호출하고 싶다면 super.메서드명()을 사용합니다. 

class Parent {

    public void printInfo() {
        System.out.println("Parent의 printInfo()");
    }

}

class Child extends Parent {

    public void printInfo() {
        System.out.println("Child의 printInfo()");
    }

    public void printAll() {
        this.printInfo();
        super.printInfo();
    }

}

2-4. super는 부모 private 멤버에 직접 접근할 수 없다

부모 클래스의 private 멤버는 부모 클래스 내부에서만 접근할 수 있습니다. 자식 클래스라도 super를 통해 직접 접근할 수 없습니다. 

class Parent {

    private String secret = "비밀";

}

class Child extends Parent {

    public void printSecret() {
        System.out.println(super.secret); // 오류
    }

}

필요하다면 부모 클래스가 protected 또는 public 메서드를 제공해야 합니다. 

class Parent {

    private String secret = "비밀";

    protected String getSecret() {
        return secret;
    }

}

class Child extends Parent {

    public void printSecret() {
        System.out.println(super.getSecret());
    }

}

2-5. super도 static 메서드에서 사용할 수 없다

super는 현재 객체의 부모 영역을 의미합니다. 따라서 this처럼 객체가 있어야 의미가 있습니다. static 메서드는 객체 없이 호출될 수 있으므로 super를 사용할 수 없습니다. 

class Parent {
    String name = "부모";
}

class Child extends Parent {

    public static void print() {
        System.out.println(super.name); // 오류
    }

}

this는 현재 객체, super는 현재 객체의 부모 영역입니다. 둘 다 객체가 있어야 의미가 있으므로 static 메서드 안에서는 사용할 수 없습니다.

3. this와 super 비교

3-1. 비교 표

구분 this super
의미 현재 객체 자신 부모 클래스 영역
주 사용 상황 필드명과 매개변수명 구분 부모 멤버와 자식 멤버 구분
접근 대상 현재 클래스의 인스턴스 멤버 부모 클래스의 멤버
static 메서드에서 사용 불가 불가
private 접근 현재 클래스의 private 멤버 접근 가능 부모 private 멤버 직접 접근 불가
연결 단원 캡슐화, 생성자 상속, 오버라이딩, 생성자

3-2. this와 this(), super와 super() 구분

괄호가 없는 this, super와 괄호가 붙은 this(), super()는 의미가 다릅니다. 

this
→ 현재 객체 자신

super
→ 부모 멤버

this()
→ 같은 클래스의 다른 생성자 호출

super()
→ 부모 생성자 호출

이번 페이지에서는 this와 super를 먼저 이해하고, 생성자 단원에서 this()와 super()를 더 자세히 공부합니다.

6. 실무 연결

6-1. DTO / Entity setter에서 this 사용

public class UserEntity {

    private Long userId;
    private String loginId;
    private String email;

    public void setUserId(Long userId) {
        this.userId = userId;
    }

    public void setLoginId(String loginId) {
        this.loginId = loginId;
    }

    public void setEmail(String email) {
        this.email = email;
    }

}

필드명과 매개변수명을 같게 쓰면 의미가 자연스럽고, this로 현재 객체의 필드를 명확히 구분할 수 있습니다.

6-2. 의미 있는 변경 메서드에서 this 사용

public class User {

    private String password;
    private String email;

    public void changePassword(String password) {
        if (password != null && password.length() >= 8) {
            this.password = password;
        }
    }

    public void changeEmail(String email) {
        if (email != null && email.contains("@")) {
            this.email = email;
        }
    }

}

6-3. 상속 구조에서 super 사용

class BaseService {

    public void log(String message) {
        System.out.println("[LOG] " + message);
    }

}

class UserService extends BaseService {

    public void createUser() {
        super.log("회원 생성 시작");
        System.out.println("회원 생성 로직 실행");
        super.log("회원 생성 완료");
    }

}

자식 클래스가 부모 클래스의 공통 기능을 재사용할 때 super를 사용할 수 있습니다.

6-4. 오버라이딩에서 super 사용

class Report {

    public void print() {
        System.out.println("기본 보고서 출력");
    }

}

class AiReport extends Report {

    public void print() {
        super.print();
        System.out.println("AI 분석 결과 추가 출력");
    }

}

부모의 기본 기능을 먼저 실행하고, 자식 클래스에서 추가 기능을 덧붙이는 구조입니다.

8. 면접 포인트

Q1. this 키워드는 무엇인가요?

답변: this는 현재 객체 자기 자신을 가리키는 키워드입니다. 인스턴스 멤버에 접근할 때 사용하며, 특히 멤버 변수와 매개변수 이름이 같을 때 둘을 구분하기 위해 사용합니다.

Q2. this는 언제 주로 사용하나요?

답변: setter나 생성자에서 멤버 변수와 매개변수 이름이 같을 때 주로 사용합니다.
예를 들어 this.name = name;에서 this.name은 현재 객체의 필드이고, name은 매개변수입니다.

Q3. this를 static 메서드에서 사용할 수 있나요?

답변: 사용할 수 없습니다. this는 현재 객체를 의미하지만, static 메서드는 객체 생성 없이 클래스명으로 호출될 수 있기 때문에 현재 객체라는 개념이 없습니다.

Q4. super 키워드는 무엇인가요?

답변: super는 부모 클래스의 멤버에 접근할 때 사용하는 키워드입니다. 자식 클래스에서 부모 클래스의 필드나 메서드를 명시적으로 호출할 때 사용합니다.

Q5. super로 부모의 private 멤버에 접근할 수 있나요?

답변: 접근할 수 없습니다. 부모의 private 멤버는 부모 클래스 내부에서만 접근 가능하기 때문에, 자식 클래스에서도 super를 통해 직접 접근할 수 없습니다.
필요하다면 부모 클래스에서 protected 또는 public 메서드를 제공해야 합니다.

Q6. super를 static 메서드에서 사용할 수 있나요?

답변: 사용할 수 없습니다. super는 현재 객체의 부모 영역을 의미하므로 객체가 있어야 사용할 수 있습니다.
static 메서드는 객체 없이 호출될 수 있기 때문에 super를 사용할 수 없습니다.

Q7. this와 super의 차이는 무엇인가요?

답변: this는 현재 객체 자신을 의미하고, super는 현재 객체의 부모 클래스 영역을 의미합니다.
this는 현재 클래스의 인스턴스 멤버에 접근할 때 사용하고, super는 부모 클래스의 멤버에 접근할 때 사용합니다.

Q8. this와 this(), super와 super()의 차이는 무엇인가요?

답변: this는 현재 객체 자신을 의미하는 참조 키워드입니다. this()는 같은 클래스의 다른 생성자를 호출하는 문법입니다.
super는 부모 클래스의 멤버에 접근하는 참조 키워드입니다. super()는 부모 클래스의 생성자를 호출하는 문법입니다.

9. 오늘의 핵심 정리

  1. this는 현재 객체 자기 자신을 의미한다.
  2. this는 자기 자신의 인스턴스형 멤버에 접근할 때 사용한다.
  3. 멤버 변수와 매개변수 이름이 같을 때 this로 구분한다.
  4. this는 static 메서드 안에서 사용할 수 없다.
  5. super는 부모 클래스의 멤버에 접근할 때 사용한다.
  6. super는 상속 관계에서 사용한다.
  7. super로 부모의 private 멤버에는 직접 접근할 수 없다.
  8. super도 static 메서드 안에서 사용할 수 없다.
  9. this는 현재 클래스 영역, super는 부모 클래스 영역을 가리킨다고 이해하면 쉽다.
  10. this와 super는 캡슐화, 생성자, 상속, 다형성과 계속 연결된다.

학습 메모

이 페이지는 Java의 this와 super 단원입니다.
this는 현재 객체의 인스턴스 멤버에 접근할 때 사용하고, super는 상속 관계에서 부모 클래스의 멤버에 접근할 때 사용합니다.
둘 다 객체를 기준으로 동작하므로 static 메서드 안에서는 사용할 수 없습니다. 또한 this, super와 this(), super()는 의미가 다르므로 반드시 구분해야 합니다.
