03. Java 프로그램 구조

Java 프로그램은 package, import, class, main 메서드를 중심으로 구성됩니다.
이 페이지에서는 Java 파일의 기본 뼈대와 실행 시작점,
그리고 실무 프로젝트 구조로 확장되는 흐름을 공부합니다.

이번 페이지의 학습 목표

이 페이지에서는 Java 코드가 어떤 기본 구조로 작성되는지 공부합니다.
Java 프로그램은 대부분 class를 중심으로 작성되며,
package, import, class, main 메서드가 Java 파일의 기본 뼈대를 이룹니다.

기본 구조 이해

Java 파일이
package, import, class 순서로 구성된다는 점을 이해합니다.

main 메서드 이해

JVM이 프로그램을 실행할 때
main 메서드를 시작점으로 찾는다는 점을 이해합니다.

파일명 규칙 이해

public class 이름과 Java 파일명이 같아야 하는 이유를
이해합니다.

실무 구조 연결

class 구조가 Spring Boot의 Controller, Service, Repository
구조로 확장되는 흐름을 이해합니다.

1. 핵심 개념 설명

1-1. Java 프로그램은 클래스 단위로 작성된다

Java는 대부분의 코드를 class 안에 작성합니다.
변수, 메서드, main 메서드도 기본적으로 클래스 내부에 들어갑니다.
따라서 Java를 공부할 때 가장 먼저 익숙해져야 하는 구조는 클래스입니다. 

public class Hello {
}

초급 단계에서는 class를 Java 코드를 담는 큰 상자로 이해하면 좋습니다.
이후 객체지향을 배우면 class는 객체를 만들기 위한 설계도라는 의미로 더 깊게 확장됩니다.

정리: Java 프로그램은 클래스 단위로 작성되며, 클래스 안에 데이터와 기능이 함께 들어갈 수 있습니다.

1-2. Java 파일의 기본 작성 순서

Java 파일은 보통 아래 순서로 작성합니다.
모든 파일에 package와 import가 반드시 필요한 것은 아니지만,
실무 프로젝트에서는 거의 항상 package와 import가 등장합니다. 

Java 파일 기본 구조

package 패키지명;

import 사용할클래스;

public class 클래스명 {

    멤버변수;

    메서드;

    public static void main(String[] args) {
        실행문;
    }

}

작성 순서는 중요합니다. package가 있다면 가장 위에 와야 하고, import는 package 아래, class 위에 작성합니다.

  1. package 선언부
  2. import 선언부
  3. class 선언부
  4. 멤버변수와 메서드
  5. main 메서드

1-3. package 선언부

package는 클래스를 폴더처럼 묶어서 관리하기 위한 선언입니다.
프로젝트가 작을 때는 클래스가 몇 개 없어서 package의 필요성이 잘 느껴지지 않을 수 있습니다.
하지만 프로젝트가 커지면 클래스 수가 많아지고, 역할별로 정리할 필요가 생깁니다. 

package basic;

위 코드는 이 Java 파일이 basic이라는 패키지에 속한다는 의미입니다.
package 선언은 반드시 파일의 가장 위쪽에 작성해야 하며, import보다 먼저 와야 합니다. 

package basic;

basic 폴더
└─ Hello.java

1-4. import 선언부

import는 다른 패키지에 있는 클래스를 현재 파일에서 사용하겠다는 선언입니다.
예를 들어 사용자 입력을 받을 때 자주 사용하는 Scanner 클래스는 java.util 패키지에 들어 있습니다. 

import java.util.Scanner;

import를 작성하면 아래처럼 클래스 이름만으로 사용할 수 있습니다. 

Scanner sc = new Scanner(System.in);

만약 import를 사용하지 않으면 전체 경로를 매번 작성해야 합니다. 

java.util.Scanner sc = new java.util.Scanner(System.in);

1-5. java.lang 패키지는 자동 import된다

Java에서 가장 기본적으로 사용되는 클래스들은 java.lang 패키지에 들어 있습니다.
이 패키지는 자동으로 import되기 때문에 직접 import하지 않아도 사용할 수 있습니다.

  • String
  • System
  • Math
  • Integer
  • Thread
String name = "Java";
System.out.println(name);

위 코드에서 String과 System은 import 없이 사용할 수 있습니다. 이것은 Java가 java.lang.*을 기본적으로 사용할 수 있게 해주기 때문입니다.

1-6. public class와 파일명 규칙

Java에서는 public class가 있는 경우, 그 클래스 이름과 파일명이 반드시 같아야 합니다. 

// 파일명: Hello.java

public class Hello {
}

위 코드는 올바른 예입니다. public class Hello이므로 파일명도 Hello.java여야 합니다. 

// 파일명: Test.java

public class Hello {
}

위 코드는 잘못된 예입니다. 파일명과 public class 이름이 다르기 때문에 컴파일 오류가 발생할 수 있습니다.

1-7. main 메서드

Java 프로그램의 시작점은 main 메서드입니다.
JVM은 프로그램을 실행할 때 정해진 형태의 main 메서드를 찾아서 그 안의 코드를 실행합니다. 

public static void main(String[] args) {
}
구성 의미
public 어디서든 접근할 수 있다는 의미입니다.
static 객체를 생성하지 않아도 실행할 수 있다는 의미입니다.
void 반환값이 없다는 의미입니다.
main JVM이 프로그램 시작점으로 찾는 메서드 이름입니다.
String[] args 프로그램 실행 시 전달받을 수 있는 문자열 배열입니다.

핵심: 초급 단계에서는 public static void main(String[] args) 구조를 먼저 정확히 익히는 것이 중요합니다.

1-8. 실행문과 세미콜론

Java의 실행문 대부분은 세미콜론 ;으로 끝납니다. 세미콜론은 Java에서 하나의 문장이 끝났음을 나타냅니다. 

System.out.println("Hello");
int age = 20;
String name = "홍길동";

반대로 클래스 선언이나 메서드 선언처럼 중괄호 블록을 여는 구조에는 일반적으로 세미콜론을 붙이지 않습니다.

1-9. 중괄호와 코드 블록

Java에서 중괄호 { }는 코드의 범위를 나타냅니다.
클래스의 범위, 메서드의 범위, 조건문과 반복문의 범위가 모두 중괄호로 구분됩니다. 

public class Hello {

    public static void main(String[] args) {

        System.out.println("Hello Java");

    }

}
class 블록
└─ main 메서드 블록
└─ 실행문

중괄호는 열었으면 반드시 닫아야 합니다. 중괄호 짝이 맞지 않으면 컴파일 오류가 발생합니다.

1-10. 들여쓰기

Java에서 들여쓰기는 문법적으로 필수는 아니지만, 코드를 읽기 쉽게 만들기 위해 반드시 지켜야 하는 습관입니다.
특히 팀 프로젝트에서는 들여쓰기와 코드 정렬이 코드 품질에 큰 영향을 줍니다. 

public class Hello {

    public static void main(String[] args) {
        System.out.println("읽기 좋은 코드");
    }

}
public class Hello {
public static void main(String[] args) {
System.out.println("읽기 어려운 코드");
}
}

3. 코드 예제

3-1. 가장 기본적인 Java 구조

public class Hello {

    public static void main(String[] args) {

        System.out.println("Hello Java");

    }

}

이 예제는 Java 프로그램의 가장 기본적인 형태입니다.
Hello 클래스 안에 main 메서드가 있고, main 메서드 안에서 출력문이 실행됩니다.

3-2. package와 import가 있는 구조

package basic;

import java.util.Scanner;

public class InputExample {

    public static void main(String[] args) {

        Scanner sc = new Scanner(System.in);

        System.out.print("이름 입력: ");
        String name = sc.nextLine();

        System.out.println("안녕하세요, " + name + "님!");

        sc.close();

    }

}

이 예제에서는 basic 패키지를 사용하고, 사용자 입력을 받기 위해 java.util.Scanner를 import합니다.

3-3. 멤버변수와 메서드가 있는 구조

public class Student {

    String name;
    int age;

    public void introduce() {
        System.out.println("이름: " + name);
        System.out.println("나이: " + age);
    }

    public static void main(String[] args) {

        Student student = new Student();

        student.name = "홍길동";
        student.age = 20;

        student.introduce();

    }

}
코드 요소 의미
Student 클래스 이름입니다.
name, age 클래스가 가지는 멤버변수입니다.
introduce() 학생 정보를 출력하는 메서드입니다.
main() 프로그램 시작점입니다.
student Student 객체를 참조하는 변수입니다.

3-4. 실행 흐름 예제

public class ProgramFlow {

    public static void main(String[] args) {

        System.out.println("1. 프로그램 시작");
        System.out.println("2. Java 구조 학습");
        System.out.println("3. 프로그램 종료");

    }

}

출력 결과는 다음과 같습니다.

1. 프로그램 시작
2. Java 구조 학습
3. 프로그램 종료

Java의 기본 실행 흐름은 main 메서드 안에서 위에서 아래로 진행됩니다.

3-5. 한 파일에 여러 클래스 작성하기

// 파일명: MainClass.java

public class MainClass {

    public static void main(String[] args) {
        Helper helper = new Helper();
        helper.printMessage();
    }

}

class Helper {

    public void printMessage() {
        System.out.println("Helper 클래스 실행");
    }

}

Java에서는 한 파일 안에 여러 클래스를 작성할 수 있습니다. 하지만 public class는 한 파일에 하나만 작성할 수 있고, public class 이름은 파일명과 같아야 합니다.

4. 실무 연결

4-1. package는 프로젝트 구조가 된다

초급 예제에서는 package basic; 정도로 간단하게 시작하지만, 실무에서는 package가 프로젝트 구조를 결정합니다.
특히 Spring Boot 프로젝트에서는 역할별로 패키지를 나누어 코드를 관리합니다.

패키지 역할
controller 사용자의 요청을 받는 클래스가 위치합니다.
service 비즈니스 로직을 처리하는 클래스가 위치합니다.
repository DB 접근을 담당하는 클래스가 위치합니다.
entity DB 테이블과 연결되는 객체가 위치합니다.
dto 데이터 전달용 객체가 위치합니다.
config 프로젝트 설정 클래스가 위치합니다.

4-2. class는 역할 단위로 분리된다

처음에는 모든 코드를 main 메서드 안에 작성하지만,
실무에서는 하나의 클래스가 모든 일을 담당하지 않도록 역할을 나눕니다. 

public class UserController {
}

public class UserService {
}

public class UserRepository {
}

이렇게 나누면 코드가 커져도 유지보수하기 쉬워집니다. Java의 class 구조는 이후 객체지향 프로그래밍과 Spring MVC 구조의 기초가 됩니다.

4-3. import를 보면 이 클래스가 사용하는 기능을 알 수 있다

실무 코드에서는 import 목록만 보아도 해당 클래스가 어떤 기능을 사용하는지 어느 정도 파악할 수 있습니다. 

import java.util.List;
import java.time.LocalDateTime;
import org.springframework.stereotype.Service;

위 import를 보면 이 클래스는 목록 자료구조, 날짜/시간, Spring의 Service 기능과 관련이 있다는 것을 추측할 수 있습니다.

4-4. Spring Boot도 main 메서드에서 시작한다

Spring Boot 프로젝트도 결국 Java 프로그램이기 때문에 main 메서드에서 시작합니다. 

@SpringBootApplication
public class ProjectApplication {

    public static void main(String[] args) {
        SpringApplication.run(ProjectApplication.class, args);
    }

}

즉, 지금 배우는 public static void main(String[] args) 구조는
단순한 기초 예제가 아니라 실제 백엔드 서버 실행 구조와도 연결됩니다.

5. 자주 하는 실수

실수 1. 파일명과 public class 이름이 다른 경우

// 파일명: Test.java

public class Hello {
}

올바른 예시는 다음과 같습니다.

// 파일명: Hello.java

public class Hello {
}

실수 2. package보다 import를 먼저 작성하는 경우

import java.util.Scanner;

package basic;

public class Hello {
}

올바른 순서는 package가 먼저입니다.

package basic;

import java.util.Scanner;

public class Hello {
}

실수 3. main 메서드 형태가 다른 경우

public void main(String[] args) {
}

위 코드는 static이 빠져 있습니다. JVM이 일반적인 프로그램 시작점으로 인식하는 형태는 다음과 같습니다. 

public static void main(String[] args) {
}

실수 4. 세미콜론을 빠뜨리는 경우

System.out.println("Hello")

올바른 실행문은 세미콜론으로 끝납니다.

System.out.println("Hello");

실수 5. 중괄호 짝이 맞지 않는 경우

public class Hello {

    public static void main(String[] args) {

        System.out.println("Hello");

}

올바른 예시는 다음과 같습니다.

public class Hello {

    public static void main(String[] args) {

        System.out.println("Hello");

    }

}

실수 6. import하지 않고 Scanner를 사용하는 경우

public class InputExample {

    public static void main(String[] args) {

        Scanner sc = new Scanner(System.in);

    }

}

Scanner는 java.util 패키지에 있으므로 import가 필요합니다. 

import java.util.Scanner;

public class InputExample {

    public static void main(String[] args) {

        Scanner sc = new Scanner(System.in);

    }

}

6. 면접 포인트

Q1. Java 프로그램의 기본 구조를 설명해보세요.

답변: Java 프로그램은 일반적으로 package 선언, import 선언, class 선언으로 구성됩니다.
class 내부에는 멤버변수와 메서드가 들어갈 수 있으며, 실행 가능한 Java 프로그램은 main 메서드를 시작점으로 실행됩니다.

Q2. main 메서드는 왜 필요한가요?

답변: main 메서드는 Java 프로그램의 시작 지점입니다.
JVM은 프로그램을 실행할 때 public static void main(String[] args) 형태의 메서드를 찾아 실행합니다.

Q3. public class 이름과 파일명이 같아야 하는 이유는 무엇인가요?

답변: Java에서는 public class를 외부에서 접근 가능한 대표 클래스로 보기 때문에,
컴파일러가 해당 public class를 찾기 쉽도록 파일명과 public class 이름을 일치시켜야 합니다.

Q4. package의 역할은 무엇인가요?

답변: package는 관련 있는 클래스들을 묶어서 관리하기 위한 이름 공간입니다.
폴더 구조와 연결되며, 클래스 이름 충돌을 방지하고 프로젝트를 역할별로 정리하는 데 사용됩니다.

Q5. import는 왜 사용하나요?

답변: import는 다른 패키지에 있는 클래스를 현재 파일에서 간단한 이름으로 사용하기 위해 작성합니다.
import하지 않으면 클래스의 전체 경로를 매번 작성해야 합니다.

Q6. java.lang 패키지는 왜 import하지 않아도 되나요?

답변: java.lang 패키지는 Java 프로그램에서 가장 기본적으로 사용되는 클래스들을 포함하기 때문에 자동으로 import됩니다.
String, System, Math 같은 클래스가 여기에 포함됩니다.

Q7. 세미콜론은 어떤 역할을 하나요?

답변: 세미콜론은 Java에서 하나의 실행문이 끝났음을 나타냅니다.
변수 선언문, 출력문, 대입문 같은 대부분의 실행문은 세미콜론으로 끝납니다.

7. 오늘의 핵심 정리

  1. Java 프로그램은 클래스 단위로 작성된다.
  2. package는 클래스를 묶는 선언이며 파일 최상단에 위치한다.
  3. import는 다른 패키지의 클래스를 현재 파일에서 사용하기 위한 선언이다.
  4. java.lang 패키지는 자동 import된다.
  5. public class 이름은 파일명과 같아야 한다.
  6. main 메서드는 Java 프로그램의 시작점이다.
  7. 실행문은 대부분 세미콜론으로 끝난다.
  8. 중괄호는 코드 블록의 범위를 나타낸다.
  9. 들여쓰기는 문법 필수는 아니지만 가독성을 위해 중요하다.
  10. 실무에서는 class를 Controller, Service, Repository처럼 역할별로 나누어 사용한다.
