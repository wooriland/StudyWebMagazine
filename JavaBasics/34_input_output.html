34. Java IO Input / Output Stream

Java IO는 프로그램이 파일, 키보드, 콘솔, 네트워크 같은 외부 대상과
데이터를 읽고 쓰기 위해 사용하는 입출력 기능입니다. 이번 단원에서는
InputStream, OutputStream, Reader, Writer, File, BufferedReader, ObjectStream을 중심으로 공부합니다.

이번 페이지의 학습 목표

Java IO는 단순히 콘솔에 값을 출력하거나 키보드 입력을 받는 기능만 의미하지 않습니다.
실제 개발에서는 파일 업로드, 파일 다운로드, 로그 분석, CSV 처리, 이미지 저장, 설정 파일 읽기, 객체 저장, 네트워크 데이터 송수신처럼 다양한 상황에서 사용됩니다.
이번 페이지에서는 IO 클래스를 무작정 외우는 것이 아니라, 데이터가 어디서 들어오고, 어디로 나가며, 어떤 스트림 조합을 선택해야 하는지를 이해하는 것을 목표로 합니다.

텍스트 파일, 이미지 파일, 객체 저장,
로그 분석 상황에 맞는
스트림 조합을 익힙니다.

1. IO 기본 개념

1-1. IO란?

IO는 Input / Output의 줄임말입니다. Java 프로그램이 외부에서 데이터를 읽어오거나, 프로그램 안의 데이터를 외부로 내보내는 작업을 의미합니다. 

Input
외부 데이터 → Java 프로그램

Output
Java 프로그램 → 외부 대상

예를 들어 사용자가 키보드로 값을 입력하면 데이터가 프로그램 안으로 들어오기 때문에 입력입니다.
반대로 프로그램이 콘솔에 문장을 출력하거나 파일에 내용을 저장하면 출력입니다.

구분 의미 예시
Input 외부 데이터가 프로그램 안으로 들어옴 키보드 입력, 파일 읽기, 네트워크 수신
Output 프로그램 안의 데이터가 외부로 나감 콘솔 출력, 파일 저장, 네트워크 전송

1-2. 데이터 소스와 데이터 목적지

IO를 이해할 때는 데이터가 어디서 출발하는지와 어디로 도착하는지를 먼저 생각해야 합니다.

Data Source

데이터가 시작되는 곳입니다.

  • 파일
  • 키보드
  • 네트워크
  • 메모리
  • 데이터베이스

Data Destination

데이터가 최종적으로 도착하는 곳입니다.

  • 파일
  • 콘솔 화면
  • 프린터
  • 네트워크
  • 메모리

파일을 읽는 상황에서는 파일이 Data Source입니다. 파일에 저장하는 상황에서는 파일이 Data Destination입니다.

1-3. 스트림이란?

스트림은 데이터가 한 방향으로 흐르는 통로입니다. 물이 수도관을 따라 흐르듯이, 데이터도 스트림을 따라 순서대로 이동합니다. 

파일
↓
입력 스트림
↓
Java 프로그램

Java 프로그램
↓
출력 스트림
↓
파일

스트림은 데이터를 한 번에 통째로 순간이동시키는 것이 아니라, 순서대로 흘려보냅니다. 그래서 스트림은 기본적으로 FIFO, 즉 먼저 들어간 데이터가 먼저 나오는 구조를 가집니다.

2. 스트림의 특징

2-1. FIFO 구조

FIFO는 First In First Out의 줄임말입니다. 먼저 들어간 데이터가 먼저 나오는 구조입니다. 

입력 순서
A → B → C

처리 순서
A → B → C

스트림에 들어간 데이터는 중간 데이터를 마음대로 먼저 꺼내는 구조가 아니라, 들어온 순서대로 처리됩니다.

2-2. 단방향 구조

스트림은 기본적으로 단방향입니다. 읽기용 스트림과 쓰기용 스트림이 구분됩니다.

방향 대표 클래스 역할
입력 InputStream, Reader 외부 데이터를 Java 프로그램 안으로 읽어옵니다.
출력 OutputStream, Writer Java 프로그램의 데이터를 외부로 내보냅니다.

FileInputStream은 읽기 전용이고, FileOutputStream은 쓰기 전용입니다. 하나의 스트림이 읽기와 쓰기를 동시에 담당하지 않습니다.

2-3. Blocking

스트림은 데이터가 준비될 때까지 기다리는 경우가 있습니다. 이 상태를 Blocking이라고 합니다. 

int data = System.in.read();

위 코드는 사용자가 키보드로 값을 입력할 때까지 다음 코드로 넘어가지 않습니다. 프로그램이 멈춘 것처럼 보이지만, 실제로는 입력을 기다리는 상태입니다. 

System.in.read()
↓
사용자 입력 대기
↓
입력 발생
↓
다음 코드 실행

3. Java IO의 큰 분류

3-1. 데이터 단위에 따른 분류

Java IO는 데이터를 어떤 단위로 처리하느냐에 따라 바이트 스트림과 문자 스트림으로 나뉩니다.

구분 바이트 스트림 문자 스트림
처리 단위 1바이트 문자
최상위 클래스 InputStream, OutputStream Reader, Writer
주 사용 대상 이미지, PDF, 동영상, ZIP, 바이너리 파일 txt, csv, json, xml, html, log 파일
대표 클래스 FileInputStream, FileOutputStream FileReader, FileWriter

3-2. 연결 방식에 따른 분류

스트림은 실제 데이터 대상과 직접 연결할 수 있는지에 따라 노드 스트림과 필터 스트림으로 나뉩니다. 

노드 스트림
파일, 키보드, 콘솔과 직접 연결

필터 스트림
노드 스트림에 연결해서 부가 기능 제공
구분 설명 대표 클래스
Node Stream 데이터 소스나 목적지에 직접 연결됩니다. FileInputStream, FileReader, System.in
Filter Stream 노드 스트림에 붙어서 성능이나 편의 기능을 제공합니다. BufferedReader, DataInputStream, ObjectInputStream

3-3. 브릿지 스트림

브릿지 스트림은 바이트 스트림과 문자 스트림을 연결하는 변환 스트림입니다. 대표 클래스는 InputStreamReader와 OutputStreamWriter입니다. 

System.in
바이트 입력 스트림
↓
InputStreamReader
문자 입력 스트림으로 변환
↓
BufferedReader
줄 단위 입력 가능

BufferedReader br =
        new BufferedReader(new InputStreamReader(System.in));

위 코드는 키보드 입력을 문자열 단위로 편하게 받기 위해 자주 사용하는 조합입니다.

4. System.in과 System.out

4-1. System.in

System.in은 키보드와 연결된 표준 입력 스트림입니다. 타입은 InputStream이므로 바이트 기반 입력 스트림입니다. 

import java.io.IOException;

public class SystemInExample {

    public static void main(String[] args) {
        try {
            System.out.print("문자 하나 입력: ");

            int data = System.in.read();

            System.out.println("입력된 정수 코드: " + data);
            System.out.println("입력된 문자: " + (char) data);

        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

System.in.read()는 사용자가 입력할 때까지 기다립니다. 그리고 입력된 값을 1바이트 단위로 읽어서 int로 반환합니다.

4-2. System.out

System.out은 콘솔 화면과 연결된 표준 출력 스트림입니다. 타입은 PrintStream입니다. 

public class SystemOutExample {

    public static void main(String[] args) {

        System.out.print("줄바꿈 없음");
        System.out.println("줄바꿈 있음");
        System.out.printf("이름: %s, 나이: %d%n", "홍길동", 25);
    }
}
메서드 설명
print() 출력 후 줄바꿈을 하지 않습니다.
println() 출력 후 줄바꿈을 합니다.
printf() 형식 문자열을 사용하여 출력합니다.
write() 바이트 기반으로 출력합니다.
flush() 버퍼에 남아 있는 데이터를 강제로 내보냅니다.

5. FileInputStream과 FileOutputStream

5-1. FileInputStream

FileInputStream은 파일을 읽기 위한 바이트 기반 노드 스트림입니다. 이미지, PDF, ZIP 같은 바이너리 파일을 읽을 때 사용할 수 있습니다. 

import java.io.FileInputStream;
import java.io.IOException;

public class FileInputStreamExample {

    public static void main(String[] args) {
        try (FileInputStream fis = new FileInputStream("sample.txt")) {

            int data;

            while ((data = fis.read()) != -1) {
                System.out.print((char) data);
            }

        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
sample.txt
↓
FileInputStream
↓
read()
↓
Java 프로그램

read()는 파일의 끝에 도달하면 -1을 반환합니다. 그래서 파일을 끝까지 읽을 때는 while ((data = read()) != -1) 패턴을 사용합니다.

5-2. FileOutputStream

FileOutputStream은 파일에 데이터를 쓰기 위한 바이트 기반 노드 스트림입니다. 

import java.io.FileOutputStream;
import java.io.IOException;

public class FileOutputStreamExample {

    public static void main(String[] args) {
        String text = "Hello Java IO";

        try (FileOutputStream fos = new FileOutputStream("result.txt")) {

            fos.write(text.getBytes());

            System.out.println("파일 저장 완료");

        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

new FileOutputStream("result.txt")는 기존 파일 내용이 있으면 지우고 새로 씁니다. 기존 내용 뒤에 추가하려면 생성자의 두 번째 인자로 true를 넣어야 합니다. 

FileOutputStream fos = new FileOutputStream("result.txt", true);

5-3. 바이트 배열을 사용한 파일 복사

파일을 1바이트씩 읽고 쓰면 큰 파일에서는 비효율적입니다. 그래서 보통 byte[] 배열을 버퍼처럼 사용합니다. 

import java.io.FileInputStream;
import java.io.FileOutputStream;
import java.io.IOException;

public class ByteArrayCopyExample {

    public static void main(String[] args) {
        try (
            FileInputStream fis = new FileInputStream("source.pdf");
            FileOutputStream fos = new FileOutputStream("copy.pdf")
        ) {
            byte[] buffer = new byte[1024];
            int length;

            while ((length = fis.read(buffer)) != -1) {
                fos.write(buffer, 0, length);
            }

            System.out.println("파일 복사 완료");

        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

마지막 반복에서는 버퍼가 꽉 차지 않을 수 있습니다. 그래서 반드시 fos.write(buffer, 0, length)처럼 실제로 읽은 길이만큼만 써야 합니다.

6. BufferedInputStream과 BufferedOutputStream

6-1. Buffered 계열을 사용하는 이유

BufferedInputStream과 BufferedOutputStream은 바이트 기반 스트림에 버퍼 기능을 추가하는 필터 스트림입니다. 

버퍼 없음
파일 → 1바이트 → 프로그램
파일 → 1바이트 → 프로그램
파일 → 1바이트 → 프로그램

버퍼 있음
파일 → 여러 바이트를 버퍼에 모음 → 프로그램

버퍼를 사용하면 디스크 접근 횟수를 줄일 수 있기 때문에 파일 복사 성능이 좋아집니다.

6-2. Buffered 스트림으로 이미지 복사하기

import java.io.BufferedInputStream;
import java.io.BufferedOutputStream;
import java.io.FileInputStream;
import java.io.FileOutputStream;
import java.io.IOException;

public class BufferedBinaryCopyExample {

    public static void main(String[] args) {
        try (
            BufferedInputStream bis =
                    new BufferedInputStream(new FileInputStream("source.png"));
            BufferedOutputStream bos =
                    new BufferedOutputStream(new FileOutputStream("copy.png"))
        ) {
            byte[] buffer = new byte[1024];
            int length;

            while ((length = bis.read(buffer)) != -1) {
                bos.write(buffer, 0, length);
            }

            System.out.println("이미지 파일 복사 완료");

        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

이미지, PDF, ZIP처럼 문자로 해석하면 안 되는 파일은 문자 스트림이 아니라 바이트 스트림으로 복사해야 합니다.

7. Reader와 Writer

7-1. 문자 스트림이 필요한 이유

텍스트 파일은 단순한 바이트 덩어리가 아니라 문자 데이터입니다. 특히 한글은 여러 바이트로 표현되므로, 문자 단위로 처리하는 스트림이 필요합니다.

구분 사용 대상 대표 클래스
바이트 스트림 이미지, PDF, ZIP, 동영상 FileInputStream, FileOutputStream
문자 스트림 txt, csv, json, xml, html, log FileReader, FileWriter

7-2. FileReader

import java.io.FileReader;
import java.io.IOException;

public class FileReaderExample {

    public static void main(String[] args) {
        try (FileReader fr = new FileReader("sample.txt")) {

            int data;

            while ((data = fr.read()) != -1) {
                System.out.print((char) data);
            }

        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

FileReader는 문자 단위로 파일을 읽습니다. 텍스트 파일을 처리할 때 FileInputStream보다 자연스럽습니다.

7-3. FileWriter

import java.io.FileWriter;
import java.io.IOException;

public class FileWriterExample {

    public static void main(String[] args) {
        try (FileWriter fw = new FileWriter("text_result.txt")) {

            fw.write("FileWriter로 문자열을 저장합니다.");

            System.out.println("텍스트 파일 저장 완료");

        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

FileWriter도 기본적으로 기존 파일 내용을 지우고 새로 씁니다. 기존 내용 뒤에 추가하려면 new FileWriter("text_result.txt", true)를 사용합니다.

8. BufferedReader와 BufferedWriter

8-1. BufferedReader

BufferedReader는 문자 입력 스트림에 버퍼 기능을 추가하는 필터 스트림입니다. 가장 중요한 메서드는 readLine()입니다. 

import java.io.BufferedReader;
import java.io.FileReader;
import java.io.IOException;

public class BufferedReaderExample {

    public static void main(String[] args) {
        try (
            BufferedReader br =
                    new BufferedReader(new FileReader("sample.txt"))
        ) {
            String line;

            while ((line = br.readLine()) != null) {
                System.out.println(line);
            }

        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

readLine()은 한 줄을 읽지만 줄바꿈 문자는 포함하지 않습니다. 파일 끝에 도달하면 null을 반환합니다.

8-2. BufferedWriter

BufferedWriter는 문자 출력 스트림에 버퍼 기능을 추가하는 필터 스트림입니다. 문자열을 출력할 때는 write(), 줄바꿈에는 newLine()을 사용합니다. 

import java.io.BufferedWriter;
import java.io.FileWriter;
import java.io.IOException;

public class BufferedWriterExample {

    public static void main(String[] args) {
        try (
            BufferedWriter bw =
                    new BufferedWriter(new FileWriter("result.txt"))
        ) {
            bw.write("첫 번째 줄");
            bw.newLine();
            bw.write("두 번째 줄");

            System.out.println("파일 저장 완료");

        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

8-3. 텍스트 파일 복사하기

import java.io.BufferedReader;
import java.io.BufferedWriter;
import java.io.FileReader;
import java.io.FileWriter;
import java.io.IOException;

public class TextFileCopyExample {

    public static void main(String[] args) {
        try (
            BufferedReader br =
                    new BufferedReader(new FileReader("source.txt"));
            BufferedWriter bw =
                    new BufferedWriter(new FileWriter("target.txt"))
        ) {
            String line;

            while ((line = br.readLine()) != null) {
                bw.write(line);
                bw.newLine();
            }

            System.out.println("텍스트 파일 복사 완료");

        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
source.txt
↓
FileReader
↓
BufferedReader
↓
readLine()

write()
↓
BufferedWriter
↓
FileWriter
↓
target.txt

9. InputStreamReader와 OutputStreamWriter

9-1. InputStreamReader

InputStreamReader는 바이트 입력 스트림을 문자 입력 스트림으로 변환하는 브릿지 스트림입니다. 

import java.io.BufferedReader;
import java.io.IOException;
import java.io.InputStreamReader;

public class KeyboardInputExample {

    public static void main(String[] args) {

10. DataInputStream과 DataOutputStream

10-1. Data 계열 스트림의 목적

DataInputStream과 DataOutputStream은 Java의 기본 자료형을 바이트 형태로 저장하고 다시 읽기 위한 필터 스트림입니다.

출력 메서드 입력 메서드 자료형
writeInt() readInt() int
writeDouble() readDouble() double
writeBoolean() readBoolean() boolean
writeUTF() readUTF() String

10-2. 자료형 저장하기

import java.io.DataOutputStream;
import java.io.FileOutputStream;
import java.io.IOException;

public class DataWriteExample {

    public static void main(String[] args) {
        try (
            DataOutputStream dos =
                    new DataOutputStream(new FileOutputStream("student.dat"))
        ) {
            dos.writeInt(1);
            dos.writeUTF("홍길동");
            dos.writeInt(90);
            dos.writeDouble(175.5);
            dos.writeBoolean(true);

            System.out.println("학생 데이터 저장 완료");

        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

10-3. 자료형 읽기

import java.io.DataInputStream;
import java.io.FileInputStream;
import java.io.IOException;

public class DataReadExample {

    public static void main(String[] args) {
        try (
            DataInputStream dis =
                    new DataInputStream(new FileInputStream("student.dat"))
        ) {
            int id = dis.readInt();
            String name = dis.readUTF();
            int score = dis.readInt();
            double height = dis.readDouble();
            boolean active = dis.readBoolean();

            System.out.println("번호: " + id);
            System.out.println("이름: " + name);
            System.out.println("점수: " + score);
            System.out.println("키: " + height);
            System.out.println("활성 여부: " + active);

        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

DataOutputStream으로 쓴 데이터는 DataInputStream으로 읽어야 합니다. 또한 쓴 순서와 읽는 순서가 반드시 같아야 합니다.

11. ObjectInputStream과 ObjectOutputStream

11-1. 객체 직렬화

ObjectOutputStream과 ObjectInputStream은 객체를 저장하고 읽기 위한 스트림입니다. 객체를 파일이나 네트워크로 보내려면 객체를 바이트 형태로 바꾸어야 합니다. 이 과정을 직렬화 Serialization라고 합니다. 

직렬화
객체 → 바이트 데이터

역직렬화
바이트 데이터 → 객체

직렬화할 클래스는 반드시 Serializable 인터페이스를 구현해야 합니다.

11-2. 직렬화 대상 클래스

import java.io.Serializable;

public class Member implements Serializable {

    private static final long serialVersionUID = 1L;

    private String name;
    private int age;

    public Member(String name, int age) {
        this.name = name;
        this.age = age;
    }

    @Override
    public String toString() {
        return "Member{name='" + name + "', age=" + age + "}";
    }
}

Serializable은 메서드가 없는 마커 인터페이스입니다. JVM에게 이 클래스의 객체는 직렬화할 수 있다는 표시를 해주는 역할입니다.

11-3. 객체 저장하기

import java.io.FileOutputStream;
import java.io.IOException;
import java.io.ObjectOutputStream;

public class ObjectWriteExample {

    public static void main(String[] args) {
        Member member = new Member("홍길동", 25);

        try (
            ObjectOutputStream oos =
                    new ObjectOutputStream(new FileOutputStream("member.obj"))
        ) {
            oos.writeObject(member);

            System.out.println("객체 저장 완료");

        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

11-4. 객체 읽기

import java.io.FileInputStream;
import java.io.IOException;
import java.io.ObjectInputStream;

public class ObjectReadExample {

    public static void main(String[] args) {
        try (
            ObjectInputStream ois =
                    new ObjectInputStream(new FileInputStream("member.obj"))
        ) {
            Member member = (Member) ois.readObject();

            System.out.println(member);

        } catch (IOException | ClassNotFoundException e) {
            e.printStackTrace();
        }
    }
}

readObject()는 Object 타입으로 반환되므로 원래 클래스 타입으로 형변환해야 합니다.

12-1. PrintWriter의 특징

PrintWriter는 문자 출력에 편리한 기능을 제공하는 스트림입니다. print(), println(), printf()를 사용할 수 있습니다.

13. File 클래스

13-1. File 클래스의 역할

File 클래스는 파일의 내용을 읽거나 쓰는 클래스가 아닙니다. 파일이나 디렉토리의 이름, 경로, 존재 여부, 크기, 삭제, 목록 조회 같은 파일 시스템 정보를 다루는 클래스입니다.

14. try-with-resources

14-1. 스트림은 반드시 닫아야 한다

IO 스트림은 파일, 네트워크, 콘솔 같은 외부 자원과 연결됩니다. 사용 후 닫지 않으면 자원 누수가 발생할 수 있습니다.

IO 작업이 끝난 뒤에는 close()를 호출해야 합니다. 그러나 직접 닫는 방식은 코드가 복잡하고 실수하기 쉽기 때문에 try-with-resources를 권장합니다.

14-2. try-with-resources 예제

import java.io.BufferedReader;
import java.io.FileReader;
import java.io.IOException;

public class TryWithResourcesExample {

    public static void main(String[] args) {
        try (
            BufferedReader br =
                    new BufferedReader(new FileReader("sample.txt"))
        ) {
            String line;

            while ((line = br.readLine()) != null) {
                System.out.println(line);
            }

        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
try 시작
↓
스트림 열기
↓
파일 읽기
↓
try 종료
↓
자동 close()

15. 실무에서 어떤 스트림을 선택할까?

15-1. 상황별 스트림 선택표

상황 추천 조합 이유
텍스트 파일 읽기 BufferedReader + FileReader 줄 단위로 읽기 편합니다.
텍스트 파일 쓰기 BufferedWriter + FileWriter 문자열 출력과 줄바꿈 처리가 편합니다.
인코딩 지정 텍스트 읽기 BufferedReader + InputStreamReader + FileInputStream UTF-8 같은 문자셋을 명확하게 지정할 수 있습니다.
이미지, PDF, ZIP 복사 BufferedInputStream + BufferedOutputStream 바이너리 데이터를 원본 그대로 처리합니다.
키보드 문자열 입력 BufferedReader + InputStreamReader + System.in 바이트 입력을 문자 입력으로 바꾸고 줄 단위로 읽습니다.
기본 자료형 저장 DataOutputStream int, double, boolean, UTF 문자열을 자료형 단위로 저장합니다.
객체 저장 ObjectOutputStream Serializable 객체를 직렬화합니다.
파일 정보 확인 File 파일명, 경로, 존재 여부, 크기, 디렉토리 목록을 확인합니다.

15-2. Spring Boot 프로젝트와 연결해서 보기

웹 프로젝트에서 파일 업로드 기능을 만들면 사용자가 보낸 파일을 서버에 저장해야 합니다. 이때 내부적으로는 파일의 바이트 데이터를 읽고, 서버의 특정 경로에 쓰는 과정이 필요합니다. 

사용자 이미지 업로드
↓
서버가 파일 바이트 수신
↓
저장 폴더 존재 여부 확인
↓
파일명 생성
↓
서버 디스크에 저장
↓
DB에는 파일 경로 저장

즉, Java IO는 콘솔 예제에서 끝나는 개념이 아니라 실제 웹 서비스의 파일 처리 기능과 직접 연결됩니다.

16. 자주 하는 실수

실수 1. 스트림을 닫지 않음

FileInputStream fis = new FileInputStream("sample.txt");

int data = fis.read();
try (FileInputStream fis = new FileInputStream("sample.txt")) {
    int data = fis.read();
}

IO 스트림은 외부 자원과 연결되므로 작업 후 반드시 닫아야 합니다. 가장 안전한 방법은 try-with-resources를 사용하는 것입니다.

실수 2. 이미지 파일을 문자 스트림으로 복사함

FileReader fr = new FileReader("image.png");
FileWriter fw = new FileWriter("copy.png");
FileInputStream fis = new FileInputStream("image.png");
FileOutputStream fos = new FileOutputStream("copy.png");

이미지, PDF, ZIP 같은 파일은 문자 데이터가 아니므로 바이트 스트림으로 처리해야 합니다.

실수 3. 마지막 버퍼 길이를 무시함

while ((length = fis.read(buffer)) != -1) {
    fos.write(buffer);
}
while ((length = fis.read(buffer)) != -1) {
    fos.write(buffer, 0, length);
}

마지막 반복에서는 버퍼 전체가 채워지지 않을 수 있습니다. 그래서 실제로 읽은 바이트 수인 length만큼만 써야 합니다.

실수 4. readLine()이 줄바꿈까지 읽는다고 생각함

while ((line = br.readLine()) != null) {
    bw.write(line);
}
while ((line = br.readLine()) != null) {
    bw.write(line);
    bw.newLine();
}

readLine()은 줄바꿈 문자를 포함하지 않습니다. 파일 복사 시 줄바꿈을 유지하려면 newLine()을 직접 호출해야 합니다.

실수 5. DataInputStream 읽는 순서를 틀림

// 저장 순서
dos.writeInt(1);
dos.writeUTF("홍길동");

// 잘못된 읽기 순서
String name = dis.readUTF();
int id = dis.readInt();
// 올바른 읽기 순서
int id = dis.readInt();
String name = dis.readUTF();

Data 계열 스트림은 저장한 순서와 읽는 순서가 반드시 같아야 합니다.

실수 6. Serializable을 구현하지 않고 객체를 저장함

public class Member {
    private String name;
    private int age;
}
import java.io.Serializable;

public class Member implements Serializable {
    private static final long serialVersionUID = 1L;

    private String name;
    private int age;
}

객체를 ObjectOutputStream으로 저장하려면 클래스가 Serializable을 구현해야 합니다.

17. 면접 포인트

Q1. Java IO란 무엇인가요?

답변: Java IO는 프로그램이 외부 데이터와 입력 및 출력을 수행하기 위한 기능입니다. 파일, 키보드, 콘솔, 네트워크 등과 데이터를 주고받기 위해 스트림을 사용합니다.
입력 스트림은 외부 데이터를 프로그램으로 읽어오고, 출력 스트림은 프로그램의 데이터를 외부로 내보냅니다.

Q2. Stream이란 무엇인가요?

답변: Stream은 데이터가 이동하는 단방향 통로입니다. 데이터는 스트림을 따라 순서대로 이동하며 FIFO 구조를 가집니다.
Java에서는 입력 스트림과 출력 스트림을 구분해서 사용합니다.

Q3. 바이트 스트림과 문자 스트림의 차이는 무엇인가요?

답변: 바이트 스트림은 데이터를 1바이트 단위로 처리하며 InputStream, OutputStream 계열을 사용합니다. 이미지, PDF, ZIP 같은 바이너리 파일 처리에 적합합니다.
문자 스트림은 데이터를 문자 단위로 처리하며 Reader, Writer 계열을 사용합니다. 텍스트 파일, CSV, 로그 파일 처리에 적합합니다.

Q4. 노드 스트림과 필터 스트림의 차이는 무엇인가요?

답변: 노드 스트림은 파일, 키보드, 콘솔 같은 실제 데이터 소스나 목적지에 직접 연결되는 스트림입니다.
반면 필터 스트림은 노드 스트림에 연결되어 버퍼링, 자료형 처리, 객체 처리 같은 부가 기능을 제공합니다.
예를 들어 FileReader는 노드 스트림이고, BufferedReader는 필터 스트림입니다.

Q5. BufferedReader를 사용하는 이유는 무엇인가요?

답변: BufferedReader는 문자 입력 스트림에 버퍼 기능을 추가해 성능을 높이고,
readLine() 메서드를 제공하여 텍스트 데이터를 줄 단위로 편리하게 읽을 수 있게 해줍니다.

Q6. InputStreamReader는 어떤 역할을 하나요?

답변: InputStreamReader는 바이트 기반 입력 스트림을 문자 기반 입력 스트림으로 변환하는 브릿지 스트림입니다.
대표적으로 System.in을 문자열로 처리하거나, 파일을 특정 인코딩으로 읽을 때 사용합니다.

Q7. File 클래스는 파일 내용을 읽고 쓰는 클래스인가요?

답변: 아닙니다. File 클래스는 파일이나 디렉토리의 이름, 경로, 존재 여부, 크기, 삭제, 목록 조회 같은 파일 시스템 정보를 다루는 클래스입니다.
파일 내용을 읽거나 쓰려면 FileInputStream, FileReader, FileWriter 같은 스트림 클래스를 사용해야 합니다.

Q8. try-with-resources를 사용하는 이유는 무엇인가요?

답변: IO 스트림은 파일이나 네트워크 같은 외부 자원과 연결되므로 사용 후 반드시 닫아야 합니다.
try-with-resources를 사용하면 AutoCloseable을 구현한 자원을 try 블록이 끝날 때 자동으로 닫아주기 때문에 자원 누수를 방지하고 코드도 간결해집니다.

Q9. 직렬화란 무엇인가요?

답변: 직렬화는 메모리에 있는 객체를 파일 저장이나 네트워크 전송이 가능한 바이트 형태로 변환하는 과정입니다.
Java에서는 ObjectOutputStream으로 직렬화하고, ObjectInputStream으로 역직렬화할 수 있습니다. 직렬화 대상 클래스는 Serializable을 구현해야 합니다.

18. 최종 정리

18-1. 이번 단원 핵심 요약

  1. IO는 프로그램이 외부와 데이터를 주고받는 기능입니다.
  2. Stream은 데이터가 한 방향으로 흐르는 통로입니다.
  3. 입력 스트림은 외부 데이터를 프로그램으로 읽어옵니다.
  4. 출력 스트림은 프로그램 데이터를 외부로 내보냅니다.
  5. 바이트 스트림은 1바이트 단위로 데이터를 처리합니다.
  6. 문자 스트림은 문자 단위로 데이터를 처리합니다.
  7. 노드 스트림은 파일, 키보드, 콘솔에 직접 연결됩니다.
  8. 필터 스트림은 노드 스트림에 부가 기능을 제공합니다.
  9. 브릿지 스트림은 바이트 스트림과 문자 스트림을 연결합니다.
  10. Buffered 계열은 성능과 편의 기능을 제공합니다.
  11. Data 계열은 Java 기본 자료형을 저장하고 읽습니다.
  12. Object 계열은 객체를 직렬화하고 역직렬화합니다.
  13. File 클래스는 파일 내용이 아니라 파일과 디렉토리 정보를 다룹니다.
  14. IO 자원은 반드시 닫아야 하며, try-with-resources를 권장합니다.

18-2. 반드시 외워야 할 기본 조합

// 텍스트 파일 읽기
BufferedReader br =
        new BufferedReader(new FileReader("sample.txt"));

// 텍스트 파일 쓰기
BufferedWriter bw =
        new BufferedWriter(new FileWriter("result.txt"));

// 키보드 입력
BufferedReader br =
        new BufferedReader(new InputStreamReader(System.in));

// 바이트 파일 복사
BufferedInputStream bis =
        new BufferedInputStream(new FileInputStream("source.bin"));

BufferedOutputStream bos =
        new BufferedOutputStream(new FileOutputStream("target.bin"));

// 인코딩 지정 텍스트 읽기
BufferedReader br =
        new BufferedReader(
                new InputStreamReader(
                        new FileInputStream("korean.txt"),
                        StandardCharsets.UTF_8
                )
        );

학습 메모

이 페이지는 Java IO 단원입니다. IO는 프로그램이 외부 데이터와 입력 및 출력을 수행하는 기능이며, Java에서는 스트림을 통해 데이터를 읽고 씁니다.

가장 중요한 구분

InputStream / OutputStream은 바이트 기반이고, Reader / Writer는 문자 기반입니다.

노드와 필터

노드 스트림은 파일이나 키보드에 직접 연결됩니다. 필터 스트림은 노드 스트림에 연결되어 버퍼링, 자료형 처리, 객체 처리 같은 기능을 추가합니다.

브릿지 스트림

InputStreamReader는 바이트 입력을 문자 입력으로 바꿉니다. OutputStreamWriter는 문자 출력을 바이트 출력 쪽으로 연결합니다.

실무 선택 기준

텍스트 파일은 Reader / Writer, 이미지와 PDF는 InputStream / OutputStream 계열을 사용합니다.

필수 패턴

텍스트 파일을 줄 단위로 읽을 때는 BufferedReader와 readLine() 패턴을 먼저 익히는 것이 좋습니다.

주의할 점

IO 스트림은 사용 후 반드시 닫아야 합니다. 실무와 학습 코드 모두 try-with-resources 방식을 권장합니다.

면접 핵심

바이트 스트림과 문자 스트림의 차이, 노드 스트림과 필터 스트림의 차이, 직렬화와 Serializable, File 클래스의 역할은 자주 질문됩니다.
