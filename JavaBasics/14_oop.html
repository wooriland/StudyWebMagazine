14. 객체지향 프로그래밍

객체지향 프로그래밍은 현실 세계의 사물, 현상, 사건을 객체로 바라보고
객체의 속성과 행위를 클래스로 모델링하여 프로그램을 구성하는 방식입니다.
이 페이지에서는 객체, 클래스, 객체 모델링, 추상화, 캡슐화, 상속성, 다형성의 기초를 공부합니다.

이번 페이지의 학습 목표

Java는 객체지향 언어입니다.
따라서 Java를 제대로 이해하려면 변수와 제어문을 넘어, 현실의 대상을 프로그램 안에서 어떻게 객체로 표현하는지 이해해야 합니다.
객체지향은 단순히 class 문법을 배우는 것이 아니라, 프로그램을 역할과 책임 중심으로 나누는 사고 방식입니다.

객체 이해

현실의 사물, 개념, 사건을 프로그램의 객체로 바라보는 방법을 익힙니다.

속성과 행위

객체의 속성은 필드로, 행위는 메서드로 표현한다는 점을 이해합니다.

클래스와 객체

클래스는 설계도이고, 객체는 설계도로 만든 실제 대상이라는 차이를 익힙니다.

OOP 4대 특징

추상화, 캡슐화, 상속성, 다형성의 기본 의미를 이해합니다.

1. 객체지향 프로그래밍의 기본 개념

1-1. 객체지향 프로그래밍이란?

객체지향 프로그래밍은 영어로 Object Oriented Programming, 줄여서 OOP라고 합니다.
현실 세계의 사물, 현상, 사건, 개념을 객체로 바라보고, 그 객체의 속성과 행위를 프로그램에 반영하는 방식입니다. 

    현실 세계
    회원, 계좌, 자동차, 주문, 매물, 문의
            ↓
    객체로 분석
            ↓
    속성과 행위 추출
            ↓
    클래스로 설계
            ↓
    new로 객체 생성
            ↓
    프로그램에서 사용

예를 들어 부동산 서비스에서는 User, Property, Inquiry, Report, Recommendation 같은 객체를 만들 수 있습니다.

1-2. 객체란?

객체는 프로그램에서 의미 있는 단위로 다룰 수 있는 대상입니다. 객체는 손으로 만질 수 있는 사물뿐 아니라, 주문, 결제, 로그인, 문의, 신고 같은 개념이나 사건도 될 수 있습니다.

객체 종류 예시 설명
유형 객체 자동차, 사람, 책, 컴퓨터 현실에서 물리적으로 존재하는 대상
개념 객체 계좌, 권한, 등급, 추천 결과 물리적 대상은 아니지만 프로그램에서 관리해야 하는 개념
사건 객체 주문, 결제, 로그인, 문의, 신고 시스템 안에서 발생하는 사건이나 흐름

1-3. 객체는 속성과 행위를 가진다

객체는 속성과 행위를 가집니다. Java에서는 객체의 속성을 필드 또는 멤버변수로 표현하고, 객체의 행위를 메서드로 표현합니다. 

객체 Object

┌────────────────────┐
│ 속성 Field         │
│ - name             │
│ - balance          │
│ - accountNo        │
│                    │
│ 행위 Method        │
│ - deposit()        │
│ - withdraw()       │
│ - printInfo()      │
└────────────────────┘

예를 들어 계좌 객체는 예금주, 계좌번호, 잔고를 속성으로 가지고, 입금, 출금, 계좌 정보 출력 기능을 행위로 가질 수 있습니다.

1-4. 객체 모델링이란?

객체 모델링은 현실의 객체에서 프로그램에 필요한 속성과 행위를 뽑아내는 과정입니다.
현실의 모든 정보를 그대로 가져오는 것이 아니라, 현재 프로그램의 목적에 필요한 정보만 선택하는 것이 중요합니다. 

현실의 회원
이름, 나이, 주소, 취미, 성격, 이메일, 가입일, 권한, 로그인 기록 ...

프로그램에서 필요한 회원 객체
- userId
- loginId
- password
- name
- email
- userType
- login()
- logout()
- changePassword()

1-5. 클래스란?

클래스는 객체를 만들기 위한 설계도입니다. 클래스에는 객체가 가져야 할 속성과 행위를 정의합니다. 하지만 클래스 자체는 아직 실제 객체가 아닙니다. 

class Account {

    String name;
    String accountNo;
    int balance;

}

위 코드는 계좌 객체를 만들기 위한 설계도입니다. 실제 계좌 객체는 new를 사용해 생성합니다. 

Account account = new Account();

1-6. 클래스와 객체의 차이

구분 클래스 객체
의미 설계도 설계도로 만든 실제 대상
예시 Account new Account()
메모리 클래스 선언만으로 일반 객체가 생성되지는 않음 new 후 메모리에 생성됨
역할 속성과 행위를 정의 실제 값을 가지고 동작 
클래스 Account
┌────────────────────┐
│ String name        │
│ String accountNo   │
│ int balance        │
│ deposit()          │
│ withdraw()         │
└────────────────────┘
        ↓ new

    객체 account1
    name = "홍길동"
    balance = 10000

    객체 account2
    name = "김철수"
    balance = 50000

2. 객체지향 프로그래밍 순서

2-1. OOP 개발 흐름

객체지향 프로그래밍은 무작정 클래스를 만드는 것이 아니라, 필요한 객체를 찾고, 객체의 속성과 행위를 정리한 뒤, 클래스를 만들고 객체를 생성해 사용하는 흐름으로 진행됩니다. 

1. 프로그램에 필요한 객체를 뽑아낸다.
                ↓
2. 객체의 속성과 행위를 모델링한다.
                ↓
3. 클래스를 구성한다.
                ↓
4. 객체를 생성하고 사용한다.

2-2. 은행 계좌 프로그램 예시

1단계. 객체 추출
- 회원
- 계좌
- 거래내역

2단계. 객체 모델링
계좌 객체
속성: 예금주, 계좌번호, 잔고
행위: 입금하다, 출금하다, 계좌정보를 출력하다

3단계. 클래스 구성
Account 클래스 작성

4단계. 객체 생성 및 사용
new Account()
deposit()
withdraw()
printAccountInfo()

2-3. 객체지향 사고 방식

절차지향은 “무엇을 어떤 순서로 처리할까?”에 집중합니다.
객체지향은 “누가 어떤 책임을 가질까?”에 집중합니다.

절차 중심 사고 객체 중심 사고
회원 정보를 입력받는다. LoginRequest가 입력값을 담는다.
아이디와 비밀번호를 검사한다. AuthService가 로그인 검증을 담당한다.
로그인 결과를 출력한다. LoginResponse가 결과 데이터를 담는다.

3. OOP의 주요 특징

3-1. OOP 4대 특징

객체지향 프로그래밍의 주요 특징은 추상화, 캡슐화, 상속성, 다형성입니다.
이번 페이지에서는 전체 개념을 먼저 잡고, 이후 단원에서 클래스, 생성자, 상속, 다형성을 더 자세히 공부합니다. 

추상화
→ 필요한 특징만 뽑기

캡슐화
→ 데이터를 보호하기

상속성
→ 기존 클래스를 확장하기

다형성
→ 같은 이름, 다른 동작

3-2. 추상화

추상화는 어떤 대상에서 프로그램 목적에 필요한 특징만 뽑아내고, 중요하지 않은 부분은 생략하는 것입니다. 

현실의 자동차
색상, 제조사, 모델명, 속도, 엔진 온도, 타이어 마모도,
시트 재질, 창문 상태, 보험 정보, 정비 이력 ...

주차 관리 프로그램에서 필요한 자동차
- 차량 번호
- 소유자
- 입차 시간
- 출차 시간
- 요금 계산

프로그램의 목적이 달라지면 추상화 결과도 달라집니다. 자동차 경주 게임과 주차 관리 시스템은 같은 자동차를 다루더라도 필요한 속성이 다릅니다.

3-3. 캡슐화

캡슐화는 객체의 데이터를 외부에서 함부로 바꾸지 못하게 보호하고, 정해진 메서드를 통해 접근하게 하는 방식입니다. 

class SafeAccount {

    private int balance;

    public void deposit(int amount) {
        if (amount > 0) {
            balance += amount;
        }
    }

    public int getBalance() {
        return balance;
    }

}

위 코드에서는 balance를 private으로 숨기고, deposit()과 getBalance()를 통해서만 접근하게 합니다.

3-4. 상속성

상속성은 기존 클래스를 바탕으로 새로운 클래스를 만드는 것입니다. 공통 속성과 기능은 부모 클래스에 두고, 자식 클래스는 필요한 기능을 추가할 수 있습니다. 

class User {
    String name;
    String email;

    void login() {
        System.out.println("로그인");
    }
}

class AdminUser extends User {

    void manageUsers() {
        System.out.println("회원 관리");
    }

}

상속 관계 확인: “관리자는 회원이다”처럼 말할 수 있으면 상속 관계로 볼 수 있습니다. AdminUser is a User

3-5. 다형성

다형성은 같은 이름이나 같은 타입이 상황에 따라 다르게 동작하는 성질입니다. 대표적으로 오버로딩과 오버라이딩이 있습니다.

오버로딩

같은 클래스 안에서 같은 이름의 메서드를 매개변수 형태를 다르게 하여 여러 개 만드는 것입니다. 

class Calculator {

    int add(int a, int b) {
        return a + b;
    }

    int add(int a, int b, int c) {
        return a + b + c;
    }

}

오버라이딩

부모에게 물려받은 메서드를 자식 클래스에서 다시 정의하는 것입니다. 

class Animal {
    void sound() {
        System.out.println("소리를 냅니다.");
    }
}

class Dog extends Animal {
    void sound() {
        System.out.println("멍멍");
    }
}

4. 코드 예제

4-1. 계좌 클래스 만들기

class Account {

    String name;
    String accountNo;
    int balance;

    void deposit(int amount) {
        balance += amount;
    }

    int withdraw(int amount) {
        if (balance < amount) {
            return 0;
        }

        balance -= amount;
        return amount;
    }

    void printAccountInfo() {
        System.out.println("예금주: " + name);
        System.out.println("계좌번호: " + accountNo);
        System.out.println("잔고: " + balance);
    }

}

name, accountNo, balance는 속성이고, deposit(), withdraw(), printAccountInfo()는 행위입니다.

4-2. 객체 생성과 사용

public class AccountMain {

    public static void main(String[] args) {

        Account account = new Account();

        account.name = "홍길동";
        account.accountNo = "123-456";
        account.balance = 10000;

        account.deposit(5000);

        account.printAccountInfo();

    }

}
예금주: 홍길동
계좌번호: 123-456
잔고: 15000

4-3. 같은 클래스로 여러 객체 만들기

public class AccountMain {

    public static void main(String[] args) {

        Account account1 = new Account();
        account1.name = "홍길동";
        account1.accountNo = "111-111";
        account1.balance = 10000;

        Account account2 = new Account();
        account2.name = "김철수";
        account2.accountNo = "222-222";
        account2.balance = 50000;

        account1.deposit(5000);
        account2.withdraw(10000);

        account1.printAccountInfo();
        account2.printAccountInfo();

    }

}

Account 클래스는 하나지만, account1과 account2는 서로 다른 객체입니다.

4-4. 회원 객체 예제

class User {

    Long userId;
    String loginId;
    String password;
    String name;
    String email;
    String userType;

    void login() {
        System.out.println(loginId + "님이 로그인했습니다.");
    }

    void logout() {
        System.out.println(loginId + "님이 로그아웃했습니다.");
    }

    void changePassword(String newPassword) {
        password = newPassword;
        System.out.println("비밀번호가 변경되었습니다.");
    }

}
public class UserMain {

    public static void main(String[] args) {

        User user = new User();

        user.userId = 1L;
        user.loginId = "test01";
        user.password = "1234";
        user.name = "홍길동";
        user.email = "test01@test.com";
        user.userType = "USER";

        user.login();
        user.changePassword("5678");
        user.logout();

    }

}

5-4. 객체지향이 필요한 이유

  • 현실 개념에 맞게 코드를 구조화할 수 있습니다.
  • 관련 데이터와 기능을 하나의 클래스에 묶을 수 있습니다.
  • 코드 재사용성이 좋아집니다.
  • 유지보수가 쉬워집니다.
  • 기능 확장이 쉬워집니다.
  • 팀 프로젝트에서 역할 분리가 쉬워집니다.

6. 자주 하는 실수

실수 1. 클래스와 객체를 같은 것으로 생각함

클래스
→ 객체를 만들기 위한 설계도

객체
→ 클래스를 바탕으로 new를 통해 생성된 실제 대상

Account account = new Account();

여기서 Account는 클래스이고, new Account()는 객체 생성입니다.

실수 2. 객체 생성 없이 멤버를 사용하려 함

Account.name = "홍길동";

name이 인스턴스 멤버라면 객체를 먼저 만들어야 합니다. 

Account account = new Account();
account.name = "홍길동";

실수 3. 속성과 행위를 구분하지 못함

속성
→ 객체가 가지고 있는 데이터
예: 회원 이름, 계좌 잔고, 매물 가격

행위
→ 객체가 할 수 있는 기능
예: 로그인하다, 입금하다, 가격을 변경하다

실수 4. 모든 것을 한 클래스에 몰아넣음

실수 5. 캡슐화 없이 필드를 직접 수정함

account.balance = -10000;

초기 학습 단계에서는 필드를 직접 접근하기도 하지만, 실무에서는 보통 private으로 보호하고 메서드를 통해 접근합니다. 

account.deposit(10000);
account.withdraw(5000);

실수 6. 객체 모델링에서 너무 많은 속성을 넣음

처음부터 현실의 모든 정보를 클래스에 넣으려고 하면 클래스가 복잡해집니다. 현재 프로그램에 필요한 속성과 행위만 먼저 뽑는 것이 좋습니다.

예를 들어 주차 관리 프로그램에서는 자동차의 엔진 구조보다 차량번호, 입차시간, 출차시간, 요금 계산이 더 중요합니다.

7. 면접 포인트

Q1. 객체지향 프로그래밍이란 무엇인가요?

답변: 객체지향 프로그래밍은 현실 세계의 사물, 현상, 사건을 객체로 보고, 객체의 속성과 행위를 클래스로 모델링하여 프로그램을 구성하는 방식입니다.

Q2. 객체란 무엇인가요?

답변: 객체는 프로그램에서 의미 있는 단위로 다루는 대상입니다.
현실의 사물뿐 아니라 주문, 결제, 로그인, 계좌 같은 개념이나 사건도 객체가 될 수 있습니다. 객체는 속성과 행위를 가집니다.

Q3. 클래스와 객체의 차이는 무엇인가요?

답변: 클래스는 객체를 만들기 위한 설계도이고, 객체는 클래스를 기반으로 new를 통해 생성된 실제 인스턴스입니다.
같은 클래스로 여러 객체를 만들 수 있으며, 각 객체는 서로 다른 값을 가질 수 있습니다.

Q4. 객체의 속성과 행위는 Java에서 무엇으로 표현되나요?

답변: 객체의 속성은 Java에서 멤버변수 또는 필드로 표현하고, 객체의 행위는 메서드로 표현합니다.

Q5. 객체 모델링이란 무엇인가요?

답변: 객체 모델링은 현실 세계의 객체에서 프로그램에 필요한 속성과 행위를 추출하여 클래스로 표현할 수 있도록 정리하는 과정입니다.

Q6. OOP의 주요 특징은 무엇인가요?

답변: OOP의 주요 특징은 추상화, 캡슐화, 상속성, 다형성입니다.
추상화는 필요한 특징만 뽑아내는 것이고, 캡슐화는 데이터를 보호하고 메서드를 통해 접근하게 하는 것이며,
상속성은 기존 클래스를 확장해 새로운 클래스를 만드는 것이고, 다형성은 같은 이름이나 타입이 상황에 따라 다르게 동작하는 성질입니다.

Q7. 객체지향을 사용하는 이유는 무엇인가요?

답변: 객체지향을 사용하면 현실 개념에 맞게 코드를 구조화할 수 있고, 관련 데이터와 기능을 하나의 클래스에 묶을 수 있으며, 재사용성, 유지보수성, 확장성이 좋아집니다.
또한 팀 프로젝트에서 역할별로 클래스를 나누기 쉬워집니다.

8. 오늘의 핵심 정리

  1. OOP는 현실 세계의 사물, 현상, 사건을 객체화해서 프로그램에 반영하는 방식이다.
  2. 객체는 속성과 행위를 가진다.
  3. Java에서 속성은 멤버변수 또는 필드로 표현한다.
  4. Java에서 행위는 메서드로 표현한다.
  5. 객체 모델링은 객체의 속성과 행위를 뽑아내는 과정이다.
  6. 클래스는 객체를 만들기 위한 설계도다.
  7. 객체는 클래스를 바탕으로 new를 통해 생성된다.
  8. 같은 클래스로 여러 객체를 만들 수 있다.
  9. 객체지향 프로그래밍 순서는 객체 추출 → 객체 모델링 → 클래스 구성 → 객체 생성 및 사용이다.
  10. OOP의 주요 특징은 추상화, 캡슐화, 상속성, 다형성이다.
  11. 추상화는 필요한 특징만 뽑아내는 것이다.
  12. 캡슐화는 데이터를 보호하고 메서드를 통해 접근하게 하는 것이다.
  13. 상속은 기존 클래스를 바탕으로 새로운 클래스를 만드는 것이다.
  14. 다형성은 같은 이름이나 타입이 상황에 따라 다르게 동작하는 성질이다.
