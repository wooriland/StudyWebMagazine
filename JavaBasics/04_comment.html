04. 주석과 문서화 주석

Java의 주석은 컴파일러가 실행하지 않는 사람을 위한 설명입니다.
이 페이지에서는 //, /* */, /** */의 차이와
javadoc, @param, @return을 활용한 문서화 방법을 공부합니다.

이번 페이지의 학습 목표

주석은 단순히 코드 옆에 적는 메모가 아니라, 코드의 의도와 사용 방법을 설명하는 중요한 도구입니다.
이번 페이지에서는 Java의 세 가지 주석 방식과 문서화 주석을 이용한 API 문서 생성 흐름을 공부합니다.

주석의 역할 이해

주석이 컴퓨터가 아니라 사람을 위한 설명이라는 점을 이해합니다.

주석 종류 구분

한 줄 주석, 여러 줄 주석, 문서화 주석의 차이를 구분합니다.

JavaDoc 이해

javadoc 도구로 HTML 문서를 생성하는 흐름을 이해합니다.

좋은 주석 습관

코드의 동작보다 의도와 이유를 설명하는 주석을 작성합니다.

1. 핵심 개념 설명

1-1. 주석이란?

주석은 Java 코드 안에 작성하지만 컴파일러가 실행 코드로 해석하지 않는 설명문입니다. 즉, 주석은 컴퓨터에게 내리는 명령이 아니라 개발자와 협업자를 위한 기록입니다. 

Java 코드
├─ 컴파일러가 읽는 부분
│   └─ 클래스, 변수, 메서드, 실행문
│
└─ 컴파일러가 무시하는 부분
    └─ 주석

정리: 코드는 컴퓨터를 위한 명령이고, 주석은 사람을 위한 설명입니다.

1-2. 한 줄 주석

한 줄 주석은 //로 작성합니다. // 뒤의 내용은 그 줄 끝까지 주석으로 처리됩니다. 짧은 설명, 간단한 메모, 코드 오른쪽 설명에 적합합니다. 

// 화면에 인사말을 출력한다.
System.out.println("Hello Java");

int age = 20; // 사용자 나이

1-3. 여러 줄 주석

여러 줄 주석은 /* */로 작성합니다. 긴 설명을 남기거나 여러 줄 코드를 잠시 비활성화할 때 사용할 수 있습니다. 

/*
    이 프로그램은 Java 주석을 연습하기 위한 예제입니다.
    한 줄 주석과 여러 줄 주석의 차이를 확인합니다.
*/
public class CommentExample {
}

다만 사용하지 않는 코드를 오래 주석 처리해서 남겨두는 것은 좋지 않습니다.
실무에서는 Git 같은 버전 관리 도구로 이전 코드를 확인할 수 있으므로, 불필요한 코드는 삭제하는 편이 더 깔끔합니다.

1-4. 문서화 주석

문서화 주석은 /** */ 형태로 작성합니다. 일반 여러 줄 주석과 비슷해 보이지만 목적이 다릅니다. 문서화 주석은 javadoc 도구가 읽어서 HTML API 문서를 만들 수 있습니다. 

/**
* 회원 정보를 관리하는 클래스입니다.
*/
public class UserManager {
}
종류 문법 주요 목적
한 줄 주석 // 짧은 설명, 간단한 메모
여러 줄 주석 /* */ 긴 설명, 여러 줄 코드 임시 비활성화
문서화 주석 /** */ javadoc HTML 문서 생성

1-5. JavaDoc이란?

javadoc은 Java 소스코드의 문서화 주석을 읽어서 HTML 문서를 생성하는 도구입니다. 클래스 설명, 메서드 설명, 매개변수 설명, 반환값 설명 등을 문서로 정리할 수 있습니다. 

Calculator.java
↓
/**
* 문서화 주석
*/
↓
javadoc 명령어 실행
↓
docs 폴더 생성
↓
HTML API 문서 생성

mkdir docs
javadoc -author -version -encoding UTF-8 -charset UTF-8 -docencoding UTF-8 -d docs Calculator.java

1-6. 문서화 주석 태그

문서화 주석에는 특별한 태그를 사용할 수 있습니다. 이 태그들은 javadoc 문서에서 클래스, 메서드, 매개변수, 반환값을 체계적으로 보여주는 역할을 합니다.

태그 의미 사용 위치
@author 작성자 클래스 설명
@version 버전 클래스 설명
@param 매개변수 설명 메서드 설명
@return 반환값 설명 반환값이 있는 메서드
@throws 발생 가능한 예외 설명 예외가 발생할 수 있는 메서드
@see 관련 참고 항목 참조 문서 연결
@since 기능이 추가된 버전 클래스 또는 메서드 설명

2. 그림처럼 이해하기

2-1. 주석 종류 비교

// 한 줄 주석
짧은 설명, 간단한 메모

/* 여러 줄 주석 */
긴 설명, 여러 줄 코드 임시 비활성화

/** 문서화 주석 */
javadoc HTML 문서 생성용

2-2. 좋은 주석 판단 기준

코드만 봐도 알 수 있는가?
├─ 예  → 주석 불필요
└─ 아니오 → 의도, 이유, 정책을 주석으로 보완

좋은 주석은 코드의 동작을 그대로 반복하는 것이 아니라, 코드만으로 알기 어려운 배경, 정책, 제약 조건, 의도를 설명합니다.

3. 코드 예제

3-1. 한 줄 주석 예제

public class LineCommentExample {

    public static void main(String[] args) {

        // 화면에 인사말을 출력한다.
        System.out.println("Hello Java");

        int age = 20; // 사용자 나이

    }

}

3-2. 여러 줄 주석 예제

public class BlockCommentExample {

    public static void main(String[] args) {

        /*
            아래 코드는 사용자의 이름과 나이를 출력하는 예제입니다.
            여러 줄 설명이 필요할 때는 블록 주석을 사용할 수 있습니다.
        */
        String name = "홍길동";
        int age = 20;

        System.out.println(name);
        System.out.println(age);

    }

}

3-3. 문서화 주석 예제

/**
* 계산 기능을 제공하는 예제 클래스입니다.
*
* @author ICT5
* @version 1.0
*/
public class Calculator {

    /**
    * 두 정수를 더해서 결과를 반환합니다.
    *
    * @param num1 첫 번째 숫자
    * @param num2 두 번째 숫자
    * @return 두 숫자의 합
    */
    public int getSum(int num1, int num2) {
        return num1 + num2;
    }

}

3-4. 실무형 주석 예제

public class AccountRecoveryPolicy {

    /**
    * 탈퇴 계정 복구 가능 여부를 판단합니다.
    *
    * @param daysAfterDelete 탈퇴 후 경과 일수
    * @return 복구 가능하면 true, 불가능하면 false
    */
    public boolean canRecoverAccount(int daysAfterDelete) {

        // 정책상 탈퇴 후 30일 이내의 계정만 복구 신청이 가능하다.
        return daysAfterDelete <= 30;

    }

}

이 예제의 주석은 단순히 코드 내용을 반복하지 않고, 왜 30이라는 기준을 사용하는지 정책적 이유를 설명합니다.

4. 실무 연결

4-1. 주석은 협업 도구다

팀 프로젝트에서는 다른 사람이 내 코드를 읽어야 합니다.
복잡한 조건문, 계산 공식, 비즈니스 규칙, 외부 API 연동 방식, 예외 처리 이유 등은 주석으로 의도를 남겨두면 유지보수에 큰 도움이 됩니다. 

// 탈퇴 후 30일 이내에는 계정 복구 신청이 가능하다.
if (daysAfterDelete <= 30) {
    allowRecovery();
}

4-2. 공통 메서드에는 JavaDoc이 유용하다

실무에서는 모든 메서드에 문서화 주석을 쓰지는 않더라도, 여러 사람이 사용하는 공통 유틸 메서드나 라이브러리성 코드에는 JavaDoc이 유용합니다. 

/**
* 금액을 원화 표기 형식으로 변환합니다.
*
* @param amount 변환할 금액
* @return 콤마와 원 단위가 포함된 문자열
*/
public String formatWon(int amount) {
    return String.format("%,d원", amount);
}

4-3. 주석보다 좋은 이름이 먼저다

주석을 많이 쓰는 것보다 먼저 변수명과 메서드명을 명확하게 짓는 것이 중요합니다. 좋은 이름은 주석 없이도 코드의 의미를 어느 정도 설명합니다. 

int d = 30; // 탈퇴 후 경과 일수
int daysAfterDelete = 30;
// 탈퇴 계정은 정책상 30일 이내에만 복구 가능하다.
int recoveryLimitDays = 30;

5. 자주 하는 실수

실수 1. 코드 그대로를 반복하는 주석

// age가 20이다.
int age = 20;

더 좋은 주석은 값의 목적을 설명합니다.

// 테스트용 기본 나이값
int age = 20;

실수 2. 주석과 코드가 서로 다른 경우

// 10% 할인 적용
discountRate = 0.2;

코드는 20% 할인인데 주석은 10%라고 되어 있습니다. 이런 주석은 오히려 잘못된 정보를 줍니다. 

// 이벤트 정책에 따라 20% 할인 적용
discountRate = 0.2;

실수 3. 오래된 코드를 주석으로 남겨두기

// oldLogin();
// oldTokenCheck();
// oldSessionCheck();

newLogin();

개발 중 잠깐은 괜찮지만 오래 남겨두면 코드가 지저분해집니다. 실무에서는 Git을 통해 이전 코드를 확인할 수 있으므로 불필요한 코드는 삭제하는 것이 좋습니다.

실수 4. @param 이름을 실제 매개변수와 다르게 쓰기

/**
* @param number 첫 번째 숫자
*/
public int getSum(int num1, int num2) {
    return num1 + num2;
}

올바른 예시는 다음과 같습니다.

/**
* @param num1 첫 번째 숫자
* @param num2 두 번째 숫자
*/
public int getSum(int num1, int num2) {
    return num1 + num2;
}

실수 5. void 메서드에 @return 작성하기

/**
* 인사말을 출력합니다.
*
* @return 출력 결과
*/
public void printHello() {
    System.out.println("Hello");
}

반환값이 없다면 일반적으로 @return을 쓰지 않습니다.

/**
* 인사말을 출력합니다.
*/
public void printHello() {
    System.out.println("Hello");
}

6. 면접 포인트

Q1. Java 주석의 종류를 설명해보세요.

답변: Java 주석에는 한 줄 주석, 여러 줄 주석, 문서화 주석이 있습니다.
한 줄 주석은 //, 여러 줄 주석은 /* */, 문서화 주석은 /** */를 사용합니다. 문서화 주석은 javadoc 도구로 API 문서를 생성할 수 있습니다.

Q2. 주석은 컴파일 결과에 영향을 주나요?

답변: 아닙니다. 주석은 컴파일러가 실행 코드로 해석하지 않기 때문에 프로그램 실행 결과에 영향을 주지 않습니다. 주석은 개발자가 코드의 의도나 설명을 남기기 위한 용도입니다.

Q3. JavaDoc은 무엇인가요?

답변: JavaDoc은 Java 소스코드의 문서화 주석을 읽어서 HTML 형식의 API 문서를 생성하는 도구입니다. 클래스, 메서드, 매개변수, 반환값, 예외 등을 문서화하는 데 사용됩니다.

Q4. @param과 @return의 차이는 무엇인가요?

답변: @param은 메서드의 매개변수를 설명할 때 사용하고, @return은 메서드가 반환하는 값을 설명할 때 사용합니다.

Q5. 좋은 주석이란 무엇인가요?

답변: 좋은 주석은 코드 자체만으로 알기 어려운 의도, 이유, 정책, 제약 조건을 설명하는 주석입니다.
코드가 그대로 보여주는 내용을 반복하는 주석은 좋은 주석이라고 보기 어렵습니다.

7. 오늘의 핵심 정리

  1. 주석은 컴파일러가 실행 코드로 해석하지 않는다.
  2. 주석은 사람을 위한 설명이다.
  3. 한 줄 주석은 //를 사용한다.
  4. 여러 줄 주석은 /* */를 사용한다.
  5. 문서화 주석은 /** */를 사용한다.
  6. 문서화 주석은 javadoc으로 HTML 문서를 만들 수 있다.
  7. @param은 매개변수를 설명한다.
  8. @return은 반환값을 설명한다.
  9. 좋은 주석은 코드의 동작보다 의도와 이유를 설명한다.
  10. 주석보다 먼저 변수명, 메서드명, 코드 구조를 명확하게 만드는 습관이 중요하다.
