06. 자료형

자료형은 변수에 저장할 데이터의 종류를 정하는 Java의 핵심 문법입니다.
이 페이지에서는 기본 자료형과 참조형, 정수형, 실수형, 문자형, 논리형,
그리고 자료형 선택 기준과 실무에서 자주 발생하는 실수를 공부합니다.

이번 페이지의 학습 목표

변수는 값을 저장하는 공간이고, 자료형은 그 공간에 어떤 종류의 값을 저장할 수 있는지 결정합니다.
Java는 자료형을 엄격하게 구분하는 언어이기 때문에 자료형을 정확히 이해해야 변수 선언, 연산, 형변환, 객체지향, DTO, Entity 설계를 제대로 할 수 있습니다.

자료형 개념 이해

자료형이 변수에 저장할 값의 종류를 정한다는 점을 이해합니다.

기본형과 참조형 구분

Primitive Type과 Reference Type의 차이를 구분합니다.

숫자 자료형 이해

정수형과 실수형의 크기, 기본 타입, 접미사 규칙을 익힙니다.

실무 자료형 선택

DTO, Entity, 금액, ID, 상태값에 적절한 자료형을 선택하는 감각을 익힙니다.

1. 핵심 개념 설명

1-1. 자료형이란?

자료형은 변수에 저장할 수 있는 데이터의 종류를 의미합니다. 같은 변수라도 어떤 자료형으로 선언하느냐에 따라 저장할 수 있는 값의 종류와 크기가 달라집니다. 

int age = 20;
double height = 175.5;
char grade = 'A';
boolean isStudent = true;
String name = "홍길동";
자료형 저장할 수 있는 값 예시
int 정수 20
double 실수 175.5
char 문자 하나 'A'
boolean 참 또는 거짓 true, false
String 문자열 "홍길동"

정리: 변수는 값을 담는 공간이고, 자료형은 그 공간에 어떤 값을 담을 수 있는지 정하는 규칙입니다.

1-2. 자료형이 필요한 이유

Java는 자료형을 엄격하게 구분합니다. 자료형이 맞지 않는 값을 변수에 넣으려고 하면 컴파일 오류가 발생합니다. 

int age = "스무 살";

위 코드는 오류입니다. int는 정수를 저장하는 자료형인데, "스무 살"은 문자열이기 때문입니다.

  • 자료형은 값의 종류를 명확하게 합니다.
  • 자료형은 메모리를 얼마나 사용할지 결정합니다.
  • 자료형은 잘못된 값이 들어가는 것을 막아줍니다.
  • 자료형은 연산 결과를 예측할 수 있게 합니다.

1-3. Java 자료형의 큰 분류

Java 자료형은 크게 기본 자료형과 참조형으로 나뉩니다. 기본 자료형은 값 자체를 저장하는 자료형이고, 참조형은 객체의 위치 정보를 저장하는 자료형입니다. 

Java Data Type
├─ Primitive Type
│  ├─ 정수형: byte, short, int, long
│  ├─ 실수형: float, double
│  ├─ 문자형: char
│  └─ 논리형: boolean
│
└─ Reference Type
├─ String
├─ 배열
├─ 클래스
└─ 인터페이스

1-4. 기본 자료형 Primitive Type

기본 자료형은 Java가 기본으로 제공하는 자료형입니다. 객체가 아니라 값 자체를 저장한다고 이해하면 됩니다. Java의 기본 자료형은 총 8개입니다.

분류 자료형 크기 저장 값
정수형 byte 1byte 작은 정수
정수형 short 2byte 정수
정수형 int 4byte 일반 정수
정수형 long 8byte 큰 정수
실수형 float 4byte 실수
실수형 double 8byte 더 정밀한 실수
문자형 char 2byte 문자 하나
논리형 boolean 1bit 개념 true, false

2. 정수형과 실수형

2-1. 정수형 자료형

정수형은 소수점이 없는 숫자를 저장할 때 사용합니다. Java에서 소수점 없는 숫자는 기본적으로 int로 인식됩니다. 

byte smallNumber = 10;
short shortNumber = 100;
int age = 20;
long population = 5000000000L;
자료형 크기 범위
byte 1byte -128 ~ 127
short 2byte -32,768 ~ 32,767
int 4byte 약 -21억 ~ 21억
long 8byte 매우 큰 정수

2-2. 정수의 기본 자료형은 int

Java는 소수점 없는 숫자를 기본적으로 int로 인식합니다. 따라서 아주 큰 숫자를 long에 저장할 때는 숫자 뒤에 L을 붙이는 것이 안전합니다. 

long bigNumber = 10000000000;
long bigNumber = 10000000000L;

핵심: 큰 정수 리터럴에는 L을 붙여서 Java에게 이 숫자가 long이라고 알려줘야 합니다.

2-3. byte 범위 초과

자료형마다 저장할 수 있는 값의 범위가 있습니다. byte는 -128부터 127까지만 저장할 수 있습니다. 

byte b = 127;
byte b = 128;

2-4. int보다 작은 정수형의 연산

byte, short, char는 연산할 때 보통 int로 변환됩니다. 그래서 byte + byte 결과를 다시 byte에 바로 넣으면 오류가 발생할 수 있습니다. 

byte a = 20;
byte b = 10;

byte c = a + b;

더 자연스러운 방식은 결과를 int로 받는 것입니다.

byte a = 20;
byte b = 10;

int c = a + b;

강제로 byte에 넣고 싶다면 형변환이 필요합니다.

byte c = (byte)(a + b);

2-5. 실수형 자료형

실수형은 소수점이 있는 숫자를 저장할 때 사용합니다. Java에서 소수점이 있는 숫자는 기본적으로 double로 인식됩니다. 

float temperature = 36.5F;
double height = 175.5;
자료형 크기 특징
float 4byte 사용 시 숫자 뒤에 F를 붙이는 것이 일반적입니다.
double 8byte Java의 기본 실수형이며 더 정밀합니다.

2-6. 실수의 기본 자료형은 double

Java는 3.14 같은 소수점 숫자를 기본적으로 double로 봅니다. 따라서 float 변수에 소수 값을 넣을 때는 F를 붙여야 합니다. 

float pi = 3.14;
float pi = 3.14F;
double pi = 3.14;

2-7. float과 double 연산

float과 double을 함께 연산하면 결과는 더 큰 타입인 double이 됩니다. 

float f = 10.5F;
double d = 20.5;

double result = f + d;

아래처럼 결과를 float로 바로 받으면 오류가 발생할 수 있습니다.

float result = f + d;

3. 문자형, 논리형, 참조형

3-1. 문자형 char

char는 문자 하나를 저장하는 자료형입니다. 문자 하나는 작은따옴표를 사용합니다. 

char grade = 'A';
char korean = '가';

큰따옴표를 사용하면 문자 하나가 아니라 문자열이 되므로 오류가 발생합니다.

char grade = "A";

3-2. char는 숫자처럼 연산될 수 있다

char는 문자처럼 보이지만 내부적으로는 숫자 코드값으로 저장됩니다. 예를 들어 'A'의 코드값은 65입니다. 

char ch = 'A';

System.out.println(ch);
System.out.println(ch + 1);

출력 결과는 다음과 같습니다.

A
66
char ch = 'A';

겉보기 값: A
내부 코드값: 65

ch + 1 = 66

3-3. 논리형 boolean

boolean은 참과 거짓을 저장하는 자료형입니다. 저장할 수 있는 값은 true와 false 두 가지뿐입니다. 

boolean isLogin = true;
boolean isDeleted = false;

Java에서는 0이나 1을 boolean 값으로 사용할 수 없습니다.

boolean isLogin = 1;

3-4. 참조형 Reference Type

참조형은 값 자체를 직접 저장한다기보다 객체가 있는 위치를 참조하는 자료형입니다. 대표적으로 String, 배열, 클래스, 인터페이스가 있습니다. 

String name = "홍길동";
int[] scores = {90, 80, 70};
Student student = new Student();

핵심: String은 자주 사용해서 기본형처럼 보이지만, 실제로는 클래스 기반의 참조형입니다.

3-5. char와 String의 차이

구분 자료형 따옴표 예시
문자 하나 char 작은따옴표 'A'
문자열 String 큰따옴표 "Alice" 
char grade = 'A';
String name = "Alice";

3-6. 기본 자료형과 참조형 차이

구분 기본 자료형 참조형
저장 내용 값 자체 객체의 위치 정보
예시 int, double, char, boolean String, 배열, 클래스
기본값 숫자형 0, boolean false 등 null
메서드 사용 불가 가능
객체 여부 객체 아님 객체

4. 기본값과 자료형 선택 기준

4-1. 자료형의 기본값

멤버변수는 초기화하지 않아도 자료형에 따라 기본값이 들어갑니다. 하지만 지역변수는 자동 기본값을 사용할 수 없으므로 반드시 직접 초기화해야 합니다.

자료형 기본값
byte, short, int 0
long 0L
float 0.0F
double 0.0
char '\u0000'
boolean false
참조형 null

4-2. 자료형 선택 기준

초급 단계에서는 아래 기준으로 자료형을 선택하면 됩니다.

  • 일반 정수는 int를 사용합니다.
  • 큰 정수나 ID 값은 long 또는 Long을 사용합니다.
  • 일반 실수는 double을 사용합니다.
  • 문자 하나는 char를 사용합니다.
  • 문자열은 String을 사용합니다.
  • 참/거짓 상태는 boolean을 사용합니다.

금액 계산처럼 정확도가 중요한 경우에는 double 대신 BigDecimal을 고려해야 합니다. 초급 단계에서는 “실수 계산에는 오차가 생길 수 있다”는 점만 기억해도 충분합니다.

5. 그림처럼 이해하기

5-1. 기본 자료형 메모리 이해

int age = 20;

메모리
┌──────────────┐
│ age          │
│ 20           │
└──────────────┘

기본 자료형은 값 자체를 저장한다고 이해하면 됩니다.

5-2. 참조형 메모리 이해

String name = "홍길동";

변수 name
┌──────────────┐
│ 문자열 위치  │
└──────┬───────┘
    ↓
문자열 객체
┌──────────────┐
│ "홍길동"      │
└──────────────┘

참조형은 객체의 위치 정보를 저장한다고 이해하면 됩니다.

5-3. 정수와 실수의 기본 타입

100    → int로 인식
100L   → long으로 인식

3.14   → double로 인식
3.14F  → float으로 인식

6. 코드 예제

6-1. 기본 자료형 전체 예제

public class PrimitiveTypeExample {

    public static void main(String[] args) {

        byte byteValue = 127;
        short shortValue = 32000;
        int intValue = 2100000000;
        long longValue = 10000000000L;

        float floatValue = 3.14F;
        double doubleValue = 3.141592;

        char charValue = 'A';
        boolean booleanValue = true;

        System.out.println(byteValue);
        System.out.println(shortValue);
        System.out.println(intValue);
        System.out.println(longValue);
        System.out.println(floatValue);
        System.out.println(doubleValue);
        System.out.println(charValue);
        System.out.println(booleanValue);

    }

}

6-2. String과 char 비교

public class CharStringExample {

    public static void main(String[] args) {

        char grade = 'A';
        String name = "Alice";

        System.out.println(grade);
        System.out.println(name);

    }

}

6-3. long과 float 접미사 예제

public class SuffixExample {

    public static void main(String[] args) {

        long bigNumber = 10000000000L;
        float pi = 3.14F;

        System.out.println(bigNumber);
        System.out.println(pi);

    }

}

6-4. byte 연산 예제

public class ByteOperationExample {

    public static void main(String[] args) {

        byte a = 20;
        byte b = 10;

        int result = a + b;

        System.out.println(result);

    }

}

byte + byte 연산 결과는 int로 받는 것이 안전합니다.

6-5. char 연산 예제

public class CharOperationExample {

    public static void main(String[] args) {

        char ch = 'A';

        System.out.println(ch);
        System.out.println(ch + 1);

    }

}

출력 결과는 다음과 같습니다.

A
66

7. 실무 연결

7-1. DTO와 Entity의 필드는 자료형으로 설계된다

Spring Boot에서 DTO나 Entity를 만들 때 필드마다 자료형을 선택해야 합니다. 자료형 선택은 단순 문법이 아니라 데이터의 성격을 설계하는 일입니다. 

public class UserResponseDto {

    private Long userId;
    private String loginId;
    private String email;
    private boolean active;

}
  • userId는 식별자이므로 Long을 사용합니다.
  • loginId와 email은 문자열이므로 String을 사용합니다.
  • active는 활성 여부이므로 boolean을 사용합니다.

7-2. 매물 정보 자료형 예시

public class PropertyResponseDto {

    private Long propertyId;
    private String address;
    private Long dealAmount;
    private double area;
    private boolean recommended;

}
필드 자료형 선택 이유
propertyId Long 식별자 값이므로 큰 정수와 null 가능성을 고려합니다.
address String 주소는 문자열 데이터입니다.
dealAmount Long 금액은 큰 정수가 될 수 있습니다.
area double 면적은 소수점이 있을 수 있습니다.
recommended boolean 추천 여부는 참/거짓 상태입니다.

7-3. 금액 계산에서는 BigDecimal을 고려한다

실수형은 이진 표현 방식 때문에 정밀한 계산에서 오차가 발생할 수 있습니다.
그래서 돈 계산처럼 정확도가 중요한 경우에는 double보다 BigDecimal을 사용하는 경우가 많습니다. 

BigDecimal price = new BigDecimal("350000000");

학습 포인트: 지금은 BigDecimal을 깊게 다루지 않아도 됩니다. 다만 “돈 계산에 실수형을 무조건 쓰면 위험할 수 있다”는 점은 기억해두는 것이 좋습니다.

8. 자주 하는 실수

실수 1. int 범위를 넘는 숫자 저장

int number = 3000000000;

큰 정수는 long을 사용하고 L을 붙입니다.

long number = 3000000000L;

실수 2. long 값에 L을 붙이지 않음

long number = 10000000000;
long number = 10000000000L;

실수 3. float 값에 F를 붙이지 않음

float pi = 3.14;
float pi = 3.14F;

실수 4. char에 큰따옴표 사용

char grade = "A";
char grade = 'A';

실수 5. String에 작은따옴표 사용

String grade = 'A';
String grade = "A";

실수 6. boolean에 0 또는 1 사용

boolean isLogin = 1;
boolean isLogin = true;

실수 7. 기본형과 참조형의 null 차이를 모름

int age = null;
Integer age = null;

기본형은 null을 저장할 수 없고, 참조형은 null을 가질 수 있습니다. 이 내용은 Wrapper 클래스 단원에서 더 자세히 연결됩니다.

9. 면접 포인트

Q1. Java의 자료형은 어떻게 분류되나요?

답변: Java 자료형은 기본 자료형과 참조형으로 나뉩니다.
기본 자료형에는 byte, short, int, long, float, double, char, boolean이 있고, 참조형에는 String, 배열, 클래스, 인터페이스 등이 있습니다.

Q2. int와 long의 차이는 무엇인가요?

답변: int는 4byte 정수형이고, long은 8byte 정수형입니다. long은 int보다 더 큰 범위의 정수를 저장할 수 있으며, 큰 정수 리터럴에는 L을 붙이는 것이 안전합니다.

Q3. float과 double의 차이는 무엇인가요?

답변: float은 4byte 실수형이고, double은 8byte 실수형입니다.
Java에서 소수점이 있는 숫자는 기본적으로 double로 인식되며, float 값을 사용할 때는 숫자 뒤에 F를 붙여야 합니다.

Q4. char와 String의 차이는 무엇인가요?

답변: char는 문자 하나를 저장하는 기본 자료형이고 작은따옴표를 사용합니다. String은 문자열을 저장하는 참조형이며 큰따옴표를 사용합니다.

Q5. boolean에는 어떤 값을 저장할 수 있나요?

답변: boolean에는 true 또는 false만 저장할 수 있습니다. Java에서는 0이나 1을 boolean 값으로 사용할 수 없습니다.

Q6. 기본 자료형과 참조형의 차이는 무엇인가요?

답변: 기본 자료형은 값 자체를 저장하고, 참조형은 객체의 위치 정보를 저장합니다.
기본 자료형에는 int, double, char, boolean 등이 있고, 참조형에는 String, 배열, 클래스 등이 있습니다.

Q7. String은 기본 자료형인가요?

답변: 아닙니다. String은 기본 자료형이 아니라 클래스 기반의 참조형입니다. 하지만 Java에서 문자열을 자주 사용하기 때문에 기본형처럼 편리하게 사용할 수 있도록 지원됩니다.

10. 오늘의 핵심 정리

  1. 자료형은 변수에 저장할 데이터의 종류를 정한다.
  2. Java 자료형은 기본 자료형과 참조형으로 나뉜다.
  3. 기본 자료형은 byte, short, int, long, float, double, char, boolean이다.
  4. 정수의 기본 자료형은 int이다.
  5. 실수의 기본 자료형은 double이다.
  6. long 값에는 L을 붙이는 것이 안전하다.
  7. float 값에는 F를 붙여야 한다.
  8. char는 문자 하나이며 작은따옴표를 사용한다.
  9. String은 문자열이며 큰따옴표를 사용한다.
  10. String은 기본 자료형이 아니라 참조형이다.
  11. boolean은 true 또는 false만 저장한다.
  12. 기본 자료형은 값 자체, 참조형은 객체의 위치 정보를 저장한다.
