07. 상수

상수는 한 번 값이 정해지면 다시 변경할 수 없는 저장 공간입니다.
이 페이지에서는 final, static final, 리터럴과 상수의 차이,
매직 넘버를 줄이고 유지보수하기 쉬운 코드를 만드는 방법을 공부합니다.

이번 페이지의 학습 목표

변수는 값을 바꿀 수 있는 저장 공간이고, 상수는 값을 바꿀 수 없는 저장 공간입니다.
상수는 단순히 값을 고정하는 문법이 아니라, 코드의 의미를 분명하게 만들고 실수로 값이 변경되는 것을 막아주는 중요한 도구입니다.

상수 개념 이해

상수가 한 번 초기화된 뒤 변경할 수 없는 값이라는 점을 이해합니다.

final 이해

final 키워드가 값 재대입을 막는 역할을 한다는 점을 익힙니다.

static final 이해

클래스 전체에서 공유되는 공통 고정값의 구조를 이해합니다.

실무 상수 활용

권한, 상태, 정책값, 메시지, API 경로를 상수로 관리하는 방법을 익힙니다.

1. 핵심 개념 설명

1-1. 상수란?

상수는 한 번 값이 정해지면 다시 변경할 수 없는 저장 공간입니다. 변수는 실행 중 값을 바꿀 수 있지만, 상수는 초기화된 뒤 값을 다시 대입할 수 없습니다. 

int score = 80;
score = 90;

위 코드는 변수이기 때문에 값 변경이 가능합니다.

final int MAX_SCORE = 100;
MAX_SCORE = 90; // 오류

정리: 변수는 변경 가능한 값이고, 상수는 변경 불가능한 값입니다.

1-2. final 키워드

Java에서는 final 키워드를 사용해서 상수를 만듭니다. final이 붙은 변수는 한 번 초기화된 뒤 다시 값을 대입할 수 없습니다. 

final int MAX_SCORE = 100;
구성 요소 의미
final 값을 다시 변경할 수 없게 만듭니다.
int 저장할 값의 자료형입니다.
MAX_SCORE 상수명입니다.
100 상수에 저장되는 값입니다.

1-3. 상수 선언과 초기화

상수는 보통 선언과 동시에 초기화합니다. 이렇게 작성하면 상수가 어떤 값을 가지는지 한눈에 확인할 수 있고, 초기화되지 않은 상태로 남는 실수를 줄일 수 있습니다. 

final int MAX_USER_COUNT = 100;
final double PI = 3.141592;
final String SERVICE_NAME = "ICT5 Study Magazine";

선언만 먼저 하고 나중에 한 번 초기화하는 것도 가능하지만, 한 번 값이 들어간 뒤에는 다시 변경할 수 없습니다. 

final int PASS_SCORE;

PASS_SCORE = 60;
// PASS_SCORE = 70; // 오류

1-4. 상수명 작성 관례

일반 변수는 보통 camelCase를 사용하지만, 상수는 보통 대문자와 언더바를 사용하는 UPPER_SNAKE_CASE 방식으로 작성합니다. 

final int MAX_SCORE = 100;
final int MIN_AGE = 19;
final String DEFAULT_ROLE = "USER";
final double PI = 3.141592;
max score     → MAX_SCORE
default role  → DEFAULT_ROLE
service name  → SERVICE_NAME

이렇게 작성하면 코드만 봐도 이 값이 변경되지 않는 고정값이라는 것을 쉽게 알아볼 수 있습니다.

1-5. 리터럴과 상수의 차이

리터럴은 코드에 직접 적은 값입니다. 상수는 그 리터럴에 의미 있는 이름을 붙인 변경 불가능한 저장 공간입니다. 

System.out.println(100);
System.out.println("ADMIN");

위 코드의 100, "ADMIN"은 리터럴입니다. 

final int MAX_SCORE = 100;
final String ROLE_ADMIN = "ADMIN";

위 코드의 MAX_SCORE, ROLE_ADMIN은 상수입니다.

구분 리터럴 상수
의미 코드에 직접 적은 값 이름이 붙은 고정값
예시 100, "ADMIN" MAX_SCORE, ROLE_ADMIN
장점 짧고 간단함 의미 부여, 재사용, 유지보수에 유리함

1-6. static final 상수

실무에서는 공통으로 사용하는 고정값에 static final을 많이 사용합니다.
static은 객체를 만들지 않고 클래스명으로 접근할 수 있게 하고, final은 값을 변경할 수 없게 합니다. 

public class UserPolicy {

    public static final int MIN_AGE = 19;
    public static final int MAX_LOGIN_FAIL_COUNT = 5;
    public static final String DEFAULT_ROLE = "USER";

}
System.out.println(UserPolicy.MIN_AGE);
System.out.println(UserPolicy.DEFAULT_ROLE);

핵심: static final은 클래스 전체에서 공유되는 고정값을 표현할 때 사용합니다.

1-7. final 기본형과 final 참조형

기본 자료형에 final을 붙이면 값 자체를 변경할 수 없습니다. 하지만 참조형에 final을 붙이면 참조변수가 다른 객체를 가리키는 것을 막는 의미가 됩니다. 

final int NUMBER = 10;
NUMBER = 20; // 오류
final int[] scores = {90, 80, 70};

scores[0] = 100; // 가능
// scores = new int[]{1, 2, 3}; // 오류

정리: final 기본형은 값 변경 불가, final 참조형은 다른 객체로 참조 변경 불가입니다.

2. 그림처럼 이해하기

100

상수
MAX_SCORE = 100

리터럴은 값 자체이고, 상수는 그 값에 의미 있는 이름을 붙인 것입니다.

2-3. static final 구조

UserPolicy 클래스

UserPolicy.MIN_AGE
UserPolicy.DEFAULT_ROLE

final int[] scores = {90, 80, 70};

    배열 객체

3. 코드 예제

3-1. 기본 상수 예제

public class ConstantBasic {

    public static void main(String[] args) {

        final int MAX_SCORE = 100;

        System.out.println(MAX_SCORE);

        // MAX_SCORE = 90; // 오류

    }

}

3-2. 합격 기준 상수

public class PassScoreExample {

    public static void main(String[] args) {

        final int PASS_SCORE = 60;

        int javaScore = 75;

        if (javaScore >= PASS_SCORE) {
            System.out.println("합격입니다.");
        } else {
            System.out.println("불합격입니다.");
        }

    }

}

60을 직접 조건문에 쓰는 대신 PASS_SCORE라는 상수명을 사용하면 합격 기준이라는 의미가 더 분명해집니다.

3-3. static final 상수 클래스

public class AppConstants {

    public static final String SERVICE_NAME = "ICT5 Study Magazine";

    public static final int PASS_SCORE = 60;
    public static final int ADULT_AGE = 19;

    public static final String ROLE_USER = "USER";
    public static final String ROLE_ADMIN = "ADMIN";

    public static final int MAX_LOGIN_FAIL_COUNT = 5;
    public static final int ACCOUNT_RECOVERY_LIMIT_DAYS = 30;

}

3-4. 상수 사용 예제

public class ConstantUseExample {

    public static void main(String[] args) {

        System.out.println(AppConstants.SERVICE_NAME);

        int score = 80;

        if (score >= AppConstants.PASS_SCORE) {
            System.out.println("합격입니다.");
        }

    }

}

3-5. 권한 상수 예제

public class RoleCheckExample {

    public static final String ROLE_ADMIN = "ADMIN";
    public static final String ROLE_USER = "USER";

    public static void main(String[] args) {

        String role = "ADMIN";

        if (role.equals(ROLE_ADMIN)) {
            System.out.println("관리자 페이지에 접근합니다.");
        }

    }

}

3-6. final 참조형 예제

public class FinalReferenceExample {

    public static void main(String[] args) {

        final int[] scores = {90, 80, 70};

        scores[0] = 100;

        System.out.println(scores[0]);

        // scores = new int[]{1, 2, 3}; // 오류

    }

}

4. 실무 연결

4-1. 정책값은 상수로 관리한다

실무에서는 정책값이 자주 등장합니다. 이런 값은 여러 곳에서 반복될 수 있으므로 상수로 관리하면 유지보수가 쉬워집니다. 

public static final int ACCOUNT_RECOVERY_LIMIT_DAYS = 30;
public static final int PASSWORD_MIN_LENGTH = 8;
public static final int MAX_LOGIN_FAIL_COUNT = 5;

예를 들어 계정 복구 가능 기간이 30일에서 60일로 바뀌면, 상수 값 하나만 수정하면 됩니다.

4-2. 권한과 상태 코드는 상수로 관리한다

문자열을 직접 작성하면 오타 위험이 있습니다. 권한이나 상태 코드는 상수로 관리하면 실수를 줄일 수 있습니다. 

if (role.equals("ADMN")) {
    showAdminPage();
}
public static final String ROLE_ADMIN = "ADMIN";

if (role.equals(ROLE_ADMIN)) {
    showAdminPage();
}

4-3. 메시지도 상수화할 수 있다

public static final String MSG_LOGIN_FAIL = "로그인에 실패했습니다.";
public static final String MSG_ACCESS_DENIED = "접근 권한이 없습니다.";

반복해서 사용하는 메시지나 정책적으로 관리해야 하는 메시지는 상수로 분리할 수 있습니다. 다만 모든 메시지를 무조건 상수로 만들 필요는 없습니다.

4-4. API 경로나 설정값에도 상수가 쓰인다

public static final String API_USERS = "/api/users";
public static final String API_PROPERTIES = "/api/properties";

경로가 여러 곳에서 사용된다면 상수로 분리해두는 편이 관리하기 쉽습니다.

4-5. static final과 enum

단순한 고정값은 static final로 충분하지만, 선택지가 정해진 값의 묶음이라면 enum을 고려할 수 있습니다. 

public static final String STATUS_ACTIVE = "ACTIVE";
public static final String STATUS_DELETED = "DELETED";
public enum UserStatus {
    ACTIVE,
    DELETED,
    PENDING
}

단순 고정값은 static final, 정해진 선택지 묶음은 enum을 고려합니다.

5. 매직 넘버 줄이기

5-1. 매직 넘버란?

코드 안에 의미 없이 직접 박혀 있는 숫자나 문자열을 매직 넘버라고 부릅니다. 매직 넘버는 코드의 의미를 흐리게 만들고 유지보수를 어렵게 합니다. 

if (loginFailCount >= 5) {
    lockAccount();
}

여기서 5가 무엇을 의미하는지 바로 알기 어렵습니다. 

final int MAX_LOGIN_FAIL_COUNT = 5;

if (loginFailCount >= MAX_LOGIN_FAIL_COUNT) {
    lockAccount();
}

상수로 바꾸면 5가 로그인 실패 최대 허용 횟수라는 의미가 분명해집니다.

5-2. 문자열 매직 값 줄이기

if (role.equals("ADMIN")) {
    showAdminPage();
}
final String ROLE_ADMIN = "ADMIN";

if (role.equals(ROLE_ADMIN)) {
    showAdminPage();
}

문자열 상수는 오타를 줄이고 의미를 명확하게 해줍니다.

6. 자주 하는 실수

실수 1. final 상수 값을 다시 변경하려 함

final int MAX_SCORE = 100;
MAX_SCORE = 90;

final 값은 다시 대입할 수 없습니다.

실수 2. 상수명을 일반 변수처럼 작성

final int maxScore = 100;

문법적으로는 가능하지만 상수 관례상 좋지 않습니다. 

final int MAX_SCORE = 100;

실수 3. 공통값을 여러 곳에서 반복 선언

final int PASS_SCORE = 60;

여러 클래스에서 반복된다면 공통 상수로 분리하는 것이 좋습니다. 

public static final int PASS_SCORE = 60;

실수 4. 모든 값을 무조건 상수로 만들기

final int TEMP_COUNT = 1;

잠깐 쓰고 끝나는 값이라면 일반 지역변수로 충분할 수 있습니다. 상수는 의미 있는 고정값, 여러 곳에서 재사용되는 값, 정책이나 설정값에 사용하는 것이 좋습니다.

실수 5. final 참조형을 완전 불변이라고 착각하기

final int[] scores = {90, 80, 70};

scores[0] = 100; // 가능

final은 참조변수가 다른 객체를 가리키는 것을 막을 뿐, 배열 내부 값 변경까지 막지는 않습니다.

실수 6. 문자열 상수를 여러 클래스에 흩어놓기

public static final String ADMIN = "ADMIN";

권한, 상태, 메시지, API 경로 상수가 여러 클래스에 흩어져 있으면 관리하기 어렵습니다. 역할별 상수 클래스로 묶는 것이 좋습니다. 

RoleConstants
StatusConstants
MessageConstants
ApiPathConstants

7. 면접 포인트

Q1. 상수란 무엇인가요?

답변: 상수는 한 번 값이 초기화되면 다시 변경할 수 없는 값입니다. Java에서는 final 키워드를 사용해서 상수를 만들 수 있습니다.

Q2. 변수와 상수의 차이는 무엇인가요?

답변: 변수는 실행 중 값이 변경될 수 있지만, 상수는 한 번 초기화된 뒤 다시 값을 변경할 수 없습니다. Java에서는 상수를 만들 때 final 키워드를 사용합니다.

Q3. final 키워드는 어떤 역할을 하나요?

답변: final은 값을 다시 대입할 수 없게 만드는 키워드입니다. 기본 자료형에서는 값 변경을 막고, 참조형에서는 다른 객체로 참조를 변경하는 것을 막습니다.

Q4. static final은 언제 사용하나요?

답변: static final은 클래스 전체에서 공유되는 고정값을 표현할 때 사용합니다.
정책값, 상태 코드, 권한 코드, 메시지, API 경로처럼 여러 곳에서 재사용되는 값에 주로 사용합니다.

Q5. 리터럴과 상수의 차이는 무엇인가요?

답변: 리터럴은 코드에 직접 적은 값이고, 상수는 리터럴에 의미 있는 이름을 붙인 변경 불가능한 값입니다. 상수를 사용하면 코드의 의미가 명확해지고 유지보수가 쉬워집니다.

Q6. final 참조형은 내부 값도 변경할 수 없나요?

답변: 아닙니다. final이 참조형에 붙으면 참조변수가 다른 객체를 가리키지 못하게 막습니다.
하지만 참조하고 있는 객체 내부 상태는 변경될 수 있습니다. 예를 들어 final 배열은 다른 배열로 바꿀 수는 없지만 배열 요소는 변경할 수 있습니다.

Q7. 매직 넘버를 상수로 바꾸는 이유는 무엇인가요?

답변: 매직 넘버는 코드 안에 의미 없이 직접 적힌 숫자나 문자열입니다. 상수로 바꾸면 값의 의미가 명확해지고, 같은 값이 여러 곳에서 사용될 때 한 곳에서 관리할 수 있어 유지보수가 쉬워집니다.

8. 오늘의 핵심 정리

  1. 상수는 한 번 값이 정해지면 변경할 수 없다.
  2. Java에서는 final 키워드로 상수를 만든다.
  3. 상수명은 보통 대문자와 언더바를 사용한다.
  4. 리터럴은 코드에 직접 적힌 값이다.
  5. 상수는 리터럴에 의미 있는 이름을 붙인 고정값이다.
  6. static final은 클래스 전체에서 공유되는 고정값이다.
  7. 상수는 매직 넘버를 줄이고 코드 의미를 명확하게 만든다.
  8. 상수는 유지보수를 쉽게 만든다.
  9. final 참조형은 참조 변경을 막지만 내부 값 변경까지 항상 막지는 않는다.
  10. 실무에서는 권한, 상태, 정책값, 메시지, API 경로 등에 상수를 사용한다.

final과 static final을 이해하면 코드 안의 매직 넘버를 줄이고, 권한 코드, 상태 코드, 정책값, 메시지, API 경로처럼 반복되는 값을 더 안전하게 관리할 수 있습니다.
상수는 단순 문법이 아니라 유지보수하기 좋은 코드를 만드는 습관과 연결됩니다.
