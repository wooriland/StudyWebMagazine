21. 다형성

다형성은 같은 이름의 메서드가 입력값이나 실제 객체에 따라 다르게 동작하는 객체지향의 핵심 개념입니다.
이 페이지에서는 오버로딩, 오버라이딩, 부모 타입 참조변수, 부모 타입 배열,
instanceof와 형변환까지 단계적으로 공부합니다.

이번 페이지의 학습 목표

다형성은 객체지향에서 “같은 명령을 내려도 객체에 따라 다르게 동작하는 구조”입니다.
Java 기초에서는 주로 오버로딩과 오버라이딩을 통해 다형성을 이해합니다.
특히 오버라이딩은 상속, 부모 타입 참조변수, 배열, 형변환과 연결되며 다음 객체지향 단원의 기반이 됩니다.

오버로딩 이해

같은 클래스 안에서 같은 이름의 메서드를 여러 개 정의하는 방법을 익힙니다.

오버라이딩 이해

상속 관계에서 부모 메서드를 자식 클래스에 맞게 재정의하는 방법을 익힙니다.

부모 타입 참조

부모 타입으로 자식 객체를 참조했을 때 실행되는 메서드 기준을 이해합니다.

실무 확장성

조건문을 줄이고 새 기능을 쉽게 추가하는 다형성 설계 흐름을 익힙니다.

1. 다형성의 기본 개념

1-1. 다형성이란?

다형성은 하나의 이름이나 타입이 여러 형태로 동작할 수 있는 성질입니다. Java 기초에서는 주로 메서드를 기준으로 이해합니다. 

다형성
→ 같은 이름의 메서드가 상황에 따라 다르게 동작하는 것

대표 형태:
1. 오버로딩
2. 오버라이딩

1-2. 다형성의 두 가지 핵심 형태

구분 의미 핵심 기준
오버로딩 같은 클래스 안에서 같은 이름의 메서드를 여러 개 정의 매개변수의 개수, 타입, 순서가 달라야 함
오버라이딩 상속 관계에서 부모 메서드를 자식 클래스가 다시 정의 부모 메서드와 선언부가 같거나 호환되어야 함

2. 오버로딩 Overloading

2-1. 오버로딩이란?

오버로딩은 하나의 클래스 안에서 같은 이름의 메서드를 여러 개 정의하는 것입니다. 같은 기능 계열을 같은 이름으로 묶어 사용할 수 있게 해줍니다. 

class Calculator {

    public int add(int a, int b) {
        return a + b;
    }

    public double add(double a, double b) {
        return a + b;
    }

    public int add(int a, int b, int c) {
        return a + b + c;
    }

}
add(int, int)
add(double, double)
add(int, int, int)

같은 이름 add
하지만 매개변수 목록이 다르다.

2-2. 오버로딩이 필요한 이유

오버로딩을 사용하면 같은 의미의 기능을 같은 이름으로 제공할 수 있습니다. 메서드 이름을 여러 개 외우지 않아도 되고, 사용하는 입장에서 코드가 자연스러워집니다. 

calculator.add(10, 20);
calculator.add(10.5, 20.3);
calculator.add(10, 20, 30);

핵심: 모두 더하기 기능이므로 addInt(), addDouble()처럼 이름을 나누지 않고 add() 하나의 이름으로 묶을 수 있습니다.

2-3. 오버로딩 성립 조건

  1. 메서드 이름이 같아야 합니다.
  2. 매개변수의 개수가 다르거나,
  3. 매개변수의 타입이 다르거나,
  4. 매개변수 타입의 순서가 달라야 합니다.
  5. 반환 타입만 다른 것은 오버로딩이 아닙니다.

2-4. 매개변수 개수가 다른 오버로딩

class Printer {

    public void print(String text) {
        System.out.println(text);
    }

    public void print(String text, int count) {
        for (int i = 0; i < count; i++) {
            System.out.println(text);
        }
    }

}

2-5. 매개변수 타입이 다른 오버로딩

class Printer {

    public void print(String text) {
        System.out.println(text);
    }

    public void print(int number) {
        System.out.println(number);
    }

    public void print(double number) {
        System.out.println(number);
    }

}

2-6. 매개변수 순서가 다른 오버로딩

class UserService {

    public void createUser(String loginId, int age) {
        System.out.println("아이디: " + loginId + ", 나이: " + age);
    }

    public void createUser(int age, String loginId) {
        System.out.println("나이: " + age + ", 아이디: " + loginId);
    }

}
createUser(String, int)
createUser(int, String)

매개변수 타입의 순서가 다르므로 오버로딩 가능

2-7. 반환 타입만 다른 것은 오버로딩이 아니다

class Calculator {

    public int add(int a, int b) {
        return a + b;
    }

    public double add(int a, int b) {
        return a + b;
    }

}

위 코드는 오버로딩이 아닙니다. 메서드 이름과 매개변수 목록이 완전히 같기 때문입니다. 

add(int, int)
add(int, int)

반환 타입만 다름
→ Java가 호출할 메서드를 구분할 수 없음
→ 오버로딩 성립 안 함

3. 오버라이딩 Overriding

3-1. 오버라이딩이란?

오버라이딩은 상속 관계에서 부모로부터 상속받은 메서드를 자식 클래스가 다시 정의하는 것입니다. 부모의 공통 동작을 자식에게 맞게 바꿀 수 있습니다. 

class Animal {

    public void sound() {
        System.out.println("동물이 소리를 냅니다.");
    }

}

class Dog extends Animal {

    @Override
    public void sound() {
        System.out.println("멍멍");
    }

}

3-2. 오버라이딩이 필요한 이유

부모 클래스는 공통적인 동작을 정의하고, 자식 클래스는 자기 역할에 맞게 동작을 바꿀 수 있습니다. 

class Dog extends Animal {

    @Override
    public void sound() {
        System.out.println("멍멍");
    }

}

class Cat extends Animal {

    @Override
    public void sound() {
        System.out.println("야옹");
    }

}
Animal animal1 = new Dog();
Animal animal2 = new Cat();

animal1.sound();
animal2.sound();
멍멍
야옹

3-3. 오버라이딩 성립 조건

  1. 상속 관계여야 합니다.
  2. 메서드 이름이 같아야 합니다.
  3. 매개변수 개수가 같아야 합니다.
  4. 매개변수 타입이 같아야 합니다.
  5. 반환 타입이 같거나 호환되어야 합니다.
  6. 접근 범위를 부모보다 좁게 만들 수 없습니다.
  7. 부모 메서드의 예외보다 더 넓은 예외를 던질 수 없습니다.

초급 정리: 부모 메서드의 선언부를 거의 그대로 맞추고, 중괄호 안의 실행 내용만 자식 클래스에 맞게 바꾸는 것입니다.

3-4. @Override 어노테이션

오버라이딩할 때는 @Override를 붙이는 것이 좋습니다. 이 표시는 “부모 메서드를 재정의하겠다”는 의도를 컴파일러에게 알려줍니다. 

class AdminUser extends User {

    @Override
    public void printRole() {
        System.out.println("관리자");
    }

}

@Override를 붙이면 메서드 이름이나 매개변수를 잘못 썼을 때 컴파일러가 오류로 알려주기 때문에 실수를 줄일 수 있습니다.

3-5. 접근 지정자와 오버라이딩

오버라이딩할 때 자식 메서드는 부모 메서드보다 접근 범위를 좁게 만들 수 없습니다. 

class Parent {

    public void print() {
    }

}

class Child extends Parent {

    private void print() {
    }

}
class Child extends Parent {

    @Override
    public void print() {
    }

}
부모 protected → 자식 public 가능
부모 public    → 자식 public만 가능

부모보다 접근 범위를 좁게 만들 수 없음

3-6. private과 default 메서드 주의

private 메서드
→ 자식에서 직접 접근 불가
→ 오버라이딩 대상이 아님

default 메서드
→ 같은 패키지면 오버라이딩 가능
→ 다른 패키지면 접근 불가라 오버라이딩 아님

public/protected 메서드
→ 오버라이딩에 주로 사용

4. 부모 타입 참조변수와 다형성

4-1. 부모 타입으로 자식 객체 참조

상속 관계에서는 부모 타입 참조변수로 자식 객체를 참조할 수 있습니다. 

Animal animal = new Dog();

animal.sound();
참조변수 타입:
Animal

실제 객체 타입:
Dog

컴파일 시점:
Animal에 sound()가 있는지 확인

실행 시점:
Dog의 sound() 실행

4-2. 오버라이딩 메서드는 실제 객체 기준으로 실행된다

4-3. 부모 타입 배열과 다형성

부모 타입 배열에는 여러 자식 객체를 담을 수 있습니다. 같은 메서드를 호출해도 각 객체가 자기 방식으로 동작합니다. 

Animal[] animals = new Animal[3];

animals[0] = new Dog();
animals[1] = new Cat();
animals[2] = new Cow();

for (int i = 0; i < animals.length; i++) {
    animals[i].sound();
}
멍멍

반복문 안에서 Dog, Cat, Cow를 각각 조건문으로 구분하지 않아도, 실제 객체가 자기에게 맞는 sound()를 실행합니다.

4-4. 자식 고유 메서드와 형변환

부모 타입 참조변수는 부모 기준의 멤버만 접근할 수 있습니다. 자식 고유 메서드에 접근하려면 형변환이 필요합니다. 

class Dog extends Animal {

    @Override
    public void sound() {
        System.out.println("멍멍");
    }

    public void bark() {
        System.out.println("강아지가 짖습니다.");
    }

}
Animal animal = new Dog();

animal.sound(); // 가능
animal.bark();  // 오류
if (animal instanceof Dog) {
    Dog dog = (Dog) animal;
    dog.bark();
}

5. 오버로딩과 오버라이딩 비교

5-1. 비교 표

구분 오버로딩 오버라이딩
적용 위치 같은 클래스 안 상속 관계
메서드 이름 같아야 함 같아야 함
매개변수 달라야 함 같아야 함
반환 타입 구분 기준 아님 같거나 호환되어야 함
목적 같은 기능 이름을 다양한 입력에 대응 부모 기능을 자식에 맞게 재정의
결정 시점 컴파일 시점 실행 시점
예시 add(int, int), add(double, double) Animal.sound() → Dog.sound()

8. 자주 하는 실수

실수 1. 반환 타입만 다르게 해서 오버로딩하려 함

public int add(int a, int b) {
    return a + b;
}

public double add(int a, int b) {
    return a + b;
}

매개변수 목록이 같기 때문에 오버로딩이 아닙니다.

실수 2. 오버라이딩하려고 했는데 매개변수가 달라짐

class Parent {
                public void print(int num) {
                }
            }

            class Child extends Parent {
                public void print(String num) {
                }
            }

이 코드는 오버라이딩이 아니라 자식 클래스에 새로운 메서드를 추가한 오버로딩에 가깝습니다.

실수 3. @Override를 붙이지 않음

class Child extends Parent {
    public void pritn() {
    }
}
class Child extends Parent {

    @Override
    public void print() {
    }

}

실수 4. 부모 public 메서드를 자식에서 private으로 줄임

class Parent {
    public void print() {
    }
}

class Child extends Parent {
    private void print() {
    }
}

오버라이딩할 때 접근 범위를 부모보다 좁게 만들 수 없습니다.

실수 5. 부모 타입 참조변수로 자식 고유 메서드 호출

Animal animal = new Dog();

animal.bark();
if (animal instanceof Dog) {
    ((Dog) animal).bark();
}

실수 6. instanceof로 모든 분기를 처리하려 함

if (animal instanceof Dog) {
    System.out.println("멍멍");
} else if (animal instanceof Cat) {
    System.out.println("야옹");
}
animal.sound();

각 자식 클래스가 sound()를 오버라이딩하게 만드는 것이 더 객체지향적인 방향입니다.

9. 면접 포인트

Q1. 다형성이란 무엇인가요?

답변: 다형성은 하나의 이름이나 타입이 여러 형태로 동작할 수 있는 객체지향의 특징입니다. Java에서는 주로 오버로딩과 오버라이딩을 통해 다형성을 구현합니다.

Q2. 오버로딩이란 무엇인가요?

답변: 오버로딩은 하나의 클래스 안에서 같은 이름의 메서드를 여러 개 정의하는 것입니다. 메서드 이름은 같아야 하고, 매개변수의 개수, 타입, 순서 중 하나 이상이 달라야 합니다.

Q3. 오버로딩에서 반환 타입만 다르면 성립하나요?

답변: 성립하지 않습니다. 오버로딩은 메서드 이름과 매개변수 목록으로 구분되며, 반환 타입만 다른 것은 오버로딩으로 인정되지 않습니다.

Q4. 오버라이딩이란 무엇인가요?

답변: 오버라이딩은 상속 관계에서 부모 클래스의 메서드를 자식 클래스가 다시 정의하는 것입니다. 메서드 이름, 매개변수, 반환 타입이 부모 메서드와 같거나 호환되어야 합니다.

Q5. 오버라이딩할 때 접근 지정자를 더 좁게 만들 수 있나요?

답변: 불가능합니다. 부모 메서드보다 접근 범위를 좁게 만들 수 없습니다. 예를 들어 부모가 public이면 자식도 public이어야 합니다.

Q6. 부모 타입 참조변수로 자식 객체를 참조했을 때 오버라이딩 메서드는 무엇이 실행되나요?

답변: 참조변수 타입이 부모여도 실제 객체가 자식 객체라면, 자식 클래스에서 오버라이딩한 메서드가 실행됩니다.

Q7. 오버로딩과 오버라이딩의 차이는 무엇인가요?

답변: 오버로딩은 같은 클래스 안에서 같은 이름의 메서드를 매개변수를 다르게 하여 여러 개 정의하는 것입니다.
오버라이딩은 상속 관계에서 부모 메서드를 자식 클래스가 다시 정의하는 것입니다.
오버로딩은 컴파일 시점에 결정되고, 오버라이딩은 실행 시점의 실제 객체를 기준으로 결정됩니다.

Q8. @Override를 사용하는 이유는 무엇인가요?

답변: @Override는 해당 메서드가 부모 메서드를 오버라이딩한다는 의도를 명확히 보여줍니다.
또한 메서드 이름이나 매개변수를 잘못 작성했을 때 컴파일러가 오류로 알려주기 때문에 실수를 줄일 수 있습니다.

Q9. 다형성을 사용하면 어떤 장점이 있나요?

답변: 다형성을 사용하면 조건문을 줄이고, 객체별 동작을 각 클래스에 분리할 수 있습니다.
또한 새로운 자식 클래스가 추가되더라도 기존 코드를 크게 수정하지 않고 확장할 수 있어 유지보수성이 좋아집니다.

10. 오늘의 핵심 정리

  1. 다형성은 같은 이름의 메서드가 상황에 따라 여러 형태로 동작하는 것이다.
  2. Java에서 다형성은 주로 오버로딩과 오버라이딩으로 이해한다.
  3. 오버로딩은 같은 클래스 안에서 같은 이름의 메서드를 여러 개 정의하는 것이다.
  4. 오버로딩은 매개변수의 개수, 타입, 순서가 달라야 한다.
  5. 오버로딩은 반환 타입만 다르게 해서는 성립하지 않는다.
  6. 오버라이딩은 상속 관계에서 부모 메서드를 자식이 다시 정의하는 것이다.
  7. 오버라이딩은 메서드명, 매개변수, 반환 타입이 부모 메서드와 맞아야 한다.
  8. 오버라이딩할 때 접근 범위를 부모보다 좁게 만들 수 없다.
  9. 부모 타입 참조변수로 자식 객체를 참조할 수 있다.
  10. 오버라이딩된 메서드는 참조변수 타입이 아니라 실제 객체 기준으로 실행된다.
  11. 부모 타입 배열에 여러 자식 객체를 담고 같은 메서드를 호출하면 각 객체가 자기 방식으로 동작한다.
  12. 자식 고유 메서드에 접근하려면 형변환이 필요하다.
  13. 다형성을 잘 활용하면 조건문을 줄이고 확장성 있는 구조를 만들 수 있다.

학습 메모

이 페이지는 Java 다형성 단원입니다.
다형성은 같은 이름의 메서드가 입력값이나 실제 객체에 따라 다르게 동작하는 구조입니다.
오버로딩은 같은 클래스 안에서 매개변수를 다르게 하여 같은 이름의 메서드를 여러 개 정의하는 것이고,
오버라이딩은 상속 관계에서 부모 메서드를 자식 클래스가 다시 정의하는 것입니다.
특히 부모 타입 참조변수로 자식 객체를 참조했을 때 오버라이딩된 메서드는 실제 객체 기준으로 실행된다는 점을 꼭 기억해야 합니다.
