30. 스레드

스레드는 하나의 프로그램 안에서 동시에 실행되는 작은 작업 흐름입니다.
Thread, Runnable, start(), run(), sleep(), join(), synchronized까지 공부합니다.

이번 페이지의 학습 목표

스레드는 하나의 프로세스 안에서 실제 작업을 수행하는 실행 흐름입니다.
Java에서는 Thread 클래스와 Runnable 인터페이스로 스레드를 만들 수 있으며, 멀티 스레드 환경에서는 실행 순서와 공유 자원 문제를 반드시 고려해야 합니다.

스레드 개념

프로세스와 스레드, 단일 스레드와 멀티 스레드를 구분합니다.

생성 방법

Thread 상속과 Runnable 구현 방식을 익힙니다.

start와 run

새 스레드를 시작하는 방법과 일반 메서드 호출의 차이를 이해합니다.

동기화

공유 자원을 안전하게 다루기 위한 synchronized를 이해합니다.

1. 스레드 기본 개념

1-1. 프로세스와 스레드

프로세스는 실행 중인 프로그램이고, 스레드는 그 프로세스 안에서 실제 작업을 수행하는 실행 흐름입니다. 

프로세스
→ 실행 중인 프로그램

스레드
→ 프로세스 안에서 실제 작업을 수행하는 실행 흐름

예:
음악 앱 프로세스
├─ 음악 재생 스레드
├─ 가사 표시 스레드
├─ 다운로드 스레드
└─ 사용자 입력 처리 스레드

1-2. 단일 스레드

단일 스레드는 하나의 작업 흐름만 있는 프로그램입니다. Java 프로그램은 기본적으로 main() 메서드에서 시작하며, 이 기본 실행 흐름을 main 스레드라고 합니다. 

public class MainThreadExample {

    public static void main(String[] args) {

        System.out.println("작업 1");
        System.out.println("작업 2");
        System.out.println("작업 3");

    }

}
main 스레드

작업 1 → 작업 2 → 작업 3 → 종료

1-3. 멀티 스레드

멀티 스레드는 하나의 프로그램 안에서 여러 작업 흐름이 동시에 실행되는 구조입니다. 

main 스레드      화면 처리 ───── 사용자 입력 ───── 종료
worker 스레드    파일 다운로드 ─ 데이터 처리 ───── 종료

특징:
한 프로그램 안에서 여러 작업이 동시에 진행되는 것처럼 보인다.

예를 들어 파일을 다운로드하면서 화면을 조작하거나, 음악을 재생하면서 가사를 표시하는 상황에서 멀티 스레드가 사용됩니다.

2. Java에서 스레드를 만드는 방법

2-1. 두 가지 방법

방법 작성 방식 실행 방식
Thread 클래스 상속 extends Thread thread.start()
Runnable 인터페이스 구현 implements Runnable new Thread(task).start()

2-2. Thread 클래스를 상속받는 방법

class MyThread extends Thread {

    @Override
    public void run() {
        System.out.println("스레드 실행");
    }

}
MyThread thread = new MyThread();
thread.start();

Thread를 상속받으면 run() 메서드에 스레드가 수행할 작업을 작성합니다. 실행할 때는 run()이 아니라 반드시 start()를 호출합니다.

2-3. Runnable 인터페이스를 구현하는 방법

class MyRunnable implements Runnable {

    @Override
    public void run() {
        System.out.println("스레드 실행");
    }

}
MyRunnable task = new MyRunnable();

Thread thread = new Thread(task, "작업 스레드");

thread.start();

Runnable 방식은 작업 내용과 실제 스레드 객체를 분리할 수 있습니다. Java는 클래스 다중 상속을 허용하지 않기 때문에 실무에서는 더 유연한 방식입니다.

2-4. Thread 상속과 Runnable 구현 비교

구분 Thread 상속 Runnable 구현
작성 방식 extends Thread implements Runnable
실행 방법 직접 start() 호출 Thread에 넣고 start() 호출
다른 클래스 상속 불가능 가능
역할 분리 스레드와 작업이 한 클래스에 있음 작업과 실행 스레드 분리
실무 활용 단순 예제에 적합 더 자주 사용

3. run()과 start()의 차이

3-1. run() 직접 호출

MyThread thread = new MyThread();
thread.run();

run()을 직접 호출하면 새 스레드가 생성되지 않습니다. 그냥 현재 스레드에서 일반 메서드를 호출한 것과 같습니다. 

thread.run()

main 스레드
작업 1 → run() 내용 실행 → 작업 2 → 종료

새 스레드 생성 안 됨

3-2. start() 호출

MyThread thread = new MyThread();
thread.start();

start()를 호출해야 새로운 스레드가 시작됩니다. JVM이 내부적으로 해당 스레드의 run()을 호출합니다. 

thread.start()

main 스레드
작업 1 ───── 작업 2 ───── 종료

새 스레드
        run() 내용 실행 ───── 종료

3-3. 실행 순서는 고정되지 않는다

멀티 스레드에서는 여러 스레드가 동시에 실행되므로, 출력 순서가 매번 같다고 보장할 수 없습니다. 

실행할 때마다 결과가 달라질 수 있음

main 스레드:   A ───── B ───── C
worker 스레드:   1 ───── 2 ───── 3

출력 예:
A 1 B 2 C 3
또는
1 A 2 B 3 C

4. Thread 주요 메서드

4-1. 주요 메서드 표

메서드 의미
start() 새 스레드 시작, JVM이 run() 호출
run() 스레드가 수행할 작업 내용
currentThread() 현재 실행 중인 스레드 반환
setName(), getName() 스레드 이름 설정과 조회
sleep() 현재 실행 중인 스레드를 일정 시간 멈춤
yield() 현재 스레드가 CPU 실행 기회를 양보
join() 대상 스레드가 종료될 때까지 기다림
setPriority(), getPriority() 스레드 우선순위 설정과 조회
wait(), notify() 스레드 대기와 깨우기

4-4. sleep()

sleep()은 현재 실행 중인 스레드를 지정한 시간 동안 멈추게 합니다. InterruptedException 예외 처리가 필요합니다. 

try {
    Thread.sleep(1000);
} catch (InterruptedException e) {
    System.out.println("스레드가 중단되었습니다.");
}
Thread.sleep(1000)
→ 현재 실행 중인 스레드를 1초 동안 멈춤

4-5. yield()

yield()는 현재 실행 중인 스레드가 CPU 실행 기회를 양보하도록 요청하는 메서드입니다. 단, 반드시 양보된다고 보장되지는 않습니다. 

Thread.yield();

yield()는 “잠깐 양보해볼게”에 가깝습니다. 실제 실행 순서는 JVM과 운영체제 스케줄러에 따라 달라질 수 있습니다.

4-6. join()

join()은 대상 스레드가 끝날 때까지 현재 스레드가 기다리게 합니다. 

class WorkerThread extends Thread {

    @Override
    public void run() {

        for (int i = 1; i <= 3; i++) {
            System.out.println("작업 중: " + i);
        }

    }

}

public class JoinExample {

    public static void main(String[] args) {

        WorkerThread worker = new WorkerThread();

        worker.start();

        try {
            worker.join();
        } catch (InterruptedException e) {
            System.out.println("대기 중 중단됨");
        }

        System.out.println("작업 완료 후 main 종료");

    }

}
worker.start()
    ↓
main 스레드가 worker.join() 호출
    ↓
worker가 끝날 때까지 main 대기
    ↓
worker 종료 후 main 계속 실행

5. 스레드 우선순위와 데몬 스레드

5-1. 스레드 우선순위

Thread thread = new Thread(task);

thread.setPriority(Thread.MAX_PRIORITY);

System.out.println(thread.getPriority());
상수 의미
Thread.MAX_PRIORITY 최대 우선권
Thread.NORM_PRIORITY 보통 우선권
Thread.MIN_PRIORITY 최소 우선권

우선순위가 높다고 해서 반드시 먼저 실행되는 것은 아닙니다. 운영체제와 JVM 실행 환경에 따라 달라질 수 있습니다.

5-2. 데몬 스레드

데몬 스레드는 일반 스레드를 보조하는 백그라운드 스레드입니다. 일반 스레드가 모두 끝나면 데몬 스레드는 함께 종료될 수 있습니다. 

Thread daemon = new Thread(task);
daemon.setDaemon(true);
daemon.start();
데몬 스레드 예:
- 자동 저장
- 로그 기록
- 백그라운드 감시
- 보조 작업

주의:
setDaemon(true)는 start() 전에 호출해야 한다.

6. 공유 자원 문제와 동기화

6-1. 공유 자원 문제

멀티 스레드에서 가장 중요한 문제는 여러 스레드가 같은 데이터를 동시에 수정하는 상황입니다. 

계좌 잔액: 10000

스레드 A: 잔액 10000 확인
스레드 B: 잔액 10000 확인

스레드 A: 7000 출금
스레드 B: 7000 출금

결과:
데이터가 꼬일 수 있음

6-2. synchronized란?

synchronized는 공유 자원에 lock을 걸어 한 번에 하나의 스레드만 접근하게 하는 동기화 방법입니다. 

synchronized
→ 하나의 스레드가 작업 중이면 다른 스레드는 기다림
→ 공유 자원 보호
→ 데이터 꼬임 방지

6-3. synchronized 블록

class Counter {

    private int count = 0;

    public void increase() {

        synchronized (this) {
            count++;
            System.out.println(Thread.currentThread().getName() + ": " + count);
        }

    }

}

필요한 코드 일부만 잠글 수 있어 메서드 전체를 잠그는 것보다 세밀하게 제어할 수 있습니다.

6-4. synchronized 메서드

class Counter {

    private int count = 0;

    public synchronized void increase() {
        count++;
        System.out.println(Thread.currentThread().getName() + ": " + count);
    }

}

메서드 전체를 동기화하는 방식입니다. 작성은 간단하지만 불필요하게 넓은 범위를 잠글 수 있습니다.

6-5. synchronized 블록과 메서드 비교

구분 synchronized 블록 synchronized 메서드
잠금 범위 필요한 코드 일부 메서드 전체
장점 성능상 유리할 수 있음 작성이 간단함
단점 lock 대상을 신중히 선택해야 함 불필요하게 넓게 잠글 수 있음
예 synchronized(this) { } public synchronized void method()

6-6. wait()와 notify()

wait()와 notify()는 스레드 간 협력에 사용합니다. 둘 다 synchronized 영역 안에서 사용해야 합니다. 

wait()
→ 현재 스레드를 대기 상태로 만듦

notify()
→ wait()로 대기 중인 스레드 중 하나를 깨움

주의:
wait()와 notify()는 synchronized 영역 안에서 사용해야 한다.

7-4. synchronized 예제

class Counter {

    private int count = 0;

    public synchronized void increase() {
        count++;
        System.out.println(Thread.currentThread().getName() + ": " + count);
    }

}

class CountThread extends Thread {

    private Counter counter;

    public CountThread(Counter counter, String name) {
        super(name);
        this.counter = counter;
    }

    @Override
    public void run() {
        for (int i = 0; i < 5; i++) {
            counter.increase();
        }
    }

}

public class SynchronizedExample {

    public static void main(String[] args) {

        Counter counter = new Counter();

        CountThread t1 = new CountThread(counter, "스레드 A");
        CountThread t2 = new CountThread(counter, "스레드 B");

        t1.start();
        t2.start();

    }

}

8-3. 스레드 사용 시 주의할 점

  1. 실행 순서가 항상 일정하지 않습니다.
  2. 공유 자원을 동시에 수정하면 데이터가 꼬일 수 있습니다.
  3. synchronized를 너무 많이 쓰면 성능이 떨어질 수 있습니다.
  4. 데드락이 발생할 수 있습니다.
  5. start() 대신 run()을 호출하면 새 스레드가 시작되지 않습니다.
  6. sleep()은 예외 처리가 필요합니다.

8-4. 데드락

데드락은 서로가 가진 lock을 기다리다가 아무도 진행하지 못하는 상태입니다. 

스레드 A
→ lock 1을 가지고 있음
→ lock 2를 기다림

스레드 B
→ lock 2를 가지고 있음
→ lock 1을 기다림

결과:
둘 다 영원히 기다림

9. 자주 하는 실수

실수 1. start() 대신 run()을 호출함

MyThread thread = new MyThread();
thread.run(); // 새 스레드 생성 안 됨
MyThread thread = new MyThread();
thread.start();

실수 2. 한 스레드 객체에 start()를 두 번 호출함

MyThread thread = new MyThread();

thread.start();
thread.start(); // 오류
new MyThread().start();
new MyThread().start();

실수 3. 멀티 스레드 출력 순서가 항상 같다고 생각함

멀티 스레드는 실행 순서가 매번 달라질 수 있다.
출력 순서에 의존하는 코드를 작성하면 위험하다.

실수 4. 공유 자원을 동기화하지 않음

count++;

count++는 한 줄로 보이지만 내부적으로는 읽기, 증가, 저장 과정이 있습니다. 여러 스레드가 동시에 실행하면 값이 꼬일 수 있습니다. 

public synchronized void increase() {
    count++;
}

실수 5. sleep() 예외 처리를 하지 않음

Thread.sleep(1000); // InterruptedException 처리 필요
try {
    Thread.sleep(1000);
} catch (InterruptedException e) {
    System.out.println("중단됨");
}

실수 6. synchronized 범위를 너무 넓게 잡음

public synchronized void process() {
    // 오래 걸리는 작업 전체
}
public void process() {
    // 공유 자원과 무관한 작업

    synchronized (this) {
        // 공유 자원 수정 부분만 잠금
    }

10. 면접 포인트

Q1. 스레드란 무엇인가요?

답변: 스레드는 하나의 프로세스 안에서 실행되는 작업 흐름입니다. 하나의 프로그램 안에서 여러 작업을 동시에 처리하기 위해 사용합니다.

Q2. 프로세스와 스레드의 차이는 무엇인가요?

답변: 프로세스는 실행 중인 프로그램이고, 스레드는 그 프로세스 안에서 실제 작업을 수행하는 실행 흐름입니다. 하나의 프로세스는 하나 이상의 스레드를 가질 수 있습니다.

Q3. Java에서 스레드를 만드는 방법은 무엇인가요?

답변: Thread 클래스를 상속받는 방법과 Runnable 인터페이스를 구현하는 방법이 있습니다. 두 방식 모두 run()에 작업 내용을 작성하고, 실행할 때는 start()를 호출합니다.

Q4. run()과 start()의 차이는 무엇인가요?

답변: run()을 직접 호출하면 새 스레드가 만들어지지 않고 일반 메서드처럼 현재 스레드에서 실행됩니다. start()를 호출해야 새 스레드가 생성되고, JVM이 내부적으로 run()을 호출합니다.

Q5. Runnable 방식을 사용하는 이유는 무엇인가요?

답변: Java는 클래스 단일 상속만 지원하기 때문에 Thread를 상속받으면 다른 클래스를 상속받을 수 없습니다. Runnable을 구현하면 다른 클래스를 상속받으면서도 스레드 작업을 정의할 수 있고, 작업과 스레드 실행 객체를 분리할 수 있어 더 유연합니다.

Q6. sleep()은 무엇인가요?

답변: sleep()은 현재 실행 중인 스레드를 지정한 시간 동안 멈추게 하는 메서드입니다. InterruptedException 처리가 필요합니다.

Q7. join()은 무엇인가요?

답변: join()은 대상 스레드가 종료될 때까지 현재 스레드를 기다리게 하는 메서드입니다. 예를 들어 main 스레드에서 worker.join()을 호출하면 worker 스레드가 끝난 뒤 main 스레드가 이어서 실행됩니다.

Q8. synchronized는 무엇인가요?

답변: synchronized는 여러 스레드가 공유 자원에 동시에 접근하지 못하도록 lock을 거는 동기화 방법입니다. 한 번에 하나의 스레드만 해당 코드나 메서드에 접근하도록 제한해 데이터 꼬임을 방지합니다.

Q9. synchronized 블록과 synchronized 메서드의 차이는 무엇인가요?

답변: synchronized 블록은 필요한 코드 일부만 잠글 수 있고, synchronized 메서드는 메서드 전체를 잠급니다. 블록은 더 세밀하게 제어할 수 있고, 메서드는 작성이 간단합니다.

Q10. wait()와 notify()는 무엇인가요?

답변: wait()는 현재 스레드를 대기 상태로 만들고, notify()는 wait()로 대기 중인 스레드 중 하나를 깨워 실행 가능 상태로 만듭니다. 둘 다 synchronized 영역 안에서 사용해야 합니다.

11. 오늘의 핵심 정리

  1. 스레드는 하나의 프로세스 안에서 실행되는 작업 흐름이다.
  2. main 메서드도 main 스레드에서 실행된다.
  3. 멀티 스레딩은 하나의 프로그램 안에서 여러 스레드가 동시에 작업하는 것이다.
  4. Java는 Thread 클래스와 Runnable 인터페이스로 스레드를 만들 수 있다.
  5. Thread를 상속받으면 run()을 오버라이딩하고 start()로 실행한다.
  6. Runnable을 구현하면 run()을 오버라이딩하고 Thread 객체에 넣어 start()로 실행한다.
  7. run()을 직접 호출하면 새 스레드가 만들어지지 않는다.
  8. start()를 호출해야 새 스레드가 시작되고 JVM이 run()을 호출한다.
  9. currentThread()는 현재 실행 중인 스레드를 반환한다.
  10. getName()과 setName()으로 스레드 이름을 다룰 수 있다.
  11. sleep()은 현재 스레드를 일정 시간 멈추게 한다.
  12. yield()는 다른 스레드에게 실행 기회를 양보한다.
  13. join()은 대상 스레드가 끝날 때까지 기다린다.
  14. 스레드 우선순위에는 MAX_PRIORITY, MIN_PRIORITY, NORM_PRIORITY가 있다.
  15. synchronized는 공유 자원에 lock을 걸어 한 번에 하나의 스레드만 접근하게 한다.
  16. synchronized는 블록 또는 메서드에 사용할 수 있다.
  17. wait()는 스레드를 대기 상태로 만들고 notify()는 대기 중인 스레드를 깨운다.
  18. 멀티 스레드에서는 실행 순서가 항상 고정되지 않는다.
  19. 공유 자원을 사용할 때는 동기화 문제를 반드시 고려해야 한다.

스레드는 하나의 프로세스 안에서 실행되는 작업 흐름이며, Java에서는 Thread 클래스와 Runnable 인터페이스로 만들 수 있습니다.
run()을 직접 호출하면 새 스레드가 만들어지지 않고, start()를 호출해야 새 스레드가 시작됩니다.
멀티 스레드 환경에서는 실행 순서가 고정되지 않으며, 공유 자원을 사용할 때는 synchronized로 동기화 문제를 고려해야 합니다.
