15. 클래스와 객체

클래스는 객체를 만들기 위한 설계도이고, 객체는 클래스를 바탕으로 생성된 실제 대상입니다.
이 페이지에서는 클래스 선언, 멤버 변수, 메서드, 객체 생성, 인스턴스 멤버,
static 멤버, 객체 배열까지 Java 코드 흐름에 맞춰 공부합니다.

이번 페이지의 학습 목표

객체지향 프로그래밍의 개념을 실제 Java 코드로 옮기려면 클래스와 객체를 정확히 이해해야 합니다.
클래스는 객체의 속성과 기능을 정의하는 설계도이고, 객체는 new를 통해 메모리에 생성된 실제 대상입니다.

클래스 이해

클래스가 객체를 만들기 위한 설계도이자 개발자가 만드는 참조 자료형임을 이해합니다.

멤버 이해

멤버 변수는 속성, 메서드는 행위를 표현한다는 점을 익힙니다.

객체 생성과 접근

new로 객체를 만들고 .으로 멤버에 접근하는 방법을 익힙니다.

static과 객체 배열

인스턴스 멤버와 static 멤버의 차이, 객체 배열 사용 시 주의점을 이해합니다.

1. 클래스와 객체의 기본 개념

1-1. 클래스란?

클래스는 객체를 만들기 위한 설계도입니다. 객체가 어떤 속성을 가지고 어떤 기능을 수행할 수 있는지를 클래스 안에 정의합니다. 

class Account {

    String name;
    String accountNo;
    int balance;

}
클래스 Account
→ 실제 계좌가 아니라 계좌 객체를 만들기 위한 설계도

속성:
- name
- accountNo
- balance

1-2. 객체란?

객체는 클래스를 바탕으로 new를 통해 메모리에 만들어진 실제 대상입니다. 

Account account = new Account();
Account       → 클래스명
account       → 객체를 가리키는 참조 변수
new Account() → 객체 생성

new Account()가 실행되면 실제 Account 객체가 만들어지고, account 변수는 그 객체를 가리킵니다.

1-3. 클래스와 객체의 차이

구분 클래스 객체
의미 설계도 설계도로 만든 실제 대상
예시 Account new Account()
역할 속성과 기능 정의 실제 값을 저장하고 기능 실행
메모리 설계 정보 new로 생성 시 메모리에 올라감

비유: 클래스는 붕어빵 틀이고, 객체는 붕어빵입니다. 하나의 틀로 여러 개의 붕어빵을 만들 수 있듯이, 하나의 클래스로 여러 객체를 만들 수 있습니다.

1-4. 클래스도 데이터 타입이다

클래스는 단순히 코드 묶음이 아니라 개발자가 직접 만드는 참조 자료형입니다. 

int age;
double avg;
boolean active;

Account account;
User user;
Student student;
int, double, boolean
→ Java가 제공하는 기본 자료형

Account, User, Student
→ 개발자가 만든 참조 자료형

2. 클래스 선언과 멤버

2-1. 클래스 선언 문법

클래스는 class 키워드를 사용해 선언합니다. 클래스명은 보통 대문자로 시작하는 파스칼 케이스를 사용합니다. 

public class User {

    String name;
    int age;

    void introduce() {
        System.out.println("이름: " + name);
        System.out.println("나이: " + age);
    }

}

클래스명 예시: User, Account, Student, Property

2-2. 클래스의 멤버

클래스 안에 들어가는 구성 요소를 멤버라고 합니다. 대표적인 멤버는 멤버 변수와 메서드입니다. 

클래스의 멤버

1. 멤버 변수
→ 객체의 속성 표현

2. 메서드
→ 객체의 행위 표현

class Account {

    String name;
    String accountNo;
    int balance;

    void deposit(int amount) {
        balance += amount;
    }

}

name, accountNo, balance는 멤버 변수이고, deposit()은 메서드입니다.

2-3. 멤버 변수

멤버 변수는 객체가 가지는 데이터를 표현합니다. 객체지향에서 말하는 객체의 속성이 Java 코드에서는 멤버 변수 또는 필드가 됩니다. 

class User {

    String loginId;
    String name;
    String email;

}
객체의 속성 = 멤버 변수 = 필드

회원 객체의 속성:
- loginId
- name
- email

2-4. 메서드

메서드는 객체가 수행할 수 있는 기능입니다. 객체지향에서 말하는 객체의 행위가 Java 코드에서는 메서드가 됩니다. 

class User {

    String loginId;

    void login() {
        System.out.println(loginId + "님이 로그인했습니다.");
    }

}
객체의 행위 = 메서드

회원 객체의 행위:
- login()
- logout()
- changePassword()

3. 객체 생성과 멤버 접근

3-1. 객체 생성 문법

객체는 new 키워드를 사용해 생성합니다. 

클래스명 참조변수 = new 클래스명();
Account account = new Account();

현재는 생성자를 깊게 배우기 전이므로, 우선 new 클래스명() 형태로 객체를 만든다고 이해하면 됩니다.

3-2. 점(.)으로 멤버 접근하기

객체를 만든 뒤에는 점 .을 사용해 멤버 변수와 메서드에 접근합니다. 

Account account = new Account();

account.name = "홍길동";
account.accountNo = "123-456";
account.balance = 10000;

account.deposit(5000);
account.name
→ account 객체의 name 변수 접근

account.deposit()
→ account 객체의 deposit 메서드 호출

3-3. 참조 변수 이해하기

클래스 타입 변수에는 객체 자체가 통째로 들어가는 것이 아니라, 객체를 가리키는 참조값이 들어간다고 이해하면 좋습니다. 

Account account = new Account();

account
↓
Account 객체
┌──────────────────┐
│ name             │
│ accountNo        │
│ balance          │
└──────────────────┘

3-4. null과 객체

클래스 타입 변수는 참조형이므로 null을 가질 수 있습니다. null은 아직 어떤 객체도 가리키지 않는 상태입니다. 

Account account = null;

account.name = "홍길동";

위 코드는 객체가 없는 상태에서 멤버에 접근하므로 오류가 발생할 수 있습니다. 

Account account = new Account();

account.name = "홍길동";

4. 인스턴스 멤버와 static 멤버

4-1. 인스턴스 멤버

static이 붙지 않은 멤버는 인스턴스 멤버입니다. 인스턴스 멤버는 객체마다 따로 존재합니다. 

class User {

    String name;

    void introduce() {
        System.out.println(name);
    }

}
User user1 = new User();
User user2 = new User();

user1.name = "홍길동";
user2.name = "김철수";
user1.name → 홍길동
user2.name → 김철수

인스턴스 멤버는 객체마다 따로 값을 가진다.

4-2. static 정적 멤버

static이 붙은 멤버는 객체가 아니라 클래스에 소속됩니다. 그래서 객체를 만들지 않고 클래스명으로 접근할 수 있습니다. 

class Counter {

    static int count;

}
Counter.count = 10;

System.out.println(Counter.count);

4-3. 인스턴스 멤버와 static 멤버 비교

구분 인스턴스 멤버 static 멤버
선언 String name; static int count;
소속 객체 클래스
접근 객체명.멤버 클래스명.멤버
값 객체마다 다를 수 있음 클래스 전체에서 공유
예시 학생 이름, 계좌 잔고 학교 이름, 전체 객체 수

4-4. static 멤버 예제

class Student {

    String name;
    int score;

    static String schoolName = "ICT학교";

    void printInfo() {
        System.out.println("이름: " + name);
        System.out.println("점수: " + score);
        System.out.println("학교: " + schoolName);
    }

}
name, score
→ 객체마다 다른 값

schoolName
→ 모든 Student 객체가 공유하는 값

5. 코드 예제

5-1. Account 클래스 만들기

class Account {

    String name;
    String accountNo;
    int balance;

    public void deposit(int amount) {
        balance += amount;
    }

    public int withdraw(int amount) {
        if (balance < amount) {
            return 0;
        } else {
            balance -= amount;
            return amount;
        }
    }

    public void getAccountInfo() {
        System.out.println("예금주: " + name);
        System.out.println("계좌번호: " + accountNo);
        System.out.println("예금잔고: " + balance);
    }

}

name, accountNo, balance는 계좌의 속성이고, deposit(), withdraw(), getAccountInfo()는 계좌의 행위입니다.

5-2. Account 객체 사용하기

public class AccountMain {

    public static void main(String[] args) {

        Account account = new Account();

        account.name = "홍길동";
        account.accountNo = "123-456";
        account.balance = 10000;

        account.deposit(5000);

        int money = account.withdraw(3000);

        System.out.println("출금액: " + money);
        account.getAccountInfo();

    }

}
출금액: 3000
예금주: 홍길동
계좌번호: 123-456
예금잔고: 12000

5-3. 같은 클래스로 여러 객체 만들기

public class AccountMultiMain {

    public static void main(String[] args) {

        Account account1 = new Account();
        account1.name = "홍길동";
        account1.accountNo = "111-111";
        account1.balance = 10000;

        Account account2 = new Account();
        account2.name = "김철수";
        account2.accountNo = "222-222";
        account2.balance = 50000;

        account1.deposit(5000);
        account2.withdraw(10000);

        account1.getAccountInfo();
        account2.getAccountInfo();

    }

}

Account 클래스는 하나지만, account1과 account2는 서로 다른 객체입니다.

5-4. Student 클래스와 객체

class Student {

    String name;
    int age;
    int score;

    void introduce() {
        System.out.println("안녕하세요. 저는 " + name + "입니다.");
    }

    void printScore() {
        System.out.println("점수: " + score);
    }

}
public class StudentMain {

    public static void main(String[] args) {

        Student student = new Student();

        student.name = "홍길동";
        student.age = 20;
        student.score = 90;

        student.introduce();
        student.printScore();

    }

}
안녕하세요. 저는 홍길동입니다.
점수: 90

5-5. 객체 배열 예제

public class StudentArrayMain {

    public static void main(String[] args) {

        Student[] students = new Student[3];

        students[0] = new Student();
        students[1] = new Student();
        students[2] = new Student();

        students[0].name = "홍길동";
        students[0].score = 90;

        students[1].name = "김철수";
        students[1].score = 80;

        students[2].name = "이영희";
        students[2].score = 95;

        for (int i = 0; i < students.length; i++) {
            students[i].printScore();
        }

    }

}

6. 객체 배열 이해하기

6-1. 객체 배열은 배열 생성과 객체 생성을 따로 생각한다

객체도 배열에 저장할 수 있습니다. 하지만 객체 배열을 만들었다고 해서 객체가 자동으로 만들어지는 것은 아닙니다. 

Student[] students = new Student[3];
       students
┌──────┬──────┬──────┐
│ null │ null │ null │
└──────┴──────┴──────┘
   [0]    [1]    [2]

각 칸은 Student 객체를 담을 수 있는 자리일 뿐입니다. 실제 객체를 사용하려면 각 칸에 new Student()를 넣어야 합니다. 

students[0] = new Student();
students[1] = new Student();
students[2] = new Student();

6-2. 객체 배열 그림

students[0] = new Student();

         students
┌──────────┬──────┬──────┐
│ Student  │ null │ null │
└──────────┴──────┴──────┘
     [0]      [1]    [2]

7. 실무 연결

7-1. Entity 클래스

Spring 프로젝트에서 Entity 클래스는 DB 테이블과 가까운 객체입니다. 

public class UserEntity {

    private Long userId;
    private String loginId;
    private String email;
    private String userType;

}
USERS 테이블의 한 행
↓
UserEntity 객체 하나

7-2. DTO 클래스

DTO는 요청이나 응답 데이터를 담는 객체입니다. 

public class LoginRequestDto {

    private String loginId;
    private String password;

}

public class LoginResponseDto {

    private Long userId;
    private String loginId;
    private String userType;
    private String accessToken;

}

7-3. Service 클래스

Service 클래스는 실제 비즈니스 로직을 담당합니다. 

public class AuthService {

    public void login() {
        System.out.println("로그인 처리");
    }

}
Controller → 요청을 받음
Service    → 핵심 로직 처리
Repository → DB 접근
DTO        → 요청/응답 데이터 전달
Entity     → DB 데이터 표현

7-4. static 메서드의 실무 사용

static 메서드는 객체 생성 없이 사용할 수 있는 공통 기능에 자주 사용합니다. 

public class MathUtil {

    public static int add(int a, int b) {
        return a + b;
    }

}
int result = MathUtil.add(10, 20);

8. 자주 하는 실수

실수 1. 클래스 선언만 하고 객체가 생겼다고 착각

class Account {
    String name;
}

위 코드는 설계도만 만든 것입니다. 실제 객체를 사용하려면 아래처럼 객체를 생성해야 합니다. 

Account account = new Account();

실수 2. 객체 생성 없이 인스턴스 멤버 접근

Account.name = "홍길동";
Account account = new Account();
account.name = "홍길동";

실수 3. null 상태에서 멤버 접근

Account account = null;

account.name = "홍길동";

객체가 없는 상태에서 멤버에 접근하면 NullPointerException이 발생할 수 있습니다. 

Account account = new Account();

account.name = "홍길동";

실수 4. 객체 배열 생성과 객체 생성을 혼동

Student[] students = new Student[3];

students[0].name = "홍길동";
Student[] students = new Student[3];

students[0] = new Student();
students[0].name = "홍길동";

실수 5. static 멤버를 객체마다 따로 있다고 생각

static 멤버
→ 객체마다 따로 있는 값이 아니라 클래스에 하나만 존재하는 공유 값

예:
Student.schoolName

실수 6. 클래스명 작성 규칙을 지키지 않음

class student {
}
class Student {
}

Java에서 클래스명은 보통 대문자로 시작하는 파스칼 케이스를 사용합니다.

9. 면접 포인트

Q1. 클래스란 무엇인가요?

답변: 클래스는 객체를 만들기 위한 설계도입니다.
객체가 가져야 할 속성과 기능을 멤버 변수와 메서드로 정의합니다. 또한 클래스는 개발자가 직접 정의할 수 있는 참조 자료형입니다.

Q2. 객체란 무엇인가요?

답변: 객체는 클래스를 바탕으로 new 키워드를 통해 메모리에 생성된 실제 대상입니다. 객체는 클래스에 정의된 멤버 변수를 통해 값을 저장하고, 메서드를 통해 기능을 수행합니다.

Q3. 클래스와 객체의 차이는 무엇인가요?

답변: 클래스는 설계도이고 객체는 설계도를 바탕으로 만들어진 실제 인스턴스입니다. 클래스 하나로 여러 객체를 만들 수 있으며, 각 객체는 서로 다른 값을 가질 수 있습니다.

Q4. 멤버 변수와 메서드는 무엇인가요?

답변: 멤버 변수는 클래스 안에 선언되어 객체의 속성을 표현하는 변수입니다. 메서드는 클래스 안에 선언되어 객체의 행위를 표현하는 함수입니다.

Q5. 인스턴스 멤버와 static 멤버의 차이는 무엇인가요?

답변: 인스턴스 멤버는 객체마다 따로 존재하며 객체를 생성한 후 객체명으로 접근합니다. static 멤버는 클래스에 소속되어 클래스 로드 시 준비되고, 보통 클래스명으로 접근합니다.

Q6. static 멤버는 언제 사용하나요?

답변: 모든 객체가 공유해야 하는 값이나 객체 생성 없이 호출해야 하는 공통 기능에 사용합니다.
예를 들어 공통 상수, 유틸리티 메서드, 전체 객체 수를 세는 count 변수 등에 사용할 수 있습니다.

Q7. 객체 배열을 만들 때 주의할 점은 무엇인가요?

답변: 객체 배열을 생성하면 객체가 저장될 배열 공간만 만들어지고, 각 칸의 객체가 자동으로 생성되는 것은 아닙니다.
따라서 각 배열 요소에 new로 객체를 생성해서 넣어야 멤버에 접근할 수 있습니다.

10. 오늘의 핵심 정리

  1. 클래스는 객체를 만들기 위한 설계도다.
  2. 객체는 클래스를 바탕으로 new를 통해 생성된 실제 대상이다.
  3. 클래스의 멤버에는 멤버 변수와 메서드가 있다.
  4. 멤버 변수는 객체의 속성을 표현한다.
  5. 메서드는 객체의 행위를 표현한다.
  6. 객체 생성 문법은 클래스명 참조변수 = new 클래스명(); 형태다.
  7. 인스턴스 멤버는 객체명.멤버로 접근한다.
  8. static 멤버는 클래스명.멤버로 접근한다.
  9. 인스턴스 멤버는 객체마다 따로 존재한다.
  10. static 멤버는 클래스 단위로 공유된다.
  11. 클래스도 개발자가 만드는 참조 자료형이다.
  12. 클래스 타입 변수는 null을 가질 수 있다.
  13. null 상태에서 멤버에 접근하면 NullPointerException이 발생할 수 있다.
  14. 객체 배열은 배열 생성과 객체 생성을 따로 생각해야 한다.

학습 메모

이 페이지는 Java 클래스와 객체 단원입니다.
교안에서는 클래스를 객체의 설계도이자 하나의 자료구조, 데이터 타입으로 설명하고, 클래스의 멤버를 멤버 변수와 메서드로 구분합니다.
객체 생성은 클래스명 인스턴스변수 = new 생성자(); 형태이며, 인스턴스 멤버는 객체명.멤버, static 멤버는 클래스명.멤버로 접근한다는 점을 반드시 기억해야 합니다.
