27. 인터페이스

인터페이스는 클래스가 반드시 구현해야 할 기능 약속입니다.
interface, implements, 인터페이스 상수, 추상 메서드, 다형성, 다중 구현, 실무 Service 구조까지 공부합니다.

이번 페이지의 학습 목표

인터페이스는 “이 기능은 반드시 있어야 한다”는 규칙을 정하는 문법입니다.
클래스는 인터페이스를 구현하면서 약속된 메서드를 반드시 오버라이딩해야 합니다.
이를 통해 기능 명세와 실제 구현을 분리하고, 여러 구현체를 같은 타입으로 다룰 수 있습니다.

인터페이스 개념

클래스가 지켜야 하는 기능 약속이라는 개념을 이해합니다.

implements

클래스가 인터페이스를 구현하는 방법을 익힙니다.

다형성

인터페이스 타입으로 여러 구현 클래스를 다루는 구조를 이해합니다.

실무 연결

Service 인터페이스와 구현 클래스 구조를 이해합니다.

1. 인터페이스 기본 개념

1-1. 인터페이스란?

인터페이스는 클래스가 반드시 구현해야 할 기능 목록을 정해두는 구조입니다. 클래스가 객체의 설계도라면, 인터페이스는 클래스가 따라야 할 설계 규칙이라고 볼 수 있습니다. 

interface Animal {
    void sound();
}
Animal 인터페이스

sound();

의미:
Animal 인터페이스를 구현하는 클래스는
반드시 sound() 기능을 가져야 한다.

1-2. interface 선언 방법

interface 인터페이스명 {
    // 상수
    // 추상 메서드
}
interface RemoteControl {

    void turnOn();

    void turnOff();

}

위 인터페이스는 “리모컨 역할을 하는 클래스라면 켜기와 끄기 기능을 반드시 가져야 한다”는 약속입니다.

1-3. 인터페이스의 기본 멤버

멤버 자동으로 붙는 키워드 의미
변수 public static final 인터페이스 상수
메서드 public abstract 구현 클래스가 완성해야 할 추상 메서드 
public interface MyInterface {

    int MONEY = 100;

    void func();

}

위 코드는 실제로 아래처럼 해석됩니다.

public interface MyInterface {

    public static final int MONEY = 100;

    public abstract void func();

}

2. 인터페이스의 상수와 추상 메서드

2-1. 인터페이스 변수는 상수다

인터페이스 안에서 선언한 변수는 자동으로 public static final이 됩니다. 따라서 객체 없이 인터페이스명으로 접근할 수 있고, 값을 바꿀 수 없습니다. 

interface MyInterface {

    int MONEY = 100;

}

System.out.println(MyInterface.MONEY);
MyInterface.MONEY = 200; // 오류
인터페이스 변수

public
→ 어디서든 접근 가능

static
→ 객체 없이 인터페이스명으로 접근 가능

final
→ 값 변경 불가

결론:
인터페이스 변수는 상수다.

2-2. 인터페이스 메서드는 추상 메서드다

인터페이스 안에서 선언한 메서드는 자동으로 public abstract가 됩니다. 즉, 몸체가 없고 구현 클래스가 반드시 완성해야 합니다. 

interface MyInterface {

    void func();

}

위 메서드는 실제로 아래처럼 해석됩니다.

public abstract void func();

2-3. 인터페이스 메서드는 default 접근 지정자가 아니다

겉으로는 접근 지정자가 없어 보이지만, 인터페이스 메서드는 실제로 public abstract입니다. 

interface MyInterface {
    void func();
}

그래서 구현 클래스에서 오버라이딩할 때 접근 범위를 줄일 수 없으며, 반드시 public을 붙여야 합니다.

3. implements와 구현 클래스

3-1. 클래스가 인터페이스를 구현할 때는 implements

클래스가 인터페이스를 사용할 때는 extends가 아니라 implements를 사용합니다. 

interface MyInterface {

    void func();

}

class GeneralClass implements MyInterface {

    @Override
    public void func() {
        System.out.println("func 구현");
    }

}
클래스 extends 클래스
클래스 implements 인터페이스
인터페이스 extends 인터페이스

3-2. 구현 클래스의 의무

인터페이스를 구현한 클래스는 인터페이스 안의 모든 추상 메서드를 반드시 오버라이딩해야 합니다. 

interface Animal {
    void sound();
}

class Dog implements Animal {

    @Override
    public void sound() {
        System.out.println("멍멍");
    }

}

3-3. 구현하지 않으면 오류

interface Animal {
    void sound();
}

class Dog implements Animal {
}

Dog가 sound()를 구현하지 않았기 때문에 오류가 발생합니다.

해결 방법 1: 메서드를 구현합니다.

class Dog implements Animal {

    @Override
    public void sound() {
        System.out.println("멍멍");
    }

}

해결 방법 2: 구현 클래스도 추상 클래스로 선언합니다.

abstract class Dog implements Animal {
}

3-4. 오버라이딩할 때 public 필수

interface MyInterface {
    void func();
}

class GeneralClass implements MyInterface {

    void func() { // 오류
        System.out.println("구현");
    }

}
class GeneralClass implements MyInterface {

    @Override
    public void func() {
        System.out.println("구현");
    }

}

인터페이스 메서드는 실제로 public abstract이므로, 구현 클래스에서 더 좁은 접근 범위로 오버라이딩할 수 없습니다.

4. 인터페이스와 객체 생성

4-1. 인터페이스는 직접 객체 생성이 불가능하다

인터페이스는 구현되지 않은 추상 메서드의 약속만 가지고 있으므로, 직접 new 해서 객체를 만들 수 없습니다. 

Animal animal = new Animal(); // 오류

대신 인터페이스를 구현한 클래스의 객체를 만들 수 있습니다. 

Animal animal = new Dog();

4-2. 인터페이스 타입 참조변수

인터페이스는 직접 객체 생성은 안 되지만, 타입으로 사용할 수 있습니다. 

Animal animal = new Dog();
Animal animal
→ 인터페이스 타입 참조변수

new Dog()
→ Animal 인터페이스를 구현한 클래스의 객체

animal.sound()
→ Dog가 구현한 sound() 실행

5. 인터페이스와 다형성

5-1. 같은 인터페이스, 다른 구현

interface Animal {
    void sound();
}

class Dog implements Animal {

    @Override
    public void sound() {
        System.out.println("멍멍");
    }

}

class Cat implements Animal {

    @Override
    public void sound() {
        System.out.println("야옹");
    }

}
Animal animal1 = new Dog();
Animal animal2 = new Cat();

animal1.sound();
animal2.sound();
멍멍
야옹

참조변수 타입은 Animal로 같지만, 실제 객체가 무엇인지에 따라 실행 결과가 달라집니다.

5-2. 배열과 반복문에서의 다형성

Animal[] animals = {
    new Dog(),
    new Cat()
};

for (Animal animal : animals) {
    animal.sound();
}
멍멍
야옹
배열 타입:
Animal[]

실제 객체:
Dog
Cat

실행:
Dog → 멍멍
Cat → 야옹

5-3. 인터페이스 다형성의 장점

Dog만 처리하는 코드
→ Dog에 강하게 묶임

Animal 인터페이스로 처리하는 코드
→ Dog, Cat, Bird 등 다양한 구현체 처리 가능

인터페이스 타입으로 코드를 작성하면 구체적인 구현 클래스가 바뀌어도 사용하는 쪽 코드를 크게 바꾸지 않아도 됩니다.

6. 다중 구현

6-1. Java는 클래스 다중 상속을 허용하지 않는다

class Child extends Parent1, Parent2 { // 오류
}

Java에서 클래스는 하나만 상속받을 수 있습니다. 하지만 인터페이스는 여러 개를 동시에 구현할 수 있습니다.

6-2. 여러 인터페이스 구현

interface Flyable {
    void fly();
}

interface Swimmable {
    void swim();
}

class Duck implements Flyable, Swimmable {

    @Override
    public void fly() {
        System.out.println("오리가 납니다.");
    }

    @Override
    public void swim() {
        System.out.println("오리가 헤엄칩니다.");
    }

}
Duck duck = new Duck();

duck.fly();
duck.swim();
오리가 납니다.
오리가 헤엄칩니다.
Duck
→ Flyable 기능 구현
→ Swimmable 기능 구현

인터페이스는 능력을 여러 개 붙이는 방식으로 이해하면 좋다.

6-3. 클래스 상속과 인터페이스 구현 함께 사용

class Parent {

    public void common() {
        System.out.println("부모 공통 기능");
    }

}

interface Flyable {
    void fly();
}

interface Swimmable {
    void swim();
}

class Duck extends Parent implements Flyable, Swimmable {

    @Override
    public void fly() {
        System.out.println("오리가 납니다.");
    }

    @Override
    public void swim() {
        System.out.println("오리가 헤엄칩니다.");
    }

}
class Duck extends Parent implements Flyable, Swimmable

Parent 클래스는 하나만 상속
Flyable, Swimmable 인터페이스는 여러 개 구현

7. 인터페이스끼리의 상속

7-1. 인터페이스가 인터페이스를 상속할 때는 extends

클래스가 인터페이스를 사용할 때는 implements를 쓰지만, 인터페이스가 인터페이스를 상속할 때는 extends를 사용합니다. 

interface MyInter1 {
    void method1();
}

interface MyInter2 {
    void method2();
}

interface MyInter3 extends MyInter1, MyInter2 {
    void method3();
}

MyInter3을 구현하는 클래스는 세 메서드를 모두 구현해야 합니다. 

class MyClass implements MyInter3 {

    @Override
    public void method1() {
    }

    @Override
    public void method2() {
    }

    @Override
    public void method3() {
    }

}

7-2. 인터페이스의 접근 지정자

최상위 인터페이스는 보통 public 또는 default 접근만 사용할 수 있습니다. 

public interface MyInterface {
}

interface MyInterface {
}
private interface MyInterface {
}

protected interface MyInterface {
}

8. 추상 클래스와 인터페이스 비교

8-1. 비교 표

구분 추상 클래스 인터페이스
키워드 abstract class interface
클래스가 사용할 때 extends implements
객체 생성 직접 생성 불가 직접 생성 불가
일반 필드 가능 기본적으로 상수
생성자 가능 불가능
일반 메서드 가능 기초 교안 기준 추상 메서드 중심
추상 메서드 가능 가능
다중 적용 클래스는 단일 상속 여러 인터페이스 구현 가능
목적 공통 기능 + 강제 구현 기능 규칙, 약속 정의

8-2. 선택 기준

추상 클래스
→ 공통 코드가 많고, 부모-자식 관계가 강할 때

9. 실무 연결

9-1. 기능 명세와 구현 분리

public interface UserService {

    UserResponseDto findById(Long userId);

    void register(UserRequestDto requestDto);

}
public class UserServiceImpl implements UserService {

    @Override
    public UserResponseDto findById(Long userId) {
        System.out.println("회원 조회");
        return null;
    }

    @Override
    public void register(UserRequestDto requestDto) {
        System.out.println("회원 가입");
    }

}
UserService
→ 기능 명세

UserServiceImpl
→ 실제 구현

9-2. 결제 방식 교체 예제

interface PaymentService {
    void pay(int amount);
}

class CardPaymentService implements PaymentService {

    @Override
    public void pay(int amount) {
        System.out.println(amount + "원을 카드로 결제합니다.");
    }

}

class KakaoPaymentService implements PaymentService {

    @Override
    public void pay(int amount) {
        System.out.println(amount + "원을 카카오페이로 결제합니다.");
    }

}
PaymentService paymentService = new CardPaymentService();
paymentService.pay(10000);

paymentService = new KakaoPaymentService();
paymentService.pay(10000);

사용하는 쪽은 PaymentService라는 약속만 알고 있으면 됩니다. 실제 결제 방식은 구현 클래스를 바꾸는 방식으로 교체할 수 있습니다.

9-3. 인터페이스를 사용하는 이유

  1. 기능 명세와 구현을 분리할 수 있습니다.
  2. 구현 클래스를 바꿔도 사용하는 쪽 코드를 줄일 수 있습니다.
  3. 테스트용 가짜 구현체를 만들기 쉽습니다.
  4. 서로 다른 클래스들을 같은 타입으로 다룰 수 있습니다.
  5. 클래스 다중 상속의 한계를 보완할 수 있습니다.

12. 면접 포인트

Q1. 인터페이스란 무엇인가요?

답변: 인터페이스는 클래스가 반드시 구현해야 할 기능 목록을 정의하는 구조입니다.
클래스가 객체의 설계도라면, 인터페이스는 클래스의 설계도 또는 기능 약속이라고 볼 수 있습니다.

Q2. 인터페이스의 멤버는 무엇으로 구성되나요?

답변: 기초적으로 인터페이스의 멤버는 상수와 추상 메서드로 구성됩니다.
인터페이스 변수는 자동으로 public static final이 되고, 인터페이스 메서드는 자동으로 public abstract가 됩니다.

Q3. 클래스가 인터페이스를 사용할 때 어떤 키워드를 쓰나요?

답변: implements 키워드를 사용합니다. 클래스는 클래스를 상속할 때 extends를 사용하고, 인터페이스를 구현할 때 implements를 사용합니다.

Q4. 인터페이스를 구현한 클래스는 무엇을 해야 하나요?

답변: 인터페이스에 선언된 모든 추상 메서드를 반드시 오버라이딩해야 합니다. 만약 구현하지 않으려면 해당 클래스도 abstract class로 선언해야 합니다.

Q5. 인터페이스 메서드를 구현할 때 왜 public을 붙여야 하나요?

답변: 인터페이스의 메서드는 생략되어 있어도 실제로는 public abstract입니다.
오버라이딩할 때 접근 범위를 더 좁게 만들 수 없기 때문에 구현 메서드에는 반드시 public을 붙여야 합니다.

Q6. 인터페이스는 객체 생성이 가능한가요?

답변: 직접 객체 생성은 불가능합니다. 하지만 인터페이스 타입 참조변수로 구현 클래스의 객체를 참조할 수 있습니다. 예를 들어 Animal animal = new Dog();는 가능합니다.

Q7. 인터페이스와 다형성은 어떤 관계가 있나요?

답변: 인터페이스 타입으로 여러 구현 클래스 객체를 같은 방식으로 다룰 수 있습니다.
예를 들어 Animal 인터페이스 타입으로 Dog, Cat 객체를 참조하고 sound()를 호출하면 각 구현 클래스가 오버라이딩한 메서드가 실행됩니다.

Q8. 인터페이스를 여러 개 구현할 수 있나요?

답변: 네. Java는 클래스 다중 상속은 허용하지 않지만, 인터페이스는 여러 개를 동시에 implements할 수 있습니다.

Q9. 인터페이스가 인터페이스를 상속할 때는 어떤 키워드를 사용하나요?

답변: extends를 사용합니다. 인터페이스는 여러 인터페이스를 동시에 extends할 수 있습니다.

Q10. 추상 클래스와 인터페이스의 차이는 무엇인가요?

답변: 추상 클래스는 공통 필드, 생성자, 일반 메서드, 추상 메서드를 함께 가질 수 있고, 상속 관계에서 공통 기능과 강제 구현을 함께 제공할 때 사용합니다.
인터페이스는 주로 클래스가 반드시 구현해야 할 기능 규칙이나 약속을 정의할 때 사용합니다. 또한 클래스는 하나만 상속할 수 있지만, 인터페이스는 여러 개 구현할 수 있습니다.

13. 오늘의 핵심 정리

  1. 인터페이스는 클래스가 구현해야 할 기능 규칙이다.
  2. 클래스가 객체의 설계도라면, 인터페이스는 클래스의 설계도라고 볼 수 있다.
  3. 인터페이스는 interface 키워드로 선언한다.
  4. 인터페이스의 변수는 자동으로 public static final이 된다.
  5. 인터페이스의 메서드는 자동으로 public abstract가 된다.
  6. 클래스가 인터페이스를 구현할 때는 implements를 사용한다.
  7. 인터페이스를 구현한 클래스는 모든 추상 메서드를 오버라이딩해야 한다.
  8. 인터페이스 메서드를 오버라이딩할 때는 반드시 public을 붙여야 한다.
  9. 인터페이스는 직접 new 해서 객체 생성할 수 없다.
  10. 인터페이스 타입 참조변수는 구현 클래스 객체를 참조할 수 있다.
  11. 인터페이스를 사용하면 다형성을 구현할 수 있다.
  12. 클래스는 하나만 extends할 수 있지만 인터페이스는 여러 개 implements할 수 있다.
  13. 클래스 상속과 인터페이스 구현을 함께 사용할 수 있다.
  14. 인터페이스가 인터페이스를 상속할 때는 extends를 사용한다.
  15. 인터페이스는 여러 인터페이스를 동시에 extends할 수 있다.
  16. 인터페이스는 기능 명세와 구현을 분리할 때 유용하다.

학습 메모

이 페이지는 Java 인터페이스 단원입니다.
인터페이스는 클래스가 반드시 구현해야 할 기능 약속이며, 클래스는 implements 키워드로 인터페이스를 구현합니다.
인터페이스의 변수는 자동으로 public static final 상수이고, 메서드는 자동으로 public abstract 추상 메서드입니다.
구현 클래스에서 메서드를 오버라이딩할 때는 반드시 public을 붙여야 하며, 인터페이스 타입 참조변수를 사용하면 여러 구현체를 같은 방식으로 다룰 수 있습니다.
