22. static block

static block은 클래스가 JVM에 로드될 때 실행되는 정적 초기화 블록입니다.
main()보다 먼저 실행되며, 주로 static 변수 초기화에 사용됩니다.
이 페이지에서는 실행 시점, 접근 가능한 멤버, 생성자와의 차이, 초기화 순서를 공부합니다.

이번 페이지의 학습 목표

static block은 객체가 생성될 때 실행되는 코드가 아니라, 클래스가 메모리에 로드될 때 한 번 실행되는 코드입니다.
그래서 객체마다 다른 값을 초기화하기보다는 클래스 단위의 공유 값, 즉 static 변수를 초기화할 때 사용합니다.

실행 시점 이해

static block이 main()보다 먼저 실행되는 이유를 이해합니다.

static 초기화

클래스 단위로 공유되는 static 변수를 초기화하는 방법을 익힙니다.

접근 범위 이해

static 멤버와 인스턴스 멤버의 접근 차이를 구분합니다.

생성자와 비교

static block과 생성자의 실행 횟수와 목적 차이를 이해합니다.

1. static block의 기본 개념

1-1. static block이란?

static block은 클래스가 JVM에 의해 로드될 때 실행되는 초기화 블록입니다. 다른 이름으로는 static Initializer라고도 부릅니다. 

static {
    // 클래스가 로드될 때 실행할 코드
}
class StaticBlockTest {

    static {
        System.out.println("static block 실행");
    }

    public static void main(String[] args) {
        System.out.println("main 실행");
    }

}
static block 실행
main 실행

핵심: static block은 객체 생성보다 먼저, 그리고 main() 실행보다 먼저 실행될 수 있습니다.

1-2. static block 실행 시점

static block은 객체를 만들 때 실행되는 것이 아니라, 클래스가 메모리에 로드될 때 실행됩니다. 

프로그램 실행 요청
        ↓
클래스 로드
        ↓
static 변수 메모리 생성
        ↓
static block 실행
        ↓
main() 실행

1-3. main보다 먼저 실행된다는 의미

class StaticBlockTest {

    static int num;

    static {
        num = 10;
        System.out.println("static block 실행");
    }

    public static void main(String[] args) {
        System.out.println("main 실행");
        System.out.println("num = " + num);
    }

}
static block 실행
main 실행
num = 10
1. main이 시작되기 전에 static block이 먼저 실행된다.
2. static block에서 num 값을 10으로 초기화한다.
3. main에서는 이미 초기화된 num 값을 사용할 수 있다.

2. static 변수 초기화

2-1. static 변수 초기화에 사용

static block의 대표적인 용도는 static 변수 초기화입니다. static 변수는 객체마다 따로 존재하는 값이 아니라 클래스에 하나만 존재하는 공유 값입니다. 

class AppConfig {

    static String appName;
    static int maxUserCount;

    static {
        appName = "Java Study Magazine";
        maxUserCount = 100;
    }

}
System.out.println(AppConfig.appName);
System.out.println(AppConfig.maxUserCount);
Java Study Magazine
100

2-2. static 변수와 인스턴스 변수 비교

구분 static 변수 인스턴스 변수
소속 클래스 객체
메모리 생성 시점 클래스 로드 시 객체 생성 시
접근 방법 클래스명.변수명 객체명.변수명
공유 여부 모든 객체가 공유 객체마다 따로 가짐
static block에서 직접 접근 가능 불가능

2-3. static block 안의 지역 변수

static block 안에서도 일반 블록처럼 지역 변수를 선언할 수 있습니다. 단, 블록 안에서 선언한 지역 변수는 그 블록 내부에서만 사용할 수 있습니다. 

class Test {

    static int num;

    static {
        num = 10;

        int sum = num * 100;

        System.out.println("num은 " + num + ", sum=" + sum);
    }

}
num
→ 클래스 전체에서 사용할 수 있는 static 변수

sum
→ static block 안에서만 사용할 수 있는 지역 변수

3. static block에서 접근 가능한 멤버

3-1. static 멤버는 직접 접근 가능

class StaticBlockTest {

    static int num;
    String name;

    static {
        num = 10;          // 가능
        // name = "홍길동"; // 오류
    }

}
num
→ static 변수
→ 클래스가 로드될 때 메모리에 생김
→ static block에서 접근 가능

name
→ 인스턴스 변수
→ 객체를 new 해야 메모리에 생김
→ static block 실행 시점에는 객체가 없음
→ 직접 접근 불가

3-2. static 메서드는 호출 가능

class StaticBlockTest {

    static {
        staticPrint();       // 가능
        // nonStaticPrint(); // 오류
    }

    public void nonStaticPrint() {
        System.out.println("인스턴스 메서드");
    }

    public static void staticPrint() {
        System.out.println("정적 메서드");
    }

}
staticPrint()
→ static 메서드
→ 객체 없이 호출 가능

nonStaticPrint()
→ 인스턴스 메서드
→ 객체가 있어야 호출 가능

3-3. static block에서 직접 접근 불가능한 것

static block

직접 접근 가능:
- static 변수
- static 메서드
- static block 내부 지역 변수

직접 접근 불가:
- 인스턴스 변수
- 인스턴스 메서드
- this
- super

3-4. 객체를 만들면 인스턴스 멤버 접근은 가능하지만 권장하지는 않음

static block 안에서 인스턴스 멤버에 직접 접근할 수는 없지만, 객체를 직접 만들면 접근할 수는 있습니다. 

class Test {

    String name = "홍길동";

    static {
        Test test = new Test();
        System.out.println(test.name);
    }

}

하지만 static block은 클래스 초기화용이고, 인스턴스 변수는 객체 생성 후 생성자나 메서드에서 다루는 것이 자연스럽습니다. 초급 단계에서는 static block을 static 변수 초기화에 집중해서 사용하는 것이 좋습니다.

4. static block과 생성자 비교

4-1. static block과 생성자의 차이

구분 static block 생성자
실행 시점 클래스 로드 시 객체 생성 시
실행 횟수 클래스당 1번 객체 생성할 때마다
주 용도 static 변수 초기화 인스턴스 변수 초기화
인스턴스 멤버 직접 접근 불가능 가능
main보다 먼저 실행 가능 객체를 만들어야 실행

4-2. 실행 횟수 비교 예제

class Test {

    static {
        System.out.println("static block");
    }

    public Test() {
        System.out.println("constructor");
    }

    public static void main(String[] args) {
        System.out.println("main");

        new Test();
        new Test();
    }

}
static block
main
constructor
constructor

static block은 클래스 로드 시 한 번 실행되고, 생성자는 객체를 만들 때마다 실행됩니다.

5. static 초기화 순서

5-1. static block이 여러 개 있을 때

static block은 한 클래스 안에 여러 개 작성할 수 있습니다. 실행 순서는 코드에 작성된 위에서 아래 순서입니다. 

class Test {

    static {
        System.out.println("첫 번째 static block");
    }

    static {
        System.out.println("두 번째 static block");
    }

    public static void main(String[] args) {
        System.out.println("main");
    }

}
첫 번째 static block
두 번째 static block
main

5-2. static 변수 초기화와 static block 실행 순서

static 변수 초기화와 static block은 코드에 작성된 순서대로 실행됩니다. 

class Test {

    static int num = 10;

    static {
        num = 20;
        System.out.println("static block num = " + num);
    }

    static int count = 30;

    public static void main(String[] args) {
        System.out.println("main num = " + num);
        System.out.println("main count = " + count);
    }

}
static block num = 20
main num = 20
main count = 30
1. static int num = 10;
2. static block 실행 → num = 20;
3. static int count = 30;
4. main 실행

5-3. 초기화 순서 착각 주의

class Test {

    static {
        System.out.println(num);
    }

    static int num = 10;

}

static 초기화는 위에서 아래로 진행됩니다. 초급 단계에서는 static 변수를 static block보다 위에 선언하는 습관이 좋습니다. 

static int num;

static {
    num = 10;
}

6. 실무 연결

6-1. 코드 테이블 초기화

import java.util.HashMap;
import java.util.Map;

class UserTypeCode {

    static Map<String, String> userTypes = new HashMap<>();

    static {
        userTypes.put("U", "일반 회원");
        userTypes.put("A", "관리자");
        userTypes.put("R", "공인중개사");
    }

}
System.out.println(UserTypeCode.userTypes.get("A"));
관리자

6-2. 복잡한 static 변수 초기화

단순 대입이면 변수 선언과 동시에 초기화하면 되지만, 여러 줄의 초기화 로직이 필요하면 static block을 사용할 수 있습니다. 

class SystemConfig {

    static int maxCount;
    static String mode;

    static {
        maxCount = 100;
        mode = "DEV";

        System.out.println("초기 설정 완료");
    }

}

6-3. 현대 Java의 대체 문법

간단한 Map 초기화라면 static block 대신 Map.of() 같은 문법을 사용할 수도 있습니다. 

static Map<String, String> userTypes = Map.of(
    "U", "일반 회원",
    "A", "관리자",
    "R", "공인중개사"
);

그래도 static block을 이해하면 오래된 코드, 교안 예제, 복잡한 초기화 로직을 읽을 때 도움이 됩니다.

6-4. static block 사용 기준

사용하기 좋은 경우:
- static final Map 초기 데이터 구성
- 복잡한 static 변수 초기화
- 클래스 로딩 시 한 번만 필요한 설정
- 테스트용 샘플 데이터 초기화
- 간단한 상수 목록 또는 코드 테이블 초기화

주의할 경우:
- 너무 오래 걸리는 작업
- 예외가 자주 발생하는 작업
- 외부 자원 연결
- 복잡한 비즈니스 로직

9. 면접 포인트

Q1. static block이란 무엇인가요?

답변: static block은 클래스가 JVM에 로드될 때 실행되는 정적 초기화 블록입니다. static { } 형태로 작성하며, 주로 static 변수를 초기화할 때 사용합니다.

Q2. static block은 main보다 먼저 실행되나요?

답변: 네. static block은 클래스 로드 시 실행되므로 main 메서드보다 먼저 실행됩니다.

Q3. static block은 몇 번 실행되나요?

답변: 클래스가 로드될 때 한 번 실행됩니다. 객체를 여러 번 생성해도 static block은 객체 생성 때마다 실행되지 않습니다.

Q4. static block에서 인스턴스 변수에 직접 접근할 수 있나요?

답변: 직접 접근할 수 없습니다. static block은 객체 생성 전에도 실행될 수 있지만, 인스턴스 변수는 객체가 생성되어야 메모리에 존재하기 때문입니다.

Q5. static block에서 static 메서드는 호출할 수 있나요?

답변: 호출할 수 있습니다. static 메서드는 객체 없이 클래스 단위로 존재하므로 static block에서 직접 호출할 수 있습니다.

Q6. static block과 생성자의 차이는 무엇인가요?

답변: static block은 클래스 로드 시 한 번 실행되고 static 변수 초기화에 주로 사용됩니다. 생성자는 객체 생성 시마다 실행되고 인스턴스 변수 초기화에 주로 사용됩니다.

Q7. static block이 여러 개 있으면 실행 순서는 어떻게 되나요?

답변: 코드에 작성된 위에서 아래 순서대로 실행됩니다. static 변수 초기화와 static block도 작성 순서에 따라 실행됩니다.

Q8. static block은 실무에서 언제 사용할 수 있나요?

답변: 복잡한 static 변수 초기화, static Map 초기 데이터 구성, 클래스 로딩 시 한 번만 필요한 설정 등에 사용할 수 있습니다.
다만 너무 복잡한 로직을 static block에 넣는 것은 피하는 것이 좋습니다.

10. 오늘의 핵심 정리

  1. static block은 static Initializer라고도 한다.
  2. static block은 static { } 형태로 작성한다.
  3. static block은 클래스가 JVM에 로드될 때 실행된다.
  4. static block은 main보다 먼저 실행된다.
  5. static block은 static 변수 초기화에 주로 사용된다.
  6. static block 안에서는 static 멤버에만 직접 접근할 수 있다.
  7. 인스턴스 변수와 인스턴스 메서드는 객체가 있어야 하므로 static block에서 직접 접근할 수 없다.
  8. static block은 클래스당 한 번 실행된다.
  9. 생성자는 객체를 만들 때마다 실행되지만, static block은 클래스 로드 시 한 번만 실행된다.
  10. static block이 여러 개 있으면 위에서 아래 순서로 실행된다.
  11. static 변수 초기화와 static block은 코드 작성 순서대로 실행된다.
  12. static block은 복잡한 static 변수 초기화가 필요할 때 사용할 수 있다.

학습 메모

이 페이지는 Java의 static block 단원입니다.
static block은 클래스가 JVM에 로드될 때 main()보다 먼저 한 번 실행되는 정적 초기화 블록입니다.
주로 static 변수를 초기화할 때 사용하며, 객체가 생성되어야 존재하는 인스턴스 변수나 인스턴스 메서드에는 직접 접근할 수 없습니다.
생성자는 객체를 만들 때마다 실행되지만, static block은 클래스당 한 번만 실행된다는 차이를 꼭 기억해야 합니다.
