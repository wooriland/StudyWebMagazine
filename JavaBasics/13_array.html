13. 배열

배열은 하나의 이름으로 같은 자료형의 데이터를 여러 개 저장하는 구조입니다.
이 페이지에서는 배열 선언, 메모리 할당, 초기화, 인덱스 접근, length,
반복문 활용, 2차원 배열까지 단계별로 공부합니다.

이번 페이지의 학습 목표

변수는 하나의 이름으로 하나의 값만 저장하지만, 배열은 하나의 이름으로 같은 자료형의 값을 여러 개 저장할 수 있습니다.
배열은 점수 목록, 이름 목록, 메뉴 목록, 2차원 표 데이터처럼 같은 성격의 데이터를 묶어 관리할 때 사용합니다.

배열 개념 이해

배열이 같은 자료형 데이터를 여러 개 저장하는 구조라는 점을 이해합니다.

선언과 메모리 할당

int[] scores, new int[3] 구조를 익힙니다.

인덱스와 length

인덱스가 0부터 시작하고 마지막 인덱스가 length - 1임을 이해합니다.

2차원 배열

행과 열 구조의 배열을 중첩 반복문으로 다루는 방법을 익힙니다.

1. 배열의 기본 개념

1-1. 배열이란?

배열은 하나의 이름으로 같은 자료형의 데이터를 여러 개 저장하는 자료구조입니다.
교안에서도 배열을 하나의 이름으로 하나의 데이터 형식을 여러 개 저장 가능한 공간, 즉 동종 데이터를 저장하는 자료구조로 설명합니다. 

int[] scores = {100, 90, 80};
scores
┌─────┬────┬────┐
│ 100 │ 90 │ 80 │
└─────┴────┴────┘
[0]   [1]  [2]

배열은 같은 자료형의 값만 저장합니다. int[] 배열에는 정수 값을, String[] 배열에는 문자열 값을 저장합니다.

1-2. 배열이 필요한 이유

점수 3개 정도는 변수 3개로 저장할 수 있습니다. 하지만 점수가 100개라면 변수 100개를 만드는 것은 비효율적입니다. 

int score1 = 100;
int score2 = 90;
int score3 = 80;
// int score100까지 계속 만들어야 한다면 관리가 어려워집니다.
int[] scores = new int[100];

배열을 사용하면 하나의 이름으로 여러 값을 묶을 수 있고, 반복문을 사용해 전체 데이터를 효율적으로 처리할 수 있습니다.

1-3. 변수와 배열의 차이

변수
score1 = 100
score2 = 90
score3 = 80

배열
scores
┌─────┬────┬────┐
│ 100 │ 90 │ 80 │
└─────┴────┴────┘
[0]   [1]  [2]

변수는 각각 따로 관리하지만, 배열은 같은 종류의 값을 하나의 묶음으로 관리합니다.

2. 배열 선언, 메모리 할당, 초기화

2-1. 배열 선언

배열 선언은 배열을 가리킬 변수를 만드는 단계입니다. 교안에서는 데이터형[] 배열명 또는 데이터형 배열명[] 두 가지 문법을 제시합니다. 

int[] j;
int j[];

둘 다 가능하지만 Java에서는 보통 첫 번째 방식을 더 추천합니다. 자료형 옆에 []를 붙이면 “int 배열”이라는 의미가 더 잘 보입니다. 

int[] scores;
String[] names;
double[] averages;

2-2. 배열 메모리 할당

배열은 선언만으로는 값을 저장할 공간이 만들어지지 않습니다. 실제 배열 공간은 new 키워드로 만들어야 합니다. 

int[] scores;

scores = new int[3];
int[] scores;
→ 배열을 가리킬 변수만 선언

scores = new int[3];
→ int 값 3개를 저장할 공간 생성

교안에서는 배열 메모리 할당 문법을 배열명 = new 데이터형[배열크기]로 정리합니다.

2-3. 선언과 메모리 할당 동시에 하기

배열은 선언과 동시에 메모리 할당을 할 수 있습니다. 

int[] scores = new int[3];
scores
┌───┬───┬───┐
│ 0 │ 0 │ 0 │
└───┴───┴───┘
[0] [1] [2]

배열의 인덱스는 0부터 시작합니다. 크기가 3인 배열의 사용 가능한 인덱스는 0, 1, 2입니다.

2-4. 배열 초기화

배열의 각 칸에 값을 넣으려면 인덱스를 사용합니다. 

int[] scores = new int[3];

scores[0] = 100;
scores[1] = 90;
scores[2] = 80;
scores
┌─────┬────┬────┐
│ 100 │ 90 │ 80 │
└─────┴────┴────┘
[0]   [1]  [2]

2-5. 선언, 할당, 초기화를 동시에 하기

교안에서는 배열 선언, 메모리 할당, 초기화를 동시에 하는 방법도 제시합니다. 

int[] scores = new int[]{100, 90, 80};

더 간단하게는 아래처럼 작성할 수 있습니다.

int[] scores = {100, 90, 80};

정리: 배열 값을 직접 나열하면 배열 크기는 자동으로 결정됩니다. 위 예제의 배열 크기는 3입니다.

2-6. 배열의 기본값

교안에서는 배열이 메모리 할당 후 초기화하지 않아도 디폴트 값으로 초기화된다고 설명합니다. 단, 먼저 반드시 new로 메모리 할당을 해야 합니다. 

int[] numbers = new int[3];

System.out.println(numbers[0]);
System.out.println(numbers[1]);
System.out.println(numbers[2]);
0
0
0
자료형 기본값
정수형 byte, short, int, long 0
실수형 float, double 0.0
char '\u0000'
boolean false
참조형 String, 클래스 등 null

3. 인덱스와 length

3-1. 배열 인덱스

배열의 각 칸은 인덱스로 접근합니다. Java 배열의 인덱스는 1이 아니라 0부터 시작합니다. 

int[] scores = {100, 90, 80};

System.out.println(scores[0]);
System.out.println(scores[1]);
System.out.println(scores[2]);
100
90
80

암기: 첫 번째 값은 배열명[0], 마지막 값은 배열명[배열명.length - 1]입니다.

3-2. 배열의 크기 length

배열의 크기는 배열명.length로 확인합니다. 

int[] scores = {100, 90, 80};

System.out.println(scores.length);
3

문자열의 길이는 length() 메서드를 사용하지만, 배열의 길이는 length 속성처럼 사용합니다. 

배열 길이     → scores.length
문자열 길이   → name.length()

3-3. 배열 인덱스 오류

배열 크기가 3이면 인덱스는 0, 1, 2만 사용할 수 있습니다. scores[3]은 네 번째 칸을 의미하므로 오류가 발생합니다. 

int[] scores = {100, 90, 80};

System.out.println(scores[3]);
scores[0] → 가능
scores[1] → 가능
scores[2] → 가능
scores[3] → 오류

이런 오류는 ArrayIndexOutOfBoundsException으로 나타납니다.

4. 배열과 반복문

4-1. 배열은 반복문과 함께 사용할 때 강력하다

배열은 같은 자료형 데이터를 순서대로 저장하기 때문에 반복문과 잘 어울립니다. 배열 전체를 출력하거나 합계를 구할 때 반복문을 사용합니다. 

int[] scores = {100, 90, 80};

for (int i = 0; i < scores.length; i++) {
    System.out.println(scores[i]);
}
i = 0 → scores[0] 출력
i = 1 → scores[1] 출력
i = 2 → scores[2] 출력
i = 3 → i < scores.length false → 반복 종료

4-2. 배열 반복문의 기본 조건

배열을 처음부터 끝까지 반복할 때는 보통 아래 조건을 사용합니다. 

for (int i = 0; i < 배열명.length; i++) {
    // 배열명[i] 사용
}

i <= 배열명.length로 작성하면 마지막에 배열 범위를 벗어나게 됩니다. 

for (int i = 0; i <= scores.length; i++) {
    System.out.println(scores[i]);
}

4-3. 합계와 평균 구하기

배열은 합계와 평균 계산에 자주 사용됩니다. 이 예제에는 변수, 자료형, 형변환, 연산자, 제어문, 배열 개념이 함께 들어 있습니다. 

int[] scores = {100, 90, 80};

int sum = 0;

for (int i = 0; i < scores.length; i++) {
    sum += scores[i];
}

double average = (double) sum / scores.length;

System.out.println("합계: " + sum);
System.out.println("평균: " + average);
합계: 270
평균: 90.0

5. 2차원 배열

5-1. 2차원 배열이란?

2차원 배열은 배열 안에 배열이 들어 있는 구조입니다. 행과 열을 가진 표 형태로 이해하면 쉽습니다. 

int[][] data = new int[2][3];
2행 3열 배열

      0열  1열  2열
0행    □    □    □
1행    □    □    □

교안에서도 data = new int[2][3]을 2행 3열의 2차원 배열로 설명합니다.

5-2. 2차원 배열 초기화

int[][] data = new int[][] {
    {10, 20, 30},
    {40, 50, 60}
};

더 간단하게는 아래처럼 작성할 수 있습니다.

int[][] data = {
    {10, 20, 30},
    {40, 50, 60}
};
       0열  1열  2열
0행    10   20   30
1행    40   50   60

data[0][0] → 10
data[0][2] → 30
data[1][1] → 50

5-3. 2차원 배열과 중첩 반복문

2차원 배열은 행과 열을 모두 순회해야 하므로 중첩 반복문을 사용합니다. 

int[][] data = {
    {10, 20, 30},
    {40, 50, 60}
};

for (int row = 0; row < data.length; row++) {
    for (int col = 0; col < data[row].length; col++) {
        System.out.print(data[row][col] + " ");
    }
    System.out.println();
}
10 20 30
40 50 60

data.length는 행의 개수, data[row].length는 해당 행의 열 개수를 의미합니다.

6. 실무 연결

6-1. 여러 점수 저장

int[] scores = {90, 80, 70, 100};

점수, 순위, 통계값처럼 같은 자료형의 값이 여러 개 있을 때 배열로 묶을 수 있습니다.

6-2. 문자열 분리 결과 저장

String text = "java,sql,html";

String[] subjects = text.split(",");
subjects[0] = "java"
subjects[1] = "sql"
subjects[2] = "html"

6-3. 고정된 메뉴 목록

String[] menus = {"회원가입", "로그인", "종료"};

for (int i = 0; i < menus.length; i++) {
    System.out.println((i + 1) + ". " + menus[i]);
}

6-4. 배열과 컬렉션의 연결

배열은 크기가 고정되어 있습니다. 데이터를 계속 추가하거나 삭제해야 한다면 이후에 배우는 컬렉션을 사용합니다.

구분 배열 컬렉션
크기 고정 동적 변경 가능
저장 타입 기본형/참조형 가능 객체 저장
예시 int[] scores ArrayList<Integer>

8. 자주 하는 실수

실수 1. 배열 선언만 하고 메모리 할당을 하지 않음

int[] scores;

scores[0] = 100;
int[] scores = new int[3];

scores[0] = 100;

실수 2. 인덱스가 1부터 시작한다고 착각

int[] scores = {100, 90, 80};

System.out.println(scores[1]);

위 코드는 첫 번째 값이 아니라 두 번째 값 90을 출력합니다. 첫 번째 값은 scores[0]입니다.

실수 3. 마지막 인덱스를 length로 착각

int[] scores = {100, 90, 80};

System.out.println(scores[scores.length]);
System.out.println(scores[scores.length - 1]);

실수 4. 반복문 조건에 <= 사용

for (int i = 0; i <= scores.length; i++) {
    System.out.println(scores[i]);
}
for (int i = 0; i < scores.length; i++) {
    System.out.println(scores[i]);
}

실수 5. 2차원 배열에서 열 길이를 잘못 사용

for (int row = 0; row < data.length; row++) {
    for (int col = 0; col < data.length; col++) {
        System.out.println(data[row][col]);
    }
}
for (int row = 0; row < data.length; row++) {
    for (int col = 0; col < data[row].length; col++) {
        System.out.println(data[row][col]);
    }
}

실수 6. 배열 크기를 나중에 바꿀 수 있다고 생각함

int[] numbers = new int[3];

위 배열은 크기가 3으로 고정됩니다. 더 많은 값을 저장해야 한다면 새 배열을 만들거나 컬렉션을 사용해야 합니다.

9. 면접 포인트

Q1. 배열이란 무엇인가요?

답변: 배열은 하나의 이름으로 같은 자료형의 데이터를 여러 개 저장할 수 있는 자료구조입니다. 각 요소는 인덱스로 접근하며, 인덱스는 0부터 시작합니다.

Q2. 배열의 인덱스는 몇 번부터 시작하나요?

답변: 배열의 인덱스는 0부터 시작합니다. 따라서 크기가 3인 배열의 인덱스는 0, 1, 2이고 마지막 인덱스는 length - 1입니다.

Q3. 배열의 크기는 어떻게 확인하나요?

답변: 배열의 크기는 배열명.length로 확인합니다. 예를 들어 scores.length는 scores 배열의 요소 개수를 의미합니다.

Q4. 배열과 일반 변수의 차이는 무엇인가요?

답변: 일반 변수는 하나의 이름으로 하나의 값만 저장하지만, 배열은 하나의 이름으로 같은 자료형의 값을 여러 개 저장할 수 있습니다.

Q5. 배열과 컬렉션의 차이는 무엇인가요?

답변: 배열은 크기가 고정되어 있고 기본형과 참조형을 모두 저장할 수 있습니다.
컬렉션은 크기를 동적으로 변경할 수 있지만 객체만 저장합니다. 그래서 int 같은 기본형은 컬렉션에 직접 저장할 수 없고 Integer 같은 Wrapper 클래스를 사용합니다.

Q6. 2차원 배열은 어떻게 이해하면 되나요?

답변: 2차원 배열은 행과 열을 가진 표 형태의 배열로 이해할 수 있습니다. 예를 들어 int[][] data = new int[2][3]은 2행 3열의 int 배열입니다.

Q7. 배열에서 자주 발생하는 예외는 무엇인가요?

답변: 배열에서 자주 발생하는 예외는 ArrayIndexOutOfBoundsException입니다. 배열의 사용 가능한 인덱스 범위를 벗어나 접근할 때 발생합니다.

10. 오늘의 핵심 정리

  1. 배열은 하나의 이름으로 같은 자료형 데이터를 여러 개 저장하는 구조다.
  2. 배열은 동종 데이터 저장에 사용한다.
  3. 배열 선언은 int[] scores; 형태를 추천한다.
  4. 배열은 new 자료형[크기]로 메모리 할당한다.
  5. 배열 인덱스는 0부터 시작한다.
  6. 배열의 마지막 인덱스는 length - 1이다.
  7. 배열의 크기는 배열명.length로 확인한다.
  8. 배열은 반복문과 함께 사용할 때 효율적이다.
  9. 배열은 메모리 할당 후 기본값으로 초기화된다.
  10. 2차원 배열은 행과 열 구조로 이해할 수 있다.
  11. 2차원 배열 출력에는 중첩 반복문이 자주 쓰인다.
  12. 배열은 한 번 크기를 정하면 크기를 바꿀 수 없다.

학습 메모

이 페이지는 Java 배열 단원입니다.
교안에서는 배열을 하나의 이름으로 같은 데이터 형식을 여러 개 저장하는 공간으로 설명하고,
배열 선언, 메모리 할당, 초기화, 2차원 배열, length, 기본값 초기화를 핵심으로 다룹니다.
배열은 이후 객체지향, 상속, 컬렉션, Stream을 이해하기 위한 기초 자료구조이므로 인덱스가 0부터 시작한다는 점과 마지막 인덱스가 length - 1이라는 점을 반드시 익혀야 합니다.
