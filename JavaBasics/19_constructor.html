19. 생성자

생성자는 객체가 생성될 때 가장 먼저 실행되는 특수한 메서드입니다.
이 페이지에서는 기본 생성자, 매개변수가 있는 생성자, 생성자 오버로딩,
this를 활용한 필드 초기화, 생성자와 setter의 차이를 공부합니다.

이번 페이지의 학습 목표

생성자는 객체가 처음 만들어질 때 객체의 초기 상태를 결정합니다. 생성자를 이해하면 객체를 불완전한 상태로 두지 않고, 필요한 값을 객체 생성 시점에 안전하게 넣을 수 있습니다.

생성자 개념

객체 생성 시 최초로 실행되는 특수한 메서드임을 이해합니다.

기본 생성자

생성자가 없을 때 컴파일러가 제공하는 기본 생성자의 조건을 익힙니다.

인자 생성자

객체 생성과 동시에 필드를 초기화하는 방법을 익힙니다.

생성자 오버로딩

여러 생성자로 다양한 방식의 객체 생성을 제공하는 구조를 이해합니다.

1. 생성자의 기본 개념

1-1. 생성자란?

생성자는 객체가 생성될 때 가장 먼저 실행되는 특수한 메서드입니다. 일반적으로 객체의 멤버 변수를 초기화하는 역할을 합니다. 

Account account = new Account();
new Account()
→ Account 객체 생성
→ Account 생성자 실행
→ 필드 초기화
→ account 참조변수가 객체를 가리킴

class Account {

    public Account() {
        System.out.println("Account 생성자 실행");
    }

}

1-2. 생성자의 특징

  1. 생성자 이름은 클래스명과 동일해야 합니다.
  2. 생성자는 반환 타입을 가지면 안 됩니다.
  3. 객체 생성 시 new와 함께 자동 호출됩니다.
  4. 주된 역할은 멤버 변수를 초기화하는 것입니다.
  5. 접근 지정자는 보통 public을 사용합니다.
public class Account {

    public Account() {
        System.out.println("생성자 실행");
    }

}

1-3. 생성자는 반환 타입이 없다

생성자는 반환 타입을 쓰지 않습니다. void를 붙이면 생성자가 아니라 일반 메서드가 됩니다. 

public Account() {
}
public void Account() {
}
Account()
→ 생성자

void Account()
→ 생성자가 아니라 Account라는 이름의 일반 메서드

1-4. 생성자의 역할

생성자의 가장 중요한 역할은 객체가 처음 만들어질 때 필드를 초기화하는 것입니다. 

class Account {

    String name;
    String accountNo;
    int balance;

    public Account(String name, String accountNo, int balance) {
        this.name = name;
        this.accountNo = accountNo;
        this.balance = balance;
    }

}
Account account = new Account("홍길동", "123-456", 10000);
name      → 홍길동
accountNo → 123-456
balance   → 10000

2. 기본 생성자

2-1. 기본 생성자란?

기본 생성자는 매개변수가 없는 생성자입니다. 아무 인자 없이 객체를 만들 수 있게 해줍니다. 

public Account() {
}
Account account = new Account();

2-2. 생성자가 없으면 컴파일러가 기본 생성자를 제공한다

클래스 안에 생성자를 하나도 작성하지 않으면, 컴파일러가 기본 생성자를 자동으로 제공한 것처럼 처리합니다. 

class Account {

    String name;
    int balance;

}

위 클래스에는 생성자가 보이지 않지만 아래 코드가 가능합니다. 

Account account = new Account();
class Account {
    String name;
    int balance;
}

생성자를 하나도 작성하지 않음
        ↓
컴파일러가 기본 생성자 자동 제공
        ↓
new Account() 가능

2-3. 생성자를 하나라도 만들면 기본 생성자는 자동 제공되지 않는다

매개변수가 있는 생성자를 하나라도 직접 만들면, 컴파일러는 기본 생성자를 자동으로 제공하지 않습니다. 

class Account {

    String name;
    int balance;

    public Account(String name, int balance) {
        this.name = name;
        this.balance = balance;
    }

}
Account account = new Account(); // 오류
Account account = new Account("홍길동", 10000);

기본 생성자도 필요하다면 직접 작성해야 합니다. 

class Account {

    String name;
    int balance;

    public Account() {
    }

    public Account(String name, int balance) {
        this.name = name;
        this.balance = balance;
    }

}

3. 매개변수가 있는 생성자

3-1. 인자 생성자란?

매개변수가 있는 생성자는 객체를 만들면서 필요한 값을 전달받아 필드를 초기화합니다. 

class User {

    private String loginId;
    private String email;

    public User(String loginId, String email) {
        this.loginId = loginId;
        this.email = email;
    }

}
User user = new User("test01", "test01@test.com");

장점: 객체 생성 시점에 필수값을 강제로 넣을 수 있어 객체가 불완전한 상태로 만들어지는 것을 줄일 수 있습니다.

3-2. 생성자에서 this 사용

생성자에서도 필드명과 매개변수명이 같으면 this를 사용해 구분합니다. 

public Account(String name, String accountNo, int balance) {
    this.name = name;
    this.accountNo = accountNo;
    this.balance = balance;
}
this.name
→ 현재 객체의 name 필드

name
→ 생성자로 전달받은 name 매개변수

3-3. 생성자에서 검증하기

생성자는 객체가 처음 만들어지는 순간 실행되므로, 잘못된 초기값을 막기에 좋습니다. 

class Account {

    private String name;
    private int balance;

    public Account(String name, int balance) {
        this.name = name;

        if (balance >= 0) {
            this.balance = balance;
        } else {
            this.balance = 0;
        }
    }

}
Account account = new Account("홍길동", -10000);
전달값 balance = -10000
        ↓
생성자 검증
        ↓
필드 balance = 0

4. 생성자 오버로딩

4-1. 생성자 오버로딩이란?

생성자 오버로딩은 같은 클래스 안에서 매개변수의 개수, 타입, 순서를 다르게 하여 여러 생성자를 정의하는 것입니다. 이를 통해 객체를 다양한 방식으로 생성할 수 있습니다. 

class Account {

    private String name;
    private String accountNo;
    private int balance;

    public Account() {
    }

    public Account(String name) {
        this.name = name;
    }

    public Account(String name, String accountNo) {
        this.name = name;
        this.accountNo = accountNo;
    }

    public Account(String name, String accountNo, int balance) {
        this.name = name;
        this.accountNo = accountNo;
        this.balance = balance;
    }

}
Account a1 = new Account();
Account a2 = new Account("홍길동");
Account a3 = new Account("홍길동", "123-456");
Account a4 = new Account("홍길동", "123-456", 10000);

4-2. 오버로딩 조건

생성자 이름은 모두 클래스명으로 같기 때문에, 오버로딩은 매개변수의 개수, 타입, 순서가 달라야 성립합니다. 

public Account(String name) {
}

public Account(String accountNo) {
}

위 두 생성자는 매개변수 이름만 다르고 타입과 개수가 같으므로 오버로딩이 아닙니다. 

public Account(String name) {
}

public Account(String name, int balance) {
}
public Account(String name) {
}

public Account(int balance) {
}

4-3. this()와 super() 맛보기

생성자 오버로딩이 많아지면 코드 중복이 생길 수 있습니다. 이때 같은 클래스의 다른 생성자를 호출하는 this()를 사용할 수 있습니다.
부모 생성자를 호출할 때는 super()를 사용합니다. 

this()
→ 같은 클래스의 다른 생성자 호출

super()
→ 부모 클래스의 생성자 호출

주의:
this()와 super()는 생성자 안에서만 호출 가능
반드시 생성자의 첫 번째 문장이어야 함
생성자 안에서 this()와 super()를 함께 쓸 수 없음

이번 페이지에서는 생성자 자체를 먼저 이해하고, 이후 상속 단원에서 super()를 더 자세히 연결해서 보면 됩니다.

5. 생성자와 setter의 차이

5-1. 생성자와 setter 비교

구분 생성자 setter
실행 시점 객체 생성 시 1번 실행 객체 생성 후 필요할 때 여러 번 호출 가능
목적 초기값 설정 값 변경
호출 방식 new 클래스명(...) 객체명.set필드명(...)
필수값 강제 가능 상대적으로 약함

5-2. 생성자 방식

User user = new User("test01", "test@test.com");

객체를 만들 때 반드시 loginId와 email을 넣어야 합니다.

5-3. setter 방식

User user = new User();

user.setLoginId("test01");
user.setEmail("test@test.com");

setter를 호출하지 않으면 객체가 불완전한 상태로 남을 수 있습니다.

6. 객체 생성 흐름

6-1. 객체 생성과 생성자 호출 흐름

Account account = new Account("홍길동", "123-456", 10000);
1. new 키워드 실행
2. Account 객체 메모리 생성
3. 필드 기본값 초기화
name = null

balance = 0
4. 생성자 Account(String, String, int) 실행
5. 전달받은 값으로 필드 초기화

accountNo = "123-456"
balance = 10000
6. 생성된 객체의 참조값이 account에 저장

6-2. 생성 전후 상태 비교

new Account("홍길동", "123-456", 10000)

객체 생성 직후 기본값:
name = null
accountNo = null
balance = 0

생성자 실행 후:
name = "홍길동"
accountNo = "123-456"
balance = 10000

7-4. 생성자 오버로딩 예제

class Account {

    private String name;
    private String accountNo;
    private int balance;

    public Account() {
        System.out.println("기본 생성자");
    }

    public Account(String name) {
        this.name = name;
    }

    public Account(String name, String accountNo) {
        this.name = name;
        this.accountNo = accountNo;
    }

    public Account(String name, String accountNo, int balance) {
        this.name = name;
        this.accountNo = accountNo;

        if (balance >= 0) {
            this.balance = balance;
        }
    }

    public void printAccountInfo() {
        System.out.println("예금주: " + name);
        System.out.println("계좌번호: " + accountNo);
        System.out.println("잔고: " + balance);
    }

}
public class AccountMain {

    public static void main(String[] args) {

        Account a1 = new Account();
        Account a2 = new Account("홍길동");
        Account a3 = new Account("김철수", "111-222");
        Account a4 = new Account("이영희", "333-444", 50000);

        a4.printAccountInfo();

    }

}
기본 생성자
예금주: 이영희
계좌번호: 333-444
잔고: 50000

7-5. 생성자와 setter 함께 사용

class Account {

    private String name;
    private int balance;

    public Account(String name, int balance) {
        this.name = name;
        setBalance(balance);
    }

    public void setBalance(int balance) {
        if (balance >= 0) {
            this.balance = balance;
        }
    }

    public void printInfo() {
        System.out.println("예금주: " + name);
        System.out.println("잔고: " + balance);
    }

}

생성자 안에서 setter를 호출해 검증 로직을 재사용하고 있습니다.

10. 면접 포인트

Q1. 생성자란 무엇인가요?

답변: 생성자는 객체가 생성될 때 최초로 실행되는 특수한 메서드입니다. 주로 객체의 멤버 변수를 초기화하는 역할을 합니다.

Q2. 생성자의 특징은 무엇인가요?

답변: 생성자 이름은 클래스명과 같아야 하며, 반환 타입을 가지면 안 됩니다. 객체 생성 시 new와 함께 자동으로 호출되고, 주로 멤버 변수를 초기화하는 데 사용됩니다.

Q3. 기본 생성자란 무엇인가요?

답변: 기본 생성자는 매개변수가 없는 생성자입니다. 클래스에 생성자를 하나도 작성하지 않으면 컴파일러가 기본 생성자를 자동으로 제공합니다.

Q4. 인자 생성자를 만들면 기본 생성자는 자동으로 생기나요?

답변: 자동으로 생기지 않습니다. 생성자를 하나라도 직접 작성하면 컴파일러는 기본 생성자를 제공하지 않습니다. 따라서 기본 생성자가 필요하면 직접 작성해야 합니다.

Q5. 생성자 오버로딩이란 무엇인가요?

답변: 생성자 오버로딩은 같은 클래스 안에서 매개변수의 개수, 타입, 순서를 다르게 하여 여러 개의 생성자를 정의하는 것입니다.
이를 통해 다양한 방식으로 객체를 초기화할 수 있습니다.

Q6. 생성자와 메서드의 차이는 무엇인가요?

답변: 생성자는 클래스명과 이름이 같고 반환 타입이 없으며, 객체 생성 시 자동 호출됩니다. 일반 메서드는 이름이 자유롭고 반환 타입이 있으며, 객체 생성 후 직접 호출해야 합니다.

Q7. 생성자와 setter의 차이는 무엇인가요?

답변: 생성자는 객체 생성 시 초기값을 설정하기 위해 한 번 실행됩니다. setter는 객체 생성 후 값을 변경하기 위해 필요할 때 호출합니다.
생성자는 필수값을 강제하는 데 유리하고, setter는 변경 가능한 값을 수정하는 데 사용됩니다.

Q8. 생성자에서 this를 사용하는 이유는 무엇인가요?

답변: 생성자의 매개변수명과 필드명이 같을 때 둘을 구분하기 위해 this를 사용합니다. this.name은 현재 객체의 필드이고, name은 생성자로 전달받은 매개변수입니다.

11. 오늘의 핵심 정리

  1. 생성자는 객체가 생성될 때 최초로 실행되는 특수한 메서드다.
  2. 생성자 이름은 클래스명과 같아야 한다.
  3. 생성자는 반환 타입을 가지면 안 된다.
  4. 생성자의 접근 지정자는 주로 public을 사용한다.
  5. 생성자의 주된 역할은 멤버 변수를 초기화하는 것이다.
  6. 생성자를 하나도 만들지 않으면 컴파일러가 기본 생성자를 제공한다.
  7. 생성자를 하나라도 만들면 컴파일러가 기본 생성자를 제공하지 않는다.
  8. 기본 생성자가 필요하면 직접 작성해야 한다.
  9. 매개변수가 있는 생성자는 객체 생성 시 초기값을 전달받을 수 있다.
  10. 생성자도 오버로딩할 수 있다.
  11. 생성자에서는 this를 사용해 필드와 매개변수를 구분할 수 있다.
  12. 생성자는 객체가 처음부터 올바른 상태로 만들어지도록 돕는다.
  13. 생성자와 setter는 모두 값을 넣을 수 있지만 실행 시점과 목적이 다르다.
  14. this()는 같은 클래스의 다른 생성자를 호출하고, super()는 부모 생성자를 호출한다.

생성자는 객체가 생성될 때 최초로 실행되는 특수한 메서드이며, 클래스명과 이름이 같고 반환 타입을 가지지 않습니다.
생성자를 하나도 작성하지 않으면 컴파일러가 기본 생성자를 제공하지만, 생성자를 하나라도 직접 작성하면 기본 생성자는 자동 제공되지 않습니다.
생성자는 객체의 초기 상태를 안전하게 만드는 핵심 문법입니다.
