29. 내부 클래스

내부 클래스는 클래스 안에 선언된 클래스입니다.
외부 클래스와 밀접한 보조 클래스를 함께 묶어 관리할 때 사용하며,
내부 멤버 클래스, 내부 정적 클래스, 내부 로컬 클래스, 내부 익명 클래스로 나누어 공부합니다.

이번 페이지의 학습 목표

내부 클래스는 외부 클래스와 강하게 연결된 클래스를 외부 클래스 안에 넣어 관리하는 문법입니다.
특히 외부 클래스의 멤버 접근, 내부 클래스의 생성 방식, 익명 클래스의 의미를 정확히 구분하는 것이 중요합니다.

내부 클래스 개념

클래스 안에 클래스를 선언하는 이유를 이해합니다.

4가지 종류

멤버, 정적, 로컬, 익명 클래스를 구분합니다.

객체 생성 방식

각 내부 클래스의 객체 생성 방법을 익힙니다.

익명 클래스

인터페이스를 즉석에서 구현하는 구조를 이해합니다.

1. 내부 클래스 기본 개념

1-1. 내부 클래스란?

내부 클래스는 클래스 안에 선언된 클래스입니다. 외부 클래스와 밀접하게 관련된 보조 클래스를 외부 클래스 내부에 묶어 관리할 수 있습니다. 

class Outer {

    class Inner {

    }

}
Outer
→ 외부 클래스

Inner
→ Outer 안에 정의된 내부 클래스

내부 클래스
→ Inner Class
→ Nested Class
→ 클래스 안의 클래스

1-2. 내부 클래스를 사용하는 이유

  1. 외부 클래스와 매우 밀접한 클래스를 내부에 숨길 수 있습니다.
  2. 외부 클래스의 멤버를 편하게 사용할 수 있습니다.
  3. 특정 메서드 안에서만 필요한 클래스를 만들 수 있습니다.
  4. 한 번만 사용할 구현 클래스를 즉석에서 만들 수 있습니다.
  5. GUI 이벤트 처리나 Runnable 같은 일회성 구현체에 사용할 수 있습니다.

1-3. 자동차와 엔진 예시

class Car {

    private String model = "Sonata";

    class Engine {

        public void start() {
            System.out.println(model + " 엔진 시작");
        }

    }

}

Engine은 Car와 강하게 연결된 보조 클래스입니다. 이런 경우 Car 안에 Engine을 선언해 의미를 더 명확하게 표현할 수 있습니다.

1-4. 내부 클래스 구조 그림

Outer 클래스

┌─────────────────────────────┐
│ Outer                       │
│                             │
│  outerValue                 │
│  outerMethod()              │
│                             │
│  ┌───────────────────────┐  │
│  │ Inner                 │  │
│  │ innerValue            │  │
│  │ innerMethod()         │  │
│  └───────────────────────┘  │
└─────────────────────────────┘

2. 내부 클래스의 종류

2-1. 네 가지 내부 클래스

종류 선언 위치 이름 객체 생성 특징
내부 멤버 클래스 클래스 안 있음 외부 객체를 먼저 만든 뒤 생성
내부 정적 클래스 클래스 안 있음 외부 객체 없이 생성 가능
내부 로컬 클래스 메서드 안 있음 메서드 안에서만 생성 가능
내부 익명 클래스 메서드 안 또는 식 안 없음 선언과 동시에 객체 생성 
class Outer {

    class MemberInner {
    }

    static class StaticInner {
    }

    public void method() {

        class LocalInner {
        }

        Runnable r = new Runnable() {
            @Override
            public void run() {
            }
        };
    }
}

3. 내부 멤버 클래스

3-1. 내부 멤버 클래스란?

내부 멤버 클래스는 클래스 안에 선언된 일반 내부 클래스입니다. static이 붙지 않으며, 외부 클래스의 객체를 기준으로 생성됩니다. 

class Outer {

    class Inner {

    }

}

내부 멤버 클래스는 외부 클래스의 인스턴스 멤버와 static 멤버에 접근할 수 있습니다.

3-2. 내부 멤버 클래스 객체 생성

class Outer {

    class Inner {
        public void print() {
            System.out.println("내부 멤버 클래스");
        }
    }

}
Outer outer = new Outer();
Outer.Inner inner = outer.new Inner();

inner.print();
내부 멤버 클래스
생성 순서:

1. Outer 객체 생성
2. Outer 객체를 기준으로 Inner 객체 생성

공식:
Outer outer = new Outer();
Outer.Inner inner = outer.new Inner();

3-3. 외부 멤버 접근

class Outer {

    private int instanceValue = 10;
    private static int staticValue = 20;

    class Inner {

        public void print() {
            System.out.println(instanceValue);
            System.out.println(staticValue);
        }

    }

}

일반 내부 클래스는 외부 객체와 연결되어 생성되므로, 외부 클래스의 인스턴스 멤버를 사용할 수 있습니다.

4. 내부 정적 클래스

4-1. 내부 정적 클래스란?

내부 정적 클래스는 클래스 안에 static으로 선언된 클래스입니다. 외부 클래스 객체를 만들지 않고도 생성할 수 있습니다. 

class Outer {

    static class StaticInner {

    }

}

4-2. 내부 정적 클래스 객체 생성

class Outer {

    static class StaticInner {
        public void print() {
            System.out.println("내부 정적 클래스");
        }
    }

}
Outer.StaticInner inner = new Outer.StaticInner();

inner.print();
내부 정적 클래스
내부 정적 클래스 생성 공식:

Outer.StaticInner inner = new Outer.StaticInner();

특징:
Outer 객체 생성 필요 없음

4-3. 외부 인스턴스 멤버에는 직접 접근 불가

class Outer {

    private int instanceValue = 10;
    private static int staticValue = 20;

    static class StaticInner {

        public void print() {
            // System.out.println(instanceValue); // 오류
            System.out.println(staticValue);      // 가능
        }

    }

}

내부 정적 클래스는 외부 객체 없이 생성될 수 있기 때문에, 특정 외부 객체에 속한 인스턴스 멤버에 직접 접근할 수 없습니다.

5. 내부 로컬 클래스

5-1. 내부 로컬 클래스란?

내부 로컬 클래스는 메서드 안에 선언된 이름 있는 클래스입니다. 지역변수처럼 해당 메서드 안에서만 사용할 수 있습니다. 

class Outer {

    public void method() {

        class LocalInner {

        }

    }

}

5-2. 내부 로컬 클래스 예제

class Outer {

    public void method() {

        class LocalInner {

            public void print() {
                System.out.println("내부 로컬 클래스");
            }

        }

        LocalInner inner = new LocalInner();
        inner.print();
    }

}
Outer outer = new Outer();
outer.method();
내부 로컬 클래스
로컬 클래스 선언 위치
→ 메서드 내부

사용 가능 범위
→ 해당 메서드 내부

객체 생성 위치
→ 해당 메서드 내부

6. 내부 익명 클래스

6-1. 내부 익명 클래스란?

내부 익명 클래스는 이름이 없는 클래스입니다. 보통 한 번만 사용할 구현체를 만들 때 사용합니다. 

new 부모클래스또는인터페이스() {
    // 오버라이딩
};

익명 클래스는 클래스 상속 또는 인터페이스 구현을 선언과 동시에 처리합니다.

6-2. 클래스를 상속받는 익명 클래스

class Parent {

    public void printA() {
        System.out.println("A출력");
    }

}
Parent p = new Parent() {

    @Override
    public void printA() {
        System.out.println("오버라이딩");
    }

};

p.printA();
오버라이딩
new Parent()
→ Parent 객체를 그냥 만든 것이 아님

실제 의미:
Parent를 상속받은 이름 없는 자식 클래스 객체 생성

6-3. 인터페이스를 구현하는 익명 클래스

interface MyInter {
    void printNumber();
}

public class AnonymousInterfaceMain {

    public static void main(String[] args) {

        MyInter mi = new MyInter() {

            @Override
            public void printNumber() {
                System.out.println("익명 클래스");
            }

        };

        mi.printNumber();
    }

}
익명 클래스

이것은 인터페이스를 직접 객체 생성한 것이 아닙니다. MyInter를 구현한 이름 없는 클래스의 객체를 만든 것입니다.

6-4. 익명 클래스에서 새 메서드를 추가하면?

Parent p = new Parent() {

    public void printNew() {
        System.out.println("새 메서드");
    }

};

p.printNew(); // 오류
참조변수 타입:
Parent

호출 가능한 것:
Parent에 원래 있던 메서드
오버라이딩된 메서드

호출 불가능:
익명 클래스에서 새로 추가한 고유 메서드

7. 내부 클래스 객체 생성 방법 정리

종류 객체 생성 방법
내부 멤버 클래스 Outer outer = new Outer();
Outer.Inner inner = outer.new Inner();
내부 정적 클래스 Outer.StaticInner inner = new Outer.StaticInner();
내부 로컬 클래스 메서드 안에서 LocalInner inner = new LocalInner();
내부 익명 클래스 Parent p = new Parent() { ... };
MyInter mi = new MyInter() { ... };

8. 컴파일 결과 파일명

8-1. 내부 클래스도 class 파일로 컴파일된다

종류 컴파일 결과 파일명
내부 멤버 클래스 Outer$Inner.class
내부 정적 클래스 Outer1$Inner1.class
내부 로컬 클래스 Outer3$1Inner3.class
내부 익명 클래스 Outer4$1.class

내부 클래스는 코드상으로는 클래스 안에 있지만, 컴파일 후에는 별도의 .class 파일로 만들어집니다.

9. 실무 연결

9-1. GUI 이벤트 처리

button.addActionListener(new ActionListener() {

    @Override
    public void actionPerformed(ActionEvent e) {
        System.out.println("버튼 클릭");
    }

});

한 번만 사용할 이벤트 처리 클래스를 따로 파일로 만들지 않아도 됩니다.

9-2. Runnable 구현

Runnable task = new Runnable() {

    @Override
    public void run() {
        System.out.println("작업 실행");
    }

};

Thread thread = new Thread(task);
thread.start();

나중에 람다식을 배우면 같은 코드를 더 짧게 표현할 수 있습니다. 

Runnable task = () -> System.out.println("작업 실행");

9-3. 일회성 구현체

interface Printer {
    void print();
}

public class Main {

    public static void main(String[] args) {

        Printer printer = new Printer() {

            @Override
            public void print() {
                System.out.println("일회성 출력");
            }
        };

        printer.print();
    }
}

별도의 PrinterImpl 클래스를 만들지 않고, 필요한 위치에서 즉석으로 구현할 수 있습니다.

10. 자주 하는 실수

실수 1. 내부 멤버 클래스를 외부 객체 없이 생성함

Outer.Inner inner = new Outer.Inner(); // 오류
Outer outer = new Outer();
Outer.Inner inner = outer.new Inner();

실수 2. 내부 정적 클래스에서 외부 인스턴스 멤버에 직접 접근함

class Outer {

    int value = 10;

    static class Inner {
        public void print() {
            System.out.println(value); // 오류
        }
    }
}

내부 정적 클래스는 외부 객체 없이 생성될 수 있으므로, 특정 외부 객체에 속한 인스턴스 변수에 직접 접근할 수 없습니다.

실수 3. 로컬 클래스를 메서드 밖에서 사용함

class Outer {

    public void method() {
        class LocalInner {
        }
    }

    public void otherMethod() {
        LocalInner inner = new LocalInner(); // 오류
    }
}

실수 4. 익명 클래스에서 추가한 새 메서드를 호출하려 함

Parent p = new Parent() {

    public void newMethod() {
        System.out.println("새 메서드");
    }

};

p.newMethod(); // 오류

실수 5. 인터페이스를 직접 new 했다고 오해함

MyInter mi = new MyInter() {
    @Override
    public void printNumber() {
        System.out.println("익명 클래스");
    }
};

이것은 인터페이스를 직접 객체 생성한 것이 아니라, 인터페이스를 구현한 이름 없는 클래스 객체를 만든 것입니다.

11. 면접 포인트

Q1. 내부 클래스란 무엇인가요?

답변: 내부 클래스는 클래스 안에 선언된 클래스입니다. 외부 클래스와 밀접한 보조 클래스를 내부에 묶어 관리할 때 사용합니다.

Q2. 내부 클래스의 종류는 무엇이 있나요?

답변: 내부 멤버 클래스, 내부 정적 클래스, 내부 로컬 클래스, 내부 익명 클래스가 있습니다.

Q3. 내부 멤버 클래스 객체는 어떻게 생성하나요?

답변: 먼저 외부 클래스 객체를 생성한 뒤, 그 외부 객체를 이용해 내부 클래스 객체를 생성합니다. Outer outer = new Outer(); Outer.Inner inner = outer.new Inner();

Q4. 내부 정적 클래스는 어떻게 생성하나요?

답변: 외부 객체를 만들 필요 없이 외부 클래스명으로 생성합니다. Outer.StaticInner inner = new Outer.StaticInner();

Q5. 내부 정적 클래스가 외부 인스턴스 멤버에 직접 접근할 수 없는 이유는 무엇인가요?

답변: 내부 정적 클래스는 외부 클래스 객체 없이도 생성될 수 있습니다. 따라서 특정 외부 객체에 속한 인스턴스 멤버에 직접 접근할 수 없습니다.

Q6. 내부 로컬 클래스란 무엇인가요?

답변: 내부 로컬 클래스는 메서드 안에 선언된 이름 있는 클래스입니다. 지역변수처럼 해당 메서드 안에서만 사용할 수 있습니다.

Q7. 익명 클래스란 무엇인가요?

답변: 익명 클래스는 이름이 없는 클래스입니다. 상위 클래스를 상속하거나 인터페이스를 구현하는 일회성 객체를 만들 때 사용합니다.

Q8. 인터페이스를 new 하는 익명 클래스 코드는 무슨 의미인가요?

답변: 인터페이스를 직접 객체 생성하는 것이 아닙니다. 그 인터페이스를 구현한 이름 없는 클래스를 즉석에서 만들고, 그 객체를 인터페이스 타입 참조변수에 저장하는 것입니다.

Q9. 익명 클래스에서 새 메서드를 추가하면 호출할 수 있나요?

답변: 참조변수 타입에 따라 다릅니다. 부모 클래스나 인터페이스 타입으로 참조하고 있다면, 그 타입에 선언되지 않은 새 메서드는 호출할 수 없습니다.

Q10. 내부 클래스는 언제 사용하나요?

답변: 외부 클래스와 강하게 관련된 보조 클래스를 숨기고 싶을 때, 특정 메서드 안에서만 필요한 클래스를 만들 때,
GUI 이벤트 처리나 Runnable처럼 일회성 구현체가 필요할 때 사용합니다.

12. 오늘의 핵심 정리

  1. 내부 클래스는 클래스 안에 선언된 클래스다.
  2. 내부 클래스는 Inner Class 또는 Nested Class라고도 한다.
  3. 일반 내부 클래스는 외부 클래스의 멤버를 자유롭게 사용할 수 있다.
  4. 내부 정적 클래스는 외부 클래스의 non-static 멤버를 직접 사용할 수 없다.
  5. 내부 멤버 클래스는 외부 객체를 먼저 만든 뒤 생성한다.
  6. 내부 정적 클래스는 외부 객체 없이 외부클래스명.내부클래스명으로 생성한다.
  7. 내부 로컬 클래스는 메서드 안에 선언된 이름 있는 클래스다.
  8. 내부 로컬 클래스는 메서드 안에서만 사용할 수 있다.
  9. 내부 익명 클래스는 이름 없는 클래스다.
  10. 익명 클래스는 상속 또는 인터페이스 구현을 즉석에서 처리할 때 사용한다.
  11. 익명 클래스는 GUI 이벤트 처리에서 자주 사용된다.
  12. 인터페이스의 익명 클래스는 인터페이스를 직접 new 하는 것이 아니라, 인터페이스를 구현한 이름 없는 클래스를 생성하는 것이다.
  13. 내부 멤버 클래스의 class 파일명은 Outer$Inner.class 형태다.
  14. 내부 로컬 클래스는 Outer$1Inner.class 형태로 컴파일된다.
  15. 내부 익명 클래스는 Outer$1.class 형태로 컴파일된다.

학습 메모

이 페이지는 Java 내부 클래스 단원입니다.
내부 클래스는 클래스 안에 선언된 클래스이며, 외부 클래스와 밀접한 보조 클래스를 함께 관리할 때 사용합니다.
내부 멤버 클래스는 외부 객체를 먼저 만든 뒤 생성하고, 내부 정적 클래스는 외부 객체 없이 생성할 수 있습니다.
내부 로컬 클래스는 메서드 안에서만 사용할 수 있으며, 내부 익명 클래스는 이름 없는 일회성 구현체를 만들 때 사용합니다.
