33. 정규표현식 Regular Expression

정규표현식은 문자열이 특정한 패턴을 만족하는지 검사하거나,
문자열 안에서 필요한 데이터를 찾아 추출하고 치환할 때 사용하는 표현식입니다.
Pattern, Matcher, matches(), find(), group(), replaceAll()을 중심으로 공부합니다.

이번 페이지의 학습 목표

정규표현식은 회원가입, 로그인, 검색, 관리자 필터링, 로그 분석처럼 문자열 형식을 검사하거나 특정 값을 추출해야 하는 상황에서 자주 사용됩니다.
이번 단원에서는 정규표현식 기호를 단순 암기하는 것이 아니라, Java 코드에서 실제로 어떻게 읽고 적용하는지 이해하는 것을 목표로 합니다.

Regex 개념

문자열이 특정 패턴을 만족하는지 검사하는 원리를 이해합니다.

기본 문법

., *, +, ?, ^, $, [], {}를 익힙니다.

Java 적용

Pattern, Matcher, matches(), find(), group()을 사용합니다.

실무 검증

아이디, 이메일, 휴대폰 번호, 비밀번호, 한글 이름 검증 예제를 익힙니다.

1. 정규표현식 기본 개념

1-1. 정규표현식이란?

정규표현식, 또는 Regex는 문자열이 특정 규칙을 만족하는지 검사하기 위한 문자열 패턴 표현 방법입니다.
예를 들어 사용자가 입력한 값이 숫자인지, 이메일 형식인지, 휴대폰 번호 형식인지 확인할 때 사용할 수 있습니다. 

사용자 입력값
    ↓
정규표현식 패턴 검사
    ↓
규칙에 맞으면 true
규칙에 맞지 않으면 false

예를 들어 "^\\d+$"는 처음부터 끝까지 숫자만 있는 문자열인지 검사하는 정규표현식입니다. 

String input = "12345";

boolean result = input.matches("^\\d+$");

System.out.println(result);
true

1-2. 정규표현식이 필요한 이유

단순히 문자열에 @가 포함되어 있는지만 확인하면 이메일 검증이 정확하지 않습니다. 예를 들어 "abc@", "@naver.com", "abc@@naver.com"처럼 잘못된 값도 통과할 수 있습니다. 

String email = "abc@";

if (email.contains("@")) {
    System.out.println("이메일처럼 보임");
}

그래서 문자열의 구조 전체를 검사할 수 있는 정규표현식이 필요합니다. 

String email = "test@naver.com";
String regex = "^[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\\.[A-Za-z]{2,}$";

System.out.println(email.matches(regex));

1-3. 실무에서 자주 쓰이는 곳

사용 위치 정규표현식 활용 예시
회원가입 아이디, 비밀번호, 이메일, 휴대폰 번호 형식 검사
로그인 아이디 또는 이메일 입력 형식 확인
관리자 검색 전화번호에서 하이픈 제거, 숫자만 추출
게시판 검색 특정 키워드 패턴 검색
로그 분석 로그 문자열에서 사용자 ID, 상태 코드, 에러 코드 추출
데이터 정리 여러 공백을 하나로 줄이거나 특수문자 제거

2. Java에서 정규표현식 사용하기

2-1. String.matches() 사용

가장 간단한 정규표현식 검사는 String.matches()를 사용하는 방식입니다. 이 메서드는 문자열 전체가 정규표현식과 일치하면 true, 아니면 false를 반환합니다. 

public class RegexMatchesExample {

    public static void main(String[] args) {

        String input = "12345";
        String regex = "^\\d+$";

        boolean result = input.matches(regex);

        System.out.println(result);
    }
}
true

matches()는 부분 검색이 아니라 전체 문자열 검사입니다. 그래서 회원가입 입력값 검증처럼 전체 형식을 검사할 때 자주 사용합니다.

2-2. Pattern과 Matcher 사용

정규표현식을 더 세밀하게 사용하려면 Pattern과 Matcher 클래스를 사용합니다. 

import java.util.regex.Pattern;
import java.util.regex.Matcher;

public class RegexPatternExample {

    public static void main(String[] args) {

        Pattern pattern = Pattern.compile("^\\d{13}$");
        Matcher matcher = pattern.matcher("1111112222222");

        if (matcher.matches()) {
            System.out.println("패턴 일치");
        } else {
            System.out.println("패턴 불일치");
        }
    }
}
패턴 일치
Pattern
→ 정규표현식 규칙을 준비한다.

Matcher
→ 준비된 규칙을 실제 문자열에 적용한다.

matches()
→ 전체 문자열이 패턴과 일치하는지 검사한다.

2-3. Pattern과 Matcher 역할 비교

클래스 / 메서드 역할
Pattern 정규표현식 패턴 자체를 컴파일하여 준비합니다.
Pattern.compile() 문자열로 작성한 정규표현식을 Pattern 객체로 만듭니다.
Matcher 패턴을 검사 대상 문자열에 적용합니다.
matcher.matches() 전체 문자열이 패턴과 일치하는지 검사합니다.
matcher.find() 문자열 안에서 패턴과 일치하는 부분을 찾습니다.
matcher.group() 찾은 문자열 또는 그룹으로 묶은 값을 가져옵니다.

3. matches(), find(), group() 차이

3-1. matches()는 전체 일치 검사

import java.util.regex.Pattern;
import java.util.regex.Matcher;

public class MatchesExample {

    public static void main(String[] args) {

        Pattern pattern = Pattern.compile("\\d+");
        Matcher matcher = pattern.matcher("12345");

        System.out.println(matcher.matches());
    }
}
true

matches()는 검사 대상 문자열 전체가 정규표현식과 일치해야 합니다. 

Pattern pattern = Pattern.compile("\\d+");
Matcher matcher = pattern.matcher("abc123def");

System.out.println(matcher.matches());
false
검사 문자열: abc123def
정규식: \d+

질문:
전체 문자열이 숫자로만 이루어져 있는가?

결과:
false

3-2. find()는 부분 검색

Pattern pattern = Pattern.compile("\\d+");
Matcher matcher = pattern.matcher("abc123def");

System.out.println(matcher.find());
true

find()는 전체 문자열이 일치하지 않아도, 문자열 안에 패턴과 일치하는 부분이 있으면 true를 반환합니다. 

검사 문자열: abc123def
정규식: \d+

찾은 부분:
123

결과:
true

3-3. group()은 찾은 값을 꺼낸다

import java.util.regex.Pattern;
import java.util.regex.Matcher;

public class GroupExample {

    public static void main(String[] args) {

        String text = "상품 가격은 12000원이고 배송비는 3000원입니다.";

        Pattern pattern = Pattern.compile("\\d+");
        Matcher matcher = pattern.matcher(text);

        while (matcher.find()) {
            System.out.println(matcher.group());
        }
    }
}
12000
3000

find()가 패턴에 맞는 부분을 찾고, group()이 현재 찾은 값을 반환합니다.

3-4. 세 메서드 비교

메서드 검사 방식 주요 사용 상황
matches() 전체 문자열이 패턴과 일치해야 함 회원가입 입력값 검증
find() 문자열 안에서 패턴과 맞는 부분을 찾음 로그, 문장, 긴 텍스트에서 값 찾기
group() 찾은 값 또는 그룹 값을 가져옴 숫자, 이름, 코드, 날짜 추출

4. 기본 정규표현식 기호

4-1. 기본 기호 한눈에 보기

기호 의미 예시
. 임의의 한 글자 a.b → aab, acb
* 앞 문자가 0개 이상 a*b → b, ab, aaab
+ 앞 문자가 1개 이상 a+b → ab, aaab
? 앞 문자가 0개 또는 1개 a?b → b, ab
^ 문자열 시작 ^ab → ab로 시작
$ 문자열 끝 ab$ → ab로 끝남
[] 문자 집합 [abc] → a, b, c 중 하나
[^] 제외 문자 집합 [^0-9] → 숫자가 아닌 문자
{} 반복 횟수 a{3} → aaa
() 그룹 a(bc){2} → abcbc
| OR a(b|c)d → abd, acd

4-3. *, +, ? 비교

표현 의미 가능한 값
a*b a가 0개 이상 + b b, ab, aab, aaab
a+b a가 1개 이상 + b ab, aab, aaab
a?b a가 0개 또는 1개 + b b, ab 
public class RepeatRegexExample {

    public static void main(String[] args) {

        System.out.println("b".matches("a*b"));
        System.out.println("b".matches("a+b"));
        System.out.println("b".matches("a?b"));
    }

false
true

4-4. {} : 반복 횟수 지정

표현 의미
a{3} a가 정확히 3개
a{1,} a가 1개 이상
a{1,3} a가 1개 이상 3개 이하
a{0,} a*와 같음
a{1,} a+와 같음
a{0,1} a?와 같음 
public class CountRegexExample {

    public static void main(String[] args) {

        System.out.println("aaa".matches("a{3}"));
        System.out.println("aa".matches("a{3}"));
        System.out.println("aaaa".matches("a{1,3}"));
    }
}
true
false
false

4-5. ^와 $ : 시작과 끝

^는 문자열의 시작, $는 문자열의 끝을 의미합니다. 입력값 전체를 검사할 때는 보통 둘을 함께 사용합니다. 

public class StartEndRegexExample {

    public static void main(String[] args) {

        String regex = "^\\d{3}$";

        System.out.println("123".matches(regex));
        System.out.println("1234".matches(regex));
        System.out.println("abc123".matches(regex));
    }
}
true
false
false
^       \d{3}       $
시작     숫자 3개      끝

즉,
처음부터 끝까지 숫자 3개여야 한다.

4-7. ()와 |

()는 여러 문자를 하나의 그룹으로 묶고, |는 OR 조건을 의미합니다. 

public class GroupOrRegexExample {

    public static void main(String[] args) {

        System.out.println("abcbc".matches("a(bc){2}"));

        System.out.println("abd".matches("a(b|c)d"));
        System.out.println("acd".matches("a(b|c)d"));
        System.out.println("aed".matches("a(b|c)d"));
    }
}

true
true
false
a(bc){2}
→ a + bc + bc
→ abcbc

a(b|c)d
→ a + b 또는 c + d
→ abd 또는 acd

5. 자주 쓰는 축약 표현

5-1. 축약 표현 표

의미 정규표현식 Java 문자열
숫자 \d "\\d"
숫자가 아닌 문자 \D "\\D"
공백 문자 \s "\\s"
공백이 아닌 문자 \S "\\S"
영문자, 숫자, 언더바 \w "\\w"
영문자, 숫자, 언더바가 아닌 문자 \W "\\W"
알파벳 \p{Alpha} "\\p{Alpha}"
숫자 \p{Digit} "\\p{Digit}"
알파벳 또는 숫자 \p{Alnum} "\\p{Alnum}"

Java 문자열 안에서는 백슬래시를 한 번 더 써야 합니다. 정규식에서 \d라고 쓰는 표현은 Java 코드에서는 "\\d"가 됩니다.

6-6. 비밀번호 검사

조건은 다음과 같습니다.

  1. 8자 이상 20자 이하입니다.
  2. 영문 대소문자, 숫자, 특수문자 !@#$%^&*만 허용합니다.
  3. 숫자를 최소 1개 포함해야 합니다.
  4. 특수문자를 최소 1개 포함해야 합니다.
public class PasswordRegexExample {

    public static void main(String[] args) {

        String regex = "^(?=.*\\d)(?=.*[!@#$%^&*])[A-Za-z0-9!@#$%^&*]{8,20}$";

        System.out.println("abc12345".matches(regex));
        System.out.println("abcdefg!".matches(regex));
        System.out.println("abc1234!".matches(regex));
        System.out.println("ABC1234!".matches(regex));
        System.out.println("abc 1234!".matches(regex));
    }
}

false

8. 정규표현식 읽는 방법

8-1. 복잡한 정규식은 나누어서 읽는다

정규표현식은 한 번에 읽으려고 하면 어렵습니다. 반드시 왼쪽에서 오른쪽으로 쪼개서 읽어야 합니다. 

String regex = "^01[016789]-?\\d{3,4}-?\\d{4}$";
^
→ 문자열 시작

01
→ 01로 시작

[016789]
→ 0, 1, 6, 7, 8, 9 중 하나

-?
→ 하이픈이 0개 또는 1개

\d{3,4}
→ 숫자 3개 이상 4개 이하

-?
→ 하이픈이 0개 또는 1개

\d{4}
→ 숫자 4개

$
→ 문자열 끝

8-2. Java 문자열과 정규표현식을 구분한다

원하는 정규표현식 Java 코드에서 쓰는 문자열
\d "\\d"
\D "\\D"
\s "\\s"
\. "\\."

정규표현식에서 숫자를 의미하는 것은 \d지만, Java 문자열 안에서는 "\\d"로 작성해야 합니다. 이것은 Java의 문자열 이스케이프 규칙 때문입니다.

9. 자주 하는 실수

실수 1. 백슬래시를 하나만 작성함

String regex = "\d+";
String regex = "\\d+";

Java 문자열에서는 백슬래시 자체를 표현하기 위해 \\를 사용해야 합니다.

실수 2. 점을 진짜 점으로 착각함

String regex = ".+@.+..+";
String regex = ".+@.+\\..+";

정규표현식에서 .은 아무 문자 한 글자를 의미합니다. 실제 점을 검사하려면 \\.를 사용해야 합니다.

실수 3. matches()와 find()를 혼동함

Pattern pattern = Pattern.compile("\\d+");
Matcher matcher = pattern.matcher("abc123def");

System.out.println(matcher.matches());
false
Pattern pattern = Pattern.compile("\\d+");
Matcher matcher = pattern.matcher("abc123def");

System.out.println(matcher.find());
true

matches()는 전체 일치 검사이고, find()는 부분 검색입니다.

실수 4. ^와 $를 빼먹음

String regex = "\\d{3}";
String regex = "^\\d{3}$";

입력값 전체가 특정 형식인지 검사할 때는 시작과 끝을 명확하게 지정하는 것이 좋습니다.

System.out.println("a".matches("[abc]"));
System.out.println("abc".matches("[abc]"));

실수 6. 정규표현식 하나에 모든 검증을 몰아넣음

비밀번호처럼 조건이 복잡한 경우, 모든 조건을 정규표현식 하나로 처리하면 가독성이 떨어질 수 있습니다. 실무에서는 정규표현식과 일반 Java 조건문을 나누어 사용하는 것이 더 좋을 때도 많습니다. 

String password = "abc1234!";

boolean lengthCheck = password.length() >= 8 && password.length() <= 20;
boolean hasNumber = password.matches(".*\\d.*");
boolean hasSpecial = password.matches(".*[!@#$%^&*].*");
boolean noSpace = !password.matches(".*\\s.*");

if (lengthCheck && hasNumber && hasSpecial && noSpace) {
    System.out.println("사용 가능한 비밀번호");
} else {
    System.out.println("사용 불가능한 비밀번호");
}

10. 면접 포인트

Q1. 정규표현식이란 무엇인가요?

답변: 정규표현식은 문자열이 특정한 패턴을 만족하는지 검사하거나, 문자열 안에서 특정 패턴을 찾아 추출 또는 치환할 때 사용하는 표현식입니다.
Java에서는 Pattern, Matcher 클래스를 사용하거나 String.matches(), replaceAll(), split() 같은 메서드를 통해 사용할 수 있습니다.

Q2. Java에서 Pattern과 Matcher의 역할은 무엇인가요?

답변: Pattern은 정규표현식 패턴을 컴파일해서 준비하는 클래스이고, Matcher는 준비된 패턴을 실제 문자열에 적용하는 클래스입니다.
Pattern.compile()로 패턴을 만들고, pattern.matcher()로 검사 대상 문자열을 연결한 뒤 matches(), find(), group() 등을 사용할 수 있습니다.

Q3. matches()와 find()의 차이는 무엇인가요?

답변: matches()는 문자열 전체가 정규표현식과 일치해야 true를 반환합니다. 반면 find()는 문자열 중 일부라도 정규표현식과 일치하는 부분이 있으면 true를 반환합니다.
입력값 전체 검증에는 matches(), 로그나 문장 안에서 특정 값을 찾을 때는 find()를 주로 사용합니다.

Q4. Java 정규표현식에서 \\d처럼 백슬래시를 두 번 쓰는 이유는 무엇인가요?

답변: 정규표현식에서 \d는 숫자를 의미합니다.
하지만 Java 문자열에서 백슬래시는 이스케이프 문자이기 때문에, 실제 정규표현식 엔진에 \d를 전달하려면 Java 문자열 안에서는 "\\d"처럼 백슬래시를 두 번 작성해야 합니다.

Q5. ^와 $는 무엇인가요?

답변: ^는 문자열의 시작을 의미하고, $는 문자열의 끝을 의미합니다. 입력값 전체가 특정 형식과 일치하는지 검사할 때 두 기호를 함께 사용합니다.
예를 들어 "^\\d{3}$"는 전체 문자열이 숫자 3개로만 이루어져 있는지 검사합니다.

Q6. *, +, ?의 차이는 무엇인가요?

답변: *는 바로 앞 문자가 0개 이상 반복되는 경우를 의미하고, +는 1개 이상 반복되는 경우를 의미합니다. ?는 0개 또는 1개만 허용합니다.
예를 들어 a*b는 b, ab, aab가 가능하지만, a+b는 b가 불가능합니다.

Q7. 정규표현식의 단점은 무엇인가요?

답변: 정규표현식은 짧은 코드로 강력한 문자열 검사를 할 수 있지만, 복잡해질수록 가독성이 크게 떨어질 수 있습니다.
그래서 복잡한 검증 정책은 정규표현식 하나에 모두 넣기보다, 정규표현식과 Java 조건문을 적절히 나누어 작성하는 것이 유지보수에 좋습니다.
