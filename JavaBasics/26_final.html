26. final 지정자

final은 더 이상 변경하지 못하게 막는 Java의 지정자입니다.
변수에 붙으면 값 재대입을 막고, 메서드에 붙으면 오버라이딩을 막으며, 클래스에 붙으면 상속을 막습니다.
이 페이지에서는 final 변수, 상수, 메서드, 클래스, static final, abstract와의 차이를 공부합니다.

이번 페이지의 학습 목표

final은 “여기서 끝, 더 이상 바꾸지 마라”라는 의미를 가진 지정자입니다.
변수에 붙이면 값을 바꿀 수 없고, 메서드에 붙이면 자식 클래스가 재정의할 수 없고, 클래스에 붙이면 다른 클래스가 상속할 수 없습니다.

final 변수

한 번 초기화한 값을 다시 대입할 수 없다는 의미를 이해합니다.

static final 상수

프로그램 전체에서 공유되는 변경 불가능한 클래스 상수를 익힙니다.

final 메서드와 클래스

오버라이딩과 상속을 막는 구조를 이해합니다.

abstract와 비교

abstract와 final이 동시에 사용될 수 없는 이유를 이해합니다.

1. final의 기본 개념

1-1. final이란?

final은 Java에서 변경을 막기 위한 지정자입니다. 붙는 위치에 따라 의미가 달라집니다.

사용 위치 의미 막는 것
final 변수 값을 다시 대입할 수 없음 값 변경
final 메서드 자식 클래스에서 재정의할 수 없음 오버라이딩
final 클래스 다른 클래스가 상속할 수 없음 상속

핵심: final은 “변경 금지”입니다. 변수에는 값 변경 금지, 메서드에는 오버라이딩 금지, 클래스에는 상속 금지의 의미로 적용됩니다.

1-2. final의 핵심 문장

final 변수
→ 값을 다시 넣지 마라.

final 메서드
→ 자식이 메서드 내용을 바꾸지 마라.

final 클래스
→ 이 클래스를 부모로 삼지 마라.

2. final 변수

2-1. final 변수란?

변수 앞에 final을 붙이면 값을 한 번만 대입할 수 있습니다. 한 번 값이 정해진 뒤에는 다시 바꿀 수 없습니다. 

final int MAX_COUNT = 10;

System.out.println(MAX_COUNT);
final int MAX_COUNT = 10;

MAX_COUNT = 20; // 오류
final int count = 10;

count
┌──────┐
│  10  │
└──────┘

count = 20;
→ 불가능

의미:
한 번 정한 값은 다시 바꿀 수 없다.

2-2. 일반 변수와 final 변수 비교

int count = 10;
count = 20; // 가능
final int count = 10;
count = 20; // 오류

일반 변수는 값을 다시 바꿀 수 있지만, final 변수는 최초 1회 초기화 후 값을 바꿀 수 없습니다.

2-3. final 변수는 반드시 초기화해야 한다

final 변수는 반드시 한 번은 초기화되어야 합니다. 선언과 동시에 초기화할 수도 있고, 선언 후 한 번만 대입할 수도 있습니다. 

final int number;
number = 10;
final int number;
number = 10;
number = 20; // 오류

final은 “아예 값을 넣지 말라”가 아니라, “한 번만 값을 정하고 다시 바꾸지 말라”는 의미입니다.

2-4. blank final

선언할 때 바로 초기화하지 않고, 생성자에서 반드시 한 번 초기화하는 final 변수를 blank final이라고 합니다. 

class User {

    private final String loginId;

    public User(String loginId) {
        this.loginId = loginId;
    }

    public String getLoginId() {
        return loginId;
    }

}
loginId
→ 객체 생성 시 한 번 정해짐
→ 이후 변경 불가

사용하기 좋은 값:
- 회원 로그인 ID
- 주문 번호
- 계좌 번호
- 생성 시점에 정해지는 고유 식별값

2-5. final 매개변수와 지역변수

final은 메서드의 매개변수나 지역변수에도 붙일 수 있습니다. 이 경우 메서드 내부에서 값을 다시 대입할 수 없습니다. 

public void print(final String message) {
    System.out.println(message);

    // message = "변경"; // 오류
}
public void calculate() {
    final int taxRate = 10;

    // taxRate = 20; // 오류

    System.out.println(taxRate);
}

3. final 상수와 static final

3-1. final 변수는 상수처럼 사용할 수 있다

final 변수는 한 번 값이 정해지면 다시 바뀌지 않으므로 상수처럼 사용할 수 있습니다. 

final double PI = 3.141592;
final int MAX_LOGIN_FAIL_COUNT = 5;
final int DEFAULT_PAGE_SIZE = 20;

상수명은 보통 대문자와 언더바 _를 사용합니다. 

public static final int MAX_COUNT = 100;
public static final int DEFAULT_PAGE_SIZE = 20;
public static final String ADMIN_ROLE = "ADMIN";

3-2. public static final의 의미

public static final int MAX_COUNT = 100;
키워드 의미
public 어디서든 접근 가능
static 객체를 만들지 않고 클래스명으로 접근 가능
final 값을 다시 바꿀 수 없음 
class AppConfig {
    public static final int MAX_COUNT = 100;
}

System.out.println(AppConfig.MAX_COUNT);

3-3. static과 final의 차이

구분 static final
의미 클래스 소속, 공유 변경 금지
값 변경 가능 불가능
접근 방식 클래스명으로 접근 가능 변수 성격에 따라 다름
목적 공유 불변 
class Counter {
    static int count = 0;
    final int maxCount = 10;
}

Counter.count = 1;
Counter.count = 2; // 가능
Counter counter = new Counter();

counter.maxCount = 20; // 오류

3-4. static final은 클래스 상수

static
→ 클래스에 하나만 존재

final
→ 값 변경 불가

static final
→ 클래스에 하나만 존재하고 변경할 수 없는 상수

class Config {
    public static final int MAX_PAGE_SIZE = 100;
}

System.out.println(Config.MAX_PAGE_SIZE);

4. final 메서드

4-1. final 메서드란?

메서드 앞에 final을 붙이면 자식 클래스에서 오버라이딩할 수 없습니다. 

class Parent {

    public final void print() {
        System.out.println("부모의 final 메서드");
    }

}
class Child extends Parent {

    @Override
    public void print() { // 오류
        System.out.println("자식에서 재정의");
    }

}
Parent

final print()
    ↓ 상속

Child

print() 재정의 시도
→ 오류

의미:
부모가 정한 메서드 동작을 자식이 바꿀 수 없다.

4-2. final 메서드를 사용하는 이유

자식 클래스가 마음대로 바꾸면 안 되는 중요한 처리 흐름을 보호할 때 사용합니다. 

class LoginProcessor {

    public final void login() {
        checkInput();
        authenticate();
        createToken();
    }

    protected void checkInput() {
        System.out.println("입력값 검증");
    }

    protected void authenticate() {
        System.out.println("사용자 인증");
    }

    protected void createToken() {
        System.out.println("토큰 생성");
    }

}

login()의 전체 흐름은 바뀌면 안 되므로 final로 막을 수 있습니다. 단, 내부 단계 메서드는 필요에 따라 자식이 사용할 수 있게 protected로 둘 수 있습니다.

5. final 클래스

5-1. final 클래스란?

클래스 앞에 final을 붙이면 다른 클래스가 상속할 수 없습니다. 

final class SecurityUtil {
}
class MySecurityUtil extends SecurityUtil { // 오류
}
final class SecurityUtil

SecurityUtil
    ↓ extends 시도

MySecurityUtil
→ 오류

의미:
이 클래스는 부모 클래스가 될 수 없다.

5-2. final 클래스를 사용하는 이유

상속을 허용하면 자식 클래스가 부모 클래스의 동작을 바꿀 수 있습니다. 보안, 안정성, 불변성 때문에 상속을 막고 싶을 때 final class를 사용할 수 있습니다. 

public final class String {
    // Java의 String 클래스는 final 클래스
}

대표적으로 Java의 String 클래스는 final 클래스입니다. 문자열의 동작이 상속으로 바뀌지 않도록 막는 구조입니다.

5-3. 유틸 클래스 상속 방지

public final class StringUtil {

    private StringUtil() {
    }

    public static boolean isBlank(String value) {
        return value == null || value.trim().isEmpty();
    }

}
final class
→ 상속 방지

private 생성자
→ 외부 객체 생성 방지

static 메서드
→ 클래스명으로 바로 사용

6. abstract와 final

6-1. abstract와 final은 동시에 사용할 수 없다

abstract와 final은 목적이 완전히 반대입니다.

구분 abstract final
클래스 상속받아 자식이 완성해야 함 상속을 막음
메서드 자식이 반드시 오버라이딩해야 함 오버라이딩을 막음
변수 붙일 수 없음 값 재할당을 막음 
abstract final class Animal {
}
abstract final void sound();
abstract
→ 자식이 상속해서 완성해라.

final
→ 아무도 상속하거나 재정의하지 마라.

결론:
서로 모순이므로 함께 사용할 수 없다.

6-2. abstract와 final을 그림처럼 비교

abstract
→ 아직 미완성
→ 자식이 상속해서 완성해야 함

final
→ 여기서 완성
→ 더 이상 바꾸거나 확장하지 못함

7. final 참조변수 주의

7-1. final 참조변수는 참조 대상 변경을 막는다

final이 참조변수에 붙으면 다른 객체로 다시 대입할 수 없습니다. 하지만 참조하고 있는 객체 내부의 값은 그 객체가 허용한다면 변경될 수 있습니다. 

final ArrayList<String> list = new ArrayList<>();

list.add("Java");   // 가능
list.add("Oracle"); // 가능
list = new ArrayList<>(); // 오류
final ArrayList<String> list = new ArrayList<>();

list 변수
→ 다른 ArrayList 객체로 갈아끼울 수 없음

list가 가리키는 ArrayList 내부
→ add(), remove()가 가능하면 내용 변경 가능

7-2. 객체 내부까지 불변으로 만들려면?

final 참조변수는 참조 변경을 막을 뿐, 객체 내부 상태 변경을 항상 막는 것은 아닙니다. 객체 자체를 불변으로 만들려면 별도의 설계가 필요합니다. 

final class Money {

    private final int amount;

    public Money(int amount) {
        this.amount = amount;
    }

    public int getAmount() {
        return amount;
    }

}
불변 객체 설계 예:
1. class를 final로 선언
2. 필드를 private final로 선언
3. setter를 만들지 않음
4. 생성자에서 값을 초기화

8. 실무 연결

8-1. 설정값 상수

public class AuthConstants {

    public static final int REFRESH_TOKEN_EXPIRE_DAYS = 14;

문자열이나 숫자를 여러 곳에 직접 쓰는 것보다, 상수로 분리하면 오타를 줄이고 유지보수하기 쉬워집니다.

8-2. 고유 식별값 보호

class User {

    private final String loginId;

    public User(String loginId) {
        this.loginId = loginId;
    }

    public String getLoginId() {
        return loginId;
    }

}

loginId처럼 객체 생성 후 바뀌면 안 되는 값은 final로 보호할 수 있습니다.

8-3. 불변 객체 설계의 일부

final class Money {

    private final int amount;

    public Money(int amount) {
        this.amount = amount;
    }

    public int getAmount() {
        return amount;
    }

}

final class, private final field, setter 없음의 조합은 객체의 상태를 안전하게 유지하는 데 도움이 됩니다.

10. 자주 하는 실수

실수 1. final 변수를 다시 대입함

final int count = 10;

count = 20; // 오류

실수 2. final 상수를 초기화하지 않음

final int MAX_COUNT;

System.out.println(MAX_COUNT); // 오류
final int MAX_COUNT;

System.out.println(MAX_COUNT);

list.add("Java");        // 가능
list.remove("Java");     // 가능

list = new ArrayList<>(); // 오류

}

11. 면접 포인트

Q1. final 지정자는 무엇인가요?

답변: final은 더 이상 변경하지 못하게 막는 지정자입니다. 변수에 붙으면 값을 재대입할 수 없고, 메서드에 붙으면 오버라이딩할 수 없고, 클래스에 붙으면 상속할 수 없습니다.

Q2. final 변수를 상수라고 볼 수 있나요?

답변: 네. final 변수는 한 번 초기화된 후 값을 다시 대입할 수 없기 때문에 상수처럼 사용할 수 있습니다.
일반적으로 상수는 public static final로 선언하고 이름은 대문자와 언더바를 사용합니다.

Q3. public static final의 의미는 무엇인가요?

답변: public은 어디서든 접근 가능하다는 뜻이고, static은 객체 생성 없이 클래스명으로 접근할 수 있다는 뜻이며, final은 값을 다시 바꿀 수 없다는 뜻입니다.
따라서 public static final은 전역적으로 접근 가능한 클래스 상수를 만들 때 자주 사용합니다.

Q4. final 메서드는 무엇인가요?

답변: final 메서드는 자식 클래스에서 오버라이딩할 수 없는 메서드입니다. 부모 클래스가 정한 중요한 동작을 자식 클래스가 바꾸지 못하게 할 때 사용합니다.

Q5. final 클래스는 무엇인가요?

답변: final 클래스는 다른 클래스가 상속할 수 없는 클래스입니다. 상속을 통해 기능이 변경되는 것을 막고 싶을 때 사용합니다.

Q6. abstract와 final을 함께 사용할 수 있나요?

답변: 사용할 수 없습니다. abstract는 자식 클래스가 상속해서 완성해야 한다는 의미이고, final은 상속이나 오버라이딩을 막는 의미이기 때문에 서로 목적이 반대입니다.

Q7. static과 final의 차이는 무엇인가요?

답변: static은 클래스 소속이라는 의미이고, final은 변경 금지라는 의미입니다. static 변수는 값 변경이 가능하지만, final 변수는 값을 다시 대입할 수 없습니다.
static final은 클래스에 하나만 존재하고 변경할 수 없는 클래스 상수입니다.

Q8. final 참조변수는 객체 내부 값도 바꿀 수 없나요?

답변: 아닙니다. final 참조변수는 다른 객체로 다시 대입할 수 없다는 뜻입니다. 참조하고 있는 객체 내부의 상태는 그 객체가 변경을 허용한다면 바뀔 수 있습니다.

Q9. blank final이란 무엇인가요?

답변: blank final은 선언할 때 바로 초기화하지 않고 생성자에서 한 번 초기화하는 final 변수입니다.
객체마다 다른 값을 가지지만 생성 이후에는 바뀌면 안 되는 값에 사용합니다.

Q10. final을 실무에서 언제 사용하나요?

답변: 변하지 않는 상수, 중요한 설정값, 고유 식별값, 오버라이딩되면 안 되는 메서드, 상속되면 안 되는 클래스, 불변 객체 설계 등에 사용합니다.

12. 오늘의 핵심 정리

  1. final은 더 이상 변경하지 못하게 막는 지정자다.
  2. final 변수는 값을 다시 대입할 수 없다.
  3. final 변수는 상수처럼 사용할 수 있다.
  4. 상수명은 보통 대문자와 언더바로 작성한다.
  5. public static final은 클래스 상수 선언에 자주 사용한다.
  6. final 메서드는 오버라이딩할 수 없다.
  7. final 클래스는 상속할 수 없다.
  8. abstract는 상속과 오버라이딩을 요구하는 방향이다.
  9. final은 상속과 오버라이딩을 막는 방향이다.
  10. abstract와 final은 목적이 반대라 동시에 사용할 수 없다.
  11. static은 공유의 의미이고, final은 변경 금지의 의미다.
  12. static final은 공유되고 변경 불가능한 클래스 상수다.
  13. final 참조변수는 다른 객체로 다시 대입할 수 없지만, 객체 내부 상태는 변경될 수 있다.
  14. blank final은 생성자에서 한 번 초기화하는 final 변수다.

학습 메모

이 페이지는 Java final 지정자 단원입니다.
final은 변수, 메서드, 클래스에 붙을 수 있으며 각각 값 변경 금지, 오버라이딩 금지, 상속 금지의 의미를 가집니다.
static final은 클래스 상수로 자주 사용되고, abstract와 final은 목적이 반대이므로 함께 사용할 수 없습니다.
또한 final 참조변수는 다른 객체로 다시 대입할 수 없다는 뜻이지, 객체 내부 상태 변경을 항상 막는 것은 아니라는 점을 꼭 기억해야 합니다.
