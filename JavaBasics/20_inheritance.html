20. 상속

상속은 부모 클래스의 공통 속성과 기능을 자식 클래스가 물려받아 확장하는 객체지향 문법입니다.
이 페이지에서는 extends, is-a 관계, super, 생성자 실행 순서,
부모 타입 참조변수, 형변환, 부모 타입 배열까지 공부합니다.

이번 페이지의 학습 목표

상속은 객체지향 프로그래밍의 핵심 개념 중 하나입니다.
공통 기능은 부모 클래스에 두고, 자식 클래스는 그 기능을 물려받아 자신만의 기능을 추가합니다.
상속을 이해하면 코드 재사용, 확장, 오버라이딩, 다형성의 흐름을 자연스럽게 이해할 수 있습니다.

상속 개념

부모 클래스와 자식 클래스의 관계를 이해합니다.

extends 문법

Java에서 상속을 선언하는 기본 문법을 익힙니다.

생성자와 super

자식 객체 생성 시 부모 생성자가 먼저 실행되는 흐름을 이해합니다.

부모 타입 참조

부모 타입으로 자식 객체를 참조하는 구조와 접근 범위 제한을 익힙니다.

1. 상속의 기본 개념

1-1. 상속이란?

상속은 기존 클래스의 필드와 메서드를 물려받아 새로운 클래스를 만드는 문법입니다. 기존 클래스는 부모 클래스, 새로 확장하는 클래스는 자식 클래스라고 합니다. 

class Parent {
}

class Child extends Parent {
}
Parent
→ 부모 클래스, 상위 클래스, super class

Child
→ 자식 클래스, 하위 클래스, sub class

extends
→ 상속을 표현하는 키워드

1-2. 상속을 사용하는 이유

상속을 사용하는 가장 큰 이유는 코드 재사용과 확장입니다.
여러 클래스가 공통으로 가지는 속성과 기능을 부모 클래스에 모아두면, 자식 클래스는 중복 코드를 작성하지 않고 필요한 기능만 추가할 수 있습니다. 

일반 회원
- 이름
- 이메일
- 로그인 기능

관리자 회원
- 이름
- 이메일
- 로그인 기능
- 회원 관리 기능

공인중개사 회원
- 이름
- 이메일
- 로그인 기능
- 매물 등록 기능

공통 부분:
이름, 이메일, 로그인 기능
→ User 부모 클래스로 분리

2. is-a 관계와 상속 판단 기준

2-1. is-a 관계란?

상속은 “자식 is a 부모” 관계가 자연스럽게 성립할 때 사용하는 것이 좋습니다. 

Dog is an Animal.
개는 동물이다.
→ 자연스러움 → 상속 가능

AdminUser is a User.
관리자는 회원이다.
→ 자연스러움 → 상속 가능

RealtorUser is a User.
공인중개사는 회원이다.
→ 자연스러움 → 상속 가능

2-2. 상속으로 만들면 어색한 관계

Car is an Engine.
자동차는 엔진이다.
→ 어색함

Property is a Realtor.
매물은 공인중개사다.
→ 어색함

User is a LoginService.
회원은 로그인 서비스다.
→ 어색함

이런 경우는 상속보다 포함 관계, 즉 has-a 관계로 설계하는 것이 좋습니다. 

class Car {
    private Engine engine;
}

class Property {
    private RealtorUser realtor;
}

정리: 상속은 is-a 관계, 조합 또는 포함은 has-a 관계일 때 자연스럽습니다.

4. 상속과 접근 지정자

4-1. 자식 클래스가 물려받는 것

자식 클래스는 부모 클래스의 필드와 메서드를 물려받을 수 있습니다. 하지만 접근 지정자에 따라 직접 접근 가능 여부가 달라집니다.

접근 지정자 자식 클래스 접근 가능 여부 설명
public 가능 어디서나 접근 가능
protected 가능 같은 패키지 또는 상속 관계에서 접근 가능
default 같은 패키지면 가능 패키지가 다르면 접근 불가
private 직접 접근 불가 부모 클래스 내부에서만 접근 가능

4-2. private 멤버는 자식에서 직접 접근할 수 없다

class Parent {

    private String secret = "비밀";

}

class Child extends Parent {

    void printSecret() {
        System.out.println(secret); // 오류
    }

}

부모의 private 멤버는 자식 객체 안에 포함되더라도, 자식 클래스에서 직접 접근할 수 없습니다. 

class Parent {

    private String secret = "비밀";

    protected String getSecret() {
        return secret;
    }

}

class Child extends Parent {

    void printSecret() {
        System.out.println(getSecret());
    }

}

부모의 private 필드는 직접 접근하지 못하지만, 부모가 제공한 protected 또는 public 메서드로 간접 접근할 수 있습니다.

5. super와 생성자

5-1. super로 부모 멤버 접근

class Parent {

    String name = "부모";

}

class Child extends Parent {

    String name = "자식";

    void printName() {
        System.out.println(this.name);
        System.out.println(super.name);
    }

}
자식
부모
this.name
→ 자식 클래스 영역의 name

super.name
→ 부모 클래스 영역의 name

5-2. 부모 메서드 호출

class Parent {

    void printInfo() {
        System.out.println("부모 정보");
    }

}

class Child extends Parent {

    void printInfo() {
        System.out.println("자식 정보");
    }

    void callParent() {
        super.printInfo();
    }

}

5-3. 생성자 실행 순서

자식 객체가 만들어질 때는 부모 영역도 함께 만들어져야 합니다. 그래서 부모 생성자가 먼저 실행되고, 그 다음 자식 생성자가 실행됩니다. 

class Parent {

    public Parent() {
        System.out.println("부모 생성자");
    }

}

class Child extends Parent {

    public Child() {
        System.out.println("자식 생성자");
    }

}
Child child = new Child();
부모 생성자
자식 생성자
new Child()

1. Parent 영역 생성
2. Parent 생성자 실행
3. Child 영역 생성
4. Child 생성자 실행

5-4. 부모 기본 생성자가 없을 때

부모 클래스에 인자 생성자만 있고 기본 생성자가 없으면, 자식 생성자에서 부모 생성자를 명시적으로 호출해야 할 수 있습니다. 

class Parent {

    public Parent(String name) {
    }

}

class Child extends Parent {

    public Child() {
        // 컴파일러가 super(); 자동 호출 시도
        // 그런데 Parent()가 없음 → 오류
    }

}

해결 방법 1: 부모에 기본 생성자를 추가합니다.

class Parent {

    public Parent() {
    }

    public Parent(String name) {
    }

}

해결 방법 2: 자식 생성자에서 부모 인자 생성자를 명시적으로 호출합니다.

class Child extends Parent {

    public Child() {
        super("홍길동");
    }

}

6. Java 상속의 특징

6-1. Java는 단일 상속만 가능하다

Java 클래스는 부모 클래스를 하나만 상속받을 수 있습니다. 

class Child extends Parent {
}
class Child extends Parent1, Parent2 {
}

여러 기능 규약을 연결하고 싶을 때는 나중에 배우는 인터페이스를 사용합니다. 

class Child extends Parent implements Interface1, Interface2 {
}

6-2. 부모 타입 참조변수로 자식 객체 참조

상속 관계에서는 부모 타입 참조변수로 자식 객체를 참조할 수 있습니다. 

User user = new AdminUser();

이것이 가능한 이유는 “관리자는 회원이다”라는 is-a 관계가 성립하기 때문입니다. 

AdminUser admin = new User();

반대로 모든 회원이 관리자는 아니므로 위 코드는 불가능합니다.

6-3. 부모 타입 참조변수의 접근 범위 제한

class User {

    void login() {
        System.out.println("User 로그인");
    }

}

class AdminUser extends User {

    void login() {
        System.out.println("Admin 로그인");
    }

    void manageUsers() {
        System.out.println("회원 관리");
    }

}
User user = new AdminUser();

user.login();       // 가능
user.manageUsers(); // 오류
접근 가능:
1. 부모 클래스에 정의된 필드와 메서드
2. 자식이 오버라이딩한 부모 메서드

접근 불가:
1. 자식 클래스에만 새로 추가된 고유 필드
2. 자식 클래스에만 새로 추가된 고유 메서드

6-4. 자식 타입으로 형변환

부모 타입 참조변수로 접근할 수 없는 자식 고유 멤버에 접근하려면 자식 타입으로 형변환해야 합니다. 

User user = new AdminUser();

if (user instanceof AdminUser) {
    AdminUser admin = (AdminUser) user;
    admin.manageUsers();
}

실제 객체가 정말 AdminUser일 때만 안전하게 형변환할 수 있으므로, instanceof로 확인하는 습관이 좋습니다.

6-5. 부모 타입 배열

User[] users = new User[3];

users[0] = new User();
users[1] = new AdminUser();
users[2] = new RealtorUser();
배열 타입은 User[]

실제 객체:
users[0] → User 객체
users[1] → AdminUser 객체
users[2] → RealtorUser 객체

}

실수 6. 상속을 코드 재사용만 보고 남용

공통 코드가 있다고 무조건 상속을 쓰는 것은 위험합니다. 상속은 부모와 자식이 강하게 묶이므로 반드시 is-a 관계인지 확인해야 합니다.

판단 기준: “자식은 부모이다”라고 말했을 때 자연스러우면 상속, “A는 B를 가진다”가 자연스러우면 조합을 고려합니다.

10. 면접 포인트

Q1. 상속이란 무엇인가요?

답변: 상속은 부모 클래스의 필드와 메서드를 자식 클래스가 물려받아 확장하는 객체지향 문법입니다. Java에서는 extends 키워드를 사용합니다.

Q2. 상속을 사용하는 이유는 무엇인가요?

답변: 공통 속성과 기능을 부모 클래스에 모아 코드 중복을 줄이고, 자식 클래스에서 기능을 확장할 수 있기 때문입니다. 또한 다형성의 기반이 됩니다.

Q3. Java에서 다중 상속이 가능한가요?

답변: 클래스는 다중 상속이 불가능하고 단일 상속만 가능합니다. 즉 extends로 상속받을 수 있는 클래스는 하나뿐입니다. 다만 인터페이스는 여러 개 구현할 수 있습니다.

Q4. 부모의 private 멤버는 자식 클래스에서 접근할 수 있나요?

답변: 직접 접근할 수 없습니다.
private 멤버는 선언된 클래스 내부에서만 접근 가능하기 때문에, 자식 클래스에서도 직접 접근할 수 없습니다.
필요하다면 부모 클래스가 protected 또는 public 메서드를 제공해야 합니다.

Q5. 자식 객체를 생성하면 생성자 실행 순서는 어떻게 되나요?

답변: 부모 생성자가 먼저 실행되고 그 다음 자식 생성자가 실행됩니다. 자식 객체 안에는 부모 영역도 포함되기 때문에 부모 영역을 먼저 초기화해야 합니다.

Q6. 부모 클래스에 기본 생성자가 없고 인자 생성자만 있으면 어떻게 해야 하나요?

답변: 자식 생성자에서 super(값)를 사용해 부모의 인자 생성자를 명시적으로 호출해야 합니다. 또는 부모 클래스에 기본 생성자를 직접 추가할 수 있습니다.

Q7. 부모 타입 참조변수로 자식 객체를 참조할 수 있나요?

답변: 가능합니다. 예를 들어 User user = new AdminUser();처럼 자식 객체를 부모 타입으로 참조할 수 있습니다. 이는 AdminUser is a User 관계가 성립하기 때문입니다.

Q8. 부모 타입 참조변수로 자식 고유 메서드를 호출할 수 있나요?

답변: 바로 호출할 수 없습니다. 부모 타입 참조변수는 부모 클래스 기준의 멤버만 접근할 수 있습니다. 자식 고유 메서드에 접근하려면 자식 타입으로 형변환해야 합니다.

Q9. 오버라이딩된 메서드는 부모 타입 참조변수로 호출하면 무엇이 실행되나요?

답변: 참조변수 타입이 부모여도 실제 객체가 자식 객체라면, 자식 클래스에서 오버라이딩한 메서드가 실행됩니다.

Q10. 상속과 조합의 차이는 무엇인가요?

답변: 상속은 is-a 관계일 때 사용합니다. 예를 들어 AdminUser is a User 관계는 상속이 자연스럽습니다.
조합은 has-a 관계일 때 사용합니다. 예를 들어 Car has an Engine 관계는 조합이 자연스럽습니다.

11. 오늘의 핵심 정리

  1. 상속은 부모 클래스의 속성과 기능을 자식 클래스가 물려받아 확장하는 것이다.
  2. Java에서는 상속에 extends 키워드를 사용한다.
  3. 상속은 “자식 is a 부모” 관계가 성립할 때 사용하는 것이 좋다.
  4. 상속을 사용하면 코드 재사용성과 확장성이 좋아진다.
  5. Java 클래스는 단일 상속만 가능하다.
  6. 자식 클래스는 부모의 public, protected 멤버에 접근할 수 있다.
  7. 부모의 private 멤버에는 자식도 직접 접근할 수 없다.
  8. 자식 객체 생성 시 부모 생성자가 먼저 실행된다.
  9. 자식 생성자에서는 기본적으로 부모의 기본 생성자 super()가 호출된다.
  10. 부모에게 기본 생성자가 없고 인자 생성자만 있으면 자식에서 super(값)를 명시해야 할 수 있다.
  11. 부모 타입 참조변수로 자식 객체를 참조할 수 있다.
  12. 부모 타입 참조변수는 부모 기준의 멤버만 접근할 수 있다.
  13. 자식이 오버라이딩한 메서드는 부모 타입 참조변수로 호출해도 자식 메서드가 실행된다.
  14. 자식 고유 멤버에 접근하려면 자식 타입으로 형변환해야 한다.
  15. 부모 타입 배열에는 부모 객체와 자식 객체를 함께 저장할 수 있다.
