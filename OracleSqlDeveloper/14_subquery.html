Oracle SQL Developer - Chapter 14

SUBQUERY

서브쿼리는 SQL문 안에 포함된 또 다른 SELECT문입니다.
이번 단원에서는 WHERE절 서브쿼리, SELECT절 서브쿼리, FROM절 서브쿼리, 단일행 서브쿼리, 다중행 서브쿼리,
다중 컬럼 서브쿼리, 상관 서브쿼리를 SQL Developer에서 직접 실행하며 이해할 수 있도록 정리합니다.

학습 목표 SQL 안에서 먼저 조회한 결과를
바깥 SQL의 조건, 컬럼, 임시 테이블로 활용한다.
실습 방향 EMP와 DEPT를 이용해 최소 연봉자, 평균 이하 연봉자,
직무별 최고 연봉자를 조회한다.
프로젝트 연결 평균 가격 이하 매물, 게시글별 댓글 수,
지역별 최고가 매물 조회와 연결한다.
Overview

1. 서브쿼리란?

서브쿼리는 하나의 SQL문 안에 포함된 또 다른 SELECT문입니다.
바깥 SQL이 직접 값을 알 수 없을 때, 안쪽 SELECT문이 먼저 값을 구하고 바깥 SQL은 그 결과를 조건값, 출력 컬럼, 임시 테이블처럼 사용합니다.

외부 쿼리 최종적으로 사용자가 보고 싶은 데이터를 조회합니다.
서브쿼리 외부 쿼리에서 사용할 값을 먼저 계산합니다.
결과 활용 조건, 컬럼, 임시 테이블 형태로
서브쿼리 결과를 사용합니다.

한 줄 정리

서브쿼리는 바깥 쿼리가 사용할 값을 안쪽 쿼리가 먼저 구해주는 구조입니다.
“최저 연봉이 얼마인지 먼저 구하고, 그 연봉을 받는 직원을 찾는다”처럼 생각하면 이해하기 쉽습니다.

Why Subquery?

2. 서브쿼리가 필요한 이유

조건값을 사람이 직접 알고 있을 때는 WHERE절에 값을 바로 넣을 수 있습니다. 하지만 실무에서는 조건값이 데이터에 따라 계속 바뀌거나, 먼저 계산해야 하는 경우가 많습니다.

직접 값을 넣는 방식

SELECT ENAME, SAL, JOB
FROM EMP
WHERE SAL = 800;

최저 연봉이 800이라는 사실을 사람이 미리 알고 있어야 합니다.
데이터가 바뀌면 SQL도 직접 수정해야 합니다.

서브쿼리 방식

SELECT ENAME, SAL, JOB
FROM EMP
WHERE SAL = (
    SELECT MIN(SAL)
    FROM EMP
);

DB가 먼저 최저 연봉을 계산하고, 바깥 쿼리가 그 값을 조건으로 사용합니다.

서브쿼리 실행 흐름

1. 안쪽 SELECT 실행
   SELECT MIN(SAL) FROM EMP

2. 결과값 반환
   예: 800

3. 바깥 SELECT 실행
   WHERE SAL = 800

4. 최저 연봉자 출력
Visual

3. 그림처럼 이해하기

서브쿼리는 바깥 쿼리의 조건을 완성하기 위해 먼저 실행되는 작은 SQL입니다. 특히 평균, 최댓값, 최솟값처럼 먼저 계산해야 하는 값이 있을 때 자주 사용합니다.

1 EMP 전체 급여 확인

800, 1600, 1250, 2975, 5000...

→
2 서브쿼리 실행

SELECT AVG(SAL) FROM EMP

→
3 평균값 반환

예: 2073.21

→
4 바깥 쿼리 실행

WHERE SAL <= 2073.21

평균 이하 연봉자 조회

SELECT ENAME, SAL, DEPTNO, HIREDATE
FROM EMP
WHERE SAL <= (
    SELECT AVG(SAL)
    FROM EMP
);
Position

4. 서브쿼리의 위치

서브쿼리는 WHERE절, SELECT절, FROM절 등에 위치할 수 있습니다. 위치에 따라 역할이 달라지므로, 먼저 “서브쿼리가 어디에 있는가?”를 확인해야 합니다.

WHERE절 서브쿼리

조건값으로 사용합니다.

SELECT ENAME, SAL
FROM EMP
WHERE SAL > (
    SELECT AVG(SAL)
    FROM EMP
);

SELECT절 서브쿼리

결과 컬럼으로 사용합니다.

SELECT E.ENAME,
       (
           SELECT D.DNAME
           FROM DEPT D
           WHERE D.DEPTNO = E.DEPTNO
       ) AS DNAME
FROM EMP E;

FROM절 서브쿼리

임시 테이블처럼 사용합니다.

SELECT E.ENAME, E.SAL
FROM (
    SELECT ENAME, SAL
    FROM EMP
    WHERE SAL >= 3000
) E;
Syntax

5. 서브쿼리 기본 문법과 작성 규칙

서브쿼리는 괄호로 감싸고, 보통 연산자의 오른쪽에 작성합니다. 가장 중요한 것은 서브쿼리 결과가 하나인지 여러 개인지 판단하고 그에 맞는 연산자를 사용하는 것입니다.

기본 구조

SELECT 컬럼명
FROM 테이블명
WHERE 컬럼명 연산자 (
    SELECT 컬럼명
    FROM 테이블명
);

작성 규칙

1. 서브쿼리는 괄호로 감싼다.
2. 서브쿼리는 보통 연산자의 오른쪽에 작성한다.
3. 단일행 결과에는 단일행 연산자를 사용한다.
4. 다중행 결과에는 IN, NOT IN을 사용한다.
5. SELECT절 서브쿼리는 한 행당 한 값만 반환해야 한다.
Single Row

6. 단일행 서브쿼리

단일행 서브쿼리는 결과가 한 행, 한 값으로 나오는 서브쿼리입니다. `MIN`, `MAX`, `AVG`, `COUNT`처럼 하나의 집계값을 반환하는 경우가 많습니다.

사용 연산자 = > >= < <= <> !=

6-1. 최저 연봉자 조회

SELECT ENAME,
       SAL,
       JOB,
       SAL * 1.5 AS "150% 인상 연봉"
FROM EMP
WHERE SAL = (
    SELECT MIN(SAL)
    FROM EMP
);

EMP 전체 최저 연봉을 먼저 구하고, 그 연봉을 받는 직원을 조회합니다.

6-2. 평균 이하 연봉자 조회

SELECT ENAME,
       SAL,
       DEPTNO,
       HIREDATE
FROM EMP
WHERE SAL <= (
    SELECT AVG(SAL)
    FROM EMP
);

전체 평균 연봉 이하의 직원을 조회합니다.

6-3. 평균보다 많이 받는 직원 조회

SELECT ENAME,
       SAL
FROM EMP
WHERE SAL > (
    SELECT AVG(SAL)
    FROM EMP
);

교안의 평균 급여보다 많이 받는 사람 예제와 같은 구조입니다.

Multi Row

7. 다중행 서브쿼리

다중행 서브쿼리는 결과가 여러 행으로 나오는 서브쿼리입니다. 여러 행을 반환하는 서브쿼리에는 `=`, `>` 같은 단일행 연산자를 사용하면 안 되고, `IN`, `NOT IN` 같은 복수행 연산자를 사용해야 합니다.

대표 연산자 IN NOT IN

7-1. 최대/최소 연봉자 조회

SELECT SAL,
       ENAME,
       JOB
FROM EMP
WHERE SAL IN (
    (SELECT MAX(SAL) FROM EMP),
    (SELECT MIN(SAL) FROM EMP)
)
ORDER BY SAL;

전체 최고 연봉과 최저 연봉을 먼저 구하고, 그 둘 중 하나에 해당하는 직원을 조회합니다.

7-2. 직무별 최고 연봉 값 목록

SELECT JOB, MAX(SAL)
FROM EMP
GROUP BY JOB;

JOB별 최고 연봉을 구하면 직무 개수만큼 여러 행이 반환됩니다.

Multi Column

8. 다중 컬럼 서브쿼리

다중 컬럼 서브쿼리는 하나의 컬럼이 아니라 여러 컬럼 조합을 비교하는 서브쿼리입니다.
직무별 최고 연봉자를 정확히 찾으려면 `SAL`만 비교하는 것보다 `(JOB, SAL)` 조합을 비교하는 것이 좋습니다.

조합 비교 구조

바깥 쿼리의 (JOB, SAL)

예:
('CLERK', 1300)
('SALESMAN', 1600)

서브쿼리 결과:
('CLERK', 1300)
('SALESMAN', 1600)
('MANAGER', 2975)

목록에 조합이 있으면 출력

8-1. 직무별 최고 연봉자 조회

SELECT ENAME,
       SAL,
       JOB,
       DEPTNO,
       HIREDATE
FROM EMP
WHERE (JOB, SAL) IN (
    SELECT JOB, MAX(SAL)
    FROM EMP
    GROUP BY JOB
);

8-2. 컬럼 개수 불일치 예

-- 잘못된 예
SELECT ENAME, JOB, SAL
FROM EMP
WHERE (JOB, SAL) IN (
    SELECT MAX(SAL)
    FROM EMP
    GROUP BY JOB
);

바깥 조건은 두 컬럼 `(JOB, SAL)`을 비교하는데, 서브쿼리는 한 컬럼만 반환하므로 잘못된 구조입니다.

SELECT Subquery

9. SELECT절 서브쿼리

SELECT절 서브쿼리는 결과 컬럼 중 하나로 서브쿼리 결과를 표시하는 방식입니다. 바깥 쿼리의 각 행마다 하나의 값을 계산해서 붙이는 느낌으로 이해하면 됩니다.

9-1. 부서명 가져오기

SELECT E.ENAME,
       E.SAL,
       E.DEPTNO,
       (
           SELECT D.DNAME
           FROM DEPT D
           WHERE D.DEPTNO = E.DEPTNO
       ) AS DNAME
FROM EMP E;

각 사원의 DEPTNO를 기준으로 DEPT에서 부서명을 찾아 결과 컬럼으로 붙입니다.

9-2. 게시글별 댓글 수 예시

SELECT B.BOARD_ID,
       B.TITLE,
       B.WRITER_ID,
       (
           SELECT COUNT(*)
           FROM COMMENTS C
           WHERE C.BOARD_ID = B.BOARD_ID
       ) AS COMMENT_COUNT
FROM BOARD B;

교안의 메모글별 댓글 수 예제를 게시판 프로젝트 형태로 바꾼 예시입니다.

9-3. SELECT절 서브쿼리 주의

-- 잘못된 예
SELECT ENAME,
       (
           SELECT DNAME
           FROM DEPT
       ) AS DNAME
FROM EMP;

SELECT절 서브쿼리는 각 행마다 하나의 값만 반환해야 합니다. DEPT 전체를 조회하면 여러 행이 반환되어 오류가 발생할 수 있습니다.

FROM Subquery

10. FROM절 서브쿼리, 인라인 뷰

FROM절 서브쿼리는 서브쿼리 결과를 하나의 임시 테이블처럼 사용하는 방식입니다.
먼저 복잡한 결과를 만들고, 그 결과에 별칭을 붙여 바깥 쿼리에서 다시 조회하거나 JOIN할 수 있습니다.

10-1. 기본 구조

SELECT 컬럼명
FROM (
    SELECT 컬럼명
    FROM 테이블명
    WHERE 조건
) 별칭;

10-2. 직무별 최고 연봉자 결과를 임시 테이블로 사용

SELECT E.ENAME,
       E.SAL,
       D.DEPTNO,
       E.HIREDATE,
       D.DNAME
FROM (
    SELECT ENAME,
           SAL,
           DEPTNO,
           HIREDATE
    FROM EMP
    WHERE (JOB, SAL) IN (
        SELECT JOB, MAX(SAL)
        FROM EMP
        GROUP BY JOB
    )
) E
JOIN DEPT D
ON E.DEPTNO = D.DEPTNO;

FROM절 서브쿼리 E에서 먼저 직무별 최고 연봉자를 만들고, 그 결과를 DEPT와 JOIN합니다.

Correlated

11. 상관 서브쿼리

상관 서브쿼리는 서브쿼리가 바깥 쿼리의 컬럼을 참조하는 구조입니다. 바깥 쿼리의 각 행마다 서브쿼리가 그 행의 값을 참고해서 실행되는 형태로 이해하면 됩니다.

게시글별 댓글 수

SELECT B.BOARD_ID,
       B.TITLE,
       (
           SELECT COUNT(*)
           FROM COMMENTS C
           WHERE C.BOARD_ID = B.BOARD_ID
       ) AS COMMENT_COUNT
FROM BOARD B;

서브쿼리 안의 B.BOARD_ID는 바깥 쿼리의 게시글 ID입니다.

사원별 매니저 이름

SELECT ENAME || '의 매니저는 ' ||
       (
           SELECT ENAME
           FROM EMP
           WHERE EMPNO = E1.MGR
       ) || '입니다.' AS "매니저 정보"
FROM EMP E1
WHERE MGR IS NOT NULL;

서브쿼리 안에서 바깥 쿼리의 E1.MGR 값을 참조합니다.

Subquery vs Join

12. 서브쿼리와 JOIN 비교

서브쿼리와 JOIN은 비슷한 결과를 만들 수 있지만 목적이 다릅니다. JOIN은 여러 테이블의 컬럼을 함께 출력할 때 유리하고, 서브쿼리는 먼저 계산한 값을 조건이나 보조 컬럼으로 사용할 때 유리합니다.

JOIN으로 부서명 출력

SELECT E.ENAME,
       E.SAL,
       D.DNAME
FROM EMP E JOIN DEPT D
ON E.DEPTNO = D.DEPTNO;

여러 테이블의 컬럼을 함께 가져올 때 자연스럽습니다.

SELECT절 서브쿼리로 부서명 출력

SELECT E.ENAME,
       E.SAL,
       (
           SELECT D.DNAME
           FROM DEPT D
           WHERE D.DEPTNO = E.DEPTNO
       ) AS DNAME
FROM EMP E;

각 사원 행마다 부서명을 하나씩 계산해서 붙입니다.

서브쿼리가 자연스러운 경우

SELECT ENAME, SAL
FROM EMP
WHERE SAL > (
    SELECT AVG(SAL)
    FROM EMP
);

평균처럼 먼저 계산해야 하는 값을 조건으로 사용할 때 적합합니다.

Practice

13. 실습 문제 연결

실습 문제 교안의 SUBQUERY 문제는 최소 연봉자, 최대·최소 연봉자, 평균 이하 연봉자, 직무별 최고 연봉자, 서브쿼리와 JOIN 결합 문제로 구성되어 있습니다.
문제를 읽을 때는 서브쿼리가 “무엇을 먼저 구하는지”를 먼저 파악해야 합니다.

문제 1. 최소 연봉자 조회

SELECT ENAME, SAL, JOB, SAL * 1.5
FROM EMP
WHERE SAL = (
    SELECT MIN(SAL)
    FROM EMP
);

최소 연봉을 먼저 구하고, 그 연봉을 받는 직원을 조회합니다.

문제 2. 최대/최소 연봉자 조회

SELECT SAL, ENAME, JOB
FROM EMP
WHERE SAL IN (
    (SELECT MAX(SAL) FROM EMP),
    (SELECT MIN(SAL) FROM EMP)
)
ORDER BY SAL;

최고 연봉과 최저 연봉 중 하나에 해당하는 직원을 조회합니다.

문제 3. 평균 이하 연봉자 조회

SELECT ENAME, SAL, DEPTNO, HIREDATE
FROM EMP
WHERE SAL <= (
    SELECT AVG(SAL)
    FROM EMP
);

문제 4. 직무별 최고 연봉자 조회

SELECT ENAME, SAL, DEPTNO, HIREDATE
FROM EMP E
WHERE (JOB, SAL) IN (
    SELECT JOB, MAX(SAL)
    FROM EMP
    GROUP BY JOB
);

해답 SQL 기준으로는 JOB별 최고 연봉자를 찾는 구조입니다.

문제 5. 직무별 최고 연봉자 + 부서명

SELECT E.ENAME,
       E.SAL,
       D.DEPTNO,
       E.HIREDATE,
       D.DNAME
FROM EMP E JOIN DEPT D
ON E.DEPTNO = D.DEPTNO
WHERE (E.JOB, E.SAL) IN (
    SELECT JOB, MAX(SAL)
    FROM EMP
    GROUP BY JOB
);

문제 읽는 순서

1. 바깥 쿼리는 무엇을 출력하는가?
2. 서브쿼리는 무엇을 먼저 계산하는가?
3. 서브쿼리 결과는 한 값인가, 여러 행인가?
4. 단일행이면 =, >, < 계열을 확인한다.
5. 다중행이면 IN, NOT IN을 확인한다.
6. 다중 컬럼이면 비교 컬럼 개수가 맞는지 확인한다.
Project

14. Java / Spring / Oracle 프로젝트 연결

서브쿼리는 실무 프로젝트에서 평균값 기준 필터링, 최고값/최저값 찾기, 게시글별 댓글 수 계산, 사용자별 최근 로그 조회, 추천 후보 추출 등에 자주 사용됩니다.

평균 가격 이하 매물 조회

SELECT PROPERTY_ID,
       TITLE,
       DEAL_AMOUNT
FROM PROPERTY
WHERE DEAL_AMOUNT <= (
    SELECT AVG(DEAL_AMOUNT)
    FROM PROPERTY
);

전체 평균 가격 이하의 매물을 조회합니다.

최고가 매물 조회

SELECT PROPERTY_ID,
       TITLE,
       DEAL_AMOUNT
FROM PROPERTY
WHERE DEAL_AMOUNT = (
    SELECT MAX(DEAL_AMOUNT)
    FROM PROPERTY
);

관리자 대시보드의 최고가 매물 카드에 사용할 수 있습니다.

지역별 최고가 매물 조회

SELECT PROPERTY_ID,
       TITLE,
       REGION_ID,
       DEAL_AMOUNT
FROM PROPERTY
WHERE (REGION_ID, DEAL_AMOUNT) IN (
    SELECT REGION_ID, MAX(DEAL_AMOUNT)
    FROM PROPERTY
    GROUP BY REGION_ID
);

REGION_ID와 DEAL_AMOUNT의 조합을 비교하는 다중 컬럼 서브쿼리입니다.

Mistake

15. 자주 하는 실수

  • 실수 1. 단일행 연산자에 다중행 서브쿼리를 사용한다. GROUP BY가 있는 서브쿼리는 여러 행을 반환할 수 있으므로 = 대신 IN을 검토해야 합니다.
  • 실수 2. 다중행 서브쿼리에 = 를 사용한다. 여러 행을 반환하는 서브쿼리에는 IN, NOT IN 같은 다중행 연산자를 사용해야 합니다.
  • 실수 3. 서브쿼리를 괄호로 감싸지 않는다. 서브쿼리는 WHERE SAL = (SELECT MIN(SAL) FROM EMP)처럼 괄호로 감싸야 합니다.
  • 실수 4. SELECT절 서브쿼리가 여러 행을 반환한다. SELECT절 서브쿼리는 바깥 쿼리의 각 행마다 하나의 값만 반환해야 합니다.
  • 실수 5. 다중 컬럼 서브쿼리의 컬럼 개수를 맞추지 않는다. (JOB, SAL)을 비교한다면 서브쿼리도 JOB, MAX(SAL)처럼 두 컬럼을 반환해야 합니다.
  • 실수 6. 서브쿼리와 JOIN을 무조건 같은 것으로 생각한다. JOIN은 여러 테이블 컬럼을 함께 출력할 때, 서브쿼리는 먼저 계산한 값을 조건이나 컬럼으로 사용할 때 유리합니다.
  • 실수 7. NOT IN과 NULL 문제를 모른다. NOT IN 서브쿼리 결과에 NULL이 포함되면 의도와 다른 결과가 나올 수 있으므로 NULL을 제거하는 조건을 고려해야 합니다.
Interview

16. 면접 포인트

Q1. 서브쿼리란 무엇인가요?

서브쿼리는 SQL문 안에 포함된 또 다른 SELECT문입니다. 보통 괄호로 감싸서 작성하며, 바깥 쿼리에서 사용할 조건값, 결과 컬럼, 임시 테이블을 만들 때 사용합니다.

Q2. 단일행 서브쿼리와 다중행 서브쿼리의 차이는 무엇인가요?

단일행 서브쿼리는 결과가 한 값으로 나오며 =, >, <, >=, <= 같은 단일행 연산자를 사용합니다.
다중행 서브쿼리는 여러 행을 반환하며 IN, NOT IN 같은 다중행 연산자를 사용합니다.

Q3. 서브쿼리는 어디에 사용할 수 있나요?

서브쿼리는 WHERE절, SELECT절, FROM절 등에 사용할 수 있습니다. WHERE절에서는 조건값, SELECT절에서는 보조 컬럼, FROM절에서는 임시 테이블처럼 사용합니다.

Q4. 다중행 서브쿼리에 = 를 사용하면 어떤 문제가 생기나요?

= 연산자는 하나의 값과 비교하는 단일행 연산자입니다. 서브쿼리가 여러 행을 반환하면 어떤 값과 비교해야 할지 알 수 없으므로 오류가 발생할 수 있습니다.
이런 경우 IN 같은 다중행 연산자를 사용해야 합니다.

Q5. 다중 컬럼 서브쿼리는 무엇인가요?

다중 컬럼 서브쿼리는 여러 컬럼의 조합을 비교하는 서브쿼리입니다.
예를 들어 직무별 최고 연봉자를 찾을 때 (JOB, SAL) IN (SELECT JOB, MAX(SAL) FROM EMP GROUP BY JOB)처럼 사용할 수 있습니다.

Q6. SELECT절 서브쿼리는 어떤 경우에 사용하나요?

SELECT절 서브쿼리는 각 행에 추가로 계산된 단일 값을 붙이고 싶을 때 사용합니다. 예를 들어 게시글 목록에서 각 게시글의 댓글 수를 함께 보여줄 때 사용할 수 있습니다.

Q7. FROM절 서브쿼리는 무엇인가요?

FROM절 서브쿼리는 서브쿼리 결과를 하나의 임시 테이블처럼 사용하는 방식입니다.
복잡한 조건으로 먼저 결과 집합을 만든 뒤, 그 결과를 다시 조인하거나 필터링할 때 사용합니다.

Q8. 상관 서브쿼리란 무엇인가요?

상관 서브쿼리는 서브쿼리가 바깥 쿼리의 컬럼을 참조하는 구조입니다.
예를 들어 게시글별 댓글 수를 구할 때 서브쿼리 안에서 C.BOARD_ID = B.BOARD_ID처럼 바깥 쿼리의 게시글 ID를 참조하면 상관 서브쿼리입니다.

읽는 순서

1. 바깥 쿼리가 무엇을 출력하려는지 본다.
2. 서브쿼리가 무엇을 먼저 계산하는지 본다.
3. 서브쿼리 결과가 단일 값인지 여러 행인지 판단한다.
4. 단일 값이면 =, >, < 같은 연산자를 확인한다.
5. 여러 행이면 IN, NOT IN을 확인한다.
6. SELECT절 서브쿼리라면 각 행마다 하나의 값을 반환하는지 본다.
7. FROM절 서브쿼리라면 임시 테이블처럼 사용되는지 본다.

학습 메모

오늘의 핵심

서브쿼리는 SQL 안에 들어가는 또 다른 SELECT문입니다. 바깥 쿼리에서 필요한 값을 안쪽 쿼리가 먼저 계산해준다고 이해하면 됩니다.

기본 문법

SELECT 컬럼명
FROM 테이블명
WHERE 컬럼명 연산자 (
    SELECT 컬럼명
    FROM 테이블명
);

서브쿼리 위치

  • WHERE절 : 조건값으로 사용
  • SELECT절 : 결과 컬럼으로 사용
  • FROM절 : 임시 테이블처럼 사용

연산자 선택

  • 단일행 : =, >, >=, <, <=, <>, !=
  • 다중행 : IN, NOT IN
  • 다중 컬럼 : (JOB, SAL) IN (...)

실습 확인 순서

  1. MIN(SAL) 단독 실행
  2. 최저 연봉자 서브쿼리 실행
  3. AVG(SAL)로 평균 이하 연봉자 조회
  4. MAX/MIN을 IN으로 연결
  5. JOB별 MAX(SAL) 결과 확인
  6. (JOB, SAL) IN 다중 컬럼 서브쿼리 실행
  7. SELECT절 서브쿼리로 부서명 또는 댓글 수 조회
  8. FROM절 서브쿼리와 JOIN 결합 확인

주의할 점

서브쿼리 결과가 여러 행인데 = 연산자를 사용하면 오류가 날 수 있습니다.
SELECT절 서브쿼리는 각 행마다 하나의 값만 반환해야 하며, 다중 컬럼 비교는 바깥 컬럼 수와 서브쿼리 컬럼 수가 일치해야 합니다.

프로젝트 연결

평균 가격 이하 매물, 최고가 매물, 지역별 최고가 매물, 게시글별 댓글 수, 사용자별 최근 조회 로그 같은 기능은 서브쿼리와 자연스럽게 연결됩니다.
