Overview

1. ORDER BY란?

ORDER BY는 SELECT 조회 결과의 행 순서를 정렬하는 절입니다.
SELECT가 어떤 컬럼을 보여줄지 결정하고, WHERE가 어떤 행을 남길지 결정한다면, ORDER BY는 최종 결과를 어떤 순서로 보여줄지 결정합니다.

FROM 어느 테이블에서
데이터를 가져올지 결정합니다.
WHERE 조건에 맞는 행만 남깁니다.
SELECT 남은 행에서
어떤 컬럼을 출력할지 결정합니다.
ORDER BY 최종 조회 결과의 행 순서를
정렬합니다.

한 줄 정리

ORDER BY는 조회 결과를 사용자가 보기 좋은 순서로 정리하는 절입니다. 정렬 순서가 중요하다면 반드시 ORDER BY를 작성해야 합니다.

Syntax

2. ORDER BY 기본 문법

ORDER BY는 SELECT문의 맨 마지막에 작성합니다. ASC는 오름차순, DESC는 내림차순입니다. ASC는 기본값이므로 생략할 수 있습니다.

2-1. 기본 구조

SELECT 컬럼명
FROM 테이블명
WHERE 조건
ORDER BY 정렬기준 [ASC | DESC];

2-2. 오름차순 정렬

SELECT ENAME, SAL
FROM EMP
ORDER BY SAL ASC;

SAL이 낮은 값에서 높은 값으로 정렬됩니다. ASC는 기본값이므로 생략할 수 있습니다.

2-3. 내림차순 정렬

SELECT ENAME, SAL
FROM EMP
ORDER BY SAL DESC;

SAL이 높은 값에서 낮은 값으로 정렬됩니다. 내림차순은 반드시 DESC를 명시해야 합니다.

2-4. WHERE와 함께 사용

SELECT ENAME, JOB, SAL
FROM EMP
WHERE JOB = 'SALESMAN'
ORDER BY SAL;

먼저 JOB이 SALESMAN인 행만 남긴 뒤, 그 결과를 SAL 오름차순으로 정렬합니다.

Visual

4. 그림처럼 이해하기

ORDER BY는 조회 결과의 행을 특정 기준으로 다시 줄 세우는 과정입니다. 숫자, 문자, 날짜는 각각 정렬 방향을 다르게 해석할 수 있습니다.

ENAME SAL JOB HIREDATE DEPTNO
SMITH 800 CLERK 1980-12-17 20
ALLEN 1600 SALESMAN 1981-02-20 30
WARD 1250 SALESMAN 1981-02-22 30
KING 5000 PRESIDENT 1981-11-17 10
ORDER BY SAL ASC
800 1250 1600 5000

급여가 낮은 값에서 높은 값으로 정렬됩니다.

ORDER BY SAL DESC
5000 1600 1250 800

급여가 높은 값에서 낮은 값으로 정렬됩니다.

ORDER BY HIREDATE ASC
과거 → 최근

날짜 오름차순은 오래된 날짜에서 최근 날짜 순서입니다.

ORDER BY HIREDATE DESC
최근 → 과거

날짜 내림차순은 최근 날짜에서 오래된 날짜 순서입니다.

ASC / DESC

5. 오름차순과 내림차순

ASC는 오름차순, DESC는 내림차순입니다. ASC는 기본값이라 생략할 수 있지만, DESC는 반드시 작성해야 합니다.

ASC 오름차순

작은 값에서 큰 값으로 정렬합니다.

SELECT ENAME, SAL
FROM EMP
ORDER BY SAL ASC;

ASC 생략

ORDER BY의 기본 방향은 오름차순입니다.

SELECT ENAME, SAL
FROM EMP
ORDER BY SAL;

DESC 내림차순

큰 값에서 작은 값으로 정렬합니다.

SELECT ENAME, SAL
FROM EMP
ORDER BY SAL DESC;

문자 정렬

문자는 알파벳 또는 사전식 순서로 정렬됩니다.

SELECT ENAME, JOB, SAL
FROM EMP
ORDER BY ENAME ASC;
WHERE + ORDER BY

6. WHERE와 ORDER BY 함께 사용하기

WHERE는 먼저 행을 걸러내고, ORDER BY는 남은 결과를 정렬합니다. 즉, 조건 검색 후 정렬하는 흐름으로 이해하면 됩니다.

6-1. SALESMAN 중 급여 낮은 순

SELECT ENAME, DEPTNO, SAL, COMM
FROM EMP
WHERE JOB = 'SALESMAN'
ORDER BY SAL;

6-2. SALESMAN 중 보너스 100 이상, 급여 낮은 순

SELECT ENAME, DEPTNO, SAL, COMM
FROM EMP
WHERE JOB = 'SALESMAN'
  AND COMM >= 100
ORDER BY SAL;

실습 문제 교안의 ORDER BY 문제와 직접 연결되는 형태입니다.

6-3. 급여 1500 이상, 급여 높은 순

SELECT ENAME, JOB, SAL
FROM EMP
WHERE SAL >= 1500
ORDER BY SAL DESC;

6-4. 이름에 A가 들어간 직원 정렬

SELECT *
FROM EMP
WHERE ENAME LIKE '%A%'
ORDER BY SAL ASC;
Multi Sort

7. 여러 컬럼으로 정렬하기

여러 컬럼으로 정렬하면 첫 번째 기준으로 먼저 정렬하고, 첫 번째 기준의 값이 같은 행들끼리 두 번째 기준으로 다시 정렬합니다.

여러 컬럼 정렬의 의미

ORDER BY SAL ASC, DEPTNO DESC

1차 정렬: SAL 오름차순
2차 정렬: SAL이 같은 행끼리 DEPTNO 내림차순

7-1. 급여 오름차순 + 부서번호 내림차순

SELECT ENAME, SAL, DEPTNO
FROM EMP
ORDER BY SAL ASC, DEPTNO DESC;

7-2. 직무 오름차순 + 급여 내림차순

SELECT ENAME, JOB, SAL
FROM EMP
ORDER BY JOB ASC, SAL DESC;

7-3. 실습 문제 연결

SELECT *
FROM EMP
WHERE ENAME LIKE '%A%'
ORDER BY SAL ASC, DEPTNO DESC;

이름에 A가 들어간 사원을 조회하고, 먼저 급여 오름차순, 급여가 같으면 부서번호 내림차순으로 정렬합니다.

Alias / Expression

8. 별칭, 표현식, 컬럼 번호로 정렬하기

ORDER BY에서는 컬럼명뿐 아니라 SELECT절의 별칭, 계산 표현식, 컬럼 위치 번호도 사용할 수 있습니다.
다만 실무에서는 가독성과 유지보수를 위해 컬럼명이나 별칭 사용을 권장합니다.

별칭으로 정렬

SELECT ENAME AS "사원명",
       SAL * 12 AS "연봉"
FROM EMP
ORDER BY "연봉" DESC;

한글 별칭은 큰따옴표로 감쌉니다.

표현식으로 정렬

SELECT ENAME, SAL, SAL * 12
FROM EMP
ORDER BY SAL * 12 DESC;

계산 결과를 기준으로 정렬합니다.

영문 별칭으로 정렬

SELECT ENAME,
       SAL,
       SAL * 12 AS ANNUAL_SAL
FROM EMP
ORDER BY ANNUAL_SAL DESC;

실무에서는 영문 별칭이 유지보수에 더 편리할 수 있습니다.

컬럼 위치 번호로 정렬

SELECT ENAME, JOB, SAL
FROM EMP
ORDER BY 3 DESC;

SELECT절의 세 번째 컬럼인 SAL 기준으로 내림차순 정렬합니다.

Hidden Column

9. SELECT에 없는 컬럼으로 정렬하기

Oracle에서는 SELECT 결과에 출력하지 않은 컬럼도 ORDER BY 기준으로 사용할 수 있습니다. 하지만 학습할 때는 정렬 기준 컬럼도 함께 출력하는 것이 결과를 확인하기 쉽습니다.

가능하지만 확인이 어려운 예

SELECT ENAME, JOB
FROM EMP
ORDER BY SAL DESC;

결과에는 SAL이 보이지 않지만 SAL 기준으로 정렬됩니다.

학습할 때 권장하는 예

SELECT ENAME, JOB, SAL
FROM EMP
ORDER BY SAL DESC;

정렬 기준인 SAL을 함께 출력하면 결과를 확인하기 쉽습니다.

NULL Sort

10. NULL이 있는 컬럼 정렬

COMM처럼 NULL 값이 있는 컬럼을 정렬할 때는 NULL이 어느 위치에 나오는지도 확인해야 합니다.
Oracle에서는 NULLS FIRST, NULLS LAST를 사용해 NULL의 위치를 명시할 수 있습니다.

10-1. COMM 오름차순 정렬

SELECT ENAME, COMM
FROM EMP
ORDER BY COMM;

10-2. COMM 내림차순 정렬

SELECT ENAME, COMM
FROM EMP
ORDER BY COMM DESC;

10-3. NULL을 앞에 표시

SELECT ENAME, COMM
FROM EMP
ORDER BY COMM ASC NULLS FIRST;

10-4. NULL을 뒤에 표시

SELECT ENAME, COMM
FROM EMP
ORDER BY COMM ASC NULLS LAST;
Practice

11. 실습 문제 연결

실습 문제 교안의 ORDER BY 문제는 정렬 방향과 여러 컬럼 정렬을 확인하는 데 좋습니다.
문제를 읽을 때는 “어떤 데이터를 조회하는지”와 “어떤 순서로 정렬하는지”를 분리해서 읽어야 합니다.

문제 1. 연봉 높은 순

EMP 테이블에서 연봉이 가장 높은 직원부터 모든 데이터를 출력합니다.

SELECT *
FROM EMP
ORDER BY SAL DESC;

문제 2. SALESMAN + 보너스 100 이상 + 연봉 낮은 순

SELECT ENAME, DEPTNO, SAL, COMM
FROM EMP
WHERE JOB = 'SALESMAN'
  AND COMM >= 100
ORDER BY SAL;

문제 3. 이름에 A 포함 + 여러 컬럼 정렬

SELECT *
FROM EMP
WHERE ENAME LIKE '%A%'
ORDER BY SAL ASC, DEPTNO DESC;

문제 읽는 순서

1. 어떤 테이블에서?
   → EMP

2. 어떤 행을 조회?
   → WHERE ENAME LIKE '%A%'

3. 어떤 기준으로 정렬?
   → SAL ASC
   → DEPTNO DESC

4. 정렬 우선순위는?
   → 1차 SAL, 2차 DEPTNO

12. Java / Spring / Oracle 프로젝트 연결

ORDER BY는 실제 프로젝트에서 목록 화면, 검색 결과, 최신순 정렬, 가격순 정렬, 추천 점수순 정렬, 페이징 처리와 직접 연결됩니다.

게시판 최신순 목록

SELECT BOARD_ID, TITLE, WRITER_ID, CREATED_AT
FROM BOARD
ORDER BY CREATED_AT DESC;

최신 글이 위에 나오도록 정렬합니다.

게시글 검색 결과 최신순

SELECT BOARD_ID, TITLE, WRITER_ID, CREATED_AT
FROM BOARD
WHERE TITLE LIKE '%' || ? || '%'
ORDER BY CREATED_AT DESC;

검색 결과를 최신 작성일 순으로 정렬합니다.

관리자 회원 목록 최근 가입순

SELECT USER_ID, LOGIN_ID, EMAIL, REGISTERED_AT
FROM USERS
ORDER BY REGISTERED_AT DESC;

관리자 페이지에서 최근 가입자를 먼저 보여줍니다.

회원 목록 아이디순

SELECT USER_ID, LOGIN_ID, EMAIL
FROM USERS
ORDER BY LOGIN_ID ASC;

아이디를 기준으로 오름차순 정렬합니다.

매물 가격 낮은 순

SELECT PROPERTY_ID, TITLE, DEAL_AMOUNT, AREA_SQM
FROM PROPERTY
WHERE DEAL_TYPE = 'SALE'
ORDER BY DEAL_AMOUNT ASC;

매매 매물 중 가격이 낮은 순으로 보여줍니다.

매물 가격 높은 순

SELECT PROPERTY_ID, TITLE, DEAL_AMOUNT, AREA_SQM
FROM PROPERTY
WHERE DEAL_TYPE = 'SALE'
ORDER BY DEAL_AMOUNT DESC;

매매 매물 중 가격이 높은 순으로 보여줍니다.

추천 점수순

SELECT USER_ID, PROPERTY_ID, SCORE
FROM PROPERTY_RECOMMENDATION
WHERE USER_ID = ?
ORDER BY SCORE DESC;

추천 점수가 높은 매물부터 출력합니다.

페이징 처리

SELECT BOARD_ID, TITLE, CREATED_AT
FROM BOARD
ORDER BY BOARD_ID DESC
OFFSET 0 ROWS FETCH NEXT 10 ROWS ONLY;

정렬 기준이 있어야 안정적인 페이징이 가능합니다.

Spring Data JPA 메서드

List<BoardEntity> findAllByOrderByCreatedAtDesc();

Pageable pageable = PageRequest.of(
    0,
    10,
    Sort.by(Sort.Direction.DESC, "createdAt")
);

JPA의 Sort와 Pageable은 SQL의 ORDER BY와 연결됩니다.

Mistake

13. 자주 하는 실수

  • 실수 1. ORDER BY 위치를 잘못 쓴다. ORDER BY는 SELECT문의 마지막에 작성해야 합니다.
  • 실수 2. DESC를 SELECT절에 붙인다. SELECT ENAME, SAL DESC가 아니라 ORDER BY SAL DESC입니다.
  • 실수 3. ASC가 기본값이라는 것을 모른다. ORDER BY SAL은 ORDER BY SAL ASC와 같습니다.
  • 실수 4. ORDER BY 없는 결과 순서를 믿는다. 정렬이 필요하면 반드시 ORDER BY를 작성해야 합니다.
  • 실수 5. 여러 컬럼 정렬의 의미를 오해한다. 첫 번째 기준으로 정렬하고, 값이 같은 행끼리 두 번째 기준이 적용됩니다.
  • 실수 6. 컬럼 번호 정렬을 남용한다. ORDER BY 3 DESC는 SELECT 컬럼 순서가 바뀌면 의미가 바뀔 수 있습니다.
  • 실수 7. 정렬 기준 컬럼을 결과에서 확인하지 않는다. 학습할 때는 정렬 기준 컬럼도 SELECT에 포함하는 것이 좋습니다.
  • 실수 8. NULL 정렬 위치를 확인하지 않는다. NULL이 포함된 컬럼은 NULL이 앞에 나오는지 뒤에 나오는지 확인해야 합니다.
Interview

14. 면접 포인트

Q1. ORDER BY절은 무엇인가요?

ORDER BY절은 SELECT 조회 결과의 행 순서를 정렬할 때 사용하는 절입니다.
숫자, 문자, 날짜 컬럼 등을 기준으로 오름차순 또는 내림차순 정렬할 수 있으며, SELECT문의 마지막에 작성합니다.

Q2. ASC와 DESC의 차이는 무엇인가요?

ASC는 오름차순이고 DESC는 내림차순입니다. 숫자는 ASC일 때 작은 값에서 큰 값으로, DESC일 때 큰 값에서 작은 값으로 정렬됩니다.
날짜는 ASC일 때 과거에서 최근, DESC일 때 최근에서 과거로 정렬됩니다.

Q3. ORDER BY를 생략하면 결과 순서가 보장되나요?

아닙니다. ORDER BY를 사용하지 않으면 조회 결과의 순서는 보장되지 않습니다. 명확한 정렬이 필요하면 반드시 ORDER BY를 사용해야 합니다.

Q4. 여러 컬럼으로 정렬할 때 어떻게 동작하나요?

먼저 첫 번째 컬럼을 기준으로 정렬하고, 첫 번째 컬럼의 값이 같은 행들끼리 두 번째 컬럼 기준으로 다시 정렬합니다.
예를 들어 ORDER BY SAL ASC, DEPTNO DESC는 급여 오름차순 후, 급여가 같은 행끼리 부서번호 내림차순을 적용합니다.

Q5. SELECT에 없는 컬럼으로 ORDER BY 할 수 있나요?

Oracle에서는 SELECT 결과에 출력하지 않은 컬럼도 ORDER BY 기준으로 사용할 수 있습니다.
다만 학습이나 유지보수 측면에서는 정렬 기준 컬럼을 함께 출력하거나 명확히 작성하는 것이 좋습니다.

Q6. ORDER BY에서 별칭을 사용할 수 있나요?

네. SELECT절에서 지정한 컬럼 별칭을 ORDER BY에서 사용할 수 있습니다.
예를 들어 SAL * 12 AS "연봉"으로 별칭을 지정했다면 ORDER BY "연봉" DESC처럼 정렬할 수 있습니다.

Q7. ORDER BY 3 DESC는 무슨 뜻인가요?

SELECT절의 세 번째 컬럼을 기준으로 내림차순 정렬한다는 의미입니다.
예를 들어 SELECT ENAME, JOB, SAL FROM EMP ORDER BY 3 DESC라면 세 번째 컬럼인 SAL 기준으로 정렬합니다.
다만 컬럼 순서가 바뀌면 의미가 달라질 수 있어 실무에서는 컬럼명이나 별칭을 사용하는 것이 좋습니다.

15. 최종 정리

ORDER BY
→ 조회 결과를 정렬하는 절

ASC
→ 오름차순
→ 기본값
→ 생략 가능

DESC
→ 내림차순
→ 직접 작성해야 함

ORDER BY 위치
→ SELECT문의 맨 마지막

숫자 정렬
→ 작은 값에서 큰 값 또는 큰 값에서 작은 값

문자 정렬
→ 알파벳 또는 사전식 정렬

날짜 정렬
→ 과거순 또는 최신순

여러 컬럼 정렬
→ 1차 기준 정렬 후 같은 값끼리 2차 기준 적용

별칭 정렬
→ SELECT에서 지정한 별칭으로 ORDER BY 가능

표현식 정렬
→ 계산식 기준 정렬 가능

컬럼 번호 정렬
→ 가능하지만 실무에서는 주의

NULL 정렬
→ NULL 위치가 결과에 영향을 줄 수 있음

읽는 순서

1. FROM을 보고 어느 테이블인지 확인한다.
2. WHERE가 있으면 어떤 행을 남기는지 확인한다.
3. SELECT를 보고 어떤 컬럼을 출력하는지 확인한다.
4. ORDER BY를 보고 어떤 기준으로 정렬하는지 확인한다.
5. ASC인지 DESC인지 확인한다.
6. 정렬 기준이 여러 개라면 1차, 2차 기준을 구분한다.

학습 메모

오늘의 핵심

ORDER BY는 조회 결과의 순서를 정하는 절입니다. 결과 순서가 중요하다면 반드시 ORDER BY를 작성해야 합니다.

기본 문법

SELECT 컬럼명
FROM 테이블명
WHERE 조건
ORDER BY 정렬기준 [ASC | DESC];

정렬 방향

  • ASC : 오름차순, 기본값, 생략 가능
  • DESC : 내림차순, 직접 작성 필요
  • 숫자 ASC : 작은 값에서 큰 값
  • 숫자 DESC : 큰 값에서 작은 값
  • 날짜 ASC : 과거에서 최근
  • 날짜 DESC : 최근에서 과거

실습 확인 순서

  1. SHOW USER로 현재 계정 확인
  2. DESC EMP로 컬럼명 확인
  3. ORDER BY SAL ASC 실행
  4. ORDER BY SAL DESC 실행
  5. ORDER BY ENAME 실행
  6. ORDER BY HIREDATE DESC 실행
  7. WHERE 조건과 ORDER BY 함께 사용
  8. 여러 컬럼 정렬 실행
  9. 별칭, 표현식, 컬럼 번호 정렬 확인

주의할 점

ORDER BY가 없으면 결과 순서는 보장되지 않습니다. 여러 컬럼 정렬은 1차 기준이 우선이고, 값이 같은 행끼리 2차 기준이 적용됩니다.

프로젝트 연결

게시판 최신순은 CREATED_AT DESC, 매물 가격 낮은 순은 DEAL_AMOUNT ASC, 추천 점수순은 SCORE DESC, 페이징은 ORDER BY와 OFFSET/FETCH가 함께 사용됩니다.
