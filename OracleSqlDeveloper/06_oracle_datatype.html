Oracle SQL Developer - Chapter 06

Oracle 자료형

Oracle 자료형은 테이블 설계의 출발점입니다. 숫자, 문자, 날짜, 긴 글, 파일 데이터를 어떤 컬럼에 어떤 형태로 저장할지 결정하며,
이후 CREATE TABLE, INSERT, SELECT, Java Entity, Spring DTO, 프로젝트 DB 설계까지 연결됩니다.

학습 목표 NUMBER, VARCHAR2, CHAR, DATE, CLOB, BLOB의
차이를 이해한다.
실습 방향 SQL Developer에서 테이블 생성, 데이터 입력,
결과 확인을 직접 실행한다.
프로젝트 연결 Oracle 자료형을
Java / Spring Entity 자료형과 연결해 이해한다.
Concept

1. 핵심 개념

자료형은 컬럼에 저장할 수 있는 데이터의 종류와 크기를 정하는 규칙입니다. 데이터베이스는 아무 값이나 아무 컬럼에 저장하지 않습니다.
숫자 컬럼에는 숫자, 문자 컬럼에는 문자, 날짜 컬럼에는 날짜가 들어가야 합니다.

NUMBER

숫자 데이터를 저장합니다.
정수, 실수, 금액, 수량, 조회수, 회원번호 등에
사용합니다.

NUMBER NUMBER(5) NUMBER(5,2)

VARCHAR2

가변 길이 문자 데이터를 저장합니다.
이름, 이메일, 제목, 주소처럼
길이가 달라지는 문자에 사용합니다.

VARCHAR2(30) VARCHAR2(100)

CHAR

고정 길이 문자 데이터를 저장합니다.
성별, 사용 여부, 회원 유형처럼
길이가 정해진 코드값에 사용합니다.

CHAR(1) CHAR(5)

DATE

날짜와 시간을 저장합니다.
가입일, 작성일, 수정일, 로그인 시간, 결제일 등에
사용합니다.

DATE SYSDATE

CLOB

긴 텍스트 데이터를 저장합니다.
게시글 본문, 공지사항 내용, 리포트 전문 등에
사용합니다.

CLOB

BLOB

이미지, 파일, 음성, PDF 같은
바이너리 데이터를 저장할 때 사용합니다.

BLOB

자료형 선택 기준

  • 계산해야 한다 → NUMBER
  • 소수점이 필요하다 → NUMBER(p, s)
  • 길이가 변하는 문자다 → VARCHAR2
  • 길이가 고정된 코드값이다 → CHAR
  • 날짜 계산이 필요하다 → DATE
  • 긴 본문이다 → CLOB
  • 파일 데이터다 → BLOB
Visual

2. 그림처럼 이해하기

자료형은 데이터를 담는 상자의 종류를 정하는 것과 같습니다. 숫자는 숫자 전용 상자에, 문자는 문자 전용 상자에, 날짜는 날짜 계산이 가능한 상자에 넣어야 합니다.

NUMBER(5,2)
1 2 3 . 4 5

전체 5자리 중 소수점 이하 2자리입니다. 정수 부분은 최대 3자리입니다.

CHAR(5)
A 공백 공백 공백 공백

한 글자만 저장해도 고정 길이 5칸을 유지합니다.

VARCHAR2(5)
A

최대 5까지 가능하지만, 실제 입력한 길이만큼 저장합니다.

DATE
2026 06 07 14:30:00

날짜뿐 아니라 시, 분, 초까지 저장할 수 있습니다.

SQL

3. SQL 예제

아래 예제는 SQL Developer에서 직접 실행하면서 자료형별 저장 방식과 결과를 확인하기 위한 실습 코드입니다.

3-1. 자료형 실습 테이블 생성

DROP TABLE DATATYPE_STUDY PURGE;

CREATE TABLE DATATYPE_STUDY (
    ID NUMBER(5),
    NAME VARCHAR2(30),
    GENDER CHAR(1),
    SCORE NUMBER(5,2),
    MEMO CLOB,
    CREATED_AT DATE
);

이 테이블은 숫자형, 문자형, 고정 문자형, 소수점 숫자형, 긴 글, 날짜형을 한 번에 확인하기 위한 실습 테이블입니다.

3-2. 테이블 구조 확인

DESC DATATYPE_STUDY;
확인할 결과 
Name        Null?    Type
----------- -------- --------------
ID                   NUMBER(5)
NAME                 VARCHAR2(30)
GENDER               CHAR(1)
SCORE                NUMBER(5,2)
MEMO                 CLOB
CREATED_AT           DATE

3-3. 정상 데이터 입력

INSERT INTO DATATYPE_STUDY (
    ID,
    NAME,
    GENDER,
    SCORE,
    MEMO,
    CREATED_AT
)
VALUES (
    1,
    '홍길동',
    'M',
    95.55,
    '오라클 자료형을 공부하고 있습니다.',
    SYSDATE
);

3-4. 날짜 출력 형식 변환

SELECT ID,
       NAME,
       GENDER,
       SCORE,
       MEMO,
       TO_CHAR(CREATED_AT, 'YYYY-MM-DD HH24:MI:SS') AS CREATED_AT
FROM DATATYPE_STUDY;

DATE 컬럼은 화면 설정에 따라 날짜만 보일 수 있습니다. 시간까지 명확히 확인하려면 TO_CHAR 함수를 사용합니다.

3-5. NUMBER(5,2) 반올림 확인

INSERT INTO DATATYPE_STUDY
VALUES (
    2,
    '김철수',
    'M',
    88.888,
    '소수점 반올림 확인',
    SYSDATE
);

SELECT ID, NAME, SCORE
FROM DATATYPE_STUDY
WHERE ID = 2;
예상 결과 
ID  NAME    SCORE
--  ------  -----
2   김철수  88.89

3-6. NUMBER 자리수 초과 오류 확인

INSERT INTO DATATYPE_STUDY
VALUES (
    3,
    '이영희',
    'F',
    1234.56,
    '자리수 초과 테스트',
    SYSDATE
);

SCORE는 NUMBER(5,2)이므로 전체 5자리 중 소수점 이하 2자리입니다. 정수 부분은 최대 3자리까지만 가능하므로 1234.56은 저장할 수 없습니다.

3-7. 한글 글자 수와 바이트 수 확인

SELECT NAME,
       LENGTH(NAME) AS CHAR_LENGTH,
       LENGTHB(NAME) AS BYTE_LENGTH
FROM DATATYPE_STUDY;

LENGTH는 글자 수를 반환하고, LENGTHB는 바이트 수를 반환합니다. 한글은 DB 문자셋에 따라 한 글자가 여러 바이트로 계산될 수 있습니다.

5. 실습 문제 연결

자료형 단원은 이후 SELECT, WHERE, ORDER BY, GROUP BY 실습과 연결됩니다. EMP와 EMPLOYEES 테이블에서 숫자형, 문자형, 날짜형 컬럼을 어떻게 사용하는지 확인하면 좋습니다.

숫자형 컬럼 조건 검색

SAL, COMM, DEPTNO는 숫자형 컬럼입니다.
BETWEEN, 비교 연산자, ORDER BY와 연결됩니다.

SELECT EMPNO, ENAME, JOB, SAL
FROM EMP
WHERE SAL BETWEEN 2500 AND 3000;

문자형 컬럼 조건 검색

ENAME, JOB은 문자형 컬럼입니다.
문자값은 반드시 작은따옴표로 감싸야 합니다.

SELECT ENAME, DEPTNO, SAL, JOB
FROM EMP
WHERE JOB = 'SALESMAN';

날짜형 컬럼 출력

HIRE_DATE는 날짜형 컬럼입니다.
화면 출력 형식은 TO_CHAR로 변환할 수 있습니다.

SELECT FIRST_NAME,
        SALARY,
        TO_CHAR (HIRE_DATE, 'YYYY-MM-DD') AS HIRE_DATE
FROM EMPLOYEES;

추가 실습 미션

  1. DATATYPE_MEMBER 테이블을 생성한다.
  2. NUMBER, VARCHAR2, CHAR, DATE, CLOB 컬럼을 각각 하나 이상 포함한다.
  3. SYSDATE를 사용해 가입일을 입력한다.
  4. TO_CHAR로 가입일을 YYYY-MM-DD HH24:MI:SS 형식으로 출력한다.
  5. NUMBER(5,2)에 소수점 세 자리 값을 넣고 반올림 결과를 확인한다.
Mistake

6. 자주 하는 실수

실수 1. NUMBER(5,2)를 정수 5자리 + 소수 2자리로 착각

NUMBER(5,2)는 전체 5자리 중 소수점 이하 2자리입니다. 따라서 정수 부분은 최대 3자리입니다. 

가능: 999.99
불가능: 1000.00

실수 2. 문자값에 작은따옴표를 붙이지 않음

-- 잘못된 예
INSERT INTO DATATYPE_STUDY (NAME)
VALUES (홍길동);

-- 올바른 예
INSERT INTO DATATYPE_STUDY (NAME)
VALUES ('홍길동');

실수 3. 날짜를 문자열처럼 대충 입력

-- 권장하지 않는 방식
INSERT INTO DATATYPE_STUDY (CREATED_AT)
VALUES ('2026-06-07');

-- 안전한 방식
INSERT INTO DATATYPE_STUDY (CREATED_AT)
VALUES (TO_DATE('2026-06-07', 'YYYY-MM-DD'));

실수 4. VARCHAR2 크기를 글자 수로만 생각

VARCHAR2는 기본적으로 byte 기준으로 해석될 수 있습니다. 한글은 한 글자가 여러 바이트를 차지할 수 있으므로 이름, 제목, 주소 컬럼은 넉넉하게 잡는 것이 좋습니다.

실수 5. 금액이나 면적을 VARCHAR2로 저장

금액, 면적, 조회수, 체류시간처럼 계산이나 정렬이 필요한 값은 NUMBER로 저장해야 합니다. 

-- 좋지 않은 설계
DEAL_AMOUNT VARCHAR2(20)

-- 좋은 설계
DEAL_AMOUNT NUMBER(15)
AREA_SQM NUMBER(8,2)
Interview

7. 면접 포인트

Q1. CHAR와 VARCHAR2의 차이는 무엇인가요?

CHAR는 고정 길이 문자열이고, VARCHAR2는 가변 길이 문자열입니다.
CHAR(10)은 한 글자만 저장해도 정해진 길이를 유지하지만, VARCHAR2(10)은 실제 입력된 길이만큼 저장합니다.
그래서 Y/N, M/F 같은 코드값은 CHAR, 이름과 이메일은 VARCHAR2를 사용합니다.

Q2. NUMBER(5,2)는 어떤 의미인가요?

전체 숫자 자릿수가 5자리이고, 그중 소수점 이하가 2자리라는 의미입니다. 따라서 정수 부분은 최대 3자리입니다. 999.99는 가능하지만 1000.00은 저장할 수 없습니다.

Q3. 날짜를 문자열이 아니라 DATE로 저장하는 이유는 무엇인가요?

DATE로 저장하면 날짜 계산, 기간 검색, 정렬이 정확해집니다. 문자열로 저장하면 형식이 섞일 수 있고 날짜 계산이 어렵습니다.
화면에 보여줄 때만 TO_CHAR로 원하는 형식으로 변환하는 것이 좋습니다.

Q4. Oracle DATE는 날짜만 저장하나요?

아닙니다. Oracle DATE는 년, 월, 일뿐 아니라 시, 분, 초까지 저장할 수 있습니다.
다만 SQL Developer 화면 설정에 따라 날짜만 보일 수 있으므로, 시간까지 확인하려면 TO_CHAR(date_column, 'YYYY-MM-DD HH24:MI:SS')를 사용합니다.

Q5. Oracle 자료형은 Java / Spring에서 어떻게 연결되나요?

NUMBER는 Integer, Long, BigDecimal로 연결할 수 있고, VARCHAR2와 CHAR는 String으로 연결합니다.
DATE는 LocalDate 또는 LocalDateTime으로 연결하고, CLOB은 긴 문자열, BLOB은 byte 배열이나 파일 데이터와 연결됩니다.

학습 메모

오늘의 핵심

자료형은 테이블 설계의 첫 번째 기준이다. 값을 저장하는 것뿐 아니라 검색, 정렬, 계산, Java Entity 매핑까지 영향을 준다.

외우기보다 이해하기

  • 계산할 값은 NUMBER
  • 일반 문자열은 VARCHAR2
  • 고정 코드값은 CHAR
  • 날짜 계산은 DATE
  • 긴 본문은 CLOB
  • 파일 데이터는 BLOB

SQL Developer 확인 순서

  1. CREATE TABLE 실행
  2. DESC로 구조 확인
  3. INSERT로 값 입력
  4. SELECT로 결과 확인
  5. 오류가 나는 값도 직접 넣어보기

프로젝트 연결

회원가입, 게시판, 매물 등록, 추천 로그, 통계 테이블을 만들 때 Oracle 자료형 선택이 곧 Java Entity 필드 타입 선택으로 이어진다.
