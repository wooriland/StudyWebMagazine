Overview

1. INSERT, UPDATE, DELETE란?

INSERT, UPDATE, DELETE는 테이블 안에 저장된 데이터를 직접 변경하는 DML 문장입니다.
테이블 구조를 만드는 CREATE TABLE이나 구조를 변경하는 ALTER TABLE과 달리, DML은 테이블의 행 데이터를 입력, 수정, 삭제합니다.

INSERT 테이블에 새 행을 입력한다.
UPDATE 이미 존재하는 행의 값을 수정한다.
DELETE 조건에 맞는 행을 삭제한다.

한 줄 정리

INSERT는 새 데이터를 넣고, UPDATE는 기존 데이터를 바꾸고, DELETE는 기존 데이터를 삭제하는 DML 문장입니다.

Concept Card

2. 단원 개념 카드

2-1. DML이란?

DML은 Data Manipulation Language, 즉 데이터 조작어입니다. 테이블 자체를 만들거나 삭제하는 것이 아니라, 테이블 안의 실제 데이터를 다룹니다.

구분 의미 대표 명령어 이번 단원 연결
DDL DB 객체 생성, 변경, 삭제 CREATE, ALTER, DROP, TRUNCATE 테이블 구조 준비
DML 테이블 데이터 조회, 입력, 수정, 삭제 SELECT, INSERT, UPDATE, DELETE 이번 단원의 핵심
DCL 권한 부여, 회수 GRANT, REVOKE 사용자 권한 관리

2-2. CRUD와 DML 연결

웹 프로젝트의 기본 기능인 CRUD는 SQL과 직접 연결됩니다. INSERT, UPDATE, DELETE를 이해하면 게시판, 회원가입, 매물 관리 기능을 훨씬 쉽게 이해할 수 있습니다.

Create

새 데이터를 생성합니다.

INSERT INTO board (...)

Read

기존 데이터를 조회합니다.

SELECT * FROM board

Update

기존 데이터를 수정합니다.

UPDATE board SET ...

Delete

기존 데이터를 삭제하거나 삭제 상태로 변경합니다.

DELETE FROM board ...
UPDATE board SET deleted_yn = 'Y'

2-3. DML과 트랜잭션

INSERT, UPDATE, DELETE는 데이터를 변경하기 때문에 COMMIT과 ROLLBACK을 함께 이해해야 합니다.
DML 실행 후 결과를 확인하고, 맞으면 COMMIT으로 확정하고, 틀리면 ROLLBACK으로 되돌립니다.

1 DML 실행

INSERT / UPDATE / DELETE

2 SELECT 확인

변경 결과 검증

3 COMMIT 또는 ROLLBACK

확정 또는 취소

중요

UPDATE와 DELETE는 WHERE절이 없으면 모든 행에 영향을 줄 수 있습니다. 실행 전 반드시 SELECT로 대상을 먼저 확인해야 합니다.

3. 실행 순서 그림 카드

3-1. INSERT는 새 행 추가

INSERT는 테이블에 새로운 행을 추가합니다. 회원가입, 게시글 작성, 매물 등록처럼 새 데이터가 생기는 기능과 연결됩니다.

INSERT 전

MEMBER_ID LOGIN_ID NAME

데이터 없음

→

INSERT 후

1 hong 홍길동

새 행 1건 추가

3-2. UPDATE는 기존 행 수정

UPDATE는 이미 존재하는 행의 컬럼 값을 바꿉니다. 회원 이메일 변경, 게시글 제목 수정, 매물 가격 변경에 사용됩니다.

UPDATE 전

EMAIL: hong@test.com
→

UPDATE 후

EMAIL: hong_new@test.com

3-3. DELETE는 행 삭제

DELETE는 조건에 맞는 행을 테이블에서 제거합니다. 다만 실무에서는 이력 보존을 위해 실제 삭제보다 논리 삭제를 많이 사용합니다.

DELETE 전

1 - hong 2 - kim
→

DELETE 후

1 - hong

kim 행 삭제

3-4. WHERE의 영향

WHERE절은 UPDATE와 DELETE에서 대상을 제한합니다. WHERE가 없으면 전체 행이 수정되거나 삭제될 수 있습니다.

WHERE 있음

UPDATE dml_member
SET active = 'N'
WHERE login_id = 'hong';

hong 회원만 수정

WHERE 없음

UPDATE dml_member
SET active = 'N';

모든 회원 수정

Setup

4. SQL Developer 실습 준비

이번 단원은 원본 EMP, DEPT 테이블을 직접 수정하지 않고, 별도의 실습 테이블을 만들어 INSERT, UPDATE, DELETE를 안전하게 연습합니다.

4-1. 기존 실습 객체 정리

DROP TABLE dml_board PURGE;
DROP TABLE dml_member PURGE;
DROP SEQUENCE dml_member_seq;
DROP SEQUENCE dml_board_seq;

처음 실행하면 ORA-00942 또는 ORA-02289 오류가 날 수 있습니다.
삭제할 테이블이나 시퀀스가 없다는 뜻이므로 첫 실습에서는 괜찮습니다.

4-2. 회원 테이블 생성

CREATE TABLE dml_member (
    member_id NUMBER,
    login_id VARCHAR2(30) NOT NULL,
    member_name VARCHAR2(50) NOT NULL,
    email VARCHAR2(100),
    active CHAR(1) DEFAULT 'Y',
    created_at DATE DEFAULT SYSDATE,
    updated_at DATE,
    CONSTRAINT pk_dml_member PRIMARY KEY (member_id),
    CONSTRAINT uk_dml_member_login UNIQUE (login_id),
    CONSTRAINT chk_dml_member_active CHECK (active IN ('Y', 'N'))
);

4-3. 게시글 테이블 생성

CREATE TABLE dml_board (
    board_id NUMBER,
    title VARCHAR2(200) NOT NULL,
    content VARCHAR2(1000),
    member_id NUMBER NOT NULL,
    deleted_yn CHAR(1) DEFAULT 'N',
    created_at DATE DEFAULT SYSDATE,
    updated_at DATE,
    CONSTRAINT pk_dml_board PRIMARY KEY (board_id),
    CONSTRAINT fk_dml_board_member
        FOREIGN KEY (member_id)
        REFERENCES dml_member(member_id),
    CONSTRAINT chk_dml_board_deleted CHECK (deleted_yn IN ('Y', 'N'))
);

4-4. 시퀀스 생성

CREATE SEQUENCE dml_member_seq
START WITH 1
INCREMENT BY 1
NOCACHE
NOCYCLE;

CREATE SEQUENCE dml_board_seq
START WITH 1
INCREMENT BY 1
NOCACHE
NOCYCLE;

4-5. 생성 결과 확인

DESC dml_member;
DESC dml_board;

SELECT table_name
FROM user_tables
WHERE table_name IN ('DML_MEMBER', 'DML_BOARD');

SELECT sequence_name,
       increment_by,
       cycle_flag,
       cache_size,
       last_number
FROM user_sequences
WHERE sequence_name IN ('DML_MEMBER_SEQ', 'DML_BOARD_SEQ')
ORDER BY sequence_name;

Transaction

6. COMMIT과 ROLLBACK

INSERT, UPDATE, DELETE는 COMMIT 전까지는 확정되지 않은 변경입니다. 실습 중에는 DML 실행 후 SELECT로 결과를 확인하고, 맞으면 COMMIT, 틀리면 ROLLBACK합니다.

6-1. ROLLBACK 실습

INSERT INTO dml_member (
    member_id,
    login_id,
    member_name
) VALUES (
    dml_member_seq.NEXTVAL,
    'rollback_test',
    '롤백테스트'
);

SELECT *
FROM dml_member
WHERE login_id = 'rollback_test';

ROLLBACK;

SELECT *
FROM dml_member
WHERE login_id = 'rollback_test';

ROLLBACK 후 결과가 없으면 되돌리기 성공입니다.

6-2. COMMIT 실습

COMMIT;

COMMIT 후에는 일반적인 ROLLBACK으로 되돌릴 수 없으므로,
반드시 결과를 확인한 뒤 실행해야 합니다.

UPDATE

7. SQL 예제 카드 - UPDATE

UPDATE는 기존 행의 컬럼 값을 수정합니다. WHERE 조건이 없으면 모든 행이 수정될 수 있으므로 주의해야 합니다.

7-1. UPDATE 기본 문법

UPDATE 테이블명
SET 컬럼명 = 새값
WHERE 조건;

7-2. 회원 이름 수정

UPDATE dml_member
SET member_name = '홍길동 수정'
WHERE login_id = 'hong';

SELECT *
FROM dml_member
WHERE login_id = 'hong';

7-3. 여러 컬럼 수정

UPDATE dml_member
SET email = 'hong_new@test.com',
    updated_at = SYSDATE
WHERE login_id = 'hong';

SELECT member_id,
       login_id,
       member_name,
       email,
       updated_at
FROM dml_member
WHERE login_id = 'hong';

7-4. 회원 비활성화

UPDATE dml_member
SET active = 'N',
    updated_at = SYSDATE
WHERE login_id = 'lee';

SELECT login_id,
       member_name,
       active,
       updated_at
FROM dml_member
WHERE login_id = 'lee';

7-5. UPDATE 후 ROLLBACK

UPDATE dml_member
SET member_name = '전체수정위험'
WHERE active = 'Y';

SELECT member_id,
       login_id,
       member_name,
       active
FROM dml_member
ORDER BY member_id;

ROLLBACK;

7-6. WHERE 없는 UPDATE 위험

-- 매우 위험
UPDATE dml_member
SET active = 'N';

모든 회원의 ACTIVE 값이 N으로 변경됩니다.
반드시 UPDATE 전 SELECT로 대상을 확인해야 합니다.

DELETE

8. SQL 예제 카드 - DELETE

DELETE는 조건에 맞는 행을 삭제합니다. WHERE 조건이 없으면 테이블의 모든 행이 삭제될 수 있습니다.

8-1. DELETE 기본 문법

DELETE FROM 테이블명
WHERE 조건;

8-2. 특정 회원 삭제

SELECT *
FROM dml_member
WHERE login_id = 'park';

DELETE FROM dml_member
WHERE login_id = 'park';

SELECT *
FROM dml_member
WHERE login_id = 'park';

8-3. DELETE 후 ROLLBACK

ROLLBACK;

SELECT *
FROM dml_member
WHERE login_id = 'park';

8-4. 게시글 삭제

DELETE FROM dml_board
WHERE board_id = 1;

SELECT *
FROM dml_board
ORDER BY board_id;

8-5. FK 때문에 삭제가 막히는 경우

DELETE FROM dml_member
WHERE member_id = 1;

해당 회원이 작성한 게시글이 있다면
ORA-02292 child record found 오류가 발생할 수 있습니다.

8-6. WHERE 없는 DELETE 위험

-- 매우 위험
DELETE FROM dml_member;

모든 회원 데이터가 삭제됩니다.
DELETE 전에는 반드시 SELECT로 삭제 대상을 확인해야 합니다.

Logical Delete

9. 논리 삭제와 물리 삭제

DELETE는 실제 행을 제거하는 물리 삭제입니다. 하지만 실무에서는 데이터 이력과 관계 보존을 위해 실제 삭제보다 논리 삭제를 자주 사용합니다.

물리 삭제

행을 실제로 테이블에서 제거합니다.

DELETE FROM dml_member
WHERE member_id = 1;

논리 삭제

행은 남겨두고 상태값만 삭제 상태로 변경합니다.

UPDATE dml_member
SET active = 'N',
    updated_at = SYSDATE
WHERE member_id = 1;

게시글 논리 삭제

UPDATE dml_board
SET deleted_yn = 'Y',
    updated_at = SYSDATE
WHERE board_id = 1;

삭제되지 않은 게시글 조회

SELECT *
FROM dml_board
WHERE deleted_yn = 'N'
ORDER BY board_id;
DELETE vs TRUNCATE

10. DELETE와 TRUNCATE 비교

DELETE와 TRUNCATE는 모두 테이블의 데이터를 삭제할 수 있지만 성격이 다릅니다.
DELETE는 조건 삭제가 가능하고 ROLLBACK이 가능하지만, TRUNCATE는 전체 삭제만 가능하고 ROLLBACK이 어렵습니다.

구분 DELETE TRUNCATE
명령 성격 DML DDL 성격
WHERE 사용 가능 불가능
일부 삭제 가능 불가능
전체 삭제 가능 가능
ROLLBACK 가능 일반적으로 어려움
속도 상대적으로 느림 빠름
테이블 구조 유지 유지

주의

조건에 맞는 일부 행만 삭제하려면 DELETE를 사용해야 합니다. TRUNCATE는 WHERE 조건을 사용할 수 없고, 실행 후 복구가 어렵기 때문에 운영 DB에서 매우 조심해야 합니다.

11. 결과 화면 확인

DML 실행 후에는 반드시 SELECT로 결과를 확인해야 합니다. SQL Developer의 메시지 영역에서는 몇 행이 입력, 수정, 삭제되었는지도 확인할 수 있습니다.

11-1. INSERT 결과 확인

SELECT *
FROM dml_member
ORDER BY member_id;

SELECT *
FROM dml_member
WHERE login_id = 'hong';

11-2. UPDATE 결과 확인

SELECT member_id,
       login_id,
       member_name,
       email,
       active,
       updated_at
FROM dml_member
WHERE login_id = 'hong';

11-3. DELETE 결과 확인

SELECT *
FROM dml_member
WHERE login_id = 'park';

결과가 없으면 삭제된 것입니다.

11-4. SQL%ROWCOUNT 확인

BEGIN
    UPDATE dml_member
    SET active = 'N'
    WHERE login_id = 'hong';

    DBMS_OUTPUT.PUT_LINE(SQL%ROWCOUNT || '개의 행이 수정되었습니다.');
END;
/

PL/SQL에서는 SQL%ROWCOUNT로
최근 DML이 몇 행에 영향을 주었는지 확인할 수 있습니다.

12. 실습 문제 카드

이번 단원은 직접 INSERT, UPDATE, DELETE를 실행하고 SELECT로 결과를 확인하는 연습이 핵심입니다.
또한 원본 EMP 테이블을 직접 수정하지 않고 복사본을 만들어 연습하는 것이 안전합니다.

문제 1. EMP 복사본 만들기

DROP TABLE emp_dml_copy PURGE;

CREATE TABLE emp_dml_copy
AS
SELECT *
FROM emp;

SELECT COUNT(*)
FROM emp_dml_copy;

DESC emp_dml_copy;

문제 2. INSERT 실습

INSERT INTO emp_dml_copy (
    empno,
    ename,
    job,
    mgr,
    hiredate,
    sal,
    comm,
    deptno
) VALUES (
    9001,
    'HONG',
    'CLERK',
    NULL,
    SYSDATE,
    2000,
    NULL,
    30
);

SELECT *
FROM emp_dml_copy
WHERE empno = 9001;

문제 3. UPDATE 실습

UPDATE emp_dml_copy
SET ename = 'KIM',
    sal = 2500
WHERE empno = 9001;

SELECT empno,
       ename,
       sal
FROM emp_dml_copy
WHERE empno = 9001;

문제 4. DELETE 실습

Mistake

13. 자주 하는 실수

  • 실수 1. INSERT에서 문자에 작은따옴표를 빼먹음 문자형 값은 반드시 작은따옴표로 감싸야 합니다.
  • 실수 2. 컬럼명을 생략한 INSERT에서 값 순서를 틀림 테이블 컬럼 순서와 값 순서가 정확히 맞아야 하므로 컬럼명을 명시하는 방식이 안전합니다.
  • 실수 3. NOT NULL 컬럼을 생략함 NOT NULL 컬럼에는 반드시 값을 넣어야 합니다.
  • 실수 4. UPDATE에 WHERE를 빼먹음 WHERE 없는 UPDATE는 모든 행을 수정합니다.
  • 실수 5. DELETE에 WHERE를 빼먹음 WHERE 없는 DELETE는 모든 행을 삭제합니다.
  • 실수 6. DML 후 바로 COMMIT함 COMMIT 전에 SELECT로 결과를 확인해야 합니다.
  • 실수 7. 부모 데이터를 먼저 삭제하려고 함 자식 테이블이 참조하고 있으면 FK 오류가 발생할 수 있습니다.
  • 실수 8. 실제 삭제와 논리 삭제를 구분하지 않음 이력 보존이 필요한 데이터는 DELETE보다 상태값 UPDATE가 더 안전할 수 있습니다.
  • 실수 9. 시퀀스 번호가 건너뛰었다고 오류로 생각함 시퀀스의 목적은 연속 번호 보장이 아니라 중복 없는 고유 번호 발급입니다.
Interview

14. 면접 질문 카드

Q1. INSERT, UPDATE, DELETE는 무엇인가요?

INSERT, UPDATE, DELETE는 테이블의 데이터를 조작하는 DML 문장입니다.
INSERT는 새 행을 추가하고, UPDATE는 기존 행의 컬럼 값을 수정하며, DELETE는 조건에 맞는 행을 삭제합니다.

Q2. INSERT에서 컬럼명을 명시하는 방식이 왜 더 안전한가요?

컬럼명을 명시하면 어떤 값이 어떤 컬럼에 들어가는지 명확합니다. 또한 테이블에 컬럼이 추가되거나 컬럼 순서가 바뀌어도 상대적으로 안전합니다.

Q3. UPDATE에서 WHERE절이 중요한 이유는 무엇인가요?

UPDATE에서 WHERE절이 없으면 테이블의 모든 행이 수정됩니다. 따라서 실행 전 SELECT로 수정 대상을 확인하고 WHERE 조건을 명확히 작성해야 합니다.

Q4. DELETE에서 WHERE절이 없으면 어떻게 되나요?

DELETE FROM 테이블명처럼 WHERE절 없이 실행하면 해당 테이블의 모든 행이 삭제됩니다. 운영 DB에서는 매우 위험한 실수입니다.

Q5. DELETE와 TRUNCATE의 차이는 무엇인가요?

DELETE는 DML이며 WHERE 조건을 사용해 일부 행만 삭제할 수 있고 COMMIT 전에는 ROLLBACK이 가능합니다.
TRUNCATE는 DDL 성격이 강하고 테이블 전체 데이터를 빠르게 삭제하지만 WHERE 조건을 사용할 수 없고 일반적으로 ROLLBACK이 어렵습니다.

Q6. COMMIT과 ROLLBACK의 차이는 무엇인가요?

COMMIT은 현재 트랜잭션에서 발생한 변경 내용을 영구적으로 반영합니다. ROLLBACK은 아직 COMMIT하지 않은 변경 내용을 취소하고 원래 상태로 되돌립니다.

Q7. 물리 삭제와 논리 삭제의 차이는 무엇인가요?

물리 삭제는 DELETE로 실제 행을 제거하는 방식입니다. 논리 삭제는 active='N' 또는 deleted_yn='Y'처럼 상태값만 변경하여 데이터는 보존하는 방식입니다.

Q8. FK가 걸린 부모 데이터를 삭제하면 어떤 문제가 생길 수 있나요?

자식 테이블이 부모 테이블의 PK를 참조하고 있다면 부모 데이터 삭제 시 child record found 오류가 발생할 수 있습니다.
이 경우 자식 데이터를 먼저 삭제하거나, 논리 삭제를 사용하거나, 설계 단계에서 ON DELETE CASCADE를 고려해야 합니다.

Summary

16. 최종 정리

INSERT
→ 테이블에 새 행을 입력한다.

UPDATE
→ 기존 행의 컬럼 값을 수정한다.

DELETE
→ 조건에 맞는 행을 삭제한다.

COMMIT
→ 변경 내용을 확정한다.

ROLLBACK
→ COMMIT 전 변경 내용을 되돌린다.

DEFAULT
→ 기본값을 입력한다.

NULL
→ 값이 없음을 의미한다.

시퀀스.NEXTVAL
→ PK 값을 자동으로 발급할 때 사용한다.

논리 삭제
→ 실제 DELETE 대신 상태값을 변경한다.

물리 삭제
→ DELETE로 실제 행을 제거한다.

학습 메모

오늘의 핵심

INSERT는 새 데이터를 넣고, UPDATE는 기존 데이터를 수정하고, DELETE는 기존 데이터를 삭제하는 DML 문장입니다.

암기 문법

INSERT INTO 테이블명 (
    컬럼명1,
    컬럼명2
) VALUES (
    값1,
    값2
);

UPDATE 테이블명
SET 컬럼명 = 새값
WHERE 조건;

DELETE FROM 테이블명
WHERE 조건;

안전 실행 순서

  1. SELECT로 대상 확인
  2. INSERT / UPDATE / DELETE 실행
  3. SELECT로 결과 확인
  4. 맞으면 COMMIT
  5. 틀리면 ROLLBACK

INSERT 주의

  • 문자형은 작은따옴표로 감싼다.
  • 숫자형은 따옴표 없이 입력한다.
  • 날짜는 SYSDATE 또는 TO_DATE를 사용한다.
  • DEFAULT 컬럼은 생략하거나 DEFAULT를 사용할 수 있다.
  • PK는 시퀀스.NEXTVAL로 자동 입력할 수 있다.

주의할 점

UPDATE와 DELETE에서 WHERE절을 빼먹으면 모든 행이 수정되거나 삭제될 수 있습니다. 운영 DB에서는 반드시 SELECT로 대상을 먼저 확인해야 합니다.

프로젝트 연결

INSERT, UPDATE, DELETE는 회원가입, 게시글 작성·수정·삭제, 매물 등록·수정·삭제, 좋아요, 문의, 로그 저장 기능과 직접 연결됩니다.
