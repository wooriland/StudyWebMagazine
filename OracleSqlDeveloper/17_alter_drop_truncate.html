Oracle SQL Developer - Chapter 17

ALTER, DROP, TRUNCATE

이번 단원에서는 이미 만들어진 테이블의 구조를 변경하는 ALTER, 테이블 자체를 삭제하는 DROP,
테이블 구조는 유지하고 데이터만 전체 삭제하는 TRUNCATE를 학습합니다.
이 문법들은 Java·Spring·Oracle 프로젝트에서
요구사항 변경, 실습 테이블 정리, 임시 분석 테이블 초기화, 운영 DB 변경 관리와 직접 연결됩니다.

학습 목표 ALTER, DROP, TRUNCATE의 차이를 이해하고
SQL Developer에서 직접 실행한다.
실습 방향 테이블 생성 → 구조 변경 → 삭제 → 복구 →
데이터 전체 비우기 흐름을 확인한다.
프로젝트 연결 Entity 변경, DTO 수정, Mapper 수정,
DB 마이그레이션과 연결한다.
Overview

1. ALTER, DROP, TRUNCATE란?

앞 단원에서는 CREATE TABLE로 테이블을 만들고 제약조건을 설정했습니다. 하지만 실제 프로젝트에서는 테이블을 한 번 만들고 끝내는 경우가 거의 없습니다.
요구사항이 바뀌면 컬럼을 추가하거나, 자료형을 늘리거나, 제약조건을 새로 걸거나, 잘못 만든 테이블을 삭제해야 합니다.

ALTER 기존 테이블 구조를 변경한다.
DROP 테이블 객체 자체를 삭제한다.
TRUNCATE 테이블 구조는 남기고 데이터만 전체 삭제한다.

한 줄 정리

ALTER는 리모델링, DROP은 철거, TRUNCATE는 내부 데이터 비우기입니다.

Concept Card

2. 핵심 개념 카드

2-1. 왜 이 문법을 배우는가?

실제 개발에서는 데이터베이스 구조가 요구사항에 따라 계속 바뀝니다.
처음에는 아이디와 비밀번호만 저장하던 회원 테이블에 이메일, 전화번호, 회원 상태, 가입일 컬럼이 추가될 수 있습니다.

요구사항 변경

회원가입에 전화번호 인증이 추가되어 PHONE 컬럼이 필요해집니다.

ALTER TABLE users
ADD phone VARCHAR2(20);

실습 테이블 정리

잘못 만든 실습 테이블이나 임시 테이블을 제거해야 할 수 있습니다.

DROP TABLE temp_test_table PURGE;

임시 데이터 초기화

분석 결과 임시 테이블의 데이터만 빠르게 비우고 구조는 유지합니다.

TRUNCATE TABLE temp_recommendation_result;

2-2. ALTER, DROP, TRUNCATE 비교

명령어 의미 테이블 구조 데이터 주요 사용 상황
ALTER 기존 테이블 구조 변경 변경됨 보통 유지 컬럼 추가, 자료형 변경, 제약조건 추가
DROP 테이블 객체 삭제 삭제됨 삭제됨 잘못 만든 테이블 삭제, 임시 테이블 정리
TRUNCATE 데이터 전체 삭제 유지됨 전체 삭제 로그/임시 테이블 전체 비우기

2-3. DDL과 자동 COMMIT

ALTER, DROP, TRUNCATE는 DDL 성격의 명령입니다. Oracle에서 DDL은 실행되면 자동 COMMIT이 발생한다고 이해해야 합니다.
따라서 일반적인 INSERT, UPDATE, DELETE처럼 ROLLBACK으로 쉽게 되돌릴 수 있다고 생각하면 위험합니다.

주의

운영 DB에서 ALTER, DROP, TRUNCATE를 실행하기 전에는 반드시 대상 테이블, 백업 여부, 관련 API, Entity, Mapper, 화면, 배치, 통계 영향을 확인해야 합니다.

3. 그림처럼 이해하기

3-1. ALTER는 테이블 리모델링

ALTER는 기존 테이블을 삭제하지 않고 구조를 변경합니다. 예를 들어 EMAIL 컬럼을 추가하면 기존 데이터는 유지되고 새 컬럼은 NULL 상태로 추가됩니다.

ALTER 전

MEMBER_ID LOGIN_ID NAME AGE
→

ALTER 후

MEMBER_ID LOGIN_ID NAME AGE EMAIL

3-2. DROP은 테이블 철거

DROP은 테이블 구조와 데이터를 모두 삭제합니다. 일반 DROP은 Oracle 휴지통에 들어갈 수 있고, PURGE를 붙이면 휴지통을 거치지 않고 완전 삭제됩니다.

1 DROP TABLE

테이블 객체 삭제

2 RECYCLEBIN

일반 DROP은 휴지통에 남을 수 있음

3 FLASHBACK

휴지통에 있으면 복구 가능

3-3. TRUNCATE는 데이터만 비우기

TRUNCATE는 테이블 구조는 그대로 두고 데이터만 모두 삭제합니다. WHERE 조건을 사용할 수 없으며, 일반적으로 ROLLBACK이 어렵습니다.

TRUNCATE 전

1 - 데이터1 2 - 데이터2 3 - 데이터3
→

TRUNCATE 후

데이터 0건

테이블 구조는 유지

4. SQL Developer 실습 준비

이번 단원은 직접 실행하면서 이해하는 것이 중요합니다. 먼저 부모 테이블, 회원 테이블, 게시글 테이블을 만든 뒤 ALTER, DROP, TRUNCATE를 순서대로 실습합니다.

4-1. 기존 실습 테이블 정리

DROP TABLE alter_board PURGE;
DROP TABLE alter_member PURGE;
DROP TABLE alter_dept PURGE;

처음 실행하면 ORA-00942 오류가 날 수 있습니다.
삭제할 테이블이 없다는 의미이므로 첫 실습에서는 괜찮습니다.

4-2. 부모 테이블 생성

CREATE TABLE alter_dept (
    dept_id NUMBER,
    dept_name VARCHAR2(50) NOT NULL,
    CONSTRAINT pk_alter_dept PRIMARY KEY (dept_id)
);

4-3. 회원 테이블 생성

CREATE TABLE alter_member (
    member_id NUMBER,
    login_id VARCHAR2(30) NOT NULL,
    member_name VARCHAR2(50) NOT NULL,
    age NUMBER,
    dept_id NUMBER,
    created_at DATE DEFAULT SYSDATE,
    CONSTRAINT pk_alter_member PRIMARY KEY (member_id),
    CONSTRAINT uk_alter_member_login UNIQUE (login_id),
    CONSTRAINT fk_alter_member_dept
        FOREIGN KEY (dept_id)
        REFERENCES alter_dept(dept_id)
);

4-4. 게시글 테이블 생성

CREATE TABLE alter_board (
    board_id NUMBER,
    title VARCHAR2(200) NOT NULL,
    content VARCHAR2(1000),
    member_id NUMBER NOT NULL,
    created_at DATE DEFAULT SYSDATE,
    CONSTRAINT pk_alter_board PRIMARY KEY (board_id),
    CONSTRAINT fk_alter_board_member
        FOREIGN KEY (member_id)
        REFERENCES alter_member(member_id)
);

ALTER

5. ALTER TABLE 예제

ALTER TABLE은 기존 테이블의 구조를 변경하는 명령입니다. 컬럼 추가, 제약조건 추가, 컬럼 삭제, 제약조건 삭제, 자료형 변경, 컬럼명 변경에 사용합니다.

5-1. 컬럼 추가: ADD

ALTER TABLE alter_member
ADD email VARCHAR2(100);

DESC alter_member;

SELECT member_id,
       login_id,
       member_name,
       email
FROM alter_member;

기존 행에는 EMAIL 값이 없기 때문에 NULL로 표시됩니다.

5-2. NOT NULL 컬럼 추가 주의

--이미 데이터가 있는 테이블에 바로 NOT NULL 컬럼 추가 시 오류 가능
ALTER TABLE alter_member
ADD phone VARCHAR2(20) NOT NULL;

기존 행에 PHONE 값이 없기 때문에 ORA-01758 오류가 날 수 있습니다.

5-3. 안전한 NOT NULL 추가 순서

ALTER TABLE alter_member
ADD phone VARCHAR2(20);

UPDATE alter_member
SET phone = '010-0000-0000';

ALTER TABLE alter_member
MODIFY phone VARCHAR2(20) NOT NULL;

5-4. CHECK 제약조건 추가

ALTER TABLE alter_member
ADD CONSTRAINT chk_alter_member_age
CHECK (age BETWEEN 0 AND 120);

SELECT constraint_name,
       constraint_type,
       search_condition
FROM user_constraints
WHERE table_name = 'ALTER_MEMBER'
ORDER BY constraint_name;

5-5. CHECK 오류 확인

INSERT INTO alter_member (
    member_id,
    login_id,
    member_name,
    age,
    dept_id,
    email,
    phone
) VALUES (
    3,
    'lee',
    '이영희',
    150,
    10,
    'lee@test.com',
    '010-1111-2222'
);

AGE가 0~120 사이가 아니므로 ORA-02290 오류가 발생해야 정상입니다.

5-6. 컬럼 삭제: DROP COLUMN

ALTER TABLE alter_member
DROP COLUMN email;

DESC alter_member;

컬럼 삭제는 해당 컬럼의 데이터도 함께 사라지므로 실무에서 신중해야 합니다.

5-7. 제약조건 삭제: DROP CONSTRAINT

ALTER TABLE alter_member
DROP CONSTRAINT chk_alter_member_age;

SELECT constraint_name,
       constraint_type
FROM user_constraints
WHERE table_name = 'ALTER_MEMBER'
ORDER BY constraint_name;

5-8. 자료형 변경: MODIFY

ALTER TABLE alter_member
MODIFY login_id VARCHAR2(50);

DESC alter_member;

자료형 크기를 늘리는 것은 비교적 안전하지만,
줄이는 것은 기존 데이터 길이에 따라 오류가 날 수 있습니다.

5-9. 자료형 줄이기 전 데이터 길이 확인

SELECT member_name,
       LENGTH(member_name) AS name_length
FROM alter_member;

5-10. 컬럼명 변경

ALTER TABLE alter_member
RENAME COLUMN member_name TO name;

DESC alter_member;

컬럼명을 변경하면 기존 SQL, Entity, DTO, Mapper도 함께 수정해야 합니다.

5-11. 테이블명 변경

RENAME alter_member TO alter_member_renamed;

SELECT table_name
FROM user_tables
WHERE table_name LIKE 'ALTER_MEMBER%';

RENAME alter_member_renamed TO alter_member;
CTAS

6. CREATE TABLE AS SELECT

CREATE TABLE AS SELECT는 기존 테이블의 구조와 데이터를 복사해서 새 테이블을 만드는 문법입니다. 실습용 복사본이나 임시 백업 테이블을 만들 때 유용합니다.
단, PK, FK, UNIQUE 같은 제약조건은 그대로 복사되지 않습니다.

6-1. 테이블 복사

CREATE TABLE alter_member_copy
AS
SELECT *
FROM alter_member;

6-2. 복사 결과 확인

SELECT *
FROM alter_member_copy;

DESC alter_member_copy;

6-3. 제약조건 비교

SELECT table_name,
       constraint_name,
       constraint_type
FROM user_constraints
WHERE table_name IN ('ALTER_MEMBER', 'ALTER_MEMBER_COPY')
ORDER BY table_name, constraint_name;

주의

CTAS는 데이터와 기본 구조 복사에는 편하지만, 제약조건까지 완벽하게 복사되는 것은 아닙니다.
실습용 복사 테이블을 만든 뒤에는 USER_CONSTRAINTS로 반드시 확인해야 합니다.

DROP

7. DROP TABLE 예제

DROP TABLE은 테이블 객체 자체를 삭제합니다. 일반 DROP은 휴지통에 들어갈 수 있고, PURGE를 붙이면 휴지통을 거치지 않고 완전 삭제됩니다.

7-1. 일반 DROP

DROP TABLE alter_member_copy;

SELECT table_name
FROM user_tables
WHERE table_name = 'ALTER_MEMBER_COPY';

7-2. 휴지통 확인

SHOW RECYCLEBIN;

-- SQL Developer에서 SHOW가 불편하면 아래 쿼리 사용
SELECT object_name,
       original_name,
       type
FROM recyclebin
ORDER BY droptime DESC;

7-3. FLASHBACK 복구

FLASHBACK TABLE alter_member_copy TO BEFORE DROP;

SELECT table_name
FROM user_tables
WHERE table_name = 'ALTER_MEMBER_COPY';

7-4. PURGE 완전 삭제

DROP TABLE alter_member_copy PURGE;

SELECT original_name
FROM recyclebin
WHERE original_name = 'ALTER_MEMBER_COPY';

7-5. 부모 테이블 삭제 오류

DROP TABLE alter_member;

ALTER_BOARD가 ALTER_MEMBER를 참조하고 있다면
ORA-02449 오류가 발생할 수 있습니다.

7-6. CASCADE CONSTRAINTS

DROP TABLE alter_member CASCADE CONSTRAINTS;

부모 테이블을 삭제하면서 자식 테이블의 FK 제약조건을 제거합니다.
단, 자식 테이블 자체가 삭제되는 것은 아닙니다.

TRUNCATE

8. TRUNCATE TABLE 예제

TRUNCATE TABLE은 테이블 구조는 유지하고 데이터만 전체 삭제합니다. WHERE 조건을 사용할 수 없고, 일반적으로 ROLLBACK이 어렵기 때문에 주의해야 합니다.

8-1. 실습 테이블 생성

DROP TABLE truncate_test PURGE;

CREATE TABLE truncate_test (
    id NUMBER PRIMARY KEY,
    title VARCHAR2(100)
);

8-2. 데이터 입력

INSERT INTO truncate_test VALUES (1, '데이터1');
INSERT INTO truncate_test VALUES (2, '데이터2');
INSERT INTO truncate_test VALUES (3, '데이터3');

COMMIT;

SELECT *
FROM truncate_test;

8-3. TRUNCATE 실행

TRUNCATE TABLE truncate_test;

SELECT COUNT(*)
FROM truncate_test;

DESC truncate_test;

데이터는 0건이지만 테이블 구조는 그대로 남아 있어야 합니다.

8-4. DELETE와 TRUNCATE 비교

-- 일부 삭제 가능
DELETE FROM truncate_test
WHERE id = 1;

-- 전체 데이터 빠른 삭제
TRUNCATE TABLE truncate_test;
구분 DELETE FROM TRUNCATE TABLE
명령 성격 DML DDL 성격
WHERE 사용 가능 불가능
일부 삭제 가능 불가능
전체 삭제 가능 가능
ROLLBACK 가능 일반적으로 어려움
속도 상대적으로 느림 빠름

Mistake

11. 자주 하는 실수

  • 실수 1. 운영 DB에서 DROP이나 TRUNCATE를 가볍게 실행함 DROP은 테이블 자체를 삭제하고 TRUNCATE는 데이터를 전체 삭제하므로 운영 DB에서는 백업과 승인 절차가 필요합니다.
  • 실수 2. NOT NULL 컬럼을 기존 데이터 있는 테이블에 바로 추가함 기존 행에 새 컬럼 값이 없기 때문에 오류가 날 수 있습니다.
  • 실수 3. 컬럼 크기를 줄이다가 오류 발생 기존 데이터 길이가 새 자료형 크기보다 크면 MODIFY가 실패할 수 있습니다.
  • 실수 4. 제약조건을 수정하려고 함 제약조건은 보통 DROP CONSTRAINT 후 ADD CONSTRAINT로 다시 생성합니다.
  • 실수 5. CTAS로 복사하면 제약조건도 복사된다고 생각함 CTAS는 데이터와 기본 구조는 복사하지만 PK, FK, UNIQUE는 그대로 복사되지 않습니다.
  • 실수 6. DROP TABLE PURGE 후 FLASHBACK을 기대함 PURGE를 붙이면 휴지통을 거치지 않으므로 일반적인 FLASHBACK 복구가 어렵습니다.
  • 실수 7. TRUNCATE에 WHERE를 쓰려고 함 TRUNCATE는 전체 삭제만 가능하며 조건 삭제가 필요하면 DELETE를 사용해야 합니다.
  • 실수 8. 부모 테이블을 자식 테이블보다 먼저 DROP하려고 함 FK 참조 관계 때문에 삭제가 막힐 수 있으며 CASCADE CONSTRAINTS 사용 여부를 신중히 판단해야 합니다.
Interview

12. 면접 포인트

Q1. ALTER TABLE은 무엇인가요?

ALTER TABLE은 이미 생성된 테이블의 구조를 변경하는 DDL 명령입니다. 컬럼 추가, 컬럼 삭제, 자료형 변경, 컬럼명 변경, 제약조건 추가 및 삭제에 사용합니다.

Q2. DROP TABLE과 TRUNCATE TABLE의 차이는 무엇인가요?

DROP TABLE은 테이블 객체 자체를 삭제하여 구조와 데이터가 모두 사라집니다. TRUNCATE TABLE은 테이블 구조는 유지하고 내부 데이터만 전체 삭제합니다.

Q3. DELETE와 TRUNCATE의 차이는 무엇인가요?

DELETE는 DML이고 WHERE 조건으로 일부 행을 삭제할 수 있으며 ROLLBACK이 가능합니다.
TRUNCATE는 테이블 전체 데이터를 빠르게 삭제하는 DDL 성격의 명령이고, WHERE 조건을 사용할 수 없으며 일반적으로 ROLLBACK이 어렵습니다.

Q4. DROP TABLE과 DROP TABLE PURGE의 차이는 무엇인가요?

일반 DROP TABLE은 Oracle 휴지통에 들어갈 수 있어 FLASHBACK으로 복구할 수 있는 경우가 있습니다.
DROP TABLE PURGE는 휴지통을 거치지 않고 완전 삭제하므로 일반적인 복구가 어렵습니다.

Q5. CREATE TABLE AS SELECT의 특징은 무엇인가요?

기존 테이블의 구조와 데이터를 복사해서 새 테이블을 만드는 문법입니다.
실습용 복사본이나 임시 백업 테이블을 만들 때 유용하지만, PK, FK, UNIQUE 같은 제약조건은 그대로 복사되지 않습니다.

Q6. 이미 데이터가 있는 테이블에 NOT NULL 컬럼을 추가할 때 주의할 점은 무엇인가요?

기존 행에 새 컬럼 값이 없기 때문에 바로 NOT NULL 컬럼을 추가하면 오류가 날 수 있습니다.
먼저 NULL 허용 컬럼으로 추가하고, 기존 데이터에 값을 채운 뒤, MODIFY로 NOT NULL을 적용하는 방식이 안전합니다.

Q7. 부모 테이블을 DROP할 때 자식 테이블이 참조하고 있으면 어떻게 되나요?

자식 테이블이 부모 테이블의 PK나 UNIQUE 키를 FK로 참조하고 있으면 부모 테이블 삭제가 막힐 수 있습니다.
이때 자식 FK를 먼저 삭제하거나, 자식 테이블을 먼저 삭제하거나, DROP TABLE 부모테이블 CASCADE CONSTRAINTS를 사용할 수 있습니다.

Project

13. Java·Spring·Oracle 프로젝트 연결

ALTER, DROP, TRUNCATE는 단순 SQL 문법이 아니라 실제 프로젝트의 요구사항 변경, DB 마이그레이션, 배포 스크립트, Entity 수정, DTO 수정, Mapper 수정과 직접 연결됩니다.

ALTER와 요구사항 변경

ALTER TABLE users
ADD phone VARCHAR2(20);

회원가입에 전화번호 인증이 추가되면
DB 컬럼뿐 아니라 Entity, DTO, 화면도 함께 수정해야 합니다.

MODIFY와 Entity 길이 변경

ALTER TABLE users
MODIFY email VARCHAR2(100);

Java Entity의 @Column(length = 100), DTO의 @Size(max = 100)도
같이 맞춰야 합니다.

DROP COLUMN의 영향

ALTER TABLE users
DROP COLUMN nickname;

Entity 필드, DTO, Mapper SQL, Repository 쿼리, React 화면,
관리자 페이지에 영향을 줄 수 있습니다.

TRUNCATE와 임시 분석 테이블

TRUNCATE TABLE temp_recommendation_result;

추천 결과 임시 테이블이나 배치 중간 결과 테이블을
빠르게 비울 때 사용할 수 있습니다.

DROP과 배포 관리

DROP TABLE temp_test_table PURGE;

개발 중 임시 테이블 삭제에는 유용하지만 운영 DB에서는 매우 위험합니다.

DB 마이그레이션 연결

V1__create_users.sql
V2__add_phone_to_users.sql
V3__modify_email_length.sql
V4__drop_unused_column.sql

Flyway나 Liquibase 같은 도구로 DB 변경 이력을 관리할 수 있습니다.

학습 메모

오늘의 핵심

ALTER는 구조 변경, DROP은 테이블 삭제, TRUNCATE는 구조를 유지한 채 데이터만 전체 삭제하는 명령입니다.

암기 문법

ALTER TABLE 테이블명 ADD 컬럼명 자료형;

ALTER TABLE 테이블명
ADD CONSTRAINT 제약조건명 제약조건;

ALTER TABLE 테이블명 DROP COLUMN 컬럼명;

ALTER TABLE 테이블명 DROP CONSTRAINT 제약조건명;

ALTER TABLE 테이블명 MODIFY 컬럼명 새자료형;

ALTER TABLE 테이블명
RENAME COLUMN 기존컬럼명 TO 새컬럼명;

DROP TABLE 테이블명;

TRUNCATE TABLE 테이블명;

DROP 관련 문법

  • DROP TABLE : 테이블 삭제, 휴지통에 남을 수 있음
  • DROP TABLE PURGE : 휴지통 없이 완전 삭제
  • FLASHBACK TABLE : 휴지통 테이블 복구
  • PURGE RECYCLEBIN : 휴지통 비우기
  • CASCADE CONSTRAINTS : 참조 제약조건까지 정리

DELETE vs TRUNCATE

  • DELETE : WHERE 가능, 일부 삭제 가능, ROLLBACK 가능
  • TRUNCATE : WHERE 불가, 전체 삭제, 빠르지만 복구 어려움

주의할 점

DROP과 TRUNCATE는 운영 DB에서 매우 위험합니다. 실행 전 반드시 대상 테이블, 백업, 관련 API, Entity, Mapper, 화면 영향을 확인해야 합니다.

프로젝트 연결

DB ALTER는 Entity 수정, DTO 수정, Repository/Mapper 수정, Service 로직 수정, React 화면 수정, 배포 스크립트 관리로 이어집니다.
