Oracle SQL Developer - Chapter 25

분석 함수 Analytic Function

분석 함수는 조회 결과의 행을 줄이지 않고,
각 행에 순위, 그룹별 통계, 이전 행, 다음 행, 페이징 번호 같은 분석 결과를 붙이는 SQL 문법입니다.
이번 단원에서는 OVER, PARTITION BY, ORDER BY, ROW_NUMBER, RANK, DENSE_RANK,
SUM·AVG OVER, LAG, LEAD를 실습하고 Java·Spring·Oracle 프로젝트의 목록 화면과 연결합니다.

학습 목표 GROUP BY와 분석 함수의 차이를 이해하고
행을 유지한 채 통계와 순위를 붙인다.
실습 방향 EMP 기준 평균·순위 실습 →
HR.EMPLOYEES 페이징 → 게시판 이전글/다음글
프로젝트 연결 게시판 페이징, 상품 Top-N,
부서별 순위, 관리자 리포트 SQL과 연결한다.
Overview

1. 분석 함수는 왜 배우는가?

일반 그룹 함수와 GROUP BY는 데이터를 요약해서 행 수를 줄입니다. 예를 들어 부서별 평균 급여를 구하면 부서별 한 줄만 남습니다.
하지만 실무에서는 직원 목록을 그대로 보여주면서 각 직원의 부서 평균 급여, 부서 내 순위, 전체 순번, 이전 데이터와 다음 데이터까지 함께 보여줘야 할 때가 많습니다.

행 유지 직원 목록, 상품 목록, 게시글 목록의 행을
그대로 유지한다.
분석값 추가 평균, 합계, 순위, 순번, 이전글, 다음글을 각 행에 붙인다.
실무 화면 연결 페이징, Top-N, 관리자 리포트,
상세 페이지 네비게이션에 사용한다.

한 줄 정리

분석 함수는 GROUP BY처럼 집계하거나 순위를 계산하지만, GROUP BY와 달리 원본 행을 줄이지 않고 각 행에 분석 결과를 붙입니다.

Concept Card

2. 단원 개념 카드

2-1. GROUP BY와 분석 함수의 핵심 차이

GROUP BY는 그룹별로 결과를 요약합니다. 분석 함수는 행을 유지하면서 그룹별 계산 결과를 붙입니다.

GROUP BY

SELECT DEPTNO,
       ROUND(AVG(SAL), 2) AS DEPT_AVG_SAL
FROM EMP
GROUP BY DEPTNO
ORDER BY DEPTNO;

부서별 평균 급여를 구하지만 결과는 부서별 한 줄입니다.
직원 이름, 사번, 개인 급여는 사라집니다.

분석 함수

SELECT EMPNO,
       ENAME,
       DEPTNO,
       SAL,
       ROUND(AVG(SAL) OVER (PARTITION BY DEPTNO), 2) AS DEPT_AVG_SAL
FROM EMP
ORDER BY DEPTNO, EMPNO;

직원 행은 그대로 유지되고,
각 직원 행마다 자신이 속한 부서의 평균 급여가 붙습니다.

2-2. 분석 함수 기본 문법

기본형 
분석함수([인자]) OVER (
    PARTITION BY 그룹기준컬럼
    ORDER BY 정렬기준컬럼
)
분석함수 What

무엇을 계산할지 정합니다.
AVG, SUM, ROW_NUMBER, RANK, LAG, LEAD 등이 들어갑니다.

OVER Analytic Mode

일반 집계가 아니라 행을 유지하는 분석 함수 방식으로 계산하겠다는 표시입니다.

PARTITION BY Group Range

부서별, 직무별, 카테고리별처럼 분석 범위를 나누는 기준입니다.

ORDER BY Rank Order

순위, 순번, 누적합, 이전/다음 행을 계산할 때 기준이 되는 정렬입니다.

2-3. 분석 함수 종류

분류 대표 함수 의미 실무 연결
집계 분석 함수 AVG, SUM, MIN, MAX, COUNT 그룹별 평균, 합계, 최소, 최대, 건수를 각 행에 표시 부서별 평균, 카테고리별 매출 합계, 회원 등급별 인원수
순위 분석 함수 ROW_NUMBER, RANK, DENSE_RANK 정렬 기준에 따라 순번 또는 순위를 부여 페이징, 급여 순위, 상품 판매 순위, Top-N
순서 분석 함수 LAG, LEAD 현재 행 기준 이전 행 또는 다음 행의 값을 가져옴 게시판 이전글·다음글, 전월 대비 매출 비교
누적 분석 SUM OVER ORDER BY 정렬 순서에 따라 현재 행까지 누적 계산 월별 누적 매출, 부서별 누적 급여, 누적 포인트

3. 그림처럼 이해하기

3-1. GROUP BY는 행을 줄인다

1 EMP 14행

직원 데이터 전체를 읽습니다.

2 DEPTNO 그룹화

10, 20, 30번 부서로 묶습니다.

3 AVG(SAL)

부서별 평균 급여를 계산합니다.

4 부서별 1행

결과는 부서 수만큼만 나옵니다.

3-2. 분석 함수는 행을 유지한다

1 EMP 14행

직원 데이터 전체를 읽습니다.

2 PARTITION

DEPTNO별 내부 그룹을 만듭니다.

3 AVG OVER

각 부서 안에서 평균을 계산합니다.

4 각 행에 붙임

직원 행은 유지되고 평균값만 추가됩니다.

3-3. PARTITION BY 그림

DEPTNO = 10

CLARK KING MILLER

10번 부서 안에서 평균·순위 계산

DEPTNO = 20

SMITH JONES SCOTT FORD

20번 부서 안에서 다시 1등부터 시작

DEPTNO = 30

ALLEN WARD BLAKE MARTIN

30번 부서 안에서 별도 분석

PARTITION BY 핵심

PARTITION BY DEPTNO는 전체 직원을 부서별 작은 묶음으로 나누고, 각 묶음 안에서 평균, 합계, 순위 등을 계산하게 만듭니다.

Execution Order

4. 실행 순서 그림 카드

분석 함수 결과를 WHERE절에서 바로 사용할 수 없는 이유는 SELECT문의 실행 순서와 관련이 있습니다.
WHERE는 SELECT보다 먼저 실행되고, 분석 함수 별칭은 SELECT 단계에서 만들어집니다.

1 FROM

테이블과 조인 결과를
만든다.

2 WHERE

분석 전에 대상 행을
먼저 거른다.

3 GROUP BY

필요하면 그룹을 만든다.

4 HAVING

그룹 조건을 거른다.

5 SELECT

분석 함수와 별칭이
계산된다.

6 ORDER BY

최종 결과를 정렬한다.

잘못된 예

SELECT ENAME,
       SAL,
       ROW_NUMBER() OVER (ORDER BY SAL DESC) AS RN
FROM EMP
WHERE RN <= 5;

RN은 SELECT 단계에서 만들어지는 별칭입니다.
WHERE 단계에서는 아직 RN을 알 수 없습니다.

올바른 예

SELECT *
FROM (
    SELECT E.*,
           ROW_NUMBER() OVER (ORDER BY SAL DESC, EMPNO) AS RN
    FROM EMP E
)
WHERE RN <= 5
ORDER BY RN;

안쪽 쿼리에서 RN을 만들고, 바깥 쿼리에서 RN 조건을 걸어야 합니다.

5. 기본 문법 카드

5-1. 전체 평균을 각 행에 표시

AVG(SAL) OVER ()

전체 결과 집합을 하나의 분석 범위로 보고 평균을 각 행에 표시합니다.

5-2. 부서별 평균을 각 행에 표시

AVG(SAL) OVER (PARTITION BY DEPTNO)

DEPTNO별로 나누어 평균을 계산하고 각 직원 행에 붙입니다.

5-3. 전체 순번

ROW_NUMBER() OVER (ORDER BY SAL DESC)

급여 높은 순서대로 중복 없는 1, 2, 3, 4 순번을 붙입니다.

5-4. 전체 순위

RANK() OVER (ORDER BY SAL DESC)

동점자는 같은 순위이고, 다음 순위는 건너뜁니다.

5-5. 연속 순위

DENSE_RANK() OVER (ORDER BY SAL DESC)

동점자는 같은 순위이지만, 다음 순위는 건너뛰지 않습니다.

5-6. 부서별 순위

RANK() OVER (
    PARTITION BY DEPTNO
    ORDER BY SAL DESC
)

부서가 바뀔 때마다 순위가 다시 1부터 시작합니다.

5-7. 이전 행

LAG(ENAME, 1, '이전 없음') OVER (ORDER BY EMPNO)

현재 행 기준 바로 이전 행의 ENAME을 가져옵니다.

5-8. 다음 행

LEAD(ENAME, 1, '다음 없음') OVER (ORDER BY EMPNO)

현재 행 기준 바로 다음 행의 ENAME을 가져옵니다.

7. SQL 예제 카드 - GROUP BY와 OVER 비교

7-1. GROUP BY로 부서 평균

SELECT DEPTNO,
       ROUND(AVG(SAL), 2) AS DEPT_AVG_SAL
FROM EMP
GROUP BY DEPTNO
ORDER BY DEPTNO;
결과 확인 부서별 한 줄만 출력됩니다. 직원 개별 행은 사라집니다.

7-2. 분석 함수로 부서 평균

SELECT EMPNO,
       ENAME,
       DEPTNO,
       SAL,
       ROUND(AVG(SAL) OVER (PARTITION BY DEPTNO), 2) AS DEPT_AVG_SAL
FROM EMP
ORDER BY DEPTNO, EMPNO;
결과 확인 직원 행은 유지되고, 같은 부서끼리 같은 평균값이 표시됩니다.

7-3. 전체 평균과 부서 평균 동시 표시

SELECT EMPNO,
       ENAME,
       DEPTNO,
       SAL,
       ROUND(AVG(SAL) OVER (), 2) AS ALL_AVG_SAL,
       ROUND(AVG(SAL) OVER (PARTITION BY DEPTNO), 2) AS DEPT_AVG_SAL
FROM EMP
ORDER BY DEPTNO, SAL DESC;

ALL_AVG_SAL은 모든 행에서 같고, DEPT_AVG_SAL은 같은 부서끼리 같습니다.

7-4. 부서 평균 이상/미만 비교

SELECT EMPNO,
       ENAME,
       DEPTNO,
       SAL,
       ROUND(AVG(SAL) OVER (PARTITION BY DEPTNO), 2) AS DEPT_AVG_SAL,
       CASE
           WHEN SAL >= AVG(SAL) OVER (PARTITION BY DEPTNO)
           THEN '부서 평균 이상'
           ELSE '부서 평균 미만'
       END AS COMPARE_RESULT
FROM EMP
ORDER BY DEPTNO, SAL DESC;

분석 함수 결과를 CASE와 결합하면 화면용 판단 컬럼을 만들 수 있습니다.

8. SQL 예제 카드 - 집계 함수를 분석 함수처럼 사용

AVG, SUM, MIN, MAX, COUNT 같은 그룹 함수도 OVER와 함께 사용하면 행을 유지한 채 그룹별 집계값을 표시할 수 있습니다.

8-1. 부서별 최소·최대·합계·평균을 각 행에 표시

SELECT ENAME,
       DEPTNO,
       SAL,
       MIN(SAL) OVER (PARTITION BY DEPTNO) AS DEPT_MIN_SAL,
       MAX(SAL) OVER (PARTITION BY DEPTNO) AS DEPT_MAX_SAL,
       SUM(SAL) OVER (PARTITION BY DEPTNO) AS DEPT_SUM_SAL,
       ROUND(AVG(SAL) OVER (PARTITION BY DEPTNO), 2) AS DEPT_AVG_SAL
FROM EMP
ORDER BY DEPTNO, SAL DESC;
결과 확인 같은 DEPTNO 직원들은 DEPT_MIN_SAL, DEPT_MAX_SAL, DEPT_SUM_SAL, DEPT_AVG_SAL이 같습니다.

8-2. 직무별 총 급여를 각 행에 표시

SELECT EMPNO,
       ENAME,
       JOB,
       SAL,
       SUM(SAL) OVER (PARTITION BY JOB) AS JOB_TOTAL_SAL
FROM EMP
ORDER BY JOB, SAL DESC;

JOB이 같은 직원들은 JOB_TOTAL_SAL 값이 같습니다.

8-3. GROUP BY 방식과 비교

SELECT JOB,
       SUM(SAL) AS JOB_TOTAL_SAL
FROM EMP
GROUP BY JOB
ORDER BY JOB;

이 방식은 직무별 한 줄만 나오므로 직원 이름은 같이 볼 수 없습니다.

8-4. 부서별 직원 수 표시

SELECT EMPNO,
       ENAME,
       DEPTNO,
       COUNT(*) OVER (PARTITION BY DEPTNO) AS DEPT_EMP_COUNT
FROM EMP
ORDER BY DEPTNO, EMPNO;

각 직원 행마다 해당 부서의 직원 수가 표시됩니다.

SQL Example

9. SQL 예제 카드 - ROW_NUMBER / RANK / DENSE_RANK

ROW_NUMBER() 1, 2, 3, 4

동점이어도 무조건 서로 다른 번호를 붙입니다.

RANK() 1, 2, 2, 4

동점자는 같은 순위이고 다음 순위는 건너뜁니다.

DENSE_RANK() 1, 2, 2, 3

동점자는 같은 순위이지만
다음 순위는 건너뛰지 않습니다.

9-1. 세 함수 한 번에 비교

SELECT ENAME,
       SAL,
       ROW_NUMBER() OVER (ORDER BY SAL DESC) AS ROW_NUM,
       RANK() OVER (ORDER BY SAL DESC) AS RANK_NUM,
       DENSE_RANK() OVER (ORDER BY SAL DESC) AS DENSE_RANK_NUM
FROM EMP
ORDER BY SAL DESC;
결과 확인 SCOTT과 FORD처럼 SAL이 같은 행에서 ROW_NUMBER, RANK, DENSE_RANK 차이를 확인합니다.

9-2. 안정적인 ROW_NUMBER

SELECT ENAME,
       SAL,
       EMPNO,
       ROW_NUMBER() OVER (ORDER BY SAL DESC, EMPNO ASC) AS RN
FROM EMP
ORDER BY RN;

SAL이 같을 때 EMPNO로 한 번 더 정렬해 페이징 결과를 안정적으로 만듭니다.

9-3. 부서별 급여 순위

SELECT ENAME,
       DEPTNO,
       SAL,
       RANK() OVER (PARTITION BY DEPTNO ORDER BY SAL DESC) AS DEPT_SAL_RANK
FROM EMP
ORDER BY DEPTNO, DEPT_SAL_RANK;

부서가 바뀔 때마다 순위가 다시 1부터 시작합니다.

9-4. HR 부서별 급여 순위

SELECT EMPLOYEE_ID,
       LAST_NAME,
       DEPARTMENT_ID,
       SALARY,
       RANK() OVER (
           PARTITION BY DEPARTMENT_ID
           ORDER BY SALARY DESC
       ) AS DEPT_SALARY_RANK
FROM EMPLOYEES
ORDER BY DEPARTMENT_ID, DEPT_SALARY_RANK;

DEPARTMENT_ID가 NULL인 직원도 하나의 파티션처럼 묶일 수 있습니다.

SQL Example

10. SQL 예제 카드 - Top-N과 페이징

분석 함수의 ROW_NUMBER는 Top-N과 페이징에 매우 자주 사용됩니다. 정확히 N개 행을 잘라야 하는 목록 화면에서는 RANK보다 ROW_NUMBER가 더 적합합니다.

10-1. 급여 Top 5

SELECT *
FROM (
    SELECT E.*,
           ROW_NUMBER() OVER (ORDER BY SAL DESC, EMPNO) AS RN
    FROM EMP E
)
WHERE RN <= 5
ORDER BY RN;

급여 높은 순서로 정확히 5행을 가져옵니다.

10-2. 급여 6번째부터 10번째까지

SELECT *
FROM (
    SELECT E.*,
           ROW_NUMBER() OVER (ORDER BY SAL DESC, EMPNO) AS RN
    FROM EMP E
)
WHERE RN BETWEEN 6 AND 10
ORDER BY RN;

정렬된 결과의 6번째부터 10번째 행을 가져옵니다.

10-3. WITH절로 페이징

WITH ranked_emp AS (
    SELECT E.*,
           ROW_NUMBER() OVER (ORDER BY SAL DESC, EMPNO) AS RN
    FROM EMP E
)
SELECT EMPNO,
       ENAME,
       SAL,
       DEPTNO,
       RN
FROM ranked_emp
WHERE RN BETWEEN 6 AND 10
ORDER BY RN;

복잡한 쿼리는 WITH절로 이름을 붙이면 읽기 쉬워집니다.

10-4. 동점자 포함 Top-N

SELECT *
FROM (
    SELECT E.*,
           RANK() OVER (ORDER BY SAL DESC) AS SAL_RANK
    FROM EMP E
)
WHERE SAL_RANK <= 5
ORDER BY SAL_RANK, SAL DESC;

5등과 동점인 직원까지 포함하려면 RANK를 사용합니다.

페이징 공식

SELECT *
FROM (
    SELECT T.*,
           ROW_NUMBER() OVER (ORDER BY 정렬컬럼 DESC, PK컬럼) AS RN
    FROM 테이블 T
)
WHERE RN BETWEEN 시작번호 AND 끝번호
ORDER BY RN;

11. SQL 예제 카드 - LAG와 LEAD

LAG는 현재 행 기준 이전 행의 값을 가져오고, LEAD는 현재 행 기준 다음 행의 값을 가져옵니다. 게시판 상세 페이지의 이전글·다음글, 전월 대비 매출 비교에 자주 사용됩니다.

이전 행 LAG()

현재 행보다 정렬상 앞에 있는 행

← 현재 행 →
다음 행 LEAD()

현재 행보다 정렬상 뒤에 있는 행

11-1. 이전 사원번호, 다음 사원번호

SELECT EMPNO,
       ENAME,
       LAG(EMPNO) OVER (ORDER BY EMPNO) AS PREV_EMPNO,
       LEAD(EMPNO) OVER (ORDER BY EMPNO) AS NEXT_EMPNO
FROM EMP
ORDER BY EMPNO;

첫 번째 행의 PREV_EMPNO와 마지막 행의 NEXT_EMPNO는 NULL입니다.

11-2. 기본값 지정

SELECT EMPNO,
       ENAME,
       LAG(ENAME, 1, '이전 없음') OVER (ORDER BY EMPNO) AS PREV_ENAME,
       LEAD(ENAME, 1, '다음 없음') OVER (ORDER BY EMPNO) AS NEXT_ENAME
FROM EMP
ORDER BY EMPNO;

이전 또는 다음 행이 없을 때 표시할 기본값을 지정할 수 있습니다.

11-3. 게시판 이전글·다음글 실습용 CTE

WITH BBS AS (
    SELECT 101 AS ID, '첫 번째 글' AS TITLE FROM DUAL UNION ALL
    SELECT 102 AS ID, '두 번째 글' AS TITLE FROM DUAL UNION ALL
    SELECT 103 AS ID, '세 번째 글' AS TITLE FROM DUAL UNION ALL
    SELECT 104 AS ID, '네 번째 글' AS TITLE FROM DUAL UNION ALL
    SELECT 105 AS ID, '다섯 번째 글' AS TITLE FROM DUAL
),
BBS_NAV AS (
    SELECT ID,
           TITLE,
           LAG(ID) OVER (ORDER BY ID DESC) AS NEWER_ID,
           LAG(TITLE) OVER (ORDER BY ID DESC) AS NEWER_TITLE,
           LEAD(ID) OVER (ORDER BY ID DESC) AS OLDER_ID,
           LEAD(TITLE) OVER (ORDER BY ID DESC) AS OLDER_TITLE
    FROM BBS
)
SELECT *
FROM BBS_NAV
WHERE ID = 103;
결과 확인 ID DESC 기준에서 103번 글의 위쪽 최신 글은 104번, 아래쪽 오래된 글은 102번입니다.
SQL Example

12. SQL 예제 카드 - 누적 합계 맛보기

분석 함수 안에 ORDER BY를 넣으면 정렬 순서에 따른 누적 계산도 가능합니다. 처음에는 어렵게 느껴질 수 있지만, 매출 누적합이나 포인트 누적합에서 자주 사용됩니다.

12-1. 부서별 총합 반복 표시

SELECT ENAME,
       DEPTNO,
       SAL,
       SUM(SAL) OVER (PARTITION BY DEPTNO) AS DEPT_TOTAL_SAL
FROM EMP
ORDER BY DEPTNO, SAL DESC;

같은 부서 직원들에게 부서 총합이 동일하게 표시됩니다.

12-2. 부서별 누적 급여

SELECT ENAME,
       DEPTNO,
       SAL,
       SUM(SAL) OVER (
           PARTITION BY DEPTNO
           ORDER BY SAL DESC
           ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
       ) AS DEPT_RUNNING_TOTAL
FROM EMP
ORDER BY DEPTNO, SAL DESC;

부서 안에서 급여 높은 순서로 현재 행까지의 누적 합계를 계산합니다.

Mistake

15. 자주 하는 실수

  • 실수 1. GROUP BY와 PARTITION BY를 같은 결과라고 생각함 둘 다 그룹 기준을 만들지만 GROUP BY는 행을 줄이고, PARTITION BY는 행을 유지합니다.
  • 실수 2. 분석 함수 결과 별칭을 WHERE에서 바로 사용함 RN, SAL_RANK 같은 별칭은 SELECT 단계에서 만들어지므로 WHERE에서 바로 사용할 수 없습니다.
  • 실수 3. 페이징에 RANK를 사용함 RANK는 동점자 때문에 결과 행 수가 일정하지 않을 수 있습니다. 일반 페이징은 ROW_NUMBER가 적합합니다.
  • 실수 4. ROW_NUMBER 정렬 기준을 하나만 둠 SALARY가 같은 행이 있으면 순서가 불안정할 수 있으므로 PK 컬럼을 추가하는 것이 좋습니다.
  • 실수 5. 분석 함수 안의 ORDER BY와 최종 ORDER BY를 혼동함 OVER 안의 ORDER BY는 계산 기준이고, SELECT 맨 끝의 ORDER BY는 화면 출력 순서입니다.
  • 실수 6. 부서별 순위에서 PARTITION BY를 빼먹음 PARTITION BY DEPTNO가 없으면 부서별 순위가 아니라 전체 직원 기준 순위가 됩니다.
  • 실수 7. LAG와 LEAD 방향을 UI 용어로만 외움 LAG는 정렬 기준상 이전 행, LEAD는 정렬 기준상 다음 행입니다. 이전글·다음글 명칭은 ORDER BY 방향에 따라 달라질 수 있습니다.
  • 실수 8. 조인 후 행 수가 늘어난 상태에서 ROW_NUMBER를 붙임 1:N 조인으로 행이 늘어난 경우 직원 순번이 아니라 직원-상세행 순번이 될 수 있습니다.
  • 실수 9. 분석 전에 필터링해야 할 조건을 바깥 WHERE에 둠 재직자 중 순위를 매기려면 안쪽 쿼리에서 WHERE RETIRE_YN = 'N'을 먼저 적용해야 합니다.
  • 실수 10. 누적합의 윈도우 범위를 이해하지 않고 사용함 누적합은 ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW처럼 범위를 명확히 쓰면 이해하기 쉽습니다.
Interview

16. 면접 질문 카드

Q1. 분석 함수란 무엇인가요?

분석 함수는 조회 결과의 행을 유지하면서 각 행에 순위, 그룹별 집계값, 이전 행, 다음 행 같은 분석 결과를 붙이는 함수입니다.
GROUP BY와 달리 결과 행 수를 줄이지 않는다는 점이 핵심입니다.

Q2. GROUP BY와 분석 함수의 차이는 무엇인가요?

GROUP BY는 그룹당 한 행을 반환하므로 행 수가 줄어듭니다. 분석 함수는 원래 행을 유지하면서 각 행에 그룹 기준 계산 결과를 붙입니다.

Q3. OVER는 무슨 뜻인가요?

OVER는 함수를 분석 함수 방식으로 사용하겠다는 표시입니다. AVG(SAL)은 일반 집계 함수이지만 AVG(SAL) OVER(...)는 행을 유지하면서 평균을 붙입니다.

Q4. PARTITION BY는 무엇인가요?

분석 함수 안에서 데이터를 그룹처럼 나누는 기준입니다. PARTITION BY DEPTNO는 부서별로 분석 범위를 나누겠다는 뜻입니다.

Q5. 분석 함수 안의 ORDER BY와 마지막 ORDER BY는 같은가요?

다릅니다. OVER 안의 ORDER BY는 순위나 누적 계산의 기준이고, SELECT문 마지막 ORDER BY는 최종 결과 화면의 정렬 기준입니다.

Q6. ROW_NUMBER, RANK, DENSE_RANK의 차이는 무엇인가요?

ROW_NUMBER는 동점이어도 고유 번호를 붙입니다. RANK는 동점자에게 같은 순위를 주고 다음 순위를 건너뜁니다.
DENSE_RANK는 동점자에게 같은 순위를 주지만 다음 순위를 건너뛰지 않습니다.

Q7. 페이징에는 어떤 함수를 쓰는 것이 좋나요?

일반적인 페이징에는 ROW_NUMBER가 적합합니다. 페이지당 정확히 10개씩 가져와야 하므로 동점자 때문에 행 수가 달라질 수 있는 RANK는 주의해야 합니다.

Q8. 분석 함수 결과를 WHERE절에서 바로 사용할 수 있나요?

바로 사용할 수 없습니다. 분석 함수는 SELECT 단계에서 계산되므로 서브쿼리나 WITH절로 감싼 후 바깥 WHERE에서 조건을 걸어야 합니다.

Q9. LAG와 LEAD는 어디에 쓰나요?

LAG는 이전 행, LEAD는 다음 행의 값을 가져옵니다. 게시판 이전글·다음글, 전월 대비 매출, 이전 주문과 현재 주문 비교에 사용합니다.

Q10. 실무에서 분석 함수는 어디에 많이 쓰나요?

게시판 페이징, 상품 판매 순위, 부서별 급여 순위, 회원 포인트 순위, 카테고리별 Top-N, 관리자 리포트, 누적 매출, 이전글·다음글 기능에 많이 사용합니다.

→ DENSE_RANK()

→ LAG()

다음 행 가져오기
→ LEAD()

학습 메모

오늘의 핵심

분석 함수는 행을 줄이지 않고 각 행에 분석 결과를 붙입니다. GROUP BY가 “요약 보고서”라면 분석 함수는 “목록 화면 + 분석값”입니다.

기본 문법

분석함수() OVER (
    PARTITION BY 그룹기준
    ORDER BY 정렬기준
)

GROUP BY와 비교

GROUP BY
→ 그룹당 한 줄
→ 행 수 감소

분석 함수
→ 원본 행 유지
→ 각 행에 분석값 추가

순위 함수 차이

ROW_NUMBER : 1, 2, 3, 4
RANK       : 1, 2, 2, 4
DENSE_RANK : 1, 2, 2, 3

페이징 공식

WITH ranked AS (
    SELECT T.*,
           ROW_NUMBER() OVER (ORDER BY 정렬컬럼 DESC, PK컬럼) AS RN
    FROM 테이블 T
)
SELECT *
FROM ranked
WHERE RN BETWEEN :startRow AND :endRow
ORDER BY RN;

LAG / LEAD

LAG(컬럼) OVER (ORDER BY 정렬컬럼)
→ 이전 행

LEAD(컬럼) OVER (ORDER BY 정렬컬럼)
→ 다음 행

주의

분석 함수 결과 별칭은 WHERE에서 바로 사용할 수 없습니다. 반드시 서브쿼리나 WITH절로 감싼 뒤 바깥 WHERE에서 조건을 거세요.

프로젝트 연결

Spring 프로젝트에서 page, size를 받으면 Service에서 startRow와 endRow를 계산하고, Mapper SQL에서 ROW_NUMBER로 구간을 잘라 조회합니다.
