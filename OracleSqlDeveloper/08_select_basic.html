SELECT 기본 조회

SELECT는 테이블에 저장된 데이터를 조회하는 가장 기본적인 SQL입니다.
이번 단원에서는 전체 조회, 특정 컬럼 조회, 산술 표현식, 별칭, DISTINCT, WHERE 맛보기까지
SQL Developer에서 직접 실행하며 결과 화면을 읽는 방법을 정리합니다.

학습 목표 SELECT 기본 문법과 결과
화면의 컬럼 출력 방식을 이해한다.
실습 방향 SCOTT 계정의 EMP 테이블을 기준으로
전체 조회와 컬럼 조회를 실행한다.
프로젝트 연결 로그인, 목록 조회, 상세 조회, 추천 데이터 조회가
SELECT와 연결된다.
Overview

1. SELECT 문이란?

SELECT는 테이블에 저장된 데이터를 조회하는 SQL입니다.
SQL의 종류 중에서는 DML에 속하지만, INSERT, UPDATE, DELETE와 달리 데이터를 변경하지 않고 필요한 데이터를 결과 화면으로 가져오는 역할을 합니다.

FROM 어느 테이블에서 데이터를 가져올지 결정합니다.
SELECT 그 테이블에서 어떤 컬럼을 보여줄지 결정합니다.
Result Grid SQL Developer 결과 화면에
조회 결과가 표 형태로 출력됩니다.

한 줄 정리

SELECT는 어느 테이블에서 어떤 컬럼을 가져와 보여줄지 결정하는 조회 명령입니다. 처음에는 SELECT 컬럼명 FROM 테이블명 구조를 정확히 읽는 것이 가장 중요합니다.

Syntax

2. SELECT 기본 형식

SELECT 문은 간단한 조회에서 시작해 WHERE, GROUP BY, HAVING, ORDER BY까지 확장될 수 있습니다.
지금 단계에서는 SELECT와 FROM을 중심으로 이해하고, WHERE는 특정 행 검색 맛보기로만 다룹니다.

2-1. 가장 기본 형태

SELECT 컬럼명
FROM 테이블명;

하나의 테이블에서 하나의 컬럼을 조회하는 가장 기본적인 구조입니다.

2-2. 여러 컬럼 조회

SELECT 컬럼명1, 컬럼명2, 컬럼명3
FROM 테이블명;

여러 컬럼을 조회할 때는 컬럼명 사이를 쉼표로 구분합니다.

2-3. SELECT 문의 전체 형식

SELECT 컬럼명, 컬럼명, ...
FROM 테이블명
[WHERE 찾는 조건]
[GROUP BY 결과 데이터 그룹화]
[HAVING 그룹에서 찾는 조건]
[ORDER BY 데이터 정렬 표현];

대괄호로 표시한 부분은 선택 사항입니다. 이번 단원에서는 SELECT, FROM, 산술 표현식, 별칭, DISTINCT, WHERE 맛보기에 집중합니다.

Visual

4. 그림처럼 이해하기

테이블은 행과 컬럼으로 이루어진 표입니다. SELECT는 그중 어떤 세로줄, 즉 어떤 컬럼을 결과 화면으로 가져올지 정하는 명령입니다.

EMPNO ENAME JOB SAL DEPTNO
7369 SMITH CLERK 800 20
7499 ALLEN SALESMAN 1600 30
7521 WARD SALESMAN 1250 30
SELECT *
EMPNO ENAME JOB SAL DEPTNO

모든 컬럼을 조회합니다.
학습할 때 전체 데이터를 빠르게 확인하기 좋습니다.

SELECT ENAME, SAL
ENAME SAL

필요한 컬럼만 골라서 조회합니다.
실무에서는 이 방식이 더 권장됩니다.

WHERE
DEPTNO = 10

컬럼이 아니라 행을 거르는 조건입니다.
SELECT는 컬럼 선택, WHERE는 행 필터링입니다.

Prepare

5. SQL Developer 실습 준비

SELECT 실습을 시작하기 전에 현재 접속 계정과 테이블 목록, 테이블 구조를 확인해야 합니다. 이번 단원에서는 SCOTT 계정의 EMP 테이블을 기준으로 실습합니다.

5-1. 현재 사용자 확인

SHOW USER;

현재 어떤 계정으로 접속했는지 확인합니다. SCOTT 계정으로 접속했다면 EMP, DEPT 같은 예제 테이블을 사용할 수 있습니다.

5-2. 현재 계정의 테이블 목록 확인

SELECT table_name
FROM user_tables;

현재 접속 계정이 가진 테이블 목록을 확인합니다. EMP, DEPT, BONUS, SALGRADE가 보이면 SELECT 실습을 진행하기 좋습니다.

5-3. EMP 테이블 구조 확인

DESC EMP;

ENAME    사원이름
JOB      직무
MGR      관리자 번호
HIREDATE 입사일
SAL      급여
COMM     커미션
DEPTNO   부서번호
SELECT *

6. 모든 컬럼 조회

모든 컬럼을 조회할 때는 별표(*)를 사용합니다. 학습할 때 테이블 전체 구조와 데이터를 빠르게 확인하기 좋습니다.

EMP 테이블 전체 조회

SELECT *
FROM EMP;

EMP 테이블의 모든 컬럼과 모든 행을 조회합니다. SQL Developer에서는 결과 그리드에 EMPNO, ENAME, JOB, MGR, HIREDATE, SAL, COMM, DEPTNO 컬럼이 모두 표시됩니다.

실무에서는 SELECT *를 조심하기

SELECT *는 편리하지만 필요 없는 컬럼까지 모두 가져옵니다. 실무에서는 성능, 보안, API 응답 구조의 명확성을 위해 필요한 컬럼만 명시하는 것이 좋습니다.

Columns

7. 특정 컬럼 조회

특정 컬럼만 조회하려면 SELECT 뒤에 원하는 컬럼명을 쉼표로 구분해서 작성합니다. 결과 화면의 컬럼 순서는 SELECT에 작성한 순서대로 표시됩니다.

7-1. 사원번호, 이름, 급여 조회

SELECT EMPNO, ENAME, SAL
FROM EMP;

EMP 테이블에서 EMPNO, ENAME, SAL 컬럼만 조회합니다.

7-2. 컬럼 출력 순서 바꾸기

SELECT ENAME, SAL, EMPNO
FROM EMP;

결과 화면은 ENAME, SAL, EMPNO 순서로 출력됩니다. 테이블에 저장된 원래 컬럼 순서와 관계없이 SELECT에 적은 순서가 결과 순서가 됩니다.

7-3. 직무와 부서번호 조회

SELECT ENAME, JOB, DEPTNO
FROM EMP;

사원의 이름, 직무, 부서번호만 확인하고 싶을 때 사용하는 기본 조회입니다.

Expression

8. 산술 표현식

SELECT 문에서는 컬럼 값을 그대로 출력할 수도 있지만, 숫자 컬럼을 이용해 계산 결과를 함께 출력할 수도 있습니다.
산술 연산자는 더하기(+), 빼기(-), 곱하기(*), 나누기(/)를 사용할 수 있습니다.

8-1. 연봉 계산

SELECT ENAME, SAL, SAL * 12
FROM EMP;

SAL이 월급이라고 가정하면 SAL * 12는 연봉 계산 결과입니다. 이 결과는 실제 테이블 컬럼이 아니라 조회 중 계산된 임시 결과입니다.

8-2. 급여 10% 인상 계산

SELECT ENAME, SAL, SAL * 1.1
FROM EMP;

현재 급여에 10% 인상률을 적용한 값을 계산해서 보여줍니다.

8-3. 급여에 100 더하기

SELECT ENAME, SAL, SAL + 100
FROM EMP;

기존 급여에 100을 더한 값을 조회 결과로 출력합니다.

실제 컬럼과 계산 컬럼 구분

SAL
→ EMP 테이블에 실제 저장된 컬럼

SAL * 12
→ SELECT 실행 중 계산한 결과

SAL * 12 AS "연봉"
→ 계산 결과에 보기 좋은 이름을 붙인 출력 컬럼
Alias

9. 열 별칭

열 별칭은 조회 결과의 컬럼명을 보기 좋게 바꾸는 기능입니다. 특히 계산식이나 한글 컬럼명을 화면에 보여줄 때 유용합니다.

9-1. AS를 사용한 별칭

SELECT ENAME AS NAME,
       SAL AS SALARY
FROM EMP;

ENAME 컬럼은 NAME으로, SAL 컬럼은 SALARY로 출력됩니다.

9-2. 한글 별칭

SELECT ENAME AS "사원명",
       SAL AS "급여"
FROM EMP;

한글 별칭이나 공백이 있는 별칭은 큰따옴표로 감싸는 것이 안전합니다.

9-3. 산술 표현식에 별칭 붙이기

SELECT ENAME AS "사원명",
       SAL AS "월급",
       SAL * 12 AS "연봉"
FROM EMP;

SAL * 12 같은 계산 결과에 "연봉"이라는 별칭을 붙여 결과 화면을 읽기 쉽게 만듭니다.

DISTINCT

10. 중복 행 제거

DISTINCT는 조회 결과에서 중복된 행을 제거할 때 사용합니다. 직무 종류, 부서번호 종류처럼 중복을 제거한 목록이 필요할 때 유용합니다.

10-1. 직무 전체 조회

SELECT JOB
FROM EMP;

같은 직무가 여러 사원에게 반복되어 출력될 수 있습니다.

10-2. 직무 종류만 조회

SELECT DISTINCT JOB
FROM EMP;

중복된 직무를 제거하고 직무 종류만 출력합니다.

10-3. 여러 컬럼 조합으로 중복 제거

SELECT DISTINCT JOB, DEPTNO
FROM EMP;

JOB만 중복 제거하는 것이 아니라 JOB과 DEPTNO의 조합을 기준으로 중복이 제거됩니다.

DISTINCT 위치 주의

DISTINCT는 SELECT 바로 뒤에 작성합니다. 그리고 여러 컬럼을 함께 작성하면 각각의 컬럼이 아니라 컬럼 조합 전체를 기준으로 중복이 제거됩니다.

WHERE

11. WHERE 맛보기 - 특정 행 검색

WHERE는 원하는 행만 조회하기 위한 조건절입니다.
이번 단원에서는 SELECT 기본 흐름을 이해하기 위한 맛보기로만 다루고, 다음 SQL 연산자 단원에서 BETWEEN, IN, LIKE, IS NULL 등을 자세히 공부합니다.

11-1. 부서번호가 10인 사원 조회

SELECT EMPNO, ENAME, DEPTNO
FROM EMP
WHERE DEPTNO = 10;

숫자 조건은 작은따옴표 없이 작성하는 것이 기본입니다.

11-2. 급여가 3000 이상인 사원 조회

SELECT ENAME, SAL
FROM EMP
WHERE SAL >= 3000;

비교 연산자를 사용해 조건에 맞는 행만 조회합니다.

11-3. 직무가 MANAGER인 사원 조회

SELECT ENAME, JOB
FROM EMP
WHERE JOB = 'MANAGER';

문자 조건은 작은따옴표로 감싸야 합니다.

Practice

12. 실습 문제 연결

실습 문제 교안의 초반 문제들은 SELECT 기본 조회와 직접 연결됩니다. 문제를 읽을 때 먼저 테이블명, 출력 컬럼, 조건을 분리해서 보는 습관을 들이면 좋습니다.

문제 1. 출력 컬럼 읽기

EMP 테이블에서 직책이 MANAGER이면서
연봉이 2500 이상 3000 이하인 직원의 사원번호, 이름, 직책, 연봉을 출력합니다. 

SELECT EMPNO, ENAME, JOB, SAL
FROM EMP
WHERE JOB = 'MANAGER'
  AND SAL BETWEEN 2500 AND 3000;

이번 단원에서 먼저 읽어야 할 부분은
SELECT EMPNO, ENAME, JOB, SAL FROM EMP입니다.

문제 2. 컬럼 순서 맞추기

부서 코드가 30인 직원 중 연봉이 1500 이상인 직원의
이름, 부서 코드, 연봉, 직무를 출력합니다. 

SELECT ENAME, DEPTNO, SAL, JOB
FROM EMP
WHERE DEPTNO = 30
  AND SAL >= 1500;

문제에서 요구한 출력 순서가 이름, 부서 코드, 연봉, 직무이므로 SELECT 컬럼 순서도 맞춰줍니다.

문제 3. 전체 데이터 조회

문제에서 모든 데이터 또는 모든 컬럼을 출력하라고 하면
SELECT *를 사용할 수 있습니다. 

SELECT *
FROM EMP;

다만 실무에서는 필요한 컬럼만 명시하는 것이 더 좋은 습관입니다.

문제 읽는 순서

1. 어떤 테이블에서?
   → EMP

2. 어떤 컬럼을 출력?
   → EMPNO, ENAME, JOB, SAL

3. 어떤 조건이 있는가?
   → JOB = 'MANAGER'
   → SAL BETWEEN 2500 AND 3000
Project

13. Java / Spring / Oracle 프로젝트 연결

SELECT 기본 조회는 Java와 Spring 프로젝트에서 가장 많이 사용되는 SQL 흐름입니다. 로그인, 상세 조회, 목록 조회, 통계, 추천 시스템 모두 SELECT를 기반으로 합니다.

로그인 기능

아이디로 회원 정보를 조회한 뒤 비밀번호 해시를 비교합니다.

SELECT USER_ID, LOGIN_ID, PASSWORD_HASH, USER_TYPE, ACTIVE
FROM USERS
WHERE LOGIN_ID = ?;

회원 상세 조회

마이페이지에서 사용자 정보를 보여주기 위해 필요한 컬럼만 조회합니다.

SELECT USER_ID,
       LOGIN_ID,
       EMAIL,
       USER_TYPE,
       REGISTERED_AT
FROM USERS
WHERE USER_ID = ?;

게시글 목록 조회

게시판 목록 화면에 필요한 컬럼만 조회합니다.

SELECT BOARD_ID,
       TITLE,
       WRITER_ID,
       CREATED_AT
FROM BOARD
ORDER BY BOARD_ID DESC;

게시글 상세 조회

상세 페이지에서는 목록보다 더 많은 컬럼, 특히 본문 CONTENT가 필요합니다.

SELECT BOARD_ID,
       TITLE,
       CONTENT,
       WRITER_ID,
       CREATED_AT,
       UPDATED_AT
FROM BOARD
WHERE BOARD_ID = ?;

매물 카드 조회

매물 목록 카드에 필요한 기본 정보를 조회합니다.

SELECT PROPERTY_ID,
       TITLE,
       DEAL_AMOUNT,
       AREA_SQM,
       PROPERTY_TYPE,
       DEAL_TYPE
FROM PROPERTY;

추천 시스템 데이터 조회

사용자 행동 로그를 조회해
Python 분석이나 추천 모델의 입력 데이터로 사용할 수 있습니다.

SELECT USER_ID,
       PROPERTY_ID,
       STAY_SECONDS,
       VIEWED_AT
FROM PROPERTY_VIEW_LOG
WHERE USER_ID = ?;

Mistake

14. 자주 하는 실수

  • 실수 1. SELECT와 FROM 순서를 바꾼다. SELECT ENAME FROM EMP;처럼 SELECT 다음 FROM 순서로 작성해야 합니다.
  • 실수 2. 컬럼 사이의 쉼표를 빼먹는다. 여러 컬럼을 조회할 때는 SELECT EMPNO, ENAME, SAL처럼 쉼표로 구분해야 합니다.
  • 실수 3. 존재하지 않는 컬럼명을 사용한다. EMP 테이블에는 NAME이 아니라 ENAME 컬럼이 있습니다. 조회 전 DESC EMP;로 구조를 확인하세요.
  • 실수 4. 문자 조건에 작은따옴표를 빼먹는다. WHERE JOB = 'MANAGER'처럼 문자값은 작은따옴표로 감싸야 합니다.
  • 실수 5. 별칭에 공백이 있는데 큰따옴표를 쓰지 않는다. AS "사원 이름"처럼 공백이 있는 별칭은 큰따옴표를 사용합니다.
  • 실수 6. DISTINCT가 한 컬럼에만 적용된다고 착각한다. SELECT DISTINCT JOB, DEPTNO는 JOB과 DEPTNO 조합 기준으로 중복을 제거합니다.
  • 실수 7. SELECT *를 실무에서도 계속 사용한다. 학습용 확인에는 편하지만 실무에서는 필요한 컬럼만 명시하는 것이 좋습니다.
  • 실수 8. 계산 결과가 실제 테이블에 저장된다고 생각한다. SAL * 12 AS "연봉"은 조회 결과일 뿐 실제 컬럼이 생성되는 것은 아닙니다.
Interview

15. 면접 포인트

Q1. SELECT 문은 무엇인가요?

SELECT 문은 테이블에 저장된 데이터를 조회할 때 사용하는 SQL입니다. DML에 속하며, 데이터를 변경하지 않고 필요한 컬럼과 행을 결과로 가져옵니다.
기본 형식은 SELECT 컬럼명 FROM 테이블명입니다.

Q2. SELECT *와 특정 컬럼 조회의 차이는 무엇인가요?

SELECT *는 테이블의 모든 컬럼을 조회하는 방식이고, 특정 컬럼 조회는 필요한 컬럼만 명시해서 가져오는 방식입니다.
실무에서는 성능, 보안, API 응답 구조의 명확성을 위해 필요한 컬럼만 조회하는 것이 좋습니다.

Q3. SELECT 문의 실행 순서는 어떻게 되나요?

기본 실행 순서는 FROM, WHERE, GROUP BY, HAVING, SELECT, ORDER BY입니다.
먼저 테이블을 결정하고, 조건으로 행을 걸러낸 뒤, 필요한 컬럼을 출력하고, 마지막으로 정렬합니다.

Q4. 컬럼 별칭은 왜 사용하나요?

컬럼 별칭은 조회 결과의 컬럼명을 사용자가 이해하기 쉽게 바꾸기 위해 사용합니다. 특히 계산식이나 한글 컬럼명, 보고서 출력에서 유용합니다.

Q5. DISTINCT는 어떤 역할을 하나요?

DISTINCT는 조회 결과에서 중복된 행을 제거할 때 사용합니다. 여러 컬럼을 함께 작성하면 각 컬럼의 조합을 기준으로 중복이 제거됩니다.

Q6. WHERE와 SELECT의 역할 차이는 무엇인가요?

WHERE는 어떤 행을 조회할지 결정하는 조건절이고, SELECT는 그 행들 중 어떤 컬럼을 출력할지 결정하는 절입니다. 쉽게 말해 WHERE는 행 필터링, SELECT는 컬럼 선택입니다.

Q7. 산술 표현식은 실제 컬럼을 생성하나요?

아닙니다. SAL * 12 AS "연봉" 같은 산술 표현식은 조회 시점에 계산된 결과를 보여주는 것이고, 실제 테이블에 새로운 컬럼을 생성하지 않습니다.

Summary

16. 최종 정리

SELECT
→ 테이블에서 데이터를 조회하는 DML 명령

SELECT *
→ 모든 컬럼 조회

SELECT 컬럼1, 컬럼2
→ 원하는 컬럼만 조회

산술 표현식
→ 조회 중 계산 결과 출력

AS 별칭
→ 결과 컬럼명을 보기 좋게 변경

DISTINCT
→ 중복 결과 제거

WHERE
→ 원하는 행만 검색

읽는 순서

1. FROM을 보고 어느 테이블인지 확인한다.
2. SELECT를 보고 어떤 컬럼을 출력하는지 확인한다.
3. WHERE가 있으면 어떤 행을 거르는지 확인한다.
4. 별칭이 있으면 결과 화면에서 어떤 이름으로 보이는지 확인한다.
5. DISTINCT가 있으면 중복 제거 기준이 무엇인지 확인한다.

학습 메모

오늘의 핵심

SELECT는 테이블에서 데이터를 조회하는 명령입니다. 가장 먼저 FROM으로 테이블을 확인하고, SELECT로 출력할 컬럼을 확인하는 습관을 들이세요.

기본 문법

SELECT 컬럼명, 컬럼명
FROM 테이블명;

실습 순서

  1. SHOW USER로 현재 계정 확인
  2. user_tables로 테이블 목록 확인
  3. DESC EMP로 테이블 구조 확인
  4. SELECT *로 전체 조회
  5. 필요한 컬럼만 선택해서 조회
  6. 산술 표현식과 별칭 적용
  7. DISTINCT와 WHERE 맛보기 실행

외우기 좋은 구분

  • SELECT는 컬럼 선택
  • FROM은 테이블 선택
  • WHERE는 행 필터링
  • AS는 출력 이름 변경
  • DISTINCT는 중복 제거

주의할 점

SELECT *는 학습할 때 편하지만 실무에서는 필요한 컬럼만 명시하는 습관을 들이는 것이 좋습니다.
특히 사용자 비밀번호, 개인정보, 대용량 본문 컬럼이 포함된 테이블에서는 더욱 주의해야 합니다.

프로젝트 연결

로그인, 회원 상세 조회, 게시판 목록, 게시글 상세, 매물 카드, 추천 로그 조회는 모두 SELECT 기본 조회에서 출발합니다.
