주요 내장 함수 Built-in Function

Oracle 내장 함수는 데이터베이스에 저장된 원본 값을 조회하면서
검색하기 좋게 바꾸고, 화면에 보여주기 좋게 가공하고, 계산·분기·순위 결과를 만드는 도구입니다.
이번 단원에서는 NVL, 문자열 함수, TO_CHAR, TO_DATE, 날짜 함수, 수학 함수,
DECODE, CASE, 분석 함수까지 Java·Spring·Oracle 프로젝트 흐름과 연결해서 학습합니다.

학습 목표 원본 데이터를 함수로 가공해
조회 결과와 검색 조건을 설계한다.
실습 방향 DUAL 테스트 → EMP/EMPLOYEES 적용 →
마스킹·날짜 포맷·순위 쿼리 작성
프로젝트 연결 검색 API, 목록 화면, 개인정보 마스킹,
페이징, 리포트 SQL과 연결한다.
Overview

1. 내장 함수는 왜 배우는가?

테이블에 저장된 값은 보통 원본 데이터입니다. 하지만 실제 화면, 검색 API, 엑셀 다운로드, 관리자 리포트에서는 원본 그대로보다 가공된 값이 필요한 경우가 많습니다.

DB 원본 값 
LAST_NAME    PHONE_NUMBER     HIRE_DATE     SALARY
Robert       515.123.4567     2013-12-12    10000
→
화면 출력 값 
Robert | 515XXXXXXXX | 2013년 12월 12일 | 고액연봉 | $10,000
검색 보정 UPPER, LOWER, INSTR로 대소문자와 포함 검색 처리
출력 가공 TO_CHAR, LPAD, RPAD로 날짜·금액·문자열 표시
분석 처리 CASE, RANK, ROW_NUMBER로 등급·순위·페이징 구현

한 줄 정리

내장 함수는 저장된 원본 데이터를 바꾸는 명령이 아니라, SELECT 결과나 WHERE 조건 안에서 값을 임시로 변환해 사용하는 도구입니다.

2. 단원 개념 카드

2-1. 함수는 값을 넣으면 결과를 돌려주는 변환 상자다

함수는 괄호 안의 값을 입력으로 받아 새로운 결과를 반환합니다. 입력값은 문자 상수, 숫자 상수, 날짜, 컬럼명, 계산식 모두 가능합니다.

결과 'oracle'

2-2. 함수의 주요 사용 위치

SELECT 절 Output

조회 결과를 화면에 보여주기 좋은 형태로 바꿉니다.

SELECT TO_CHAR(HIRE_DATE, 'YYYY-MM-DD')
FROM EMPLOYEES;
WHERE 절 Search

검색 조건을 만들기 위해 컬럼 값을 변환합니다.

SELECT *
FROM EMPLOYEES
WHERE INSTR(UPPER(LAST_NAME), 'AR') > 0;
ORDER BY 절 Sort

가공된 값이나 계산 결과를 기준으로 정렬합니다.

SELECT ENAME, SAL
FROM EMP
ORDER BY LENGTH(ENAME), SAL DESC;
분석 절 OVER Window

행을 유지하면서 그룹 평균, 순위, 이전글·다음글을 붙입니다.

SELECT ENAME, SAL,
       RANK() OVER (ORDER BY SAL DESC) AS SAL_RANK
FROM EMP;

3-2. SELECT 실행 순서 그림 카드

함수와 별칭을 어디에서 사용할 수 있는지 이해하려면 SELECT문의 실행 순서를 알아야 합니다.

1 FROM

어느 테이블에서 읽을지 결정

2 WHERE

행을 먼저 필터링

3 GROUP BY

그룹 생성

4 HAVING

그룹 조건 필터링

5 SELECT

함수 계산, 별칭 생성

6 ORDER BY

최종 결과 정렬

왜 중요한가?

SELECT 별칭은 SELECT 단계에서 만들어집니다. WHERE는 SELECT보다 먼저 실행되므로 WHERE에서 SELECT 별칭을 바로 사용할 수 없습니다.
분석 함수 결과도 WHERE에서 바로 쓰지 말고 서브쿼리나 WITH절로 감싸야 합니다.

NULL

6. SQL 예제 카드 - NULL 처리 함수

NULL은 0이 아니라 값이 없다는 의미입니다. NULL이 포함된 산술 계산 결과는 NULL이 될 수 있으므로 급여, 포인트, 할인 금액 계산에서는 반드시 주의해야 합니다.

6-1. NULL 계산 확인

SELECT 1000 + NULL AS RESULT
FROM DUAL;
결과 확인 RESULT는 1000이 아니라 NULL입니다.

6-2. NVL로 보너스 NULL 처리

SELECT ENAME,
       SAL,
       COMM,
       SAL + COMM AS TOTAL_ORIGINAL,
       SAL + NVL(COMM, 0) AS TOTAL_NVL
FROM EMP
ORDER BY EMPNO;

COMM이 NULL인 직원도 TOTAL_NVL은 정상적으로 계산됩니다.

6-3. NVL2로 보너스 상태 표시

SELECT ENAME,
       COMM,
       NVL2(COMM, '보너스 있음', '보너스 없음') AS COMM_STATE
FROM EMP
ORDER BY EMPNO;

COMM이 0인 경우는 NULL이 아니므로 “보너스 있음”으로 처리됩니다.

6-4. 주문 금액 계산 예시

SELECT ORDER_ID,
       PRODUCT_PRICE,
       SHIPPING_FEE,
       COUPON_DISCOUNT,
       PRODUCT_PRICE + SHIPPING_FEE - NVL(COUPON_DISCOUNT, 0) AS PAY_AMOUNT
FROM ORDERS;

쿠폰 할인 금액이 NULL이어도 결제 금액이 NULL이 되지 않도록 처리합니다.

7. SQL 예제 카드 - 문자열 함수

대소문자 LOWER / UPPER / INITCAP

검색어와 저장값의 대소문자를 통일합니다.

길이 LENGTH / LENGTHB

문자 수와 바이트 수를 확인합니다.

추출·검색 SUBSTR / INSTR

일부 문자열을 자르거나 포함 위치를 찾습니다.

마스킹 LPAD / RPAD

자리수를 맞추거나 오른쪽·왼쪽을 특정 문자로 채웁니다.

7-1. 대소문자 변환

SELECT LOWER('ORACLE') AS LOWER_RESULT,
       UPPER('oracle') AS UPPER_RESULT,
       INITCAP('oracle sql') AS INITCAP_RESULT
FROM DUAL;

7-2. 대소문자 구분 없는 검색

SELECT EMPLOYEE_ID,
       FIRST_NAME,
       LAST_NAME
FROM EMPLOYEES
WHERE UPPER(LAST_NAME) = UPPER('king');

7-3. 문자열 연결

SELECT FIRST_NAME || ' ' || LAST_NAME AS FULL_NAME
FROM EMPLOYEES;

SELECT ENAME || '의 직무는 ' || JOB || '입니다.' AS INFO
FROM EMP;

7-4. LENGTH와 LENGTHB

SELECT LENGTH('오라클') AS LEN_CHAR,
       LENGTHB('오라클') AS LEN_BYTE
FROM DUAL;
예상 결과 LEN_CHAR = 3, LEN_BYTE = 9

7-5. LPAD와 RPAD

SELECT LPAD('9', 2, '0') AS LPAD_CODE,
       RPAD('9', 2, '0') AS RPAD_CODE,
       LPAD('HELLO', 10, 'X') AS LEFT_FILL,
       RPAD('HELLO', 10, 'X') AS RIGHT_FILL
FROM DUAL;

LPAD('9', 2, '0')은 09, RPAD('9', 2, '0')은 90입니다.

7-6. INSTR과 SUBSTR

SELECT INSTR('ABCDEFG', 'DE') AS POSITION,
       SUBSTR('123456789', 3, 3) AS CUT_RESULT
FROM DUAL;
예상 결과 POSITION = 4, CUT_RESULT = 345

7-7. 이름 마스킹

SELECT ENAME,
       RPAD(SUBSTR(ENAME, 1, 1), LENGTH(ENAME), '*') AS MASKED_NAME
FROM EMP
ORDER BY ENAME;

7-8. REPLACE와 TRIM

SELECT REPLACE('HELLO WORLD', 'HELLO', 'JAVA') AS REPLACED,
       '[' || TRIM('  ORACLE  ') || ']' AS TRIMMED
FROM DUAL;

8. SQL 예제 카드 - 형 변환 함수

형 변환 함수는 데이터 타입을 바꾸는 함수입니다. 숫자와 날짜를 화면용 문자열로 만들거나, 사용자가 입력한 문자열을 숫자·날짜 타입으로 바꿀 때 사용합니다.

TO_CHAR

숫자 또는 날짜를 문자로 변환합니다.

TO_CHAR(SALARY, 'FM$999,999')
TO_CHAR(HIRE_DATE, 'YYYY-MM-DD')

TO_NUMBER

숫자처럼 생긴 문자를 숫자로 변환합니다.

TO_NUMBER('1,234', '9,999') + 10

TO_DATE

날짜처럼 생긴 문자를 DATE 타입으로 변환합니다.

TO_DATE('2024-12-25', 'YYYY-MM-DD')

포맷 주의

문자열 모양과 포맷 문자열이 일치해야 안전합니다.

YYYY = 연도
MM   = 월
DD   = 일
HH24 = 24시간
MI   = 분
SS   = 초

8-1. 숫자 포맷

SELECT TO_CHAR(123, '0999') AS ZERO_FILL,
       TO_CHAR(1234, 'FM999,999.00') AS NUMBER_TEXT,
       TO_CHAR(1234, 'FM$999,999.00') AS DOLLAR_TEXT
FROM DUAL;

8-2. 급여 출력 포맷

SELECT ENAME,
       SAL,
       TO_CHAR(SAL, 'FM999,999') AS SAL_TEXT,
       TO_CHAR(SAL * 12, 'FM$999,999') AS YEAR_SAL_TEXT
FROM EMP
ORDER BY SAL DESC;

정렬은 문자 포맷 결과가 아니라 원본 숫자 SAL로 하는 것이 안전합니다.

8-3. 날짜 포맷

SELECT SYSDATE AS NOW_DATE,
       TO_CHAR(SYSDATE, 'YYYY-MM-DD HH24:MI:SS') AS NOW_TEXT,
       TO_CHAR(SYSDATE, 'YYYY"년" MM"월" DD"일"') AS NOW_KOR
FROM DUAL;

8-4. 문자 → 숫자

SELECT TO_NUMBER('123') + TO_NUMBER('456') AS RESULT1,
       TO_NUMBER('1,234', '9,999') + 10 AS RESULT2
FROM DUAL;

8-5. 문자 → 날짜

SELECT TO_DATE('2024-12-25', 'YYYY-MM-DD') AS CHRISTMAS,
       TO_DATE('2024년12월25일', 'YYYY"년"MM"월"DD"일"') AS CHRISTMAS_KOR
FROM DUAL;

8-6. 날짜 검색에서 피해야 할 방식

-- 인덱스 사용에 불리할 수 있음
SELECT *
FROM ORDERS
WHERE TO_CHAR(ORDER_DATE, 'YYYY-MM-DD') = '2024-12-25';

날짜 컬럼은 가능하면 날짜 타입 그대로 범위 조건으로 검색합니다.

8-7. 날짜 검색 권장 방식

SELECT *
FROM ORDERS
WHERE ORDER_DATE >= DATE '2024-12-25'
  AND ORDER_DATE <  DATE '2024-12-26';

8-8. Java 날짜 파라미터와 연결

SELECT *
FROM ORDERS
WHERE ORDER_DATE >= TO_DATE(:startDate, 'YYYY-MM-DD')
  AND ORDER_DATE <  TO_DATE(:endDate, 'YYYY-MM-DD') + 1;
Date & Math

9. SQL 예제 카드 - 날짜 함수와 수학 함수

9-1. 현재 날짜와 하루 계산

SELECT SYSDATE AS TODAY,
       SYSDATE + 1 AS TOMORROW,
       SYSDATE - 1 AS YESTERDAY
FROM DUAL;

Oracle에서 날짜에 숫자 1을 더하면 하루가 더해집니다.

9-2. 두 날짜 사이의 일수

SELECT TO_DATE('2024-12-25', 'YYYY-MM-DD') - SYSDATE AS DAYS_LEFT
FROM DUAL;

날짜끼리 빼면 두 날짜 사이의 일수가 나옵니다.

9-3. 근속 개월 수

SELECT ENAME,
       HIREDATE,
       FLOOR(MONTHS_BETWEEN(SYSDATE, HIREDATE)) AS WORK_MONTHS
FROM EMP
ORDER BY WORK_MONTHS DESC;

9-4. 구독 만료일 계산

SELECT SYSDATE AS START_DATE,
       ADD_MONTHS(SYSDATE, 1) AS EXPIRE_DATE
FROM DUAL;

9-5. 수학 함수 한 번에 확인

SELECT ROUND(123.456, 2) AS ROUND_RESULT,
       FLOOR(123.956) AS FLOOR_RESULT,
       CEIL(123.056) AS CEIL_RESULT,
       MOD(5, 2) AS MOD_RESULT,
       POWER(2, 3) AS POWER_RESULT,
       SQRT(10) AS SQRT_RESULT
FROM DUAL;

9-6. 총 페이지 수 계산

SELECT CEIL(101 / 10) AS TOTAL_PAGE
FROM DUAL;

게시글 101개를 10개씩 보여주면 총 11페이지가 필요합니다.

DECODE / CASE

10. SQL 예제 카드 - 조건 분기 함수

조건 분기 함수는 DB에 저장된 코드값이나 숫자 범위를 사람이 읽기 쉬운 라벨로 바꿉니다. Java의 switch, if/else if/else와 비슷하게 이해하면 됩니다.

DECODE

값 비교 중심, Oracle 전용 함수

DECODE(JOB,
       'MANAGER', '매니저',
       'SALESMAN', '영업맨',
       '기타')
↔

CASE

값 비교와 범위 조건 모두 가능

CASE
    WHEN SAL >= 3000 THEN '고액'
    WHEN SAL >= 2000 THEN '중간'
    ELSE '보통'
END

10-1. DECODE로 직무 표시

SELECT ENAME,
       JOB,
       DECODE(JOB,
              'MANAGER', '매니저',
              'SALESMAN', '영업맨',
              '기타') AS JOB_KO
FROM EMP;

10-2. CASE로 급여 등급 표시

SELECT ENAME,
       SAL,
       CASE
           WHEN SAL >= 3000 THEN '고액 연봉자'
           WHEN SAL >= 2000 THEN '중간 연봉자'
           ELSE '하위 연봉자'
       END AS SAL_GRADE
FROM EMP
ORDER BY SAL DESC;

10-3. 값 비교 CASE

SELECT ENAME,
       JOB,
       CASE JOB
           WHEN 'MANAGER' THEN '매니저'
           WHEN 'SALESMAN' THEN '영업맨'
           ELSE '기타'
       END AS JOB_KO
FROM EMP;

10-4. CASE 조건 순서 주의

-- 잘못된 순서
CASE
    WHEN SALARY >= 5000 THEN '중간'
    WHEN SALARY >= 10000 THEN '고액연봉'
    ELSE '보통'
END

10000 이상도 먼저 5000 이상 조건에 걸려 “중간”이 됩니다.

Analytic

11. SQL 예제 카드 - 분석 함수와 Top-N 연결

분석 함수는 GROUP BY처럼 그룹 단위 계산을 하면서도 원래 행을 줄이지 않습니다. 그래서 목록 화면에 부서 평균, 급여 순위, 페이지 번호를 함께 붙일 때 유용합니다.

GROUP BY

SELECT DEPTNO,
       AVG(SAL) AS AVG_SAL
FROM EMP
GROUP BY DEPTNO;

부서별 한 줄만 나오므로 행 수가 줄어듭니다.

분석 함수

SELECT ENAME,
       DEPTNO,
       SAL,
       AVG(SAL) OVER (PARTITION BY DEPTNO) AS DEPT_AVG_SAL
FROM EMP;

직원 행은 유지되고 각 행마다 부서 평균이 붙습니다.

11-1. 전체 급여 순위

SELECT ENAME,
       SAL,
       ROW_NUMBER() OVER (ORDER BY SAL DESC) AS ROW_NUM,
       RANK() OVER (ORDER BY SAL DESC) AS RANK_NUM,
       DENSE_RANK() OVER (ORDER BY SAL DESC) AS DENSE_RANK_NUM
FROM EMP
ORDER BY SAL DESC;

11-2. 부서별 급여 순위

SELECT ENAME,
       DEPTNO,
       SAL,
       RANK() OVER (PARTITION BY DEPTNO ORDER BY SAL DESC) AS DEPT_SAL_RANK
FROM EMP
ORDER BY DEPTNO, DEPT_SAL_RANK;

11-3. ROW_NUMBER로 페이징

WITH ranked_emp AS (
    SELECT E.*,
           ROW_NUMBER() OVER (ORDER BY SAL DESC, EMPNO) AS RN
    FROM EMP E
)
SELECT EMPNO,
       ENAME,
       SAL,
       RN
FROM ranked_emp
WHERE RN BETWEEN 6 AND 10
ORDER BY RN;

11-4. LAG와 LEAD로 이전글·다음글

SELECT *
FROM (
    SELECT B.*,
           LEAD(ID) OVER (ORDER BY ID DESC) AS PREV_ID,
           LAG(ID) OVER (ORDER BY ID DESC) AS NEXT_ID
    FROM BBS B
)
WHERE ID = :boardId;

순위 함수 선택 기준

페이징 처리            → ROW_NUMBER()
동점자를 같은 순위로    → RANK()
동점 후 순위 건너뜀 없음 → DENSE_RANK()

문제 35. LAST_NAME에 ar 포함, 전화번호 마스킹

입사일은 YYYY-MM-DD 형식으로 출력합니다. 

SELECT LAST_NAME,

WHERE INSTR(UPPER(LAST_NAME), 'AR') > 0;

  • 전화번호 앞 3자리만 보이는가?
  • 입사일이 YYYY-MM-DD 형식인가?

문제 36. LAST_NAME이 t로 끝나는 직원 조회

LIKE 없이 함수로 LAST_NAME의 마지막 글자를 확인합니다.
EMAIL 마스킹, SALARY 등급, 금액 포맷, 한글 날짜 포맷을 함께 적용합니다. 

SELECT LAST_NAME,
       RPAD(SUBSTR(EMAIL, 1, 1), LENGTH(EMAIL), '*') AS EMAIL,
       CASE
           WHEN SALARY >= 10000 THEN '고액연봉'
           WHEN SALARY >= 5000 THEN '중간'
           ELSE '보통'

       TO_CHAR(SALARY, 'FM$999,999') AS SALARY,
       TO_CHAR(HIRE_DATE, 'YYYY"년" MM"월" DD"일"') AS HIRE_DATE
FROM EMPLOYEES
WHERE SUBSTR(LOWER(LAST_NAME), -1) = 't'
ORDER BY LAST_NAME;

  • EMAIL은 첫 글자만 남겼는가?
  • CASE 조건 순서가 큰 금액부터인가?
  • SALARY와 HIRE_DATE가 화면용 포맷인가?

ORA-01861: literal does not match format string

SELECT TO_DATE('2024-12-25', 'YYYY/MM/DD')
FROM DUAL;

문자열은 하이픈인데 포맷은 슬래시라서 맞지 않습니다.

SELECT TO_DATE('2024-12-25', 'YYYY-MM-DD')
FROM DUAL;

ORA-01722: invalid number

SELECT TO_NUMBER('ABC') + 10
FROM DUAL;

숫자로 바꿀 수 없는 문자를 TO_NUMBER로 변환하려고 할 때 발생합니다.

ORA-00904: invalid identifier

SELECT SAL * 12 AS YEAR_SAL
FROM EMP
WHERE YEAR_SAL >= 30000;

WHERE 단계에서는 SELECT 별칭 YEAR_SAL이 아직 만들어지지 않았습니다.

SELECT ENAME,
       SAL,

FROM EMP
WHERE SAL_RANK <= 3;

분석 함수 결과를 조건으로 쓰려면 서브쿼리나 WITH절로 감쌉니다.

Mistake

14. 자주 하는 실수

  • 실수 1. NULL과 0을 같은 값으로 생각함 NULL은 값이 없음이고, 0은 값이 있는 숫자입니다. 계산에는 NVL 처리가 필요할 수 있습니다.
  • 실수 2. LPAD와 RPAD 방향을 반대로 기억함 LPAD는 왼쪽 채우기, RPAD는 오른쪽 채우기입니다. LPAD('9', 2, '0')은 09입니다.
  • 실수 3. Oracle SUBSTR도 Java처럼 0부터 시작한다고 생각함 Oracle SUBSTR의 위치는 1부터 시작합니다. Java substring 인덱스는 0부터 시작합니다.
  • 실수 4. INSTR 결과를 true/false로 생각함 INSTR는 위치를 반환합니다. 찾지 못하면 0이므로 WHERE에서는 INSTR(...) > 0으로 씁니다.
  • 실수 5. 날짜 포맷에서 MM과 MI를 혼동함 MM은 월, MI는 분입니다. 시간 포맷은 HH24:MI:SS로 작성합니다.
  • 실수 6. 날짜 컬럼을 TO_CHAR로 변환해서 WHERE 검색함 출력은 TO_CHAR로 하되, 검색은 날짜 타입 범위 조건을 사용하는 것이 좋습니다.
  • 실수 7. CASE 조건 순서를 작은 범위부터 작성함 급여 등급처럼 범위가 겹치면 큰 조건 또는 더 구체적인 조건부터 작성합니다.
  • 실수 8. TO_CHAR 결과를 숫자로 착각함 TO_CHAR 결과는 문자입니다. 계산과 숫자 정렬에는 원본 숫자 컬럼을 사용합니다.
  • 실수 9. ROW_NUMBER 정렬 기준을 하나만 둠 동점이 있는 페이징은 ORDER BY SAL DESC, EMPNO처럼 PK를 추가해 안정적인 순서를 만듭니다.
  • 실수 10. SQL에서 처리할 일과 Java에서 처리할 일을 구분하지 않음 검색·정렬·집계는 SQL에 적합하고, 다국어 표시나 복잡한 정책은 Java/Spring 계층이 더 적합할 수 있습니다.
Interview

15. 면접 질문 카드

Q1. Oracle 내장 함수는 왜 사용하나요?

조회 결과를 원하는 형태로 가공하거나, 검색 조건을 보정하거나, NULL을 처리하거나, 날짜·숫자 형식을 변환하거나, 순위와 분석 결과를 만들기 위해 사용합니다.

Q2. NVL과 NVL2의 차이는 무엇인가요?

NVL은 NULL일 때 대체값 하나를 반환합니다. NVL2는 NULL이 아닐 때의 값과 NULL일 때의 값을 각각 지정할 수 있습니다.

Q3. LIKE 없이 포함 검색을 하려면 어떻게 하나요?

INSTR(컬럼, 찾을문자열) > 0 조건을 사용할 수 있습니다. 대소문자를 무시하려면 UPPER 또는 LOWER를 함께 사용합니다.

Q4. LENGTH와 LENGTHB의 차이는 무엇인가요?

LENGTH는 문자 수를 반환하고, LENGTHB는 바이트 수를 반환합니다. 한글처럼 멀티바이트 문자는 두 결과가 다를 수 있습니다.

Q5. TO_CHAR, TO_DATE, TO_NUMBER의 차이는 무엇인가요?

TO_CHAR는 숫자나 날짜를 문자로 변환합니다. TO_DATE는 문자를 날짜로 변환하고, TO_NUMBER는 문자를 숫자로 변환합니다.

Q6. 날짜 검색에서 TO_CHAR(date_col)을 WHERE에 쓰면 왜 주의해야 하나요?

컬럼에 함수를 적용하면 인덱스 사용에 불리할 수 있습니다. 날짜 컬럼은 날짜 타입 그대로 시작일 이상, 다음날 미만 범위 조건으로 검색하는 것이 안전합니다.

Q7. DECODE와 CASE의 차이는 무엇인가요?

DECODE는 Oracle 전용 함수이며 값 비교 중심입니다. CASE는 범위 조건과 복합 조건을 표현할 수 있어 더 유연합니다.

Q8. GROUP BY와 분석 함수의 차이는 무엇인가요?

GROUP BY는 그룹당 한 행을 반환하여 행 수가 줄어듭니다. 분석 함수는 원래 행을 유지하면서 각 행에 그룹 기준 계산 결과를 붙입니다.

Q9. ROW_NUMBER, RANK, DENSE_RANK의 차이는 무엇인가요?

ROW_NUMBER는 중복 없이 순번을 부여합니다. RANK는 동점에 같은 순위를 주고 다음 순위를 건너뜁니다.
DENSE_RANK는 동점에 같은 순위를 주지만 다음 순위를 건너뛰지 않습니다.

Q10. 분석 함수 결과를 WHERE에서 바로 사용할 수 있나요?

바로 사용할 수 없습니다. 분석 함수는 SELECT 단계에서 계산되므로 서브쿼리나 WITH절로 감싼 뒤 바깥 WHERE에서 조건을 걸어야 합니다.

Q11. SQL에서 마스킹하는 것과 Java에서 마스킹하는 것의 차이는?

SQL에서 마스킹하면 조회 단계에서 원본 노출을 줄일 수 있습니다. Java에서 마스킹하면 DB 종속성이 줄고 화면 정책 변경에 대응하기 쉽습니다.
개인정보 보호가 중요하면 DB 조회 단계부터 마스킹하는 설계를 고려합니다.

Q12. Spring 프로젝트에서 Oracle 함수를 사용할 때 주의할 점은?

Oracle 전용 함수는 DB 종속성을 높입니다.
검색·집계·페이징처럼 DB에서 처리해야 하는 일에는 적합하지만, 다국어 라벨이나 화면 포맷은 Java 또는 Front-end 계층에서 처리하는 것이 더 유연할 수 있습니다.

오늘의 핵심

내장 함수는 DB 원본 데이터를 저장 단계에서 바꾸는 것이 아니라, 조회하는 순간 원하는 형태로 가공하는 도구입니다.

암기 문법

SELECT 함수(컬럼)
FROM 테이블
WHERE 함수(컬럼) 조건
ORDER BY 함수(컬럼);

문자열 마스킹 공식

RPAD(SUBSTR(컬럼, 1, 보여줄글자수), LENGTH(컬럼), '마스킹문자')

예: 이메일 첫 글자만 남기고 나머지를 *로 표시합니다.

날짜 포맷 공식

TO_CHAR(날짜컬럼, 'YYYY-MM-DD HH24:MI:SS')
TO_DATE('2024-12-25', 'YYYY-MM-DD')

MM은 월, MI는 분입니다.

CASE 공식

CASE
    WHEN 조건1 THEN 결과1
    WHEN 조건2 THEN 결과2
    ELSE 기본값
END

범위 조건은 큰 조건 또는 더 구체적인 조건부터 작성합니다.

분석 함수 공식

ROW_NUMBER() OVER (ORDER BY 정렬컬럼)
RANK() OVER (PARTITION BY 그룹컬럼 ORDER BY 정렬컬럼)
AVG(컬럼) OVER (PARTITION BY 그룹컬럼)

주의할 점

WHERE절에서 컬럼에 함수를 씌우면 인덱스 사용에 불리할 수 있습니다. 특히 날짜 검색은 TO_CHAR보다 날짜 타입 범위 조건을 우선 고려합니다.

프로젝트 연결

Spring Boot에서는 검색 조건, 페이징, 순위, 리포트 SQL은 DB 함수로 처리하기 좋고, 다국어 라벨과 화면 포맷은 Java 또는 Front-end에서 처리하는 것이 더 유연할 수 있습니다.
