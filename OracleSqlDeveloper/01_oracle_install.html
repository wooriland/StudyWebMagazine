Oracle SQL Developer · 01

Oracle 설치와 SQL Developer 접속

Oracle Database를 로컬 PC에 설치하고 SQL Developer로 접속하는 과정을 학습합니다.
이 단원에서는 Oracle Database와 SQL Developer의 역할 차이, SYSTEM 계정,
localhost, 1521 포트, XEPDB1 서비스 이름을 이해하고, 이후 모든 SQL 실습을 진행할 수 있는 기본 환경을 준비합니다.

1. 이 단원에서 배우는 것

이번 단원은 Oracle SQL 문법을 본격적으로 배우기 전에, 실습을 실행할 수 있는 환경을 준비하는 단계입니다.
Oracle Database를 설치하고 SQL Developer로 접속할 수 있어야 이후 SELECT, WHERE, JOIN, SUBQUERY, PL/SQL 같은 모든 실습을 진행할 수 있습니다.
중요한 점은 Oracle Database와 SQL Developer가 서로 다른 역할을 한다는 것입니다.
Oracle Database는 실제 데이터가 저장되고 SQL이 처리되는 서버이고, SQL Developer는 그 서버에 접속해서 SQL을 작성하고 실행 결과를 확인하는 도구입니다.

Oracle Database 데이터 저장과 SQL 처리 담당
SQL Developer Oracle에 접속해 SQL 작성과 결과 확인
SQL Database에 보내는 명령어

2. Oracle Database와 SQL Developer의 차이

Oracle을 처음 공부할 때 가장 먼저 구분해야 하는 것은 Oracle Database와 SQL Developer입니다. 둘은 함께 사용되지만 같은 프로그램이 아닙니다.

Oracle Database

실제 데이터가 저장되는 데이터베이스 서버입니다.
사용자, 테이블, 데이터, 권한, 트랜잭션, 뷰, 인덱스, PL/SQL
객체가 모두 Oracle Database 안에서 관리됩니다.

  • 데이터 저장
  • SQL 실행 처리
  • 계정과 권한 관리
  • 트랜잭션 관리
  • 테이블, 뷰, 인덱스 관리

SQL Developer

Oracle Database에 접속해서 SQL을 작성하고 실행하는 클라이언트 도구입니다.
SQL Developer는 데이터를 직접 저장하는 서버가 아니라,
Oracle 서버에 명령을 보내고 결과를 화면에 보여주는 역할을 합니다.

  • 접속 정보 입력
  • SQL Worksheet 제공
  • SQL 실행
  • 결과 그리드 표시
  • 오류 메시지 확인

비유로 이해하기

Oracle Database는 실제 창고이고, SQL Developer는 창고 관리 프로그램입니다.
SQL은 창고에 내리는 명령이고, 테이블은 창고 안의 정리된 선반, 데이터는 선반 안에 들어 있는 물건이라고 볼 수 있습니다.

3. 설치 전 확인 사항

Oracle은 단순한 실행 프로그램이 아니라 Windows 서비스, 포트, 환경 변수, 데이터베이스 파일과 연결되는 서버 프로그램입니다. 그래서 설치 전에 기본 환경을 확인해야 합니다.

  1. 컴퓨터명이 영문인지 확인 컴퓨터명이나 사용자명, 설치 경로에 한글 또는 특수문자가 포함되어 있으면 Oracle 설치나 서비스 실행 과정에서 문제가 생길 수 있습니다.
    학습 환경에서는 오류 원인을 줄이기 위해 영문 환경을 권장합니다.
  2. 이벤트 서비스가 실행 중인지 확인 Oracle 설치 과정에서는 Windows 서비스와 로그가 등록됩니다. 이벤트 관련 서비스가 정상적으로 동작하지 않으면 설치나 실행 과정에서 문제가 발생할 수 있습니다.
  3. 관리자 권한으로 설치 준비 Oracle은 서비스 등록, 포트 사용, 환경 변수 설정이 필요하므로 setup.exe를 실행할 때 관리자 권한으로 실행하는 것이 안전합니다.

4. Oracle 설치 흐름

Oracle 설치는 다운로드, 압축 해제, setup.exe 실행, SYSTEM 암호 설정, 설치 완료 후 접속 확인 순서로 진행됩니다.

01 Oracle 다운로드

Oracle 공식 사이트에서 Database 항목으로 이동한 뒤, 운영체제에 맞는 Oracle XE 버전을 다운로드합니다.

02 압축 해제

다운로드한 zip 파일을 압축 해제합니다. 버전에 따라 설치 파일 구성이 다를 수 있으므로 안내에 따라 준비합니다.

03 setup.exe 실행

가능하면 관리자 권한으로 setup.exe를 실행합니다. Oracle은 Windows 서비스와 포트를 사용하므로 권한 문제가 없어야 합니다.

04 SYSTEM 암호 설정

설치 중 관리자 계정인 SYSTEM의 암호를 설정합니다. 암호는 숫자가 아닌 문자로 시작하는 형태가 안전합니다.

05 설치 완료

설치가 완료되면 Oracle 서비스와 Listener가 실행될 준비가 됩니다. 이후 SQL Developer로 접속 테스트를 진행합니다.

5. SQL Developer 접속 정보 이해

Oracle 설치 후 SQL Developer에서 새 접속을 만들 때는 사용자 이름, 비밀번호, 호스트, 포트, 서비스 이름을 정확히 입력해야 합니다.

항목 입력 예시 의미
Name SYSTEM계정 SQL Developer에서 보이는 접속 이름입니다. 자유롭게 정할 수 있습니다.
사용자 이름 system 실제 Oracle 사용자 계정입니다.
비밀번호 설치 시 설정한 비밀번호 SYSTEM 계정의 암호입니다. 교안 예시와 다를 수 있습니다.
호스트 이름 localhost 내 PC에 설치된 Oracle에 접속한다는 뜻입니다.
포트 1521 Oracle Listener가 접속 요청을 받는 기본 포트입니다.
서비스 이름 XEPDB1 Oracle XE에서 주로 사용하는 기본 PDB 이름입니다.

주의할 점

SQL Developer 접속 화면에서 SID와 서비스 이름을 헷갈리면 접속에 실패할 수 있습니다. Oracle XE 학습 환경에서는 일반적으로 서비스 이름에 XEPDB1을 입력합니다.

6. 접속 흐름을 그림처럼 이해하기

SQL Developer에서 접속 버튼을 누르면 내부적으로 다음 흐름이 진행됩니다.

SQL Developer SQL 작성 도구
→
localhost:1521 내 PC의 Oracle Listener
→
XEPDB1 접속할 PDB
→
SYSTEM 관리자 계정

이 구조는 나중에 Java JDBC나 Spring Boot datasource 설정과 그대로 연결됩니다.
SQL Developer에서 접속이 성공했다는 것은 host, port, service name, username, password가 정상이라는 뜻이므로 애플리케이션 연결 테스트의 기준이 됩니다.

7. 설치와 접속 확인 SQL

접속에 성공했다면 SQL Worksheet에서 아래 SQL을 실행해봅니다. 이번 단원에서는 SQL 문법을 완벽히 분석하기보다, 설치와 접속이 정상인지 확인하는 것이 목적입니다.

현재 사용자 확인

SHOW USER;

현재 SQL Developer에서 어떤 Oracle 계정으로 접속했는지 확인합니다. SYSTEM 계정으로 접속했다면 USER is "SYSTEM"과 비슷한 결과가 나옵니다.

PDB 목록 확인

SELECT name
FROM v$pdbs;

현재 Oracle 안에 어떤 PDB가 있는지 확인합니다. Oracle XE 환경에서는 보통 XEPDB1이 보입니다.

서비스 목록 확인

SELECT name, pdb
FROM v$services
ORDER BY name;

SQL Developer의 서비스 이름과 실제 PDB 연결 상태를 확인합니다. 접속 오류가 발생할 때 서비스 이름을 점검하는 데 도움이 됩니다.

CDB 이름 확인

SELECT name
FROM v$database;

현재 접속한 Oracle의 CDB 이름을 확인합니다. Oracle XE 환경에서는 보통 XE 계열 이름을 확인할 수 있습니다.

현재 계정의 테이블 목록 확인

SELECT table_name
FROM user_tables;

현재 접속한 계정이 소유한 테이블 목록을 확인합니다. SYSTEM 계정에서는 학습용 EMP, DEPT 테이블이 바로 보이지 않을 수 있습니다.

접근 가능한 테이블 확인

SELECT owner, table_name
FROM all_tables
ORDER BY owner, table_name;

현재 사용자가 접근 권한을 가진 테이블을 확인합니다. 다른 스키마의 테이블이 포함될 수 있습니다.

8. Java / Spring 프로젝트와의 연결

SQL Developer에서 사용하는 접속 정보는 나중에 Java JDBC, Spring Boot datasource 설정과 거의 그대로 연결됩니다.
그래서 SQL Developer 접속 성공은 프로젝트 DB 연결을 검증하는 첫 단계가 됩니다.

SQL Developer 접속 정보

Host         : localhost
Port         : 1521
Service Name : XEPDB1
User         : system 또는 app_user
Password     : 계정 비밀번호

Spring Boot datasource 예시

spring.datasource.url=jdbc:oracle:thin:@localhost:1521/XEPDB1
spring.datasource.username=app_user
spring.datasource.password=app_password
spring.datasource.driver-class-name=oracle.jdbc.OracleDriver

실무나 프로젝트에서는 SYSTEM 계정을 애플리케이션 접속용으로 사용하지 않습니다.
SYSTEM은 관리자 작업용이고, Spring Boot 프로젝트에서는 별도의 일반 사용자 계정을 만든 뒤 필요한 권한만 부여해서 사용하는 것이 안전합니다.

9. 자주 하는 실수

  • SQL Developer만 설치하고 Oracle Database도 설치됐다고 생각함 SQL Developer는 접속 도구입니다. 실제 Oracle Database 서버가 설치되어 있어야 접속할 수 있습니다.
  • SYSTEM 비밀번호를 교안 예시 그대로 입력함 비밀번호는 설치할 때 직접 설정한 값입니다. 교안에 나온 값은 예시일 수 있습니다.
  • SID와 서비스 이름을 헷갈림 Oracle XE 학습 환경에서는 보통 SID가 아니라 서비스 이름에 XEPDB1을 입력합니다.
  • 포트를 잘못 입력함 기본 포트는 1521입니다. 설치 과정에서 변경하지 않았다면 1521을 사용합니다.
  • SCOTT / HR 계정으로 바로 접속하려고 함 설치 직후에는 SCOTT이나 HR 계정이 잠겨 있거나 준비되지 않았을 수 있습니다. 먼저 SYSTEM 계정으로 접속을 확인합니다.
  • 접속 후 현재 계정을 확인하지 않음 여러 접속을 만들어두면 계정을 헷갈리기 쉽습니다. SQL 실행 전 SHOW USER;로 현재 계정을 확인하는 습관이 필요합니다.

10. 면접 포인트

Q1. Oracle Database와 SQL Developer의 차이는 무엇인가요?

Oracle Database는 실제 데이터를 저장하고 SQL을 처리하는 서버입니다.
SQL Developer는 Oracle Database에 접속해서 SQL을 작성하고 실행 결과를 확인하는 클라이언트 도구입니다.

Q2. Oracle XE에서 XEPDB1은 무엇인가요?

XEPDB1은 Oracle XE에서 기본으로 제공되는 PDB입니다. SQL Developer나 JDBC URL에서 서비스 이름으로 XEPDB1을 지정하면 해당 PDB에 접속하게 됩니다.

Q3. Host, Port, Service Name은 각각 무엇인가요?

Host는 Oracle이 실행 중인 서버 주소이고, 로컬 설치 환경에서는 보통 localhost입니다. Port는 Oracle Listener가 접속 요청을 받는 번호이며 기본값은 1521입니다.
Service Name은 접속할 데이터베이스 서비스 또는 PDB 이름입니다.

Q4. SYSTEM 계정은 어떤 용도로 사용하나요?

SYSTEM은 관리자 계정입니다. 설치 확인, 사용자 생성, 권한 부여, 계정 잠금 해제, PDB 확인 같은 관리 작업에 사용합니다.
일반 애플리케이션 접속용으로 사용하는 것은 권장되지 않습니다.

Q5. SQL Developer 접속 성공이 Spring Boot 프로젝트에서 왜 중요한가요?

SQL Developer 접속이 성공했다는 것은 DB 서버 주소, 포트, 서비스 이름, 계정, 비밀번호가 정상이라는 뜻입니다.
Spring Boot도 같은 정보를 사용해 JDBC URL로 Oracle에 접속하기 때문에 SQL Developer 접속은 datasource 설정의 사전 검증 단계가 됩니다.

11. 이번 단원 핵심 정리

Oracle 설치는 실제 데이터베이스 서버를 준비하는 과정이고, SQL Developer 접속은 그 서버에 SQL을 실행할 수 있는 통로를 여는 과정입니다.
Oracle XE 실습 환경에서는 보통 localhost, 1521, XEPDB1, system 계정 구조로 접속합니다.
접속 후에는 SHOW USER, SELECT name FROM v$pdbs, SELECT name FROM v$database 같은 SQL로 현재 계정과 데이터베이스 상태를 확인합니다.
