Oracle SQL Developer - Chapter 12

INNER JOIN

INNER JOIN은 두 개 이상의 테이블에 나누어 저장된 데이터를 관계 컬럼을 기준으로 연결해 조회하는 문법입니다.
이번 단원에서는 SCOTT 계정의 EMP와 DEPT, HR 계정의 EMPLOYEES, JOBS, DEPARTMENTS, LOCATIONS를 활용해
PK/FK 관계, 표준 JOIN, Oracle식 JOIN, 테이블 별칭, ON과 WHERE의 차이를 정리합니다.

학습 목표 두 개 이상의 테이블을 공통 컬럼으로 연결해
하나의 결과처럼 조회한다.
실습 방향 EMP와 DEPT를 DEPTNO로 연결하고,
HR 예제로 3개 테이블 JOIN까지 실행한다.
프로젝트 연결 게시글+작성자, 댓글+회원, 매물+등록자,
문의+매물+회원 조회와 연결한다.
Overview

1. JOIN이란?

JOIN은 두 개 이상의 테이블에 나누어 저장된 데이터를 관계 조건을 기준으로 연결해서 조회하는 문법입니다.
데이터베이스에서는 중복을 줄이기 위해 사원 정보, 부서 정보, 사용자 정보, 게시글 정보처럼 성격이 다른 데이터를 여러 테이블로 나누어 저장합니다.
하지만 화면이나 보고서에서는 이 데이터들을 함께 보여줘야 할 때가 많습니다.

테이블 분리 EMP에는 사원 정보, DEPT에는 부서 정보를 저장합니다.
관계 컬럼 EMP.DEPTNO와 DEPT.DEPTNO처럼
공통 관계 컬럼을 찾습니다.
INNER JOIN 조건이 일치하는 행끼리 연결해
하나의 결과로 조회합니다.

한 줄 정리

INNER JOIN은 양쪽 테이블에서 연결 조건이 일치하는 행만 합쳐서 보여주는 조회 방식입니다. 한 테이블에 없는 정보를 다른 테이블에서 가져와 함께 출력할 때 사용합니다.

Why Join?

2. JOIN이 필요한 이유

모든 정보를 하나의 테이블에 저장하면 같은 정보가 반복됩니다.
예를 들어 사원 테이블에 부서명과 지역까지 모두 저장하면 20번 부서의 모든 사원 행마다 RESEARCH, DALLAS가 반복됩니다.
부서명이 바뀌면 여러 행을 동시에 수정해야 하므로 데이터 관리가 어려워집니다.

한 테이블에 모두 저장한 경우

ENAME   SAL   DEPTNO   DNAME       LOC
------  ----  ------   ----------  --------
SMITH   800   20       RESEARCH    DALLAS
JONES   2975  20       RESEARCH    DALLAS
SCOTT   3000  20       RESEARCH    DALLAS

부서명과 지역이 사원 행마다 반복됩니다. 중복이 많고 수정에도 불리합니다.

테이블을 나누어 저장한 경우

EMP
EMPNO, ENAME, SAL, DEPTNO

DEPT
DEPTNO, DNAME, LOC

사원 정보와 부서 정보를 나누어 저장하고, 필요할 때 DEPTNO로 연결합니다.

JOIN의 역할

EMP.DEPTNO
→ 사원이 속한 부서번호

DEPT.DEPTNO
→ 부서 테이블의 부서번호

EMP.DEPTNO = DEPT.DEPTNO
→ 같은 부서번호끼리 연결
Visual

3. 그림처럼 이해하기

INNER JOIN은 두 테이블의 공통 컬럼 값이 같은 행만 연결합니다.
EMP에는 사원명과 급여가 있고, DEPT에는 부서명과 지역이 있으므로 DEPTNO를 기준으로 두 테이블을 연결합니다.

EMP

ENAME SAL DEPTNO
SMITH 800 20
ALLEN 1600 30
KING 5000 10

DEPT

DEPTNO DNAME LOC
10 ACCOUNTING NEW YORK
20 RESEARCH DALLAS
30 SALES CHICAGO
40 OPERATIONS BOSTON
ON E.DEPTNO = D.DEPTNO 같은 부서번호를 가진 행끼리 연결
ENAME SAL DEPTNO DNAME LOC
SMITH 800 20 RESEARCH DALLAS
ALLEN 1600 30 SALES CHICAGO
KING 5000 10 ACCOUNTING NEW YORK

INNER JOIN에서 빠지는 데이터

DEPT의 40번 부서처럼 한쪽 테이블에는 있지만 다른 테이블과 연결되는 행이 없으면 INNER JOIN 결과에서는 제외됩니다.
이런 연결되지 않은 데이터까지 보고 싶을 때는 OUTER JOIN을 사용합니다.

PK / FK

4. PK와 FK로 이해하는 JOIN

JOIN은 보통 부모 테이블의 PK와 자식 테이블의 FK를 기준으로 연결합니다.
DEPT는 부서 기준 정보를 가진 부모 테이블이고, EMP는 각 사원이 어느 부서에 속하는지를 DEPTNO로 참조하는 자식 테이블입니다.

부모 테이블: DEPT

DEPT.DEPTNO
→ 부서 테이블의 대표 번호
→ Primary Key 역할

자식 테이블: EMP

EMP.DEPTNO
→ 사원이 속한 부서번호
→ DEPT.DEPTNO를 참조하는 Foreign Key 역할

관계 구조

DEPT.DEPTNO  ← 부모 PK
     ↑
     │
EMP.DEPTNO   ← 자식 FK

JOIN 조건:
ON E.DEPTNO = D.DEPTNO
Syntax

5. INNER JOIN 기본 문법

표준 INNER JOIN은 `FROM 테이블1 JOIN 테이블2 ON 연결조건` 형태로 작성합니다. `INNER`는 생략할 수 있으며, 조인 조건은 `ON`절에 작성합니다.

5-1. 기본 구조

SELECT 별칭1.컬럼명,
       별칭2.컬럼명
FROM 테이블1 별칭1
INNER JOIN 테이블2 별칭2
ON 별칭1.공통컬럼 = 별칭2.공통컬럼;

5-2. EMP와 DEPT 기본 JOIN

SELECT E.ENAME,
       E.SAL,
       E.DEPTNO,
       D.DNAME
FROM EMP E INNER JOIN DEPT D
ON E.DEPTNO = D.DEPTNO;

5-3. INNER 생략

SELECT E.ENAME,
       E.SAL,
       E.DEPTNO,
       D.DNAME
FROM EMP E JOIN DEPT D
ON E.DEPTNO = D.DEPTNO;

`JOIN`만 작성해도 일반적으로 INNER JOIN과 같은 의미입니다.

Alias

6. 테이블 별칭

JOIN에서는 테이블명이 길어지고 컬럼 출처가 중요해지므로 테이블 별칭을 자주 사용합니다. Oracle에서는 테이블 별칭에 `AS`를 사용하지 않고, `테이블명 별칭` 형태로 작성합니다.

올바른 테이블 별칭

FROM EMP E
JOIN DEPT D
ON E.DEPTNO = D.DEPTNO

EMP를 E로, DEPT를 D로 줄여 사용합니다.

Oracle에서 피해야 할 형태

FROM EMP AS E
JOIN DEPT AS D
ON E.DEPTNO = D.DEPTNO

Oracle에서는 테이블 별칭에 AS를 사용하지 않는다고 기억하는 것이 좋습니다.

별칭 사용 예

SELECT E.ENAME,
       E.SAL,
       D.DNAME
FROM EMP E
JOIN DEPT D
ON E.DEPTNO = D.DEPTNO;

별칭 혼용 주의

-- 좋지 않은 예
SELECT EMP.ENAME, D.DNAME
FROM EMP E
JOIN DEPT D
ON E.DEPTNO = D.DEPTNO;

`EMP E`라고 별칭을 붙였으면
이후에는 `E.ENAME`처럼 별칭을 일관되게 사용하는 것이 좋습니다.

Column Name

7. 같은 이름의 컬럼 구분하기

JOIN 대상 테이블에 같은 이름의 컬럼이 있으면 반드시 테이블명 또는 별칭을 붙여 구분해야 합니다. EMP와 DEPT에는 모두 DEPTNO가 있으므로 `E.DEPTNO`, `D.DEPTNO`처럼 작성해야 합니다.

모호한 컬럼명

SELECT DEPTNO, ENAME, DNAME
FROM EMP E
JOIN DEPT D
ON E.DEPTNO = D.DEPTNO;

DEPTNO가 EMP에도 있고 DEPT에도 있어 모호해질 수 있습니다.

명확한 컬럼명

SELECT E.DEPTNO,
       E.ENAME,
       D.DNAME
FROM EMP E
JOIN DEPT D
ON E.DEPTNO = D.DEPTNO;

어느 테이블의 컬럼인지 별칭으로 명확히 표시합니다.

Standard vs Oracle

8. 표준 INNER JOIN과 Oracle식 JOIN 비교

교안은 표준 SQL식 INNER JOIN과 Oracle식 JOIN을 함께 제시합니다. 두 방식 모두 같은 결과를 만들 수 있지만, 표준 JOIN은 조인 조건과 검색 조건이 분리되어 읽기 쉽습니다.

표준 INNER JOIN

SELECT E.ENAME,
       E.SAL,
       E.DEPTNO,
       D.DNAME
FROM EMP E INNER JOIN DEPT D
ON E.DEPTNO = D.DEPTNO
WHERE E.SAL >= 3000
ORDER BY E.ENAME DESC;

ON에는 조인 조건, WHERE에는 검색 조건을 작성합니다.

Oracle식 JOIN

SELECT E.ENAME,
       E.SAL,
       E.DEPTNO,
       D.DNAME
FROM EMP E, DEPT D
WHERE E.DEPTNO = D.DEPTNO
  AND E.SAL >= 3000
ORDER BY E.ENAME DESC;

WHERE절에 조인 조건과 검색 조건이 함께 들어갑니다.

학습 추천

처음 공부할 때는 표준 INNER JOIN을 기준으로 익히고, Oracle식 JOIN은 실습 문제와 기존 코드 해석을 위해 함께 알아두면 좋습니다.

ON vs WHERE

9. ON과 WHERE의 차이

INNER JOIN에서는 검색 조건을 ON에 함께 쓸 수도 있고 WHERE에 쓸 수도 있습니다.
하지만 학습 단계에서는 ON은 테이블 연결 조건, WHERE는 검색 조건으로 구분해서 작성하는 것이 이해하기 쉽습니다.

권장 학습 형태

SELECT E.ENAME,
       E.SAL,
       D.DNAME
FROM EMP E
JOIN DEPT D
ON E.DEPTNO = D.DEPTNO
WHERE E.SAL >= 3000;

ON은 연결, WHERE는 필터링 역할로 분리되어 있습니다.

ON에 조건 추가

SELECT E.ENAME,
       E.SAL,
       D.DNAME
FROM EMP E
JOIN DEPT D
ON E.DEPTNO = D.DEPTNO
AND E.SAL >= 3000;

INNER JOIN에서는 비슷한 결과를 낼 수 있지만,
처음에는 역할 분리 방식이 더 읽기 좋습니다.

잘못된 순서

SELECT E.ENAME, E.SAL, D.DNAME
FROM EMP E JOIN DEPT D
WHERE E.SAL >= 3000
ON E.DEPTNO = D.DEPTNO;

올바른 순서

SELECT E.ENAME, E.SAL, D.DNAME
FROM EMP E JOIN DEPT D
ON E.DEPTNO = D.DEPTNO
WHERE E.SAL >= 3000;
2 Tables

10. 2개 테이블 INNER JOIN 실습

SCOTT 계정의 EMP와 DEPT는 INNER JOIN을 연습하기 가장 좋은 기본 예제입니다. EMP에는 사원 정보가 있고, DEPT에는 부서 정보가 있습니다.

10-1. EMP 단독 조회

SELECT ENAME, SAL, DEPTNO
FROM EMP;

사원명, 급여, 부서번호는 나오지만 부서명은 나오지 않습니다.

10-2. DEPT 단독 조회

SELECT DEPTNO, DNAME, LOC
FROM DEPT;

부서번호, 부서명, 지역은 나오지만 사원명은 나오지 않습니다.

10-3. EMP + DEPT JOIN

SELECT E.ENAME,
       E.SAL,
       E.DEPTNO,
       D.DNAME,
       D.LOC
FROM EMP E INNER JOIN DEPT D
ON E.DEPTNO = D.DEPTNO;

10-4. 조건과 정렬 추가

SELECT E.ENAME,
       E.SAL,
       E.DEPTNO,
       D.DNAME
FROM EMP E INNER JOIN DEPT D
ON E.DEPTNO = D.DEPTNO
WHERE E.SAL >= 3000
ORDER BY E.ENAME DESC;
3 Tables

11. 3개 테이블 INNER JOIN

실무에서는 3개 이상의 테이블을 연결하는 경우가 많습니다. 세 개 이상의 테이블을 JOIN할 때는 각 테이블 사이의 부모-자식 관계와 연결 컬럼을 먼저 파악해야 합니다.

HR 계정 관계 예시

EMPLOYEES.JOB_ID
→ JOBS.JOB_ID

EMPLOYEES.DEPARTMENT_ID
→ DEPARTMENTS.DEPARTMENT_ID

DEPARTMENTS.LOCATION_ID
→ LOCATIONS.LOCATION_ID

11-1. EMPLOYEES + JOBS + DEPARTMENTS

SELECT E.LAST_NAME || ' ' || E.FIRST_NAME AS "이름",
       E.SALARY,
       J.JOB_TITLE,
       D.DEPARTMENT_NAME
FROM EMPLOYEES E
JOIN JOBS J
ON E.JOB_ID = J.JOB_ID
JOIN DEPARTMENTS D
ON E.DEPARTMENT_ID = D.DEPARTMENT_ID
WHERE E.SALARY >= 10000;

11-2. EMPLOYEES + DEPARTMENTS + LOCATIONS

SELECT E.LAST_NAME || E.FIRST_NAME AS "이름",
       E.SALARY,
       D.DEPARTMENT_NAME,
       L.CITY
FROM EMPLOYEES E
JOIN DEPARTMENTS D
ON E.DEPARTMENT_ID = D.DEPARTMENT_ID
JOIN LOCATIONS L
ON L.LOCATION_ID = D.LOCATION_ID;

11-3. Oracle식 3개 테이블 조인

SELECT E.LAST_NAME || E.FIRST_NAME AS "이름",
       E.SALARY,
       D.DEPARTMENT_NAME,
       L.CITY
FROM EMPLOYEES E, DEPARTMENTS D, LOCATIONS L
WHERE E.DEPARTMENT_ID = D.DEPARTMENT_ID
  AND L.LOCATION_ID = D.LOCATION_ID;

12. INNER JOIN과 OUTER JOIN 맛보기

INNER JOIN은 연결 조건이 일치하는 데이터만 보여줍니다. 반면 OUTER JOIN은 연결되지 않은 데이터도 기준 테이블에 따라 보여줄 수 있습니다.
이번 단원에서는 INNER JOIN이 중심이므로 차이만 가볍게 기억하면 됩니다.

INNER JOIN

SELECT D.DEPTNO, D.DNAME, E.ENAME
FROM EMP E
JOIN DEPT D
ON E.DEPTNO = D.DEPTNO;

사원이 없는 40번 부서는 결과에서 빠질 수 있습니다.

OUTER JOIN 방향

DEPT에는 있지만 EMP에 연결된 사원이 없는 부서도 보고 싶다면
OUTER JOIN을 배워야 합니다.
이 내용은 다음 OUTER JOIN 단원에서 자세히 다룹니다.

Practice

13. 실습 문제 연결

실습 문제 교안의 조인 파트는 2개 테이블 조인과 3개 테이블 조인을 모두 포함합니다.
문제를 읽을 때는 “어떤 테이블들을 연결하는지”, “연결 조건이 무엇인지”, “검색 조건과 정렬 조건이 무엇인지”를 나누어 읽어야 합니다.

문제 1. EMP + DEPT 조인

SELECT E.ENAME,
       E.SAL,
       E.DEPTNO,
       D.DNAME
FROM EMP E INNER JOIN DEPT D
ON E.DEPTNO = D.DEPTNO
WHERE E.SAL >= 3000
ORDER BY E.ENAME DESC;

사원 정보와 부서명을 함께 조회하고,
급여 3000 이상만 사원명 내림차순으로 출력합니다.

문제 2. Oracle식 JOIN

SELECT E.ENAME,
       E.SAL,
       E.DEPTNO,
       D.DNAME
FROM EMP E, DEPT D
WHERE E.DEPTNO = D.DEPTNO
  AND E.SAL >= 3000
ORDER BY E.ENAME DESC;

같은 결과를 Oracle식 조인 문법으로 작성한 형태입니다.

문제 3. EMPLOYEES + JOBS + DEPARTMENTS

SELECT E.LAST_NAME || ' ' || E.FIRST_NAME AS "이름",
       E.SALARY,
       J.JOB_TITLE,
       D.DEPARTMENT_NAME
FROM EMPLOYEES E
JOIN JOBS J
ON E.JOB_ID = J.JOB_ID
JOIN DEPARTMENTS D
ON E.DEPARTMENT_ID = D.DEPARTMENT_ID
WHERE E.SALARY >= 10000;

문제 4. EMPLOYEES + DEPARTMENTS + LOCATIONS

SELECT E.LAST_NAME || E.FIRST_NAME AS "이름",
       E.SALARY,
       D.DEPARTMENT_NAME,
       L.CITY
FROM EMPLOYEES E
JOIN DEPARTMENTS D
ON E.DEPARTMENT_ID = D.DEPARTMENT_ID
JOIN LOCATIONS L
ON L.LOCATION_ID = D.LOCATION_ID;

문제 읽는 순서

1. 몇 개의 테이블을 연결하는가?
   → EMP, DEPT

2. 연결 기준은 무엇인가?
   → E.DEPTNO = D.DEPTNO

3. 출력 컬럼은 무엇인가?
   → ENAME, SAL, DEPTNO, DNAME

4. 검색 조건은 무엇인가?
   → SAL >= 3000

5. 정렬 조건은 무엇인가?
   → ENAME DESC

14. Java / Spring / Oracle 프로젝트 연결

INNER JOIN은 실제 프로젝트에서 매우 자주 사용됩니다.
단일 테이블에는 ID만 저장하고, 화면에는 이름, 제목, 상태, 가격, 작성자 정보처럼 다른 테이블의 정보가 함께 필요하기 때문입니다.

게시글 목록 + 작성자 이름

SELECT B.BOARD_ID,
       B.TITLE,
       U.NICKNAME,
       B.CREATED_AT
FROM BOARD B
INNER JOIN USERS U
ON B.WRITER_ID = U.USER_ID
ORDER BY B.BOARD_ID DESC;

BOARD의 WRITER_ID와 USERS의 USER_ID를 연결합니다.

댓글 목록 + 댓글 작성자

SELECT C.COMMENT_ID,
       C.CONTENT,
       U.NICKNAME,
       C.CREATED_AT
FROM COMMENTS C
INNER JOIN USERS U
ON C.USER_ID = U.USER_ID
WHERE C.BOARD_ID = ?
ORDER BY C.CREATED_AT ASC;

매물 목록 + 등록자 정보

SELECT P.PROPERTY_ID,
       P.TITLE,
       P.DEAL_AMOUNT,
       U.LOGIN_ID AS REALTOR_LOGIN_ID
FROM PROPERTY P
INNER JOIN USERS U
ON P.USER_ID = U.USER_ID;

문의 목록 + 매물 + 문의자

SELECT I.INQUIRY_ID,
       I.MESSAGE,
       P.TITLE AS PROPERTY_TITLE,
       U.LOGIN_ID AS USER_LOGIN_ID,
       I.CREATED_AT
FROM INQUIRY I
INNER JOIN PROPERTY P
ON I.PROPERTY_ID = P.PROPERTY_ID
INNER JOIN USERS U
ON I.USER_ID = U.USER_ID
ORDER BY I.CREATED_AT DESC;

Mistake

15. 자주 하는 실수

  • 실수 1. 조인 조건을 빼먹는다. 조인 조건 없이 두 테이블을 나열하면 모든 행이 서로 조합되는 카테시안 곱이 발생할 수 있습니다.
  • 실수 2. 같은 이름의 컬럼에 테이블 별칭을 붙이지 않는다. EMP와 DEPT 모두 DEPTNO를 가지므로 E.DEPTNO, D.DEPTNO처럼 명확히 작성해야 합니다.
  • 실수 3. ON과 WHERE 순서를 바꾼다. 표준 JOIN에서는 ON 조인 조건을 먼저 작성하고, 그 뒤에 WHERE 검색 조건을 작성합니다.
  • 실수 4. 테이블 별칭에 AS를 사용한다. Oracle에서는 테이블 별칭을 FROM EMP E처럼 작성합니다.
  • 실수 5. 별칭을 붙여놓고 원래 테이블명을 섞어 쓴다. EMP E라고 별칭을 붙였다면 이후에는 E.ENAME처럼 별칭을 일관되게 사용합니다.
  • 실수 6. ON과 WHERE의 역할을 구분하지 못한다. ON은 테이블 연결 조건, WHERE는 연결된 결과에서 행을 거르는 검색 조건입니다.
  • 실수 7. INNER JOIN과 OUTER JOIN 결과 차이를 모른다. INNER JOIN은 연결 조건이 일치하는 행만 출력하므로 연결되지 않은 기준 데이터는 빠질 수 있습니다.
Interview

16. 면접 포인트

Q1. JOIN이란 무엇인가요?

JOIN은 두 개 이상의 테이블에 나누어 저장된 데이터를 관계 조건을 기준으로 연결해서 조회하는 SQL 문법입니다.
보통 부모 테이블의 Primary Key와 자식 테이블의 Foreign Key를 기준으로 연결합니다.

Q2. INNER JOIN은 무엇인가요?

INNER JOIN은 두 테이블 사이에서 조인 조건이 일치하는 행만 조회하는 조인입니다.
예를 들어 EMP.DEPTNO와 DEPT.DEPTNO가 같은 행끼리 연결하면 사원 정보와 부서 정보를 함께 조회할 수 있습니다.

Q3. INNER JOIN과 OUTER JOIN의 차이는 무엇인가요?

INNER JOIN은 조인 조건이 양쪽 테이블에서 일치하는 행만 출력합니다.
OUTER JOIN은 조인 조건이 일치하지 않아도 기준 테이블의 행을 출력할 수 있습니다.
예를 들어 사원이 없는 부서까지 보고 싶다면 OUTER JOIN을 사용해야 합니다.

Q4. ON절과 WHERE절의 차이는 무엇인가요?

ON절은 테이블끼리 어떻게 연결할지 정하는 조인 조건을 작성하는 곳입니다. WHERE절은 조인된 결과에서 원하는 행만 필터링하는 검색 조건을 작성하는 곳입니다.

Q5. 조인할 때 같은 이름의 컬럼이 있으면 어떻게 하나요?

같은 이름의 컬럼이 여러 테이블에 존재하면 반드시 테이블명이나 별칭을 붙여 구분해야 합니다.
예를 들어 EMP와 DEPT 모두 DEPTNO가 있으므로 E.DEPTNO 또는 D.DEPTNO처럼 작성해야 합니다.

Q6. Oracle에서 테이블 별칭에 AS를 사용할 수 있나요?

Oracle에서는 컬럼 별칭에는 AS를 사용할 수 있지만, 테이블 별칭에는 AS를 사용하지 않는 것이 원칙입니다. 따라서 FROM EMP E처럼 작성합니다.

Q7. 조인 조건을 빼먹으면 어떤 문제가 생기나요?

조인 조건을 빼먹으면 두 테이블의 모든 행이 서로 조합되는 카테시안 곱이 발생할 수 있습니다. 예를 들어 EMP가 14행, DEPT가 4행이면 56행이 나올 수 있습니다.

Summary

17. 최종 정리

JOIN
→ 두 개 이상의 테이블에서 데이터를 함께 조회하는 방법

INNER JOIN
→ 조인 조건이 일치하는 행만 조회

ON
→ 테이블 간 연결 조건 작성

PK
→ 부모 테이블에서 행을 대표하는 키

FK
→ 자식 테이블에서 부모 테이블을 참조하는 키

테이블 별칭
→ 긴 테이블명을 짧게 줄여 SQL을 간결하게 작성

동일 컬럼명
→ E.DEPTNO, D.DEPTNO처럼 테이블 별칭을 붙여 구분

표준 INNER JOIN
→ FROM EMP E INNER JOIN DEPT D ON E.DEPTNO = D.DEPTNO

Oracle식 JOIN
→ FROM EMP E, DEPT D WHERE E.DEPTNO = D.DEPTNO

읽는 순서

1. FROM에서 기준 테이블을 확인한다.
2. JOIN으로 어떤 테이블을 연결하는지 확인한다.
3. ON에서 연결 조건을 확인한다.
4. SELECT에서 어느 테이블의 어떤 컬럼을 출력하는지 확인한다.
5. WHERE가 있으면 검색 조건을 확인한다.
6. ORDER BY가 있으면 정렬 조건을 확인한다.

학습 메모

오늘의 핵심

INNER JOIN은 여러 테이블에 나뉜 데이터를 연결 조건이 맞는 행끼리 합쳐서 조회하는 문법입니다. 가장 먼저 어떤 테이블들이 어떤 컬럼으로 연결되는지 찾아야 합니다.

기본 문법

SELECT 별칭1.컬럼명,
       별칭2.컬럼명
FROM 테이블1 별칭1
INNER JOIN 테이블2 별칭2
ON 별칭1.공통컬럼 = 별칭2.공통컬럼
WHERE 검색조건
ORDER BY 정렬조건;

암기 포인트

  • JOIN : 여러 테이블 연결 조회
  • INNER JOIN : 조건이 맞는 행만 출력
  • ON : 테이블 연결 조건
  • WHERE : 연결 후 검색 조건
  • PK : 부모 테이블 대표 키
  • FK : 부모 키를 참조하는 자식 컬럼

실습 확인 순서

  1. SHOW USER로 현재 계정 확인
  2. DESC EMP와 DESC DEPT로 컬럼 확인
  3. EMP 단독 조회
  4. DEPT 단독 조회
  5. DEPTNO 기준 INNER JOIN 실행
  6. WHERE 조건 추가
  7. ORDER BY 정렬 추가
  8. HR 계정에서 3개 테이블 JOIN 실행

주의할 점

조인 조건을 빼먹으면 카테시안 곱이 발생할 수 있습니다.
같은 컬럼명이 여러 테이블에 있으면 반드시 E.DEPTNO처럼 별칭을 붙이고, Oracle에서는 테이블 별칭에 AS를 사용하지 않는다고 기억하세요.
