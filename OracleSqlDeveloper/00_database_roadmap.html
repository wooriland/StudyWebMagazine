Oracle SQL Developer · 00

Database 학습 로드맵

Oracle Database를 단순한 SQL 문법 모음으로 보지 않고,
사용자와 계정, 테이블, 데이터, SQL, 제약조건, 트랜잭션, 뷰와 인덱스,
그리고 PL/SQL까지 연결된 하나의 데이터 관리 시스템으로 이해하는 첫 단원입니다.

1. Database를 공부한다는 것

Database는 데이터를 아무렇게나 저장하는 공간이 아니라, 데이터를 일정한 구조에 맞게 저장하고 필요할 때 조회, 입력, 수정, 삭제할 수 있도록 관리하는 시스템입니다.
엑셀에서 행과 열로 데이터를 정리하듯이, 데이터베이스도 테이블이라는 구조 안에 데이터를 저장합니다. 하지만 데이터베이스는 엑셀보다 훨씬 엄격합니다.
컬럼마다 자료형을 정하고, 중복을 막거나, 비어 있으면 안 되는 값을 제한하거나, 테이블 사이의 관계를 유지할 수 있습니다.
따라서 Oracle Database를 공부한다는 것은 단순히 SELECT 문법을 외우는 것이 아니라,
데이터가 어떤 구조로 저장되고, 어떤 계정이 접근하며, 어떤 SQL로 조작되고, 어떤 규칙으로 안전하게 관리되는지를 이해하는 과정입니다.

2. Oracle Database 전체 구조

앞으로 공부할 Oracle Database는 다음과 같은 큰 구조로 이해할 수 있습니다. 각 항목은 이후 단원에서 하나씩 자세히 학습하게 됩니다.

  1. 사용자 / 계정 Oracle에 접속하는 주체입니다. SYSTEM은 관리자 계정, SCOTT과 HR은 SQL 실습에 자주 사용하는 샘플 계정입니다.
  2. 테이블 데이터를 행과 열로 저장하는 기본 단위입니다. EMP, DEPT, EMPLOYEES, DEPARTMENTS 같은 테이블을 사용해 실습합니다.
  3. 데이터 테이블 안에 저장되는 실제 값입니다. 한 줄은 행 또는 레코드라고 하고, 세로 항목은 컬럼이라고 합니다.
  4. SQL 데이터베이스에 명령을 내리는 언어입니다. SELECT, INSERT, UPDATE, DELETE, CREATE, ALTER, DROP 등을 학습합니다.
  5. 제약조건 데이터가 잘못 저장되지 않도록 막는 규칙입니다. PRIMARY KEY, FOREIGN KEY, NOT NULL, UNIQUE, CHECK, DEFAULT를 배웁니다.
  6. 트랜잭션 여러 데이터 변경 작업을 하나의 작업 단위로 관리하는 개념입니다. COMMIT, ROLLBACK, SAVEPOINT를 사용합니다.
  7. 뷰 / 인덱스 VIEW는 복잡한 조회 결과를 가상의 테이블처럼 사용하는 기능이고, INDEX는 검색 성능을 높이기 위한 객체입니다.
  8. PL/SQL SQL에 변수, 조건문, 반복문, 예외 처리 같은 프로그래밍 기능을 더한 Oracle의 절차형 언어입니다.

3. SQL 학습 순서 전체 지도

Oracle SQL은 처음부터 어려운 JOIN이나 PL/SQL로 들어가면 흐름을 놓치기 쉽습니다.
먼저 접속과 테이블 구조를 확인하고, SELECT로 데이터를 조회한 뒤, 조건, 정렬, 그룹, 조인, 서브쿼리, 테이블 생성, 트랜잭션 순서로 확장하는 것이 좋습니다.

localhost, 1521, XEPDB1

SELECT 문으로 테이블의

04

WHERE, BETWEEN, IN, LIKE, IS NULL, ORDER BY를

4. SQL Developer에서 먼저 확인할 기본 SQL

이 단원에서는 복잡한 SQL을 작성하기보다, SQL Developer에서 내가 어떤 계정으로 접속했는지, 어떤 테이블을 사용할 수 있는지 확인하는 것이 중요합니다.

현재 사용자 확인

SHOW USER;

현재 SQL Developer에서 접속한 계정이 SYSTEM인지, SCOTT인지, HR인지 확인합니다.

현재 계정의 테이블 목록 확인

SELECT table_name
FROM user_tables;

현재 접속한 계정이 소유한 테이블 목록을 확인합니다. SCOTT 계정이라면 EMP, DEPT 같은 테이블을 기대할 수 있습니다.

접근 가능한 전체 테이블 확인

SELECT owner, table_name
FROM all_tables
ORDER BY owner, table_name;

현재 사용자가 접근 권한을 가진 테이블을 확인합니다. 다른 스키마의 테이블이 포함될 수 있습니다.

테이블 구조 확인

DESC emp;

DESC employees;

테이블의 컬럼명, NULL 허용 여부, 자료형을 확인합니다. SQL을 작성하기 전 테이블 구조를 먼저 확인하는 습관이 중요합니다.

CDB / PDB 확인

SELECT name
FROM v$pdbs;

SELECT name
FROM v$database;

SYSTEM 계정에서 Oracle의 PDB 목록과 CDB 이름을 확인합니다. Oracle XE 실습에서는 보통 XEPDB1에 접속합니다.

5. 이번 단원에서 기억할 핵심

Oracle Database는 SQL 문법만 배우는 과목이 아닙니다.
데이터를 저장하는 테이블, 테이블을 소유하는 사용자, 데이터를 안전하게 지키는 제약조건, 작업을 확정하거나 취소하는 트랜잭션,
조회를 편하게 하는 VIEW, 검색을 빠르게 하는 INDEX, 그리고 데이터베이스 내부 로직을 작성하는 PL/SQL까지 연결해서 이해해야 합니다.

최종 정리: Oracle Database는 데이터를 테이블에 저장하고, 사용자와 권한으로 접근을 통제하며,
SQL로 데이터를 조회·입력·수정·삭제하고, 제약조건으로 무결성을 지키며, 트랜잭션으로 작업의 확정과 취소를 관리하는 시스템입니다.

Oracle Database는 데이터를 테이블에 저장하고, 사용자와 권한으로 접근을 통제하며, SQL로 데이터를 조회·입력·수정·삭제하고,
제약조건과 트랜잭션으로 데이터의 안정성을 지키며, VIEW, INDEX, PL/SQL로 조회 편의성, 성능, 내부 처리 로직까지 확장할 수 있는 시스템입니다.
