Oracle SQL Developer · 04

SCOTT / HR 계정과 샘플 스키마

Oracle SQL 학습에서 자주 사용하는 SCOTT 계정과 HR 계정의 역할을 이해하고,
각 계정이 가진 샘플 테이블의 구조를 확인합니다.
SCOTT은 EMP, DEPT를 중심으로 SQL 기초를 연습하기 좋고,
HR은 EMPLOYEES, DEPARTMENTS, JOBS, LOCATIONS, COUNTRIES를 활용해
JOIN과 실무형 관계형 데이터 조회를 연습하기에 적합합니다.

1. 이 단원에서 배우는 것

이번 단원은 Oracle SQL 문법을 본격적으로 공부하기 전에, SQL 실습에 사용할 대표 샘플 계정인 SCOTT과 HR을 이해하는 단원입니다.
SQL 문법은 결국 테이블을 대상으로 실행되기 때문에, 어떤 계정에 어떤 테이블이 있는지 먼저 알아야 안정적으로 실습할 수 있습니다.
SCOTT 계정은 EMP, DEPT처럼 구조가 단순한 테이블을 가지고 있어서 SELECT, WHERE, ORDER BY, GROUP BY, JOIN의 기초를 익히기에 좋습니다.
HR 계정은 EMPLOYEES, DEPARTMENTS, JOBS, LOCATIONS, COUNTRIES처럼 여러 테이블이 연결된 구조를 가지고 있어 실무형 JOIN과 관계형 데이터 조회를 연습하기 좋습니다.

SCOTT EMP, DEPT 중심의 SQL 기초 실습 계정
HR 인사 관리 구조를 가진 실무형 샘플 계정
Sample Schema SQL 실습을 위해 미리 준비된 테이블과 데이터 묶음

2. 사용자 계정과 스키마 이해하기

Oracle에서는 사용자 계정과 스키마를 거의 함께 이해해야 합니다.
사용자 계정은 Oracle Database에 접속하기 위한 이름이고, 스키마는 그 사용자가 소유한 테이블, 뷰, 시퀀스, 프로시저 같은 객체들의 묶음입니다.

사용자 계정

Oracle Database에 로그인하기 위한 계정입니다.
예를 들어 SYSTEM, SCOTT, HR, APP_USER 같은 이름이 사용자 계정입니다.

  • Oracle에 접속하는 주체
  • 비밀번호를 가지고 있음
  • 권한에 따라 할 수 있는 작업이 달라짐
  • SQL Developer 접속 시 사용자 이름에 입력함

스키마

특정 사용자가 소유한 데이터베이스 객체들의 공간입니다.
SCOTT 사용자가 EMP와 DEPT 테이블을 가지고 있다면,
그것을 SCOTT 스키마의 테이블이라고 볼 수 있습니다.

  • 사용자가 소유한 객체 묶음
  • 테이블, 뷰, 시퀀스 등을 포함
  • 계정명과 스키마명이 같은 경우가 많음
  • 실습 문제의 기준이 되는 데이터 공간

비유로 이해하기

사용자 계정은 사무실에 들어가기 위한 출입증이고, 스키마는 그 사용자가 사용하는 사무실 공간입니다.
테이블은 사무실 안의 문서함이고, 데이터는 문서함 안에 들어 있는 실제 문서라고 생각하면 됩니다.

3. SCOTT 계정 이해하기

SCOTT은 Oracle SQL 학습에서 오래 사용되어 온 대표적인 샘플 계정입니다. 테이블 구조가 단순하고 데이터 양도 적절하기 때문에 SQL 기초 문법을 연습하기 좋습니다.

SCOTT Schema 기초 SQL 실습용 샘플 계정

EMP

직원 정보를 담은 테이블입니다.
사원번호, 이름, 직무, 관리자 번호, 입사일, 급여, 보너스, 부서번호를 다룹니다.

  • EMPNO : 사원번호
  • ENAME : 사원이름
  • JOB : 직무
  • MGR : 관리자 사원번호
  • HIREDATE : 입사일
  • SAL : 급여
  • COMM : 보너스
  • DEPTNO : 부서번호

DEPT

부서 정보를 담은 테이블입니다.
부서번호, 부서명, 부서 위치를 가지고 있으며,
EMP 테이블의 DEPTNO와 연결됩니다.

  • DEPTNO : 부서번호
  • DNAME : 부서명
  • LOC : 부서 위치

SCOTT 테이블 관계

EMP.DEPTNO  ───── 연결 ─────  DEPT.DEPTNO

직원 테이블의 부서번호           부서 테이블의 부서번호

이 관계를 이용하면 직원 이름, 직무, 급여와 함께 부서명, 부서 위치를 함께 조회할 수 있습니다. 그래서 SCOTT 계정은 JOIN을 처음 연습하기에도 좋습니다.

4. HR 계정 이해하기

HR은 Human Resources, 즉 인사 관리 샘플 스키마입니다.
SCOTT보다 테이블 수가 많고 관계가 더 현실적이어서, 여러 테이블을 연결하는 JOIN 실습과 실무형 조회 연습에 적합합니다.

HR Schema 인사 관리 구조를 가진 실무형 샘플 계정

EMPLOYEES

직원 정보를 담는 핵심 테이블입니다.

  • EMPLOYEE_ID : 직원 ID
  • FIRST_NAME, LAST_NAME : 이름
  • JOB_ID : 직무 ID
  • SALARY : 급여
  • DEPARTMENT_ID : 부서 ID

DEPARTMENTS

부서 정보를 담는 테이블입니다.

  • DEPARTMENT_ID : 부서 ID
  • DEPARTMENT_NAME : 부서명
  • MANAGER_ID : 부서 관리자
  • LOCATION_ID : 위치 ID

JOBS

직무 정보를 담는 테이블입니다.

  • JOB_ID : 직무 ID
  • JOB_TITLE : 직무명
  • MIN_SALARY : 최소 급여
  • MAX_SALARY : 최대 급여

LOCATIONS

부서가 위치한 도시 정보를 담는 테이블입니다.

  • LOCATION_ID : 위치 ID
  • CITY : 도시명
  • COUNTRY_ID : 국가 ID

COUNTRIES

국가 정보를 담는 테이블입니다.

  • COUNTRY_ID : 국가 ID
  • COUNTRY_NAME : 국가명
  • REGION_ID : 지역 ID

REGIONS / JOB_HISTORY

지역 정보와 직원의 직무 이력을 관리하는 보조 테이블입니다. HR 스키마의 관계형 구조를 더 확장해서 이해할 때 사용합니다.

5. 그림처럼 이해하기

SCOTT과 HR은 모두 XEPDB1 안에서 실습에 사용하는 샘플 계정입니다. 다만 SCOTT은 단순한 구조, HR은 더 실무적인 다중 테이블 구조를 가지고 있습니다.

DEPT

SQL 기초 문법과 단순 JOIN 실습에 적합합니다.

HR

DEPARTMENTS
JOBS
LOCATIONS
COUNTRIES

실무형 JOIN과 다중 테이블 관계 이해에 적합합니다.

HR 테이블 관계 흐름

EMPLOYEES.DEPARTMENT_ID
        ↓
DEPARTMENTS.DEPARTMENT_ID

HR 스키마는 직원, 부서, 직무, 위치, 국가 정보가 분리되어 있고, 필요한 정보를 JOIN으로 연결해서 가져오는 구조입니다.

6. SCOTT / HR 계정 활성화 흐름

SCOTT과 HR은 설치 직후 바로 사용할 수 없는 경우가 있습니다.
따라서 SYSTEM 계정으로 접속해서 계정이 존재하는지 확인하고, 필요하면 비밀번호 설정이나 잠금 해제를 진행해야 합니다.

01 SYSTEM 접속

관리자 계정인 SYSTEM으로 XEPDB1에 접속합니다. 샘플 계정 확인과 활성화는 관리자 권한이 필요합니다.

02 계정 존재 확인

ALL_USERS에서 SCOTT과 HR 사용자가 존재하는지 확인합니다.

03 잠금 해제

HR처럼 잠겨 있는 계정은 ALTER USER로 비밀번호를 설정하고 잠금을 해제합니다.

04 계정 접속

SQL Developer에서 SCOTT 또는 HR 계정으로 새 접속을 만들고 XEPDB1에 접속합니다.

05 테이블 확인

USER_TABLES와 DESC 명령으로 샘플 테이블과 컬럼 구조를 확인합니다.

7. SQL Developer에서 직접 확인할 SQL

이번 단원에서 SQL은 샘플 계정과 테이블이 정상적으로 준비되어 있는지 확인하는 데 사용합니다. 본격적인 문제 풀이 전, 현재 계정과 테이블 구조를 반드시 확인해야 합니다.

현재 접속 계정 확인

SHOW USER;

현재 SQL Developer에서 어떤 계정으로 접속했는지 확인합니다. SCOTT 문제를 풀 때는 SCOTT, HR 문제를 풀 때는 HR인지 확인해야 합니다.

SCOTT / HR 사용자 존재 확인

SELECT username
FROM all_users
WHERE username IN ('SCOTT', 'HR')
ORDER BY username;

SYSTEM 계정에서 실행합니다. SCOTT과 HR 사용자가 XEPDB1 안에 준비되어 있는지 확인합니다.

HR 비밀번호 설정과 잠금 해제

ALTER USER hr IDENTIFIED BY hr;

ALTER USER hr ACCOUNT UNLOCK;

HR 계정이 잠겨 있거나 비밀번호를 모를 때 SYSTEM 계정으로 실행합니다. 첫 번째 SQL은 비밀번호 설정, 두 번째 SQL은 계정 잠금 해제입니다.

현재 계정의 테이블 목록 확인

SELECT table_name
FROM user_tables
ORDER BY table_name;

현재 접속한 계정이 가진 테이블 목록을 확인합니다. SCOTT에서는 EMP, DEPT를 확인하고 HR에서는 EMPLOYEES, DEPARTMENTS를 확인합니다.

SCOTT 테이블 구조 확인

DESC emp;

DESC dept;

SCOTT 계정에서 직원 테이블과 부서 테이블의 컬럼 구조를 확인합니다. EMP의 급여 컬럼은 SAL입니다.

HR 테이블 구조 확인

DESC employees;

DESC departments;

DESC jobs;

DESC locations;

DESC countries;

HR 계정에서 인사 관리 샘플 테이블들의 컬럼 구조를 확인합니다. EMPLOYEES의 급여 컬럼은 SALARY입니다.

샘플 데이터 확인

SELECT *
FROM emp;

SELECT *
FROM dept;

SELECT *
FROM employees;

SELECT *
FROM departments;

실제 데이터가 들어 있는지 확인합니다. 계정에 따라 사용할 수 있는 테이블이 다르므로 현재 접속 계정을 먼저 확인해야 합니다.

SQL Developer에서 SQL을 실행한 뒤 결과 그리드에서 아래 항목을 확인합니다. 이 확인이 끝나야 실습 문제를 안정적으로 풀 수 있습니다.

구분 확인 SQL 기대 결과

10. Java / Spring 프로젝트와의 연결

SCOTT과 HR은 학습용 샘플 스키마이지만, 실제 Java / Spring Boot 프로젝트에서 사용하는 애플리케이션 계정을 이해하는 준비 단계가 됩니다.

학습용 계정 구조

XEPDB1
├─ SCOTT
│   ├─ EMP
│   └─ DEPT
│
└─ HR
    ├─ EMPLOYEES
    ├─ DEPARTMENTS
    └─ JOBS

SCOTT과 HR은 SQL 문법을 연습하기 위한 샘플 계정입니다.

프로젝트용 계정 구조

XEPDB1
└─ APP_USER
    ├─ USERS
    ├─ BOARD
    ├─ PROPERTY
    ├─ PROPERTY_LIKE
    └─ RECOMMENDATION_LOG

Spring Boot 프로젝트에서는 별도의 APP_USER 계정으로 접속해
프로젝트 테이블을 관리하는 것이 일반적입니다.

SCOTT / HR에서 USER_TABLES와 DESC로 테이블 구조를 확인하는 습관은 나중에 Spring Boot의 JPA Entity와 실제 Oracle 테이블을 연결할 때도 그대로 사용됩니다.

11. 자주 하는 실수

  • SYSTEM 계정에서 EMP 테이블을 찾으려고 함 EMP는 보통 SCOTT 계정의 테이블입니다. 먼저 SHOW USER;로 현재 계정을 확인해야 합니다.
  • HR 계정에서 EMP를 조회함 HR의 직원 테이블은 EMP가 아니라 EMPLOYEES입니다.
  • SCOTT의 SAL과 HR의 SALARY를 헷갈림 SCOTT.EMP의 급여 컬럼은 SAL이고, HR.EMPLOYEES의 급여 컬럼은 SALARY입니다.
  • HR 계정이 잠겨 있는데 비밀번호만 계속 입력함 HR 계정은 비밀번호 설정뿐 아니라 ACCOUNT UNLOCK도 필요할 수 있습니다.
  • XEPDB1이 아닌 다른 컨테이너에서 계정을 확인함 SCOTT / HR은 보통 XEPDB1 안에서 확인합니다. 컨테이너가 다르면 계정이 보이지 않을 수 있습니다.
  • 테이블 구조를 확인하지 않고 문제부터 풀기 시작함 문제를 풀기 전 DESC로 컬럼명을 확인해야 합니다. 컬럼명을 잘못 쓰면 ORA-00904 같은 오류가 발생할 수 있습니다.

12. 면접 포인트

Q1. Oracle에서 사용자와 스키마의 관계를 설명해보세요.

사용자는 Oracle에 접속하는 계정이고, 스키마는 그 사용자가 소유한 데이터베이스 객체들의 집합입니다.
예를 들어 SCOTT 사용자가 EMP와 DEPT 테이블을 소유하고 있다면, SCOTT 스키마 안에 EMP와 DEPT가 있다고 볼 수 있습니다.

Q2. SCOTT 계정은 어떤 용도로 사용하나요?

SCOTT 계정은 Oracle SQL 학습에서 전통적으로 사용하는 샘플 계정입니다.
EMP와 DEPT 테이블을 중심으로 SELECT, WHERE, ORDER BY, GROUP BY, JOIN, SUBQUERY 같은 SQL 기초 문법을 연습하기에 좋습니다.

Q3. HR 계정은 어떤 용도로 사용하나요?

HR 계정은 Human Resources 샘플 스키마입니다.
EMPLOYEES, DEPARTMENTS, JOBS, LOCATIONS, COUNTRIES 같은 테이블을 가지고 있어 다중 JOIN과 실무형 관계형 데이터 조회를 연습하기 좋습니다.

Q4. SCOTT의 EMP와 HR의 EMPLOYEES는 어떤 차이가 있나요?

SCOTT의 EMP는 EMPNO, ENAME, JOB, SAL, DEPTNO 같은 단순한 컬럼을 가진 기초 실습용 직원 테이블입니다.
HR의 EMPLOYEES는 EMPLOYEE_ID, FIRST_NAME, LAST_NAME, JOB_ID, SALARY, DEPARTMENT_ID 등 더 실무형 구조를 가진 직원 테이블입니다.

Q5. SQL 실습 전 SHOW USER를 실행하는 이유는 무엇인가요?

현재 어떤 계정으로 접속했는지 확인하기 위해서입니다. SCOTT과 HR은 가지고 있는 테이블이 다르므로, 계정을 확인하지 않으면 테이블이 없다는 오류가 발생할 수 있습니다.

Q6. HR 계정이 접속되지 않을 때 어떤 점을 확인하겠습니까?

먼저 HR 계정이 존재하는지 ALL_USERS에서 확인하고, 필요한 경우 SYSTEM 계정으로 비밀번호를 설정하고 계정 잠금을 해제합니다.
또한 XEPDB1에 접속하고 있는지도 확인해야 합니다.

13. 이번 단원 핵심 정리

SCOTT과 HR은 Oracle SQL 학습을 위한 대표 샘플 계정입니다.
SCOTT은 EMP, DEPT를 중심으로 SQL 기초 문법을 연습하기 좋고,
HR은 EMPLOYEES, DEPARTMENTS, JOBS, LOCATIONS, COUNTRIES를 중심으로 다중 JOIN과 실무형 조회를 연습하기 좋습니다.
실습 문제를 풀기 전에는 반드시 SHOW USER, USER_TABLES, DESC 명령으로 현재 계정과 테이블 구조를 확인해야 합니다.

학습 메모

이 페이지는 Oracle SQL 학습에서 자주 사용하는 SCOTT 계정과 HR 계정의 역할을 이해하는 단원입니다.
SCOTT은 EMP, DEPT 테이블을 중심으로 SELECT, WHERE, ORDER BY, GROUP BY, JOIN 기초를 연습하기 좋고,
HR은 EMPLOYEES, DEPARTMENTS, JOBS, LOCATIONS, COUNTRIES 테이블을 활용해 실무형 JOIN과 다중 테이블 관계를 연습하기 좋습니다.
실습 문제를 풀기 전에는 반드시 SHOW USER로 현재 계정을 확인하고, USER_TABLES와 DESC 명령으로 테이블 목록과 컬럼 구조를 확인하는 습관을 가져야 합니다.
