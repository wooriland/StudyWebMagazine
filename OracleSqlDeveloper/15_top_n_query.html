TOP-N 쿼리와 페이징

TOP-N 쿼리는 정렬된 결과 중 상위 N개 데이터만 가져오는 SQL 기법이고,
페이징은 정렬된 결과 중 특정 구간만 잘라서 가져오는 SQL 기법입니다.
이 단원은 게시판 목록, 매물 목록, 관리자 통계, 추천 결과, 검색 결과 페이지 구현에 직접 연결됩니다.

핵심 공식 offset = (page - 1) × size
핵심 흐름 조건 필터링 → 정렬 → 일부 행만 조회
프로젝트 연결 게시판, 매물 목록, 관리자 TOP 통계와 연결한다.
Overview

1. TOP-N 쿼리와 페이징이란?

TOP-N 쿼리는 전체 데이터 중에서 조건에 맞는 모든 데이터를 가져오는 것이 아니라, 특정 기준으로 정렬한 뒤 위에서 몇 개만 가져오는 쿼리입니다. 페이징은 정렬된 결과 중 앞의 일부를 건너뛰고, 현재 페이지에 필요한 개수만 가져오는 방식입니다.

TOP-N 정렬된 결과 중 위에서 N개만 조회
페이징 정렬된 결과 중 특정 구간만 조회
실무 연결 게시판, 매물 목록, 검색 결과, 관리자 통계

한 줄 정리

TOP-N은 “위에서 N개만 가져오기”이고, 페이징은 “앞에서 일부를 건너뛰고 필요한 개수만 가져오기”입니다.

Concept Card

2. 핵심 개념 카드

2-1. TOP-N 쿼리란?

TOP-N 쿼리는 반드시 정렬 기준이 있어야 의미가 있습니다. 단순히 3개를 가져오는 것이 아니라, 무엇을 기준으로 상위 3개인지가 중요합니다.

다음 요구사항은 모두 TOP-N 쿼리입니다.

  • 연봉이 높은 직원 상위 3명 조회
  • 최근 등록된 게시글 10개 조회
  • 조회수가 높은 매물 TOP 5 조회
  • 가격이 낮은 상품 20개 조회
  • 위험도 점수가 높은 매물 TOP 10 조회
TOP-N 쿼리 정렬 기준 + 행 개수 제한

2-2. 페이징이란?

페이징은 전체 조회 결과를 한 번에 모두 출력하지 않고 일정 개수씩 나누어 보여주는 방식입니다. 게시판, 검색 결과, 상품 목록, 매물 목록에서 거의 필수적으로 사용됩니다.

페이지 한 페이지 크기 조회 구간 OFFSET 값
1페이지 10개 1번 ~ 10번 0
2페이지 10개 11번 ~ 20번 10
3페이지 10개 21번 ~ 30번 20
4페이지 10개 31번 ~ 40번 30

2-3. 페이징 공식

페이징 공식 offset = (page - 1) × size 
-- page = 현재 페이지 번호
-- size = 한 페이지에 보여줄 데이터 개수

offset = (page - 1) * size

2-4. Oracle에서 TOP-N과 페이징을 구현하는 방식

ROWNUM 방식

Oracle 구버전에서도 사용 가능하지만, 정렬 후 자르려면 인라인 뷰가 필요합니다.

OFFSET / FETCH 방식

Oracle 12c 이상에서 지원하며, 문법이 직관적이고 실무 페이징에 적합합니다.

분석 함수 방식

ROW_NUMBER(), RANK(), DENSE_RANK()를 사용하며 그룹별 TOP-N에 강합니다.

WITH절 방식

복잡한 서브쿼리에 이름을 붙여 가독성을 높이는 방식입니다.

3. 그림처럼 이해하기

3-1. TOP-N 쿼리 흐름

연봉이 높은 직원 상위 3명을 가져온다고 생각해 봅시다. 먼저 전체 데이터를 가져온 뒤, 연봉 기준으로 정렬하고, 그중 위에서 3명만 남깁니다.

1단계 전체 데이터

EMP 테이블 전체 직원 조회

→
2단계 정렬

SAL DESC 기준 정렬

→
3단계 행 제한

위에서 3개만 조회

정렬 전

  1. SMITH - 800
  2. ALLEN - 1600
  3. WARD - 1250
  4. JONES - 2975
  5. KING - 5000

연봉 내림차순 정렬 후

  1. KING - 5000
  2. SCOTT - 3000
  3. FORD - 3000
  4. JONES - 2975
  5. BLAKE - 2850

TOP 3 결과

  1. KING - 5000
  2. SCOTT - 3000
  3. FORD - 3000

3-2. 페이징 흐름

페이징은 정렬된 결과에서 앞의 일부를 건너뛰고, 현재 페이지에 필요한 개수만 가져오는 방식입니다.

1페이지 1 ~ 10

OFFSET 0

2페이지 11 ~ 20

OFFSET 10

3페이지 21 ~ 30

OFFSET 20

4페이지 31 ~ 40

OFFSET 30

3-3. ROWNUM 방식의 3단계 구조

ROWNUM으로 구간 조회를 할 때는 한 번에 처리하지 않고, 정렬 → 번호 부여 → 구간 필터링의 3단계로 나누어 생각해야 합니다.

가장 안쪽 쿼리 ORDER BY로 먼저 정렬 SELECT * FROM emp ORDER BY sal DESC
중간 쿼리 ROWNUM에 별칭 부여 SELECT sorted_emp.*, ROWNUM rnum FROM (...)
가장 바깥 쿼리 BETWEEN으로 구간 조회 WHERE rnum BETWEEN 6 AND 10
SQL Examples

4. SQL 예제

4-1. Oracle 12c 이상: 연봉 상위 3명

SELECT empno, ename, job, sal
FROM emp
ORDER BY sal DESC
FETCH FIRST 3 ROWS ONLY;

4-2. OFFSET을 명시한 TOP-N

SELECT empno, ename, job, sal
FROM emp
ORDER BY sal DESC
OFFSET 0 ROWS FETCH NEXT 3 ROWS ONLY;

4-3. 조건이 있는 TOP-N

SELECT ename, sal, job, deptno
FROM emp
WHERE deptno = 30
ORDER BY sal DESC
OFFSET 0 ROWS FETCH NEXT 3 ROWS ONLY;

4-4. JOIN이 있는 TOP-N

SELECT e.ename,
       e.sal,
       e.job,
       e.deptno,
       d.dname
FROM emp e
JOIN dept d
  ON e.deptno = d.deptno
WHERE e.deptno = 30
ORDER BY e.sal DESC
OFFSET 0 ROWS FETCH NEXT 3 ROWS ONLY;

4-5. 입사일 기준 최근 직원 5명

SELECT first_name,
       salary,
       hire_date,
       department_name,
       city
FROM employees e
JOIN departments d
  ON e.department_id = d.department_id
JOIN locations l
  ON d.location_id = l.location_id
ORDER BY hire_date DESC
OFFSET 0 ROWS FETCH NEXT 5 ROWS ONLY;

4-6. 특정 구간 조회: 6번째부터 10번째까지

SELECT *
FROM emp
ORDER BY sal DESC
OFFSET 5 ROWS FETCH NEXT 5 ROWS ONLY;

4-7. ROWNUM 방식 TOP-N

SELECT empno, ename, job, sal
FROM (
    SELECT empno, ename, job, sal
    FROM emp
    ORDER BY sal DESC
)
WHERE ROWNUM <= 3;

4-8. ROWNUM 방식 페이징

SELECT *
FROM (
    SELECT sorted_emp.*, ROWNUM rnum
    FROM (
        SELECT *
        FROM emp
        ORDER BY sal DESC
    ) sorted_emp
)
WHERE rnum BETWEEN 6 AND 10;

4-9. ROW_NUMBER() 분석 함수 방식

SELECT *
FROM (
    SELECT empno,
           ename,
           job,
           sal,
           ROW_NUMBER() OVER (ORDER BY sal DESC) AS rn
    FROM emp
)
WHERE rn BETWEEN 6 AND 10;

4-10. 부서별 TOP-N

SELECT *
FROM (
    SELECT empno,
           ename,
           deptno,
           sal,
           ROW_NUMBER() OVER (
               PARTITION BY deptno
               ORDER BY sal DESC
           ) AS rn
    FROM emp
)
WHERE rn <= 3
ORDER BY deptno, rn;

4-11. 게시판 페이징

SELECT board_id,
       title,
       writer,
       created_at,
       view_count
FROM board
ORDER BY board_id DESC
OFFSET :offset ROWS FETCH NEXT :size ROWS ONLY;

4-12. 내집마련 매물 목록 페이징

SELECT property_id,
       title,
       deal_amount,
       area_sqm,
       property_type,
       created_at
FROM property
WHERE listing_status = 'ACTIVE'
ORDER BY created_at DESC, property_id DESC
OFFSET :offset ROWS FETCH NEXT :size ROWS ONLY;

WITH TIES 주의

WITH TIES는 동점자를 함께 가져올 수 있기 때문에 결과 행 수가 고정되지 않습니다. 따라서 일반적인 게시판이나 매물 목록 페이징에서는 사용하지 않는 것이 좋습니다.

Mistake

6. 자주 하는 실수

  • 실수 1. ORDER BY 없이 TOP-N을 사용함 정렬 기준이 없으면 “상위”라는 의미가 사라집니다.
  • 실수 2. ROWNUM을 먼저 자르고 나중에 정렬함 전체 연봉 상위 3명이 아니라 먼저 선택된 3명을 정렬한 결과가 될 수 있습니다.
  • 실수 3. ROWNUM BETWEEN을 바로 사용함 ROWNUM은 결과에 포함되는 행에 붙는 번호이므로 별칭을 만든 뒤 BETWEEN을 적용해야 합니다.
  • 실수 4. OFFSET 계산을 잘못함 2페이지에서 10개씩 보여줄 때 OFFSET은 2가 아니라 10입니다.
  • 실수 5. 정렬 기준이 불안정함 같은 값이 있는 컬럼만 정렬하면 페이지 결과가 흔들릴 수 있으므로 PK를 보조 정렬 기준으로 추가합니다.
  • 실수 6. WHERE 조건 위치를 잘못 둠 “30번 부서 안에서 상위 3명”이면 부서 조건을 먼저 적용한 뒤 정렬해야 합니다.
Interview

7. 면접 포인트

Q1. TOP-N 쿼리란 무엇인가요?

TOP-N 쿼리는 정렬된 결과에서 상위 N개의 행만 조회하는 쿼리입니다. 예를 들어 연봉 상위 3명, 최근 게시글 10개, 조회수 높은 상품 5개를 조회할 때 사용합니다.

Q2. Oracle에서 ROWNUM을 사용할 때 주의할 점은 무엇인가요?

ROWNUM은 ORDER BY 결과에 최종적으로 붙는 번호가 아니라, Oracle이 행을 읽어오는 과정에서 부여하는 가상의 번호입니다. 따라서 정렬 기준이 있는 TOP-N 쿼리에서는 인라인 뷰 안에서 먼저 ORDER BY를 수행하고, 바깥 쿼리에서 ROWNUM 조건을 적용해야 합니다.

Q3. Oracle 12c 이상에서 페이징 쿼리는 어떻게 작성하나요?

Oracle 12c 이상에서는 OFFSET과 FETCH를 사용합니다. OFFSET은 앞에서 건너뛸 행 수이고, FETCH NEXT는 가져올 행 수입니다. 

SELECT *
FROM board
ORDER BY board_id DESC
OFFSET 10 ROWS FETCH NEXT 10 ROWS ONLY;

Q4. ROWNUM 방식과 OFFSET/FETCH 방식의 차이는 무엇인가요?

ROWNUM 방식은 Oracle 구버전에서도 사용할 수 있지만, 정렬 후 구간 조회를 하려면 인라인 뷰를 여러 번 감싸야 해서 쿼리가 복잡해질 수 있습니다. OFFSET/FETCH 방식은 Oracle 12c 이상에서 지원되며, ORDER BY 뒤에 바로 작성할 수 있어 더 직관적입니다.

Q5. ROW_NUMBER()와 ROWNUM의 차이는 무엇인가요?

ROWNUM은 Oracle이 조회 과정에서 부여하는 가상의 번호입니다. ROW_NUMBER()는 분석 함수로, OVER(ORDER BY ...)에 지정한 정렬 기준에 따라 명확하게 순번을 계산합니다.

Q6. RANK()와 DENSE_RANK()의 차이는 무엇인가요?

둘 다 같은 값에는 같은 순위를 부여합니다. RANK()는 동점자가 있으면 다음 순위를 건너뛰고, DENSE_RANK()는 다음 순위를 건너뛰지 않습니다.

RANK()

1, 2, 2, 4

DENSE_RANK()

1, 2, 2, 3

학습 메모

오늘의 핵심

TOP-N은 정렬된 결과에서 위에서 N개만 가져오고, 페이징은 정렬된 결과에서 특정 구간만 가져오는 쿼리입니다.

페이징 공식

offset = (page - 1) × size

page가 1이고 size가 10이면 offset은 0입니다.
page가 2이고 size가 10이면 offset은 10입니다.

암기 SQL

SELECT *
FROM board
ORDER BY board_id DESC
OFFSET :offset ROWS
FETCH NEXT :size ROWS ONLY;

학습 순서

  1. TOP-N의 의미 이해
  2. ROWNUM의 동작 순서 이해
  3. OFFSET/FETCH 문법 익히기
  4. ROW_NUMBER() 분석 함수 비교
  5. 게시판/매물 목록 페이징으로 연결

주의할 점

TOP-N은 반드시 정렬 기준이 필요합니다. ROWNUM은 ORDER BY보다 먼저 붙을 수 있으므로, 정렬이 필요한 경우 인라인 뷰를 먼저 사용해야 합니다.

프로젝트 연결

게시판 목록, 검색 결과, 매물 목록, 관리자 TOP 통계, 추천 결과 페이지는 모두 TOP-N과 페이징 쿼리로 연결됩니다.
