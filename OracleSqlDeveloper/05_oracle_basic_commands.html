Oracle SQL Developer · 05

Oracle 기본 명령어

Oracle SQL 실습을 시작하기 전에 반드시 알아야 할 기본 명령어를 학습합니다.
현재 접속 계정을 확인하는 SHOW USER, 현재 사용자의 테이블 목록을 확인하는 USER_TABLES,
권한 있는 테이블을 확인하는 ALL_TABLES, 테이블 구조를 확인하는 DESC,
그리고 SQL Plus에서 접속 사용자를 바꾸는 CONN 명령어까지 정리합니다.
이 단원은 이후 SELECT, WHERE, JOIN, Spring Boot Oracle 연결 점검의 기본 루틴이 됩니다.

1. 이 단원에서 배우는 것

이번 단원은 SQL 문법을 깊게 작성하기 전에, Oracle에서 현재 작업 환경을 확인하는 기본 명령어를 배우는 단원입니다.
SQL을 실행하기 전에 내가 어떤 계정으로 접속했는지, 이 계정이 어떤 테이블을 가지고 있는지, 테이블 구조가 어떻게 생겼는지 확인해야 합니다.
초보자가 SQL을 공부할 때 자주 겪는 오류는 문법 자체보다 계정 착각, 테이블명 착각, 컬럼명 착각에서 많이 발생합니다.
예를 들어 SCOTT 계정에는 EMP 테이블이 있지만, HR 계정에는 EMP가 아니라 EMPLOYEES 테이블이 있습니다.

SHOW USER 현재 어떤 계정으로 접속했는지 확인합니다.
USER_TABLES 현재 사용자가 소유한 테이블 목록을 확인합니다.
DESC 테이블의 컬럼명, NULL 여부, 자료형을 확인합니다.

2. Oracle 기본 명령어가 필요한 이유

SQL을 작성하기 전에 가장 먼저 확인해야 하는 것은 현재 접속 상태입니다.
같은 SQL이라도 어느 계정에서 실행하느냐에 따라 정상 실행될 수도 있고, 테이블이 없다는 오류가 발생할 수도 있습니다.

확인 없이 SQL을 실행하는 경우

현재 계정을 확인하지 않고 바로 SQL을 실행하면,
계정이 달라 테이블이 보이지 않거나
컬럼명을 잘못 입력하는 실수가 발생할 수 있습니다. 

SELECT *
FROM emp;

이 SQL은 SCOTT 계정에서는 가능할 수 있지만, HR 계정에서는 EMP 테이블이 없어서 오류가 날 수 있습니다.

기본 명령어로 확인 후 실행하는 경우

현재 사용자와 테이블 목록, 테이블 구조를 먼저 확인하면
어떤 테이블과 컬럼을 사용해야 하는지 명확해집니다. 

SHOW USER;

SELECT table_name
FROM user_tables
ORDER BY table_name;

DESC emp;

이 루틴을 먼저 실행하면 계정 착각과 컬럼명 오류를 크게 줄일 수 있습니다.

핵심: Oracle 기본 명령어는 SQL 문법을 작성하기 전, 현재 계정과 테이블 구조를 확인하는 준비 도구입니다.

SQL 실습을 시작할 때는 아래 흐름을 습관처럼 따라가면 좋습니다. 이 흐름은 SCOTT, HR, 프로젝트용 APP_USER 계정 모두에 적용됩니다.

SCOTT, HR, SYSTEM,

4. 현재 사용자 확인: SHOW USER

SHOW USER는 현재 Oracle 세션에서 어떤 사용자 계정으로 접속했는지 확인하는 명령어입니다. SQL 실습을 시작할 때 가장 먼저 실행하는 습관을 들이면 좋습니다.

기본 사용

SHOW USER;

실행 결과가 USER is "SCOTT"이면 현재 SCOTT 계정으로 접속한 것입니다. USER is "HR"이면 HR 계정으로 접속한 것입니다.

왜 필요한가?

SCOTT 계정에는 EMP, DEPT가 있고, HR 계정에는 EMPLOYEES, DEPARTMENTS가 있습니다. 현재 계정을 확인하지 않으면 테이블명을 잘못 사용하기 쉽습니다.

5. 현재 사용자 테이블 목록 확인: USER_TABLES

USER_TABLES는 현재 접속한 사용자가 직접 소유한 테이블 목록을 보여주는 데이터 딕셔너리 뷰입니다. SQL 문제를 풀기 전, 현재 계정에 어떤 테이블이 있는지 확인할 때 사용합니다.

현재 사용자 테이블 목록 조회

SELECT table_name
FROM user_tables
ORDER BY table_name;

SCOTT 계정에서는 EMP, DEPT, BONUS, SALGRADE 같은 테이블을 기대할 수 있습니다. HR 계정에서는 EMPLOYEES, DEPARTMENTS, JOBS 등이 보일 수 있습니다.

특정 테이블 존재 여부 확인

SELECT table_name
FROM user_tables
WHERE table_name = 'USERS';

Spring Boot 프로젝트에서 USERS 테이블이 실제 Oracle에 있는지 확인할 때 사용할 수 있습니다. Oracle은 일반적으로 테이블명을 대문자로 저장하므로 대문자로 확인하는 것이 안전합니다.

6. 권한 있는 테이블 확인: ALL_TABLES

ALL_TABLES는 현재 사용자가 소유하지 않았더라도, 접근 권한이 있는 다른 스키마의 테이블까지 확인할 수 있는 데이터 딕셔너리 뷰입니다.

USER_TABLES

SELECT table_name
FROM user_tables;

현재 사용자가 직접 소유한 테이블만 보여줍니다.
즉 “내 계정의 테이블”을 확인하는 데 사용합니다.

ALL_TABLES

SELECT owner, table_name
FROM all_tables
ORDER BY owner, table_name;

현재 사용자가 접근 권한을 가진 모든 테이블을 보여줍니다.
다른 스키마의 테이블도 권한이 있다면 표시될 수 있습니다.

그림처럼 이해하기

현재 사용자: APP_USER

USER_TABLES
└─ APP_USER가 직접 소유한 테이블
    ├─ USERS
    ├─ BOARD
    └─ PROPERTY

ALL_TABLES
└─ APP_USER가 접근 가능한 테이블
    ├─ APP_USER.USERS
    ├─ APP_USER.BOARD
    ├─ COMMON.CODE_TABLE
    └─ SCOTT.EMP

7. 간단 객체 조회: SELECT * FROM tab

SELECT * FROM tab;은 현재 사용자의 테이블이나 뷰 같은 객체를 간단히 확인할 때 사용할 수 있습니다.
다만 표준 SQL은 아니므로, 정확한 테이블 확인에는 USER_TABLES를 사용하는 습관이 좋습니다.

간단 객체 목록 확인

SELECT *
FROM tab;

현재 계정의 객체를 빠르게 확인할 수 있습니다. 하지만 학습과 실무에서는 USER_TABLES, USER_VIEWS 같은 데이터 딕셔너리 뷰를 더 명확하게 사용하는 것이 좋습니다.

8. 테이블 구조 확인: DESC

DESC는 테이블의 컬럼명, NULL 허용 여부, 자료형을 확인하는 명령어입니다. SQL을 작성하기 전에 컬럼명을 정확히 확인할 때 사용합니다.

SCOTT 테이블 구조 확인

DESC emp;

DESC dept;

SCOTT의 EMP 테이블에서는 EMPNO, ENAME, JOB, SAL, DEPTNO 같은 컬럼을 확인할 수 있습니다.

HR 테이블 구조 확인

DESC employees;

DESC departments;

HR의 EMPLOYEES 테이블에서는 EMPLOYEE_ID, FIRST_NAME, LAST_NAME, SALARY, DEPARTMENT_ID 같은 컬럼을 확인할 수 있습니다.

컬럼명 착각 주의

SCOTT의 급여 컬럼은 SAL이고, HR의 급여 컬럼은 SALARY입니다. 이런 차이는 DESC를 먼저 확인하면 쉽게 예방할 수 있습니다.

9. 로그인 사용자 변경: CONN

CONN은 SQL Plus에서 현재 접속을 다른 사용자로 바꿀 때 사용하는 명령어입니다.
SQL Developer에서는 보통 접속 목록을 따로 만들어두지만, SQL Plus를 사용할 때는 CONN을 자주 사용합니다.

SCOTT 계정으로 접속 전환

CONN scott/tiger@localhost:1521/XEPDB1;

SHOW USER;

접속 후 SHOW USER를 실행해서 현재 사용자가 SCOTT으로 바뀌었는지 확인합니다.

HR 계정으로 접속 전환

CONN hr/hr@localhost:1521/XEPDB1;

SHOW USER;

접속 후 현재 사용자가 HR인지 확인합니다. 포트가 기본값 1521인 경우 환경에 따라 생략 가능한 경우도 있습니다.

10. 관리자 계정 접속과 주의점

Oracle에는 일반 사용자 계정뿐 아니라 관리자 계정도 있습니다. SYSTEM은 관리자 계정이고, SYSDBA는 더 강한 최고 관리자 권한입니다.
일반 SQL 실습은 관리자 계정보다 SCOTT, HR, APP_USER 같은 일반 계정에서 진행하는 것이 안전합니다.

SYSTEM 접속 예시

sqlplus system/비밀번호@localhost:1521/XEPDB1

사용자 생성, 계정 잠금 해제, 권한 부여 등 관리자 작업에 사용합니다.

SYSDBA 접속 예시

sqlplus / nolog

CONN / AS SYSDBA;

매우 강한 권한을 가진 접속 방식입니다.
학습 초반에는 필요한 경우에만 신중하게 사용해야 합니다.

11. ED 명령어와 SQL 파일 실행

ED 명령어는 SQL Plus에서 SQL 파일을 편집기로 열어 작성할 때 사용합니다. 긴 SQL이나 PL/SQL 블록을 작성할 때 유용합니다.

01 파일 열기 
ED MYBOARD
02 SQL 작성 후 저장 
--메모장에서 SQL 작성
--PL/SQL 블록 마지막 / 필요
03 파일 실행 
@MYBOARD
04 다시 수정 
ED MYBOARD

SQL Developer를 주로 사용할 경우 당장 자주 쓰지는 않지만, SQL Plus 환경에서 스크립트를 실행하거나 PL/SQL을 실습할 때 알아두면 좋습니다.

12. 실습 문제 연결

이번 단원은 실습 문제를 직접 푸는 단원은 아니지만, 모든 실습 문제를 풀기 전에 반드시 거쳐야 할 확인 루틴을 학습하는 단원입니다.

SCOTT 문제 풀이 전

EMP 테이블을 사용하는 문제를 풀기 전에는 현재 계정이 SCOTT인지,
EMP 테이블이 있는지, EMP 컬럼 구조가 어떻게 생겼는지 확인합니다. 

SHOW USER;

SELECT table_name
FROM user_tables
ORDER BY table_name;

DESC emp;

HR 문제 풀이 전

EMPLOYEES, DEPARTMENTS, LOCATIONS 테이블을 사용하는
JOIN 문제를 풀기 전에는 HR 계정과 테이블 구조를 먼저 확인합니다. 

SHOW USER;

SELECT table_name
FROM user_tables
ORDER BY table_name;

DESC employees;
DESC departments;
DESC locations;

13. Java / Spring 프로젝트와의 연결

Oracle 기본 명령어는 단순 SQL 학습용이 아니라, Java / Spring Boot 프로젝트에서 DB 연결 오류를 점검할 때도 그대로 사용됩니다.

Spring Boot datasource

spring.datasource.url=jdbc:oracle:thin:@localhost:1521/XEPDB1
spring.datasource.username=app_user
spring.datasource.password=app_password

Spring Boot가 접속하는 계정이 app_user라면,
SQL Developer에서도 같은 계정으로 접속해 테이블을 확인해야 합니다.

테이블 존재 확인

SHOW USER;

SELECT table_name
FROM user_tables
WHERE table_name = 'USERS';

DESC users;

JPA Entity나 MyBatis Mapper가 참조하는 테이블과 컬럼이
실제 Oracle에 존재하는지 확인합니다.

프로젝트 점검 루틴: Spring Boot에서 DB 오류가 발생하면 먼저 SQL Developer에서 같은 계정으로 접속한 뒤 SHOW USER, USER_TABLES, DESC로 실제 DB 상태를 확인합니다.

14. 자주 하는 실수

  • 현재 계정을 확인하지 않고 SQL을 실행함 EMP 테이블이 없다는 오류가 나면 문법 문제가 아니라 계정이 다를 수 있습니다. 먼저 SHOW USER를 실행해야 합니다.
  • USER_TABLES와 ALL_TABLES를 같은 것으로 생각함 USER_TABLES는 내가 소유한 테이블, ALL_TABLES는 내가 접근 가능한 테이블입니다.
  • SELECT * FROM tab을 표준 SQL로 생각함 TAB은 Oracle에서 간단 객체 조회용으로 사용할 수 있지만, 정확한 확인에는 USER_TABLES를 사용하는 것이 좋습니다.
  • DESC 없이 컬럼명을 추측함 SCOTT의 급여 컬럼은 SAL, HR의 급여 컬럼은 SALARY입니다. 컬럼명을 모를 때는 DESC를 먼저 확인합니다.
  • 관리자 계정으로 모든 실습을 진행함 SYSTEM과 SYSDBA는 관리자 작업용입니다. 일반 SQL 실습은 SCOTT, HR, APP_USER 같은 일반 계정에서 진행하는 것이 좋습니다.
  • 테이블이 없으면 바로 재설치를 생각함 테이블이 보이지 않을 때는 먼저 현재 계정, 테이블 목록, 접근 권한을 확인해야 합니다.

15. 면접 포인트

Q1. SHOW USER는 어떤 명령어인가요?

현재 Oracle 세션에서 어떤 사용자 계정으로 접속했는지 확인하는 명령어입니다. SQL 실습이나 운영 작업 전 현재 계정을 확인하기 위해 사용합니다.

Q2. USER_TABLES와 ALL_TABLES의 차이는 무엇인가요?

USER_TABLES는 현재 접속한 사용자가 소유한 테이블 목록을 보여줍니다.
ALL_TABLES는 현재 사용자가 접근 권한을 가진 모든 테이블을 보여주며, 다른 스키마의 테이블도 포함될 수 있습니다.

Q3. DESC 명령어는 언제 사용하나요?

DESC 테이블명은 테이블의 구조를 확인할 때 사용합니다. 컬럼명, NULL 허용 여부, 자료형을 볼 수 있어 SQL 작성 전에 컬럼명을 확인할 때 유용합니다.

Q4. SQL Developer와 SQL Plus의 차이는 무엇인가요?

SQL Developer는 GUI 기반의 Oracle 접속 및 SQL 실행 도구이고, SQL Plus는 명령줄 기반의 Oracle 접속 도구입니다.
SQL Developer는 결과를 표 형태로 보기 편하고, SQL Plus는 빠른 접속이나 스크립트 실행에 유용합니다.

Q5. CONN 명령어는 어떤 용도로 사용하나요?

CONN은 SQL*Plus에서 현재 접속을 다른 사용자 계정으로 바꿀 때 사용하는 명령어입니다. 예를 들어 CONN scott/tiger@localhost:1521/XEPDB1처럼 사용합니다.

Q6. Spring Boot 프로젝트에서 USER_TABLES를 확인하는 이유는 무엇인가요?

JPA Entity나 MyBatis Mapper가 실제 Oracle 테이블과 연결되기 때문에, 해당 테이블이 현재 접속 계정에 존재하는지 확인해야 합니다.
예를 들어 USERS 테이블이 있는지 확인할 수 있습니다.

16. 이번 단원 핵심 정리

Oracle 기본 명령어는 SQL 실습 전 현재 상태를 확인하는 도구입니다.
SHOW USER는 현재 접속 계정을 확인하고, USER_TABLES는 현재 사용자가 소유한 테이블, ALL_TABLES는 접근 권한이 있는 테이블을 확인합니다.
DESC는 테이블의 컬럼명과 자료형을 확인하며, CONN은 SQL*Plus에서 접속 사용자를 바꿀 때 사용합니다.
앞으로 SQL 문제를 풀기 전에는 SHOW USER, USER_TABLES, DESC 순서로 확인하는 습관을 가져야 합니다.
