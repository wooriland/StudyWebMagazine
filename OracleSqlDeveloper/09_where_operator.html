Overview

1. WHERE 조건절이란?

WHERE는 테이블의 모든 행 중에서 조건에 맞는 행만 골라내는 절입니다. SELECT가 어떤 컬럼을 보여줄지 결정한다면, WHERE는 어떤 행을 가져올지 결정합니다.

FROM 어느 테이블에서 데이터를 가져올지 결정합니다.
WHERE 조건에 맞는 행만 남깁니다.
SELECT 남은 행에서 어떤 컬럼을 출력할지 결정합니다.

한 줄 정리

WHERE는 행을 고르는 필터입니다. SELECT는 세로줄인 컬럼을 선택하고, WHERE는 가로줄인 행을 조건으로 걸러냅니다.

Syntax

2. WHERE 기본 문법

WHERE 조건은 보통 컬럼명 연산자 값 형태로 작성합니다. 숫자 조건은 작은따옴표 없이 쓰고, 문자 조건은 작은따옴표로 감싸야 합니다.

2-1. 기본 구조

SELECT 컬럼명
FROM 테이블명
WHERE 조건;

2-2. 숫자 조건

SELECT ENAME, SAL
FROM EMP
WHERE SAL >= 3000;

SAL 값이 3000 이상인 행만 조회합니다.

2-3. 문자 조건

SELECT ENAME, JOB
FROM EMP
WHERE JOB = 'MANAGER';

문자값인 MANAGER는 작은따옴표로 감싸야 합니다.

2-4. 날짜 조건

SELECT ENAME, HIREDATE
FROM EMP
WHERE HIREDATE >= TO_DATE('1981-01-01', 'YYYY-MM-DD');

날짜 조건은 환경별 날짜 형식 차이를 줄이기 위해 TO_DATE를 사용하는 것이 안전합니다.

Visual

3. 그림처럼 이해하기

WHERE는 전체 테이블을 조건이라는 체에 걸러서 필요한 행만 남기는 과정입니다.

EMPNO ENAME JOB SAL DEPTNO
7369 SMITH CLERK 800 20
7566 JONES MANAGER 2975 20
7698 BLAKE MANAGER 2850 30
7839 KING PRESIDENT 5000 10

WHERE JOB = 'MANAGER'의 흐름

FROM EMP
→ EMP 테이블 전체를 가져온다.

WHERE JOB = 'MANAGER'
→ JOB이 MANAGER인 행만 남긴다.

SELECT ENAME, JOB, SAL
→ 남은 행에서 ENAME, JOB, SAL 컬럼만 출력한다.
Comparison

4. 비교 연산자

비교 연산자는 컬럼 값과 특정 값을 비교할 때 사용합니다. Oracle에서는 같지 않다는 의미로 !=와 <>를 모두 사용할 수 있습니다.

=

같다

WHERE JOB = 'MANAGER'

>

크다

WHERE SAL > 3000

>=

크거나 같다

WHERE SAL >= 3000

<

작다

WHERE SAL < 1500

<=

작거나 같다

WHERE SAL <= 1500

!= / <>

같지 않다

WHERE JOB != 'SALESMAN'

비교 연산자 실습

SELECT ENAME, JOB, SAL
FROM EMP
WHERE JOB = 'MANAGER';

SELECT ENAME, SAL
FROM EMP
WHERE SAL >= 3000;

SELECT ENAME, JOB
FROM EMP
WHERE JOB != 'SALESMAN';

SELECT ENAME, JOB
FROM EMP
WHERE JOB <> 'SALESMAN';
Logic

5. 논리 연산자 AND / OR / NOT

논리 연산자는 여러 조건을 조합할 때 사용합니다. AND는 모든 조건을 만족해야 하고, OR은 하나라도 만족하면 되며, NOT은 조건을 반대로 뒤집습니다.

AND

모든 조건을 만족해야 합니다.

SELECT EMPNO, ENAME, JOB, SAL
FROM EMP
WHERE JOB = 'MANAGER'
  AND SAL >= 2500;

OR

조건 중 하나라도 만족하면 됩니다.

SELECT ENAME, DEPTNO
FROM EMP
WHERE DEPTNO = 10
   OR DEPTNO = 20;

NOT

조건을 부정합니다.

SELECT ENAME, SAL
FROM EMP
WHERE NOT SAL >= 2000;

괄호 사용

AND와 OR가 섞이면 괄호로 의도를 명확히 합니다.

SELECT ENAME, DEPTNO, SAL
FROM EMP
WHERE (DEPTNO = 10 OR DEPTNO = 20)
  AND SAL >= 3000;
BETWEEN

6. BETWEEN 연산자

BETWEEN은 특정 범위 안에 있는 값을 조회할 때 사용합니다. BETWEEN A AND B는 A 이상 B 이하이며, 양쪽 끝값을 모두 포함합니다.

BETWEEN의 의미

WHERE SAL BETWEEN 1500 AND 3000

같은 의미:

WHERE SAL >= 1500
  AND SAL <= 3000

6-1. 급여 범위 조회

SELECT ENAME, SAL
FROM EMP
WHERE SAL BETWEEN 1500 AND 3000;

6-2. 실습 문제 연결

SELECT EMPNO, ENAME, JOB, SAL
FROM EMP
WHERE JOB = 'MANAGER'
  AND SAL BETWEEN 2500 AND 3000;

직무가 MANAGER이고 급여가 2500 이상 3000 이하인 사원을 조회합니다.

IN

7. IN / NOT IN 연산자

IN은 여러 값 중 하나에 해당하는 행을 조회할 때 사용합니다. OR 조건을 여러 번 쓰는 것보다 간결하고 읽기 쉽습니다.

IN의 의미

WHERE DEPTNO IN (10, 20)

같은 의미:

WHERE DEPTNO = 10
   OR DEPTNO = 20

7-1. 숫자 목록 조회

SELECT ENAME, DEPTNO
FROM EMP
WHERE DEPTNO IN (10, 20);

7-2. 문자 목록 조회

SELECT ENAME, JOB
FROM EMP
WHERE JOB IN ('MANAGER', 'CLERK');

7-3. NOT IN

SELECT ENAME, DEPTNO
FROM EMP
WHERE DEPTNO NOT IN (10, 20);

7-4. 실습 문제 연결

SELECT *
FROM EMP
WHERE SAL < 2000
  AND DEPTNO IN (20, 30);

급여가 2000 미만이고 부서번호가 20 또는 30인 사원을 조회합니다.

LIKE

8. LIKE 연산자

LIKE는 문자열 패턴 검색에 사용합니다. 정확히 같은 값을 찾는 =와 달리, 시작 문자, 끝 문자, 포함 문자, 특정 위치 문자를 검색할 수 있습니다.

%

0개 이상의 여러 글자

LIKE 'A%'
LIKE '%A'
LIKE '%A%'

_

정확히 한 글자

LIKE '_A%'
LIKE '__A%'

8-1. A로 시작하는 이름

SELECT ENAME
FROM EMP
WHERE ENAME LIKE 'A%';

8-2. A로 끝나는 이름

SELECT ENAME
FROM EMP
WHERE ENAME LIKE '%A';

8-3. A가 포함된 이름

SELECT ENAME
FROM EMP
WHERE ENAME LIKE '%A%';

8-4. 두 번째 글자가 A인 이름

SELECT ENAME
FROM EMP
WHERE ENAME LIKE '_A%';

8-5. HR 계정 예제: S로 시작하는 LAST_NAME

SELECT FIRST_NAME || ' ' || LAST_NAME AS "성 명",
       COMMISSION_PCT,
       SALARY
FROM EMPLOYEES
WHERE COMMISSION_PCT IS NOT NULL
  AND SALARY >= 5000
  AND LAST_NAME LIKE 'S%';

8-6. HR 계정 예제: 두 번째 글자가 a인 LAST_NAME

SELECT FIRST_NAME || ' ' || LAST_NAME AS "성 명",
       COMMISSION_PCT,
       SALARY
FROM EMPLOYEES
WHERE COMMISSION_PCT IS NOT NULL
  AND SALARY >= 5000
  AND LAST_NAME LIKE '_a%';
NULL

9. IS NULL / IS NOT NULL

NULL은 값이 없다는 뜻입니다. 0이나 공백과 다르며, 일반 값처럼 =로 비교하지 않습니다. NULL을 찾을 때는 반드시 IS NULL, NULL이 아닌 값을 찾을 때는 IS NOT NULL을 사용합니다.

NULL인 값 조회

SELECT ENAME, COMM
FROM EMP
WHERE COMM IS NULL;

NULL이 아닌 값 조회

SELECT ENAME, COMM
FROM EMP
WHERE COMM IS NOT NULL;

잘못된 NULL 비교

-- 잘못된 예
SELECT ENAME, COMM
FROM EMP
WHERE COMM = NULL;

실습 문제 연결

SELECT *
FROM EMP
WHERE SAL >= 1500
  AND COMM IS NOT NULL;

커미션을 받는 직원 중 급여가 1500 이상인 직원을 조회합니다.

Concat

10. 문자열 연결 연산자 ||

Oracle에서 ||는 문자열을 연결할 때 사용합니다. 이름과 성을 합치거나, 조회 결과를 문장처럼 출력할 때 유용합니다.

10-1. 사원 소개 문장 만들기

SELECT ENAME || '의 직무는 ' || JOB || '입니다.' AS "사원 소개"
FROM EMP;

10-2. HR 계정 이름 연결

SELECT FIRST_NAME || ' ' || LAST_NAME AS "성 명"
FROM EMPLOYEES;
Priority

11. 연산자 우선순위와 괄호

AND와 OR를 함께 사용할 때는 의도와 다르게 조건이 해석될 수 있습니다. 복잡한 조건은 괄호를 사용해서 의미를 명확히 표현하는 것이 좋습니다.

의도가 불분명한 조건

SELECT ENAME, DEPTNO, SAL
FROM EMP
WHERE DEPTNO = 10 OR DEPTNO = 20
  AND SAL >= 3000;

괄호로 명확히 표현

SELECT ENAME, DEPTNO, SAL
FROM EMP
WHERE (DEPTNO = 10 OR DEPTNO = 20)
  AND SAL >= 3000;

IN으로 간단하게 표현

SELECT ENAME, DEPTNO, SAL
FROM EMP
WHERE DEPTNO IN (10, 20)
  AND SAL >= 3000;

실무 습관

조건이 길어질수록 괄호를 사용해 의도를 명확히 표시해야 합니다.
SQL은 작동하는 것보다 정확히 의도대로 작동하는 것이 더 중요합니다.

Practice

12. 실습 문제 연결

실습 문제 교안의 초반 문제들은 WHERE 조건절과 SQL 연산자를 직접 사용하는 문제입니다. 문제를 읽을 때는 출력 컬럼, 테이블명, 조건을 분리해서 확인하면 좋습니다.

문제 1. BETWEEN

SELECT EMPNO, ENAME, JOB, SAL
FROM EMP
WHERE JOB = 'MANAGER'
  AND SAL BETWEEN 2500 AND 3000;

직무가 MANAGER이고 급여가 2500 이상 3000 이하인 사원을 조회합니다.

문제 2. != 연산자

SELECT ENAME, DEPTNO, SAL, JOB
FROM EMP
WHERE DEPTNO = 30
  AND SAL >= 1500
  AND JOB != 'SALESMAN';

부서번호가 30이고 급여가 1500 이상이며 SALESMAN이 아닌 직원을 조회합니다.

문제 3. IN 연산자

SELECT *
FROM EMP
WHERE SAL < 2000
  AND DEPTNO IN (20, 30);

급여가 2000 미만이고 부서번호가 20 또는 30인 직원을 조회합니다.

문제 4. IS NOT NULL

SELECT *
FROM EMP
WHERE SAL >= 1500
  AND COMM IS NOT NULL;

커미션을 받는 직원 중 급여가 1500 이상인 직원을 조회합니다.

문제 5. LIKE

SELECT *
FROM EMP
WHERE ENAME LIKE '%A%';

이름에 A가 포함된 사원을 조회합니다.

문제 읽는 순서

1. 어떤 테이블에서?
   → EMP

2. 어떤 컬럼을 출력?
   → EMPNO, ENAME, JOB, SAL

3. 어떤 조건인가?
   → JOB = 'MANAGER'
   → SAL BETWEEN 2500 AND 3000
Project

13. Java / Spring / Oracle 프로젝트 연결

WHERE 조건절은 Java와 Spring 프로젝트의 거의 모든 조회 기능과 연결됩니다. 로그인, 상세 조회, 게시글 검색, 매물 필터, 추천 로그 분석 모두 WHERE 조건에서 출발합니다.

로그인

입력한 아이디에 해당하는 회원을 조회합니다.

SELECT USER_ID, LOGIN_ID, PASSWORD_HASH, USER_TYPE
FROM USERS
WHERE LOGIN_ID = ?;

회원 상세 조회

특정 회원 한 명을 PK 기준으로 조회합니다.

SELECT USER_ID, LOGIN_ID, EMAIL, USER_TYPE
FROM USERS
WHERE USER_ID = ?;

게시글 검색

제목에 검색어가 포함된 게시글을 찾습니다.

SELECT BOARD_ID, TITLE, WRITER_ID, CREATED_AT
FROM BOARD
WHERE TITLE LIKE '%' || ? || '%';

게시글 상세 조회

목록에서 클릭한 게시글 ID로 상세 데이터를 조회합니다.

SELECT BOARD_ID, TITLE, CONTENT, WRITER_ID, CREATED_AT
FROM BOARD
WHERE BOARD_ID = ?;

매물 필터 검색

가격 범위, 매물 유형, 거래 유형을 조합해 조회합니다.

SELECT PROPERTY_ID, TITLE, DEAL_AMOUNT, AREA_SQM
FROM PROPERTY
WHERE DEAL_AMOUNT BETWEEN 300000000 AND 700000000
  AND PROPERTY_TYPE IN ('APARTMENT', 'OFFICETEL')
  AND DEAL_TYPE = 'SALE';

추천 로그 분석

특정 사용자의 관심도 높은 행동 데이터를 추출합니다.

SELECT USER_ID, PROPERTY_ID, STAY_SECONDS, VIEWED_AT
FROM PROPERTY_VIEW_LOG
WHERE USER_ID = ?
  AND STAY_SECONDS >= 60;

삭제되지 않은 데이터 조회

논리 삭제 구조에서 아직 삭제되지 않은 사용자만 조회합니다.

SELECT USER_ID, LOGIN_ID, EMAIL
FROM USERS
WHERE DELETED_AT IS NULL;

Spring Data JPA 메서드

JPA 메서드 이름도 WHERE 조건으로 변환될 수 있습니다.

Optional<UsersEntity> findByLoginId(String loginId);

List<PropertyEntity> findByDealAmountBetween(
    Long minAmount,
    Long maxAmount
);
Mistake

14. 자주 하는 실수

  • 실수 1. 문자값에 작은따옴표를 빼먹는다. WHERE JOB = 'MANAGER'처럼 문자 조건은 작은따옴표로 감싸야 합니다.
  • 실수 2. NULL을 = 로 비교한다. COMM = NULL이 아니라 COMM IS NULL을 사용해야 합니다.
  • 실수 3. BETWEEN이 끝값을 제외한다고 생각한다. BETWEEN 1500 AND 3000은 1500과 3000을 모두 포함합니다.
  • 실수 4. LIKE의 %와 _를 혼동한다. %는 0개 이상의 여러 글자, _는 정확히 한 글자입니다.
  • 실수 5. AND와 OR를 괄호 없이 섞는다. 복합 조건은 괄호로 의도를 명확히 표현해야 합니다.
  • 실수 6. IN과 = 를 혼동한다. 여러 값 중 하나를 비교할 때는 DEPTNO IN (10, 20)을 사용합니다.
  • 실수 7. NOT 조건을 반대로 해석한다. NOT SAL >= 2000은 2000 이상이 아니라 2000 미만이라는 뜻입니다.
  • 실수 8. IS NOT NULL과 != NULL을 혼동한다. NULL이 아닌 값을 찾을 때는 IS NOT NULL을 사용해야 합니다.
Interview

15. 면접 포인트

Q1. WHERE절은 무엇인가요?

WHERE절은 SELECT 문에서 조건에 맞는 행만 조회하기 위해 사용하는 조건절입니다. SELECT가 어떤 컬럼을 출력할지 결정한다면, WHERE는 어떤 행을 가져올지 결정합니다.

Q2. 비교 연산자에는 어떤 것들이 있나요?

SQL의 비교 연산자에는 =, >, >=, <, <=, <>, !=가 있습니다. =는 같다, <>와 !=는 같지 않다는 의미입니다.

Q3. BETWEEN은 어떤 의미인가요?

BETWEEN A AND B는 A 이상 B 이하의 범위를 의미합니다. 양쪽 경계값을 모두 포함하며, SAL BETWEEN 1500 AND 3000은 SAL >= 1500 AND SAL <= 3000과 같습니다.

Q4. IN과 OR의 관계를 설명해 주세요.

IN은 여러 값 중 하나에 해당하는 행을 조회할 때 사용합니다. DEPTNO IN (10, 20)은 DEPTNO = 10 OR DEPTNO = 20과 같은 의미입니다.

Q5. LIKE에서 %와 _의 차이는 무엇인가요?

%는 0개 이상의 여러 문자열을 의미하고, _는 정확히 한 글자를 의미합니다. LIKE 'A%'는 A로 시작하는 문자열, LIKE '_A%'는 두 번째 글자가 A인 문자열을 찾습니다.

Q6. NULL은 어떻게 비교하나요?

NULL은 값이 없다는 의미이므로 = 또는 !=로 비교하지 않습니다. NULL인 값을 찾을 때는 IS NULL, NULL이 아닌 값을 찾을 때는 IS NOT NULL을 사용합니다.

Q7. AND와 OR를 함께 사용할 때 주의할 점은 무엇인가요?

AND와 OR를 함께 사용하면 우선순위 때문에 의도와 다르게 해석될 수 있습니다. 복합 조건에서는 괄호를 사용해 의도를 명확히 표현하는 것이 좋습니다.

Q8. 문자열 연결 연산자는 무엇인가요?

Oracle에서 문자열 연결 연산자는 ||입니다. FIRST_NAME || ' ' || LAST_NAME처럼 사용하면 여러 문자열을 하나로 연결할 수 있습니다.

→ 0개 이상의 문자열

_
→ 정확히 한 글자

IS NULL
→ NULL 값 조회

IS NOT NULL
→ NULL이 아닌 값 조회

||
→ 문자열 연결

읽는 순서

1. FROM을 보고 어느 테이블인지 확인한다.
2. WHERE를 보고 어떤 행을 남기는지 확인한다.
3. SELECT를 보고 어떤 컬럼을 출력하는지 확인한다.
4. AND / OR / NOT으로 조건 조합을 확인한다.
5. BETWEEN, IN, LIKE, IS NULL의 의미를 해석한다.

학습 메모

오늘의 핵심

WHERE는 원하는 행만 남기는 조건절입니다. SELECT가 컬럼을 고른다면, WHERE는 행을 고릅니다.

기본 문법

SELECT 컬럼명
FROM 테이블명
WHERE 조건;

연산자 암기

  • 비교 =, !=, <>, >, >=, <, <=
  • 논리 AND, OR, NOT
  • 범위 BETWEEN A AND B
  • 목록 IN, NOT IN
  • 패턴 LIKE, %, _
  • NULL IS NULL, IS NOT NULL
  • 문자열 연결 ||

실습 확인 순서

  1. SHOW USER로 현재 계정 확인
  2. DESC EMP로 컬럼명 확인
  3. 비교 연산자 실행
  4. AND / OR / NOT 실행
  5. BETWEEN과 비교 연산자 결과 비교
  6. IN과 OR 결과 비교
  7. LIKE의 %, _ 차이 확인
  8. IS NULL과 IS NOT NULL 결과 확인

주의할 점

NULL은 = 로 비교하지 않습니다. LIKE의 %와 _를 구분해야 하며, AND와 OR가 함께 나오면 괄호를 사용하는 습관을 들이세요.

프로젝트 연결

로그인은 WHERE LOGIN_ID = ?, 검색은 LIKE, 가격 필터는 BETWEEN, 체크박스 필터는 IN, 삭제되지 않은 데이터 조회는 IS NULL과 연결됩니다.
