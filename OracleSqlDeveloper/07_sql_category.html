Oracle SQL Developer - Chapter 07

SQL의 종류

SQL은 데이터베이스에 명령을 내리는 언어입니다.
이번 단원에서는 SQL을 목적에 따라 DDL, DML, DCL, TCL로 구분하고,
각 명령이 Oracle DB와 Java / Spring 프로젝트에서 어떻게 사용되는지 정리합니다.

학습 목표 SQL을 구조, 데이터, 권한, 트랜잭션 기준으로 분류한다.
실습 방향 SQL Developer에서
CREATE, INSERT, UPDATE, COMMIT 등을 직접 실행한다.
프로젝트 연결 회원가입, 로그인, 게시판 CRUD, @Transactional
흐름과 연결한다.
Overview

1. SQL은 왜 종류를 나누어 공부할까?

SQL은 데이터베이스와 대화하기 위한 언어입니다. 하지만 모든 SQL이 같은 목적을 가지는 것은 아닙니다.
어떤 SQL은 테이블 구조를 만들고, 어떤 SQL은 테이블 안의 데이터를 조회하거나 수정하며, 어떤 SQL은 사용자 권한을 제어하고, 어떤 SQL은 작업을 확정하거나 취소합니다.

구조를 다룬다 테이블 생성, 컬럼 추가, 테이블 삭제
데이터를 다룬다 조회, 입력, 수정, 삭제
권한을 다룬다 사용자에게 권한 부여 또는 회수

한 줄 정리

SQL의 종류를 구분하는 이유는 명령의 목적과 위험도가 다르기 때문입니다.
구조를 바꾸는 SQL, 데이터를 바꾸는 SQL, 권한을 바꾸는 SQL, 트랜잭션을 제어하는 SQL은 실행 전 확인해야 할 포인트가 서로 다릅니다.

Category

2. SQL의 전체 분류

SQL은 목적에 따라 크게 DDL, DML, DCL, TCL로 나누어 이해할 수 있습니다.
교안에서는 DDL, DML, DCL을 중심으로 설명하지만, 실제 프로젝트에서는 COMMIT과 ROLLBACK을 다루는 TCL도 함께 이해해야 합니다.

DDL

Data Definition Language

데이터베이스 객체의 구조를
생성, 변경, 삭제하는 SQL입니다.

  • CREATE
  • ALTER
  • DROP
  • TRUNCATE
  • RENAME

DML

Data Manipulation Language

테이블 안의 실제 데이터를
조회, 입력, 수정, 삭제하는
SQL입니다.

  • SELECT
  • INSERT
  • UPDATE
  • DELETE

DCL

Data Control Language

사용자에게 권한을 부여하거나
회수하는 SQL입니다.

  • GRANT
  • REVOKE

TCL

Transaction Control Language

데이터 변경 작업을 확정하거나
되돌리는 SQL입니다.

  • COMMIT
  • ROLLBACK
  • SAVEPOINT
Visual

3. 그림처럼 이해하기

SQL의 종류는 건물을 관리하는 일에 비유하면 이해하기 쉽습니다.
DDL은 건물 구조를 만들고 바꾸는 일, DML은 방 안의 물건을 다루는 일, DCL은 출입 권한을 관리하는 일, TCL은 작업을 확정하거나 취소하는 일입니다.

DDL
건물 설계 방 추가 철거

테이블 생성, 컬럼 추가, 테이블 삭제처럼 데이터베이스 구조를 다룹니다.

DML
물건 넣기 찾기 수정 삭제

테이블 안의 데이터를 조회, 입력, 수정, 삭제합니다.

DCL
권한 부여 권한 회수

사용자가 어떤 테이블을 조회하거나 수정할 수 있는지 제어합니다.

TCL
확정 취소 중간 저장

INSERT, UPDATE, DELETE 같은 변경 작업을 확정하거나 되돌립니다.

테이블 기준으로 보는 SQL 종류

테이블 자체를 만든다        → CREATE TABLE → DDL
테이블 구조를 바꾼다        → ALTER TABLE  → DDL
테이블 자체를 삭제한다      → DROP TABLE   → DDL

테이블 안의 데이터를 조회한다 → SELECT       → DML
테이블 안에 데이터를 넣는다   → INSERT       → DML
테이블 안의 데이터를 수정한다 → UPDATE       → DML
테이블 안의 데이터를 삭제한다 → DELETE       → DML

사용자에게 권한을 준다      → GRANT        → DCL
사용자 권한을 회수한다      → REVOKE       → DCL

변경 사항을 확정한다        → COMMIT       → TCL
변경 사항을 취소한다        → ROLLBACK     → TCL
DDL

4. DDL - 데이터베이스 구조를 다루는 SQL

DDL은 Data Definition Language의 약자로, 데이터베이스 객체의 구조를 정의하는 SQL입니다. 테이블을 생성하거나, 컬럼을 추가하거나, 테이블 자체를 삭제할 때 사용합니다.

4-1. CREATE TABLE - 테이블 생성

CREATE TABLE SQL_CATEGORY_MEMBER (
    MEMBER_ID NUMBER(5),
    MEMBER_NAME VARCHAR2(30),
    EMAIL VARCHAR2(100),
    CREATED_AT DATE
);

데이터를 넣은 것이 아니라, 데이터가 들어갈 테이블 구조를 만든 것입니다. 따라서 이 SQL은 DDL입니다.

4-2. ALTER TABLE - 컬럼 추가

ALTER TABLE SQL_CATEGORY_MEMBER
ADD PHONE VARCHAR2(20);

기존 테이블에 PHONE 컬럼을 추가합니다. 테이블 구조가 변경되므로 DDL입니다.

4-3. DROP TABLE - 테이블 삭제

DROP TABLE SQL_CATEGORY_MEMBER;

테이블 안의 데이터만 삭제하는 것이 아니라 테이블 자체를 삭제합니다. 실행 전 반드시 대상 테이블을 확인해야 합니다.

4-4. TRUNCATE TABLE - 전체 데이터 삭제

TRUNCATE TABLE SQL_CATEGORY_MEMBER;

테이블 구조는 남기고 데이터만 전체 삭제합니다. DELETE와 비슷해 보이지만 DDL 성격이 강하며 일반적으로 ROLLBACK이 어렵기 때문에 주의해야 합니다.

DML

5. DML - 테이블 안의 데이터를 다루는 SQL

DML은 Data Manipulation Language의 약자로, 테이블 안의 실제 데이터를 조회, 입력, 수정, 삭제하는 SQL입니다. 개발자가 기능 구현 중 가장 자주 사용하는 SQL 분류입니다.

5-1. SELECT - 데이터 조회

SELECT MEMBER_ID, MEMBER_NAME, EMAIL
FROM SQL_CATEGORY_MEMBER
WHERE MEMBER_ID = 1;

데이터를 변경하지 않고 조건에 맞는 데이터를 조회합니다. SELECT는 데이터를 변경하지 않지만 DML에 포함됩니다.

5-2. INSERT - 데이터 입력

INSERT INTO SQL_CATEGORY_MEMBER (
    MEMBER_ID,
    MEMBER_NAME,
    EMAIL,
    CREATED_AT
)
VALUES (
    1,
    '홍길동',
    'hong@example.com',
    SYSDATE
);

테이블 안에 새로운 행을 추가합니다. 회원가입, 게시글 작성, 댓글 등록 기능과 연결됩니다.

5-3. UPDATE - 데이터 수정

UPDATE SQL_CATEGORY_MEMBER
SET EMAIL = 'newhong@example.com'
WHERE MEMBER_ID = 1;

기존 데이터를 수정합니다. WHERE 조건을 빠뜨리면 모든 행이 수정될 수 있으므로 반드시 주의해야 합니다.

5-4. DELETE - 데이터 삭제

DELETE FROM SQL_CATEGORY_MEMBER
WHERE MEMBER_ID = 1;

조건에 맞는 데이터를 삭제합니다. WHERE 조건 없이 실행하면 테이블의 모든 데이터가 삭제될 수 있습니다.

DCL

6. DCL - 사용자 권한을 다루는 SQL

DCL은 Data Control Language의 약자로, 사용자에게 권한을 부여하거나 회수할 때 사용합니다. 일반 기능 구현보다 DB 계정 관리, 권한 문제 해결, 운영 환경 설정에서 중요합니다.

GRANT - 권한 부여

GRANT SELECT ON SCOTT.EMP TO USER1;

USER1에게 SCOTT.EMP 테이블을 조회할 수 있는 권한을 부여합니다.

REVOKE - 권한 회수

REVOKE SELECT ON SCOTT.EMP FROM USER1;

USER1에게 부여했던 SCOTT.EMP 조회 권한을 회수합니다.

권한 오류와 연결

Oracle에서 ORA-00942: table or view does not exist 오류가 발생하면, 실제로 테이블이 없을 수도 있지만 현재 접속 계정에 조회 권한이 없어서 발생할 수도 있습니다.

TCL

7. TCL - 트랜잭션을 다루는 SQL

TCL은 Transaction Control Language의 약자로, 여러 데이터 변경 작업을 하나의 작업 단위로 묶고 그 작업을 확정하거나 취소할 때 사용합니다.
Oracle 학습에서는 COMMIT, ROLLBACK, SAVEPOINT가 중요합니다.

1 DML 실행

INSERT, UPDATE, DELETE로
데이터를 변경합니다.

2 결과 확인

SELECT로 변경 결과가
의도한 대로인지 확인합니다.

3 COMMIT

문제가 없다면 변경 사항을 확정합니다.

4 ROLLBACK

문제가 있다면 변경 사항을 취소합니다.

7-1. COMMIT - 변경 사항 확정

INSERT INTO SQL_CATEGORY_MEMBER
VALUES (
    2,
    '김철수',
    'kim@example.com',
    SYSDATE
);

COMMIT;

7-2. ROLLBACK - 변경 사항 취소

INSERT INTO SQL_CATEGORY_MEMBER
VALUES (
    3,
    '이영희',
    'lee@example.com',
    SYSDATE
);

ROLLBACK;

7-3. SAVEPOINT - 중간 저장 지점

INSERT INTO SQL_CATEGORY_MEMBER
VALUES (4, '박민수', 'park@example.com', SYSDATE);

SAVEPOINT point1;

UPDATE SQL_CATEGORY_MEMBER
SET EMAIL = 'wrong@example.com'
WHERE MEMBER_ID = 4;

ROLLBACK TO point1;

SAVEPOINT를 사용하면 트랜잭션 전체가 아니라 특정 지점 이후의 작업만 되돌릴 수 있습니다.

CREATE TABLE 또는

INSERT, UPDATE, DELETE로

변경되었는지 확인합니다.

문제가 없으면 COMMIT,
문제가 있으면 ROLLBACK
합니다.

CREATE TABLE SQL_CATEGORY_STUDY (

FROM SQL_CATEGORY_STUDY

Practice

9. 실습 문제 연결

실습 문제 교안의 초반 문제들은 대부분 SELECT 중심의 DML 문제입니다.
WHERE, BETWEEN, IN, LIKE, IS NULL, ORDER BY, GROUP BY가 함께 나오지만, 큰 분류로 보면 모두 데이터를 조회하는 SELECT 문장입니다.

SELECT + WHERE

조건에 맞는 직원을 조회하는 DML 예제입니다.

SELECT EMPNO, ENAME, JOB, SAL
FROM EMP
WHERE JOB = 'MANAGER'
  AND SAL BETWEEN 2500 AND 3000;

SELECT + ORDER BY

연봉이 높은 순서대로 조회하는 DML 예제입니다.

SELECT *
FROM EMP
ORDER BY SAL DESC;

SELECT + GROUP BY

부서별 최고 연봉을 구하는 DML 예제입니다.

SELECT MAX(SALARY), DEPARTMENT_ID
FROM EMPLOYEES
GROUP BY DEPARTMENT_ID
ORDER BY MAX(SALARY) DESC;

SELECT + JOIN

여러 테이블을 연결해 필요한 정보를 조회하는 DML 예제입니다.

SELECT ENAME, SAL, E.DEPTNO, DNAME
FROM EMP E JOIN DEPT D
ON E.DEPTNO = D.DEPTNO
WHERE SAL >= 3000
ORDER BY ENAME DESC;

10. Java / Spring / Oracle 프로젝트 연결

Spring Boot 프로젝트에서는 SQL을 직접 작성하지 않아도 내부적으로는 SQL이 실행됩니다. JPA, MyBatis, JDBC 모두 결국 Oracle DB에 SQL 명령을 전달합니다.

회원가입

사용자 정보를 새로 저장하므로 DML의 INSERT와 연결됩니다.

INSERT INTO USERS (
    USER_ID,
    LOGIN_ID,
    EMAIL,
    PASSWORD_HASH,
    REGISTERED_AT
)
VALUES (?, ?, ?, ?, SYSDATE);

로그인

아이디로 회원 정보를 조회하므로 DML의 SELECT와 연결됩니다.

SELECT *
FROM USERS
WHERE LOGIN_ID = ?;

회원정보 수정

기존 사용자 정보를 변경하므로 DML의 UPDATE와 연결됩니다.

UPDATE USERS
SET EMAIL = ?,
    UPDATED_AT = SYSDATE
WHERE USER_ID = ?;

회원 탈퇴

실무에서는 DELETE보다 상태값을 바꾸는 논리 삭제를 자주 사용합니다.

UPDATE USERS
SET ACTIVE = 'N',
    DELETED_AT = SYSDATE
WHERE USER_ID = ?;

DB 권한 설정

애플리케이션 계정에 테이블 접근 권한을 부여할 때 DCL을 사용합니다.

GRANT SELECT, INSERT, UPDATE, DELETE
ON USERS
TO APP_USER;

@Transactional

Spring의 트랜잭션 처리는 COMMIT, ROLLBACK 개념과 연결됩니다.

@Transactional
public void createOrder(OrderRequest request) {
    orderRepository.save(order);
    paymentRepository.save(payment);
    inventoryService.decreaseStock(productId);
}

Spring 트랜잭션 흐름

Service 메서드 시작
→ INSERT 주문
→ INSERT 결제
→ UPDATE 재고
→ 모두 성공하면 COMMIT
→ 중간에 예외가 발생하면 ROLLBACK
Mistake

11. 자주 하는 실수

  • 실수 1. DELETE와 DROP을 혼동한다. DELETE는 테이블 안의 데이터를 삭제하는 DML이고, DROP은 테이블 자체를 삭제하는 DDL입니다.
  • 실수 2. DELETE와 TRUNCATE를 같은 것으로 생각한다. DELETE는 WHERE 조건과 ROLLBACK이 가능하지만, TRUNCATE는 전체 삭제이며 일반적으로 되돌리기 어렵습니다.
  • 실수 3. UPDATE에 WHERE를 빼먹는다. WHERE 없이 UPDATE를 실행하면 모든 행이 수정될 수 있습니다.
  • 실수 4. DELETE에 WHERE를 빼먹는다. WHERE 없이 DELETE를 실행하면 모든 데이터가 삭제될 수 있습니다.
  • 실수 5. SELECT는 DML이 아니라고 생각한다. SELECT는 데이터를 변경하지 않지만, 테이블 데이터를 조회하는 DML입니다.
  • 실수 6. COMMIT 이후에도 ROLLBACK이 된다고 생각한다. COMMIT된 변경 사항은 일반적인 ROLLBACK으로 되돌릴 수 없습니다.
  • 실수 7. SQL Developer 자동 커밋 설정을 확인하지 않는다. 자동 커밋이 켜져 있으면 ROLLBACK 실습 결과가 예상과 다르게 나올 수 있습니다.
Interview

12. 면접 포인트

Q1. SQL의 종류를 설명해 주세요.

SQL은 목적에 따라 DDL, DML, DCL, TCL로 나눌 수 있습니다.
DDL은 데이터베이스 객체의 구조를 생성, 변경, 삭제하는 명령이고, DML은 테이블 안의 데이터를 조회, 입력, 수정, 삭제하는 명령입니다.
DCL은 권한을 부여하거나 회수하는 명령이고, TCL은 트랜잭션을 확정하거나 되돌리는 명령입니다.

Q2. DDL과 DML의 차이는 무엇인가요?

DDL은 테이블이나 뷰 같은 데이터베이스 객체의 구조를 다루고, DML은 테이블 안의 실제 데이터를 다룹니다.
쉽게 말하면 DDL은 그릇을 만들거나 바꾸는 명령이고, DML은 그릇 안의 데이터를 넣고 조회하고 수정하고 삭제하는 명령입니다.

Q3. DELETE, DROP, TRUNCATE의 차이는 무엇인가요?

DELETE는 테이블 안의 행 데이터를 삭제하는 DML이고 WHERE 조건을 사용할 수 있습니다.
DROP은 테이블 자체를 삭제하는 DDL입니다.
TRUNCATE는 테이블 구조는 남기고 전체 데이터를 삭제하는 DDL 성격의 명령이며, 일반적으로 DELETE보다 빠르지만 되돌리기 어렵습니다.

Q4. COMMIT과 ROLLBACK은 무엇인가요?

COMMIT은 현재 트랜잭션에서 수행한 변경 사항을 데이터베이스에 확정하는 명령입니다.
ROLLBACK은 마지막 COMMIT 이후 수행한 변경 사항을 취소하는 명령입니다.

Q5. SELECT는 DML인가요?

네. SELECT는 데이터를 변경하지는 않지만 테이블 안의 데이터를 조회하는 명령이므로 DML에 포함됩니다.

Q6. UPDATE나 DELETE 실행 시 가장 조심해야 할 점은 무엇인가요?

WHERE 조건을 반드시 확인해야 합니다. WHERE 조건 없이 실행하면 테이블의 모든 행이 수정되거나 삭제될 수 있습니다.
실무에서는 먼저 같은 조건으로 SELECT를 실행해 대상 데이터를 확인한 뒤 UPDATE나 DELETE를 실행하는 습관이 중요합니다.

Q7. Spring의 @Transactional은 SQL 분류 중 무엇과 연결되나요?

@Transactional은 TCL의 COMMIT, ROLLBACK 개념과 연결됩니다. 메서드 내부의 여러 DML 작업이 모두 성공하면 COMMIT되고, 중간에 예외가 발생하면 ROLLBACK됩니다.

Summary

13. 최종 정리

DDL
→ 구조를 다루는 SQL
→ CREATE, ALTER, DROP, TRUNCATE, RENAME

DML
→ 데이터를 다루는 SQL
→ SELECT, INSERT, UPDATE, DELETE

DCL
→ 권한을 다루는 SQL
→ GRANT, REVOKE

TCL
→ 트랜잭션을 다루는 SQL
→ COMMIT, ROLLBACK, SAVEPOINT

구분 기준

테이블 구조가 바뀌면 DDL, 테이블 안의 데이터가 조회되거나 변경되면 DML, 사용자 권한이 바뀌면 DCL, 작업의 확정과 취소를 다루면 TCL입니다.

학습 메모

오늘의 핵심

SQL은 목적에 따라 나누어야 합니다. 명령어를 외우는 것보다 “구조를 다루는가, 데이터를 다루는가, 권한을 다루는가, 트랜잭션을 다루는가”를 먼저 판단해야 합니다.

분류 암기법

  • CREATE, ALTER, DROP → DDL
  • SELECT, INSERT, UPDATE, DELETE → DML
  • GRANT, REVOKE → DCL
  • COMMIT, ROLLBACK, SAVEPOINT → TCL

실습 확인 순서

  1. DDL 실행 후에는 DESC로 구조 확인
  2. DML 실행 후에는 SELECT로 데이터 확인
  3. UPDATE / DELETE 전에는 먼저 SELECT로 대상 확인
  4. 문제가 없으면 COMMIT
  5. 문제가 있으면 ROLLBACK

주의할 명령어

DROP, TRUNCATE, WHERE 없는 UPDATE, WHERE 없는 DELETE는 매우 위험합니다. 학습 중에도 실행 전 대상 테이블과 조건을 반드시 확인하세요.

프로젝트 연결

회원가입은 INSERT, 로그인은 SELECT, 회원정보 수정은 UPDATE, 권한 설정은 GRANT, Spring의 @Transactional은 COMMIT / ROLLBACK 개념과 연결됩니다.
