Oracle SQL Developer - Chapter 11

그룹 함수와 GROUP BY

그룹 함수는 여러 행을 계산해 하나의 통계 결과로 요약하는 함수입니다.
이번 단원에서는 COUNT, SUM, AVG, MAX, MIN과 GROUP BY, HAVING의 차이를 정리하고,
SQL Developer에서 직접 실행하면서 통계 조회의 흐름을 익힙니다.

학습 목표 여러 행을 하나의 통계 결과로 요약하는
그룹 함수의 개념을 이해한다.
실습 방향 EMP와 EMPLOYEES 테이블에서
COUNT, AVG, GROUP BY, HAVING을 직접 실행한다.
프로젝트 연결 관리자 대시보드, 통계 API, 매물 분석,
추천 로그 분석과 연결한다.
Overview

1. 그룹 함수란?

그룹 함수는 여러 행을 입력받아 하나의 결과값을 반환하는 함수입니다.
일반 SELECT가 사원 한 명 한 명의 정보를 보여준다면, 그룹 함수는 전체 사원 수, 평균 급여, 최고 급여, 최저 급여처럼 여러 행을 요약한 통계값을 보여줍니다.

여러 행 EMP 테이블의 여러 사원 데이터
그룹 함수 COUNT, SUM, AVG, MAX, MIN으로 계산
하나의 결과 전체 개수, 합계, 평균, 최고값, 최저값

한 줄 정리

그룹 함수는 여러 행을 통계값 하나로 요약하는 함수입니다. 목록 조회가 “행을 보여주는 것”이라면, 그룹 함수는 “행을 계산하는 것”입니다.

Functions

2. 대표 그룹 함수

Oracle에서 자주 사용하는 그룹 함수는 COUNT, SUM, AVG, MAX, MIN입니다.
각각 “몇 개인가”, “합계는 얼마인가”, “평균은 얼마인가”, “가장 큰 값은 무엇인가”, “가장 작은 값은 무엇인가”를 구할 때 사용합니다.

COUNT(*)

전체 행 개수를 셉니다.

SELECT COUNT(*)
FROM EMP;

COUNT(컬럼)

해당 컬럼이 NULL이 아닌 행만 셉니다.

SELECT COUNT(COMM)
FROM EMP;

SUM(컬럼)

숫자 컬럼의 합계를 구합니다.

SELECT SUM(SAL)
FROM EMP;

AVG(컬럼)

숫자 컬럼의 평균을 구합니다.

SELECT AVG(SAL)
FROM EMP;

MAX(컬럼)

가장 큰 값을 구합니다.

SELECT MAX(SAL)
FROM EMP;

MIN(컬럼)

가장 작은 값을 구합니다.

SELECT MIN(SAL)
FROM EMP;

부서번호별로 행을 묶고 각 그룹마다 그룹 함수를 계산합니다.

Syntax

4. 기본 문법

그룹 함수만 사용할 때는 전체 테이블을 하나의 그룹처럼 보고 계산합니다. 특정 기준별 통계를 구하려면 GROUP BY를 사용합니다.

4-1. 전체 통계

SELECT COUNT(*),
       SUM(SAL),
       AVG(SAL),
       MAX(SAL),
       MIN(SAL)
FROM EMP;

EMP 테이블 전체를 하나의 그룹처럼 보고 통계를 계산합니다.

4-2. 기준별 통계

SELECT DEPTNO, AVG(SAL)
FROM EMP
GROUP BY DEPTNO;

DEPTNO별로 행을 묶고 각 부서별 평균 급여를 계산합니다.

4-3. 전체 구문

SELECT 그룹기준컬럼, 그룹함수(컬럼)
FROM 테이블명
WHERE 행조건
GROUP BY 그룹기준컬럼
HAVING 그룹조건
ORDER BY 정렬기준;

WHERE는 그룹화 전 행 조건, HAVING은 그룹화 후 그룹 조건, ORDER BY는 최종 결과 정렬입니다.

COUNT

6. COUNT 함수

COUNT는 행의 개수를 셀 때 사용합니다. 특히 COUNT(*)와 COUNT(컬럼명)의 차이를 반드시 이해해야 합니다.

COUNT(*)

NULL 여부와 관계없이 전체 행 수를 셉니다.

SELECT COUNT(*)
FROM EMP;

COUNT(COMM)

COMM 컬럼이 NULL이 아닌 행만 셉니다.

SELECT COUNT(COMM)
FROM EMP;

비교 실습

SELECT COUNT(*) AS "전체 사원 수",
       COUNT(COMM) AS "커미션 받는 사원 수"
FROM EMP;

전체 행 수와 커미션 값이 있는 행 수의 차이를 확인합니다.

조건 COUNT

SELECT COUNT(*)
FROM EMP
WHERE JOB = 'SALESMAN';

직무가 SALESMAN인 사원 수만 계산합니다.

SUM / AVG / MAX / MIN

7. 합계, 평균, 최댓값, 최솟값

SUM과 AVG는 숫자 컬럼의 합계와 평균을 계산하고, MAX와 MIN은 숫자, 문자, 날짜 컬럼에서 최댓값과 최솟값을 구할 수 있습니다.

7-1. 전체 급여 합계

SELECT SUM(SAL)
FROM EMP;

7-2. 전체 급여 평균

SELECT AVG(SAL)
FROM EMP;

7-3. 최고 급여와 최저 급여

SELECT MAX(SAL),
       MIN(SAL)
FROM EMP;

7-4. 최근 입사일과 가장 오래된 입사일

SELECT MAX(HIREDATE) AS "최근 입사일",
       MIN(HIREDATE) AS "가장 오래된 입사일"
FROM EMP;

7-5. 직무가 SALESMAN인 직원 평균 급여

SELECT AVG(SAL)
FROM EMP
WHERE JOB = 'SALESMAN';

7-6. SUM / COUNT로 평균 계산

SELECT SUM(SAL) / COUNT(*)
FROM EMP
WHERE JOB = 'SALESMAN';

실습 문제 교안에서 사용한 방식입니다.

GROUP BY

8. GROUP BY로 기준별 통계 구하기

GROUP BY는 행을 특정 기준으로 묶습니다. 그룹화된 각 그룹마다 COUNT, SUM, AVG, MAX, MIN 같은 그룹 함수를 계산할 수 있습니다.

8-1. 부서별 평균 급여

SELECT DEPTNO, AVG(SAL)
FROM EMP
GROUP BY DEPTNO;

DEPTNO별로 행을 묶고 각 부서의 평균 급여를 계산합니다.

8-2. 직무별 사원 수

SELECT JOB, COUNT(*)
FROM EMP
GROUP BY JOB;

JOB별로 사원이 몇 명인지 계산합니다.

8-3. 부서별 최고 급여

SELECT DEPTNO, MAX(SAL)
FROM EMP
GROUP BY DEPTNO;

8-4. 부서와 직무 조합별 평균 급여

SELECT DEPTNO, JOB, AVG(SAL)
FROM EMP
GROUP BY DEPTNO, JOB
ORDER BY DEPTNO, JOB;

DEPTNO와 JOB의 조합별로 그룹을 만듭니다.

GROUP BY를 사용할 때 SELECT절 원칙

SELECT절에 올 수 있는 것

1. GROUP BY에 적은 컬럼
2. 그룹 함수

가능:
SELECT DEPTNO, AVG(SAL)
FROM EMP
GROUP BY DEPTNO;

불가능:
SELECT DEPTNO, ENAME, AVG(SAL)
FROM EMP
GROUP BY DEPTNO;
WHERE vs HAVING

9. WHERE와 HAVING의 차이

WHERE와 HAVING은 모두 조건절이지만 적용 시점과 대상이 다릅니다. WHERE는 그룹화 전에 개별 행을 거르고, HAVING은 그룹화 후 그룹 결과를 거릅니다.

WHERE

그룹화 전 개별 행 조건입니다.

SELECT DEPTNO, AVG(SAL)
FROM EMP
WHERE SAL >= 1000
GROUP BY DEPTNO;

HAVING

그룹화 후 그룹 결과 조건입니다.

SELECT DEPTNO, AVG(SAL)
FROM EMP
GROUP BY DEPTNO
HAVING AVG(SAL) >= 2000;

WHERE + HAVING

SELECT DEPTNO, AVG(SAL)
FROM EMP
WHERE SAL >= 1000
GROUP BY DEPTNO
HAVING AVG(SAL) >= 2000;

SAL이 1000 이상인 사원만 남긴 뒤,
부서별 평균이 2000 이상인 부서만 출력합니다.

정렬까지 포함

SELECT DEPTNO, AVG(SAL)
FROM EMP
WHERE SAL >= 1000
GROUP BY DEPTNO
HAVING AVG(SAL) >= 2000
ORDER BY AVG(SAL) DESC;

최종 결과를 평균 급여 높은 순으로 정렬합니다.

HR Examples

10. HR 계정 실습 예제

HR 계정의 EMPLOYEES 테이블에서는 부서별, 직무별 연봉 통계를 연습하기 좋습니다. 실습 문제 교안의 GROUPING 문제도 HR 계정 예제를 포함합니다.

10-1. 부서별 최고 연봉

SELECT MAX(SALARY), DEPARTMENT_ID
FROM EMPLOYEES
GROUP BY DEPARTMENT_ID
ORDER BY MAX(SALARY) DESC;

부서별 최고 연봉을 구하고 최고 연봉순으로 내림차순 정렬합니다.

10-2. 조건 적용 후 부서별 평균 연봉

SELECT AVG(SALARY), DEPARTMENT_ID
FROM EMPLOYEES
WHERE COMMISSION_PCT IS NULL
  AND NOT SALARY <= 3000
GROUP BY DEPARTMENT_ID
ORDER BY AVG(SALARY) DESC;

커미션을 받는 직원과 연봉 3000 이하 직원을 제외한 뒤 부서별 평균 연봉을 구합니다.

10-3. JOB_ID와 DEPARTMENT_ID 조합별 평균

SELECT JOB_ID, DEPARTMENT_ID, AVG(SALARY)
FROM EMPLOYEES
WHERE SALARY >= 3000
GROUP BY JOB_ID, DEPARTMENT_ID
HAVING AVG(SALARY) >= 3500
ORDER BY JOB_ID DESC;

연봉 3000 이상인 직원만 대상으로 JOB_ID와 DEPARTMENT_ID 조합별 평균 연봉을 구하고, 평균 연봉이 3500 이상인 그룹만 출력합니다.

Practice

11. 실습 문제 연결

실습 문제 교안의 GROUPING 부분은 이번 단원의 핵심을 직접 확인하는 문제들입니다. 문제를 읽을 때는 먼저 “전체 통계인지”, “그룹별 통계인지”, “그룹 조건이 있는지”를 구분해야 합니다.

문제 1. 조건에 맞는 행 개수

연봉이 1000 이상이고 이름이 S로 끝나는 직원 수를 구합니다.

SELECT COUNT(*)
FROM EMP
WHERE SAL >= 1000
  AND ENAME LIKE '%S';

문제 2. SALESMAN 평균 급여

SELECT SUM(SAL) / COUNT(*)
FROM EMP
WHERE JOB = 'SALESMAN';

AVG(SAL)로도 같은 의도를 표현할 수 있습니다.

문제 3. 부서별 최고 연봉

SELECT MAX(SALARY), DEPARTMENT_ID
FROM EMPLOYEES
GROUP BY DEPARTMENT_ID
ORDER BY MAX(SALARY) DESC;

문제 4. WHERE 후 GROUP BY

SELECT AVG(SALARY), DEPARTMENT_ID
FROM EMPLOYEES
WHERE COMMISSION_PCT IS NULL
  AND NOT SALARY <= 3000
GROUP BY DEPARTMENT_ID
ORDER BY AVG(SALARY) DESC;

문제 5. GROUP BY + HAVING

SELECT JOB_ID, DEPARTMENT_ID, AVG(SALARY)
FROM EMPLOYEES
WHERE SALARY >= 3000
GROUP BY JOB_ID, DEPARTMENT_ID
HAVING AVG(SALARY) >= 3500
ORDER BY JOB_ID DESC;

문제 읽는 순서

1. 전체 통계인가, 그룹별 통계인가?
2. WHERE 조건이 있는가?
3. GROUP BY 기준 컬럼은 무엇인가?
4. HAVING 조건이 있는가?
5. ORDER BY 정렬 기준은 무엇인가?
Project

12. Java / Spring / Oracle 프로젝트 연결

그룹 함수와 GROUP BY는 실무 프로젝트에서 관리자 대시보드, 통계 API, 리포트, 추천 시스템 분석 데이터에 자주 사용됩니다.

전체 회원 수

SELECT COUNT(*)
FROM USERS;

관리자 대시보드의 전체 회원 수 카드에 사용할 수 있습니다.

활성 회원 수

SELECT COUNT(*)
FROM USERS
WHERE ACTIVE = 'Y';

ACTIVE가 Y인 사용자만 계산합니다.

삭제되지 않은 게시글 수

SELECT COUNT(*)
FROM BOARD
WHERE DELETED_YN = 'N';

논리 삭제 구조에서 실제 노출 가능한 게시글 수를 계산합니다.

작성자별 게시글 수

SELECT WRITER_ID, COUNT(*)
FROM BOARD
GROUP BY WRITER_ID;

사용자별 활동량 통계로 사용할 수 있습니다.

매물 평균 가격

SELECT AVG(DEAL_AMOUNT)
FROM PROPERTY
WHERE DEAL_TYPE = 'SALE';

매매 매물만 대상으로 평균 가격을 계산합니다.

매물 유형별 개수

SELECT PROPERTY_TYPE, COUNT(*)
FROM PROPERTY
GROUP BY PROPERTY_TYPE;

아파트, 오피스텔, 빌라 등 유형별 매물 수를 계산합니다.

매물 유형별 가격 통계

SELECT PROPERTY_TYPE,
       COUNT(*) AS PROPERTY_COUNT,
       AVG(DEAL_AMOUNT) AS AVG_PRICE,
       MAX(DEAL_AMOUNT) AS MAX_PRICE,
       MIN(DEAL_AMOUNT) AS MIN_PRICE
FROM PROPERTY
GROUP BY PROPERTY_TYPE;

관리자 통계 API에서 DTO로 변환하기 좋은 형태입니다.

추천 로그 분석

SELECT PROPERTY_ID,
       COUNT(*) AS VIEW_COUNT,
       AVG(STAY_SECONDS) AS AVG_STAY_SECONDS
FROM PROPERTY_VIEW_LOG
GROUP BY PROPERTY_ID;

조회 수와 평균 체류시간을 추천 feature로 활용할 수 있습니다.

인기 매물 필터

SELECT PROPERTY_ID, COUNT(*) AS VIEW_COUNT
FROM PROPERTY_VIEW_LOG
GROUP BY PROPERTY_ID
HAVING COUNT(*) >= 10
ORDER BY VIEW_COUNT DESC;

조회 수가 10회 이상인 매물만 인기 매물로 추출합니다.

Spring DTO Projection

@Query("""
    SELECT new com.example.PropertyTypeStatDto(
        p.propertyType,
        COUNT(p),
        AVG(p.dealAmount),
        MAX(p.dealAmount),
        MIN(p.dealAmount)
    )
    FROM PropertyEntity p
    GROUP BY p.propertyType
""")
List<PropertyTypeStatDto> findPropertyTypeStats();

JPA에서도 GROUP BY 통계 결과를 DTO로 바로 받을 수 있습니다.

Mistake

13. 자주 하는 실수

  • 실수 1. 그룹 함수와 일반 컬럼을 함께 쓰면서 GROUP BY를 안 쓴다. SELECT DEPTNO, AVG(SAL) FROM EMP;는 문제가 됩니다. DEPTNO별 평균을 원하면 GROUP BY DEPTNO가 필요합니다.
  • 실수 2. GROUP BY에 없는 일반 컬럼을 SELECT에 쓴다. SELECT DEPTNO, ENAME, AVG(SAL) FROM EMP GROUP BY DEPTNO;에서 ENAME은 그룹 기준도 그룹 함수도 아니므로 오류가 납니다.
  • 실수 3. 그룹 조건을 WHERE에 쓴다. WHERE AVG(SAL) >= 2000이 아니라 HAVING AVG(SAL) >= 2000을 사용해야 합니다.
  • 실수 4. WHERE와 HAVING을 혼동한다. WHERE는 그룹화 전 행 조건, HAVING은 그룹화 후 그룹 조건입니다.
  • 실수 5. GROUP BY 위치를 잘못 쓴다. GROUP BY는 ORDER BY보다 앞에 작성해야 합니다.
  • 실수 6. COUNT(*)와 COUNT(컬럼)을 같은 것으로 생각한다. COUNT(*)는 전체 행 수, COUNT(컬럼)은 해당 컬럼이 NULL이 아닌 행 수입니다.
  • 실수 7. AVG가 NULL을 어떻게 처리하는지 모른다. AVG(컬럼)은 NULL인 값을 제외하고 평균을 계산합니다.
  • 실수 8. 여러 컬럼 GROUP BY의 의미를 오해한다. GROUP BY DEPTNO, JOB은 DEPTNO와 JOB의 조합별 그룹을 의미합니다.
Interview

14. 면접 포인트

Q1. 그룹 함수란 무엇인가요?

그룹 함수는 여러 행을 입력받아 하나의 결과값을 반환하는 함수입니다.
대표적으로 COUNT, SUM, AVG, MAX, MIN이 있으며, 전체 개수, 합계, 평균, 최댓값, 최솟값을 구할 때 사용합니다.

Q2. COUNT(*)와 COUNT(컬럼명)의 차이는 무엇인가요?

COUNT(*)는 NULL 여부와 관계없이 전체 행 수를 셉니다.
COUNT(컬럼명)은 해당 컬럼이 NULL이 아닌 행만 셉니다.
예를 들어 COUNT(COMM)은 커미션 값이 있는 직원 수만 계산합니다.

Q3. GROUP BY는 무엇인가요?

GROUP BY는 특정 컬럼을 기준으로 행을 그룹화하는 절입니다. 그룹화된 각 그룹마다 COUNT, SUM, AVG, MAX, MIN 같은 그룹 함수를 적용할 수 있습니다.

Q4. WHERE와 HAVING의 차이는 무엇인가요?

WHERE는 그룹화 전에 개별 행을 필터링하는 조건절이고, HAVING은 그룹화 후 그룹 결과를 필터링하는 조건절입니다.
예를 들어 WHERE SAL >= 1000은 개별 사원 조건이고, HAVING AVG(SAL) >= 2000은 그룹 평균 조건입니다.

Q5. GROUP BY를 사용할 때 SELECT절에는 무엇을 쓸 수 있나요?

GROUP BY를 사용할 때 SELECT절에는 GROUP BY에 포함된 컬럼이나 그룹 함수만 올 수 있습니다.
그룹 기준이 아닌 일반 컬럼을 함께 출력하면 어떤 행의 값을 보여줘야 할지 결정할 수 없기 때문에 오류가 발생합니다.

Q6. GROUP BY와 ORDER BY의 순서는 어떻게 되나요?

작성 순서에서는 GROUP BY가 ORDER BY보다 먼저 와야 합니다. GROUP BY로 그룹 결과를 만든 뒤 ORDER BY로 최종 결과를 정렬합니다.

Q7. HAVING 없이 WHERE만으로 그룹 조건을 줄 수 있나요?

그룹 함수 결과에 대한 조건은 WHERE로 줄 수 없습니다. AVG(SAL)이나 COUNT(*) 같은 그룹 함수 결과를 조건으로 사용하려면 HAVING을 사용해야 합니다.

읽는 순서

1. FROM을 보고 어느 테이블인지 확인한다.
2. WHERE가 있으면 그룹화 전에 어떤 행을 거르는지 확인한다.
3. GROUP BY를 보고 어떤 기준으로 묶는지 확인한다.
4. SELECT의 그룹 함수를 보고 어떤 통계를 계산하는지 확인한다.
5. HAVING이 있으면 어떤 그룹만 남기는지 확인한다.
6. ORDER BY가 있으면 어떤 순서로 정렬하는지 확인한다.

학습 메모

오늘의 핵심

그룹 함수는 여러 행을 하나의 통계값으로 요약합니다. GROUP BY는 행을 기준별로 묶고, 각 그룹마다 그룹 함수를 계산하게 해줍니다.

기본 문법

SELECT 그룹기준컬럼, 그룹함수(컬럼)
FROM 테이블명
WHERE 행조건
GROUP BY 그룹기준컬럼
HAVING 그룹조건
ORDER BY 정렬기준;

그룹 함수 암기

  • COUNT(*) : 전체 행 개수
  • COUNT(컬럼) : NULL이 아닌 행 개수
  • SUM : 합계
  • AVG : 평균
  • MAX : 최댓값
  • MIN : 최솟값

WHERE vs HAVING

  • WHERE : 그룹화 전 행 조건
  • HAVING : 그룹화 후 그룹 조건
  • WHERE SAL >= 1000 : 사원 개인 조건
  • HAVING AVG(SAL) >= 2000 : 부서 평균 조건

실습 확인 순서

  1. COUNT(*)로 전체 행 수 확인
  2. COUNT(COMM)으로 NULL 제외 개수 확인
  3. SUM, AVG, MAX, MIN으로 전체 통계 확인
  4. GROUP BY DEPTNO로 부서별 통계 확인
  5. WHERE와 GROUP BY를 함께 실행
  6. HAVING으로 그룹 조건 실행
  7. ORDER BY로 통계 결과 정렬

주의할 점

GROUP BY를 사용할 때 SELECT절에는 GROUP BY에 적은 컬럼과 그룹 함수만 올 수 있습니다. 그룹 조건은 WHERE가 아니라 HAVING에 작성해야 합니다.

프로젝트 연결

관리자 대시보드의 전체 회원 수, 게시글 수, 매물 유형별 개수, 평균 가격, 최고 가격, 추천 로그 평균 체류시간은 모두 그룹 함수와 GROUP BY로 만들 수 있습니다.
