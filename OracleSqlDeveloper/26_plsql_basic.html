Oracle SQL Developer - Chapter 26

PL/SQL 기초

PL/SQL은 SQL을 변수, 조건문, 반복문, 예외 처리와 함께 묶어서
Oracle 데이터베이스 안에서 절차적으로 실행하게 해주는 프로그래밍 언어입니다.
이번 단원에서는 DECLARE, BEGIN, EXCEPTION, END 블록 구조부터
변수, %TYPE, SELECT INTO, IF, LOOP, FOR, WHILE, DML 결과 확인, 예외 처리까지 학습합니다.

학습 목표 SQL 한 문장을 넘어
여러 SQL을 프로그램 흐름으로 묶어 실행한다.
실습 방향 출력 설정 → 변수 → SELECT INTO →
조건문 → 반복문 → DML → 예외 처리
프로젝트 연결 저장 프로시저, 함수, 트리거,
Spring 프로시저 호출 구조로 이어진다.
Overview

1. PL/SQL은 왜 배우는가?

지금까지 배운 SQL은 보통 한 문장씩 실행했습니다. 하지만 실무에서는 단순히 한 번 조회하거나 한 번 수정하는 것으로 끝나지 않습니다.
먼저 데이터를 조회하고, 조건을 판단하고, 수정하고, 처리 건수를 확인하고, 오류가 나면 예외 처리를 해야 하는 경우가 많습니다.

SQL 한 문장 
UPDATE EMP
SET SAL = 1200
WHERE EMPNO = 7369;

사원 급여를 수정하는 한 문장입니다.

→
PL/SQL 업무 흐름 
1. 사원번호를 변수에 담는다.
2. UPDATE를 실행한다.
3. 수정된 행 수를 확인한다.
4. 없으면 메시지를 출력한다.
5. 오류가 발생하면 예외 처리한다.

여러 SQL과 판단 로직을 하나의 블록으로 묶습니다.

변수 SQL 결과나 입력값을 임시로 저장한다.
조건·반복 IF, LOOP, FOR, WHILE로 실행 흐름을 제어한다.
예외 처리 오류 발생 시
메시지 출력, 로그, ROLLBACK 구조로 연결한다.

한 줄 정리

SQL이 데이터를 조회·변경하는 문장이라면, PL/SQL은 그 SQL들을 변수, 조건문, 반복문, 예외 처리와 함께 묶어 실행하는 Oracle의 절차형 언어입니다.

%TYPE / %ROWTYPE

8. SQL 예제 카드 - %TYPE과 %ROWTYPE

테이블 컬럼 값을 변수에 담을 때는 직접 NUMBER, VARCHAR2를 쓰는 것보다 컬럼의 자료형을 그대로 가져오는 %TYPE을 사용하면 유지보수가 쉬워집니다. 한 행 전체를 변수로 받을 때는 %ROWTYPE을 사용할 수 있습니다.

%TYPE

DECLARE
    v_ename EMP.ENAME%TYPE;
    v_sal   EMP.SAL%TYPE;
BEGIN
    v_ename := 'SMITH';
    v_sal := 800;

    DBMS_OUTPUT.PUT_LINE(v_ename || ' / ' || v_sal);
END;
/

EMP.ENAME과 EMP.SAL 컬럼의 자료형을 그대로 가져옵니다.

%ROWTYPE

DECLARE
    v_emp EMP%ROWTYPE;
BEGIN
    SELECT *
    INTO v_emp
    FROM EMP
    WHERE EMPNO = 7369;

    DBMS_OUTPUT.PUT_LINE('사원명: ' || v_emp.ename);
    DBMS_OUTPUT.PUT_LINE('급여: ' || v_emp.sal);
    DBMS_OUTPUT.PUT_LINE('부서: ' || v_emp.deptno);
END;
/

EMP 테이블의 한 행 전체 구조를 변수 하나로 받습니다.

선택 기준

컬럼 하나의 값을 담는다
→ 테이블명.컬럼명%TYPE

한 행 전체를 담는다
→ 테이블명%ROWTYPE

SELECT INTO

10. SQL 예제 카드 - SELECT INTO

PL/SQL 안에서 SELECT 결과를 변수에 담으려면 INTO가 필요합니다. SELECT INTO는 반드시 정확히 한 행만 반환해야 합니다.

SELECT ENAME, SAL 조회

EMP 테이블에서 한 명의 사원을 찾습니다.

→
INTO v_ename, v_sal

조회 결과를 PL/SQL 변수에 담습니다.

→
사용 출력 / 조건 / DML

변수 값을 이후 로직에서 사용합니다.

10-1. 한 행 조회

DECLARE
    v_ename EMP.ENAME%TYPE;
    v_sal   EMP.SAL%TYPE;
BEGIN
    SELECT ENAME, SAL
    INTO v_ename, v_sal
    FROM EMP
    WHERE EMPNO = 7369;

    DBMS_OUTPUT.PUT_LINE('이름: ' || v_ename);
    DBMS_OUTPUT.PUT_LINE('급여: ' || v_sal);
END;
/

10-2. 컬럼 수와 변수 수 불일치

DECLARE
    v_ename EMP.ENAME%TYPE;
BEGIN
    SELECT ENAME, SAL
    INTO v_ename
    FROM EMP
    WHERE EMPNO = 7369;
END;
/

SELECT 컬럼은 2개인데 INTO 변수는 1개라 오류가 납니다.

10-3. 결과가 여러 행인 경우

DECLARE
    v_ename EMP.ENAME%TYPE;
BEGIN
    SELECT ENAME
    INTO v_ename
    FROM EMP
    WHERE DEPTNO = 20;
END;
/

20번 부서 직원이 여러 명이면 TOO_MANY_ROWS 예외가 발생합니다.

10-4. 결과가 없는 경우

DECLARE
    v_ename EMP.ENAME%TYPE;
BEGIN
    SELECT ENAME
    INTO v_ename
    FROM EMP
    WHERE EMPNO = -1;
END;
/

조회 결과가 없으면 NO_DATA_FOUND 예외가 발생합니다.

SELECT INTO 규칙

1. SELECT 컬럼 수와 INTO 변수 수가 같아야 한다.
2. 결과는 정확히 1행이어야 한다.
3. 0행이면 NO_DATA_FOUND 예외가 발생한다.
4. 2행 이상이면 TOO_MANY_ROWS 예외가 발생한다.
5. 여러 행 처리는 CURSOR 단원과 연결된다.
IF

11. SQL 예제 카드 - IF 조건문

IF문은 PL/SQL 실행 흐름을 조건에 따라 나눌 때 사용합니다. Java의 if / else if / else와 비슷하지만, PL/SQL에서는 ELSIF를 사용합니다.

Java

if (average >= 90) {
    System.out.println("A학점");
} else if (average >= 80) {
    System.out.println("B학점");
} else {
    System.out.println("F학점");
}

PL/SQL

IF average >= 90 THEN
    DBMS_OUTPUT.PUT_LINE('A학점');
ELSIF average >= 80 THEN
    DBMS_OUTPUT.PUT_LINE('B학점');
ELSE
    DBMS_OUTPUT.PUT_LINE('F학점');
END IF;

DECLARE

BEGIN
    IF MOD(v_num, 2) = 0 THEN

    ELSE
        DBMS_OUTPUT.PUT_LINE('홀수');
    END IF;
END;
/

DECLARE

    eng     NUMBER := 66;
    math    NUMBER := 89;
    average NUMBER;
BEGIN
    average := (kor + eng + math) / 3;

    DBMS_OUTPUT.PUT_LINE('평균: ' || ROUND(average, 2));

    IF average >= 90 THEN
        DBMS_OUTPUT.PUT_LINE('A학점');
    ELSIF average >= 80 THEN
        DBMS_OUTPUT.PUT_LINE('B학점');
    ELSIF average >= 70 THEN
        DBMS_OUTPUT.PUT_LINE('C학점');
    ELSIF average >= 60 THEN
        DBMS_OUTPUT.PUT_LINE('D학점');
    ELSE
        DBMS_OUTPUT.PUT_LINE('F학점');
    END IF;
END;
/
결과 확인 평균 84.67, B학점이 출력됩니다.

IF문 주의

  • ELSEIF가 아니라 ELSIF입니다.
  • 조건문은 END IF;로 끝납니다.
  • 학점처럼 범위 조건은 높은 조건부터 작성합니다.

DECLARE

15. SQL 예제 카드 - 예외 처리

예외 처리는 PL/SQL 실행 중 오류가 발생했을 때 블록이 비정상 종료되지 않도록 오류 상황을 잡아 메시지를 출력하거나 필요한 후속 처리를 하는 구조입니다.

NO_DATA_FOUND SELECT INTO 결과 0행
TOO_MANY_ROWS SELECT INTO 결과 2행 이상
ZERO_DIVIDE 0으로 나누기
DUP_VAL_ON_INDEX PK 또는 UNIQUE 중복
OTHERS 그 외 모든 예외

15-1. ZERO_DIVIDE

DECLARE
    v_result NUMBER;
BEGIN
    v_result := 10 / 0;

    DBMS_OUTPUT.PUT_LINE('결과: ' || v_result);
EXCEPTION
    WHEN ZERO_DIVIDE THEN
        DBMS_OUTPUT.PUT_LINE('0으로 나눌 수 없습니다.');
END;
/

15-2. NO_DATA_FOUND

DECLARE
    v_ename EMP.ENAME%TYPE;
BEGIN
    SELECT ENAME
    INTO v_ename
    FROM EMP
    WHERE EMPNO = -1;

    DBMS_OUTPUT.PUT_LINE('사원명: ' || v_ename);
EXCEPTION
    WHEN NO_DATA_FOUND THEN
        DBMS_OUTPUT.PUT_LINE('조회된 사원이 없습니다.');
END;
/

15-3. TOO_MANY_ROWS

DECLARE
    v_ename EMP.ENAME%TYPE;
BEGIN
    SELECT ENAME
    INTO v_ename
    FROM EMP
    WHERE DEPTNO = 20;

    DBMS_OUTPUT.PUT_LINE('사원명: ' || v_ename);
EXCEPTION
    WHEN TOO_MANY_ROWS THEN
        DBMS_OUTPUT.PUT_LINE('조회 결과가 2행 이상입니다.');
END;
/

15-4. DUP_VAL_ON_INDEX

BEGIN
    INSERT INTO dept_plsql_copy(DEPTNO, DNAME, LOC)
    VALUES(10, '중복부서', 'SEOUL');

    DBMS_OUTPUT.PUT_LINE('입력 성공');
EXCEPTION
    WHEN DUP_VAL_ON_INDEX THEN
        DBMS_OUTPUT.PUT_LINE('이미 존재하는 부서번호입니다.');
END;
/

15-5. OTHERS와 SQLCODE, SQLERRM

BEGIN
    INSERT INTO dept_plsql_copy(DEPTNO, DNAME, LOC)
    VALUES(10, '중복부서', 'SEOUL');

EXCEPTION
    WHEN DUP_VAL_ON_INDEX THEN
        DBMS_OUTPUT.PUT_LINE('중복 데이터입니다.');
    WHEN OTHERS THEN
        DBMS_OUTPUT.PUT_LINE('알 수 없는 오류가 발생했습니다.');
        DBMS_OUTPUT.PUT_LINE('오류 코드: ' || SQLCODE);
        DBMS_OUTPUT.PUT_LINE('오류 메시지: ' || SQLERRM);
END;
/

실무에서는 OTHERS에서 오류를 무시하지 말고 로그를 남기는 것이 중요합니다.

15-6. 사용자 정의 예외

DECLARE
    v_score NUMBER := 120;

BEGIN

    END IF;

EXCEPTION
    WHEN e_invalid_score THEN

END;

16. SQL 예제 카드 - COMMIT과 ROLLBACK

PL/SQL 안에서 DML을 실행하면 트랜잭션 처리를 생각해야 합니다. COMMIT은 변경 사항을 확정하고, ROLLBACK은 마지막 COMMIT 이후 변경 사항을 취소합니다.

ROLLBACK 예제

BEGIN
    UPDATE emp_plsql_copy
    SET sal = sal + 100
    WHERE deptno = 10;

    DBMS_OUTPUT.PUT_LINE(SQL%ROWCOUNT || '개의 급여가 수정되었습니다.');

    ROLLBACK;
    DBMS_OUTPUT.PUT_LINE('ROLLBACK 했습니다.');
END;
/

변경 후 ROLLBACK하면 급여 수정이 취소됩니다.

Spring 연결 주의

@Transactional
public void processOrder(OrderRequest request) {
    // Java Service 계층에서 트랜잭션 관리
}

Spring에서 호출하는 프로시저 내부에 무조건 COMMIT을 넣으면
Java 쪽 롤백 제어와 충돌할 수 있습니다.

18. Java·Spring·Oracle 프로젝트 연결

PL/SQL 기초 블록은 이후 저장 프로시저, 함수, 트리거로 확장됩니다. Spring 프로젝트에서는 MyBatis, JdbcTemplate, JPA를 통해 Oracle 프로시저를 호출할 수 있습니다.

18-1. Java Service 흐름

@Transactional
public void updateSalary(Long empno, int salary) {
    Employee emp = employeeRepository.findById(empno)
        .orElseThrow(() -> new IllegalArgumentException("사원이 없습니다."));

    emp.changeSalary(salary);
}

애플리케이션 서버에서 업무 흐름을 처리합니다.

18-2. PL/SQL 흐름

DECLARE
    v_empno EMP.EMPNO%TYPE := 7369;
    v_sal   EMP.SAL%TYPE := 1200;
BEGIN
    UPDATE EMP
    SET SAL = v_sal
    WHERE EMPNO = v_empno;

    IF SQL%NOTFOUND THEN
        RAISE_APPLICATION_ERROR(-20001, '사원이 없습니다.');
    END IF;
END;
/

Oracle DB 안에서 업무 흐름을 처리합니다.

18-3. 프로시저로 확장

CREATE OR REPLACE PROCEDURE update_salary_proc (
    p_empno IN EMP.EMPNO%TYPE,
    p_sal   IN EMP.SAL%TYPE,
    p_msg   OUT VARCHAR2
)
IS
BEGIN
    UPDATE EMP
    SET SAL = p_sal
    WHERE EMPNO = p_empno;

    IF SQL%FOUND THEN
        p_msg := SQL%ROWCOUNT || '개의 행이 수정되었습니다.';
    ELSE
        p_msg := '해당 사원이 없습니다.';
    END IF;
END;
/

        #{resultMessage, mode=OUT, jdbcType=VARCHAR}
    ) }
</select>

Mistake

19. 자주 하는 실수

  • 실수 1. 마지막 /를 빼먹음 END;는 블록의 끝이고, /는 SQL Developer에게 실행하라는 명령입니다.
  • 실수 2. END 뒤에 세미콜론을 안 붙임 END 다음에는 반드시 세미콜론을 붙여 END; 형태로 작성합니다.
  • 실수 3. SET SERVEROUTPUT ON을 안 함 DBMS_OUTPUT.PUT_LINE 결과가 안 보이면 출력 설정과 DBMS Output 창을 확인합니다.
  • 실수 4. 대입 연산자를 =로 씀 PL/SQL에서 대입은 := 이고, 비교는 = 입니다.
  • 실수 5. ELSIF를 ELSEIF로 씀 PL/SQL 조건문은 ELSEIF가 아니라 ELSIF입니다.
  • 실수 6. SELECT INTO 결과가 여러 행 SELECT INTO는 정확히 1행만 처리합니다. 여러 행은 커서가 필요합니다.
  • 실수 7. SELECT INTO 결과가 없음 결과가 없으면 NO_DATA_FOUND 예외가 발생하므로 EXCEPTION 처리를 고려합니다.
  • 실수 8. 문자열 대체 변수에 따옴표를 안 붙임 문자 입력값은 '&p_name'처럼 작은따옴표로 감싸야 합니다.
  • 실수 9. 무한 반복 LOOP문에는 EXIT WHEN 조건을 반드시 확인하고, WHILE문에서는 증가식을 빠뜨리지 않습니다.
  • 실수 10. SQL%FOUND를 직접 출력함 SQL%FOUND는 BOOLEAN이라 바로 출력하지 말고 IF문에서 판단합니다.
  • 실수 11. 원본 EMP를 바로 수정함 실습에서는 emp_plsql_copy 같은 복사 테이블을 만들어 사용하는 것이 안전합니다.
  • 실수 12. Spring 호출 프로시저에 무조건 COMMIT을 넣음 Spring의 @Transactional과 충돌할 수 있으므로 트랜잭션 책임을 명확히 해야 합니다.

Interview

21. 면접 질문 카드

Q1. PL/SQL이 무엇인가요?

PL/SQL은 Oracle에서 제공하는 SQL 확장 절차형 언어입니다. SQL의 DML과 SELECT문을 변수, 조건문, 반복문, 예외 처리와 함께 블록 단위로 실행할 수 있게 해줍니다.

Q2. SQL과 PL/SQL의 차이는 무엇인가요?

SQL은 데이터를 조회하거나 조작하는 질의어이고, PL/SQL은 SQL을 절차적으로 묶어 실행할 수 있는 프로그래밍 언어입니다.
SQL은 보통 한 문장 단위이고, PL/SQL은 블록 단위로 실행됩니다.

Q3. PL/SQL 블록 구조를 설명해보세요.

PL/SQL 블록은 DECLARE, BEGIN, EXCEPTION, END로 구성됩니다. DECLARE는 선언부, BEGIN은 실행부, EXCEPTION은 예외 처리부입니다.
BEGIN과 END는 필수이고, DECLARE와 EXCEPTION은 선택입니다.

Q4. END;와 /의 차이는 무엇인가요?

END;는 PL/SQL 블록 문법의 끝을 의미합니다. /는 SQL Developer나 SQL*Plus에게 지금까지 작성한 PL/SQL 블록을 실행하라고 알려주는 명령입니다.

Q5. PL/SQL에서 대입 연산자는 무엇인가요?

PL/SQL의 대입 연산자는 := 입니다. = 는 조건 비교에 사용합니다.

Q6. %TYPE은 왜 사용하나요?

%TYPE은 테이블 컬럼의 자료형을 변수에 그대로 적용하기 위해 사용합니다. 컬럼 타입이나 길이가 변경되어도 변수 선언을 직접 수정할 필요가 줄어 유지보수에 유리합니다.

Q7. SELECT INTO의 규칙은 무엇인가요?

SELECT 결과를 PL/SQL 변수에 담으려면 INTO가 필요합니다. SELECT 컬럼 수와 INTO 변수 수가 같아야 하며, 결과는 정확히 1행이어야 합니다.
0행이면 NO_DATA_FOUND, 2행 이상이면 TOO_MANY_ROWS 예외가 발생합니다.

Q8. 여러 행을 SELECT해서 처리하려면 어떻게 하나요?

SELECT INTO는 한 행만 처리할 수 있습니다. 여러 행을 한 행씩 처리하려면 커서를 사용해야 하며, 이후 커서 단원에서 CURSOR 선언, OPEN, FETCH, CLOSE 흐름으로 배웁니다.

Q9. SQL%FOUND와 SQL%ROWCOUNT는 무엇인가요?

SQL%FOUND는 가장 최근 DML이 하나 이상의 행에 영향을 줬으면 TRUE입니다. SQL%ROWCOUNT는 가장 최근 DML이 영향을 준 행 수를 반환합니다.

Q10. WHEN OTHERS는 무엇인가요?

WHEN OTHERS는 명시적으로 처리하지 않은 모든 예외를 잡는 구문입니다.
실무에서는 오류를 무시하지 말고 SQLCODE, SQLERRM을 로그로 남기거나 필요한 경우 다시 예외를 발생시켜야 합니다.

Q11. PL/SQL과 Java의 예외 처리는 어떻게 비교할 수 있나요?

Java의 try-catch와 PL/SQL의 BEGIN-EXCEPTION-END가 비슷합니다. 실행부에서 오류가 발생하면 EXCEPTION 영역의 WHEN 절에서 처리합니다.

Q12. Spring 프로젝트에서 PL/SQL을 사용할 때 주의할 점은?

Spring에서는 보통 @Transactional로 트랜잭션을 관리합니다.
Spring이 호출하는 프로시저 내부에서 무조건 COMMIT이나 ROLLBACK을 수행하면 애플리케이션 트랜잭션과 충돌할 수 있습니다.
또한 DBMS_OUTPUT은 사용자 응답용이 아니므로 OUT 파라미터, 결과 셋, 예외 등을 통해 결과를 전달하는 것이 좋습니다.
