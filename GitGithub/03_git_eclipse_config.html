Git/GitHub - Chapter 03

Eclipse에서 Git 사용자 설정하기

Eclipse에서 Git 사용자 설정하기는 Git으로 커밋을 만들기 전에 Git에게 “이 커밋을 작성하는 사람이 누구인지” 알려주는 준비 단계입니다.
Git은 커밋을 만들 때 파일 내용뿐만 아니라 작성자 이름, 작성자 이메일, 작성 시간, 커밋 메시지도 함께 기록합니다.
이번 단원에서는 Eclipse의 Git Configuration 화면에서 user.name과 user.email을 설정하고, 이후 커밋 실습에 필요한 Git Staging 탭까지 준비합니다.

01

Git 사용자 설정을 공부한다는 것

Git 사용자 설정은 프로젝트 파일을 수정하는 작업이 아닙니다. 대신 앞으로 Git이 커밋을 만들 때 사용할 작성자 이름과 작성자 이메일을 등록하는 작업입니다.
Git은 버전 관리 도구이기 때문에 “무엇이 바뀌었는가?”뿐만 아니라 “누가 바꿨는가?”도 함께 기록합니다.

Git History에서 누가 커밋을 만들었는지
보여주는 이름입니다.

Email

user.email

커밋 작성자 이메일입니다.
GitHub 계정과 커밋을 연결하는 데도
중요할 수 있습니다.

Commit Author

커밋에는 작성자, 날짜, 메시지,

Setup

02

커밋에는 어떤 정보가 남을까?

Git에서 커밋은 단순 저장이 아니라 하나의 버전 기록입니다.
커밋에는 파일 변경 내용뿐만 아니라 작성자 정보와 커밋 메시지도 함께 남습니다.
이번 단원에서 설정하는 값은 아래 커밋 정보 중 Author 부분에 사용됩니다.

커밋 이력 예시

commit abc1234
Author: Hong Gil Dong <hong@example.com>
Date:   2026-07-10

    Add Git Eclipse config lesson

각 정보의 의미

  • commit: 커밋을 구분하는 고유 ID입니다.
  • Author: 커밋 작성자 이름과 이메일입니다.
  • Date: 커밋이 만들어진 시간입니다.
  • Message: 이 커밋이 어떤 작업인지 설명합니다.

왜 처음에 정확히 설정해야 할까?

user.name과 user.email을 잘못 설정하면 커밋 작성자가 test, unknown, 잘못된 이메일처럼 남을 수 있습니다.
설정을 나중에 바꿔도 이미 만들어진 기존 커밋의 작성자 정보가 자동으로 바뀌지는 않습니다.

05

User Settings는 어떤 범위의 설정일까?

Eclipse Git Configuration에는 설정 범위가 여러 개 있을 수 있습니다. 처음 Git을 사용하는 단계에서는 보통 User Settings에 이름과 이메일을 등록하면 됩니다.

System

System Settings

컴퓨터 전체 시스템 수준의 Git 설정입니다.
처음 공부할 때는 보통 직접 수정하지 않습니다.

User

User Settings

현재 사용자 계정에 적용되는 Git 설정입니다.
이번 단원에서 사용하는 설정 범위입니다.

Repository

Repository Settings

특정 Git 저장소에만 적용되는 설정입니다.
프로젝트별 작성자 정보를 다르게 할 때 사용할 수 있습니다.

설정 범위 이해

System Settings
  → 컴퓨터 전체 수준

User Settings
  → 현재 사용자 계정 수준
  → 처음 공부할 때 주로 사용

Repository Settings
  → 특정 프로젝트 저장소 수준
06

Git Staging 탭 준비하기

이번 단원은 사용자 설정이 핵심이지만, Eclipse에서 Git을 공부할 때는 Git Staging 탭도 함께 준비해두는 것이 좋습니다.
이 탭은 앞으로 Add to Index, Commit, Commit and Push를 눈으로 확인하는 중요한 화면입니다.

Git Staging 탭 열기

Window
  → Show View
  → Other
  → Git 검색
  → Git Staging

Git Staging 탭에서 보는 것

  • Unstaged Changes: 아직 스테이지에 올라가지 않은 변경
  • Staged Changes: 이번 커밋에 포함될 변경
  • Commit Message: 커밋 메시지 작성 영역
  • Commit: 로컬 저장소에 커밋
  • Commit and Push: 커밋 후 원격 저장소에 업로드
Git Staging 탭의 역할

Git Staging 탭은 Git의 Working Directory → Staging Area → Local Repository 흐름을 Eclipse 화면에서 확인할 수 있게 해주는 중요한 도구입니다.

07

CLI 명령어와 연결하기

Eclipse의 User Settings에서 설정하는 값은 Git CLI의 git config 명령어와 연결됩니다.
Eclipse에서 설정하더라도 CLI 명령어를 함께 알아두면 VS Code나 Git Bash에서도 같은 원리로 사용할 수 있습니다.

Eclipse 작업 CLI 명령어 의미
User Settings에서 user.name 추가 git config --global user.name "이름" 커밋 작성자 이름을 설정합니다.
User Settings에서 user.email 추가 git config --global user.email "이메일" 커밋 작성자 이메일을 설정합니다.
설정된 이름 확인 git config user.name 현재 적용되는 작성자 이름을 확인합니다.
설정된 이메일 확인 git config user.email 현재 적용되는 작성자 이메일을 확인합니다.
전체 설정 확인 git config --list Git 설정 목록을 확인합니다.

CLI 설정 예시

git config --global user.name "Hong Gil Dong"
git config --global user.email "hong@example.com"

git config user.name
git config user.email

Git 사용자 정보는 한 번 설정하고 끝내기보다, 실제 커밋을 만들기 전에 제대로 입력되었는지 확인하는 습관이 좋습니다. 특히 팀 프로젝트나 포트폴리오 프로젝트에서는 작성자 정보가 정확해야 합니다.

이름 확인

user.name에 실제 커밋 작성자로
표시할 이름이 들어갔는지 확인합니다.

이메일 확인

user.email에 오타가 없는지,
GitHub 계정과 연결할 이메일인지 확인합니다.

User Settings 확인

System Settings가 아니라
User Settings에 설정했는지 확인합니다.

Apply 확인

값을 입력한 뒤 Apply 또는
Apply and Close를 눌렀는지 확인합니다.

Git Staging 확인

앞으로 커밋 실습을 위해
Git Staging 탭을 열 수 있는지 확인합니다.

기존 커밋 구분

설정 변경은 이후 커밋에 적용되며,
기존 커밋이 자동 변경되지는 않습니다.

09

자주 하는 실수

Git 사용자 설정은 간단해 보이지만, key 이름이나 설정 목적을 잘못 이해해서 실수하는 경우가 많습니다. 아래 항목은 처음 Git을 설정할 때 특히 조심해야 합니다.

1. key 이름을 잘못 입력

username, userName, user.mail은 잘못된 key입니다.
정확히 user.name, user.email로 입력해야 합니다.

2. 이름과 이메일을 반대로 입력

user.name에는 이름을 넣고,
user.email에는 이메일을 넣어야 합니다.

3. user.email에 비밀번호 입력

user.email은 커밋 작성자 이메일입니다.
GitHub 비밀번호나 토큰을 넣는 곳이 아닙니다.

4. Apply를 누르지 않음

값을 입력한 뒤 Apply 또는
Apply and Close를 눌러야 설정이 저장됩니다.

5. GitHub 인증과 혼동

user.name과 user.email은 커밋 작성자 정보이고,
GitHub 인증은 토큰과 권한 문제입니다.

6. 기존 커밋도 바뀐다고 생각

설정을 바꿔도 기존 커밋 작성자 정보가
자동으로 바뀌지는 않습니다. 새 커밋부터 적용됩니다.

12

면접 포인트

Q1. Git에서 user.name과 user.email은 왜 설정하나요?

Git은 커밋마다 작성자 정보를 기록합니다. user.name은 커밋 작성자 이름이고, user.email은 커밋 작성자 이메일입니다. 이 정보를 통해 Git History에서 누가 어떤 변경을 만들었는지 확인할 수 있습니다.

Q2. user.name과 GitHub 아이디는 같은 건가요?

아닙니다. user.name은 커밋에 표시되는 작성자 이름이고, GitHub 아이디는 GitHub 서비스에서 사용하는 계정명입니다.

Q3. user.email은 GitHub 로그인 이메일인가요?

반드시 로그인 이메일이라는 뜻은 아니지만, 커밋 작성자 이메일로 사용됩니다. GitHub 계정에 등록된 이메일이나 GitHub no-reply 이메일을 사용하면 커밋이 계정과 연결되기 쉽습니다.

Q4. user.name과 user.email은 GitHub 인증과 같은가요?

아닙니다. user.name과 user.email은 커밋 작성자 정보이고, GitHub 인증은 원격 저장소에 push, pull할 때 필요한 권한과 토큰 문제입니다.

Q5. Eclipse에서 Git 사용자 정보는 어디에서 설정하나요?

Window → Preference → Version Control(Team) → Git → Configuration으로 이동한 뒤, User Settings 탭에서 Add Entry를 눌러 user.email과 user.name을 추가합니다.

Q6. Git Staging 탭은 왜 중요한가요?

Git Staging 탭은 Unstaged Changes, Staged Changes, Commit Message를 확인하고 Commit 또는 Commit and Push를 실행할 수 있는 화면입니다.
초보자는 이 탭을 통해 add와 commit 흐름을 시각적으로 이해할 수 있습니다.

Q7. user.name과 user.email을 나중에 바꾸면 기존 커밋도 바뀌나요?

아닙니다. 설정을 바꿔도 이미 만들어진 기존 커밋 작성자 정보는 자동으로 바뀌지 않습니다. 새 설정은 이후 새로 만드는 커밋부터 적용됩니다.

Q8. Please tell me who you are 메시지는 무슨 뜻인가요?

Git이 커밋 작성자 정보를 찾지 못했다는 뜻입니다. user.name과 user.email을 설정해야 합니다.

Q9. 403 Permission 오류는 user.email 문제인가요?

대부분 단순한 user.email 문제가 아니라 GitHub 원격 저장소 접근 권한이나 인증 정보 문제입니다. 저장소 권한, Personal Access Token, Windows 자격 증명 정보를 확인해야 합니다.

Q10. 팀 프로젝트에서 Git 사용자 설정이 중요한 이유는 무엇인가요?

팀 프로젝트에서는 커밋 작성자를 통해 누가 어떤 변경을 했는지 추적합니다. 작성자 정보가 정확해야 문제 발생 시 담당자를 빠르게 확인할 수 있습니다.

오늘의 암기 포인트

  • user.name은 커밋 작성자 이름입니다.
  • user.email은 커밋 작성자 이메일입니다.
  • user.name과 user.email은 GitHub 로그인 정보가 아니라 커밋 작성자 정보입니다.
  • Eclipse에서는 Window → Preference → Version Control(Team) → Git → Configuration에서 설정합니다.
  • User Settings 탭에서 Add Entry로 user.email과 user.name을 추가합니다.
  • GitHub 인증은 user.name / user.email 설정과 별개이며, 원격 저장소 권한이나 토큰과 관련됩니다.
  • Git Staging 탭은 Unstaged Changes, Staged Changes, Commit Message, Commit 버튼을 확인하는 중요한 화면입니다.

헷갈리기 쉬운 개념

user.name vs GitHub ID

user.name은 커밋 작성자 이름이고,
GitHub ID는 GitHub 서비스 계정명입니다.

user.email vs GitHub Token

user.email은 커밋에 남는 이메일이고,
Token은 원격 저장소 인증에 쓰입니다.

Commit Author vs Credential

Commit Author는 기록용 정보이고,
Credential은 접근 권한을 확인하는 인증 정보입니다.

User Settings
