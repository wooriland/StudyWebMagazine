06. CSS Flexbox

이 페이지에서는 display: flex를 시작으로 Flex Container와 Flex Item, 주축과 교차축, 정렬, 줄바꿈, 크기 배분을 학습합니다.
Flexbox는 단순한 CSS 문법 암기가 아니라, Oracle에서 조회한 여러 행의 데이터를 Java/Spring 화면에서 카드·메뉴·검색 영역·대시보드 형태로 보기 좋게 배치하는 핵심 레이아웃 기술입니다.

display: flex Flex Container Flex Item Main Axis Cross Axis flex-wrap flex: 1

Concept Card

단원 개념 카드: Flexbox를 왜 배우는가?

Flexbox는 HTML 요소를 한 방향으로 유연하게 배치하기 위한 CSS 레이아웃 방식입니다.
웹 페이지에서 메뉴를 가로로 정렬하거나, 상품 카드를 여러 줄로 배치하거나, 버튼을 오른쪽 끝으로 보내거나, 박스를 정확히 가운데에 놓아야 할 때 사용합니다.

Flexbox 한 문장 정리

부모에게 display: flex를 주고, 부모 속성으로 전체 방향과 정렬을 잡으며, 자식 속성으로 개별 크기와 순서를 조절하는 CSS 레이아웃 방식입니다.

01

Flex Container

display: flex가 적용된 부모 요소입니다.
전체 배치 방향, 줄바꿈, 주축 정렬, 교차축 정렬을 결정합니다. 

.product-list {
    display: flex;
    flex-wrap: wrap;
    gap: 20px;
}
02

Flex Item

Flex Container의 직접 자식 요소입니다.
각 아이템의 크기, 증가 비율, 축소 비율, 개별 정렬, 보이는 순서를 조절합니다. 

.product-card {
    flex: 0 1 240px;
}
03

주축과 교차축

flex-direction이 주축을 결정합니다.
justify-content는 주축 정렬, align-items는 교차축 정렬입니다. 

.center-box {
    display: flex;
    justify-content: center;
    align-items: center;
}
04

실무에서 쓰는 이유

DB에서 조회한 상품, 게시글, 회원, 주문 데이터를
반복 출력하면 개수가 매번 달라집니다.
Flexbox는 이런 반복 요소를 카드 목록처럼 자연스럽게 배치합니다. 

.card-list {
    display: flex;
    flex-wrap: wrap;
}

Visual Card

그림처럼 이해: 부모가 방향을 만들고, 자식이 줄을 선다

Flexbox는 부모와 자식의 관계로 이해해야 합니다. 부모는 운동장이고, 자식은 줄 서는 학생이라고 생각하면 쉽습니다.
운동장에 display: flex라는 규칙을 붙이면 학생들이 기본적으로 왼쪽에서 오른쪽으로 줄을 섭니다.

SQL Example Card

SQL 예제 카드: Oracle 데이터가 Flexbox 카드 목록이 되는 과정

Flexbox는 SQL 문법이 아닙니다. 하지만 Java / Spring / Oracle 프로젝트에서는 SQL로 조회한 여러 행이 HTML에서 반복 출력되고, 그 반복 요소를 Flexbox가 화면에 배치합니다.

1. SQL Developer에서 상품 카드용 테이블 만들기

CREATE TABLE product_card (
    product_id   NUMBER PRIMARY KEY,
    product_name VARCHAR2(100) NOT NULL,
    category     VARCHAR2(50) NOT NULL,
    price        NUMBER NOT NULL,
    stock_qty    NUMBER DEFAULT 0,
    created_at   DATE DEFAULT SYSDATE
);

이 테이블의 한 행은 화면의 상품 카드 한 개가 됩니다. 즉 SELECT 결과가 5행이면 화면에 카드 5개가 반복 출력됩니다.

3. 화면에 보여줄 데이터 조회

SELECT
    product_id,
    product_name,
    category,
    price,
    TO_CHAR(price, 'FM999,999,999') || '원' AS price_text,
    stock_qty,
    CASE
        WHEN stock_qty = 0 THEN '품절'
        WHEN stock_qty < 10 THEN '재고 부족'
        ELSE '판매 가능'
    END AS stock_status
FROM product_card
ORDER BY product_id;

이 결과를 Spring Controller에서 products라는 이름으로 화면에 넘기면, HTML에서는 반복문으로 카드를 만들고 CSS에서는 Flexbox로 배치합니다.

4. Spring 화면에서 반복 출력되는 HTML 예시

<div class="product-list">
    <article class="product-card" th:each="product : ${products}">
        <div class="product-category" th:text="${product.category}">전자기기</div>
        <h3 class="product-name" th:text="${product.productName}">무선 마우스</h3>
        <p class="product-price" th:text="${product.priceText}">25,000원</p>
        <div class="product-bottom">
            <span th:text="${product.stockStatus}">판매 가능</span>
            <button type="button">상세보기</button>
        </div>
    </article>
</div>

Mistake Card

자주 하는 실수 카드: 화면이 안 맞을 때 먼저 확인할 것

실수 1. display: flex를 자식에게 줌

배치하고 싶은 대상이 .item이라면,
.item의 부모인 .container에 display: flex를 줘야 합니다.

실수 2. justify-content를 무조건 가로 정렬로 외움

정확한 설명은 “주축 정렬”입니다.
flex-direction: column이면 justify-content는
세로 방향 정렬에 영향을 줍니다.

실수 3. 한 줄인데 align-content를 사용함

align-content는 flex-wrap: wrap으로 여러 줄이 생겼을 때 의미가 있습니다.
한 줄 정렬은 align-items를 사용합니다.

실수 4. 카드 목록에 flex-wrap을 빼먹음

DB에서 조회되는 데이터는 개수가 고정되어 있지 않습니다.
상품·게시글·회원 카드 목록은 flex-wrap: wrap을 먼저 고려합니다.

Interview Card

면접 질문 카드: 답변 문장까지 준비하기

면접에서는 속성 이름만 말하기보다 왜 쓰는지, 어떤 문제를 해결했는지, 프로젝트와 어떻게 연결했는지까지 말해야 합니다.

Q1. Flexbox가 무엇인가요?

Flexbox는 CSS의 1차원 레이아웃 모델입니다.
부모 요소에 display: flex를 적용하면 자식 요소들이 Flex Item이 되고,
주축과 교차축을 기준으로 방향, 정렬, 간격, 크기 배분을 제어할 수 있습니다.

Q2. justify-content와 align-items의 차이는?

justify-content는 주축 방향 정렬이고,
align-items는 교차축 방향 정렬입니다.
기본값인 row에서는 주축이 가로라서
justify-content가 가로 정렬처럼 보이지만,
column에서는 세로 정렬이 됩니다.

Q3. flex: 1은 어떤 의미인가요?

flex: 1은 아이템이 컨테이너의 남은 공간을
같은 비율로 나누어 차지하도록 만드는 축약 표현입니다.
대시보드 통계 카드, 검색창 input, 본문 영역처럼
남는 공간을 채워야 할 때 자주 사용합니다.

Q4. 프로젝트에서 Flexbox를 어디에 사용했나요?

Oracle에서 상품 목록을 조회하고
Spring Controller에서 List<ProductDto>를 화면으로 전달한 뒤,
Thymeleaf에서 상품 카드를 반복 출력했습니다.
카드 목록 부모에 display: flex와 flex-wrap: wrap을 적용하여
상품 개수가 많아져도 자동으로 줄바꿈되도록 구현했습니다.

Q5. align-items와 align-content는 어떻게 다른가요?

align-items는 한 줄 안에서 아이템들을 교차축 방향으로 정렬합니다.
align-content는 여러 줄 전체를 교차축 방향으로 정렬합니다.
따라서 align-content는 flex-wrap: wrap이고
실제로 여러 줄이 생겼을 때 의미가 있습니다.

Q6. Flexbox와 Grid의 차이는?

Flexbox는 행 또는 열 중심의 1차원 배치에 적합하고,
Grid는 행과 열을 동시에 다루는 2차원 레이아웃에 적합합니다.
메뉴, 버튼 그룹, 카드 내부 정렬은 Flexbox가 좋고,
복잡한 전체 페이지 구획은 Grid가 더 적합합니다.

학습 포인트

이 페이지는 Flexbox를 단순한 CSS 문법이 아니라, Java / Spring / Oracle 프로젝트에서 조회 데이터를 화면에 배치하는 기술로 연결해서 공부하는 단원입니다.

  • 부모에게 display: flex를 주면 자식이 Flex Item이 됩니다.
  • justify-content는 주축, align-items는 교차축 정렬입니다.
  • 상품·게시글·회원 카드 목록처럼 데이터 개수가 바뀌는 화면에는 flex-wrap: wrap이 중요합니다.
  • flex: 1은 남은 공간을 비율로 나누어 차지하게 할 때 사용합니다.
  • margin-left: auto는 로그인 버튼처럼 특정 아이템만 오른쪽 끝으로 보낼 때 유용합니다.
중요: Flexbox 속성이 안 먹는 것처럼 보이면 먼저 “부모에게 줘야 하는 속성인지, 자식에게 줘야 하는 속성인지”를 확인하세요.

Flexbox 속성 요약

Container 속성

  • display : Flex Container 생성
  • flex-direction : 주축 방향 설정
  • flex-wrap : 줄바꿈 여부 설정
  • justify-content : 주축 정렬
  • align-items : 교차축 정렬
  • align-content : 여러 줄 정렬

Item 속성

  • flex-basis : 기준 크기
  • flex-grow : 남는 공간 확장 비율
  • flex-shrink : 부족한 공간 축소 비율
  • flex : grow, shrink, basis 축약
  • align-self : 특정 아이템 개별 정렬
  • order : 보이는 순서 변경
