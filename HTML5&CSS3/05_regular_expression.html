05. JavaScript Regular Expression

이 페이지에서는 HTML5 Form에 입력된 값을 JavaScript 정규표현식으로 검사하는 방법을 공부합니다.
정규표현식은 문자열이 특정한 패턴을 만족하는지 확인할 때 사용하는 표현식입니다.
test(), match(), replace(), split(), exec() 메서드와 함께 ^, $, [], {}, (), |, \d, \w, \s 같은 패턴 문자를 익힙니다.

정규표현식이 필요한 이유

HTML5 Form은 required, type="email", type="url" 같은 기본 검증 기능을 제공합니다. 하지만 실제 웹 페이지에서는 더 세밀한 검증이 필요한 경우가 많습니다.

예를 들어 이런 검사가 필요합니다.

  • 이름은 한글 2~5글자인가?
  • 아이디는 영문 소문자와 숫자 4~12자인가?
  • 이메일은 기본적인 이메일 형식을 갖추었는가?
  • 연락처는 01012345678 또는 010-1234-5678 형식인가?
  • 학습 메모가 너무 짧지 않은가?

이런 규칙을 코드로 표현할 때 사용하는 것이 바로 정규표현식 Regular Expression입니다.

JavaScript 정규표현식 기본 형태

1. 정규표현식 리터럴 방식

var pattern = /^[0-9]+$/;
var result = pattern.test("12345");

/정규표현식/flags 형태로 작성하는 방식입니다. 고정된 패턴을 사용할 때 가장 많이 사용합니다.

2. new RegExp() 방식

var pattern = new RegExp("^\\d+$");
var result = pattern.test("12345");

문자열로 정규표현식을 만들 때 사용하는 방식입니다. 이때 \d 같은 표현은 문자열 안에서 \\d처럼 한 번 더 이스케이프해야 합니다.

정규표현식 읽는 방법

^ [0-9] + $
문자열 시작 숫자 하나 1개 이상 반복 문자열 끝

/^[0-9]+$/는 처음부터 끝까지 숫자로만 이루어진 문자열을 의미합니다.

정규표현식으로 학습 기록 Form 검증하기

아래 Form은 서버와 연결되어 있지 않습니다. 대신 JavaScript에서 submit 이벤트를 가로채고, 정규표현식으로 입력값을 검사한 뒤 결과 메시지를 화면에 출력합니다.

기본 정보

한글 2~5글자로 입력합니다. 예: 홍길동, 김철수

영문 소문자와 숫자만 사용할 수 있습니다. 예: student01

기본 이메일 형식을 검사합니다. 예: study@example.com

하이픈이 있어도 되고 없어도 됩니다.

정규표현식 학습 기록

datalist를 사용하면 선택과 직접 입력을 함께 제공할 수 있습니다.

어려움 보통 이해함

최소 10자 이상 작성해야 합니다.

입력값을 작성한 뒤 정규표현식 검증하기 버튼을 눌러보세요.

이번 Form에서 사용하는 정규표현식

검사 대상 정규표현식 의미
이름 /^[가-힣]{2,5}$/ 한글 2~5글자
아이디 /^[a-z0-9]{4,12}$/ 영문 소문자와 숫자 4~12자
이메일 /^[^\s@]+@[^\s@]+\.[^\s@]+$/ 공백과 @ 위치를 기준으로 기본 이메일 형식 검사
연락처 /^010-?[0-9]{4}-?[0-9]{4}$/ 01012345678 또는 010-1234-5678 허용
학습 메모 /.{10,}/ 10자 이상 작성 여부 검사

메서드별 실습 예제

test()

문자열이 정규표현식과 일치하는지 true 또는 false로 반환합니다. 

var pattern = /^[0-9]+$/;
pattern.test("12345"); // true

match()

문자열에서 정규표현식에 맞는 값을 찾아 배열로 반환합니다. 

"가격은 3000원".match(/\d+/);
// ["3000"]

replace()

정규표현식에 맞는 부분을 다른 문자열로 바꿉니다. 

"010-1234-5678".replace(/-/g, "");
// "01012345678"

split()

정규표현식에 맞는 구분자를 기준으로 문자열을 나눕니다. 

"HTML,CSS JS".split(/[,\\s]+/);
// ["HTML", "CSS", "JS"]

exec()

첫 번째 일치 결과와 그룹을 자세히 추출할 때 사용합니다. 

var result = /(\d{3})-(\d{4})/.exec("010-1234");
result[1]; // "010"

학습 포인트

이 페이지는 서버, DB, 백엔드 API 없이 브라우저에서 정규표현식 검증 흐름을 공부하는 실습 페이지입니다.

  • submit 이벤트가 발생하면 JavaScript가 입력값을 검사합니다.
  • 입력값이 조건에 맞지 않으면 오류 메시지를 출력합니다.
  • 조건을 모두 만족하면 성공 메시지를 출력합니다.
  • 실제 서버로 데이터가 전송되지는 않습니다.
중요: 프론트엔드 정규표현식 검증은 사용자 편의를 위한 1차 검증입니다. 실제 프로젝트에서는 반드시 백엔드에서도 같은 검증을 수행해야 합니다.

정규표현식 핵심 요약

  • ^ : 문자열의 시작
  • $ : 문자열의 끝
  • [] : 문자 후보 중 하나
  • [^] : 특정 문자를 제외
  • {} : 반복 횟수
  • () : 그룹
  • | : 또는
  • \d : 숫자
  • \w : 영문자, 숫자, 밑줄
  • \s : 공백
