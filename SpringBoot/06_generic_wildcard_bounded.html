Spring Boot - Chapter 06

Generic 와일드카드와 제한 타입

Generic 기본이 “어떤 타입을 사용할 것인가”를 정하는 문법이었다면,
Generic 심화는 “어떤 범위의 타입까지 허용할 것인가”를 정하는 문법입니다.
이 단원에서는 <T extends ...>, ?, List<?>, List<Object>, List<? extends ...>, List<? super ...>를
Spring Boot 공통 응답 DTO와 페이징 응답 타입까지 연결해서 정리합니다.

<T extends ...> Wildcard ? List<?> List<Object> ? extends ? super PECS ApiResponse<T> PageResponse<T> ParameterizedTypeReference GrantedAuthority
학습 목표 1

<T extends ...>가 타입 파라미터의 상한선을
정하는 문법임을 이해합니다.

학습 목표 2

List<?>, List<Object>, ? extends, ? super의
차이를 구분합니다.

학습 목표 3

공통 API 응답, 페이징 응답, RestTemplate,
Security 권한 타입에서 Generic 심화 문법을 읽습니다.

01

Generic 심화는 타입의 범위를 정하는 문법이다

지난 단원에서는 List<UserResponse>, ResponseEntity<LoginResponse>, JpaRepository<User, Long>처럼 타입을 명확히 지정하는 방법을 배웠습니다.
이번 단원에서는 “정확히 하나의 타입”만 지정하는 것이 아니라, 어떤 타입 범위까지 허용할 것인지를 다룹니다.
예를 들어 숫자 계열만 받고 싶으면 <T extends Number>, 어떤 리스트든 읽기만 하고 싶으면 List<?>,
Number 계열을 읽고 싶으면 List<? extends Number>, Integer를 넣고 싶으면 List<? super Integer>를 사용합니다.

Bounded Type

<T extends Number>

T는 Number 또는
Number의 자식 타입만 될 수 있습니다.

Wildcard

List<?>

정확한 타입은 모르지만
어떤 타입의 리스트든 받을 수 있습니다.

Upper Bound

List<? extends Number>

Number 계열 리스트를
읽기 중심으로 처리할 때 사용합니다.

Lower Bound

List<? super Integer>

Integer 값을 리스트에 넣는
쓰기 중심 코드에 사용합니다.

핵심 문장

Generic 기본은 타입을 정하는 문법이고, Generic 심화는 타입의 허용 범위와 읽기·쓰기 방향을 정하는 문법입니다.

T는 이름이 있는 타입 파라미터이고, ?는 이름 없는 미지의 타입입니다. 두 문법은 비슷해 보이지만 사용하는 위치와 목적이 다릅니다.

04

PECS 원칙: Producer Extends, Consumer Super

와일드카드를 외울 때는 PECS 원칙이 가장 좋습니다. 데이터를 주로 읽는 쪽이면 extends, 데이터를 주로 넣는 쪽이면 super를 사용합니다.

Producer Extends

List<? extends Number>

리스트가 Number 계열 값을 생산합니다.
메소드 입장에서는 값을 꺼내서 Number로 읽을 수 있습니다. 

double sum(List<? extends Number> list) {
    double total = 0;
    for (Number n : list) {
        total += n.doubleValue();
    }
    return total;
}
Consumer Super

List<? super Integer>

리스트가 Integer 값을 소비합니다. 메소드 입장에서는 Integer를 안전하게 넣을 수 있습니다. 

void addIntegers(List<? super Integer> list) {
    list.add(1);
    list.add(2);
    list.add(3);
}
암기법

읽기 중심이면 extends, 쓰기 중심이면 super. 즉, Producer Extends, Consumer Super입니다.

05

그림처럼 이해하기

List<?>, List<Object>, extends, super는 타입 계층 그림으로 이해하면 훨씬 쉽습니다.

List<?>

모든 타입의 리스트를 받을 수 있습니다.

List<String> List<Integer> List<UserResponse> List<Object>

List<Object>

Object를 요소 타입으로 선언한 리스트만 받을 수 있습니다.

List<Object>

? extends Number

Number Integer Double Long

Number 계열을 읽기 좋지만, 어떤 하위 타입인지 몰라 추가는 어렵습니다.

? super Integer

Object Number Integer

Integer를 넣기 좋지만, 꺼낼 때는 Object로 읽는 것이 안전합니다.

06

기본 Java 코드 예제

아래 예제들은 와일드카드와 제한 타입의 가장 기본적인 사용법입니다. 읽기 중심인지, 쓰기 중심인지 먼저 판단하는 습관을 들이면 됩니다.

1) <T extends Number>: Number 계열만 허용

import java.util.List;

public class NumberUtils {

    public static <T extends Number> double sum(List<T> numbers) {
        double total = 0;

        for (T number : numbers) {
            total += number.doubleValue();
        }

        return total;
    }

    public static void main(String[] args) {
        List<Integer> integers = List.of(1, 2, 3);
        List<Double> doubles = List.of(1.5, 2.5, 3.5);

        System.out.println(sum(integers));
        System.out.println(sum(doubles));
    }
}

2) List<?>: 어떤 리스트든 출력

import java.util.List;

public class WildcardPrinter {

    public static void printList(List<?> list) {
        for (Object item : list) {
            System.out.println(item);
        }
    }

    public static void main(String[] args) {
        List<String> names = List.of("kim", "lee");
        List<Integer> numbers = List.of(1, 2, 3);

        printList(names);
        printList(numbers);
    }
}

3) List<? extends Number>: 읽기 중심

import java.util.List;

public class NumberReader {

    public static void printNumbers(List<? extends Number> numbers) {
        for (Number number : numbers) {
            System.out.println(number.doubleValue());
        }
    }

    public static void main(String[] args) {
        List<Integer> integers = List.of(1, 2, 3);
        List<Double> doubles = List.of(1.1, 2.2, 3.3);

        printNumbers(integers);
        printNumbers(doubles);
    }
}

4) List<? super Integer>: 쓰기 중심

import java.util.ArrayList;
import java.util.List;

public class IntegerWriter {

    public static void addIntegers(List<? super Integer> target) {
        target.add(1);
        target.add(2);
        target.add(3);
    }

    public static void main(String[] args) {
        List<Integer> integers = new ArrayList<>();
        List<Number> numbers = new ArrayList<>();
        List<Object> objects = new ArrayList<>();

        addIntegers(integers);
        addIntegers(numbers);
        addIntegers(objects);

        System.out.println(integers);
        System.out.println(numbers);
        System.out.println(objects);
    }
}
07

Spring Boot 공통 응답과 페이징 응답

Spring Boot에서는 Generic을 사용해 API 응답 구조를 통일할 수 있습니다. 특히 목록 조회와 페이징 조회는 ApiResponse<T>, PageResponse<T>로 재사용 구조를 만드는 것이 좋습니다.

1) ApiResponse<T>

public record ApiResponse<T>(
    boolean ok,
    String message,
    T data
) {
    public static <T> ApiResponse<T> success(T data) {
        return new ApiResponse<>(true, "요청이 성공했습니다.", data);
    }

    public static <T> ApiResponse<T> fail(String message) {
        return new ApiResponse<>(false, message, null);
    }
}

2) PageResponse<T>

import org.springframework.data.domain.Page;

import java.util.List;
import java.util.function.Function;

public record PageResponse<T>(
    List<T> content,
    int page,
    int size,
    long totalElements,
    int totalPages,
    boolean first,
    boolean last
) {
    public static <E, D> PageResponse<D> from(
            Page<E> page,
            Function<E, D> mapper
    ) {
        List<D> content = page.getContent()
                .stream()
                .map(mapper)
                .toList();

        return new PageResponse<>(
                content,
                page.getNumber(),
                page.getSize(),
                page.getTotalElements(),
                page.getTotalPages(),
                page.isFirst(),
                page.isLast()
        );
    }
}

3) PostResponse DTO

public record PostResponse(
    Long id,
    String title,
    String writer
) {
    public static PostResponse from(Post post) {
        return new PostResponse(
                post.getId(),
                post.getTitle(),
                post.getWriter()
        );
    }
}

4) Controller

@RestController
@RequestMapping("/api/posts")
@RequiredArgsConstructor
public class PostController {

    private final PostService postService;

    @GetMapping
    public ResponseEntity<ApiResponse<PageResponse<PostResponse>>> findPosts(
            @PageableDefault(size = 10, sort = "id") Pageable pageable
    ) {
        PageResponse<PostResponse> response = postService.findPosts(pageable);

        return ResponseEntity.ok(
                ApiResponse.success(response)
        );
    }
}

5) Service

@Service
@RequiredArgsConstructor
public class PostService {

    private final PostRepository postRepository;

    @Transactional(readOnly = true)
    public PageResponse<PostResponse> findPosts(Pageable pageable) {
        Page<Post> postPage = postRepository.findAll(pageable);

        return PageResponse.from(postPage, PostResponse::from);
    }
}

1) 토큰 저장 함수

export function saveToken(token, keepLogin) {
  if (keepLogin) {
    localStorage.setItem("accessToken", token);
    sessionStorage.removeItem("accessToken");
  } else {
    sessionStorage.setItem("accessToken", token);
    localStorage.removeItem("accessToken");
  }
}

export function getAccessToken() {
  return (
    localStorage.getItem("accessToken") ||
    sessionStorage.getItem("accessToken")
  );
}

export function clearToken() {
  localStorage.removeItem("accessToken");
  sessionStorage.removeItem("accessToken");
}

2) Axios Interceptor

import axios from "axios";
import { getAccessToken, clearToken } from "../utils/tokenStorage";

const apiClient = axios.create({
  baseURL: "http://localhost:8080",
  headers: {
    "Content-Type": "application/json",
  },
});

apiClient.interceptors.request.use(
  (config) => {
    const token = getAccessToken();

    if (token) {
      config.headers.Authorization = `Bearer ${token}`;
    }

    return config;
  },
  (error) => Promise.reject(error)
);

apiClient.interceptors.response.use(
  (response) => response,
  (error) => {
    if (error.response?.status === 401) {
      clearToken();
      window.location.href = "/login";
    }

    return Promise.reject(error);
  }
);

export default apiClient;

4) React 목록 렌더링

import { useEffect, useState } from "react";
import { fetchPosts } from "../api/postApi";

function PostList() {
  const [posts, setPosts] = useState([]);
  const [pageInfo, setPageInfo] = useState({
    page: 0,
    totalPages: 0,
    first: true,
    last: false,
  });

  const [message, setMessage] = useState("");

  useEffect(() => {
    async function loadPosts() {
      try {
        const result = await fetchPosts(0, 10);

        if (result.ok) {
          setPosts(result.data.content);
          setPageInfo({
            page: result.data.page,
            totalPages: result.data.totalPages,
            first: result.data.first,
            last: result.data.last,
          });
          setMessage(result.message);
        } else {
          setMessage(result.message);
        }
      } catch (error) {
        setMessage("게시글 목록을 불러오지 못했습니다.");
      }
    }

    loadPosts();
  }, []);

  return (
    <section>
      <h2>게시글 목록</h2>

      {message && <p>{message}</p>}

      <ul>
        {posts.map((post) => (
          <li key={post.id}>
            {post.title} / {post.writer}
          </li>
        ))}
      </ul>

      <p>
        현재 페이지: {pageInfo.page + 1} / {pageInfo.totalPages}
      </p>
    </section>
  );
}

export default PostList;

import { useState } from "react";
import { searchPosts } from "../api/postApi";

function PostSearchForm() {
  const [form, setForm] = useState({
    keyword: "",
    page: 0,
    size: 10,
  });

  const [posts, setPosts] = useState([]);
  const [pageInfo, setPageInfo] = useState({
    page: 0,
    totalPages: 0,
  });

  function handleChange(event) {
    const { name, value } = event.target;

    setForm((prev) => ({
      ...prev,
      [name]: name === "page" || name === "size" ? Number(value) : value,
    }));
  }

  async function handleSubmit(event) {
    event.preventDefault();

    const result = await searchPosts(form);

    if (result.ok) {
      setPosts(result.data.content);
      setPageInfo({
        page: result.data.page,
        totalPages: result.data.totalPages,
      });
    }
  }

  return (
    <section>
      <h2>게시글 검색</h2>

      <form onSubmit={handleSubmit}>
        <input
          name="keyword"
          value={form.keyword}
          onChange={handleChange}
          placeholder="검색어"
        />

        <select name="size" value={form.size} onChange={handleChange}>
          <option value="10">10개</option>
          <option value="20">20개</option>
          <option value="30">30개</option>
        </select>

        <button type="submit">검색</button>
      </form>

      <ul>
        {posts.map((post) => (
          <li key={post.id}>
            {post.title} / {post.writer}
          </li>
        ))}
      </ul>
    </section>
  );
}

export default PostSearchForm;

4) Service와 Repository

@Transactional(readOnly = true)
public PageResponse<PostResponse> search(PostSearchRequest request) {
    Pageable pageable = PageRequest.of(
            request.page(),
            request.size(),
            Sort.by(Sort.Direction.DESC, "id")
    );

    Page<Post> postPage = postRepository.findByTitleContaining(
            request.keyword(),
            pageable
    );

}

public interface PostRepository extends JpaRepository<Post, Long> {

    Page<Post> findByTitleContaining(String keyword, Pageable pageable);
}

RestTemplate과 Security에서 만나는 Generic 심화

Generic 심화는 JPA뿐 아니라 RestTemplate, Spring Security에서도 자주 등장합니다.
특히 외부 API가 JSON 배열을 반환할 때는 ParameterizedTypeReference로 List<DTO> 타입 정보를 보존해야 합니다.

1) RestTemplate에서 List<JsonPlace> 받기

ResponseEntity<List<JsonPlace>> response =
        restTemplate.exchange(
                url,
                HttpMethod.GET,
                entity,
                new ParameterizedTypeReference<List<JsonPlace>>() {}
        );

List<JsonPlace> places = response.getBody();

2) List.class 사용 시 문제

ResponseEntity<List> response = restTemplate.exchange(
        url,
        HttpMethod.GET,
        entity,
        List.class
);

// 이 경우 요소가 JsonPlace가 아니라 LinkedHashMap일 수 있습니다.

3) Spring Security 권한 타입

public class UserAdapter implements UserDetails {

    private final User user;

    @Override
    public Collection<? extends GrantedAuthority> getAuthorities() {
        return Collections.singletonList(
                new SimpleGrantedAuthority(user.getRole())
        );
    }

    @Override
    public String getUsername() {
        return user.getUsername();
    }

    @Override
    public String getPassword() {
        return user.getPassword();
    }
}
읽는 방법

Collection<? extends GrantedAuthority>는 “정확한 권한 구현 클래스는 몰라도 되지만, 최소한 GrantedAuthority로 읽을 수 있는 권한 목록”이라는 뜻입니다.

오류 메시지 읽는 법

Generic 심화 오류는 대부분 타입 범위와 읽기·쓰기 방향을 잘못 이해했을 때 발생합니다. 특히 List<Object>와 List<?>, extends와 super를 혼동할 때 많이 발생합니다.

오류 메시지
의미
확인할 것
List<String> cannot be converted to List<Object>
List<String>은 List<Object>의 자식이 아닙니다.
모든 리스트를 받으려면 List<?>를 사용합니다.
capture of ? extends Number
List<? extends Number>에 값을 추가하려 했습니다.
읽기 전용으로 쓰거나, 추가가 목적이면 ? super Integer를 사용합니다.
unexpected type
제네릭 메소드 타입 파라미터 자리에 ?를 사용했을 수 있습니다.
<?>가 아니라 <T>처럼 이름 있는 타입 파라미터를 사용합니다.
cannot find symbol: doubleValue()
T가 Number 계열이라고 보장되지 않습니다.
<T extends Number>로 제한합니다.
LinkedHashMap cannot be cast to JsonPlace
RestTemplate에서 List.class로 받아

자주 하는 실수

1. List<Object>가
모든 리스트를 받는다고 착각

모든 타입의 리스트를 받으려면
List<?>를 사용해야 합니다.

2. extends 리스트에 값 추가

List<? extends Number>는
읽기에는 좋지만 값을 추가하기 어렵습니다.

3. super 리스트에서
Integer로 바로 읽기

List<? super Integer>는
꺼낼 때 Object로 읽는 것이 안전합니다.

4. 제네릭 메소드에 <?> 사용

제네릭 메소드 타입 파라미터에는
이름이 필요하므로 <T>를 사용해야 합니다.

5. <T super Integer> 작성

타입 파라미터 선언부에는
super를 사용할 수 없습니다.
List<? super Integer>로 작성합니다.

6. T와 ?를 같은 것으로 생각

T는 이름 있는 타입이고,
?는 이름 없는 미지의 타입입니다.

7. RestTemplate에서
List.class 사용

요소 타입 정보가 사라져
LinkedHashMap이 반환될 수 있습니다.

8. React에서
페이징 객체를 배열로 착각

PageResponse는 객체이고
실제 배열은 content에 있습니다.

공통 API 응답

14

면접 포인트

Q1. <T extends Number>는 무슨 뜻인가요?

T가 Number 또는 Number의 하위 타입이어야 한다는 뜻입니다. 이렇게 제한하면 메소드 내부에서 doubleValue() 같은 Number의 메소드를 안전하게 사용할 수 있습니다.

Q2. List<?>는 무엇인가요?

어떤 타입의 요소를 가진 리스트든 받을 수 있다는 뜻입니다. 다만 정확한 요소 타입을 모르기 때문에 내부에서는 요소를 Object로 읽는 것이 안전합니다.

Q3. List<?>와 List<Object>의 차이는 무엇인가요?

List<?>는 모든 타입의 리스트를 받을 수 있지만, List<Object>는 요소 타입이 Object로 선언된 리스트만 받을 수 있습니다. List<String>은 List<Object>의 자식이 아닙니다.

Q4. List<? extends Number>는 언제 사용하나요?

Number 계열 리스트를 읽기 중심으로 처리할 때 사용합니다. Integer, Double, Long 리스트를 받을 수 있고, 요소를 Number로 읽을 수 있습니다.

Q5. List<? super Integer>는 언제 사용하나요?

Integer 값을 리스트에 추가해야 할 때 사용합니다. List<Integer>, List<Number>, List<Object>에 Integer를 안전하게 추가할 수 있습니다.

Q6. PECS 원칙이 무엇인가요?

Producer Extends, Consumer Super의 약자입니다. 값을 읽어오는 생산자 역할이면 extends, 값을 넣는 소비자 역할이면 super를 사용한다는 원칙입니다.

Q7. PageResponse<T>를 왜 만드나요?

게시글, 회원, 댓글 등 페이징 응답의 구조는 같고 content의 요소 타입만 다르기 때문입니다. Generic을 사용하면 PageResponse 하나로 여러 DTO 페이징 응답을 재사용할 수 있습니다.

Q8. ParameterizedTypeReference<List<JsonPlace>>는 왜 쓰나요?

RestTemplate에서 List.class로 받으면 요소 타입 정보가 사라질 수 있습니다. ParameterizedTypeReference를 사용하면 List<JsonPlace>라는 Generic 타입 정보를 보존할 수 있습니다.

Q9. Collection<? extends GrantedAuthority>는 어떻게 읽나요?

GrantedAuthority 또는 그 하위 타입 객체들의 컬렉션이라는 뜻입니다. Spring Security에서 권한 목록을 읽기 중심으로 처리할 때 사용됩니다.

Q10. React와 Generic 심화는 어떻게 연결되나요?

Java의 Generic 타입 구조는 Spring Boot의 JSON 응답 구조와 연결됩니다.
ApiResponse<PageResponse<PostResponse>>라면 React에서는 response.data.data.content가 실제 게시글 배열입니다.

학습 메모

오늘의 암기 포인트

  • <T extends Number>는 T가 Number 또는 Number 하위 타입이어야 한다는 뜻입니다.
  • List<?>는 어떤 타입의 리스트든 받을 수 있습니다.
  • List<Object>는 Object 요소 타입으로 선언된 리스트만 받을 수 있습니다.
  • List<? extends Number>는 읽기 중심입니다.
  • List<? super Integer>는 쓰기 중심입니다.
  • ?는 제네릭 메소드의 타입 파라미터 이름으로 사용할 수 없습니다.
  • PageResponse<T>는 페이징 응답의 content 타입을 일반화합니다.

T vs ?

T는 이름 있는 타입,
?는 이름 없는 미지의 타입입니다.

List<?> vs List<Object>

모든 리스트를 받으려면
List<?>를 사용합니다.

extends vs super

extends는 읽기 중심,
super는 쓰기 중심입니다.

<T super ...>

Java 타입 파라미터 선언부에서는
사용할 수 없는 문법입니다.

PageResponse<T>

React에서 실제 배열은
