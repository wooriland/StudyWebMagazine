01

DataSource와 HikariCP를 공부한다는 것

React에서 GET /api/posts를 호출하면 Spring Boot는 Controller, Service, Repository를 지나 DB에 접근합니다. 이때 DB에 접근하려면 반드시 Connection이 필요합니다.
JDBC를 직접 공부할 때는 DriverManager.getConnection()으로 연결을 만들었지만, 실제 웹 API 서버에서는 매 요청마다 새 연결을 만들면 성능과 안정성 문제가 생깁니다.
그래서 Spring Boot에서는 DataSource를 DB 연결의 표준 입구로 사용하고, 그 뒤에서 HikariCP가 Connection Pool을 관리합니다.
요청이 들어오면 새 Connection을 만드는 것이 아니라 Pool에서 빌리고, 작업이 끝나면 close()를 통해 Pool에 반납합니다.

DriverManager 방식과 HikariCP 방식 비교

JDBC를 처음 배울 때는 DriverManager.getConnection()으로 Connection을 직접 만듭니다.
하지만 웹 서버에서는 매 요청마다 새 연결을 생성하는 방식이 부담스럽습니다.
HikariCP는 Connection을 미리 준비해두고 요청마다 빌려주는 방식으로 이 문제를 해결합니다.

Spring Boot DB 설정과 HikariCP 설정

Spring Boot는 spring.datasource.* 설정을 보고 DataSource를 자동 구성합니다. HikariCP 관련 설정은 spring.datasource.hikari.* 아래에 작성합니다.

application.yml

server:
  port: 8080

spring:
  application:
    name: datasource-hikari-study

  datasource:
    driver-class-name: oracle.jdbc.OracleDriver
    url: ${DB_URL:jdbc:oracle:thin:@localhost:1521/XEPDB1}
    username: ${DB_USERNAME:JAVA}
    password: ${DB_PASSWORD:JAVA}
    hikari:
      pool-name: StudyHikariPool
      auto-commit: false
      connection-timeout: 30000
      maximum-pool-size: 10
      minimum-idle: 2
      idle-timeout: 600000
      max-lifetime: 1800000
driver-class-name

사용할 JDBC Driver 클래스입니다.
Oracle은 oracle.jdbc.OracleDriver를 사용합니다.

url

DB 접속 주소입니다.
PDB 사용 시 jdbc:oracle:thin:@localhost:1521/XEPDB1
형태를 자주 씁니다.

username / password

DB 계정과 비밀번호입니다. 실무에서는 환경변수로 분리합니다.

maximum-pool-size

Pool에서 가질 수 있는 최대 Connection 수입니다.
무조건 크게 잡으면 DB에 부담이 됩니다.

connection-timeout

Connection을 빌리기 위해 기다릴 최대 시간입니다.

max-lifetime

Connection 하나가
Pool 안에서 살아 있을 수 있는 최대 시간입니다.

    <dependency>

        <artifactId>lombok</artifactId>
        <optional>true</optional>

JdbcTemplate Repository 예제

JdbcTemplate은 내부적으로 DataSource를 사용해 Connection을 얻고, SQL 실행이 끝나면 자원을 반납합니다. 개발자는 SQL, 파라미터, RowMapper에 집중할 수 있습니다.

DTO

import java.time.LocalDateTime;

public record PostCreateRequest(
        String title,
        String content,
        String writer
) {
}

public record PostResponse(
        Long id,
        String title,
        String content,
        String writer,
        LocalDateTime createdAt
) {
}

Repository

import lombok.RequiredArgsConstructor;
import org.springframework.jdbc.core.JdbcTemplate;
import org.springframework.jdbc.core.RowMapper;
import org.springframework.stereotype.Repository;

import java.sql.Timestamp;
import java.util.List;

@Repository
@RequiredArgsConstructor
public class PostJdbcRepository {

    private final JdbcTemplate jdbcTemplate;

    private final RowMapper<PostResponse> postRowMapper = (rs, rowNum) -> {
        Timestamp createdAt = rs.getTimestamp("created_at");

        return new PostResponse(
                rs.getLong("id"),
                rs.getString("title"),
                rs.getString("content"),
                rs.getString("writer"),
                createdAt == null ? null : createdAt.toLocalDateTime()
        );
    };

    public List<PostResponse> findAll() {
        String sql = """
                SELECT id, title, content, writer, created_at
                FROM post
                ORDER BY id DESC
                """;

        return jdbcTemplate.query(sql, postRowMapper);
    }

    public PostResponse findById(Long id) {
        String sql = """
                SELECT id, title, content, writer, created_at
                FROM post
                WHERE id = ?
                """;

        return jdbcTemplate.queryForObject(sql, postRowMapper, id);
    }

    public int save(PostCreateRequest request) {
        String sql = """
                INSERT INTO post (title, content, writer)
                VALUES (?, ?, ?)
                """;

        return jdbcTemplate.update(
                sql,
                request.title(),
                request.content(),
                request.writer()
        );
    }
}
07

Service와 Controller 예제

Service 계층에는 @Transactional을 적용합니다. 조회만 하는 메서드는 @Transactional(readOnly = true)를 사용하고, 등록/수정/삭제는 일반 @Transactional을 사용합니다.

Service

import lombok.RequiredArgsConstructor;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

import java.util.List;

@Service
@RequiredArgsConstructor
public class PostJdbcService {

    private final PostJdbcRepository postJdbcRepository;

    @Transactional(readOnly = true)
    public List<PostResponse> findAll() {
        return postJdbcRepository.findAll();
    }

    @Transactional(readOnly = true)
    public PostResponse findById(Long id) {
        return postJdbcRepository.findById(id);
    }

    @Transactional
    public void create(PostCreateRequest request) {
        postJdbcRepository.save(request);
    }
}

Controller

import lombok.RequiredArgsConstructor;
import org.springframework.http.HttpStatus;
import org.springframework.http.ResponseEntity;
import org.springframework.web.bind.annotation.*;

import java.util.List;

@RestController
@RequestMapping("/api/jdbc/posts")
@RequiredArgsConstructor
public class PostJdbcController {

    private final PostJdbcService postJdbcService;

    @GetMapping
    public ResponseEntity<List<PostResponse>> findAll() {
        return ResponseEntity.ok(postJdbcService.findAll());
    }

    @GetMapping("/{id}")
    public ResponseEntity<PostResponse> findById(@PathVariable Long id) {
        return ResponseEntity.ok(postJdbcService.findById(id));
    }

    @PostMapping
    public ResponseEntity<Void> create(@RequestBody PostCreateRequest request) {
        postJdbcService.create(request);

        return ResponseEntity.status(HttpStatus.CREATED).build();
    }
}

DataSource 직접 등록 예제

일반적인 단일 DB 프로젝트에서는 Spring Boot 자동 설정을 우선 사용하는 것이 좋습니다.
하지만 원리 이해를 위해 @Configuration과 @Bean으로 HikariDataSource를 직접 등록하는 구조도 알아두면 좋습니다.

DatabaseConfig

import com.zaxxer.hikari.HikariConfig;
import com.zaxxer.hikari.HikariDataSource;
import org.springframework.beans.factory.annotation.Value;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

import javax.sql.DataSource;

@Configuration
public class DatabaseConfig {

    @Value("${spring.datasource.driver-class-name}")
    private String driverClassName;

    @Value("${spring.datasource.url}")
    private String jdbcUrl;

    @Value("${spring.datasource.username}")
    private String username;

    @Value("${spring.datasource.password}")
    private String password;

    @Bean
    public DataSource dataSource() {
        HikariConfig config = new HikariConfig();

        config.setPoolName("ManualHikariPool");
        config.setDriverClassName(driverClassName);
        config.setJdbcUrl(jdbcUrl);
        config.setUsername(username);
        config.setPassword(password);

        config.setAutoCommit(false);
        config.setConnectionTimeout(30000);
        config.setMaximumPoolSize(10);
        config.setMinimumIdle(2);
        config.setIdleTimeout(600000);
        config.setMaxLifetime(1800000);

        return new HikariDataSource(config);
    }
}

TransactionManager 직접 등록 예제

import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.jdbc.datasource.DataSourceTransactionManager;
import org.springframework.transaction.support.TransactionTemplate;

import javax.sql.DataSource;

@Configuration
public class TransactionConfig {

    @Bean
    public DataSourceTransactionManager transactionManager(
            DataSource dataSource
    ) {
        return new DataSourceTransactionManager(dataSource);
    }

    @Bean
    public TransactionTemplate transactionTemplate(
            DataSourceTransactionManager transactionManager
    ) {
        return new TransactionTemplate(transactionManager);
    }
}

React에서 Axios로 API 호출하기

React는 DataSource와 HikariCP를 직접 다루지 않습니다. 하지만 React의 API 요청은 결국 Spring Boot 내부에서 DataSource와 HikariCP를 통해 DB에 접근합니다.

토큰 저장 함수

export function saveToken(token, keepLogin) {
  if (keepLogin) {
    localStorage.setItem("accessToken", token);
    sessionStorage.removeItem("accessToken");
  } else {
    sessionStorage.setItem("accessToken", token);
    localStorage.removeItem("accessToken");
  }
}

export function getAccessToken() {
  return (
    localStorage.getItem("accessToken") ||
    sessionStorage.getItem("accessToken")
  );
}

export function clearToken() {
  localStorage.removeItem("accessToken");
  sessionStorage.removeItem("accessToken");
}

Axios 공통 인스턴스

import axios from "axios";
import { getAccessToken, clearToken } from "../utils/tokenStorage";

const apiClient = axios.create({
  baseURL: "http://localhost:8080",
  headers: {
    "Content-Type": "application/json",
  },
  timeout: 5000,
});

apiClient.interceptors.request.use(
  (config) => {
    const token = getAccessToken();

    if (token) {
      config.headers.Authorization = `Bearer ${token}`;
    }

    return config;
  },
  (error) => Promise.reject(error)
);

apiClient.interceptors.response.use(
  (response) => response,
  (error) => {
    if (error.response?.status === 401) {
      clearToken();
      window.location.href = "/login";
    }

    return Promise.reject(error);
  }
);

export default apiClient;

자주 하는 실수

1. Oracle Driver 누락

ojdbc11 의존성이 없으면
Oracle Driver를 찾지 못합니다.

2. JDBC Starter 누락

Spring JDBC와 JdbcTemplate을 사용하려면
spring-boot-starter-jdbc가 필요합니다.

3. JDBC URL 오타

SID 방식과 Service Name 방식을 혼동하면
ORA-12514, ORA-12505가
발생할 수 있습니다.

4. YAML 들여쓰기 오류

spring.datasource 아래 설정이
들여쓰기 오류로 적용되지 않을 수 있습니다.

5. Pool 크기를 무조건 크게 잡음

서버 인스턴스 수와 DB 최대 세션 수를
함께 고려해야 합니다.

6. Connection 반납 누락

순수 JDBC에서는 try-with-resources로
반드시 자원을 반납해야 합니다.

7. @Transactional 누락

여러 DB 작업을 하나의 작업 단위로
묶어야 할 때는
Service에 @Transactional을 사용합니다.

8. 조회 메서드에 readOnly 누락

조회 전용 Service 메서드는
@Transactional(readOnly = true)를
고려합니다.

면접 포인트

Q1. DataSource란 무엇인가요?

DB Connection을 얻기 위한 표준 인터페이스입니다. Spring Boot에서는 HikariDataSource 같은 구현체가 Bean으로 등록될 수 있습니다.

Q2. HikariCP란 무엇인가요?

JDBC Connection Pool 구현체입니다. Connection을 미리 만들어 두고 요청마다 빌려주고 반납받습니다.

Q3. Connection Pool을 쓰는 이유는 무엇인가요?

매 요청마다 DB 연결을 새로 만들면 비용이 크기 때문에, 연결을 재사용해 성능과 안정성을 높이기 위해 사용합니다.

Q4. DriverManager와 DataSource의 차이는 무엇인가요?

DriverManager는 직접 Connection을 생성하는 방식이고, DataSource는 Spring이 관리하는 DB 연결 입구입니다.

Q5. JdbcTemplate은 무엇을 대신해주나요?

Connection 획득, PreparedStatement 생성, ResultSet 처리, 자원 반납 같은 JDBC 반복 코드를 줄여줍니다.

Q6. close()를 호출하면 Connection이 끊기나요?

Connection Pool 환경에서는 보통 물리 연결 종료가 아니라 Pool에 반납하는 의미로 동작합니다.

Q7. @Transactional과 Connection은 어떤 관계인가요?

@Transactional은 트랜잭션 경계를 만들고 정상 종료 시 commit, 예외 발생 시 rollback을 수행한 뒤 Connection을 반납합니다.

Q8. JPA를 쓰면 DataSource를 몰라도 되나요?

아닙니다. JPA도 내부적으로 Hibernate, JDBC, DataSource, HikariCP 위에서 동작합니다.

학습 메모

오늘의 암기 포인트

  • JDBC는 Java에서 DB에 접근하기 위한 표준 API입니다.
  • DriverManager는 매번 직접 Connection을 만드는 기본 방식입니다.
  • DataSource는 Spring에서 DB Connection을 얻는 표준 입구입니다.
  • HikariCP는 Connection을 미리 만들어 두고 빌려주는 Connection Pool입니다.
  • Connection Pool을 쓰면 API 요청마다 새 DB 연결을 만들지 않아도 됩니다.
  • close()는 Pool 환경에서 보통 Connection 반납을 의미합니다.
  • JdbcTemplate은 JDBC 반복 코드를 줄여줍니다.
  • @Transactional은 commit과 rollback을 Spring 방식으로 관리합니다.
