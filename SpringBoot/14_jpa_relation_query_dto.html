Spring Boot - Chapter 14

JPA 연관관계, Query Method, DTO

JPA 연관관계는 DB의 외래키 관계를 Java 객체 참조로 표현하는 기술입니다.
@ManyToOne, @OneToMany, mappedBy로 Entity 간 관계를 설계하고, Query Method, @Query, Pageable로 화면에 필요한 데이터를 조회합니다.
API 응답에서는 Entity 전체를 그대로 내려주는 것이 아니라 React 화면에 필요한 데이터만 DTO로 잘라서 내려주는 구조가 중요합니다.

JPA Relation @ManyToOne @OneToMany mappedBy @JoinColumn 연관관계의 주인 FetchType.LAZY N+1 문제 fetch join Query Method @Query JPQL DTO Projection Pageable Page React 목록 화면
학습 목표 1

@ManyToOne, @OneToMany, mappedBy로
Entity 간 관계를 설계할 수 있습니다.

학습 목표 2

Query Method, @Query, Pageable로
검색·정렬·페이징 목록 API를 만들 수 있습니다.

학습 목표 3

Entity 전체가 아니라 화면에 필요한 데이터만
DTO로 내려주는 구조를 이해합니다.

01

JPA 연관관계와 DTO를 공부한다는 것

앞 단원에서는 Entity 하나를 기준으로 등록, 조회, 수정, 삭제하는 CRUD 흐름을 배웠습니다.
하지만 실제 프로젝트에서는 게시글 하나만 독립적으로 존재하지 않습니다. 게시글에는 작성자가 있고, 게시글에는 댓글이 달리고, 댓글에도 작성자가 있습니다.
DB에서는 이런 관계를 외래키로 표현하지만, JPA에서는 Entity가 다른 Entity를 객체 필드로 참조합니다.
그리고 React 화면에는 Entity 전체를 그대로 내려주는 것이 아니라, 목록 화면, 상세 화면, 댓글 영역에 필요한 데이터만 DTO로 구성해서 내려주는 것이 좋습니다.

Relation

객체 참조로 관계 표현

DB의 FK 관계를
Java Entity의 객체 참조로 표현합니다.

Owner

연관관계의 주인

외래키를 가진 Entity가
관계를 실제로 관리합니다.

Query

필요한 데이터 조회

Query Method와 @Query로
검색, JOIN, DTO 조회를 처리합니다.

DTO

화면용 응답 구조

Entity 전체가 아니라
화면에 필요한 값만 React에 내려줍니다.

핵심 문장

JPA 연관관계는 Entity끼리 객체 참조로 연결하는 기술이고, API 응답은 Entity 전체가 아니라 화면에 필요한 데이터만 DTO로 잘라서 내려주는 구조로 설계해야 합니다.

02

03

연관관계 기본 개념 카드

JPA 연관관계는 DB의 FK 관계를 객체 참조로 표현하는 방식입니다. 가장 많이 쓰는 관계는 자식 Entity에서 부모 Entity를 참조하는 @ManyToOne입니다.

N:1

@ManyToOne

자식 Entity가 부모 Entity를 참조합니다.
대부분 외래키를 가진 연관관계의 주인입니다.

@ManyToOne(fetch = FetchType.LAZY)
@JoinColumn(name = "writer_id")
private User writer;
1:N

@OneToMany

부모 Entity에서 자식 목록을 참조합니다.
보통 mappedBy로 반대편 주인을 가리킵니다.

@OneToMany(mappedBy = "post")
private List<Comment> comments;
Owner

연관관계의 주인

DB 외래키를 가진 쪽입니다.
주인 쪽에서 @JoinColumn으로
FK를 관리합니다.

// Comment가 FK 보유
@ManyToOne
@JoinColumn(name = "post_id")
private Post post;

04

LAZY, EAGER, N+1 문제

연관관계에서 가장 중요한 성능 포인트는 fetch입니다. EAGER는 즉시 로딩, LAZY는 지연 로딩입니다.
실무에서는 대부분 LAZY로 두고, 화면에서 필요한 데이터만 명시적으로 가져오는 방식이 좋습니다.

FetchType.LAZY

지연 로딩

연관 Entity를 실제로 사용할 때 조회합니다.
필요한 시점에 가져오므로
기본 전략으로 권장됩니다.

@ManyToOne(fetch = FetchType.LAZY)
private User writer;
FetchType.EAGER

즉시 로딩

Entity를 조회할 때
연관 Entity도 함께 조회합니다.
불필요한 조회가 생길 수 있습니다.

@ManyToOne(fetch = FetchType.EAGER)
private User writer;
N+1

추가 쿼리 문제

목록 1번 조회 후 반복문에서
연관 데이터 접근 시
N번의 추가 쿼리가 생길 수 있습니다.

for (Post post : posts) {
    post.getWriter().getDisplayName();
}
Solution

해결 방향

fetch join, @EntityGraph,
DTO 직접 조회, Batch Size 등을
상황에 맞게 사용합니다.

join fetch p.writer
목록 화면 주의

목록 화면에서 Entity 전체를 조회한 뒤 반복문으로 작성자, 댓글 수를 접근하면 N+1 문제가 생기기 쉽습니다.
목록 화면은 PostListResponse 같은 DTO로 필요한 값만 조회하는 구조가 좋습니다.

05

Query Method, @Query, Pageable

간단한 조회는 Repository 메서드 이름 규칙으로 만들 수 있습니다. 조건이 복잡하거나 JOIN, 집계, DTO 조회가 필요하면 @Query를 사용합니다.
목록 API에서는 Pageable과 Page를 함께 사용합니다.

Query Method

메서드 이름으로 쿼리 생성

간단한 검색 조건을 빠르게 만들 수 있습니다.

Page<Post> findByWriterIdOrderByIdDesc(
    Long writerId,
    Pageable pageable
);
@Query

복잡한 쿼리 직접 작성

JOIN, fetch join, 집계,
DTO 조회가 필요할 때 사용합니다.

@Query("""
select p
from Post p
join fetch p.writer
where p.id = :id
""")
DTO Projection

필요한 값만 조회

Entity 전체가 아니라
화면에 필요한 필드만 DTO로 바로 조회합니다.

select new ...PostListResponse(
    p.id, p.title, w.displayName
)
Pageable

검색·정렬·페이징

page, size, sort 정보를 담고
Repository 조회에 전달합니다.

GET /api/posts?page=0&size=10

Entity 코드 예제

아래 예제는 회원, 게시글, 댓글 관계를 기준으로 합니다. 게시글은 작성자 User를 @ManyToOne으로 참조하고, 댓글은 게시글 Post와 작성자 User를 각각 @ManyToOne으로 참조합니다.

User Entity

package com.example.studyapi.user.entity;

import jakarta.persistence.*;
import java.time.LocalDateTime;

@Entity
@Table(name = "users")
public class User {

    @Id
    @GeneratedValue(strategy = GenerationType.SEQUENCE, generator = "seq_users")
    @SequenceGenerator(
            name = "seq_users",
            sequenceName = "seq_users",
            allocationSize = 1
    )
    private Long id;

    @Column(nullable = false, unique = true, length = 50)
    private String username;

    @Column(nullable = false, length = 100)
    private String displayName;

    @Column(nullable = false)
    private LocalDateTime createdAt;

    protected User() {
    }

    public User(String username, String displayName) {
        this.username = username;
        this.displayName = displayName;
        this.createdAt = LocalDateTime.now();
    }

    public Long getId() {
        return id;
    }

    public String getUsername() {
        return username;
    }

    public String getDisplayName() {
        return displayName;
    }

    public LocalDateTime getCreatedAt() {
        return createdAt;
    }
}

Post Entity

package com.example.studyapi.post.entity;

import com.example.studyapi.user.entity.User;
import jakarta.persistence.*;
import org.hibernate.annotations.CreationTimestamp;

import java.time.LocalDateTime;
import java.util.ArrayList;
import java.util.List;

@Entity
@Table(name = "posts")
public class Post {

    @Id
    @GeneratedValue(strategy = GenerationType.SEQUENCE, generator = "seq_posts")
    @SequenceGenerator(
            name = "seq_posts",
            sequenceName = "seq_posts",
            allocationSize = 1
    )
    private Long id;

    @Column(nullable = false, length = 100)
    private String title;

    @Lob
    @Column(nullable = false)
    private String content;

    @ManyToOne(fetch = FetchType.LAZY, optional = false)
    @JoinColumn(name = "writer_id", nullable = false)
    private User writer;

    @OneToMany(mappedBy = "post")
    private List<Comment> comments = new ArrayList<>();

    @CreationTimestamp
    @Column(nullable = false, updatable = false)
    private LocalDateTime createdAt;

    protected Post() {
    }

    public Post(String title, String content, User writer) {
        this.title = title;
        this.content = content;
        this.writer = writer;
    }

    public void update(String title, String content) {
        this.title = title;
        this.content = content;
    }

    public Long getId() {
        return id;
    }

    public String getTitle() {
        return title;
    }

    public String getContent() {
        return content;
    }

    public User getWriter() {
        return writer;
    }

    public List<Comment> getComments() {
        return comments;
    }

    public LocalDateTime getCreatedAt() {
        return createdAt;
    }
}

Comment Entity

package com.example.studyapi.post.entity;

import com.example.studyapi.user.entity.User;
import jakarta.persistence.*;
import org.hibernate.annotations.CreationTimestamp;

import java.time.LocalDateTime;

@Entity
@Table(name = "comments")
public class Comment {

    @Id
    @GeneratedValue(strategy = GenerationType.SEQUENCE, generator = "seq_comments")
    @SequenceGenerator(
            name = "seq_comments",
            sequenceName = "seq_comments",
            allocationSize = 1
    )
    private Long id;

    @Column(nullable = false, length = 1000)
    private String content;

    @ManyToOne(fetch = FetchType.LAZY, optional = false)
    @JoinColumn(name = "post_id", nullable = false)
    private Post post;

    @ManyToOne(fetch = FetchType.LAZY, optional = false)
    @JoinColumn(name = "writer_id", nullable = false)
    private User writer;

    @CreationTimestamp
    @Column(nullable = false, updatable = false)
    private LocalDateTime createdAt;

    protected Comment() {
    }

    public Comment(String content, Post post, User writer) {
        this.content = content;
        this.post = post;
        this.writer = writer;
    }

    public Long getId() {
        return id;
    }

    public String getContent() {
        return content;
    }

    public Post getPost() {
        return post;
    }

    public User getWriter() {
        return writer;
    }

    public LocalDateTime getCreatedAt() {
        return createdAt;
    }
}

Repository

package com.example.studyapi.post.repository;

import com.example.studyapi.post.entity.Post;
import com.example.studyapi.post.dto.PostListResponse;
import org.springframework.data.domain.Page;
import org.springframework.data.domain.Pageable;
import org.springframework.data.jpa.repository.*;
import org.springframework.data.repository.query.Param;

import java.util.Optional;

public interface PostRepository extends JpaRepository<Post, Long> {

    Page<Post> findByTitleContainingOrderByIdDesc(
            String keyword,
            Pageable pageable
    );

    Page<Post> findByWriterIdOrderByIdDesc(
            Long writerId,
            Pageable pageable
    );

    @Query("""
           select p
           from Post p
           join fetch p.writer
           where p.id = :id
           """)
    Optional<Post> findByIdWithWriter(@Param("id") Long id);

    @Query("""
           select new com.example.studyapi.post.dto.PostListResponse(
               p.id,
               p.title,
               w.displayName,
               count(c.id),
               p.createdAt
           )
           from Post p
           join p.writer w
           left join p.comments c
           where (:keyword is null or :keyword = '' or p.title like concat('%', :keyword, '%'))
           group by p.id, p.title, w.displayName, p.createdAt
           order by p.id desc
           """)
    Page<PostListResponse> searchPostList(
            @Param("keyword") String keyword,
            Pageable pageable
    );
}

Service와 Controller 코드 예제

PostQueryService

package com.example.studyapi.post.service;

import com.example.studyapi.common.PageResponse;
import com.example.studyapi.post.dto.*;
import com.example.studyapi.post.entity.Comment;
import com.example.studyapi.post.entity.Post;
import com.example.studyapi.post.exception.PostNotFoundException;
import com.example.studyapi.post.repository.CommentRepository;
import com.example.studyapi.post.repository.PostRepository;
import com.example.studyapi.user.entity.User;
import com.example.studyapi.user.exception.UserNotFoundException;
import com.example.studyapi.user.repository.UserRepository;
import lombok.RequiredArgsConstructor;
import org.springframework.data.domain.Page;
import org.springframework.data.domain.Pageable;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

import java.util.List;

@Service
@RequiredArgsConstructor
public class PostQueryService {

    private final PostRepository postRepository;
    private final CommentRepository commentRepository;
    private final UserRepository userRepository;

    @Transactional
    public Long create(PostCreateRequest request) {
        User writer = userRepository.findById(request.writerId())
                .orElseThrow(() -> new UserNotFoundException(request.writerId()));

        Post post = new Post(
                request.title(),
                request.content(),
                writer
        );

        Post savedPost = postRepository.save(post);

        return savedPost.getId();
    }

    @Transactional(readOnly = true)
    public PageResponse<PostListResponse> searchPosts(
            String keyword,
            Pageable pageable
    ) {
        Page<PostListResponse> page = postRepository.searchPostList(
                keyword,
                pageable
        );

        return PageResponse.from(page);
    }

    @Transactional(readOnly = true)
    public PostDetailResponse findDetail(Long postId) {
        Post post = postRepository.findByIdWithWriter(postId)
                .orElseThrow(() -> new PostNotFoundException(postId));

        List<Comment> comments =
                commentRepository.findByPostIdOrderByIdAsc(postId);

        List<CommentResponse> commentResponses = comments.stream()
                .map(CommentResponse::from)
                .toList();

        return PostDetailResponse.of(post, commentResponses);
    }
}

PostQueryController

package com.example.studyapi.post.controller;

import com.example.studyapi.common.ApiResponse;
import com.example.studyapi.common.PageResponse;
import com.example.studyapi.post.dto.*;
import com.example.studyapi.post.service.PostQueryService;
import jakarta.validation.Valid;
import lombok.RequiredArgsConstructor;
import org.springframework.data.domain.Pageable;
import org.springframework.data.domain.Sort;
import org.springframework.data.web.PageableDefault;
import org.springframework.http.*;
import org.springframework.web.bind.annotation.*;

@RestController
@RequestMapping("/api/posts")
@RequiredArgsConstructor
public class PostQueryController {

    private final PostQueryService postQueryService;

    @PostMapping
    public ResponseEntity<ApiResponse<Long>> create(
            @Valid @RequestBody PostCreateRequest request
    ) {
        Long postId = postQueryService.create(request);

        return ResponseEntity
                .status(HttpStatus.CREATED)
                .body(ApiResponse.success("게시글이 등록되었습니다.", postId));
    }

    @GetMapping
    public ResponseEntity<ApiResponse<PageResponse<PostListResponse>>> searchPosts(
            @RequestParam(defaultValue = "") String keyword,
            @PageableDefault(
                    size = 10,
                    sort = "id",
                    direction = Sort.Direction.DESC
            )
            Pageable pageable
    ) {
        PageResponse<PostListResponse> response =
                postQueryService.searchPosts(keyword, pageable);

        return ResponseEntity.ok(ApiResponse.success(response));
    }

    @GetMapping("/{postId}")
    public ResponseEntity<ApiResponse<PostDetailResponse>> findDetail(
            @PathVariable Long postId
    ) {
        PostDetailResponse response = postQueryService.findDetail(postId);

        return ResponseEntity.ok(ApiResponse.success(response));
    }
}

export function saveToken(token, keepLogin) {
  if (keepLogin) {
    localStorage.setItem("accessToken", token);
    sessionStorage.removeItem("accessToken");
  } else {
    sessionStorage.setItem("accessToken", token);
    localStorage.removeItem("accessToken");
  }
}

export function getAccessToken() {
  return (
    localStorage.getItem("accessToken") ||
    sessionStorage.getItem("accessToken")
  );
}

export function clearToken() {
  localStorage.removeItem("accessToken");
  sessionStorage.removeItem("accessToken");
}

자주 하는 실수

1. Entity 직접 반환

Lazy 오류, 순환 참조, 민감 정보 노출
위험이 있으므로 DTO로 응답합니다.

2. 모든 관계를 양방향으로 설계

처음에는
@ManyToOne 단방향 중심으로 설계하고
필요할 때만 양방향을 추가합니다.

3. mappedBy에 컬럼명 작성

mappedBy에는 post_id가 아니라
자식 Entity의 필드명 post를 적어야 합니다.

4. @ManyToOne에
mappedBy 사용

@ManyToOne은 보통 주인 쪽이므로
mappedBy가 아니라
@JoinColumn을 사용합니다.

5. EAGER 남발

원하지 않는 JOIN과
추가 조회를 유발할 수 있으므로
LAZY를 기본으로 둡니다.

6. cascade = ALL 남발

User와 Post처럼
생명주기가 다른 Entity에는
cascade REMOVE를 함부로 쓰면 위험합니다.

7. 요청 DTO에 Entity 전체 받기

writer 객체 전체가 아니라
writerId만 받고
Service에서 User를 조회해 연결합니다.

8. page를 1부터 시작한다고 착각

Spring Data의 page는 0부터 시작합니다.
화면에서는 page + 1로 표시합니다.

면접 포인트

Q1. JPA 연관관계란 무엇인가요?

DB의 외래키 관계를 Java Entity의 객체 참조로 표현하는 것입니다.

Q2. @ManyToOne과 @OneToMany의 차이는 무엇인가요?

@ManyToOne은 자식 Entity에서 부모를 참조하고, @OneToMany는 부모 Entity에서 자식 목록을 참조합니다.

Q3. 연관관계의 주인이란 무엇인가요?

외래키를 실제로 관리하는 Entity입니다. 보통 FK를 가진 자식 Entity가 주인입니다.

Q4. mappedBy는 무엇인가요?

양방향 관계에서 주인이 아닌 쪽에 작성하며, 자식 Entity의 부모 참조 필드명을 값으로 넣습니다.

Q5. LAZY와 EAGER의 차이는 무엇인가요?

EAGER는 즉시 로딩, LAZY는 실제 사용할 때 로딩입니다. 실무에서는 대부분 LAZY를 기본으로 권장합니다.

Q6. N+1 문제란 무엇인가요?

목록 조회 1번 이후 각 데이터의 연관 Entity를 조회하기 위해 N번의 추가 쿼리가 발생하는 문제입니다.

Q7. DTO와 Entity를 분리하는 이유는 무엇인가요?

DB 구조 노출, Lazy 오류, 순환 참조, 민감 정보 노출을 막고 화면에 필요한 데이터만 내려주기 위해서입니다.

Q8. Pageable과 Page의 차이는 무엇인가요?

Pageable은 요청의 page, size, sort 정보이고, Page는 조회 결과와 전체 개수, 전체 페이지 정보를 담는 객체입니다.

학습 메모

오늘의 암기 포인트

  • JPA 연관관계는 DB의 FK 관계를 Java 객체 참조로 표현하는 것입니다.
  • @ManyToOne은 보통 자식 Entity 쪽에 붙고, 연관관계의 주인입니다.
  • @OneToMany(mappedBy = "...")는 부모 Entity에서 자식 목록을 읽기 위한 반대편입니다.
  • mappedBy에는 DB 컬럼명이 아니라 자식 Entity의 부모 참조 필드명을 적습니다.
  • 실무에서는 연관관계를 기본적으로 LAZY로 두고 필요한 화면에서 명시적으로 조회합니다.
  • N+1 문제는 목록 1번 조회 후 연관 데이터 N번 추가 조회가 발생하는 문제입니다.
  • Query Method는 간단한 조회, @Query는 복잡한 JOIN/DTO 조회에 사용합니다.
  • Entity를 직접 반환하지 말고 화면에 필요한 데이터만 DTO로 내려줍니다.
  • Pageable은 요청의 page, size, sort 정보이고, Page는 조회 결과와 페이지 메타데이터입니다.

헷갈리기 쉬운 개념

@JoinColumn vs mappedBy

@JoinColumn은
주인 쪽 FK 컬럼명,
mappedBy는
반대편의 Java 필드명입니다.

@ManyToOne vs
@OneToMany

ManyToOne은 자식에서 부모 참조,
OneToMany는 부모에서
자식 목록 참조입니다.

LAZY vs EAGER

LAZY는 필요할 때 조회,
EAGER는 즉시 조회입니다.
실무 기본은 LAZY입니다.

Entity vs DTO

Entity는 DB 매핑 객체,
DTO는 API 요청/응답 객체입니다.

Pageable vs Page

Pageable은 요청 조건,
Page는 조회 결과와
전체 페이지 정보입니다.
