01

트랜잭션이란 무엇인가?

트랜잭션은 여러 SQL을 하나의 업무 단위로 묶는 개념입니다.
SQL 하나하나가 아니라 “회원가입”, “주문”, “게시글 등록”처럼 하나의 업무가 성공했는지를 기준으로 commit과 rollback을 결정합니다.
예를 들어 회원가입은 users 테이블에 회원 정보를 저장하고, user_roles 테이블에 기본 권한을 저장하고, signup_log에 가입 로그를 남길 수 있습니다.
이 중 하나라도 실패하면 전체 작업을 취소해야 데이터가 깨지지 않습니다.

SQL 1 users INSERT

회원 기본 정보 저장

+
SQL 2 user_roles INSERT

회원 기본 권한 저장

+
SQL 3 signup_log INSERT

가입 이력 저장

핵심 문장

트랜잭션은 여러 SQL을 하나의 업무처럼 묶고, 모두 성공하면 commit, 하나라도 실패하면 rollback하는 데이터 정합성 보호 장치입니다.

초보자가 가장 많이 헷갈리는 부분은 close()를 commit()이나 rollback()과 비슷한 의미로 생각하는 것입니다. 세 메서드는 역할이 완전히 다릅니다.

메서드
역할
사용 시점

06

순수 JDBC 트랜잭션 예제

아래 예제는 회원가입 시 users 테이블과 user_roles 테이블에 함께 INSERT하는 상황입니다. 둘 중 하나라도 실패하면 전체 rollback합니다.

회원 가입 + 권한 등록 트랜잭션

import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.PreparedStatement;
import java.sql.SQLException;

public class JdbcTransactionExample {

    public static void main(String[] args) {
        String url = "jdbc:oracle:thin:@localhost:1521/XEPDB1";
        String dbUser = "JAVA";
        String dbPassword = "JAVA";

        String insertUserSql = """
            INSERT INTO users (id, username, email)
            VALUES (users_seq.NEXTVAL, ?, ?)
            """;

        String insertRoleSql = """
            INSERT INTO user_roles (id, username, role_name)
            VALUES (user_roles_seq.NEXTVAL, ?, ?)
            """;

        Connection con = null;

        try {
            Class.forName("oracle.jdbc.OracleDriver");

            con = DriverManager.getConnection(url, dbUser, dbPassword);
            con.setAutoCommit(false);

            try (PreparedStatement ps = con.prepareStatement(insertUserSql)) {
                ps.setString(1, "user1");
                ps.setString(2, "user1@example.com");

                int count = ps.executeUpdate();

                if (count != 1) {
                    throw new SQLException("회원 정보 저장 실패");
                }
            }

            try (PreparedStatement ps = con.prepareStatement(insertRoleSql)) {
                ps.setString(1, "user1");
                ps.setString(2, "ROLE_USER");

                int count = ps.executeUpdate();

                if (count != 1) {
                    throw new SQLException("회원 권한 저장 실패");
                }
            }

            con.commit();
            System.out.println("회원 가입 트랜잭션 성공");

        } catch (Exception e) {
            if (con != null) {
                try {
                    con.rollback();
                    System.out.println("오류 발생, rollback 완료");
                } catch (SQLException rollbackException) {
                    rollbackException.printStackTrace();
                }
            }

            e.printStackTrace();

        } finally {
            if (con != null) {
                try {
                    con.setAutoCommit(true);
                    con.close();
                } catch (SQLException closeException) {
                    closeException.printStackTrace();
                }
            }
        }
    }
}
setAutoCommit(false)

SQL마다 자동 commit되는 것을 막습니다.

executeUpdate()

INSERT, UPDATE, DELETE 실행 후
영향받은 행 수를 반환합니다.

commit()

모든 SQL이 성공했을 때 DB에 확정합니다.

rollback()

하나라도 실패했을 때
현재 트랜잭션을 취소합니다.

2) DataSource 기반 JDBC Service

import lombok.RequiredArgsConstructor;
import org.springframework.stereotype.Service;

import javax.sql.DataSource;
import java.sql.Connection;
import java.sql.PreparedStatement;
import java.sql.SQLException;

@Service
@RequiredArgsConstructor
public class SignupJdbcService {

    private final DataSource dataSource;

    public void signup(SignupRequest request) {
        String insertUserSql = """
            INSERT INTO users (id, username, email, password)
            VALUES (users_seq.NEXTVAL, ?, ?, ?)
            """;

        String insertRoleSql = """
            INSERT INTO user_roles (id, username, role_name)
            VALUES (user_roles_seq.NEXTVAL, ?, ?)
            """;

        Connection con = null;

        try {
            con = dataSource.getConnection();
            con.setAutoCommit(false);

            try (PreparedStatement ps = con.prepareStatement(insertUserSql)) {
                ps.setString(1, request.username());
                ps.setString(2, request.email());
                ps.setString(3, request.password());

                int count = ps.executeUpdate();

                if (count != 1) {
                    throw new SQLException("회원 정보 저장 실패");
                }
            }

            try (PreparedStatement ps = con.prepareStatement(insertRoleSql)) {
                ps.setString(1, request.username());
                ps.setString(2, "ROLE_USER");

                int count = ps.executeUpdate();

                if (count != 1) {
                    throw new SQLException("회원 권한 저장 실패");
                }
            }

            con.commit();

        } catch (Exception e) {
            if (con != null) {
                try {
                    con.rollback();
                } catch (SQLException rollbackException) {
                    rollbackException.printStackTrace();
                }
            }

            throw new RuntimeException("회원가입 처리 중 오류가 발생했습니다.", e);

        } finally {
            if (con != null) {
                try {
                    con.setAutoCommit(true);
                    con.close();
                } catch (SQLException closeException) {
                    closeException.printStackTrace();
                }
            }
        }
    }
}

3) @Transactional 방식

import lombok.RequiredArgsConstructor;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

@Service
@RequiredArgsConstructor
public class SignupJpaService {

    private final UserRepository userRepository;
    private final UserRoleRepository userRoleRepository;

    @Transactional
    public void signup(SignupRequest request) {
        User user = new User(
            request.username(),
            request.email(),
            request.password()
        );

        UserRole role = new UserRole(
            request.username(),
            "ROLE_USER"
        );

        userRepository.save(user);
        userRoleRepository.save(role);
    }
}
정리

순수 JDBC에서는 commit()과 rollback()을 직접 호출합니다. Spring Boot에서는 @Transactional이 트랜잭션 시작, commit, rollback을 대신 관리합니다.

1) 토큰 저장 함수

export function saveToken(token, keepLogin) {
  if (keepLogin) {
    localStorage.setItem("accessToken", token);
    sessionStorage.removeItem("accessToken");
  } else {
    sessionStorage.setItem("accessToken", token);
    localStorage.removeItem("accessToken");
  }
}

export function getAccessToken() {
  return (
    localStorage.getItem("accessToken") ||
    sessionStorage.getItem("accessToken")
  );
}

export function clearToken() {
  localStorage.removeItem("accessToken");
  sessionStorage.removeItem("accessToken");
}

2) Axios 공통 인스턴스와 토큰 자동 첨부

import axios from "axios";
import { getAccessToken, clearToken } from "../utils/tokenStorage";

const apiClient = axios.create({
  baseURL: "http://localhost:8080",
  headers: {
    "Content-Type": "application/json",
  },
  timeout: 5000,
});

apiClient.interceptors.request.use(
  (config) => {
    const token = getAccessToken();

    if (token) {
      config.headers.Authorization = `Bearer ${token}`;
    }

    return config;
  },
  (error) => Promise.reject(error)
);

apiClient.interceptors.response.use(
  (response) => response,
  (error) => {
    if (error.response?.status === 401) {
      clearToken();
      window.location.href = "/login";
    }

    return Promise.reject(error);
  }
);

export default apiClient;

4) 폼 입력값을 DTO로 보내기

import { useState } from "react";
import { signup } from "../api/authApi";

function SignupForm() {
  const [form, setForm] = useState({
    username: "",
    email: "",
    password: "",
  });

  const [message, setMessage] = useState("");

  function handleChange(event) {
    const { name, value } = event.target;

    setForm((prev) => ({
      ...prev,
      [name]: value,
    }));
  }

  async function handleSubmit(event) {
    event.preventDefault();

    try {
      const result = await signup(form);
      setMessage(result.message);
    } catch (error) {
      if (error.response?.status === 409) {
        setMessage("이미 사용 중인 아이디 또는 이메일입니다.");
      } else {
        setMessage("회원가입에 실패했습니다. 다시 확인해주세요.");
      }
    }
  }

  return (
    <form onSubmit={handleSubmit}>
      <input
        name="username"
        value={form.username}
        onChange={handleChange}
        placeholder="아이디"
      />

      <input
        name="email"
        value={form.email}
        onChange={handleChange}
        placeholder="이메일"
      />

      <input
        type="password"
        name="password"
        value={form.password}
        onChange={handleChange}
        placeholder="비밀번호"
      />

      <button type="submit">회원가입</button>

      {message && <p>{message}</p>}
    </form>
  );
}

export default SignupForm;

LIKE 파라미터 처리

검색 기능에서는 LIKE 조건을 자주 사용합니다. 사용자 입력값을 SQL 문자열에 직접 붙이지 말고, PreparedStatement의 파라미터로 안전하게 바인딩해야 합니다.

검색 목적
Oracle SQL
PreparedStatement용 SQL
값 포함
where loc like '%값%'
where loc like '%' || ? || '%'
값으로 시작
where loc like '값%'
where loc like ? || '%'
값으로 끝
where loc like '%값'
where loc like '%' || ?
한 글자 뒤 값 포함
where loc like '_값%'
where loc like '_' || ? || '%'

검색 Repository 예제

public List<UserResponse> searchByUsername(String keyword) {
    String sql = """
        SELECT id, username, email
        FROM users
        WHERE username LIKE '%' || ? || '%'
        ORDER BY id DESC
        """;

    List<UserResponse> users = new ArrayList<>();

    try (
        Connection con = dataSource.getConnection();
        PreparedStatement ps = con.prepareStatement(sql)
    ) {
        ps.setString(1, keyword);

        try (ResultSet rs = ps.executeQuery()) {
            while (rs.next()) {
                users.add(new UserResponse(
                    rs.getLong("id"),
                    rs.getString("username"),
                    rs.getString("email")
                ));
            }
        }

        return users;

    } catch (SQLException e) {
        throw new RuntimeException("사용자 검색 중 오류가 발생했습니다.", e);
    }
}
주의

'%?%'처럼 작성하면 ?가 파라미터가 아니라 문자열 문자로 처리됩니다. '%' || ? || '%' 또는 "%" + keyword + "%" 방식으로 처리해야 합니다.

자주 하는 실수

1. autoCommit을 끄지 않음

여러 SQL을 하나의 업무로 묶어야 하는데
자동 커밋이 켜져 있으면
일부 데이터만 반영될 수 있습니다.

2. catch에서 rollback 누락

예외가 발생했는데 rollback을 하지 않으면
실패한 작업의 일부가 남을 수 있습니다.

3. commit을 너무 일찍 호출

모든 SQL이 끝나기 전에 commit하면
이후 실패 시 앞의 작업을 되돌리기 어렵습니다.

4. close를 commit으로 착각

close는 자원 반납입니다.
트랜잭션 결과는 commit 또는 rollback으로
명확히 결정해야 합니다.

5. Connection close 누락

close를 하지 않으면 커넥션 풀이 고갈되어
API timeout이 발생할 수 있습니다.

6. autoCommit 상태 복구 누락

커넥션 풀 환경에서는 사용 후
setAutoCommit(true)로
복구하는 습관이 안전합니다.

7. LIKE 검색에 '%?%' 사용

'%?%'는 파라미터가 아닙니다.
'%' || ? || '%'를 사용합니다.

8. @Transactional에서 예외를 삼킴

예외를 catch만 하고 다시 던지지 않으면
Spring이 정상 종료로 판단할 수 있습니다.

실무 연결 카드

트랜잭션과 자원 반납은 실무 백엔드 안정성과 직접 연결됩니다. 특히 등록·수정·삭제 API, 주문/결제, 파일 업로드, 관리자 일괄 처리에서 중요합니다.

회원가입

회원 정보, 권한, 프로필, 가입 로그를
하나의 트랜잭션으로 묶어야 합니다.

주문/결제

주문, 주문상품, 재고 차감, 결제 로그 중
하나라도 실패하면 전체 rollback해야 합니다.

게시글 + 첨부파일

게시글 저장과 첨부파일 메타데이터 저장을
같은 업무 단위로 관리해야 합니다.

관리자 일괄 처리

회원 상태, 상품 가격,
게시글 공개 상태 변경은 중간 실패 시
일부만 반영되지 않도록 해야 합니다.

커넥션 풀 안정성

close 누락은 HikariCP timeout과
API 장애로 이어질 수 있습니다.

검색 기능

LIKE 검색은 PreparedStatement 파라미터로
안전하게 처리해야 합니다.

13

면접 포인트

Q1. 트랜잭션이 무엇인가요?

트랜잭션은 여러 DB 작업을 하나의 업무 단위로 묶는 개념입니다. 모든 작업이 성공하면 commit하고, 하나라도 실패하면 rollback해서 데이터 정합성을 지킵니다.

Q2. JDBC에서 트랜잭션을 어떻게 처리하나요?

Connection 객체로 처리합니다. setAutoCommit(false)로 자동 커밋을 끄고, 여러 SQL을 실행한 뒤 성공하면 commit(), 실패하면 rollback()을 호출합니다.

Q3. commit과 rollback의 차이는 무엇인가요?

commit은 트랜잭션의 변경 내용을 DB에 최종 확정하는 것이고, rollback은 현재 트랜잭션의 변경 내용을 취소하는 것입니다.

Q4. close는 commit이나 rollback과 같은 의미인가요?

아닙니다. close는 자원 반납입니다. commit은 변경 확정, rollback은 변경 취소, close는 Connection이나 Statement 같은 자원을 닫는 역할입니다.

Q5. try-with-resources를 사용하는 이유는 무엇인가요?

Connection, PreparedStatement, ResultSet처럼 반드시 닫아야 하는 자원을 자동으로 close해줍니다. 예외가 발생해도 자원 반납이 보장되어 자원 누수를 줄일 수 있습니다.

Q6. @Transactional은 어떤 역할을 하나요?

Service 메서드에서 DB 작업을 하나의 트랜잭션으로 묶어줍니다. 메서드가 정상 종료되면 commit하고, RuntimeException이 발생하면 rollback합니다.

Q7. LIKE 검색에서 PreparedStatement를 어떻게 사용하나요?

Oracle에서는 where username like '%' || ? || '%'처럼 작성하거나, Java에서 "%" + keyword + "%"로 값을 만들어 바인딩할 수 있습니다.

Q8. React에서 rollback을 직접 제어하나요?

아닙니다. React는 API 요청과 화면 처리를 담당합니다. commit과 rollback은 Spring Boot 서버의 Service 계층에서 처리합니다.

학습 메모

오늘의 암기 포인트

  • 트랜잭션은 여러 SQL을 하나의 업무 단위로 묶는 개념입니다.
  • setAutoCommit(false)는 SQL마다 자동 commit되는 것을 막습니다.
  • commit()은 DB 변경을 확정합니다.
  • rollback()은 현재 트랜잭션의 변경을 취소합니다.
  • close()는 commit이나 rollback이 아니라 자원 반납입니다.
  • try-with-resources는 JDBC 자원을 안전하게 닫아줍니다.
  • 등록·수정·삭제 API는 실패 시 rollback해야 데이터 정합성을 지킬 수 있습니다.

commit vs close

commit은 DB 변경 확정,
close는 자원 반납입니다.

rollback vs 예외 처리

예외를 잡는 것만으로는 부족하고,
DB 변경을 취소하려면
rollback이 필요합니다.

autoCommit true vs false

true는 SQL마다 자동 확정,
false는 개발자가
commit/rollback을 결정합니다.

JDBC Transaction vs
@Transactional

JDBC는 직접 제어,
@Transactional은
Spring이 트랜잭션을 관리합니다.

이 방식은 파라미터가 아닙니다.
