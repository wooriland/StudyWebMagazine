Spring Boot - Chapter 17

RestTemplate

RestTemplate은 Spring Boot 서버 내부에서 다른 서버의 REST API를 호출할 때 사용하는 동기 방식 HTTP 클라이언트입니다.
React에서 Axios가 브라우저의 API 호출 도구라면, Spring Boot에서는 RestTemplate이 서버 내부 HTTP 호출 도구입니다.
이번 단원에서는 HttpHeaders, HttpEntity, exchange(), ResponseEntity, timeout, connection pool을 중심으로 서버 간 HTTP 통신 흐름을 정리합니다.

01

RestTemplate을 공부한다는 것

RestTemplate은 Spring Boot 서버가 다른 서버에게 HTTP 요청을 보내는 도구입니다.
프론트엔드에서 React가 Axios로 Spring Boot API를 호출하듯이,
Spring Boot 서버 내부에서는 RestTemplate으로 외부 API, Python AI 서버, 결제 서버, 공공데이터 서버, 다른 Spring 서버를 호출할 수 있습니다.

Client

서버 내부 HTTP 클라이언트

브라우저가 아니라
Spring Boot 서버 안에서 실행됩니다.
서버가 서버에게 API 요청을 보낼 때
사용합니다.

Template

HTTP 요청을 쉽게 만드는 틀

URL, HTTP Method, Header, Body,
응답 타입을 정해 반복적인 HTTP 요청 코드를
일정한 패턴으로 작성하게 해줍니다.

Sync

동기 방식 호출

요청을 보내면 응답이 올 때까지
현재 스레드가 기다립니다.
그래서 timeout 설정이 중요합니다.

DTO

JSON을 Java 객체로 변환

외부 API의 JSON 응답을
Map으로 받을 수도 있지만,
실무에서는 DTO로 받아
타입 안정성을 높이는 것이 좋습니다.

핵심 문장

RestTemplate은 Spring Boot 서버 내부에서 외부 REST API를 호출하는 동기 HTTP 클라이언트입니다. 핵심 흐름은 HttpHeaders → HttpEntity → exchange() → ResponseEntity입니다.

03

RestTemplate 요청 흐름도

RestTemplate 요청은 단순히 restTemplate.exchange() 한 줄로 끝나는 것처럼 보이지만, 실제로는 URL, Header, Body, 응답 타입을 차례대로 조립하는 과정입니다.

1 URL 준비

호출할 외부 API 주소를 정합니다.
query parameter가 있으면
UriComponentsBuilder로 안전하게 조립합니다.

→
2 HttpHeaders

Content-Type, Accept, Authorization, X-API-KEY 같은
요청 부가 정보를 담습니다.

→
3 HttpEntity

요청 header와 body를 하나로 묶습니다.
GET은 body 없이 header만 담을 수 있습니다.

4 exchange()

URL, HTTP Method, HttpEntity, 응답 타입을 넣어
실제 HTTP 요청을 보냅니다.

→
5 외부 API

외부 서버, 결제 서버, Python AI 서버,
공공데이터 API 등이 요청을 처리합니다.

→
6 ResponseEntity

statusCode, headers, body를 확인합니다.
body는 DTO로 변환해 사용합니다.

한 줄 요약

HttpHeaders는 요청 송장, HttpEntity는 송장과 물건이 들어 있는 택배 상자, exchange()는 택배 발송, ResponseEntity는 배송 결과와 받은 물건입니다.

04

핵심 클래스 카드

RestTemplate 코드를 읽을 때 가장 먼저 구분해야 하는 클래스는 HttpHeaders, HttpEntity, ResponseEntity입니다.

HttpHeaders

요청 헤더

Content-Type, Accept, Authorization 같은
요청 부가 정보를 담습니다. 

HttpHeaders headers = new HttpHeaders();
headers.setContentType(MediaType.APPLICATION_JSON);
headers.setBearerAuth(token);
HttpEntity

요청 헤더 + 요청 바디

헤더와 body를 묶어서 exchange()에 넘기는
요청 상자 역할을 합니다. 

HttpEntity<RequestDto> entity =
    new HttpEntity<>(requestDto, headers);
exchange()

HTTP 요청 실행

GET, POST, PUT, DELETE를
모두 처리할 수 있는 가장 명확한
RestTemplate 호출 방식입니다. 

restTemplate.exchange(
    url,
    HttpMethod.POST,
    entity,
    ResponseDto.class
);
ResponseEntity

응답 전체

상태코드, 응답 헤더, 응답 body를
함께 담습니다.
오류 처리와 디버깅에 중요합니다. 

response.getStatusCode();
response.getHeaders();
response.getBody();

Spring Boot RestTemplate 코드 예제

실무에서는 RestTemplate을 Service 메서드 안에서 매번 new RestTemplate()으로 만들기보다, @Configuration에서 Bean으로 등록하고 주입받아 사용하는 구조가 좋습니다.

1) RestTemplateConfig - timeout 설정

package com.example.study.global.config;

import org.springframework.boot.web.client.RestTemplateBuilder;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.web.client.RestTemplate;

import java.time.Duration;

@Configuration
public class RestTemplateConfig {

    @Bean
    public RestTemplate restTemplate(RestTemplateBuilder builder) {
        return builder
                .connectTimeout(Duration.ofSeconds(3))
                .readTimeout(Duration.ofSeconds(5))
                .build();
    }
}

3) Client - 외부 API 호출 전담 클래스

package com.example.study.external.client;

import com.example.study.external.dto.ExternalPostCreateRequest;
import com.example.study.external.dto.ExternalPostResponse;
import lombok.RequiredArgsConstructor;
import org.springframework.http.HttpEntity;
import org.springframework.http.HttpHeaders;
import org.springframework.http.HttpMethod;
import org.springframework.http.MediaType;
import org.springframework.http.ResponseEntity;
import org.springframework.stereotype.Component;
import org.springframework.web.client.HttpStatusCodeException;
import org.springframework.web.client.RestTemplate;

import java.util.List;

@Component
@RequiredArgsConstructor
public class ExternalPostClient {

    private final RestTemplate restTemplate;

    public ExternalPostResponse getPost(Long id, String token) {
        String url = "https://api.example.com/posts/{id}";

        HttpHeaders headers = new HttpHeaders();
        headers.setAccept(List.of(MediaType.APPLICATION_JSON));
        headers.setBearerAuth(token);

        HttpEntity<Void> entity = new HttpEntity<>(headers);

        try {
            ResponseEntity<ExternalPostResponse> response =
                    restTemplate.exchange(
                            url,
                            HttpMethod.GET,
                            entity,
                            ExternalPostResponse.class,
                            id
                    );

            if (!response.getStatusCode().is2xxSuccessful()) {
                throw new IllegalStateException("외부 API 호출 실패: " + response.getStatusCode());
            }

            if (response.getBody() == null) {
                throw new IllegalStateException("외부 API 응답 body가 비어 있습니다.");
            }

            return response.getBody();

        } catch (HttpStatusCodeException e) {
            System.out.println("상태 코드: " + e.getStatusCode());
            System.out.println("응답 바디: " + e.getResponseBodyAsString());
            throw e;
        }
    }

    public ExternalPostResponse createPost(ExternalPostCreateRequest request, String token) {
        String url = "https://api.example.com/posts";

        HttpHeaders headers = new HttpHeaders();
        headers.setContentType(MediaType.APPLICATION_JSON);
        headers.setAccept(List.of(MediaType.APPLICATION_JSON));
        headers.setBearerAuth(token);

        HttpEntity<ExternalPostCreateRequest> entity =
                new HttpEntity<>(request, headers);

        ResponseEntity<ExternalPostResponse> response =
                restTemplate.exchange(
                        url,
                        HttpMethod.POST,
                        entity,
                        ExternalPostResponse.class
                );

        if (!response.getStatusCode().is2xxSuccessful()) {
            throw new IllegalStateException("외부 API 등록 실패: " + response.getStatusCode());
        }

        if (response.getBody() == null) {
            throw new IllegalStateException("외부 API 응답 body가 비어 있습니다.");
        }

        return response.getBody();
    }
}

4) Controller - React가 호출하는 내 API

package com.example.study.external.controller;

import com.example.study.external.client.ExternalPostClient;
import com.example.study.external.dto.ExternalPostCreateRequest;
import com.example.study.external.dto.ExternalPostResponse;
import lombok.RequiredArgsConstructor;
import org.springframework.web.bind.annotation.*;

@RestController
@RequestMapping("/api/external/posts")
@RequiredArgsConstructor
public class ExternalPostController {

    private final ExternalPostClient externalPostClient;

    @GetMapping("/{id}")
    public ExternalPostResponse getPost(@PathVariable Long id) {
        String token = "외부_API에서_요구하는_토큰";
        return externalPostClient.getPost(id, token);
    }

    @PostMapping
    public ExternalPostResponse createPost(
            @RequestBody ExternalPostCreateRequest request
    ) {
        String token = "외부_API에서_요구하는_토큰";
        return externalPostClient.createPost(request, token);
    }
}

React Axios 연결 예제

React는 외부 API를 직접 호출하지 않고, Spring Boot의 /api/external/posts를 호출합니다. 그러면 Spring Boot가 내부에서 RestTemplate으로 외부 API를 호출합니다.

1) 토큰 저장 함수

// src/api/tokenStorage.js

const ACCESS_TOKEN_KEY = "accessToken";

export function saveToken(token, keepLogin) {
  if (keepLogin) {
    localStorage.setItem(ACCESS_TOKEN_KEY, token);
    sessionStorage.removeItem(ACCESS_TOKEN_KEY);
    return;
  }

  sessionStorage.setItem(ACCESS_TOKEN_KEY, token);
  localStorage.removeItem(ACCESS_TOKEN_KEY);
}

export function getAccessToken() {
  return (
    localStorage.getItem(ACCESS_TOKEN_KEY) ||
    sessionStorage.getItem(ACCESS_TOKEN_KEY)
  );
}

export function clearToken() {
  localStorage.removeItem(ACCESS_TOKEN_KEY);
  sessionStorage.removeItem(ACCESS_TOKEN_KEY);
}

2) Axios 공통 인스턴스

// src/api/apiClient.js

import axios from "axios";
import { getAccessToken } from "./tokenStorage";

const apiClient = axios.create({
  baseURL: "http://localhost:8080",
  timeout: 5000,
  headers: {
    "Content-Type": "application/json",
  },
});

apiClient.interceptors.request.use(
  (config) => {
    const token = getAccessToken();

    if (token) {
      config.headers.Authorization = `Bearer ${token}`;
    }

    return config;
  },
  (error) => Promise.reject(error)
);

export default apiClient;

4) 폼 입력값을 DTO로 보내기

// src/pages/PostCreatePage.jsx

import { useState } from "react";
import { createExternalPost } from "../api/externalPostApi";

function PostCreatePage() {
  const [form, setForm] = useState({
    title: "",
    content: "",
    writerId: 1,
  });

  const handleChange = (e) => {
    const { name, value } = e.target;

    setForm((prev) => ({
      ...prev,
      [name]: value,
    }));
  };

  const handleSubmit = async (e) => {
    e.preventDefault();

    const requestDto = {
      title: form.title,
      content: form.content,
      writerId: Number(form.writerId),
    };

    try {
      const result = await createExternalPost(requestDto);
      console.log("등록 결과:", result);
      alert("등록 성공");
    } catch (error) {
      console.error(error);
      alert("등록 실패");
    }
  };

  return (
    <form onSubmit={handleSubmit}>
      <input
        name="title"
        value={form.title}
        onChange={handleChange}
        placeholder="제목"
      />

      <textarea
        name="content"
        value={form.content}
        onChange={handleChange}
        placeholder="내용"
      />

      <input
        name="writerId"
        type="number"
        value={form.writerId}
        onChange={handleChange}
      />

      <button type="submit">등록</button>
    </form>
  );
}

export default PostCreatePage;
07

timeout과 connection pool 카드

RestTemplate은 외부 서버 응답을 기다리는 구조이기 때문에 timeout과 connection pool 개념이 중요합니다. 외부 API가 느려지면 내 Spring Boot 서버의 응답도 함께 느려질 수 있습니다.

connectTimeout

연결 대기 시간

외부 서버와 연결을 맺기까지
기다리는 시간입니다.
서버 주소가 틀렸거나
대상 서버가 죽어 있으면 여기서 문제가 납니다.

readTimeout

응답 대기 시간

연결은 되었지만 외부 서버가
응답 데이터를 늦게 줄 때 적용됩니다.
AI 분석 API, 파일 API는
더 길게 잡을 수 있습니다.

connection pool

HTTP 연결 재사용

매번 새 연결을 만들지 않고,
기존 연결을 빌려 쓰고 반납하는 구조입니다.
요청이 많아지는 실무 시스템에서 중요합니다.

TIME_WAIT

연결 종료 후 대기 상태

요청이 많고 연결을 계속 새로 만들면
TIME_WAIT 상태가 많이 쌓여
자원 부족 문제가 생길 수 있습니다.

실무 주의

RestTemplate을 단순히 new RestTemplate()으로 쓰면 실습은 가능하지만, 운영 환경에서는 timeout, connection pool, 에러 처리, 재시도 정책을 함께 고려해야 합니다.

08

List 응답 처리 카드

외부 API가 배열 JSON을 내려줄 때 단순히 List.class로 받으면 내부 요소가 DTO가 아니라 LinkedHashMap으로 들어갈 수 있습니다.

주의

List.class 방식

빠른 테스트는 가능하지만
List 안의 요소 타입이 명확하지 않습니다.
DTO로 바로 다루기 어렵습니다. 

ResponseEntity<List> response =
    restTemplate.exchange(
        url,
        HttpMethod.GET,
        entity,
        List.class
    );
방법 1

배열로 받고 List 변환

실습에서 이해하기 쉬운 방식입니다.
응답 body를 DTO 배열로 받은 뒤 List로 바꿉니다. 

ResponseEntity<PostResponse[]> response =
    restTemplate.exchange(
        url,
        HttpMethod.GET,
        entity,
        PostResponse[].class
    );

List<PostResponse> posts =
    Arrays.asList(response.getBody());
방법 2

ParameterizedTypeReference

제네릭 타입 정보를 유지해서
List<DTO> 형태로 정확히 받을 때 사용합니다. 

ResponseEntity<List<PostResponse>> response =
    restTemplate.exchange(
        url,
        HttpMethod.GET,
        entity,
        new ParameterizedTypeReference<List<PostResponse>>() {}
    );
09

오류 메시지 카드

RestTemplate 오류는 크게 네트워크 오류, HTTP 상태코드 오류, JSON 변환 오류, 설정/의존성 오류로 나눠서 읽으면 됩니다.

오류 메시지 / 현상
의미
확인할 것
ResourceAccessException
외부 서버 접근 실패 또는 네트워크 문제입니다.
URL, 포트, 서버 실행 여부, 방화벽, timeout 설정을 확인합니다.
Connection refused
대상 서버가 연결을 거부했습니다.
외부 서버가 실행 중인지, 포트가 맞는지 확인합니다.
UnknownHostException
도메인 주소를 찾지 못했습니다.
URL 오타, DNS, 인터넷 연결 상태를 확인합니다.
Connect timed out
외부 서버와 연결을 맺는 데 시간이 초과되었습니다.
connectTimeout, 서버 주소, 네트워크 상태를 확인합니다.
Read timed out
연결은 되었지만 응답이 늦었습니다.
readTimeout, 외부 API 처리 지연, AI 분석 시간 등을 확인합니다.
400 Bad Request
요청 데이터가 외부 API 요구 형식과 다릅니다.
DTO 필드명, 필수값, JSON 구조, 날짜/숫자 형식을 확인합니다.
401 Unauthorized
인증 정보가 없거나 틀렸습니다.
Authorization 헤더, Bearer 접두사, 토큰 만료 여부를 확인합니다.
403 Forbidden
인증은 되었지만 권한이 없습니다.
사용자 role, API Key 권한, 호출 허용 IP를 확인합니다.
404 Not Found
요청한 API 경로가 없습니다.
URL, PathVariable, endpoint 문서와 실제 경로를 확인합니다.
415 Unsupported Media Type
Content-Type이 외부 API가 요구하는 형식과 다릅니다.
headers.setContentType(MediaType.APPLICATION_JSON) 설정을 확인합니다.
HttpMessageConversionException
JSON을 DTO로 변환하지 못했습니다.
DTO 필드 타입, 필드명, 중첩 구조, 배열/객체 여부를 확인합니다.
LinkedHashMap cannot be cast
List 응답을 DTO List로 잘못 처리했습니다.
배열 응답 또는 ParameterizedTypeReference를 사용합니다.
NoSuchBeanDefinitionException
RestTemplate Bean을 찾지 못했습니다.
RestTemplateConfig의 @Bean 등록과 컴포넌트 스캔 범위를 확인합니다.

오류 응답 body 확인 코드

try {
    ResponseEntity<ExternalPostResponse> response =
            restTemplate.exchange(
                    url,
                    HttpMethod.GET,
                    entity,
                    ExternalPostResponse.class
            );

    return response.getBody();

} catch (HttpStatusCodeException e) {
    System.out.println("상태 코드: " + e.getStatusCode());
    System.out.println("응답 헤더: " + e.getResponseHeaders());
    System.out.println("응답 바디: " + e.getResponseBodyAsString());

    throw e;
}

자주 하는 실수 카드

1. new RestTemplate()을 매번 생성

메서드마다 새로 만들면 timeout, pool,
공통 설정을 관리하기 어렵습니다.
Bean으로 등록하고 주입받아
사용하는 것이 좋습니다.

2. timeout 설정 누락

외부 API가 멈추면
내 서버도 계속 기다릴 수 있습니다.
connectTimeout과 readTimeout을
설정해야 합니다.

3. Content-Type 누락

JSON을 보내면서
Content-Type을 지정하지 않으면
415 Unsupported Media Type이
발생할 수 있습니다.

4. Authorization 헤더 누락

인증이 필요한 API에 토큰을 보내지 않으면
401 또는 403 오류가 발생할 수 있습니다.

5. URL을 문자열 더하기로 조립

한글, 공백, 특수문자가 들어가면
URL이 깨질 수 있습니다.
UriComponentsBuilder를
사용하는 것이 안전합니다.

6. List.class로 DTO List를 기대

List 내부 요소가
LinkedHashMap이 될 수 있습니다.
배열 또는 ParameterizedTypeReference를
사용합니다.

7. 외부 API Key를 React에 작성

React 코드는 브라우저에 노출됩니다.
API Key는 Spring Boot 서버 환경변수로
관리하는 것이 좋습니다.

8. Controller에 외부 호출 로직 집중

Controller는 요청/응답만 담당하고,
외부 API 호출은 Client 또는 Service로
분리하는 것이 좋습니다.

9. 외부 API DTO를
그대로 프론트에 노출

외부 API 구조가 바뀌면
내 프론트도 흔들립니다.
내부 응답 DTO로 변환해서
반환하는 것이 안전합니다.

10. CORS와
RestTemplate 문제 혼동

CORS는 브라우저 보안 정책입니다.
Spring Boot가 RestTemplate으로
호출하는 서버 간 통신과는 성격이 다릅니다.

면접 포인트 카드

Q1. RestTemplate이란 무엇인가요?

Spring에서 제공하는 동기 방식 HTTP 클라이언트입니다. Spring Boot 서버 내부에서 외부 REST API나 다른 서버 API를 호출할 때 사용합니다.

Q2. Axios와 RestTemplate의 차이는 무엇인가요?

Axios는 브라우저에서 실행되어 React가 Spring Boot API를 호출할 때 사용하고, RestTemplate은 Spring Boot 서버 내부에서 외부 API를 호출할 때 사용합니다.

Q3. RestTemplate의 기본 호출 흐름은 무엇인가요?

URL 준비, HttpHeaders 작성, HttpEntity 생성, exchange 호출, ResponseEntity에서 status/header/body 확인 순서입니다.

Q4. HttpHeaders는 무엇인가요?

HTTP 요청의 부가 정보를 담는 객체입니다. Content-Type, Accept, Authorization, X-API-KEY 등을 설정합니다.

Q5. HttpEntity는 무엇인가요?

요청 header와 body를 함께 담는 객체입니다. RestTemplate의 exchange()에 전달됩니다.

Q6. exchange()를 사용하는 이유는 무엇인가요?

HTTP Method, URL, Header, Body, 응답 타입을 모두 명시적으로 제어할 수 있기 때문입니다. GET, POST, PUT, DELETE를 한 패턴으로 처리할 수 있습니다.

Q7. ResponseEntity는 무엇인가요?

HTTP 응답 전체를 담는 객체입니다. 상태코드, 응답 헤더, 응답 body를 함께 확인할 수 있습니다.

Q8. timeout 설정은 왜 중요한가요?

외부 API가 응답하지 않을 때 서버가 무한정 기다리는 것을 막기 위해 필요합니다. timeout이 없으면 요청 처리 스레드가 오래 점유되어 서비스가 느려질 수 있습니다.

Q9. connection timeout과 read timeout의 차이는 무엇인가요?

connection timeout은 외부 서버와 연결을 맺기까지 기다리는 시간이고, read timeout은 연결 후 응답 데이터를 기다리는 시간입니다.

Q10. connection pool은 왜 필요한가요?

HTTP 연결을 매번 새로 만들지 않고 재사용하기 위해 필요합니다. 성능을 높이고 연결 자원 고갈을 방지하는 데 도움이 됩니다.

Q11. List.class로 응답을 받으면 어떤 문제가 생길 수 있나요?

List 내부 요소가 DTO가 아니라 LinkedHashMap으로 들어갈 수 있습니다. 배열 응답이나 ParameterizedTypeReference를 사용하는 것이 좋습니다.

Q12. 외부 API Key는 어디에 두는 것이 좋나요?

React 코드가 아니라 Spring Boot 서버 쪽에 두는 것이 좋습니다. React에 작성하면 브라우저에서 노출될 수 있습니다.

학습 메모

오늘의 암기 포인트

  • RestTemplate은 Spring Boot 서버 내부에서 외부 REST API를 호출하는 동기 HTTP 클라이언트입니다.
  • React Axios는 브라우저에서 Spring Boot를 호출하고, RestTemplate은 Spring Boot가 외부 서버를 호출합니다.
  • RestTemplate 기본 흐름은 URL → HttpHeaders → HttpEntity → exchange() → ResponseEntity입니다.
  • HttpHeaders는 Content-Type, Accept, Authorization 같은 요청 헤더를 담습니다.
  • HttpEntity는 요청 header와 body를 함께 담는 요청 상자입니다.
  • exchange()는 HTTP Method, URL, Header, Body, 응답 타입을 모두 제어할 수 있습니다.
  • ResponseEntity는 statusCode, headers, body를 함께 담습니다.
  • timeout을 설정하지 않으면 외부 API 지연이 내 서버 지연으로 이어질 수 있습니다.
  • 요청이 많은 서비스에서는 connection pool을 고려해야 합니다.
  • List 응답은 List.class보다 배열 또는 ParameterizedTypeReference로 받는 것이 안전합니다.

헷갈리기 쉬운 개념

Axios vs RestTemplate

Axios는 브라우저에서,
RestTemplate은 Spring Boot 서버 내부에서 실행됩니다.

HttpHeaders vs HttpEntity

Headers는 부가 정보,
Entity는 header와 body를 묶은 요청 상자입니다.

Request Body vs Response Body

Request Body는 내가 보내는 데이터,
Response Body는 외부 API가 돌려준 데이터입니다.

401 vs 403

401은 인증 실패,
403은 인증은 되었지만 권한이 없는 상태입니다.

connectTimeout vs readTimeout

connect는 연결 대기,
read는 응답 데이터 대기 시간입니다.

List.class vs DTO 배열

List.class는 LinkedHashMap 문제가 생길 수 있어
DTO 배열이나 ParameterizedTypeReference가 안전합니다.
