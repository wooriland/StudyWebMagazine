Spring Boot - Chapter 05

Generic 기본

Generic은 클래스나 메소드에서 사용할 타입을 미리 고정하지 않고, 사용하는 시점에 타입을 결정하는 Java 문법입니다.
이 단원에서는 타입 안정성, 제네릭 클래스, 제네릭 인터페이스, 제네릭 메소드를 정리하고,
Spring Boot에서 자주 만나는 List<User>, ResponseEntity<LoginResponse>, JpaRepository<User, Long>을 읽는 방법을 익힙니다.

01

Generic이란 무엇인가?

Generic은 타입을 나중에 정하는 문법입니다. 클래스를 만들 때 특정 타입을 고정하지 않고, 사용할 때 Box<String>, Box<Integer>, Box<UserResponse>처럼 타입을 지정합니다.
교안에서는 Generic을 JDK 5에서 도입된 문법으로 설명하며, 클래스나 메소드에서 사용할 데이터 타입을 정의 시점이 아니라 사용 시점에 결정한다고 정리합니다.
이 덕분에 컴파일 시점에 타입을 체크하고, 불필요한 형변환을 줄이고, 같은 코드를 여러 타입에 재사용할 수 있습니다.

정의 시점 Box<T>

아직 어떤 타입을 담을지
정하지 않은 일반화된 상자입니다.

→
사용 시점 Box<String>

T가 String으로 결정됩니다.

사용 시점 Box<Integer>

T가 Integer로 결정됩니다.

사용 시점 Box<UserResponse>

T가 UserResponse로 결정됩니다.

핵심 문장

Generic은 타입을 값처럼 나중에 끼워 넣는 문법이며, Spring Boot 코드의 타입 구조를 읽기 위한 기본 문법입니다.

02

Generic을 사용하는 이유

Generic을 사용하는 가장 큰 이유는 타입 안정성입니다. 잘못된 타입이 들어오는 것을 컴파일 단계에서 막아주고, 값을 꺼낼 때 강제 형변환을 하지 않아도 됩니다.

1

컴파일 시 타입 체크

List<String>에는 String만 넣을 수 있으므로
Integer를 넣으면 실행 전에 컴파일 에러가 납니다.

2

형변환 감소

String name = names.get(0)처럼
값을 꺼낼 때 직접 캐스팅하지 않아도 됩니다.

3

코드 재사용

Box<T> 하나로 String, Integer, DTO 등
여러 타입을 처리할 수 있습니다.

Generic 없음

List list = new ArrayList();

list.add("hello");
list.add(100);

String value = (String) list.get(1);

실행 중 ClassCastException이 발생할 수 있습니다.

Generic 있음

List<String> names = new ArrayList<>();

names.add("kim");
names.add("lee");
// names.add(100); 컴파일 에러

String name = names.get(0);

잘못된 타입을 컴파일 단계에서 막습니다.

03

<T>는 실제 타입이 아니라 타입이 들어갈 자리입니다. 보통 대문자 한 글자를 사용하며, 이름은 관례일 뿐 반드시 정해져 있는 것은 아닙니다.

04

제네릭 클래스, 인터페이스, 메소드

Generic은 클래스, 인터페이스, 메소드에 사용할 수 있습니다. 클래스 전체에서 타입을 공유할 수도 있고, 메소드 하나에서만 타입을 따로 선언할 수도 있습니다.

Class

제네릭 클래스

클래스 이름 뒤에 타입 파라미터를 선언합니다.

public class Box<T> {
    private T value;

    public T get() {
        return value;
    }
}
Interface

제네릭 인터페이스

구현할 때 구체 타입을 지정합니다.

public interface Repository<T, ID> {
    T save(T entity);
    T findById(ID id);
}
Method

제네릭 메소드

반환 타입 앞에 타입 파라미터를 선언합니다.

public static <T> T identity(T value) {
    return value;
}
05

그림처럼 이해하기

Generic은 “라벨이 붙은 상자”처럼 이해하면 쉽습니다. List<String>은 String 전용 상자이고, List<UserResponse>는 UserResponse 전용 상자입니다.

라벨 없음

Raw List

"hello", 100, true가 모두 들어갈 수 있습니다.

"hello" 100 true

"kim" "lee" "park"

data에 들어가는 사용자 응답 목록

06

기본 Java Generic 예제

먼저 Generic을 쓰지 않은 코드와 Generic을 사용한 코드를 비교합니다. 차이는 형변환 필요 여부와 컴파일 시 타입 체크 여부입니다.

1) Generic을 사용하지 않은 Box

public class NonGenericBox {
    private Object value;

    public void set(Object value) {
        this.value = value;
    }

    public Object get() {
        return value;
    }
}

NonGenericBox box = new NonGenericBox();

box.set("hello");

String value = (String) box.get();

2) Generic을 사용한 Box

public class Box<T> {
    private T value;

    public void set(T value) {
        this.value = value;
    }

    public T get() {
        return value;
    }
}

Box<String> stringBox = new Box<>();

stringBox.set("hello");

String value = stringBox.get();

3) 제네릭 인터페이스

public interface SimpleRepository<T, ID> {
    T save(T entity);

    T findById(ID id);

    List<T> findAll();

    void deleteById(ID id);
}

public class UserMemoryRepository
        implements SimpleRepository<User, Long> {

    @Override
    public User save(User user) {
        return user;
    }

    @Override
    public User findById(Long id) {
        return new User(id, "user1");
    }

    @Override
    public List<User> findAll() {
        return List.of(new User(1L, "user1"));
    }

    @Override
    public void deleteById(Long id) {
    }
}

4) 제네릭 메소드

public class GenericUtils {

    public static <T> T identity(T value) {
        return value;
    }
}

String text = GenericUtils.identity("hello");
Integer number = GenericUtils.identity(100);
Boolean flag = GenericUtils.identity(true);

Spring Boot Generic 예제

Spring Boot에서는 Generic이 API 응답, Repository, Optional, ResponseEntity에서 계속 등장합니다. 특히 ApiResponse<T>를 만들면 응답 구조를 통일할 수 있습니다.

1) Generic 공통 응답

public record ApiResponse<T>(
    boolean ok,
    String message,
    T data
) {
    public static <T> ApiResponse<T> success(T data) {
        return new ApiResponse<>(true, "요청이 성공했습니다.", data);
    }

    public static <T> ApiResponse<T> fail(String message) {
        return new ApiResponse<>(false, message, null);
    }
}

2) UserResponse DTO

public record UserResponse(
    Long id,
    String username,
    String email

}

3) Controller에서 List<UserResponse> 반환

@RestController
@RequestMapping("/api/users")
@RequiredArgsConstructor
public class UserController {

    private final UserService userService;

    @GetMapping
    public ResponseEntity<ApiResponse<List<UserResponse>>> findAll() {
        List<UserResponse> users = userService.findAll();

        return ResponseEntity.ok(
            ApiResponse.success(users)
        );
    }

    @GetMapping("/{id}")
    public ResponseEntity<ApiResponse<UserResponse>> findById(
            @PathVariable Long id
    ) {
        UserResponse user = userService.findById(id);

        return ResponseEntity.ok(
            ApiResponse.success(user)
        );
    }
}

React에서 Axios로 API 호출하기

Java의 Generic 타입 구조는 React에서 받는 JSON 구조와 연결됩니다.
예를 들어 ResponseEntity<ApiResponse<List<UserResponse>>>는 React에서 response.data.data가 사용자 배열이라는 뜻입니다.

1) Axios 공통 인스턴스와 토큰 자동 첨부

import axios from "axios";
import { getAccessToken, clearToken } from "../utils/tokenStorage";

const apiClient = axios.create({
  baseURL: "http://localhost:8080",
  headers: {
    "Content-Type": "application/json",
  },
});

apiClient.interceptors.request.use(
  (config) => {
    const token = getAccessToken();

    if (token) {
      config.headers.Authorization = `Bearer ${token}`;
    }

    return config;
  },
  (error) => Promise.reject(error)
);

apiClient.interceptors.response.use(
  (response) => response,
  (error) => {
    if (error.response?.status === 401) {
      clearToken();
      window.location.href = "/login";
    }

    return Promise.reject(error);
  }
);

export default apiClient;

2) 토큰 저장 함수

export function saveToken(token, keepLogin) {
  if (keepLogin) {
    localStorage.setItem("accessToken", token);
    sessionStorage.removeItem("accessToken");
  } else {
    sessionStorage.setItem("accessToken", token);
    localStorage.removeItem("accessToken");
  }
}

export function getAccessToken() {
  return (
    localStorage.getItem("accessToken") ||
    sessionStorage.getItem("accessToken")
  );
}

export function clearToken() {
  localStorage.removeItem("accessToken");
  sessionStorage.removeItem("accessToken");
}

3) 사용자 목록 조회 API

import apiClient from "./apiClient";

export async function fetchUsers() {
  const response = await apiClient.get("/api/users");
  return response.data;
}

4) React 목록 렌더링

import { useEffect, useState } from "react";
import { fetchUsers } from "../api/userApi";

function UserList() {
  const [users, setUsers] = useState([]);
  const [message, setMessage] = useState("");

  useEffect(() => {
    async function loadUsers() {
      try {
        const result = await fetchUsers();

        if (result.ok) {
          setUsers(result.data);
          setMessage(result.message);
        } else {
          setMessage(result.message);
        }
      } catch (error) {
        setMessage("사용자 목록을 불러오지 못했습니다.");
      }
    }

    loadUsers();
  }, []);

  return (
    <section>
      <h2>사용자 목록</h2>

      {message && <p>{message}</p>}

      <ul>
        {users.map((user) => (
          <li key={user.id}>
            {user.username} / {user.email}
          </li>
        ))}
      </ul>
    </section>
  );
}

export default UserList;

3) React 로그인 폼

import { useState } from "react";
import { login } from "../api/authApi";
import { saveToken } from "../utils/tokenStorage";

function LoginForm() {
  const [form, setForm] = useState({
    username: "",
    password: "",
    keepLogin: true,
  });

  const [message, setMessage] = useState("");

  function handleChange(event) {
    const { name, value, type, checked } = event.target;

    setForm((prev) => ({
      ...prev,
      [name]: type === "checkbox" ? checked : value,
    }));
  }

  async function handleSubmit(event) {
    event.preventDefault();

    try {
      const result = await login(form.username, form.password);

      if (result.ok) {
        saveToken(result.data.token, form.keepLogin);
        setMessage("로그인 성공");
      } else {
        setMessage(result.message);
      }
    } catch (error) {
      setMessage("아이디 또는 비밀번호가 일치하지 않습니다.");
    }
  }

  return (
    <form onSubmit={handleSubmit}>
      <input
        name="username"
        value={form.username}
        onChange={handleChange}
        placeholder="아이디"
      />

      <input
        type="password"
        name="password"
        value={form.password}
        onChange={handleChange}
        placeholder="비밀번호"
      />

      <label>
        <input
          type="checkbox"
          name="keepLogin"
          checked={form.keepLogin}
          onChange={handleChange}
        />
        로그인 유지
      </label>

      <button type="submit">로그인</button>

      {message && <p>{message}</p>}
    </form>
  );
}

export default LoginForm;

오류 메시지 읽는 법

Generic 관련 오류는 주로 타입이 맞지 않거나, 응답 구조를 잘못 해석하거나, Raw Type을 사용할 때 발생합니다.

오류 메시지
의미
확인할 것
ClassCastException
실제 객체 타입과 강제 형변환 타입이 다릅니다.
Raw Type을 쓰지 말고 List<String>처럼 타입을 명시합니다.
unchecked or unsafe operations
컴파일러가 타입 안정성을 보장하기 어려운 코드가 있습니다.
Raw Type, 강제 캐스팅, 제네릭 타입 누락 여부를 확인합니다.
Raw use of parameterized class
Generic 타입이 필요한 클래스에 타입 파라미터를 생략했습니다.
List users 대신 List<UserResponse> users로 작성합니다.
incompatible types
기대 타입과 실제 타입이 맞지 않습니다.
List<String>에 Integer를 넣지 않았는지 확인합니다.
cannot infer type arguments
컴파일러가 Generic 타입을 추론하지 못했습니다.
null만 전달하거나 반환 타입이 모호한지 확인합니다.
no instance(s) of type variable(s) exist
Generic 메소드의 반환 타입과 기대 타입이 맞지 않습니다.
ApiResponse<UserResponse>를
UserResponse 변수에 담고 있지 않은지 확인합니다.
Required type / Provided type
Controller 선언 타입과 실제 반환 타입이 다릅니다.
목록이면 ApiResponse<List<UserResponse>>인지 확인합니다.
Cannot read properties of undefined
React에서 JSON 경로를 잘못 읽었습니다.
response.data.token인지 response.data.data.token인지 확인합니다.
users.map is not a function
배열이 아닌 객체에 map을 호출했습니다.
공통 응답 구조라면 실제 배열은 result.data 안에 있을 수 있습니다.
400 Bad Request
React JSON과 Spring DTO 구조가 맞지 않을 수 있습니다.
@RequestBody, Content-Type, JSON key, DTO 필드명을 확인합니다.

자주 하는 실수

1. Raw Type 사용

List users처럼 쓰면
타입 안정성이 사라집니다. List<UserResponse>처럼 작성합니다.

2. Object로 모든 응답 처리

Object data는
형변환 오류를 만들 수 있습니다. ApiResponse<T>로 타입을 보존합니다.

3. List<Object>와
List<String> 혼동

String은 Object의 자식이지만, List<String>은
List<Object>의 자식이 아닙니다.

4. new T() 사용

Java Generic에서는
new T()를 사용할 수 없습니다.
외부에서 객체를 주입하거나
Supplier를 사용합니다.

5. static T 필드 사용

클래스 타입 파라미터는
객체 생성 시점에 결정되므로
static 필드에 사용할 수 없습니다.

6. 클래스 T와 메소드 T 혼동

제네릭 클래스의 T와
제네릭 메소드의 T는
서로 독립적일 수 있습니다.

실무 연결 카드

Generic은 Java 문법이지만, Spring Boot 실무 코드 거의 모든 계층에 등장합니다. Repository, DTO, API 응답, JWT 로그인, 페이징, RestTemplate 응답 타입까지 모두 Generic과 연결됩니다.

List<UserResponse>,
Map<String, Object>처럼
데이터 묶음의 타입을 명확히 표현합니다.

Repository

JpaRepository<User, Long>으로
Entity 타입과 PK 타입을 지정합니다.

API 응답 공통화

ApiResponse<T>로
여러 응답 DTO를 하나의 구조로
감쌀 수 있습니다.

면접 포인트

Q1. Generic이 무엇인가요?

Generic은 클래스나 메소드에서 사용할 타입을 정의 시점이 아니라 사용 시점에 결정하는 Java 문법입니다. 타입 안정성을 확보하고, 형변환을 줄이며, 하나의 코드로 여러 타입을 처리할 수 있게 해줍니다.

Q2. Generic을 사용하는 이유는 무엇인가요?

컴파일 시점에 타입을 체크할 수 있고, 잘못된 타입 사용을 미리 막을 수 있습니다. 또한 값을 꺼낼 때 형변환이 필요 없고, 코드 재사용성이 좋아집니다.

Q3. 제네릭 클래스와 제네릭 메소드의 차이는 무엇인가요?

제네릭 클래스는 클래스 전체에 타입 파라미터를 선언하고, 객체 생성 시 타입이 결정됩니다. 제네릭 메소드는 메소드 자체에 타입 파라미터를 선언하고, 메소드 호출 시 타입이 결정됩니다.

Q4. 왜 static T 필드는 사용할 수 없나요?

클래스의 타입 파라미터 T는 객체 생성 시점에 결정됩니다. 하지만 static 필드는 클래스 로딩 시점에 생성되고 모든 객체가 공유하므로 T를 하나로 결정할 수 없습니다.

Q5. 왜 new T()는 사용할 수 없나요?

Java Generic은 런타임에 타입 정보가 지워지는 타입 소거 방식을 사용합니다. 따라서 런타임에 T가 어떤 생성자를 가진 클래스인지 알 수 없어 new T()를 사용할 수 없습니다.

Q6. ResponseEntity<LoginResponse>는 무슨 뜻인가요?

HTTP 응답을 표현하는 객체이며, 응답 body의 타입이 LoginResponse라는 뜻입니다. 상태 코드, 헤더, body를 함께 제어할 수 있습니다.

Q7. JpaRepository<User, Long>은 무슨 뜻인가요?

User 엔티티를 관리하는 Repository이고, User의 PK 타입이 Long이라는 뜻입니다. 이 타입 정보에 따라 findById, save, findAll 같은 메소드의 타입이 결정됩니다.

Q8. React와 Generic은 어떻게 연결되나요?

Java의 Generic 타입 구조는 Spring Boot가 내려주는 JSON 구조와 연결됩니다.
ApiResponse<List<UserResponse>>라면 React에서는 실제 배열이 response.data.data에 들어있다는 것을 예측할 수 있습니다.

학습 메모

오늘의 암기 포인트

  • Generic은 타입을 사용 시점에 결정하는 Java 문법입니다.
  • <T>는 실제 타입이 아니라 타입이 들어갈 자리입니다.
  • Generic을 사용하면 컴파일 시점에 타입 체크가 가능하고 형변환을 줄일 수 있습니다.
  • List<UserResponse>는 UserResponse만 담는 목록입니다.
  • ResponseEntity<LoginResponse>는 응답 body가 LoginResponse 타입이라는 뜻입니다.
  • JpaRepository<User, Long>은 User 엔티티를 다루고 PK 타입은 Long이라는 뜻입니다.
  • Generic 타입 구조를 이해하면 React의 response.data 구조도 예측할 수 있습니다.
