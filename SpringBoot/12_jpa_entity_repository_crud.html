01

JPA Entity와 Repository CRUD를 공부한다는 것

이전 단원에서는 Spring Boot가 DataSource와 HikariCP를 통해 DB Connection을 빌려 쓰는 구조를 공부했습니다.
이번 단원에서는 그 연결 위에서 JPA가 Java 객체를 DB 테이블에 저장하고 조회하는 방법을 공부합니다.
JPA는 SQL을 완전히 없애는 기술이 아니라, Java Entity 객체 중심으로 DB 작업을 하게 해주는 ORM 표준 기술입니다.
React 게시판에서 글을 등록하면 React는 JSON을 보냅니다.
Spring Boot Controller는 이 JSON을 Request DTO로 받고, Service는 DTO를 Entity로 바꿔 Repository에 저장합니다.
Repository는 JPA와 Hibernate를 통해 SQL을 만들고, 결국 JDBC, DataSource, HikariCP를 거쳐 DB에 접근합니다.

04

Entity Annotation 카드

Entity는 DB 테이블과 매핑되는 Java 클래스입니다. JPA가 Entity를 관리하려면 @Entity, @Id 같은 핵심 어노테이션이 반드시 필요합니다.

@Entity

이 클래스가 JPA가 관리하는 Entity이며
DB 테이블과 매핑된다는 뜻입니다.

@Table

매핑할 테이블 이름을 지정합니다.
생략하면 클래스명을 기준으로 매핑됩니다.

@Id

Primary Key 필드를 지정합니다.
Entity에는 반드시 식별자가 필요합니다.

@GeneratedValue

Primary Key 값을 자동 생성하는
전략을 지정합니다.

@SequenceGenerator

Oracle Sequence와
JPA Entity의 PK 생성 전략을 연결합니다.

@Column

컬럼명, null 허용 여부, 길이,
unique 조건 등을 지정합니다.

@Lob

긴 문자열이나 바이너리 데이터를
CLOB/BLOB 계열로 저장할 때 사용합니다.

@Transient

Java 객체에는 필요하지만
DB 컬럼으로는 만들지 않을 필드에 사용합니다.

Post Entity

package com.example.studyapi.post.entity;

import jakarta.persistence.*;
import org.hibernate.annotations.CreationTimestamp;

import java.time.LocalDateTime;

@Entity
@Table(name = "posts")
public class Post {

    @Id
    @GeneratedValue(strategy = GenerationType.SEQUENCE, generator = "seq_posts")
    @SequenceGenerator(
            name = "seq_posts",
            sequenceName = "seq_posts",
            initialValue = 1,
            allocationSize = 1
    )
    private Long id;

    @Column(nullable = false, length = 100)
    private String title;

    @Lob
    @Column(nullable = false)
    private String content;

    @Column(nullable = false, length = 50)
    private String writer;

    @CreationTimestamp
    @Column(nullable = false, updatable = false)
    private LocalDateTime createdAt;

    protected Post() {
    }

    public Post(String title, String content, String writer) {
        this.title = title;
        this.content = content;
        this.writer = writer;
    }

    public void update(String title, String content) {
        this.title = title;
        this.content = content;
    }

    public Long getId() {
        return id;
    }

    public String getTitle() {
        return title;
    }

    public String getContent() {
        return content;
    }

    public String getWriter() {
        return writer;
    }

    public LocalDateTime getCreatedAt() {
        return createdAt;
    }
}

PostService

package com.example.studyapi.post.service;

import com.example.studyapi.post.dto.PostCreateRequest;
import com.example.studyapi.post.dto.PostResponse;
import com.example.studyapi.post.dto.PostUpdateRequest;
import com.example.studyapi.post.entity.Post;
import com.example.studyapi.post.exception.PostNotFoundException;
import com.example.studyapi.post.repository.PostRepository;
import lombok.RequiredArgsConstructor;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

import java.util.List;

@Service
@RequiredArgsConstructor
public class PostService {

    private final PostRepository postRepository;

    @Transactional
    public PostResponse create(PostCreateRequest request) {
        Post post = new Post(
                request.title(),
                request.content(),
                request.writer()
        );

        Post savedPost = postRepository.save(post);

        return PostResponse.from(savedPost);
    }

    @Transactional(readOnly = true)
    public List<PostResponse> findAll() {
        return postRepository.findAll()
                .stream()
                .map(PostResponse::from)
                .toList();
    }

    @Transactional(readOnly = true)
    public PostResponse findById(Long id) {
        Post post = postRepository.findById(id)
                .orElseThrow(() -> new PostNotFoundException(id));

        return PostResponse.from(post);
    }

    @Transactional(readOnly = true)
    public List<PostResponse> searchByTitle(String keyword) {
        return postRepository.findByTitleContainingOrderByIdDesc(keyword)
                .stream()
                .map(PostResponse::from)
                .toList();
    }

    @Transactional
    public PostResponse update(Long id, PostUpdateRequest request) {
        Post post = postRepository.findById(id)
                .orElseThrow(() -> new PostNotFoundException(id));

        post.update(request.title(), request.content());

        return PostResponse.from(post);
    }

    @Transactional
    public void delete(Long id) {
        if (!postRepository.existsById(id)) {
            throw new PostNotFoundException(id);
        }

        postRepository.deleteById(id);
    }
}

PostController

package com.example.studyapi.post.controller;

import com.example.studyapi.common.ApiResponse;
import com.example.studyapi.post.dto.PostCreateRequest;
import com.example.studyapi.post.dto.PostResponse;
import com.example.studyapi.post.dto.PostUpdateRequest;
import com.example.studyapi.post.service.PostService;
import jakarta.validation.Valid;
import lombok.RequiredArgsConstructor;
import org.springframework.http.HttpStatus;
import org.springframework.http.ResponseEntity;
import org.springframework.web.bind.annotation.*;

import java.util.List;

@RestController
@RequestMapping("/api/posts")
@RequiredArgsConstructor
public class PostController {

    private final PostService postService;

    @PostMapping
    public ResponseEntity<ApiResponse<PostResponse>> create(
            @Valid @RequestBody PostCreateRequest request
    ) {
        PostResponse response = postService.create(request);

        return ResponseEntity
                .status(HttpStatus.CREATED)
                .body(ApiResponse.success("게시글이 등록되었습니다.", response));
    }

    @GetMapping
    public ResponseEntity<ApiResponse<List<PostResponse>>> findAll() {
        List<PostResponse> response = postService.findAll();

        return ResponseEntity.ok(ApiResponse.success(response));
    }

    @GetMapping("/{id}")
    public ResponseEntity<ApiResponse<PostResponse>> findById(
            @PathVariable Long id
    ) {
        PostResponse response = postService.findById(id);

        return ResponseEntity.ok(ApiResponse.success(response));
    }

    @GetMapping("/search")
    public ResponseEntity<ApiResponse<List<PostResponse>>> search(
            @RequestParam(defaultValue = "") String keyword
    ) {
        List<PostResponse> response = postService.searchByTitle(keyword);

        return ResponseEntity.ok(ApiResponse.success(response));
    }

    @PutMapping("/{id}")
    public ResponseEntity<ApiResponse<PostResponse>> update(
            @PathVariable Long id,
            @Valid @RequestBody PostUpdateRequest request
    ) {
        PostResponse response = postService.update(id, request);

        return ResponseEntity.ok(
                ApiResponse.success("게시글이 수정되었습니다.", response)
        );
    }

    @DeleteMapping("/{id}")
    public ResponseEntity<Void> delete(
            @PathVariable Long id
    ) {
        postService.delete(id);

        return ResponseEntity.noContent().build();
    }
}

CRUD와 Dirty Checking 흐름

JPA 수정은 단순히 SQL UPDATE를 직접 작성하는 방식이 아닙니다.
트랜잭션 안에서 Entity를 조회하면 JPA가 그 Entity를 관리하고, 필드 값을 변경하면 커밋 시점에 변경 감지로 UPDATE SQL을 생성합니다.

Create

등록

Request DTO → Entity 생성 →
save() → INSERT

Read

조회

findById(), findAll() → Entity 조회 →
Response DTO 변환

Update

수정

findById() → Managed Entity →
update() → Dirty Checking → UPDATE

Delete

삭제

existsById() → deleteById() → DELETE

Dirty Checking 흐름

findById(id)로 조회한 Entity는 트랜잭션 안에서 Managed 상태가 됩니다.
이 상태의 Entity 값을 변경하면 JPA가 처음 스냅샷과 현재 값을 비교하고, 변경이 있으면 커밋 시점에 UPDATE SQL을 실행합니다.

React에서 Axios로 CRUD API 호출하기

React는 Entity를 직접 알 필요가 없습니다. React는 Request DTO에 맞는 JSON을 보내고, Spring Boot는 Entity로 변환한 뒤 Repository로 저장합니다.

토큰 저장 함수

export function saveToken(token, keepLogin) {
  if (keepLogin) {
    localStorage.setItem("accessToken", token);
    sessionStorage.removeItem("accessToken");
  } else {
    sessionStorage.setItem("accessToken", token);
    localStorage.removeItem("accessToken");
  }
}

export function getAccessToken() {
  return (
    localStorage.getItem("accessToken") ||
    sessionStorage.getItem("accessToken")
  );
}

export function clearToken() {
  localStorage.removeItem("accessToken");
  sessionStorage.removeItem("accessToken");
}

Axios 공통 인스턴스

import axios from "axios";
import { getAccessToken, clearToken } from "../utils/tokenStorage";

const apiClient = axios.create({
  baseURL: "http://localhost:8080",
  headers: {
    "Content-Type": "application/json",
  },
  timeout: 5000,
});

apiClient.interceptors.request.use(
  (config) => {
    const token = getAccessToken();

    if (token) {
      config.headers.Authorization = `Bearer ${token}`;
    }

    return config;
  },
  (error) => Promise.reject(error)
);

apiClient.interceptors.response.use(
  (response) => response,
  (error) => {
    if (error.response?.status === 401) {
      clearToken();
      window.location.href = "/login";
    }

    return Promise.reject(error);
  }
);

export default apiClient;

findById(id).orElseThrow(...)로 처리합니다.
PropertyReferenceException
Query Method에 작성한 필드명이 Entity에 없습니다.
findByUsername이 아니라 실제 필드명인 findByWriter인지 확인합니다.
Ambiguous handler methods mapped
Controller URL 패턴이 충돌합니다.
/posts/{id}, /posts/search처럼 경로를 명확히 분리합니다.
ORA-02289: sequence does not exist
Entity가 사용하는 Oracle Sequence가 DB에 없습니다.
CREATE SEQUENCE seq_posts와 @SequenceGenerator 이름을 확인합니다.
ORA-01400: cannot insert NULL
NOT NULL 컬럼에 null을 넣으려고 했습니다.
React JSON key, DTO 필드명, Entity 생성자를 확인합니다.
DataIntegrityViolationException
DB 제약 조건을 위반했습니다.
NOT NULL, UNIQUE, FK, 문자열 길이 초과를 확인합니다.
LazyInitializationException
영속성 컨텍스트가 닫힌 뒤 Lazy 필드에 접근했습니다.
Service 트랜잭션 안에서 DTO로 변환하고 Entity 직접 반환을 피합니다.
detached entity passed to persist
분리된 Entity를 새 Entity처럼 저장하려고 했습니다.
기존 Entity는 findById로 조회해 Managed 상태로 만든 뒤 수정합니다.

자주 하는 실수

1. Entity를 그대로 API에 사용

DB 구조와 민감 필드가 노출될 수 있으므로
Request DTO와 Response DTO를
분리합니다.

2. 기본 생성자 누락

JPA Entity는
protected 기본 생성자가 필요합니다.

3. @Id 누락

Entity에는
반드시 Primary Key 필드가 있어야 합니다.

4. Repository 제네릭 타입 오류

Entity의 @Id 타입이 Long이면
JpaRepository<Post, Long>이어야 합니다.

5. findById().get() 사용

값이 없으면 오류가 나므로
orElseThrow로 명확히 처리합니다.

6. 수정 시 새 Entity 생성

수정은 기존 Entity를 조회한 뒤
값만 변경하는 방식이 안전합니다.

7. @Transactional 누락

Dirty Checking이 제대로 동작하려면
Service 메서드에 트랜잭션 경계가 필요합니다.

8. ddl-auto=update 운영 사용

운영에서는 validate 또는 none을 사용하고
스키마 변경은 별도 관리하는 것이 안전합니다.

면접 포인트

Q1. JPA란 무엇인가요?

Java 객체와 관계형 데이터베이스 테이블을 매핑하기 위한 ORM 표준 기술입니다.

Q2. Hibernate와 Spring Data JPA의 차이는 무엇인가요?

Hibernate는 JPA 구현체이고, Spring Data JPA는 Repository 인터페이스로 JPA 사용을 쉽게 해주는 Spring 추상화 도구입니다.

Q3. Entity란 무엇인가요?

DB 테이블과 매핑되는 Java 클래스이며, JPA 영속성 컨텍스트에서 상태가 관리되는 객체입니다.

Q4. @Id와 @GeneratedValue는 무엇인가요?

@Id는 Primary Key이고, @GeneratedValue는 PK 값을 자동 생성하는 전략을 지정합니다.

Q5. JpaRepository<Post, Long>에서 Post와 Long은 무엇인가요?

Post는 관리할 Entity 타입이고, Long은 해당 Entity의 Primary Key 타입입니다.

Q6. Dirty Checking이란 무엇인가요?

트랜잭션 안에서 Managed Entity의 변경을 감지해 커밋 시 UPDATE SQL을 생성하는 기능입니다.

Q7. DTO와 Entity를 분리하는 이유는 무엇인가요?

DB 구조 노출을 막고, API 스펙과 DB 매핑 규칙을 분리하기 위해서입니다.

Q8. JPA도 DataSource와 HikariCP를 사용하나요?

네. JPA는 내부적으로 Hibernate, JDBC, DataSource, HikariCP 위에서 동작합니다.

학습 메모

오늘의 암기 포인트

  • JPA는 Java 객체와 DB 테이블을 매핑하는 ORM 표준 기술입니다.
  • Hibernate는 JPA의 대표 구현체입니다.
  • Spring Data JPA는 JpaRepository로 CRUD를 쉽게 사용하게 해줍니다.
  • @Entity는 DB 테이블과 매핑되는 Java 클래스입니다.
  • @Id는 Primary Key이고, @GeneratedValue는 PK 생성 전략입니다.
  • Oracle에서는 보통 @SequenceGenerator와 SEQUENCE 전략을 사용합니다.
  • JpaRepository<Entity, PK타입>을 상속하면 기본 CRUD 메서드가 제공됩니다.
  • 수정은 findById 후 Entity 값을 변경하고 Dirty Checking으로 처리하는 방식이 기본입니다.
  • React와 Controller 사이에는 DTO를, Repository와 DB 사이에는 Entity를 사용합니다.

헷갈리기 쉬운 개념

JPA vs Hibernate

JPA는 표준이고,
Hibernate는 구현체입니다.

Spring Data JPA vs JPA

Spring Data JPA는
Repository를 통해
JPA 사용을 쉽게 해주는 도구입니다.

Entity vs DTO

Entity는 DB 매핑 객체이고,
DTO는 API 요청/응답 객체입니다.

save() vs Dirty Checking

등록은 save(),
수정은 조회 후 변경 감지가
기본 흐름입니다.

IDENTITY vs SEQUENCE

Oracle에서는 보통
Sequence 전략을
명확히 사용하는 것이 좋습니다.
