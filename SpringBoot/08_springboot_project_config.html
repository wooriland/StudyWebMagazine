Spring Boot 프로젝트 구조와 설정

Spring Boot 프로젝트는 pom.xml로 필요한 기능을 가져오고, application.yml로 실행 환경을 설정하며,

01

Spring Boot 프로젝트는 역할별로 나뉜 집이다

Spring Boot 프로젝트는 단순한 폴더 모음이 아닙니다. 어떤 코드를 어디에 둘 것인지 정해둔 서버 프로젝트의 지도입니다.
Java 서버 코드는 src/main/java에 두고, 설정 파일과 정적 리소스, 템플릿 파일은 src/main/resources에 둡니다.
프로젝트의 루트에는 Maven 설정 파일인 pom.xml이 있습니다. 이 파일은 프로젝트가 사용할 Spring Boot starter, Java 버전, 빌드 플러그인, 패키징 방식을 관리합니다.
즉, pom.xml은 “무슨 기능을 사용할 것인가”를 정하고, application.yml은 “그 기능을 어떤 환경에서 실행할 것인가”를 정합니다.

재료 목록 pom.xml

starter, 의존성, Java 버전, 빌드 설정

생활 공간 src/main/java

Controller, Service, Repository, DTO, Entity, Config

환경 설명서 application.yml

서버 포트, DB, JPA, JWT, 정적 리소스 설정

방문자 파일 static

CSS, JS, 이미지, 정적 HTML을 그대로 제공

동적 화면 templates

Thymeleaf가 서버 데이터와 결합해 HTML 렌더링

완성 포장 target / build

java -jar로 실행할 수 있는 jar 또는 war 산출물

핵심 문장

Spring Boot 프로젝트 구조는 코드의 역할을 나누는 지도이고, pom.xml은 필요한 기능을 켜는 의존성 설정 파일, application.yml은 실행 환경 설명서입니다.

예를 들어 spring-boot-starter-web을 추가하면 Spring MVC, Jackson, 내장 Tomcat이 함께 준비됩니다.

application.yml로 실행 환경 설정하기

application.yml은 Spring Boot가 실행될 때 자동으로 읽는 설정 파일입니다. 서버 포트, DB 연결, JPA 설정, Thymeleaf 경로, 정적 리소스 경로, JWT secret 같은 값을 이 파일에서 관리합니다.

application.yml 기본 예시

server:
  port: 8080
  servlet:
    context-path: ""

spring:
  application:
    name: study-api

  datasource:
    url: jdbc:oracle:thin:@localhost:1521/XEPDB1
    username: JAVA
    password: ${DB_PASSWORD:JAVA}
    driver-class-name: oracle.jdbc.OracleDriver

  jpa:
    show-sql: true
    hibernate:
      ddl-auto: update
    properties:
      hibernate:
        format_sql: true

  thymeleaf:
    cache: false
    encoding: UTF-8
    prefix: classpath:/templates/
    suffix: .html

  mvc:
    static-path-pattern: /**

  web:
    resources:
      static-locations: classpath:/static/

jwt:
  secret: ${JWT_SECRET:change-this-secret-key-to-long-random-value}
  expiration-ms: 3600000
server.port

Spring Boot 서버가 실행될 포트입니다.

애플리케이션 공통 URL 앞 경로입니다.

spring.datasource

DB 연결 주소, 계정, 비밀번호, 드라이버입니다.

spring.jpa

Hibernate SQL 출력, DDL 자동 생성 정책입니다.

spring.thymeleaf

static과 templates 차이

static은 파일을 그대로 반환하는 정적 리소스 위치이고, templates는 Controller가 View 이름을 반환했을 때 Thymeleaf가 서버 데이터와 결합해 HTML을 만드는 위치입니다.

정적 리소스

CSS, JS, 이미지, 정적 HTML을 그대로 제공합니다.

static/style.css
→ http://localhost:8080/style.css

static/index.html
→ http://localhost:8080/index.html
서버 템플릿

src/main/resources/templates

Controller가 View 이름을 반환하면 Thymeleaf가 렌더링합니다.

@GetMapping("/posts")
public String posts() {
    return "posts/list";
}

templates/posts/list.html

“/” 요청 처리 흐름

1. Controller 매핑 있음

Controller가 View 이름을 반환하고
templates에서 HTML을 찾습니다.

2. Controller 매핑 없음

static/index.html을 먼저 찾고,
없으면 templates/index.html을 찾습니다.

3. 둘 다 없음

Whitelabel Error Page 또는 404가 발생할 수 있습니다.

React와 연결할 때

React를 별도 프로젝트로 운영하면 Spring Boot의 templates를 거의 사용하지 않고, Spring Boot는 /api/** JSON API 서버 역할을 담당합니다.
배포 단계에서는 React build 결과물을 static에 넣어 한 서버에서 서비스할 수도 있습니다.

2) 토큰 저장 함수

export function saveToken(token, keepLogin) {
  if (keepLogin) {
    localStorage.setItem("accessToken", token);
    sessionStorage.removeItem("accessToken");
  } else {
    sessionStorage.setItem("accessToken", token);
    localStorage.removeItem("accessToken");
  }
}

export function getAccessToken() {
  return (
    localStorage.getItem("accessToken") ||
    sessionStorage.getItem("accessToken")
  );
}

export function clearToken() {
  localStorage.removeItem("accessToken");
  sessionStorage.removeItem("accessToken");
}

3) Axios 공통 인스턴스

import axios from "axios";
import { getAccessToken, clearToken } from "../utils/tokenStorage";

const apiClient = axios.create({
  baseURL: import.meta.env.VITE_API_BASE_URL || "http://localhost:8080",
  headers: {
    "Content-Type": "application/json",
  },
  timeout: 5000,
});

apiClient.interceptors.request.use(
  (config) => {
    const token = getAccessToken();

    if (token) {
      config.headers.Authorization = `Bearer ${token}`;
    }

    return config;
  },
  (error) => Promise.reject(error)
);

apiClient.interceptors.response.use(
  (response) => response,
  (error) => {
    if (error.response?.status === 401) {
      clearToken();
      window.location.href = "/login";
    }

    return Promise.reject(error);
  }
);

export default apiClient;

2) React 폼 컴포넌트

import { useState } from "react";
import { createPost } from "../api/postApi";

function PostCreateForm() {
  const [form, setForm] = useState({
    title: "",
    content: "",
  });

  const [message, setMessage] = useState("");

  function handleChange(event) {
    const { name, value } = event.target;

    setForm((prev) => ({
      ...prev,
      [name]: value,
    }));
  }

  async function handleSubmit(event) {
    event.preventDefault();

    try {
      const result = await createPost(form);

      if (result.ok) {
        setMessage("게시글이 등록되었습니다.");
        setForm({
          title: "",
          content: "",
        });
      } else {
        setMessage(result.message);
      }
    } catch (error) {
      setMessage("게시글 등록에 실패했습니다.");
    }
  }

  return (
    <form onSubmit={handleSubmit}>
      <input
        name="title"
        value={form.title}
        onChange={handleChange}
        placeholder="제목"
      />

      <textarea
        name="content"
        value={form.content}
        onChange={handleChange}
        placeholder="내용"
      />

      <button type="submit">등록</button>

      {message && <p>{message}</p>}
    </form>
  );
}

export default PostCreateForm;

CORS 설정: React 포트와 Spring Boot 포트 연결

React 개발 서버와 Spring Boot API 서버의 포트가 다르면 브라우저는 다른 출처로 판단합니다.
특히 Content-Type: application/json 요청이나 Authorization 헤더가 포함된 요청은 Preflight OPTIONS 요청이 먼저 발생할 수 있습니다.

1) Security 없이 WebMvcConfigurer로 CORS 설정

@Configuration
public class WebConfig implements WebMvcConfigurer {

    @Override
    public void addCorsMappings(CorsRegistry registry) {
        registry.addMapping("/api/**")
                .allowedOrigins("http://localhost:5173", "http://localhost:3000")
                .allowedMethods("GET", "POST", "PUT", "PATCH", "DELETE", "OPTIONS")
                .allowedHeaders("*")
                .allowCredentials(true);
    }
}

2) Spring Security 사용 시 SecurityConfig에서 CORS 설정

@Configuration
@EnableMethodSecurity
public class SecurityConfig {

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .csrf(AbstractHttpConfigurer::disable)
            .cors(Customizer.withDefaults())
            .sessionManagement(session ->
                session.sessionCreationPolicy(SessionCreationPolicy.STATELESS)
            )
            .authorizeHttpRequests(auth -> auth
                .requestMatchers(HttpMethod.OPTIONS, "/**").permitAll()
                .requestMatchers("/api/auth/**", "/api/health").permitAll()
                .anyRequest().authenticated()
            );

        return http.build();
    }

    @Bean
    public CorsConfigurationSource corsConfigurationSource() {
        CorsConfiguration config = new CorsConfiguration();

        config.setAllowedOrigins(List.of(
                "http://localhost:5173",
                "http://localhost:3000"
        ));
        config.setAllowedMethods(List.of(
                "GET", "POST", "PUT", "PATCH", "DELETE", "OPTIONS"
        ));
        config.setAllowedHeaders(List.of("*"));
        config.setAllowCredentials(true);

        UrlBasedCorsConfigurationSource source =
                new UrlBasedCorsConfigurationSource();

        source.registerCorsConfiguration("/**", config);

        return source;
    }
}

자주 하는 실수

1. pom.xml 수정 후
Maven Update 누락

의존성 추가 후
Maven Dependencies가 내려오지 않으면
Maven Update Project를 실행해야 합니다.

2. Security 의존성만 추가

SecurityConfig 없이
Security starter만 추가하면
모든 API가 인증 요구 상태가 될 수 있습니다.

3. application.yml 들여쓰기 오류

YAML은 들여쓰기가 문법입니다.
탭 대신 스페이스를 사용하고
계층을 맞춰야 합니다.

4. context-path 반영 누락

서버에 context-path를 설정했다면
React Axios baseURL에도
같은 경로를 반영해야 합니다.

5. static과 templates 혼동

정적 파일은 static,
Thymeleaf 렌더링 파일은
templates에 둡니다.

6. static 경로 변경 후
URL 그대로 사용

static-path-pattern을 /resources/**로
바꾸면 URL도 /resources/style.css처럼
바뀝니다.

7. CORS 설정 누락

React와 Spring Boot 포트가 다르면
브라우저가 요청을 차단할 수 있습니다.

8. @CrossOrigin만 믿음

Security를 사용하면
Preflight가 Security Filter에서 막힐 수 있어
http.cors() 설정이 필요합니다.

9. JPA 의존성만 추가하고
DB 설정 누락

DataSource 설정이 없으면
Failed to configure a DataSource
오류가 발생할 수 있습니다.

10. 환경변수 설정 후 IDE 미재시작

환경변수를 새로 등록했다면
IDE나 실행 터미널을 재시작해야
값이 반영될 수 있습니다.

11. React 환경변수와
Spring 환경변수 혼동

React는 VITE_ 접두사를 사용하고,
Spring Boot는 application.yml에서
${KEY} 형식으로 읽습니다.

12. Entity와 DTO 위치 혼동

Entity는 DB 매핑,
DTO는 요청/응답 모양입니다.
API에는 DTO를 사용하는 것이 안전합니다.

면접 포인트

Q1. Spring Boot 프로젝트의 기본 구조를 설명해보세요.

루트에는 pom.xml이 있고, src/main/java에는 Java 서버 코드, src/main/resources에는 application.yml, static, templates가 들어갑니다.
src/test/java에는 테스트 코드가 들어가고, target 또는 build/libs에는 빌드 결과물이 생성됩니다.

Q2. pom.xml의 역할은 무엇인가요?

Maven 프로젝트 설정 파일로 프로젝트 정보, Java 버전, Spring Boot 버전, 의존성 목록, 빌드 플러그인, 패키징 방식을 관리합니다. Spring Boot에서는 starter를 통해 자동 설정의 기반이 됩니다.

Q3. Spring Boot Starter란 무엇인가요?

관련 라이브러리를 하나로 묶은 의존성 패키지입니다. spring-boot-starter-web은 Spring MVC, Jackson, 내장 Tomcat 등 웹 개발에 필요한 기능을 함께 가져옵니다.

Q4. application.properties와 application.yml은 무엇인가요?

Spring Boot의 실행 환경 설정 파일입니다. 서버 포트, DB 연결, JPA, Thymeleaf, 정적 리소스, JWT secret 같은 값을 관리합니다.
properties는 key=value 형식이고, yml은 들여쓰기 기반 계층 구조입니다.

Q5. static과 templates의 차이는 무엇인가요?

static은 CSS, JS, 이미지, 정적 HTML을 그대로 제공하는 위치입니다. templates는 Thymeleaf 템플릿을 두는 위치이며, Controller가 View 이름을 반환해야 렌더링됩니다.

Q6. 내장 Tomcat이란 무엇인가요?

Spring Boot 애플리케이션 내부에 포함된 WAS입니다. 외부 Tomcat 설치 없이 java -jar 명령만으로 HTTP 서버를 실행할 수 있게 해줍니다.

Q7. Jar와 War의 차이는 무엇인가요?

Jar는 내장 Tomcat을 포함해 단독 실행하는 방식이고, War는 외부 Tomcat 같은 WAS에 배포하는 방식입니다. 일반적인 Spring Boot API 서버는 Jar 방식이 단순합니다.

Q8. React와 Spring Boot를 같이 쓸 때 CORS가 필요한 이유는 무엇인가요?

React 개발 서버와 Spring Boot 서버는 보통 포트가 다릅니다. 브라우저는 포트가 다르면 다른 출처로 판단하므로, Spring Boot 서버가 해당 출처를 허용하도록 CORS 설정을 해야 합니다.

Q9. Spring Security 사용 시 CORS 설정은 어디에 두는 것이 좋나요?

Security Filter Chain에서 처리하는 것이 좋습니다. Preflight OPTIONS 요청이 Security Filter 단계에서 막히면 Controller의 @CrossOrigin까지 도달하지 못할 수 있기 때문입니다.

Q10. 환경변수로 민감 정보를 관리하는 이유는 무엇인가요?

DB 비밀번호, JWT secret, API key를 코드나 Git에 직접 올리면 보안 위험이 큽니다.
application.yml에서는 ${DB_PASSWORD}, ${JWT_SECRET}처럼 환경변수로 참조하고 실제 값은 실행 환경에서 주입하는 것이 좋습니다.

오늘의 암기 포인트

  • pom.xml은 프로젝트가 사용할 기능과 라이브러리를 선언합니다.
  • Spring Boot Starter는 관련 라이브러리를 안전한 버전 조합으로 묶어 제공합니다.
  • application.yml은 서버 포트, DB, JPA, JWT, static, Thymeleaf 설정을 관리합니다.
  • src/main/java에는 Controller, Service, Repository, DTO, Entity, Config가 들어갑니다.
  • static은 정적 파일을 그대로 제공하고, templates는 Thymeleaf 템플릿을 둡니다.
  • Spring Boot는 내장 Tomcat을 포함해 java -jar로 단독 실행할 수 있습니다.
  • React와 Spring Boot의 포트가 다르면 CORS와 Axios baseURL을 반드시 확인해야 합니다.

pom.xml vs application.yml

pom.xml은 기능을 가져오고,

static vs templates

static은 그대로 제공,
templates는 Controller와

server.port vs context-path

CORS는 서버 허용 정책,
Vite proxy는
개발 서버 요청 우회 기능입니다.

Jar vs War

Jar는 내장 서버로 단독 실행,
War는 외부 WAS 배포용입니다.
