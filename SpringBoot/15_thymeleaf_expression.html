Spring Boot - Chapter 15

Thymeleaf 기본 표현식

Thymeleaf는 Spring Boot에서 서버의 Model 데이터를 HTML 템플릿에 끼워 넣어 완성된 HTML을 브라우저에 내려주는 서버 사이드 템플릿 엔진입니다.
React가 Axios로 JSON DTO를 받아 브라우저에서 화면을 렌더링한다면, Thymeleaf는 @Controller가 준비한 데이터를 서버에서 HTML로 렌더링합니다.
이 단원에서는 ${...}, *{...}, @{...}, #{...}, ~{...}, th:text, th:field를 중심으로 SSR 화면의 기본 문법을 정리합니다.

Thymeleaf SSR @Controller Model templates ${...} *{...} @{...} #{...} ~{...} th:text th:utext th:field th:object th:each React CSR
학습 목표 1

Thymeleaf가 Model 데이터를 HTML에 넣어
서버에서 완성된 화면을 만드는 흐름을 이해합니다.

학습 목표 2

${}, *{}, @{}, #{}, ~{} 표현식과
th:text, th:field의 역할을 구분합니다.

학습 목표 3

React의 JSON API 렌더링 방식과
Thymeleaf의 SSR 방식을 비교해서 설명할 수 있습니다.

01

Thymeleaf를 공부한다는 것

지금까지 React와 Spring Boot를 연결할 때는 주로 @RestController가 JSON DTO를 반환하고, React가 Axios로 그 데이터를 받아 화면을 그리는 구조를 공부했습니다.
Thymeleaf는 방향이 다릅니다. Controller가 데이터를 Model에 담고, 서버가 HTML 템플릿에 데이터를 넣어 완성된 HTML을 브라우저에 내려줍니다.
그래서 Thymeleaf를 공부한다는 것은 단순히 HTML 속성 몇 개를 외우는 것이 아니라,
Controller → Model → templates/*.html → 완성된 HTML 응답이라는 서버 렌더링 흐름을 이해하는 것입니다.

Thymeleaf

서버 템플릿 엔진

Model 데이터를 HTML 템플릿에 삽입해
서버에서 완성된 HTML을 만듭니다.

SSR

서버 렌더링

브라우저가 JSON이 아니라
이미 완성된 HTML 화면을 받습니다.

Model

화면 데이터 주머니

Controller가 HTML에서 사용할
데이터를 담아 View로 전달합니다.

Expression

HTML 안의 데이터 연결

${}, *{}, @{}, #{}, ~{} 표현식으로
데이터, URL, 메시지, 조각을 연결합니다.

핵심 문장

Thymeleaf 표현식은 서버가 준비한 Model 데이터를 HTML에 연결하는 문법입니다.
React가 JSON DTO를 받아 브라우저에서 렌더링한다면, Thymeleaf는 Controller의 Model 데이터를 서버에서 HTML로 렌더링합니다.

04

Thymeleaf 표준 표현식 5가지

Thymeleaf 표현식은 HTML 안에서 서버 데이터를 꺼내거나, URL을 만들거나, 메시지를 가져오거나, 공통 조각을 삽입할 때 사용합니다.

${...}

변수 표현식

Model에 담긴 데이터를 꺼냅니다.

<span th:text="${post.title}">제목</span>
*{...}

선택 변수 표현식

th:object로 선택한 객체의
필드에 접근합니다.

<form th:object="${postForm}">
  <input th:field="*{title}">
</form>
@{...}

URL 표현식

링크, 이미지,
form action URL을 만듭니다.

<a th:href="@{/posts/{id}(id=${post.id})}">
  상세
</a>
#{...}

메시지 표현식

messages.properties의
문구를 가져옵니다.

<button th:text="#{button.save}">
  저장
</button>
~{...}

Fragment 표현식

header, footer 같은
공통 HTML 조각을 가져옵니다.

<div th:replace="~{common/footer :: footer}"></div>
05

자주 쓰는 th 속성

표현식은 보통 th:text, th:href, th:field 같은 Thymeleaf 속성 안에서 사용합니다.

th:text

태그 내부 텍스트를 안전하게 치환합니다.
기본 출력은 th:text를 사용합니다.

<p th:text="${comment.content}">댓글</p>

th:utext

HTML을 해석해서 출력합니다.
XSS 위험이 있어
신뢰 가능한 HTML에만 사용합니다.

<p th:utext="${post.htmlContent}">HTML</p>

th:field

form input의 id, name, value를
객체 필드와 자동 연결합니다.

<input th:field="*{title}">

th:href / th:src / th:action

URL 표현식과 함께
링크, 이미지, form action을 만듭니다.

<form th:action="@{/posts}" method="post">

th:if / th:unless

조건에 따라
HTML 요소를 출력하거나 숨깁니다.

<p th:if="${#lists.isEmpty(posts)}">
  게시글 없음
</p>

th:each

목록 데이터를 반복 출력합니다.
stat으로 index, count, first, last 등을
사용할 수 있습니다.

<tr th:each="post, stat : ${posts}">
  <td th:text="${stat.count}">1</td>
</tr>

PostForm

package com.example.study.post.dto;

import jakarta.validation.constraints.NotBlank;
import jakarta.validation.constraints.Size;

public class PostForm {

    @NotBlank(message = "제목은 필수입니다.")
    @Size(max = 100, message = "제목은 100자 이하로 입력하세요.")
    private String title;

    @NotBlank(message = "내용은 필수입니다.")
    private String content;

    private String writerName;

    public PostForm() {
    }

    public PostForm(String title, String content, String writerName) {
        this.title = title;
        this.content = content;
        this.writerName = writerName;
    }

    public String getTitle() {
        return title;
    }

    public String getContent() {
        return content;
    }

    public String getWriterName() {
        return writerName;
    }

    public void setTitle(String title) {
        this.title = title;
    }

    public void setContent(String content) {
        this.content = content;
    }

    public void setWriterName(String writerName) {
        this.writerName = writerName;
    }
}

        return "redirect:/posts";
    }
}

React에서 Axios로 같은 기능 구현하기

React는 Thymeleaf 표현식을 사용하지 않습니다. 대신 Axios로 JSON API를 호출하고, 받은 DTO를 state에 저장한 뒤 브라우저에서 렌더링합니다.

토큰 저장 함수

export function saveToken(token, keepLogin) {
  if (keepLogin) {
    localStorage.setItem("accessToken", token);
    sessionStorage.removeItem("accessToken");
  } else {
    sessionStorage.setItem("accessToken", token);
    localStorage.removeItem("accessToken");
  }
}

export function getAccessToken() {
  return (
    localStorage.getItem("accessToken") ||
    sessionStorage.getItem("accessToken")
  );
}

export function clearToken() {
  localStorage.removeItem("accessToken");
  sessionStorage.removeItem("accessToken");
}

Axios 공통 인스턴스

import axios from "axios";
import { getAccessToken, clearToken } from "../utils/tokenStorage";

const apiClient = axios.create({
  baseURL: "http://localhost:8080",
  headers: {
    "Content-Type": "application/json",
  },
  timeout: 5000,
});

apiClient.interceptors.request.use(
  (config) => {
    const token = getAccessToken();

    if (token) {
      config.headers.Authorization = `Bearer ${token}`;
    }

    return config;
  },
  (error) => Promise.reject(error)
);

apiClient.interceptors.response.use(
  (response) => response,
  (error) => {
    if (error.response?.status === 401) {
      clearToken();
      window.location.href = "/login";
    }

    return Promise.reject(error);
  }
);

export default apiClient;

function PostListPage() {

            <strong>{post.title}</strong>

          </li>
        ))}
      </ul>
    </main>
  );
}

자주 하는 실수

1. @RestController 사용

Thymeleaf 페이지를 반환할 때는
@RestController가 아니라
@Controller를 사용해야 합니다.

2. Model 누락

${pageTitle}을 쓰려면 Controller에서
model.addAttribute("pageTitle", ...)을
해야 합니다.

3. th:object 없이 *{...} 사용

*{title}은 th:object로
기준 객체가 선택되어 있어야
사용할 수 있습니다.

4. 일반 출력에 th:field 사용

일반 텍스트 출력은 th:text,
form 입력 요소는 th:field를 사용합니다.

5. URL 문자열 직접 조립

링크와 form action은
문자열 연결보다 @{...} URL 표현식을
사용하는 것이 좋습니다.

6. 메시지 key 누락

#{button.save}를 쓰려면
messages.properties에
button.save key가 있어야 합니다.

7. th:utext 남용

사용자 입력값에 th:utext를 쓰면
XSS 위험이 있으므로
기본은 th:text입니다.

8. 인라인 표현식 혼동

문장 중간에 값을 넣으려면
th:text보다 [[${...}]] 또는 span th:text가
적합합니다.

면접 포인트

Q1. Thymeleaf란 무엇인가요?

Spring Controller가 Model에 담은 데이터를 HTML 템플릿에 삽입해 서버에서 완성된 HTML을 만드는 Java 템플릿 엔진입니다.

Q2. SSR이란 무엇인가요?

Server Side Rendering의 약자로, 서버에서 HTML을 완성해 브라우저에 내려주는 방식입니다.

Q3. React와 Thymeleaf의 차이는 무엇인가요?

React는 JSON API를 받아 브라우저에서 렌더링하고, Thymeleaf는 서버 Model 데이터를 HTML에 넣어 완성된 화면을 내려줍니다.

Q4. ${...}는 무엇인가요?

Model에 담긴 데이터를 HTML에서 출력할 때 사용하는 변수 표현식입니다.

Q5. *{...}는 무엇인가요?

th:object로 선택한 객체의 필드에 접근하는 선택 변수 표현식입니다.

Q6. th:field는 어떤 역할을 하나요?

form 입력 요소의 id, name, value를 객체 필드와 자동으로 연결합니다.

Q7. th:text와 th:utext의 차이는 무엇인가요?

th:text는 안전한 텍스트 출력이고, th:utext는 HTML 해석 출력이라 XSS에 주의해야 합니다.

Q8. @Controller와 @RestController의 차이는 무엇인가요?

@Controller는 View 이름을 반환해 템플릿을 렌더링하고, @RestController는 데이터를 response body로 반환합니다.

학습 메모

오늘의 암기 포인트

  • Thymeleaf는 서버에서 Model 데이터를 HTML에 넣어 완성된 HTML을 만드는 템플릿 엔진입니다.
  • @Controller는 View 이름을 반환하고, @RestController는 JSON 데이터를 반환합니다.
  • ${...}는 Model 데이터를 꺼내는 변수 표현식입니다.
  • *{...}는 th:object 기준 객체의 필드에 접근하는 선택 변수 표현식입니다.
  • @{...}는 링크, 이미지, form action 같은 URL을 만드는 표현식입니다.
  • #{...}는 messages.properties의 메시지를 가져오는 표현식입니다.
  • ~{...}는 header, footer 같은 fragment 조각을 가져오는 표현식입니다.
  • th:text는 안전한 텍스트 출력이고, th:utext는 HTML을 해석하므로 XSS에 주의해야 합니다.
  • th:field는 form input의 id, name, value를 객체 필드와 연결합니다.

헷갈리기 쉬운 개념

${...} vs *{...}

${...}는 Model 전체에서 찾고,
*{...}는 th:object로
선택한 객체 안에서 찾습니다.

th:text vs [[...]]

th:text는 태그 내부 전체를 바꾸고,
[[...]]는 문장 중간에
값을 넣기 좋습니다.

th:text vs th:utext

th:text는 안전 출력,
th:utext는 HTML 해석 출력입니다.

@{...} vs #{...}

@{...}는 URL 생성,
#{...}는
메시지 파일 문구 출력입니다.

React form vs
Thymeleaf form

React는 state와 Axios,
Thymeleaf는 th:field와
form submit 중심입니다.
