Spring Framework는 Java 애플리케이션에서 객체 생성, 의존성 연결, 웹 요청 처리, DB 접근, 트랜잭션, 보안, 로깅 같은 구조를 관리해주는 프레임워크입니다.

AOP와 공통 관심사 분리

AOP는 트랜잭션, 로깅, 보안, 성능 측정처럼 여러 계층에서 반복되는 공통 기능을 실제 비즈니스 로직과 분리하는 기술입니다.
AOP는 OOP를 대체하는 기술이 아니라, OOP를 더 깔끔하게 보완하는 기술입니다.

Business Logic

회원가입 처리

회원 저장, 권한 저장, 가입 로그 저장처럼 실제 업무 로직만 작성합니다.

Before

트랜잭션 시작

Around

실행 시간 측정

After

성공 commit / 실패 rollback

@Transactional은 대표적인 AOP 예시

@Service
@RequiredArgsConstructor
public class SignupService {

    private final UserRepository userRepository;
    private final UserRoleRepository userRoleRepository;

    @Transactional
    public void signup(SignupRequest request) {
        User user = new User(request.username(), request.password());
        UserRole role = new UserRole(request.username(), "ROLE_USER");

        userRepository.save(user);
        userRoleRepository.save(role);
    }
}
핵심

개발자는 회원가입 로직만 작성하고, 트랜잭션 시작, commit, rollback은 Spring AOP가 메서드 앞뒤에서 처리합니다.

아래 코드는 React가 게시글 등록 API를 호출하고, Spring Boot가 요청 DTO를 받아 Entity로 저장한 뒤, 응답 DTO를 JSON으로 반환하는 기본 구조입니다.

1) 공통 응답 DTO

public record ApiResponse<T>(
    boolean ok,
    String message,
    T data
) {
    public static <T> ApiResponse<T> success(T data) {
        return new ApiResponse<>(true, "요청이 성공했습니다.", data);
    }

    public static <T> ApiResponse<T> fail(String message) {
        return new ApiResponse<>(false, message, null);
    }
}

2) 요청 DTO와 응답 DTO

public record PostResponse(
    Long id,

3) Entity

@Entity
public class Post {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private String title;

    private String content;

    protected Post() {
    }

    public Post(String title, String content) {
        this.title = title;
        this.content = content;
    }

    public Long getId() {
        return id;
    }

    public String getTitle() {
        return title;
    }

    public String getContent() {
        return content;
    }
}

4) Repository

public interface PostRepository extends JpaRepository<Post, Long> {
}

5) Service

@Service
@RequiredArgsConstructor
public class PostService {

    private final PostRepository postRepository;

    @Transactional
    public PostResponse create(PostCreateRequest request) {
        Post post = new Post(request.title(), request.content());
        Post savedPost = postRepository.save(post);

        return PostResponse.from(savedPost);
    }

    @Transactional(readOnly = true)
    public PostResponse findById(Long id) {
        Post post = postRepository.findById(id)
                .orElseThrow(() -> new IllegalArgumentException("게시글이 없습니다."));

        return PostResponse.from(post);
    }
}

6) Controller

@RestController
@RequestMapping("/api/posts")
@RequiredArgsConstructor
public class PostController {

    private final PostService postService;

    @PostMapping
    public ResponseEntity<ApiResponse<PostResponse>> create(
            @RequestBody PostCreateRequest request
    ) {
        PostResponse response = postService.create(request);

        return ResponseEntity.ok(
                ApiResponse.success(response)
        );
    }

    @GetMapping("/{id}")
    public ResponseEntity<ApiResponse<PostResponse>> findById(
            @PathVariable Long id
    ) {
        PostResponse response = postService.findById(id);

        return ResponseEntity.ok(
                ApiResponse.success(response)
        );
    }
}

1) 토큰 저장 함수

export function saveToken(token, keepLogin) {
  if (keepLogin) {
    localStorage.setItem("accessToken", token);
    sessionStorage.removeItem("accessToken");
  } else {
    sessionStorage.setItem("accessToken", token);
    localStorage.removeItem("accessToken");
  }
}

export function getAccessToken() {
  return (
    localStorage.getItem("accessToken") ||
    sessionStorage.getItem("accessToken")
  );
}

export function clearToken() {
  localStorage.removeItem("accessToken");
  sessionStorage.removeItem("accessToken");
}

2) Axios 공통 인스턴스와 Authorization 헤더

import axios from "axios";
import { getAccessToken, clearToken } from "../utils/tokenStorage";

const apiClient = axios.create({
  baseURL: "http://localhost:8080",
  headers: {
    "Content-Type": "application/json",
  },
  timeout: 5000,
});

apiClient.interceptors.request.use(
  (config) => {
    const token = getAccessToken();

    if (token) {
      config.headers.Authorization = `Bearer ${token}`;
    }

    return config;
  },
  (error) => Promise.reject(error)
);

apiClient.interceptors.response.use(
  (response) => response,
  (error) => {
    if (error.response?.status === 401) {
      clearToken();
      window.location.href = "/login";
    }

    return Promise.reject(error);
  }
);

export default apiClient;

4) 폼 입력값을 DTO로 보내기

import { useState } from "react";
import { createPost } from "../api/postApi";

function PostCreateForm() {
  const [form, setForm] = useState({
    title: "",
    content: "",
  });

  const [message, setMessage] = useState("");

  function handleChange(event) {
    const { name, value } = event.target;

    setForm((prev) => ({
      ...prev,
      [name]: value,
    }));
  }

  async function handleSubmit(event) {
    event.preventDefault();

    try {
      const result = await createPost(form);

      if (result.ok) {
        setMessage("게시글이 등록되었습니다.");
        setForm({
          title: "",
          content: "",
        });
      } else {
        setMessage(result.message);
      }
    } catch (error) {
      setMessage("게시글 등록에 실패했습니다.");
    }
  }

  return (
    <form onSubmit={handleSubmit}>
      <input
        name="title"
        value={form.title}
        onChange={handleChange}
        placeholder="제목"
      />

      <textarea
        name="content"
        value={form.content}
        onChange={handleChange}
        placeholder="내용"
      />

      <button type="submit">등록</button>

      {message && <p>{message}</p>}
    </form>
  );
}

export default PostCreateForm;

Spring Boot는 src/main/resources/application.properties 또는 application.yml을 자동으로 읽어 서버 포트, DB 접속 정보, JPA 설정, JWT secret 같은 환경 값을 구성합니다.

application.yml 예시

server:
  port: 8080

spring:
  application:
    name: spring-study-api

  datasource:
    url: jdbc:oracle:thin:@localhost:1521/XEPDB1
    username: JAVA
    password: ${DB_PASSWORD}
    driver-class-name: oracle.jdbc.OracleDriver

  jpa:
    show-sql: true
    hibernate:
      ddl-auto: update

jwt:
  secret: ${JWT_SECRET}
  expiration-ms: 3600000

@Value로 설정 읽기

@Component
public class AppInfo {

    @Value("${spring.application.name}")
    private String applicationName;

    @Value("${server.port}")
    private int port;

    public String summary() {

    }
}

오류 메시지 읽는 법

Spring Boot 오류는 대부분 Bean 등록, 의존성 주입, URL 매핑, HTTP method, JSON 변환, 보안 설정, CORS에서 발생합니다. 오류 메시지를 보면 어느 단계에서 막혔는지 먼저 파악해야 합니다.

오류 메시지
의미
확인할 것
Whitelabel Error Page
요청 URL을 처리할 Controller나
정적 리소스를 찾지 못했거나 서버 오류가 발생했습니다.
URL, Controller 매핑, 서버 로그의 404/500 여부를 확인합니다.
No qualifying bean of type ... available
주입받으려는 Bean을 Spring Container에서 찾지 못했습니다.
@Service, @Repository, @Component 누락 또는
Component Scan 범위를 확인합니다.
UnsatisfiedDependencyException
Bean 생성 중 필요한 의존성을 주입하지 못했습니다.
생성자 파라미터 타입이 Bean으로 등록되어 있는지 확인합니다.
Failed to configure a DataSource

DataSource를 만들 수 없습니다.

Port 8080 was already in use
8080 포트를 다른 프로세스가 사용 중입니다.
기존 서버를 종료하거나 server.port를 변경합니다.
404 Not Found
요청 URL에 매핑된 Controller 메서드가 없습니다.
@RequestMapping, @GetMapping, @PostMapping 경로를 확인합니다.
405 Method Not Allowed
URL은 맞지만 HTTP method가 다릅니다.
GET/POST/PUT/DELETE 요청 방식과 Controller 매핑을 비교합니다.
415 Unsupported Media Type
요청 Content-Type이 서버가 기대한 형식과 다릅니다.
React Axios의 Content-Type이 application/json인지 확인합니다.
400 Bad Request
JSON이 DTO로 변환되지 않았거나 필드 타입이 맞지 않습니다.
@RequestBody, JSON key, DTO 필드명, 숫자/문자열 타입을 확인합니다.
401 Unauthorized
인증되지 않은 요청입니다.
토큰 저장 여부, Authorization 헤더, Bearer 접두사, 토큰 만료를 확인합니다.
403 Forbidden
인증은 되었지만 권한이 부족합니다.
ROLE_USER, ROLE_ADMIN, SecurityConfig 권한 조건을 확인합니다.
CORS policy blocked
React 서버와 Spring Boot 서버의
출처가 달라 브라우저가 차단했습니다.
@CrossOrigin 또는 전역 CORS 설정을 확인합니다.

주입 오류가 발생합니다.

4. 필드 주입 남용

@Autowired 필드 주입보다
final 필드와 생성자 주입을 권장합니다.

5. @RequestBody 누락

React가 JSON을 보내는 경우
Controller 파라미터에
@RequestBody가 필요합니다.

6. DTO 필드명 불일치

React JSON key와
Spring DTO 필드명이 다르면
값이 제대로 매핑되지 않습니다.

7. Security 의존성 추가 후 API 막힘

Spring Security를 추가하면
기본 인증이 적용될 수 있으므로
SecurityConfig를 설정해야 합니다.

8. CORS 설정 누락

React와 Spring Boot 포트가 다르면
브라우저 CORS 정책에 막힐 수 있습니다.

9. filterChain.doFilter 누락

JWT Filter에서 다음 필터로 넘기지 않으면
Controller까지 요청이 도달하지 않습니다.

10. Entity를 그대로 응답

DB 구조 노출과 API 스펙 변경 위험이 있으므로
Response DTO로 변환해 반환합니다.

11. application.yml 들여쓰기 오류

YAML은 들여쓰기에 민감하므로
계층 구조를 정확히 맞춰야 합니다.

12. @Transactional 위치 오류

DB 변경 업무는 보통 Service 계층
public 메서드에 적용합니다.

실무 연결 카드

Spring Framework와 Boot의 핵심 개념은 실무 프로젝트의 모든 계층에 적용됩니다.
Controller, Service, Repository, DTO, Entity, Config, Security, AOP, Validation, JPA 모두 이 기본 구조 위에서 움직입니다.

REST API 서버

React, 모바일 앱, 외부 서비스가
호출할 JSON API를 제공합니다.

관리자 페이지

Thymeleaf나 React를 사용해
관리자 화면과 API 서버를 구성합니다.

DB 연동

JDBC, DataSource, HikariCP,
JPA Repository를 통해 DB에 접근합니다.

트랜잭션

회원가입, 주문, 결제, 삭제 같은 업무를
하나의 작업 단위로 묶습니다.

인증/인가

Spring Security와 JWT Filter로
로그인 사용자와 권한을 검증합니다.

AOP 로깅

Service 실행 시간, 요청 로그, 감사 로그를
공통 처리로 분리합니다.

설정 관리

DB 비밀번호, JWT secret, 외부 API key를
환경변수와 프로파일로 관리합니다.

테스트 가능 구조

DI를 사용하면 Repository나 외부 API를
가짜 객체로 교체해 테스트하기 쉬워집니다.

면접 포인트

Q1. Spring Framework가 무엇인가요?

Java 기반 애플리케이션에서 객체 생성, 의존성 관리, 웹 요청 처리, 트랜잭션, DB 접근, AOP 등을 제공하는 프레임워크입니다.
핵심은 POJO 객체를 Spring Container가 관리하도록 해서 결합도를 낮추는 것입니다.

Q2. Spring Boot와 Spring Framework의 차이는 무엇인가요?

Spring Framework는 IoC, DI, AOP, MVC 같은 핵심 기능을 제공하고, Spring Boot는 이를 쉽게 사용하도록 자동 설정, starter, 내장 Tomcat, stand-alone 실행 구조를 제공합니다.

Q3. POJO란 무엇인가요?

Plain Old Java Object의 약자로, 특정 프레임워크에 강하게 묶이지 않은 평범한 Java 객체입니다. Spring은 이런 POJO를 Bean으로 등록해 관리할 수 있습니다.

Q4. IoC와 DI는 같은 개념인가요?

IoC는 객체 생성과 흐름 제어권이 Spring으로 넘어가는 설계 원칙이고, DI는 IoC를 구현하는 대표적인 방법입니다. DI는 필요한 객체를 외부에서 주입받는 방식입니다.

Q5. Bean이 무엇인가요?

Spring Container가 생성하고 관리하는 객체입니다. Controller, Service, Repository, Component, Configuration, Bean 메서드 반환 객체 등이 Bean이 될 수 있습니다.

Q6. 생성자 주입을 권장하는 이유는 무엇인가요?

필수 의존성을 명확히 표현할 수 있고, final 필드를 사용할 수 있으며, 객체 생성 시점에 의존성이 완성되고 테스트 코드에서 직접 주입하기 쉽기 때문입니다.

Q7. AOP란 무엇인가요?

트랜잭션, 로깅, 보안, 성능 측정처럼 여러 계층에 반복되는 공통 관심사를 비즈니스 로직과 분리하는 기술입니다. 대표적인 예로 @Transactional이 있습니다.

Q8. @SpringBootApplication은 무엇을 포함하나요?

@SpringBootConfiguration, @EnableAutoConfiguration, @ComponentScan을 포함합니다.

Q9. @Controller와 @RestController의 차이는 무엇인가요?

@Controller는 보통 View를 반환하고, @RestController는 객체를 JSON/XML 응답 body로 직접 반환합니다. React API 서버에서는 보통 @RestController를 사용합니다.

Q10. React와 Spring Boot는 어떻게 역할을 나누나요?

React는 화면 렌더링, 상태 관리, Axios 요청을 담당하고, Spring Boot는 HTTP 요청 수신, DTO 변환, 비즈니스 로직, DB 접근, 인증/인가, JSON 응답을 담당합니다.

학습 메모

오늘의 암기 포인트

  • Spring Framework는 Java 객체를 만들고 연결하고 실행 흐름을 관리하는 프레임워크입니다.
  • Spring Boot는 자동 설정, starter, 내장 Tomcat으로 Spring 개발을 쉽게 해줍니다.
  • POJO는 특정 프레임워크에 강하게 묶이지 않은 평범한 Java 객체입니다.
  • IoC는 객체 생성과 제어권이 Spring Container로 넘어가는 것입니다.
  • DI는 필요한 객체를 직접 new 하지 않고 외부에서 주입받는 방식입니다.
  • Bean은 Spring Container가 생성하고 관리하는 객체입니다.
  • AOP는 트랜잭션, 로깅, 보안 같은 공통 관심사를 비즈니스 로직과 분리합니다.
  • React는 화면, Spring Boot는 서버 API를 담당합니다.

헷갈리기 쉬운 개념

Spring vs Spring Boot

Spring은 핵심 프레임워크이고,
Boot는 자동 설정과
실행 환경입니다.

IoC vs DI

IoC는 제어권 역전 원칙이고,
DI는 이를 구현하는 대표 방식입니다.

Object vs Bean

Object는 일반 객체이고,
Bean은 Spring Container가
관리하는 객체입니다.

@Controller vs
@RestController

Controller는 View 반환,
RestController는
JSON 응답에 주로 사용합니다.

Filter vs Interceptor vs AOP

Filter는 Servlet 앞단,
Interceptor는 Controller 전후,
AOP는 Bean 메서드 전후입니다.
