Spring Boot - Chapter 16

Thymeleaf Fragment와 Utility

Thymeleaf 조건/반복/Fragment/Utility는 서버가 준비한 Model 데이터를 HTML 화면에 맞게 제어하고,
반복되는 레이아웃을 재사용하며, 날짜·숫자·문자열·목록 데이터를 보기 좋게 가공하는 문법입니다.
React에서 조건부 렌더링, map(), 컴포넌트, JavaScript 포맷팅 함수로 처리하던 일을
Thymeleaf에서는 th:if, th:each, th:fragment, #lists, #strings, #temporals, #numbers로 처리합니다.

th:if th:unless th:switch th:case th:each stat.count stat.odd th:block th:fragment th:insert th:replace ~{...} #lists #strings #temporals #numbers #objects React map()
학습 목표 1

th:if, th:unless, th:switch, th:each로
서버 렌더링 화면의 조건/반복 구조를 만들 수 있습니다.

학습 목표 2

th:fragment, th:insert, th:replace로
header, footer, pagination 같은
공통 HTML 조각을 재사용할 수 있습니다.

학습 목표 3

#lists, #strings, #temporals, #numbers 등
Utility Objects로 화면 표시용 데이터를 가공할 수 있습니다.

01

조건/반복/Fragment/Utility를 공부한다는 것

이전 단원에서는 ${...}, *{...}, @{...}, #{...}, ~{...} 같은 기본 표현식으로 Model 데이터를 HTML에 연결하는 방법을 배웠습니다.
이번 단원은 그 데이터를 실제 화면 구조에 맞게 보여줄지 말지 결정하고, 반복 출력하고, 공통 레이아웃으로 재사용하고, 보기 좋은 형식으로 가공하는 단계입니다.
React에서 posts.length === 0 ? ... : ..., posts.map(...), <Header /> 컴포넌트를 쓰는 것처럼,
Thymeleaf에서는 서버에서 HTML을 만들 때 th:if, th:each, th:fragment를 사용합니다.

Condition

조건 출력

Model 데이터 상태에 따라
HTML 요소를 출력하거나 제거합니다.

Loop

반복 출력

List, 배열, Set, 숫자 범위를 반복해
테이블과 목록을 만듭니다.

Fragment

공통 UI 재사용

header, footer, pagination 같은
반복 HTML을 조각으로 분리합니다.

Utility

화면용 데이터 가공

날짜, 숫자, 문자열, 목록, null 값을
HTML에서 보기 좋게 처리합니다.

핵심 문장

Thymeleaf 조건/반복/Fragment/Utility는 서버가 준비한 Model 데이터를 HTML 화면에 맞게 제어하고,
반복되는 레이아웃을 재사용하며, 날짜·숫자·문자열·목록 데이터를 보기 좋게 가공하는 도구입니다.

04

조건문 카드

조건문은 Model 데이터의 상태에 따라 HTML 요소를 출력하거나 제거합니다. 게시글이 없는 경우, 로그인하지 않은 경우, 권한에 따라 다른 메뉴를 보여주는 경우에 사용합니다.

th:if

true일 때 출력

조건이 true일 때 해당 태그가
최종 HTML에 포함됩니다.

<p th:if="${loginUser != null}">
  로그인 상태입니다.
</p>
th:unless

false일 때 출력

조건이 false일 때 해당 태그가
최종 HTML에 포함됩니다.

<p th:unless="${loginUser != null}">
  로그인 후 이용할 수 있습니다.
</p>
th:switch

여러 조건 분기

권한, 상태값처럼 여러 값 중
하나를 선택해 출력할 때 사용합니다.

<div th:switch="${user.role}">
  <p th:case="'ADMIN'">관리자</p>
  <p th:case="*">기타</p>
</div>
Elvis

기본값 처리

값이 없을 때 기본 텍스트를 보여줍니다.

<span th:text="${post.title} ?: '제목 없음'">
  제목
</span>
05

반복문과 상태 변수 카드

th:each는 목록 데이터를 반복 출력합니다. 상태 변수는 순번, 홀짝 스타일, 첫 번째/마지막 요소 처리에 자주 사용됩니다.

th:each

목록 반복

List, 배열, Set 등을 반복해서
HTML 요소를 생성합니다.

<tr th:each="post : ${posts}">
  <td th:text="${post.title}">제목</td>
</tr>
stat.count

1부터 시작하는 순번

게시판 번호나 테이블 순번에 사용합니다.

<td th:text="${stat.count}">1</td>
stat.odd/even

홀짝 행 스타일

테이블 줄무늬 스타일을 적용할 때 유용합니다.

<tr th:classappend="${stat.odd} ? 'odd-row' : 'even-row'">
th:block

태그 없이 제어

실제 HTML 태그를 남기지 않고
반복이나 조건만 적용합니다.

<th:block th:each="pageNo : ${#numbers.sequence(1, totalPages)}">
  ...
</th:block>

06

Fragment 카드

Fragment는 React 컴포넌트처럼 공통 UI를 재사용하기 위한 Thymeleaf의 HTML 조각입니다. header, footer, sidebar, pagination을 Fragment로 분리하면 유지보수가 쉬워집니다.

th:fragment

조각 정의

공통으로 사용할 HTML 조각에
이름을 붙입니다.

<footer th:fragment="footer">
  <p>© 2026 Study</p>
</footer>
th:insert

현재 태그 안에 삽입

현재 태그를 유지하고
그 내부에 Fragment를 삽입합니다.

<div th:insert="~{common/footer :: footer}"></div>
th:replace

현재 태그를 대체

현재 태그 자체를
Fragment로 완전히 교체합니다.

<div th:replace="~{common/footer :: footer}"></div>
Parameter

값 전달

Fragment에 제목, 메뉴 목록,
활성 메뉴 같은 값을 넘길 수 있습니다.

<div th:replace="~{common/header :: pageHeader(${pageTitle}, ${menus}, ${activeMenu})}"></div>
07

Utility Objects 카드

Utility Objects는 HTML에서 데이터를 보기 좋게 가공하는 도구입니다. 날짜 포맷, 숫자 콤마, 긴 제목 자르기, 빈 목록 확인 같은 화면 표시용 처리에 사용합니다.

#lists

목록 처리

목록 비어 있음, 크기, 포함 여부, 첫/마지막 요소를 확인합니다.

${#lists.isEmpty(posts)}
${#lists.size(posts)}
#strings

문자열 처리

대소문자, 공백 제거, 문자열 자르기, 치환, 길이 확인에 사용합니다.

${#strings.length(post.title) > 20
  ? #strings.substring(post.title, 0, 20) + '...'
  : post.title}
#temporals

날짜/시간 처리

LocalDate, LocalDateTime을 화면용 문자열로 포맷합니다.

${#temporals.format(post.createdAt, 'yyyy-MM-dd HH:mm')}
#numbers

숫자 처리

천 단위 콤마, 최소 자릿수, 페이지 번호 범위 생성에 사용합니다.

${#numbers.formatInteger(price, 1, 'COMMA')}
${#numbers.sequence(1, totalPages)}
#objects

null 안전 처리

null이면 기본값을 표시합니다.

${#objects.nullSafe(post.writerName, '알 수 없음')}
#ids

반복 id 중복 방지

반복문에서 HTML id가 중복되지 않도록 순번을 붙입니다.

th:id="${#ids.seq('item_')}"

    <a th:if="${currentPage > 0}"
       th:href="@{${baseUrl}(page=${currentPage - 1})}">
        이전
    </a>

    <th:block th:if="${totalPages > 0}">
        <a th:each="pageNo : ${#numbers.sequence(1, totalPages)}"
           th:href="@{${baseUrl}(page=${pageNo - 1})}"
           th:text="${pageNo}"
           th:classappend="${pageNo - 1 == currentPage} ? 'active'">
            1
        </a>
    </th:block>

    <a th:if="${currentPage + 1 < totalPages}"
       th:href="@{${baseUrl}(page=${currentPage + 1})}">
        다음
    </a>
</nav>

React에서 Axios로 같은 데이터 다루기

React는 Thymeleaf처럼 서버에서 HTML을 받는 것이 아니라, JSON DTO를 받아 브라우저에서 조건부 렌더링과 map 렌더링을 수행합니다.

토큰 저장 함수

export function saveToken(token, keepLogin) {
  if (keepLogin) {
    localStorage.setItem("accessToken", token);
    sessionStorage.removeItem("accessToken");
  } else {
    sessionStorage.setItem("accessToken", token);
    localStorage.removeItem("accessToken");
  }
}

export function getAccessToken() {
  return (
    localStorage.getItem("accessToken") ||
    sessionStorage.getItem("accessToken")
  );
}

export function clearToken() {
  localStorage.removeItem("accessToken");
  sessionStorage.removeItem("accessToken");
}

Axios API 함수

import apiClient from "./apiClient";

export async function fetchPosts({ page = 0, size = 10, keyword = "" }) {
  const response = await apiClient.get("/api/posts", {
    params: {
      page,
      size,
      keyword,
    },
  });

  return response.data;
}

export async function createPost(form) {
  const response = await apiClient.post("/api/posts", form);
  return response.data;
}

React 조건부 렌더링과 map

function PostListPage() {
  const [posts, setPosts] = useState([]);

  useEffect(() => {
    fetchPosts({ page: 0, size: 10 }).then((result) => {
      if (result.ok) {
        setPosts(result.data.content);
      }
    });
  }, []);

  return (
    <main>
      <h1>게시글 목록</h1>

      {posts.length === 0 ? (
        <p>게시글이 없습니다.</p>
      ) : (
        <table>
          <tbody>
            {posts.map((post, index) => (
              <tr
                key={post.id}
                className={index % 2 === 0 ? "even-row" : "odd-row"}
              >
                <td>{index + 1}</td>
                <td>{post.notice ? "[공지] " : ""}{post.title}</td>
                <td>{post.writerName ?? "알 수 없음"}</td>
                <td>{post.commentCount.toLocaleString()}</td>
                <td>{post.createdAt}</td>
              </tr>
            ))}
          </tbody>
        </table>
      )}
    </main>
  );
}

자주 하는 실수

1. th:unless를 자동 else로 오해

th:unless는
앞의 th:if와 자동 연결되지 않습니다.
같은 조건을 명시해야 합니다.

2. th:text 안에 자식 태그 작성

th:text는 태그 내부 전체를 대체하므로
자식 태그가 사라질 수 있습니다.

3. 상태 변수명 오타

stat을 쓰려면
th:each="post, stat : ${posts}"처럼
명시해야 합니다.

4. Model 데이터 누락

#lists.isEmpty(posts)를 쓰려면
Controller에서
posts를 Model에 담아야 합니다.

5. Fragment 경로 오타

~{common/header :: pageHeader}는
templates/common/header.html의
pageHeader 조각을 찾습니다.

6. insert/replace 혼동

th:insert는 현재 태그를 유지하고,
th:replace는 현재 태그를
조각으로 대체합니다.

7. Utility에 비즈니스 로직 작성

할인 계산, 권한 판단 같은 복잡한 로직은
Service/DTO에서 처리하는 것이 좋습니다.

8. th:attrappend 공백 누락

class 추가에는 th:classappend를 쓰거나
attrappend 값 앞에 공백을 넣어야 합니다.

면접 포인트

Q1. th:if와 th:unless의 차이는 무엇인가요?

th:if는 조건이 true일 때 출력하고, th:unless는 조건이 false일 때 출력합니다.

Q2. th:each란 무엇인가요?

List, 배열, Set 같은 데이터를 서버에서 반복해 HTML 요소를 생성하는 Thymeleaf 속성입니다.

Q3. th:each의 상태 변수에는 무엇이 있나요?

index, count, size, current, even, odd, first, last가 있으며 순번과 홀짝 스타일 처리에 자주 사용합니다.

Q4. Fragment란 무엇인가요?

header, footer, pagination 같은 공통 HTML 조각을 정의하고 재사용하는 Thymeleaf 기능입니다.

Q5. th:insert와 th:replace의 차이는 무엇인가요?

th:insert는 현재 태그 안에 조각을 삽입하고, th:replace는 현재 태그 자체를 조각으로 대체합니다.

Q6. #lists는 언제 사용하나요?

리스트가 비어 있는지, 크기가 얼마인지, 특정 값이 포함되어 있는지 확인할 때 사용합니다.

Q7. #temporals는 언제 사용하나요?

LocalDate, LocalDateTime 같은 Java Time 객체를 화면용 날짜 문자열로 포맷할 때 사용합니다.

Q8. React map과 Thymeleaf th:each의 차이는 무엇인가요?

React map은 브라우저에서 실행되고, Thymeleaf th:each는 서버에서 HTML을 만들 때 실행됩니다.

학습 메모

오늘의 암기 포인트

  • th:if는 조건이 true일 때 HTML 요소를 출력합니다.
  • th:unless는 조건이 false일 때 HTML 요소를 출력합니다.
  • th:switch와 th:case는 여러 조건 중 하나를 선택할 때 사용합니다.
  • th:each는 List, 배열, Set 같은 데이터를 반복 출력합니다.
  • 반복 상태 변수는 index, count, size, current, even, odd, first, last를 제공합니다.
  • th:fragment는 공통 HTML 조각을 정의합니다.
  • th:insert는 현재 태그 안에 조각을 삽입하고, th:replace는 현재 태그를 조각으로 대체합니다.
  • #lists는 목록, #strings는 문자열, #temporals는 날짜, #numbers는 숫자를 처리합니다.
  • React의 조건부 렌더링과 map은 브라우저에서, Thymeleaf의 th:if와 th:each는 서버에서 실행됩니다.

헷갈리기 쉬운 개념

th:if vs th:unless

th:if는 true 출력,
th:unless는 false 출력입니다.

index vs count

index는 0부터,
count는 1부터 시작합니다.

insert vs replace

insert는 감싸는 태그 유지,
replace는 태그 자체 교체입니다.

Fragment vs
React Component

Fragment는 서버 HTML 조각,
React Component는
브라우저 JS UI 함수입니다.

Utility vs 비즈니스 로직

Utility는 표시용 가공에만 쓰고
복잡한 계산은
Service/DTO에서 처리합니다.
