Spring Boot - Chapter 00

Spring Boot 통합 로드맵

Spring Boot는 단순히 서버를 실행하는 도구가 아니라,
Controller, Service, Repository, JPA, JDBC, Security, JWT, Thymeleaf, RestTemplate을
하나의 웹 애플리케이션 흐름으로 묶어 주는 백엔드 실행 환경입니다.
이 단원에서는 앞으로 공부할 JDBC, Generic, JPA, Thymeleaf, RestTemplate, Filter/Interceptor, JWT가
실제 프로젝트 안에서 어떤 위치에 놓이는지 큰 그림으로 정리합니다.

Spring Boot MVC DI / IoC JPA JDBC DTO Thymeleaf RestTemplate JWT React API
학습 목표 1

React 요청이 Spring Boot Controller로 들어와
DB까지 이동하는 흐름을 이해합니다.

학습 목표 2

JDBC, JPA, Repository, Service, DTO가
각각 어느 계층에서 쓰이는지 구분합니다.

학습 목표 3

JWT, Filter, Interceptor, AOP가
요청 처리 앞뒤에서 어떤 공통 역할을 하는지 정리합니다.

01

Spring Boot를 공부한다는 것

Spring Boot를 공부한다는 것은 단순히 @RestController, @Service, @Repository 같은 어노테이션을 외우는 것이 아닙니다.
실제로는 하나의 HTTP 요청이 어떤 계층을 지나 DB에 접근하고, 다시 JSON 응답으로 돌아오는지를 이해하는 과정입니다.
React 화면에서 버튼을 누르면 Axios가 Spring Boot API를 호출합니다.
Spring Boot는 요청을 Controller에서 받고, Service에서 비즈니스 로직을 처리하고, Repository와 JPA를 통해 DB에 접근합니다.
JPA 아래에서는 Hibernate가 SQL을 만들고, 더 아래에서는 JDBC가 실제 DB 연결과 SQL 실행을 담당합니다.

React

화면 렌더링, 폼 입력, API 요청, 토큰 저장

→
Spring Boot

요청 처리, 비즈니스 로직, 인증, DB 접근

→
Database

회원, 게시글, 댓글, 주문, 권한 데이터 저장

핵심 문장

Spring Boot는 프론트엔드 요청과 데이터베이스 사이에서 요청을 해석하고, 규칙을 처리하고, 데이터를 저장하거나 조회하고, 결과를 응답하는 백엔드 중심축입니다.

05

기본 Spring Boot API 예제

아래 예제는 React에서 게시글 등록 요청을 보내고, Spring Boot가 DTO로 값을 받은 뒤 Service와 Repository를 통해 DB에 저장하는 기본 흐름입니다.

1) 요청 DTO

public record PostCreateRequest(
        String title,
        String content
) {
}

2) 응답 DTO

public record PostResponseDto(
        Long id,
        String title,
        String content
) {
    public static PostResponseDto from(Post post) {
        return new PostResponseDto(
                post.getId(),
                post.getTitle(),
                post.getContent()
        );
    }
}

3) Entity

import jakarta.persistence.*;
import lombok.Getter;
import lombok.NoArgsConstructor;

@Entity
@Getter
@NoArgsConstructor
public class Post {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(nullable = false)
    private String title;

    @Lob
    private String content;

    public Post(String title, String content) {
        this.title = title;
        this.content = content;
    }

    public void update(String title, String content) {
        this.title = title;
        this.content = content;
    }
}

4) Repository

import org.springframework.data.jpa.repository.JpaRepository;
import java.util.List;

public interface PostRepository extends JpaRepository<Post, Long> {

    List<Post> findByTitleContaining(String keyword);
}

5) Service

import lombok.RequiredArgsConstructor;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;
import java.util.List;

@Service
@RequiredArgsConstructor
public class PostService {

    private final PostRepository postRepository;

    @Transactional
    public Long create(PostCreateRequest request) {
        Post post = new Post(request.title(), request.content());
        return postRepository.save(post).getId();
    }

    @Transactional(readOnly = true)
    public List<PostResponseDto> findAll() {
        return postRepository.findAll()
                .stream()
                .map(PostResponseDto::from)
                .toList();
    }
}

6) Controller

import lombok.RequiredArgsConstructor;
import org.springframework.web.bind.annotation.*;
import java.util.List;

@RestController
@RequestMapping("/api/posts")
@RequiredArgsConstructor
public class PostController {

    private final PostService postService;

    @PostMapping
    public Long create(@RequestBody PostCreateRequest request) {
        return postService.create(request);
    }

    @GetMapping
    public List<PostResponseDto> findAll() {
        return postService.findAll();
    }
}

JPA를 통해 Entity를 저장하거나 조회합니다.

DTO

4) React 폼 예제

import { useState } from "react";
import { createPost } from "../api/postApi";

function PostCreateForm() {
  const [form, setForm] = useState({
    title: "",
    content: "",
  });

  const [message, setMessage] = useState("");

  function handleChange(event) {
    const { name, value } = event.target;

    setForm((prevForm) => ({
      ...prevForm,
      [name]: value,
    }));
  }

  async function handleSubmit(event) {
    event.preventDefault();

    try {
      const postId = await createPost(form);
      setMessage(`게시글이 등록되었습니다. ID: ${postId}`);
      setForm({ title: "", content: "" });
    } catch (error) {
      setMessage("게시글 등록에 실패했습니다.");
    }
  }

  return (
    <form onSubmit={handleSubmit}>
      <h2>게시글 등록</h2>

      <input
        type="text"
        name="title"
        value={form.title}
        onChange={handleChange}
        placeholder="제목"
      />

      <textarea
        name="content"
        value={form.content}
        onChange={handleChange}
        placeholder="내용"
      />

      <button type="submit">등록</button>

      {message && <p>{message}</p>}
    </form>
  );
}

export default PostCreateForm;

07

JWT 토큰 저장과 자동 첨부

로그인 성공 후 Spring Boot는 JWT를 응답합니다. React는 이 토큰을 저장하고, 이후 인증이 필요한 요청마다 Authorization: Bearer <token> 형태로 보냅니다.

1) 로그인 API 호출

import apiClient from "./apiClient";

export async function login(username, password) {
  const response = await apiClient.post("/api/auth/login", {
    username,
    password,
  });

  return response.data;
}

2) 토큰 저장

function saveToken(token, keepLogin) {
  if (keepLogin) {
    localStorage.setItem("accessToken", token);
    sessionStorage.removeItem("accessToken");
  } else {
    sessionStorage.setItem("accessToken", token);
    localStorage.removeItem("accessToken");
  }
}

function getAccessToken() {
  return (
    localStorage.getItem("accessToken") ||
    sessionStorage.getItem("accessToken")
  );
}

3) Axios Interceptor로 토큰 자동 첨부

apiClient.interceptors.request.use(
  (config) => {
    const token = getAccessToken();

    if (token) {
      config.headers.Authorization = `Bearer ${token}`;
    }

    return config;
  },
  (error) => Promise.reject(error)
);

4) 401 인증 만료 처리

apiClient.interceptors.response.use(
  (response) => response,
  (error) => {
    if (error.response?.status === 401) {
      localStorage.removeItem("accessToken");
      sessionStorage.removeItem("accessToken");

      window.location.href = "/login";
    }

    return Promise.reject(error);
  }
);

08

오류 메시지 읽는 법

Spring Boot와 React를 연결할 때 오류는 크게 URL 문제, 요청 데이터 문제, 인증 문제, CORS 문제, DB 문제로 나눌 수 있습니다.
오류 메시지를 읽을 때는 “어느 계층에서 막혔는가?”를 먼저 판단해야 합니다.

오류 메시지
의미
확인할 것
AxiosError: Network Error
React가 Spring Boot 서버에 연결하지 못했습니다.
서버 실행 여부, 포트, baseURL, 방화벽, CORS를 확인합니다.
404 Not Found
요청한 URL에 해당하는 Controller 매핑이 없습니다.
@RequestMapping, @GetMapping, API 경로를 확인합니다.
400 Bad Request
요청 JSON이 DTO 구조와 맞지 않거나 검증에 실패했습니다.
DTO 필드명, 필수값, Content-Type, Validation 메시지를 확인합니다.
401 Unauthorized
인증되지 않은 요청입니다.
토큰 저장 여부, Authorization 헤더, Bearer 접두사, 만료 여부를 확인합니다.
403 Forbidden
인증은 되었지만 권한이 부족합니다.
사용자 role, SecurityConfig 권한 설정, @PreAuthorize 조건을 확인합니다.
CORS policy
브라우저가 서로 다른 출처 간 요청을 차단했습니다.
Spring Security의 http.cors()와 CORS 허용 origin, method, header를 확인합니다.
Failed to configure a DataSource
DB 연결 설정이 없거나 잘못되었습니다.
spring.datasource.url, username, password, Oracle Driver 의존성을 확인합니다.
No qualifying bean
Spring Container가 필요한 Bean을 찾지 못했습니다.
@Service, @Repository, 패키지 위치, Component Scan 범위를 확인합니다.
Ambiguous handler methods mapped
하나의 URL을 처리할 Controller 메소드가 두 개 이상입니다.
/post/{id}와 /post/{name}처럼 겹치는 경로를 분리합니다.
LazyInitializationException
트랜잭션이 끝난 뒤 지연 로딩 필드에 접근했습니다.
Service 트랜잭션 안에서 DTO 변환, fetch join, Entity 직접 반환 여부를 확인합니다.
09

자주 하는 실수

1. React 요청 URL을 잘못 씀

axios.get("/api/posts")는
현재 React 개발 서버 기준으로
요청될 수 있습니다.
Spring Boot가 다른 포트라면
baseURL을 명확히 설정해야 합니다.

2. @RequestBody를 빼먹음

React가 JSON을 보내는 경우
Controller 파라미터에
@RequestBody가 있어야 DTO로 변환됩니다.

3. DTO 필드명과 JSON key가 다름

React가 postTitle을 보내는데
DTO가 title이면 값이 제대로
매핑되지 않을 수 있습니다.

4. Entity를 그대로 응답함

Entity를 그대로 반환하면 DB 구조 노출,
불필요한 필드 응답, 연관관계 무한 참조 문제가
생길 수 있습니다.

5. Service 없이 Controller에서 DB 처리

예제는 짧아 보이지만 실무에서는
권한, 검증, 트랜잭션, 알림 같은
규칙이 늘어납니다.
Controller는 얇게 유지하는 것이 좋습니다.

6. @Transactional을 빼먹음

JPA 변경 감지와 rollback 처리는
트랜잭션 안에서 안정적으로 동작합니다.
수정·삭제 로직은 Service 계층에서
트랜잭션을 관리해야 합니다.

7. Authorization 헤더에
Bearer를 빼먹음

JWT 필터는 보통 Bearer 접두사를
기준으로 토큰을 추출합니다.
토큰만 보내면 인증 실패가 날 수 있습니다.

8. CORS를
Controller에서만 해결하려 함

Spring Security를 사용하면
Controller에 도달하기 전
필터 단계에서 막힐 수 있습니다.
SecurityConfig에서 CORS를 설정해야 합니다.

실무 연결 카드

Spring Boot 로드맵은 실제 프로젝트 기능과 바로 연결됩니다. 아래처럼 각 기술은 실무 기능을 완성하는 데 필요한 한 조각입니다.

회원가입 / 로그인

React 폼, AuthController, UserService,
UserRepository, PasswordEncoder,
JWT 발급, Axios Interceptor가
함께 사용됩니다.

게시판 CRUD

Controller, Service, Repository,
Entity, DTO, JPA, Transaction이
기본 구조를 이룹니다.

마이페이지

JWT Filter가 인증된 사용자 정보를
SecurityContext에 저장하고,
Controller는 현재 로그인한 사용자 정보를

면접 포인트

Q1. Spring Boot는 Spring Framework와 무엇이 다른가요?

Spring Framework는 IoC, DI, AOP, MVC 같은 핵심 기능을 제공하는 프레임워크입니다.
Spring Boot는 Spring Framework를 기반으로 자동 설정, starter 의존성, 내장 Tomcat, 실행 가능한 jar 구조를 제공해 빠르게 웹 애플리케이션을 만들 수 있게 해줍니다.

Q2. Controller, Service, Repository의 역할은 무엇인가요?

Controller는 HTTP 요청과 응답을 담당합니다. Service는 비즈니스 로직과 트랜잭션을 담당합니다. Repository는 DB 접근을 담당합니다. 이 계층을 분리하면 책임이 명확해지고 유지보수가 쉬워집니다.

Q3. DTO와 Entity를 분리하는 이유는 무엇인가요?

Entity는 DB 테이블과 직접 매핑되는 내부 모델입니다. DTO는 외부 요청과 응답에 사용하는 데이터 전달 객체입니다.
분리하지 않으면 DB 구조가 외부에 노출되고, 불필요한 필드 응답, 연관관계 순환 참조 문제가 생길 수 있습니다.

Q4. JPA와 JDBC의 관계는 무엇인가요?

JDBC는 Java가 DB와 직접 통신하는 표준 API입니다. JPA는 객체와 테이블을 매핑해 객체 중심으로 DB 작업을 하게 해주는 ORM 기술입니다.
JPA 내부에서는 Hibernate가 SQL을 만들고 JDBC를 통해 DB와 통신합니다.

Q5. @Transactional은 왜 사용하나요?

여러 DB 작업을 하나의 작업 단위로 묶기 위해 사용합니다. 정상 완료 시 commit하고, 예외 발생 시 rollback합니다.
JPA에서는 트랜잭션 안에서 Entity 변경을 감지해 commit 시점에 update SQL을 실행합니다.

Q6. Filter와 Interceptor의 차이는 무엇인가요?

Filter는 Servlet Container 레벨에서 DispatcherServlet 전에 실행됩니다. Interceptor는 DispatcherServlet과 Controller 사이에서 실행됩니다.
JWT 인증, CORS는 Filter에 적합하고, Controller 전후의 요청 로그나 권한 체크는 Interceptor에 적합합니다.

Q7. JWT 인증 흐름을 설명해보세요.

사용자가 로그인하면 서버가 사용자 정보를 검증하고 JWT를 발급합니다. React는 토큰을 저장하고 이후 요청마다 Authorization 헤더에 Bearer 토큰을 담아 보냅니다.
서버의 JWT Filter는 토큰을 검증하고 인증 객체를 SecurityContext에 저장한 뒤 요청을 Controller로 전달합니다.

Q8. React에서 JWT를 어떻게 API 요청에 포함시키나요?

로그인 성공 후 받은 토큰을 localStorage 또는 sessionStorage에 저장합니다. Axios Interceptor에서 요청을 보내기 전에 토큰을 꺼내 Authorization: Bearer <token> 헤더에 추가합니다.

학습 메모

오늘의 암기 포인트

  • Spring Boot는 React와 DB 사이에서 요청 처리, 비즈니스 로직, 인증, 데이터 접근을 담당한다.
  • 전체 흐름은 React → Controller → Service → Repository → JPA → JDBC → DB 순서로 이해한다.
  • JWT가 붙으면 Controller 전에 Filter / Security가 먼저 토큰을 검사한다.
  • DTO는 외부 요청·응답용이고, Entity는 DB 테이블 매핑용이다.
  • JPA는 JDBC를 없애는 기술이 아니라, JDBC 위에서 객체 중심으로 DB 작업을 쉽게 만드는 기술이다.

헷갈리기 쉬운 개념

Controller vs Service

Controller는 요청과 응답 담당,
Service는 업무 규칙 담당입니다.

DTO vs Entity

DTO는 외부 통신용 데이터 그릇,
Entity는 DB 테이블과 연결된
내부 모델입니다.

JPA vs JDBC

JPA는 객체 중심 DB 접근 기술이고,
JDBC는 실제 DB 연결과
SQL 실행을 담당하는
기반 기술입니다.

Filter vs Interceptor

Filter는 DispatcherServlet 전에,
Interceptor는 Controller 전후에
실행됩니다.

401 vs 403

401은 인증 안 됨,
403은 인증은 되었지만
권한 부족입니다.
