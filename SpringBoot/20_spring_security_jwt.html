Spring Boot - Chapter 20

Spring Security + JWT 구현

Spring Security + JWT는 React 사용자가 로그인한 뒤 발급받은 JWT를 Spring Boot 서버가 검증하고, 인증된 사용자 요청으로 처리하게 만드는 실무형 인증 구조입니다.
React는 로그인 성공 후 토큰을 저장하고, Axios Interceptor로 Authorization: Bearer <JWT> 헤더를 자동 추가합니다.
Spring Boot는 SecurityFilterChain에 등록된 JwtAuthenticationFilter에서 토큰을 검증하고,
UserDetailsService로 DB 사용자를 조회한 뒤, SecurityContextHolder에 인증 정보를 저장합니다.

01

Spring Security + JWT를 공부한다는 것

JWT 개념만 알고 있으면 토큰이 무엇인지는 이해할 수 있습니다. 하지만 실무에서는 토큰을 발급하는 것만으로 끝나지 않습니다.
로그인 성공 시 토큰을 발급하고, React가 그 토큰을 저장하고, 이후 요청마다 토큰을 보내며, Spring Security가 Controller 전에 토큰을 검증하는 구조까지 연결해야 실제 인증 시스템이 완성됩니다.

SecurityFilterChain

보안 규칙 설정표

어떤 URL을 열어둘지,
어떤 URL은 인증이 필요한지,
JWT Filter를 어느 위치에 둘지 설정합니다.

JwtAuthenticationFilter

JWT 검사 게이트

요청마다 Authorization 헤더를 확인하고,
JWT를 검증한 뒤 인증 정보를
SecurityContextHolder에 저장합니다.

UserDetailsService

사용자 명부 조회

토큰에서 꺼낸 username으로
DB 사용자를 조회하고,
Spring Security가 이해하는
UserDetails로 변환합니다.

Axios Interceptor

React 요청 자동 보안 헤더

React에서 API 요청을 보낼 때마다
저장된 JWT를 꺼내
Authorization 헤더에 자동으로 붙입니다.

핵심 문장

Spring Security + JWT는 React 로그인 → JWT 발급 → 토큰 저장 → Authorization 헤더 전송 → JwtAuthenticationFilter 검증 → SecurityContextHolder 저장 → Controller 실행
흐름을 완성하는 인증 구조입니다.

그림처럼 이해하기

Spring Security + JWT는 건물 출입 시스템처럼 이해하면 쉽습니다. 로그인 API는 출입증 발급 창구이고, JWT는 출입증이며, JwtAuthenticationFilter는 입구 보안 게이트입니다.

로그인 API

출입증 발급 창구

사용자가 아이디와 비밀번호를 제시하면,
서버가 DB와 비밀번호를 확인한 후
JWT 출입증을 발급합니다.

JWT

출입증

사용자는 이후 API를 호출할 때마다
JWT 출입증을 제시합니다.
서버는 이 출입증의 서명과
만료 시간을 검사합니다.

JwtAuthenticationFilter

입구 보안 게이트

Controller에 들어가기 전에
출입증이 진짜인지 확인하고,
현재 사용자를
SecurityContextHolder에 기록합니다.

UserDetailsService

직원 명부 조회

토큰이 유효하더라도
DB에서 현재 사용자가 탈퇴했는지,
정지되었는지, 권한이 바뀌었는지
다시 확인할 수 있습니다.

인증과 인가 구분

인증 Authentication

“너 누구야?”를 확인합니다.
예: 로그인 성공, JWT 검증 성공

인가 Authorization

“너 이 기능 쓸 수 있어?”를 확인합니다.
예: ROLE_ADMIN만 관리자 API 접근

6) CustomUserDetailsService

package com.example.study.auth.service;

import com.example.study.auth.domain.AppUser;
import com.example.study.auth.repository.AppUserRepository;
import lombok.RequiredArgsConstructor;
import org.springframework.security.core.authority.SimpleGrantedAuthority;
import org.springframework.security.core.userdetails.*;
import org.springframework.stereotype.Service;

import java.util.Arrays;

@Service
@RequiredArgsConstructor
public class CustomUserDetailsService implements UserDetailsService {

    private final AppUserRepository appUserRepository;

    @Override
    public UserDetails loadUserByUsername(String username)
            throws UsernameNotFoundException {

        AppUser user = appUserRepository.findByUsername(username)
                .orElseThrow(() ->
                        new UsernameNotFoundException("User not found: " + username)
                );

        var authorities = Arrays.stream(user.getRoles().split(","))
                .map(String::trim)
                .map(SimpleGrantedAuthority::new)
                .toList();

        return new org.springframework.security.core.userdetails.User(
                user.getUsername(),
                user.getPassword(),
                authorities
        );
    }
}

7) JwtUtil

package com.example.study.auth.jwt;

import io.jsonwebtoken.*;
import io.jsonwebtoken.security.Keys;
import jakarta.servlet.http.HttpServletRequest;
import org.springframework.beans.factory.annotation.Value;
import org.springframework.stereotype.Component;

import javax.crypto.SecretKey;
import java.nio.charset.StandardCharsets;
import java.util.Collection;
import java.util.Date;

@Component
public class JwtUtil {

    private final SecretKey key;
    private final long jwtExpirationMs;

    public JwtUtil(
            @Value("${jwt.secret}") String secret,
            @Value("${jwt.expiration-ms}") long jwtExpirationMs
    ) {
        this.key = Keys.hmacShaKeyFor(secret.getBytes(StandardCharsets.UTF_8));
        this.jwtExpirationMs = jwtExpirationMs;
    }

    public String generateToken(String username, Collection<String> roles) {
        Date now = new Date();
        Date expiration = new Date(now.getTime() + jwtExpirationMs);

        return Jwts.builder()
                .setSubject(username)
                .claim("roles", roles)
                .setIssuedAt(now)
                .setExpiration(expiration)
                .signWith(key, SignatureAlgorithm.HS256)
                .compact();
    }

    public String resolveToken(HttpServletRequest request) {
        String bearerToken = request.getHeader("Authorization");

        if (bearerToken != null && bearerToken.startsWith("Bearer ")) {
            return bearerToken.substring(7);
        }

        return null;
    }

    public boolean validateToken(String token) {
        try {
            Jwts.parserBuilder()
                    .setSigningKey(key)
                    .build()
                    .parseClaimsJws(token);

            return true;

        } catch (io.jsonwebtoken.security.SecurityException | MalformedJwtException e) {
            System.out.println("잘못된 JWT 서명입니다: " + e.getMessage());
        } catch (ExpiredJwtException e) {
            System.out.println("만료된 JWT입니다: " + e.getMessage());
        } catch (UnsupportedJwtException e) {
            System.out.println("지원하지 않는 JWT입니다: " + e.getMessage());
        } catch (IllegalArgumentException e) {
            System.out.println("JWT 문자열이 비어 있습니다: " + e.getMessage());
        }

        return false;
    }

    public String getUsernameFromToken(String token) {
        return Jwts.parserBuilder()
                .setSigningKey(key)
                .build()
                .parseClaimsJws(token)
                .getBody()
                .getSubject();
    }
}

8) JwtAuthenticationFilter

package com.example.study.auth.jwt;

import com.example.study.auth.service.CustomUserDetailsService;
import jakarta.servlet.FilterChain;
import jakarta.servlet.ServletException;
import jakarta.servlet.http.*;
import lombok.RequiredArgsConstructor;
import org.springframework.security.authentication.UsernamePasswordAuthenticationToken;
import org.springframework.security.core.context.SecurityContextHolder;
import org.springframework.security.core.userdetails.UserDetails;
import org.springframework.security.web.authentication.WebAuthenticationDetailsSource;
import org.springframework.web.filter.OncePerRequestFilter;

import java.io.IOException;

@RequiredArgsConstructor
public class JwtAuthenticationFilter extends OncePerRequestFilter {

    private final JwtUtil jwtUtil;
    private final CustomUserDetailsService userDetailsService;

    @Override
    protected void doFilterInternal(
            HttpServletRequest request,
            HttpServletResponse response,
            FilterChain filterChain
    ) throws ServletException, IOException {

        String token = jwtUtil.resolveToken(request);

        if (token != null
                && SecurityContextHolder.getContext().getAuthentication() == null) {

            if (jwtUtil.validateToken(token)) {
                String username = jwtUtil.getUsernameFromToken(token);

                UserDetails userDetails =
                        userDetailsService.loadUserByUsername(username);

                UsernamePasswordAuthenticationToken authentication =
                        new UsernamePasswordAuthenticationToken(
                                userDetails,
                                null,
                                userDetails.getAuthorities()
                        );

                authentication.setDetails(
                        new WebAuthenticationDetailsSource().buildDetails(request)
                );

                SecurityContextHolder.getContext().setAuthentication(authentication);
            }
        }

        filterChain.doFilter(request, response);
    }
}

package com.example.study.auth.config;

import com.example.study.auth.jwt.JwtAuthenticationFilter;
import com.example.study.auth.jwt.JwtUtil;
import com.example.study.auth.service.CustomUserDetailsService;
import lombok.RequiredArgsConstructor;
import org.springframework.context.annotation.*;
import org.springframework.http.HttpMethod;
import org.springframework.security.authentication.AuthenticationManager;
import org.springframework.security.config.Customizer;
import org.springframework.security.config.annotation.authentication.configuration.AuthenticationConfiguration;
import org.springframework.security.config.annotation.method.configuration.EnableMethodSecurity;
import org.springframework.security.config.annotation.web.builders.HttpSecurity;
import org.springframework.security.config.annotation.web.configurers.AbstractHttpConfigurer;
import org.springframework.security.config.http.SessionCreationPolicy;
import org.springframework.security.crypto.bcrypt.*;
import org.springframework.security.crypto.password.PasswordEncoder;
import org.springframework.security.web.SecurityFilterChain;
import org.springframework.security.web.authentication.UsernamePasswordAuthenticationFilter;
import org.springframework.web.cors.*;

import java.util.List;

@Configuration
@EnableMethodSecurity
@RequiredArgsConstructor
public class SecurityConfig {

    private final CustomUserDetailsService userDetailsService;
    private final JwtUtil jwtUtil;

    @Bean
    public AuthenticationManager authenticationManager(
            AuthenticationConfiguration configuration
    ) throws Exception {
        return configuration.getAuthenticationManager();
    }

    @Bean
    public PasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder();
    }

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        JwtAuthenticationFilter jwtFilter =
                new JwtAuthenticationFilter(jwtUtil, userDetailsService);

        http
                .csrf(AbstractHttpConfigurer::disable)
                .cors(Customizer.withDefaults())
                .sessionManagement(session ->
                        session.sessionCreationPolicy(SessionCreationPolicy.STATELESS)
                )
                .authorizeHttpRequests(auth -> auth
                        .requestMatchers(HttpMethod.OPTIONS, "/**").permitAll()
                        .requestMatchers("/api/auth/**").permitAll()
                        .requestMatchers("/api/admin/**").hasRole("ADMIN")
                        .anyRequest().authenticated()
                )
                .addFilterBefore(
                        jwtFilter,
                        UsernamePasswordAuthenticationFilter.class
                );

        return http.build();
    }

    @Bean
    public CorsConfigurationSource corsConfigurationSource() {
        CorsConfiguration config = new CorsConfiguration();

        config.setAllowedOrigins(List.of("http://localhost:5173"));
        config.setAllowedMethods(List.of("GET", "POST", "PUT", "DELETE", "OPTIONS"));
        config.setAllowedHeaders(List.of("*"));
        config.setAllowCredentials(true);

        UrlBasedCorsConfigurationSource source =
                new UrlBasedCorsConfigurationSource();

        source.registerCorsConfiguration("/**", config);

        return source;
    }
}

10) AuthController

package com.example.study.auth.controller;

import com.example.study.auth.domain.AppUser;
import com.example.study.auth.dto.AuthRequestDto;
import com.example.study.auth.dto.AuthResponseDto;
import com.example.study.auth.jwt.JwtUtil;
import com.example.study.auth.repository.AppUserRepository;
import lombok.RequiredArgsConstructor;
import org.springframework.http.ResponseEntity;
import org.springframework.security.authentication.*;
import org.springframework.security.core.GrantedAuthority;
import org.springframework.security.crypto.password.PasswordEncoder;
import org.springframework.web.bind.annotation.*;

import java.util.List;

@RestController
@RequestMapping("/api/auth")
@RequiredArgsConstructor
public class AuthController {

    private final AuthenticationManager authenticationManager;
    private final AppUserRepository appUserRepository;
    private final PasswordEncoder passwordEncoder;
    private final JwtUtil jwtUtil;

    @PostMapping("/signup")
    public ResponseEntity<?> signup(@RequestBody AuthRequestDto request) {
        if (appUserRepository.existsByUsername(request.username())) {
            return ResponseEntity.badRequest().body("이미 존재하는 아이디입니다.");
        }

        AppUser user = AppUser.builder()
                .username(request.username())
                .password(passwordEncoder.encode(request.password()))
                .roles("ROLE_USER")
                .build();

        appUserRepository.save(user);

        return ResponseEntity.ok("회원가입 성공");
    }

    @PostMapping("/login")
    public ResponseEntity<?> login(@RequestBody AuthRequestDto request) {
        try {
            Authentication authentication = authenticationManager.authenticate(
                    new UsernamePasswordAuthenticationToken(
                            request.username(),
                            request.password()
                    )
            );

            var principal =
                    (org.springframework.security.core.userdetails.User)
                            authentication.getPrincipal();

            List<String> roles = principal.getAuthorities()
                    .stream()
                    .map(GrantedAuthority::getAuthority)
                    .toList();

            String token = jwtUtil.generateToken(principal.getUsername(), roles);

            return ResponseEntity.ok(
                    new AuthResponseDto(token, "Bearer")
            );

        } catch (BadCredentialsException e) {
            return ResponseEntity.status(401).body("아이디 또는 비밀번호가 일치하지 않습니다.");
        }
    }
}

// src/api/tokenStorage.js

const ACCESS_TOKEN_KEY = "accessToken";

export function saveAccessToken(token, keepLogin) {
  if (keepLogin) {
    localStorage.setItem(ACCESS_TOKEN_KEY, token);
    sessionStorage.removeItem(ACCESS_TOKEN_KEY);
    return;
  }

  sessionStorage.setItem(ACCESS_TOKEN_KEY, token);
  localStorage.removeItem(ACCESS_TOKEN_KEY);
}

export function getAccessToken() {
  return (
    localStorage.getItem(ACCESS_TOKEN_KEY) ||
    sessionStorage.getItem(ACCESS_TOKEN_KEY)
  );
}

export function clearAccessToken() {
  localStorage.removeItem(ACCESS_TOKEN_KEY);
  sessionStorage.removeItem(ACCESS_TOKEN_KEY);
}

import { useState } from "react";

function LoginPage() {
  const [form, setForm] = useState({

    password: "",

  });

  const handleChange = (e) => {

    } catch (error) {

      alert("아이디 또는 비밀번호를 확인하세요.");
    }
  };

  return (
    <form onSubmit={handleSubmit}>
      <input
        name="username"
        value={form.username}
        onChange={handleChange}
        placeholder="아이디"
      />

06

자주 하는 실수 카드

1. /api/auth/**를 열지 않음

로그인과 회원가입은
토큰이 없는 상태에서 호출해야 합니다.
인증 API를 permitAll로 열어두지 않으면
로그인 자체가 막힙니다.

2. filterChain.doFilter() 누락

JWT Filter에서 이 코드를 호출하지 않으면
다음 필터나 Controller로
요청이 넘어가지 않습니다.

3. JWT Filter 등록 누락

필터 클래스를 만들기만 하면
실행되지 않습니다.
SecurityConfig에서 addFilterBefore로
등록해야 합니다.

4. Filter 중복 등록

@Component 등록과
addFilterBefore 등록을 동시에 하면
필터가 두 번 실행될 수 있습니다.

5. 예전 Security 문법 사용

Spring Security 6에서는 antMatchers와
WebSecurityConfigurerAdapter가
제거되었습니다. requestMatchers와 Lambda DSL을 사용합니다.

6. 비밀번호 평문 저장

비밀번호는 PasswordEncoder로
암호화해서 저장해야 합니다.
BCryptPasswordEncoder를 많이 사용합니다.

7. ROLE_ 접두사 혼동

Spring Security 권한은 보통 ROLE_USER,
ROLE_ADMIN 형식으로 관리합니다.
hasRole과 hasAuthority
차이를 주의해야 합니다.

8. Authorization 헤더 형식 오류

올바른 형식은
Authorization: Bearer <JWT>입니다.
Bearer 뒤에는 반드시 공백이 필요합니다.

9. React 응답 필드명 불일치

서버 응답이 token인데
React에서 accessToken으로 읽으면
저장되지 않습니다.
응답 필드명을 맞춰야 합니다.

10. CORS를
@CrossOrigin만으로 해결

Security Filter에서 OPTIONS 요청이 막히면
Controller까지 도달하지 않습니다.
SecurityConfig의 cors 설정이 중요합니다.

11. OPTIONS 요청 차단

Authorization 헤더와
application/json 요청은
Preflight를 유발할 수 있습니다.
OPTIONS 요청을 허용해야 합니다.

12. Secret Key 노출

JWT Secret Key는
코드나 Git에 올리면 안 됩니다.
application.yml에서는
환경변수로 읽는 구조가 안전합니다.

오류 메시지 카드

Spring Security + JWT 오류는 대부분 토큰 헤더 누락, 토큰 만료, 권한 부족, CORS, 필터 등록 누락, Bean 등록 누락, 예전 Security 문법 사용에서 발생합니다.

오류 메시지 / 현상
의미
확인할 것
401 Unauthorized
인증 실패입니다. 토큰이 없거나 유효하지 않습니다.

Spring Security 6에서 제거된 예전 문법을 사용했습니다.
antMatchers 대신 requestMatchers를 사용합니다.

브라우저가 다른 출처 요청을 차단했습니다.
cors 설정, allowedOrigins, allowedHeaders, OPTIONS 허용 여부를 확인합니다.

09

면접 포인트 카드

Q1. Spring Security + JWT 인증 흐름을 설명해 주세요.

로그인 성공 시 서버가 JWT를 발급하고, React가 이를 저장합니다.
이후 API 요청마다 Authorization 헤더에 Bearer Token을 담아 보내면 JwtAuthenticationFilter가 토큰을 검증하고 SecurityContextHolder에 인증 정보를 저장합니다.

Q2. SecurityFilterChain은 무엇인가요?

Spring Security의 보안 규칙을 설정하는 Bean입니다. URL별 접근 권한, 세션 정책, CSRF, CORS, 커스텀 필터 위치를 설정합니다.

Q3. JwtAuthenticationFilter는 왜 필요한가요?

Controller에 도달하기 전에 JWT를 검증하기 위해 필요합니다. Controller마다 토큰 검증 코드를 반복하지 않도록 공통 인증 로직을 필터에서 처리합니다.

Q4. OncePerRequestFilter를 사용하는 이유는 무엇인가요?

JWT 검증은 요청 하나당 한 번만 실행되면 충분합니다. OncePerRequestFilter는 같은 요청에서 필터가 중복 실행되는 것을 방지하는 데 적합합니다.

Q5. addFilterBefore는 왜 사용하나요?

JWT 필터를 UsernamePasswordAuthenticationFilter보다 앞에 배치하기 위해 사용합니다. 인증 관련 필터는 순서가 중요합니다.

Q6. UserDetailsService는 무엇인가요?

DB에서 사용자 정보를 조회하고 Spring Security가 이해할 수 있는 UserDetails 객체로 변환하는 서비스입니다.

Q7. AuthenticationManager는 언제 사용하나요?

로그인 시 사용자가 입력한 username/password를 검증할 때 사용합니다. 성공하면 Authentication 객체를 반환하고 실패하면 예외가 발생합니다.

Q8. SecurityContextHolder는 무엇인가요?

현재 요청의 인증 정보를 저장하는 공간입니다. 인증 객체를 저장하면 Controller에서 @AuthenticationPrincipal로 현재 사용자를 받을 수 있습니다.

Q9. 왜 STATELESS를 설정하나요?

JWT 인증은 서버 세션을 사용하지 않고 요청마다 토큰을 검증하는 구조이기 때문입니다. 세션을 만들지 않도록 STATELESS로 설정합니다.

Q10. JWT 기반 REST API에서 CSRF를 비활성화하는 이유는 무엇인가요?

일반적인 JWT Header 인증은 세션 쿠키 기반 인증과 다르게 동작합니다.
서버 세션을 사용하지 않기 때문에 REST API에서는 보통 CSRF를 비활성화합니다.
단, Refresh Token을 Cookie로 사용하면 별도 CSRF 전략이 필요합니다.

Q11. 401과 403의 차이는 무엇인가요?

401은 인증 실패입니다. 토큰이 없거나 잘못되었거나 만료된 경우입니다. 403은 인증은 되었지만 권한이 부족한 경우입니다.

Q12. Axios Interceptor는 왜 사용하나요?

인증 API 요청마다 JWT를 Authorization 헤더에 자동으로 붙이기 위해 사용합니다. 요청마다 직접 헤더를 작성하는 중복과 실수를 줄일 수 있습니다.

학습 메모

오늘의 암기 포인트

  • Spring Security + JWT는 React 로그인부터 인증 API 호출까지 연결하는 인증 구조입니다.
  • SecurityFilterChain은 URL별 접근 규칙, 세션 정책, CORS, JWT Filter 위치를 설정합니다.
  • JwtAuthenticationFilter는 요청마다 JWT를 추출하고 검증합니다.
  • JwtUtil은 JWT 생성, 추출, 검증, username 추출을 담당합니다.
  • UserDetailsService는 DB 사용자를 조회해 UserDetails로 변환합니다.
  • AuthenticationManager는 로그인 시 username/password를 검증합니다.
  • PasswordEncoder는 비밀번호 암호화와 비교를 담당합니다.
  • SecurityContextHolder는 현재 요청의 인증 정보를 저장합니다.
  • JWT 기반 API는 보통 SessionCreationPolicy.STATELESS를 사용합니다.
  • React는 Axios Interceptor로 Authorization: Bearer JWT 헤더를 자동 추가합니다.

헷갈리기 쉬운 개념

인증 vs 인가

인증은 “너 누구야?”이고,
인가는 “너 이 기능 쓸 수 있어?”입니다.

로그인 요청 vs 인증 요청

로그인 요청은 토큰이 없으므로 permitAll이고,
인증 요청은 JWT가 필요합니다.

UserDetailsService vs UserDetails

UserDetailsService는 DB 조회 서비스이고,
UserDetails는 Security가 이해하는 사용자 정보입니다.

AuthenticationManager vs JwtFilter

AuthenticationManager는 로그인 시 비밀번호 검증,
JwtFilter는 이후 요청의 JWT 검증입니다.

401 vs 403

401은 인증 실패,
403은 인증 후 권한 부족입니다.

CORS vs JWT 오류

React 콘솔 CORS 에러는 JWT 검증 문제가 아니라
Security CORS 설정 문제일 수 있습니다.
