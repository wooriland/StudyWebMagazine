01

JDBC란 무엇인가?

JDBC는 Java Database Connectivity의 약자입니다.
Java 코드에서 Oracle, MySQL, MS-SQL 같은 데이터베이스에 연결하고, SELECT, INSERT, UPDATE, DELETE 같은 SQL을 실행할 수 있게 해주는 표준 API입니다.
Java는 Oracle DB와 직접 대화하지 않습니다.
Java는 JDBC API를 사용하고, 실제 Oracle DB와 통신하는 일은 Oracle JDBC Driver가 담당합니다.
그래서 JDBC를 공부한다는 것은 “Java가 DB에 어떻게 말을 거는지”를 배우는 것입니다.

→

통역합니다.

→
Oracle DB

실제 데이터가 저장된 공간입니다.

핵심 문장

JDBC는 Java 애플리케이션이 데이터베이스에 연결하고 SQL을 실행하기 위한 가장 기본적인 DB 접근 표준 API입니다.

02

JDBC 기본 흐름

JDBC 프로그래밍은 일정한 순서를 가집니다. 이번 단원에서는 그중에서도 드라이버 로딩 → DB 연결 → Connection 획득을 중심으로 봅니다.

1 Driver 준비

Oracle DB에 연결하려면 Oracle JDBC Driver가 필요합니다.

2 Driver 로딩

Class.forName()으로 드라이버를 JVM에 로딩합니다.

3 DriverManager

JDBC URL에 맞는 드라이버를 찾아 연결을 요청합니다.

4 Connection

Java와 DB 사이에 실제 연결 통로가 만들어집니다.

5 SQL 실행 준비

Connection으로 Statement 또는 PreparedStatement를 만듭니다.

6 결과 처리

SELECT 결과는 ResultSet으로 읽고, 작업 후 자원을 닫습니다.

이번 단원의 집중 포인트

Class.forName("oracle.jdbc.OracleDriver")와 DriverManager.getConnection(url, username, password)가 어떤 의미인지 이해하는 것이 핵심입니다.

03

DriverManager와 Connection 이해하기

JDBC 기본 흐름에서 가장 중요한 두 객체는 DriverManager와 Connection입니다.

DriverManager

등록된 JDBC Driver 중에서
JDBC URL에 맞는 드라이버를 찾고,
해당 드라이버에게 DB 연결 생성을 요청합니다. 

Connection con =
    DriverManager.getConnection(url, user, password);

Connection

Java 애플리케이션과 DB 사이에 열린 실제 연결 통로입니다.
SQL 실행 객체 생성, 트랜잭션 제어,
연결 종료의 기준이 됩니다. 

PreparedStatement ps =
    con.prepareStatement(sql);

JDBC URL

어떤 DB 서버의 어떤 서비스에 접속할지 알려주는 주소입니다.
Oracle에서는 SID 방식과 Service Name 방식이
자주 등장합니다. 

jdbc:oracle:thin:@localhost:1521/XEPDB1

05

순수 Java JDBC 연결 예제

가장 먼저 확인할 예제는 DB 연결 자체가 되는지 확인하는 코드입니다. 이 예제는 JDBC Driver 로딩, DriverManager, Connection의 역할을 보여줍니다.

1) 기본 연결 테스트

import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.SQLException;

public class JdbcConnectionTest {

    public static void main(String[] args) {
        String driver = "oracle.jdbc.OracleDriver";
        String url = "jdbc:oracle:thin:@localhost:1521/XEPDB1";
        String username = "JAVA";
        String password = "JAVA";

        Connection con = null;

        try {
            Class.forName(driver);
            System.out.println("Oracle JDBC Driver 로딩 성공");

            con = DriverManager.getConnection(url, username, password);
            System.out.println("DB 연결 성공");

        } catch (ClassNotFoundException e) {
            System.out.println("JDBC 드라이버 클래스를 찾을 수 없습니다.");
            e.printStackTrace();

        } catch (SQLException e) {
            System.out.println("DB 연결에 실패했습니다.");
            e.printStackTrace();

        } finally {
            if (con != null) {
                try {
                    con.close();
                    System.out.println("DB 연결 종료");
                } catch (SQLException e) {
                    e.printStackTrace();
                }
            }
        }
    }
}

2) try-with-resources 개선 예제

import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.SQLException;

public class JdbcConnectionTryWithResources {

    public static void main(String[] args) {
        String driver = "oracle.jdbc.OracleDriver";
        String url = "jdbc:oracle:thin:@localhost:1521/XEPDB1";
        String username = "JAVA";
        String password = "JAVA";

        try {
            Class.forName(driver);

            try (Connection con = DriverManager.getConnection(url, username, password)) {
                System.out.println("DB 연결 성공");
                System.out.println("AutoCommit 상태: " + con.getAutoCommit());
            }

            System.out.println("try 블록 종료 후 Connection 자동 close");

        } catch (ClassNotFoundException e) {
            System.out.println("드라이버 로딩 실패");
            e.printStackTrace();

        } catch (SQLException e) {
            System.out.println("DB 연결 실패");
            e.printStackTrace();
        }
    }
}
Class.forName()

JDBC Driver를 JVM에 로딩합니다.

DriverManager

URL에 맞는 드라이버를 찾아 연결을 만듭니다.

Connection

Java와 DB 사이의 실제 연결 통로입니다.

close()

사용한 DB 연결 자원을 반납합니다.

Spring Boot에서 DB 연결 확인하기

Spring Boot에서는 보통 DriverManager를 직접 사용하지 않습니다. application.yml의 설정을 바탕으로 DataSource가 자동 생성되고, HikariCP가 Connection Pool을 관리합니다.

1) Maven 의존성

<dependency>
    <groupId>com.oracle.database.jdbc</groupId>
    <artifactId>ojdbc11</artifactId>
    <scope>runtime</scope>
</dependency>

<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-jdbc</artifactId>
</dependency>

2) application.yml 설정

server:
  port: 8080

spring:
  datasource:
    driver-class-name: oracle.jdbc.OracleDriver
    url: jdbc:oracle:thin:@localhost:1521/XEPDB1
    username: JAVA
    password: JAVA

3) DB 연결 확인 Service

import lombok.RequiredArgsConstructor;
import org.springframework.stereotype.Service;

import javax.sql.DataSource;
import java.sql.Connection;

@Service
@RequiredArgsConstructor
public class DbConnectionCheckService {

    private final DataSource dataSource;

    public String checkConnection() {
        try (Connection con = dataSource.getConnection()) {
            return "DB 연결 성공: " + con.getMetaData().getURL();
        } catch (Exception e) {
            return "DB 연결 실패: " + e.getMessage();
        }
    }
}

4) DB 연결 확인 Controller

import lombok.RequiredArgsConstructor;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
@RequiredArgsConstructor
public class DbCheckController {

    private final DbConnectionCheckService dbConnectionCheckService;

    @GetMapping("/api/db/ping")
    public String ping() {
        return dbConnectionCheckService.checkConnection();
    }
}

3) React 컴포넌트

import { useEffect, useState } from "react";
import { checkDbConnection } from "../api/dbApi";

function DbStatusCard() {
  const [message, setMessage] = useState("DB 연결 확인 중...");
  const [error, setError] = useState("");

  useEffect(() => {
    async function loadDbStatus() {
      try {

      } catch (err) {
        setError("DB 연결 상태를 확인하지 못했습니다.");
      }
    }

    loadDbStatus();
  }, []);

  return (
    <section>
      <h2>DB 연결 상태</h2>
      {error ? <p>{error}</p> : <p>{message}</p>}
    </section>
  );
}

export default DbStatusCard;

토큰 저장과 DTO 전송 예제

실제 프로젝트에서는 사용자 조회 API나 DB 확인 API가 인증이 필요한 API일 수 있습니다.
이때 React는 로그인 후 받은 JWT를 저장하고, 이후 API 요청마다 Authorization 헤더에 토큰을 담아 보냅니다.

1) 토큰 저장 함수

export function saveToken(token, keepLogin) {
  if (keepLogin) {
    localStorage.setItem("accessToken", token);
    sessionStorage.removeItem("accessToken");
  } else {
    sessionStorage.setItem("accessToken", token);
    localStorage.removeItem("accessToken");
  }
}

export function getAccessToken() {
  return (
    localStorage.getItem("accessToken") ||
    sessionStorage.getItem("accessToken")
  );
}

export function clearToken() {
  localStorage.removeItem("accessToken");
  sessionStorage.removeItem("accessToken");
}

2) Axios Interceptor로 토큰 자동 첨부

apiClient.interceptors.request.use(
  (config) => {
    const token = getAccessToken();

    if (token) {
      config.headers.Authorization = `Bearer ${token}`;
    }

    return config;
  },
  (error) => Promise.reject(error)
);

3) 폼 입력값을 DTO로 보내기

// React에서 보내는 JSON
{
  "username": "user1",
  "email": "user1@example.com"
}

4) Spring Boot 요청 DTO

public record UserSearchRequest(

) {
}

5) Controller에서 DTO 받기

@PostMapping("/api/users/search")

자주 하는 실수

1. React에서 DB에 직접 연결하려 함

React는 브라우저에서 실행되므로
DB 계정과 비밀번호를 노출하면 안 됩니다.
DB 연결은 반드시
Spring Boot 서버가 담당해야 합니다.

2. Oracle Driver 의존성 누락

ojdbc11 의존성이 없으면
oracle.jdbc.OracleDriver를
찾지 못합니다.

3. JDBC URL 오타

jdbc:oracle:thin:
@localhost:1521/XEPDB1에서
@, :, / 위치가 틀리면
연결이 실패합니다.

4. SID와 Service Name 혼동

:xe는 SID 방식이고
/XEPDB1은 Service Name 방식입니다.

5. Connection close 누락

Connection을 닫지 않으면
커넥션 누수가 발생하고,
결국 HikariPool timeout 문제가
생길 수 있습니다.

6. SQL 문자열 직접 결합

사용자 입력을 SQL 문자열에 직접 붙이면
SQL Injection 위험이 있습니다.
PreparedStatement를 사용해야 합니다.

7. @RequestBody 누락

React가 JSON을 보내는 경우
Controller 파라미터에
@RequestBody가 필요합니다.

8. DTO 필드명 불일치

React JSON key와
Java DTO 필드명이 다르면
값이 제대로 매핑되지 않을 수 있습니다.

12

면접 포인트

Q1. JDBC가 무엇인가요?

JDBC는 Java Database Connectivity의 약자로, Java 애플리케이션이 데이터베이스에 연결하고 SQL을 실행하기 위한 표준 API입니다.
Java는 JDBC API를 사용하고, 실제 DB별 통신은 JDBC Driver가 담당합니다.

Q2. JDBC Driver는 왜 필요한가요?

DB마다 통신 방식이 다르기 때문에 Java의 공통 JDBC 호출을 Oracle이나 MySQL이 이해할 수 있는 요청으로 변환해주는 드라이버가 필요합니다.

Q3. Class.forName("oracle.jdbc.OracleDriver")의 역할은 무엇인가요?

Oracle JDBC Driver 클래스를 JVM에 로딩하고 DriverManager가 사용할 수 있도록 등록하는 역할을 합니다. 최신 환경에서는 자동 등록되기도 하지만 JDBC 기본 원리 이해에는 중요한 코드입니다.

Q4. DriverManager는 어떤 역할을 하나요?

DriverManager는 등록된 JDBC Driver 중 JDBC URL에 맞는 드라이버를 찾아 DB 연결을 생성하고 Connection 객체를 반환합니다.

Q5. Connection은 무엇인가요?

Connection은 Java 애플리케이션과 DB 사이의 실제 연결 통로입니다. SQL 실행 객체 생성, 트랜잭션 제어, 자원 반납의 기준이 됩니다.

Q6. 순수 JDBC와 Spring Boot DataSource의 차이는 무엇인가요?

순수 JDBC는 DriverManager.getConnection()으로 직접 연결을 만들고, Spring Boot는 DataSource와 HikariCP를 통해 Connection을 관리합니다.
실무에서는 커넥션 풀을 사용하는 DataSource 방식이 일반적입니다.

Q7. 커넥션 풀이 왜 필요한가요?

웹 애플리케이션은 동시에 많은 요청을 처리합니다. 요청마다 DB 연결을 새로 만들면 비용이 크기 때문에, 미리 연결을 만들어두고 빌려 쓰는 커넥션 풀이 필요합니다.

Q8. React가 DB에 직접 연결하면 안 되는 이유는 무엇인가요?

React는 브라우저에서 실행되기 때문에 DB 주소, 계정, 비밀번호가 사용자에게 노출될 수 있습니다. 따라서 React는 Spring Boot API를 호출하고, DB 접근은 서버가 담당해야 합니다.

학습 메모

오늘의 암기 포인트

  • JDBC는 Java가 DB에 연결하고 SQL을 실행하기 위한 표준 API입니다.
  • Class.forName()은 JDBC Driver를 JVM에 로딩합니다.
  • DriverManager는 JDBC URL에 맞는 드라이버를 찾아 연결을 만듭니다.
  • Connection은 Java와 DB 사이의 실제 연결 통로입니다.
  • Spring Boot에서는 DataSource와 HikariCP가 Connection을 관리합니다.
  • React는 DB에 직접 연결하지 않고 Spring Boot API를 호출합니다.

헷갈리기 쉬운 개념

Driver vs DriverManager

Driver는 DB별 통역자이고,
DriverManager는

Connection vs DataSource

Connection은 실제 연결이고,
DataSource는 Connection을

SID vs Service Name

:xe는 SID 방식,
/XEPDB1은
Service Name 방식입니다.

JDBC vs JPA

JDBC는 직접 DB 연결과
SQL 실행 API이고,
JPA는 JDBC 위에서 객체 중심으로
DB를 다루는 기술입니다.

401 vs DB 오류

401은 인증 문제이고,
DB 연결 실패는 datasource,
driver, URL, 계정 문제입니다.
